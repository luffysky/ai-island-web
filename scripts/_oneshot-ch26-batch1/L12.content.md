Python 是動態型別（dynamic typing）、但 2024+ 主流會加 **type hint（型別提示）** —— 不影響執行、但**救你大專案的命**。

> 💡 **type hint 不會強制執行**。寫了 `age: int = 30` 然後塞字串進去、Python 還是會跑。它是給「IDE + type checker + 同事 + 未來的自己」看的。

  
### 為什麼要 type hint

**對比有沒有 type hint：**

```text
# === ❌ 沒 type hint ===
def process(items):
    return [item.upper() for item in items]

# 看名字看不出來：
# - items 是什麼？list of string？dict？某種物件？
# - 回傳什麼？
# - 用錯了什麼時候會發現？跑到那一行才知道

# === ✓ 有 type hint ===
def process(items: list[str]) -> list[str]:
    return [item.upper() for item in items]

# 一目瞭然：
# - 吃 list of string、回 list of string
# - IDE 自動完成神準（打 items[0]. 會跳出 str 的所有方法）
# - mypy / pyright 跑型別檢查、不用真的執行就能抓 bug
```

  
### 基本 type hint

**變數：**

```text
age: int = 30
name: str = "Alice"
is_active: bool = True
pi: float = 3.14
nothing: None = None
```

**函式：**

```text
def add(a: int, b: int) -> int:       # -> int 表示「回傳 int」
    return a + b

def greet(name: str) -> str:
    return f"Hello, {name}"

def log(msg: str) -> None:            # -> None 表示「不回傳值」
    print(msg)
```

  
**容器型別（Python 3.9+ 用內建型別、不用 import）：**

```text
def get_users() -> list[str]:                    # list 裡面裝 str
    return ["Alice", "Bob"]

def count_chars(s: str) -> dict[str, int]:       # dict、key 是 str、value 是 int
    return {c: s.count(c) for c in set(s)}

def get_point() -> tuple[float, float]:          # 固定 2 個 float
    return (3.0, 4.0)

def get_nums() -> tuple[int, ...]:               # 不定長度 tuple、全是 int
    return (1, 2, 3, 4)

def get_tags() -> set[str]:                      # set 裡面裝 str
    return {"python", "ai"}
```

> 💡 **Python 3.8 以下**要 `from typing import List, Dict` 然後寫 `List[str]`、`Dict[str, int]`（首字大寫）。3.9 之後改用內建型別、更簡潔。

  
### Optional / Union（可能是 None / 多種型別）

**Optional[X]** = 「X 或 None」、常用於「可能找不到 → 回 None」的情境。

```text
from typing import Optional

def find_user(id: int) -> Optional[User]:        # 可能回 User、可能回 None
    return db.get(id)
```

  
**Python 3.10+ 用 `|`（管道、union 運算符、更簡潔）：**

```text
def find_user(id: int) -> User | None:           # 同上、新語法
    return db.get(id)

# === 多種型別組合 ===
def parse(x: int | str) -> int:                  # x 可以是 int 或 str
    return int(x)
```

  
**老式寫法（Python 3.9 以下）：**

```text
from typing import Union
def parse(x: Union[int, str]) -> int:
    return int(x)
```

  
### Callable（函式型別）

`Callable[[輸入], 輸出]` = 「**可以呼叫的東西**」(function、lambda、有 `__call__` 的物件)。

```text
from typing import Callable

# 吃「一個吃 int 回 int 的函式」+ 一個 int、回 int
def apply(func: Callable[[int], int], n: int) -> int:
    return func(n)

def double(x: int) -> int:
    return x * 2

apply(double, 5)                       # 10

# === 多參數 ===
def reducer(
    func: Callable[[int, int], int],   # 吃兩個 int 回 int
    items: list[int]
) -> int:
    ...

# === 特殊情況 ===
Callable[[], None]                     # 不吃參數、不回值
Callable[..., int]                     # 任意參數、回 int（... 是省略號）
```

  
### Generic（泛型、寫「不限定型別」但「進去出來一致」的函式）

**generic（泛型）** = 「用一個變數代表型別」、讓函式 / class 通用、但保留型別資訊。

```text
from typing import TypeVar

T = TypeVar("T")                       # 宣告「T 是個型別變數」、實際是什麼之後決定

def first(items: list[T]) -> T:        # 吃 list of T、回 T
    return items[0]

first([1, 2, 3])                       # T 推導成 int、回 int
first(["a", "b"])                      # T 推導成 str、回 str
first([True, False])                   # T 推導成 bool、回 bool
```

  
**約束 TypeVar（限定可選的型別）：**

```text
NumberT = TypeVar("NumberT", int, float)   # 只能是 int 或 float

def avg(items: list[NumberT]) -> NumberT:
    return sum(items) / len(items)
```

  
**Python 3.12+ 簡潔語法（不用先宣告 TypeVar）：**

```text
def first[T](items: list[T]) -> T:     # 直接在 [] 內定義型別參數
    return items[0]
```

  
**Class 泛型：**

```text
from typing import Generic

T = TypeVar("T")

class Container(Generic[T]):
    def __init__(self, item: T):
        self.item = item
    
    def get(self) -> T:
        return self.item

c = Container[int](42)                 # 明確說 T 是 int
n: int = c.get()                       # IDE 知道回的是 int
```

  
### 進階 type hint（Literal / TypedDict / Protocol / Final / NewType）

**Literal（限定具體值、不只限定型別）：**

```text
from typing import Literal

def set_status(status: Literal["active", "inactive", "pending"]) -> None:
    ...

set_status("active")                   # ✓
set_status("xxx")                       # ❌ mypy 抗議：不在允許值內
```

  
**TypedDict（dict 但有固定 key + 型別）：**

```text
from typing import TypedDict

class User(TypedDict):
    name: str
    age: int
    email: str

u: User = {"name": "Alice", "age": 30, "email": "..."}
# 比 dataclass 輕量、適合「我就是要 dict」（例如 JSON 輸入 / 輸出）
```

> 💡 **Pydantic** 是更強大的版本（含執行時驗證）、L26 會教、API 主流。

  
**Protocol（duck typing 的型別、「長得像就算」）：**

```text
from typing import Protocol

class Drawable(Protocol):
    def draw(self) -> None: ...        # 只定義介面、不實作

# === 任何「有 .draw() method」的物件都符合、不用繼承 Drawable ===
def render(item: Drawable) -> None:
    item.draw()

class Circle:
    def draw(self) -> None:
        print("⭕")
# 沒繼承 Drawable、但有 draw() → 自動符合
render(Circle())                       # ✓ mypy 也通過
```

> 💡 **這就是 Python 的 「duck typing」哲學**——「**如果走起來像鴨子、叫起來像鴨子、那就是鴨子**」、不在乎名義繼承。

  
**Final（標記「不可改」）：**

```text
from typing import Final

PI: Final = 3.14
PI = 4                                 # ❌ mypy 抗議：不能改 Final 變數
```

  
**NewType（取別名、區分「邏輯不同但底層同型別」的值）：**

```text
from typing import NewType

UserId = NewType("UserId", int)        # 名字叫 UserId、底層是 int

def get_user(id: UserId) -> User: ...

uid = UserId(123)
get_user(uid)                          # ✓
get_user(123)                          # ❌ mypy 抗議：普通 int 不是 UserId
# 防止把「文章 ID」當「使用者 ID」傳的 bug
```

  
### mypy / pyright：型別檢查器

**type checker** 是會「**讀你的 type hint、檢查有沒有用錯**」的工具。Python 內建不會檢查、要靠這類工具。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**（裝 mypy + 跑檢查）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
uv add --dev mypy
uv run mypy main.py
```

  
**範例：**

```text
def add(a: int, b: int) -> int:
    return a + b

result = add(1, "2")
```

跑 mypy 會印：

```text
main.py:4: error: Argument 2 to "add" has incompatible type "str"; expected "int"
```

→ **不用真的執行**就能抓出 bug、超省 debug 時間。

  
**設定（寫在 `pyproject.toml`）：**

```text
[tool.mypy]
strict = true                          # 嚴格模式（推薦新專案）
python_version = "3.12"

# === 漸進式採用（舊專案逐步加）===
[tool.mypy]
disallow_untyped_defs = true           # 新 function 強制要寫 type hint
# 舊 function 不會報錯、慢慢補
```

  
**IDE 內建（最方便、推薦）：**

| IDE | type checker |
|---|---|
| **VS Code** | 裝 Pylance 擴充、自動 type check |
| **Cursor / Zed** | 內建 Pylance、開箱即用 |
| **PyCharm** | 內建自家 type checker |

  
**Pyright（更快、Microsoft 出、Pylance 底層用的就是這個）：**

```text
uv add --dev pyright
uv run pyright
```

  
### 實戰例子（一份完整 typed code）

```text
from typing import Optional
from dataclasses import dataclass

@dataclass
class User:
    id: int
    name: str
    email: str
    age: Optional[int] = None
    is_active: bool = True

class UserRepository:
    def __init__(self, db: "Database"):    # 字串引號 = forward reference（前向引用）
        self.db = db
    
    def find_by_id(self, id: int) -> Optional[User]:
        row = self.db.query("SELECT * FROM users WHERE id = ?", [id])
        if not row:
            return None
        return User(**row)
    
    def find_active(self) -> list[User]:
        rows = self.db.query("SELECT * FROM users WHERE is_active = true")
        return [User(**row) for row in rows]
    
    def save(self, user: User) -> User:
        if user.id:
            self.db.update(user)
        else:
            user.id = self.db.insert(user)
        return user

# 好處：
# 1. IDE 完整自動完成（打 user. 跳出 id / name / email / age / is_active）
# 2. mypy 抓所有型別錯誤（傳錯型別、回傳忘了 None check）
# 3. 自己 1 個月後看仍懂（type hint 等於說明）
```

  
    **🚀 進階技巧**
    
2026 Python 加 type hint 4 大理由：(1) **IDE 補全強** —— VS Code / Cursor 自動完成神準、打字像飛 (2) **mypy 抓 bug** —— 不用跑就知道哪錯、節省 debug 時間 (3) **自我文件** —— 1 個月後自己看 / 同事看都懂 (4) **大專案不能沒有** —— 10000 行以上不加 type hint、改一個地方破 5 個地方。**新專案 day 1 加 type hint** —— 之後感謝過去的自己。**已有的舊 code** 用漸進式加（先 public function、再 internal、最後 fully typed）。

  

  
    
      ⚠️
      新手常見錯誤
    
    Python type hint 3 大新手坑：(1) **以為加了 type hint = 強制檢查** —— 錯！Python 執行時不會檢查、要靠 mypy / pyright（IDE 內建的 Pylance 就是）才會抓。寫了 `age: int` 然後塞 `"abc"` 進去、Python 還是跑得起來。(2) **混用 Python 3.9 前後的語法** —— `List[str]`（要 `from typing import List`）跟 `list[str]`（3.9+ 內建）不一樣、搞混會 import 錯誤。**統一用 3.9+ 的內建語法**。(3) **不用 Optional 就回 None** —— 寫 `def find() -> User:` 然後實際 `return None`、mypy 會抓到。**回 None 一定要標 `Optional[User]` 或 `User | None`**。
