FastAPI 不是唯一選擇——**Django 仍是大型 Python 後端主流**。這課對比 FastAPI / Django / Flask 三大主流框架、幫你「**什麼專案選什麼**」。

  
### Django vs FastAPI vs Flask（一張表對比 3 大框架）

|  | **FastAPI** | **Django** | **Flask** |
|---|---|---|---|
| **定位** | API-first、modern | Full-stack、batteries-included | Minimal、彈性 |
| **速度** | 最快（接近 Go） | 中（夠用） | 中 |
| **學習曲線** | 中 | 陡（內容多、慢慢吸收） | 簡單 |
| **內建** | 少（你選工具） | 多（ORM / admin / auth / forms / migration） | 少 |
| **async** | 原生（核心設計） | 4.0+ 支援、不完整 | 有限 |
| **適合** | API / 微服務 / AI backend | 大型 monolith / CMS / 含 UI | 小工具 / prototype |
| **業界使用** | Uber / Microsoft / Netflix | Instagram / Pinterest / Spotify | 各種 startup |
| **學習資源** | 多但較新 | 超多（15 年累積） | 多 |

  
### Django 完整範例（5 分鐘看完整 stack）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**（建立 Django 專案）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
uv add django djangorestframework
django-admin startproject myapp
cd myapp
python manage.py startapp blog
```

**這 4 行做什麼？逐個解釋：**
- `uv add django djangorestframework` —— 裝 Django + DRF（DRF = Django REST Framework、寫 API 用）
- `django-admin startproject myapp` —— 建一個叫 `myapp` 的 Django 專案
- `cd myapp` —— 進專案資料夾
- `python manage.py startapp blog` —— 建一個叫 `blog` 的 **app**（Django 把專案分成多個 app、每個 app 是一個獨立功能模組）

  
**設定（settings.py）：**

```text
INSTALLED_APPS = [
    "django.contrib.admin",                   # Admin 後台
    "django.contrib.auth",                    # 認證系統
    "rest_framework",                         # DRF
    "blog",                                   # 自己的 app
]
```

  
**定義 Model（blog/models.py）：**

```text
from django.db import models

class Post(models.Model):
    title = models.CharField(max_length=200)
    content = models.TextField()
    author = models.ForeignKey("auth.User", on_delete=models.CASCADE)
    created_at = models.DateTimeField(auto_now_add=True)
# Django ORM 跟 SQLAlchemy 不同寫法、但概念類似
```

  
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**（建立 + 套用 migration）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
python manage.py makemigrations
python manage.py migrate
```

**這 2 行做什麼？**
- `makemigrations` —— Django 自動比對 model vs DB、生 migration script（類似 alembic autogenerate）
- `migrate` —— 套用 migration 到 DB

> 💡 Django **內建** migration（alembic 那種功能）、不用另外裝。

  
### Django 強項 #1：內建 Admin 後台（0 行 code 給管理者用）

```text
# === admin.py（一行註冊就有完整 CRUD UI）===
from django.contrib import admin
from .models import Post

admin.site.register(Post)
```

**訪問 `/admin/` 直接有：**
- 完整的 CRUD UI（增刪改查）
- 搜尋 / 過濾 / 排序
- 權限管理
- 表單驗證

> 💡 **這是 Django 最強賣點**——很多公司「**只為了內建 admin**」用 Django、寫 CMS / 內容管理超快。

  
### DRF（Django REST Framework）寫 API

```text
# === serializers.py（資料轉 JSON 規則、類似 Pydantic）===
from rest_framework import serializers
from .models import Post

class PostSerializer(serializers.ModelSerializer):
    class Meta:
        model = Post
        fields = "__all__"                    # 自動把所有欄位都當 API 輸出

# === views.py（ViewSet = 一個 class 寫完所有 CRUD endpoint）===
from rest_framework import viewsets
from .models import Post
from .serializers import PostSerializer

class PostViewSet(viewsets.ModelViewSet):
    queryset = Post.objects.all()
    serializer_class = PostSerializer

# === urls.py（自動產生所有路由）===
from rest_framework.routers import DefaultRouter
from blog.views import PostViewSet

router = DefaultRouter()
router.register("posts", PostViewSet)

urlpatterns = router.urls
```

> 💡 **這 4 個檔自動產生**：
> - `GET /posts/` 列所有
> - `POST /posts/` 新增
> - `GET /posts/{id}/` 拿單一
> - `PUT /posts/{id}/` 完整更新
> - `PATCH /posts/{id}/` 部分更新
> - `DELETE /posts/{id}/` 刪除

  
### Django 6 大強項

```text
1. Django Admin
   - 內建 CRUD 後台、0 code 給管理者用

2. Django ORM
   - 簡單直觀、教學最多

3. Forms
   - 自動驗證 + UI 渲染

4. Authentication
   - 內建 login / permissions / groups

5. 大量第三方套件
   - django-rest-framework（API）
   - django-celery（背景任務）
   - django-channels（WebSocket）
   - django-allauth（OAuth）

6. Convention over configuration
   - 大家寫法一致
   - 接案 / 換工作友善（看 code 一秒進入狀況）
```

  
### Django 不適合的場景

```text
- ❌ 純 API（FastAPI 更輕、更快、type-safe）
- ❌ Microservices（Django 太重、啟動慢）
- ❌ 需要極致效能（async 支援不完整）
- ❌ 新 startup 想快學（學習曲線陡、要學 Django way）
```

  
### 怎麼選？決策樹

```text
=== 選 FastAPI 當：===
- 主要做 API
- AI / ML backend（搭 sklearn / PyTorch / HF）
- 微服務架構
- 想 modern / async / type-safe
- 中小團隊（2026 新專案主流）

=== 選 Django 當：===
- 全端 web app（含 UI、不分前後端）
- 需要 admin 後台（CMS / 內容平台）
- 大型團隊有規範
- 想找 Python 後端工作（台灣中大企業多 Django）

=== 選 Flask 當：===
- 簡單 API / prototype
- 小工具 / 一次性 script
- 學習基礎 web 概念

=== 2026 indie 推薦組合：===
- 純 API → FastAPI + SQLAlchemy + Pydantic
- 含 UI → Next.js（前端）+ FastAPI（後端）分離
- 真的要 Django → Django + DRF（不要純 Django template）
```

  
    **☕ 用人話講**
    
Django 是 Python 後端「老大哥」——Instagram / Pinterest / Spotify 用了 15 年。**2026 現實**：(1) **新專案 FastAPI 主流**（速度 + modern + AI 生態） (2) **Django 仍主流 enterprise / CMS**（生態 + admin 殺手鐧） (3) **Flask 變 prototype 用具**。**學習建議**：FastAPI 為主（2026 主流）、Django 知道存在 + 內建 admin 概念（找工作面試會問）、Flask 不用學（除非看舊 code）。
