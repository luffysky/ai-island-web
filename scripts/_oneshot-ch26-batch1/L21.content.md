**DL（Deep Learning、深度學習）** = ML 的子集、用「**神經網路（neural network）**」當主架構。2010+ 革命性突破（圖片識別 / 語音識別 / GPT 都是這條路徑）。

> 💡 **這課全程是概念**、沒程式碼。L22 才用 PyTorch 寫第一個神經網路。先把心智圖建好再學程式。

  
### 什麼是神經網路

**靈感來自人類神經元——一個收很多輸入、加權算總和、輸出一個值。**

```text
=== 單一人工神經元（perceptron）===

    x1 ─w1→  ┐
    x2 ─w2→  ├─ sum + bias ─ activation ─→ y
    x3 ─w3→  ┘

y = activation(w1*x1 + w2*x2 + w3*x3 + b)

3 個關鍵字：
- weight（權重、w）：每個輸入有多重要（model 自己學）
- bias（偏置、b）：基準偏移（model 自己學）
- activation（激活函式）：非線性轉換、讓 NN 能學複雜 pattern
```

  
**很多神經元組合 = 神經網路、堆很多層 = 「深度」神經網路：**

```text
[ Input layer ] → [ Hidden layer 1 ] → [ Hidden layer 2 ] → [ Output layer ]

各層的職責：
- Input：餵入原始資料（圖片像素 / 文字 token / 數值特徵）
- Hidden：逐層抽出越來越抽象的「特徵」
- Output：最終答案（類別 / 數值）

「深度（deep）」= 多個 hidden layer（業界一般 > 3 層算 deep）
```

  
### Activation Function（激活函式、賦予 NN 「學非線性」的能力）

**沒 activation function 會怎樣？** 整個 NN 退化成「線性疊加」、學不會複雜 pattern（跟 Linear Regression 沒差別）。

**4 種常見 activation：**

```text
=== 1. ReLU（Rectified Linear Unit、最普及）===
f(x) = max(0, x)        # 負的變 0、正的不變
- 簡單、計算快
- Hidden layer 的「預設選擇」
- 變種：GELU（Transformer 用、更平滑）

=== 2. Sigmoid ===
f(x) = 1 / (1 + e^(-x))   # 壓到 [0, 1] 之間
- 二分類 output（機率值）

=== 3. Softmax ===
- 多分類 output、所有輸出機率加起來 = 1
- 例：分 10 類、每類給一個 0~1 機率、加總 1

=== 4. Tanh ===
- 壓到 [-1, 1] 之間
- RNN 常用、現在較少
```

  
**實戰選擇對照表：**

```text
- Hidden layer        → ReLU（或 GELU）
- Output：二分類      → Sigmoid
- Output：多分類      → Softmax
- Output：回歸（數字）→ 不加 activation（直接輸出）
```

  
### 前向 + 反向傳播（forward / backward、NN 訓練核心）

**4 步循環、跑幾千 / 幾萬次、model 就學會：**

```text
=== 1. Forward（前向傳播）===
Input → Layer 1 → Layer 2 → ... → Output
每層：y = activation(W * x + b)
（每層的 W、b 是 model 要學的）

=== 2. 算 Loss（差距、預測 vs 真實答案的距離）===
- 回歸：MSE（Mean Squared Error）
- 分類：Cross-Entropy

=== 3. Backward（反向傳播、backpropagation）===
從 Output 算 gradient（梯度、Loss 對每個 weight 的偏微分）
往回傳到每一層、知道「每個 weight 該怎麼調」

=== 4. Update（梯度下降、gradient descent）===
W_new = W_old - learning_rate * gradient
（往「Loss 變小」的方向走一小步）

→ 重複 1~4 幾千 / 幾萬次（epoch）
→ Loss 越來越小、model 越來越準
```

> 💡 **直觀比喻**：Loss 是山谷高度、gradient 告訴你「下坡方向」、learning rate 是步長、訓練 = 走到谷底。

  
### Loss Function + Optimizer

**Loss（損失函式、選錯就學不會）：**

```text
- 回歸（預測數字）：MSE / MAE（L19 教過）
- 二分類：Binary Cross-Entropy
- 多分類：Cross-Entropy
```

  
**Optimizer（最佳化器、決定「怎麼走下坡」）：**

```text
=== 1. SGD（Stochastic Gradient Descent、隨機梯度下降）===
最基本、現在很少單獨用

=== 2. SGD + Momentum ===
加「慣性」、收斂更快

=== 3. Adam（Adaptive Moment Estimation、最普及）===
自動調 learning rate
新手預設選這個（lr=0.001 通常 work）

=== 4. AdamW（Adam + Weight Decay、2020+ 主流）===
Transformer / GPT 系列必用
表現比 Adam 好一點
```

  
**進階：learning rate 排程（schedule）：**

```text
- Cosine schedule：lr 隨訓練進度像 cosine 曲線下降
- Warmup：訓練開頭幾步 lr 從 0 慢慢升、避免一開始衝太快
→ Transformer 訓練必備
```

  
### 訓練常見問題 + 對策

**1. 梯度消失 / 爆炸（gradient vanishing / exploding）：**

```text
症狀：訓練到後面 gradient 變超小（消失）或超大（爆炸）、model 學不下去
對策：
- ReLU（避免 Sigmoid 在飽和區梯度幾乎 0）
- Batch Normalization（每層輸出標準化、穩定 gradient）
- Residual Connection（殘差連接、ResNet 的核心、訊號跳層走）
- Gradient Clipping（梯度太大就截斷）
```

  
**2. 過擬合（overfitting、L20 教過、DL 特別嚴重）：**

```text
對策：
- Dropout（訓練時隨機關掉部分神經元、強迫模型不依賴單一路徑）
- L1 / L2 regularization
- Early stopping
- Data augmentation（資料增強、圖片旋轉/翻轉/裁切變新資料）
```

  
**3. GPU OOM（Out of Memory、記憶體不夠、跑大 model 常遇到）：**

```text
對策：
- 減 batch size（一次餵少一點）
- Gradient accumulation（多個 small batch 累積 gradient 再 update）
- Mixed precision（FP16、用半精度浮點、省一半記憶體）
- 用更大 GPU（如 RTX 4090 / A100 / H100）
```

  
### DL vs 傳統 ML 對照表

|  | 傳統 ML | DL（深度學習） |
|---|---|---|
| **資料量** | 1k - 1M | 10K - 10B+（越多越好） |
| **特徵工程** | 手動（人工設計） | 自動（model 自己學） |
| **硬體** | CPU 就夠 | 需要 GPU（NVIDIA 居多） |
| **解釋性** | 強（看得懂為什麼） | 弱（黑盒子） |
| **適合資料** | 表格、結構化 | 圖、聲、文字（非結構化） |
| **工具** | scikit-learn / XGBoost | PyTorch / TensorFlow |
| **訓練時間** | 分鐘 / 小時 | 小時 / 天 / 週 |

  
    **☕ 用人話講**
    
神經網路 = **很多層 layer 的數學運算**、本質就是「**一堆 `W * x + b + activation`**」串起來。訓練 = forward 預測 → 比答案算 loss → backward 算 gradient → 更新 weight → 重複幾千次。**新手不用懂數學細節**——會用 PyTorch / TensorFlow 寫就行。**5 個核心關鍵字背起來**：layer / activation / loss / optimizer / overfitting。
