**「程式跑錯怎麼辦？」**——好的錯誤處理 = production 不掛、bug 容易追、使用者不會看到奇怪畫面。

> 💡 **3 個關鍵字**：
> - **exception（例外）** = 程式執行時發生的「異常情況」
> - **try / except** = 「試著做、出錯就改做別的」
> - **raise** = 主動拋出一個例外

  
### 基本 try / except

```text
try:
    result = 10 / 0
except ZeroDivisionError:           # ZeroDivisionError = 除以零錯誤
    print("不能除以 0")
```

  
**抓所有錯（不推薦、太籠統）vs 抓特定錯（推薦、精準）：**

```text
# === ❌ 不推薦：抓 Exception 抓全部 ===
try:
    risky_operation()
except Exception as e:              # as e = 把錯誤物件存進 e、可以印出來看
    print(f"出錯：{e}")
# 為什麼不好：你不知道是什麼錯、debug 時很難追

# === ✓ 推薦：分別抓特定 Exception ===
try:
    f = open("data.txt")
except FileNotFoundError:           # 檔案不存在
    print("檔案不存在")
except PermissionError:             # 權限不足
    print("沒權限")
except Exception as e:              # 其他沒料到的（放最後當保底）
    print(f"其他錯：{e}")
```

  
**重新拋出例外（處理完繼續往外丟）：**

```text
# === raise（不帶參數）= 把剛剛抓到的錯原樣再丟出去 ===
try:
    risky()
except SomeError as e:
    log(e)                          # 我這層先 log 一下
    raise                           # 繼續往外傳、給外層處理

# === raise from（保留錯誤鏈、debug 時看得到原因）===
try:
    parse_json(data)
except json.JSONDecodeError as e:
    raise ValueError("Invalid input") from e
# 跑出來會顯示「ValueError 是因為 JSONDecodeError 造成的」、追根因更快
```

  
### finally：不管有沒有出錯、都會跑

**finally** = 「保證跑」的區塊、適合「清理資源」（關檔、關連線、釋放 lock）。

```text
try:
    f = open("data.txt")
    process(f)
except FileNotFoundError:
    print("沒檔案")
finally:
    f.close()                       # 不管前面成功還是失敗、一定會關
```

  
**但更 pythonic 的做法是用 `with`（自動 close、不用寫 finally）：**

```text
with open("data.txt") as f:
    process(f)
# with 區塊結束、Python 自動 close f、就算 process(f) 中間 raise 也會 close
```

  
**完整 try / except / else / finally 4 段：**

```text
try:
    db.connect()
    do_work()
except Exception:
    db.rollback()                   # 出錯：rollback（撤銷未提交的變更）
    raise                           # 繼續丟錯、讓外層也知道
else:
    db.commit()                     # try 沒錯才跑 else（提交變更）
finally:
    db.close()                      # 不管成功失敗都關連線
```

> **`else` 的位置**：在 `except` 之後、`finally` 之前。意思是「try 區塊跑完沒 raise」才跑 else。

  
### 常見 Exception 類型（內建層級樹）

```text
BaseException                       # 所有 Exception 的祖先（不要直接抓這個）
 ├── SystemExit                     # sys.exit() 引發
 ├── KeyboardInterrupt              # 使用者按 Ctrl+C
 └── Exception                      # 一般程式錯誤的基底（你抓的都是這以下）
      ├── ArithmeticError
      │    ├── ZeroDivisionError    # 除以零
      │    └── OverflowError        # 數值溢位
      ├── LookupError
      │    ├── IndexError           # list[100] 但只有 5 個
      │    └── KeyError             # dict["不存在的 key"]
      ├── TypeError                 # 型別不對（如 str + int）
      ├── ValueError                # 型別對但值不對（如 int("abc")）
      ├── AttributeError            # 物件沒這個屬性 / 方法
      ├── ImportError               # import 失敗
      ├── FileNotFoundError         # 檔案不存在
      ├── PermissionError           # 沒權限
      └── ...
```

  
**實用範例：**

```text
# === KeyError：dict 沒這個 key ===
try:
    val = my_dict["key"]
except KeyError:
    val = "default"                 # 給預設值
# 或更 pythonic：val = my_dict.get("key", "default")

# === ValueError：字串轉數字失敗 ===
try:
    n = int(s)
except ValueError:
    print(f"'{s}' 不是有效數字")

# === AttributeError：物件沒這個欄位 ===
try:
    user.profile.email
except AttributeError:
    print("這個 user 沒設 profile / email")
```

> ⚠️ **不要直接 `except BaseException:`** —— 它包含 `KeyboardInterrupt`（Ctrl+C）、你會擋掉使用者想中斷程式的能力。要包山包海最多到 `except Exception:`。

  
### 自訂 Exception（業務邏輯用）

業務邏輯的錯誤、用「自己定義的例外類別」比較清楚——出錯時一眼看出是「使用者餘額不足」還是「找不到使用者」。

```text
# === 最簡單：繼承 Exception、什麼都不加 ===
class UserNotFoundError(Exception):
    pass

class InsufficientBalanceError(Exception):
    pass

# === 進階：帶額外欄位（如 error code）===
class PaymentError(Exception):
    def __init__(self, message, code):
        super().__init__(message)   # 呼叫父類別 __init__ 設好 message
        self.code = code            # 自己加 code 欄位
```

  
**完整使用範例：**

```text
def withdraw(user, amount):
    """提款。"""
    if amount > user.balance:
        raise InsufficientBalanceError(
            f"User {user.id} 餘額不足：餘額 {user.balance}、想提 {amount}"
        )
    user.balance -= amount
    return amount

# 呼叫端
try:
    withdraw(user, 1000)
except InsufficientBalanceError as e:
    print(f"提款失敗：{e}")          # 看到具體錯誤訊息
except UserNotFoundError:
    print("使用者不存在")
```

  
    **🚀 進階技巧**
    
例外處理 5 大原則（2026 業界共識）：(1) **抓具體例外**——不要直接 `except Exception`、debug 時找不到原因 (2) **不要默默吞錯**——`except: pass` 是大忌、永遠至少 log 一下 (3) **失敗早暴露**（fail fast）——不要繼續跑明顯錯的資料、會雪球 (4) **用 `with`**——自動處理 close / cleanup、不用手寫 finally (5) **EAFP（Easier to Ask Forgiveness than Permission、求原諒比求允許容易）**——Python 慣用「先 try、出錯再 except」、而不是「先 if 檢查再做」。
