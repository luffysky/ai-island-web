### 2026 安裝 Python：用 uv

**它解決了什麼？** 以前裝 Python 要用 `pip`（套件安裝工具）+ `pyenv`（版本管理）+ `virtualenv`（環境隔離）三個工具、互相不相容、新手容易裝壞。uv 一個工具搞定。

```text
uv 的優勢：
- 比 pip 快 10-100 倍
- 一個工具搞定：Python 版本 / venv（虛擬環境）/ 套件安裝 / lock（依賴鎖定）
- 不用手動 activate / deactivate venv（後面會解釋什麼是 venv）
```

### 傳統做法 pip + venv（仍要懂、有些公司還在用）

**venv（virtual environment、虛擬環境）** —— 每個 Python 專案有自己**獨立的套件空間**、不互相干擾。為什麼要這樣？因為 A 專案可能用 requests 2.0、B 專案用 requests 3.0、如果裝在同個全域空間就會打架。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
# 建 venv（在當前資料夾建立一個叫 .venv 的虛擬環境）
python -m venv .venv

# 啟動 venv（每次打開終端機都要重來一次）
source .venv/bin/activate    # Mac / Linux
.venv\Scripts\activate       # Windows

# 啟動成功、終端機提示符前會多 (.venv) 字樣

# 裝套件（pip 是 Python 內建的套件安裝工具）
pip install requests
pip install -r requirements.txt   # 從 requirements.txt 批次裝

# 把當前裝的套件清單寫到 requirements.txt
pip freeze > requirements.txt

# 退出 venv（不想用了）
deactivate
```

### 第一個有意義的程式：抓台北天氣

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📄 **寫進程式檔 weather.py**（在 VS Code 編輯器、新建檔案叫 weather.py、貼下面這段）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
import requests

def get_weather(city):
    # 用 wttr.in 免費天氣服務、format=j1 = 回傳 JSON 格式
    url = f"https://wttr.in/{city}?format=j1"
    response = requests.get(url)
    data = response.json()
    
    current = data["current_condition"][0]
    return {
        "city": city,
        "temp_c": current["temp_C"],
        "feels_like": current["FeelsLikeC"],
        "description": current["weatherDesc"][0]["value"],
        "humidity": current["humidity"],
    }

if __name__ == "__main__":
    weather = get_weather("Taipei")
    print(f"🌤  {weather['city']}: {weather['temp_c']}°C")
    print(f"   感覺像 {weather['feels_like']}°C")
    print(f"   {weather['description']}")
    print(f"   濕度 {weather['humidity']}%")
```

**這段在做什麼？**
- 定義一個 `get_weather` 函式、傳入城市名、回傳一個字典（dict）含天氣資訊。
- 用 `requests.get(url)` 抓 wttr.in（免費天氣服務）的資料。
- 用 f-string 把結果印得漂亮一點（含 emoji）。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━


### 常見新手坑

- **裝到 Python 2** —— 2026 別碰、用 3.12 +
- **動到系統 Python** —— Mac 內建的 Python 不要直接拿來開發、會影響作業系統。請用 uv 或 pyenv 管。
- **不用 venv 全裝到 global** —— 不同專案版本會打架、地獄。
- **檔名跟內建套件衝突** —— 例如自己寫 `json.py`、`os.py`、會 **shadow（遮蔽）** Python 內建的同名套件、之後 import 會出怪事。
- **縮排混 Tab 跟 Space** —— Python 對縮排很嚴格、混用會錯。設定編輯器只用 Space（VS Code 預設）。
