FastAPI 兩大強項：**async（高並發）** + **DI（依賴注入、解耦 + 好測試）**。

> 💡 **3 個關鍵字先記**：
> - **async** = 非同步、「等東西時不阻塞、可以做別的」
> - **await** = 「在這等」、async function 內用
> - **DI（Dependency Injection、依賴注入）** = 「不自己造、要的東西讓外面送進來」

  
### Async 路由（FastAPI 高並發秘密）

**同步（blocking、阻塞）vs Async（non-blocking、非阻塞）對比：**

```text
# === 同步（傳統、阻塞）===
@app.get("/users/{id}")
def get_user(id: int):
    user = db.query(User).filter(User.id == id).first()
    #     ↑ 等 DB 回應、這 50ms 期間 thread 卡死、不能服務別人
    return user

# === Async（非阻塞）===
@app.get("/users/{id}")
async def get_user(id: int):           # ← function 前加 async
    user = await db.fetch_one(query)   # ← I/O 前加 await
    #     ↑ 等 DB 期間、thread 去處理別的請求、回來再繼續
    return user
```

> 💡 **直觀比喻**：餐廳服務員。
> - **同步** = 一個服務員一桌客人、客人吃飯時站著等
> - **Async** = 一個服務員多桌、客人吃飯時去服務別桌、有事再回來

**這行做什麼？** 裝 `httpx`、現代版的 `requests`、**同時支援同步 + async**。FastAPI 內呼叫外部 API 用這個。

```text
import httpx

@app.get("/proxy/{url}")
async def proxy(url: str):
    async with httpx.AsyncClient() as client:
        response = await client.get(url)
    return response.json()
```

  
**並行跑多個 async（這才是 async 真正爆強的地方）：**

```text
import asyncio

@app.get("/dashboard")
async def dashboard():
    # === 3 個 API 同時打、不是一個個等 ===
    user, orders, notifications = await asyncio.gather(
        fetch_user(),
        fetch_orders(),
        fetch_notifications()
    )
    return {
        "user": user,
        "orders": orders,
        "notifications": notifications
    }
# 假設 3 個各要 100ms：
# 同步寫法：300ms（一個個等）
# Async + gather：100ms（同時跑、等最慢的）
```

> 💡 **`asyncio.gather()` = async 版的「並行打 3 個 API」**——dashboard 類場景必用。

  
### 什麼時候用 async？什麼時候不用？

```text
=== ✓ 該用 async ===
- I/O 密集（I/O bound）：等 DB、等外部 API、等檔案讀寫
- 高並發 web server（每秒上千 request）
- WebSocket / Server-Sent Events（即時通訊）

=== ✗ 不用 async ===
- CPU 密集（CPU bound）：純計算、ML 推論、影像處理
  - async 對 CPU 沒幫助、反而拖累
- 簡單 script、一次性工具

=== 混合場景：用 asyncio.to_thread ===
async def endpoint():
    data = await fetch_from_db()                  # I/O 用 async
    
    # CPU 密集丟 thread、不卡住 event loop
    result = await asyncio.to_thread(
        heavy_computation, data
    )
    return result
```

  
### Dependency Injection（DI、依賴注入）

**痛點：每個 endpoint 都要 DB session（重複 setup + teardown）**

```text
# === ❌ 沒 DI（5 個 endpoint = 5 次同樣 code）===
@app.get("/users")
def list_users():
    db = SessionLocal()
    try:
        return db.query(User).all()
    finally:
        db.close()

@app.get("/products")
def list_products():
    db = SessionLocal()                           # 一樣 code 重複
    try:
        return db.query(Product).all()
    finally:
        db.close()
```

  
**用 `Depends`（FastAPI 的 DI 機制）：**

```text
from fastapi import Depends

# === 1. 定義 dependency function ===
def get_db():
    db = SessionLocal()
    try:
        yield db                                  # yield = 生成 db 給 endpoint 用
    finally:
        db.close()                                # endpoint 跑完、自動關
# 注意是 yield 不是 return、寫法跟 context manager 類似（L11 教過）

# === 2. endpoint 用 Depends 拿 ===
@app.get("/users")
def list_users(db: Session = Depends(get_db)):
    return db.query(User).all()

@app.get("/products")
def list_products(db: Session = Depends(get_db)):
    return db.query(Product).all()

# DI 自動處理 setup → 給 db → cleanup
# endpoint 只關心「我要 db、拿來用」
```

  
### Auth Dependency（最常見、99% 後端都用）

**場景**：很多 endpoint 都要「**先驗證 token、拿到當前 user**」、寫成 dependency 一勞永逸。

```text
from fastapi import Depends, HTTPException, status
from fastapi.security import OAuth2PasswordBearer
from jose import jwt, JWTError

oauth2_scheme = OAuth2PasswordBearer(tokenUrl="token")
# ↑ 告訴 FastAPI：「token 從 Authorization: Bearer xxx header 拿」

def get_current_user(token: str = Depends(oauth2_scheme)):
    # 1. 解 JWT（L29 詳教 JWT / OAuth）
    try:
        payload = jwt.decode(token, SECRET_KEY, algorithms=["HS256"])
        user_id = payload.get("sub")
    except JWTError:
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail="Invalid token"
        )
    
    # 2. 從 DB 找 user
    user = db.query(User).filter(User.id == user_id).first()
    if not user:
        raise HTTPException(status_code=404)
    return user

# === 用：在 endpoint 加 current_user = Depends(get_current_user) ===
@app.get("/me")
def read_me(current_user: User = Depends(get_current_user)):
    return current_user
# 自動：解 token → 找 user → 拿到 → 才執行 endpoint

@app.delete("/posts/{post_id}")
def delete_post(
    post_id: int,
    current_user: User = Depends(get_current_user)
):
    # 自動有 current_user、不用每個 endpoint 重複驗 token
    ...
```

  
### 巢狀 Dependency（dependency 可以再用 dependency）

```text
def get_db():
    ...

def get_current_user(db: Session = Depends(get_db), token: str = Depends(oauth2_scheme)):
    # ↑ 這個 dependency 自己也用 db dependency
    ...

def require_admin(user: User = Depends(get_current_user)):
    # ↑ 這個 dependency 用 user dependency
    if user.role != "admin":
        raise HTTPException(403, "Admin only")
    return user

@app.delete("/users/{id}")
def admin_delete(
    id: int,
    admin: User = Depends(require_admin)
):
    # FastAPI 自動：
    # 1. get_db → 拿到 db
    # 2. get_current_user(db, token) → 拿到 user
    # 3. require_admin(user) → 驗 admin、拿到 admin
    # 4. 才執行 admin_delete
    ...
```

  
### Route-level / App-level Dependencies（middleware 概念）

```text
# === 整個 router 都要 admin 驗證 ===
admin_router = APIRouter(
    prefix="/admin",
    dependencies=[Depends(require_admin)]          # 注意是 list
)

@admin_router.delete("/users/{id}")
def delete_user(id: int):
    # require_admin 自動跑、不用在這宣告
    ...

# === 整個 app 都要某個檢查（如 API key）===
app = FastAPI(dependencies=[Depends(verify_api_key)])
```

  
### Dependency 在測試的好處（最大優勢）

DI 讓 endpoint **超好測試**——測試時可以「**換成假的 dependency**」、不打 DB / 不打外部 API。

```text
from fastapi.testclient import TestClient

def get_test_db():                                # 假的 DB（in-memory）
    return MockDB()

# === 在測試裡 override ===
app.dependency_overrides[get_db] = get_test_db

client = TestClient(app)
response = client.get("/users")
# ↑ endpoint 拿到的 db 是假的、測試不會打真 DB、超快
```

  
    **🚀 進階技巧**
    
FastAPI 2 大殺手鐧：(1) **async** —— I/O 並行、跑得快、`asyncio.gather` 並行多個請求 (2) **`Depends`** —— DI 解耦、test 友善、共用邏輯（auth / DB / settings）。**實戰建議**：(1) endpoint **預設寫 async**、用 await（除非真的純 CPU 密集才寫普通 def） (2) DB session / current_user / settings 全用 `Depends` (3) **test 用 `dependency_overrides` mock** —— 一秒切假資料、跑 100 個 test 不打 DB。

      學習加速器
