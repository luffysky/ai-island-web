**「Model 跑得很準 = 真的好？」** 不一定——可能是 **過擬合（overfitting、背答案）**。這課教你怎麼診斷 model 健康程度。

  
### 欠擬合 vs 過擬合 vs 剛好

**3 種狀態 + 像考試的比喻：**

```text
=== 欠擬合（Underfitting）===
- Model 太簡單、抓不到資料 pattern
- train 分數低、test 分數也低
- 像「考前完全沒念書」

=== 過擬合（Overfitting）===
- Model 太複雜、把訓練資料「背起來」
- train 分數很高、test 分數低（差距大）
- 像「背題庫但沒理解、考試換題目就掛」

=== 剛好（Good fit）===
- Model 學到 pattern、能類推到沒看過的資料
- train 跟 test 分數都不錯、差距小
```

  
**怎麼判斷？看 train vs test 分數差距：**

```text
train = 0.95, test = 0.95    → 剛好（差 0）✓
train = 0.60, test = 0.55    → 欠擬合（兩個都低）
train = 1.00, test = 0.70    → 過擬合（差 30%、很明顯）
```

  
### 欠擬合的 4 種對策

```text
1. 增加 model 複雜度
   - Linear Regression → Decision Tree → Random Forest → NN
   - Tree depth 加深
   - NN 加更多層

2. 加更多特徵（feature engineering）
   - 創造新特徵（L18 教過）

3. 訓練更久（更多 epoch、神經網路用）

4. 減少 regularization（正則化、見下方）
```

  
### 過擬合的 5 種對策（更常見、更重要）

**1. 簡化 model**

```text
- Tree 限制深度（max_depth=5）
- NN 減少層數 / 神經元
- 減少特徵
```

  
**2. 加更多訓練資料**（最直接、但常常拿不到更多資料）

  
**3. Regularization（正則化、加「懲罰項」防 model 太極端）**

```text
- L1（Lasso）：把不重要特徵的權重 → 0、自動選特徵
- L2（Ridge）：把所有權重壓小、不歸 0
- Dropout（神經網路用）：訓練時隨機關掉部分神經元、強迫不依賴單一路徑
```

  
**4. Early Stopping（提前停止）**

```text
監測 val 分數、發現開始變差 → 立刻停、不繼續訓練
（再訓練下去只會 overfit）
```

  
**5. Cross-Validation（CV、見下節）**

  
**範例：Random Forest / XGBoost 的防過擬合參數**

```text
# === Random Forest ===
model = RandomForestClassifier(
    n_estimators=100,                       # 100 棵樹（多投票穩定）
    max_depth=10,                           # ⚠️ 限制深度防過擬合
    min_samples_split=20,                   # 分支至少要 20 個樣本
    min_samples_leaf=10,                    # 葉節點至少 10 個樣本
)

# === XGBoost ===
model = xgb.XGBClassifier(
    max_depth=6,                            # 樹深限制
    reg_alpha=0.1,                          # L1 正則化
    reg_lambda=1.0,                         # L2 正則化
    early_stopping_rounds=10,               # 連續 10 輪沒進步就停
)
```

  
### 交叉驗證（Cross-Validation、CV）

**單一 train/test split 的問題：**

```text
- 萬一 test 集剛好簡單 → 高估 model
- 萬一 test 集剛好困難 → 低估 model
- 評估不穩、不可信
```

  
**K-Fold CV** 解法：把訓練資料切 K 份、輪流當 test、跑 K 次取平均。

```text
from sklearn.model_selection import cross_val_score

scores = cross_val_score(model, X, y, cv=5)        # 切 5 份
print(f"平均分數：{scores.mean():.3f}")
print(f"標準差：{scores.std():.3f}")
# 例：CV 0.85 ± 0.02 → 5 次都 0.83-0.87、穩定可信
# 例：CV 0.85 ± 0.10 → 5 次差很大、model 不穩、要小心
```

  
**Stratified K-Fold（分類用、保持類別比例）：**

```text
from sklearn.model_selection import StratifiedKFold

skf = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)
for train_idx, val_idx in skf.split(X, y):
    X_train, X_val = X[train_idx], X[val_idx]
    y_train, y_val = y[train_idx], y[val_idx]
    # 在這裡訓練 + 評估
```

> 💡 **不平衡資料一定用 Stratified**——保證每個 fold 都有少數類別、不會發生「某 fold 完全沒詐欺、根本沒得學」。

  
### Hyperparameter Tuning（超參數調整）

**hyperparameter（超參數）** = 你「設定」的參數（不是 model 學出來的）、例如 `max_depth=10`、`n_estimators=100`。

**3 種調參方式：**

**1. Grid Search（暴力枚舉、慢但徹底）：**

```text
from sklearn.model_selection import GridSearchCV

param_grid = {
    "n_estimators": [50, 100, 200],
    "max_depth": [3, 5, 10],
}
# 3 × 3 = 9 種組合、每個跑 5-fold CV = 45 次訓練

grid = GridSearchCV(
    RandomForestClassifier(),
    param_grid,
    cv=5,
    scoring="f1",                           # 用 F1 分數選最好
    n_jobs=-1                                # -1 = 用所有 CPU 核心
)
grid.fit(X_train, y_train)

print(grid.best_params_)                    # 最佳組合
print(grid.best_score_)                     # 最佳 F1
```

  
**2. Random Search（隨機取樣、比 Grid 快很多）：**

```text
from sklearn.model_selection import RandomizedSearchCV

random_search = RandomizedSearchCV(
    RandomForestClassifier(),
    param_distributions=param_grid,
    n_iter=50,                              # 隨機試 50 組
    cv=5
)
```

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
uv add optuna
```

**這行做什麼？** 裝 `optuna`、貝氏優化超參數的主流套件。不像 Grid 暴力試所有組合、Optuna 根據前幾次的結果「猜下一個該試什麼」、通常 100 次 Optuna ≈ 1000 次 Grid。

```text
import optuna

def objective(trial):
    # 告訴 Optuna 每個超參數的「搜尋範圍」
    params = {
        "n_estimators": trial.suggest_int("n_estimators", 50, 500),
        "max_depth": trial.suggest_int("max_depth", 3, 20),
        "learning_rate": trial.suggest_float("lr", 0.01, 0.3),
    }
    model = xgb.XGBClassifier(**params)
    return cross_val_score(model, X, y, cv=5).mean()    # 回傳分數給 Optuna

study = optuna.create_study(direction="maximize")        # 要分數越高越好
study.optimize(objective, n_trials=100)                  # 試 100 次
print(study.best_params)
```

  
### 學習曲線（Learning Curve、診斷神器）

**learning curve** = 不同訓練資料量下、train vs val 分數的曲線、一眼看出 model 健康狀況。

```text
from sklearn.model_selection import learning_curve
import matplotlib.pyplot as plt
import numpy as np

train_sizes, train_scores, val_scores = learning_curve(
    model, X, y, cv=5,
    train_sizes=np.linspace(0.1, 1.0, 10)   # 從 10% 資料到 100% 資料、切 10 點
)

plt.plot(train_sizes, train_scores.mean(axis=1), label="train")
plt.plot(train_sizes, val_scores.mean(axis=1), label="val")
plt.legend()
plt.xlabel("Training size")
plt.ylabel("Score")
plt.show()
```

  
**怎麼診斷：**

```text
=== train 高、val 低、差距大 → 過擬合 ===
對策：加資料 / 簡化 model / 加 regularization

=== train 低、val 低、差距小 → 欠擬合 ===
對策：複雜化 model / 加特徵 / 訓練更久

=== train 高、val 高、差距小 → good fit ✓ ===
（理想狀況）
```

  
    **🚀 進階技巧**
    
2026 ML 完整工作流：(1) **baseline model** —— 先跑簡單 model（Logistic Regression / RandomForest）建基準、別一開始就用 XGBoost 調 1 週 (2) **永遠用 CV** —— 不靠單一 split、CV 5-fold 是業界標準 (3) **畫 learning curve** —— 訓練前先畫一張、判斷該加資料還是調 model (4) **Optuna 調參** —— 比 Grid 快 10 倍、業界主流 (5) **記錄實驗** —— MLflow / Weights & Biases（W&B）追蹤每次實驗的參數 + 分數、不然兩週後忘光自己試過什麼。
