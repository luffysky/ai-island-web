進 AI 章—先理解 ML 三大類、什麼問題用什麼方法。

  
### ML 是什麼

```text
傳統程式：
規則 + 資料 → 答案

ML：
資料 + 答案 → 規則（自己學）

例：分辨貓狗
- 傳統：「有鬍子 + 4 腿 + 喵叫 = 貓」（規則寫不完）
- ML：給 1000 張貓 + 1000 張狗、機器自己學
```

  
### 3 大類

```text
1. 監督學習（Supervised）
   - 有「答案」訓練
   - 給 input + 正確 output、學映射

   類型：
   - 分類：垃圾信 / 正常信、貓 / 狗 / 兔子
   - 回歸：房價、氣溫預測（連續數字）

2. 非監督學習（Unsupervised）
   - 沒「答案」、自己找 pattern

   類型：
   - 聚類：客戶分群
   - 降維：PCA / t-SNE 視覺化
   - 異常偵測：詐騙、設備故障

3. 強化學習（Reinforcement）
   - Agent 在環境學
   - 試錯 + 獎勵
   - 例：AlphaGo、自駕車、LLM RLHF
```

  
### 監督學習詳解

```text
過程：
[ 訓練資料 ] X (特徵) + Y (答案)
              ↓
        [ 訓練 model ]
              ↓
[ 新資料 ] X' → 預測 Y'

例：房價預測
X：坪數、地點、屋齡、樓層
Y：房價
給 1000 個樣本 → model 學
新房 X' → 預估 Y'

常見演算法：
- Linear Regression（線性回歸）
- Logistic Regression（其實是分類）
- Decision Tree（決策樹）
- Random Forest（隨機森林）
- XGBoost / LightGBM（業界主流）
- Neural Network（深度學習）
```

  
### 非監督學習詳解

```text
沒答案、自己找 pattern：

例 1：客戶分群
1 萬個客戶資料（年齡、消費、頻率）
不知道該怎麼分類
ML 自己分：
- Group 1：年輕高頻小額
- Group 2：中年低頻大額
- Group 3：老年高頻大額

例 2：異常偵測
99% 正常 + 1% 詐欺
ML 學「正常」pattern
新交易不符合 → 標記可疑

常見：
- K-Means（聚類最普及）
- DBSCAN
- PCA / t-SNE / UMAP（降維 / 視覺化）
- Isolation Forest（異常）
```

  
### 2026 ML 全景

```text
傳統 ML（仍主流業界）：
- 表格資料、結構化
- scikit-learn + XGBoost
- 解釋性強

深度學習（DL）：
- 圖片、語音、文字
- PyTorch / TensorFlow
- 需大量資料 + GPU

LLM（大語言模型）：
- GPT-5、Claude、Gemini
- Transformer 為基礎
- 用提示而非訓練
- 改變所有 AI 工作流

選擇：
- 資料是 CSV / 表格 → 傳統 ML（XGBoost）
- 圖 / 聲 / 文字 → DL
- 對話 / 寫作 → LLM（API）
```

  
### ML 工作流（CRISP-DM）

```text
1. 業務理解（解什麼問題？成功標準？）
2. 資料理解（有什麼資料？品質？）
3. 資料準備（清洗、特徵工程、切 train/test）
4. 建模（選演算法、訓練、調參）
5. 評估（準確率？業務指標？）
6. 部署（上線、監控、優化）

實務 1 個 ML project：
- 60% 時間在資料準備
- 20% 在建模
- 10% 在評估
- 10% 在部署

「Data Cleaning is 80% of ML」
```

  
    **☕ 用人話講**
    
ML 3 大類記憶：(1) **監督**= 有答案、學映射（房價、垃圾信） (2) **非監督**= 沒答案、找 pattern（分群、異常） (3) **強化**= 試錯 + 獎勵（遊戲、LLM RLHF）。**2026 indie ML**：90% 場景用 LLM API 直接解、10% 用 scikit-learn / XGBoost—除非做專業 AI 公司、不太需要從頭訓練。
