### 本機跑 vs Production

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**（本機開發模式）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
uvicorn main:app --reload
```

**這行做什麼？** 啟動單一 worker、`--reload` 改 code 自動重啟、適合**開發時**。⚠️ **production 不要加 `--reload`**。

  
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**（Production 模式、多 worker）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
uvicorn main:app --host 0.0.0.0 --port 8000 --workers 4
```

**這行做什麼？逐個解釋：**
- `--host 0.0.0.0` —— 監聽所有網路介面（不只 localhost、外部能連）
- `--port 8000` —— 監聽 port 8000
- `--workers 4` —— 起 4 個 worker process（沒 `--reload`）

  
**或者用 Gunicorn 管 Uvicorn（更穩定、production 主流）：**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**（裝 Gunicorn + 跑）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
uv add gunicorn
gunicorn main:app -w 4 -k uvicorn.workers.UvicornWorker
```

**這 2 行做什麼？**
- `uv add gunicorn` —— 裝 Gunicorn（process manager、管理多個 worker）
- `gunicorn main:app -w 4 -k uvicorn.workers.UvicornWorker` —— 啟動 4 個 worker（`-w 4`）、用 Uvicorn worker class（`-k`）執行 FastAPI app

> 💡 **為什麼要多 worker？** Python 有 **GIL（Global Interpreter Lock、全域解釋器鎖）**——一個 process 內、同時只有 1 個 thread 能執行 Python code。多 worker = 多 process = 真平行。**慣例：workers = CPU 核心數 × 2**。

  
### Dockerfile（把整個 app 打包進 container）

**Docker** = 「**把程式 + 環境一起打包**」、到哪都能跑、不會「我電腦能跑你電腦不能」。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📄 **寫進專案根目錄 Dockerfile**（無副檔名）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
FROM python:3.12-slim                         # 基底 image：精簡版 Python 3.12

WORKDIR /app                                  # 之後指令都在 /app 內執行

# === 裝 uv（從官方 image 直接 copy binary、超快）===
COPY --from=ghcr.io/astral-sh/uv:latest /uv /uvx /bin/

# === 裝依賴（先複製 pyproject、利用 Docker layer cache）===
COPY pyproject.toml uv.lock ./
RUN uv sync --frozen --no-cache               # --frozen = 用 lock 鎖定版本

# === 複製整個專案 code（放最後、code 變動不會 invalidate 上面 cache）===
COPY . .

# === 環境變數（不 buffer log、即時看到）===
ENV PYTHONUNBUFFERED=1

EXPOSE 8000                                   # 宣告 container 用 8000 port

# === 啟動指令 ===
CMD ["uv", "run", "uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000", "--workers", "4"]
```

  
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**（建 + 跑 Docker image）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
docker build -t myapp .
docker run -p 8000:8000 myapp
```

**這 2 行做什麼？**
- `docker build -t myapp .` —— 用當前目錄的 Dockerfile 建 image、tag 名稱叫 `myapp`
- `docker run -p 8000:8000 myapp` —— 跑 container、把容器的 8000 port 對應到本機 8000

  
### docker-compose（完整 stack：app + DB + Redis + worker）

**docker-compose** = 「**一個檔定義多個 services、一起啟動**」——一鍵起 FastAPI + Postgres + Redis + Celery worker。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📄 **寫進專案根目錄 docker-compose.yml**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
version: "3.9"

services:
  # === FastAPI app ===
  app:
    build: .                                  # 用當前目錄的 Dockerfile build
    ports:
      - "8000:8000"
    environment:
      - DATABASE_URL=postgresql+asyncpg://user:pass@db/mydb
      - REDIS_URL=redis://redis:6379
    depends_on:
      - db                                    # 等 db 跟 redis 起來才啟動
      - redis
  
  # === PostgreSQL DB ===
  db:
    image: postgres:16-alpine                 # alpine = 精簡版 Linux
    environment:
      POSTGRES_USER: user
      POSTGRES_PASSWORD: pass
      POSTGRES_DB: mydb
    volumes:
      - postgres_data:/var/lib/postgresql/data    # 資料持久化、container 砍掉資料還在
    ports:
      - "5432:5432"
  
  # === Redis（給 Celery + cache）===
  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
  
  # === Celery worker（L30 教過）===
  worker:
    build: .
    command: celery -A tasks worker --loglevel=info
    depends_on:
      - redis

volumes:
  postgres_data:                              # 宣告 volume
```

  
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**（一鍵啟動全部 services）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
docker compose up
```

**這行做什麼？** 讀 `docker-compose.yml`、把 app / db / redis / worker 全部一起啟動。**`docker compose down`** 全部關掉、**`docker compose up -d`** 背景跑。

  
### 2026 部署平台選擇

**Indie / 簡單（推薦新手）：**

| 平台 | 特色 |
|---|---|
| **Railway** ⭐ | 最簡單、git push 自動 deploy、DB / Redis 一鍵加、$0-30/月、**indie 首選** |
| **Render** | 類 Railway、有免費 tier |
| **Fly.io** | 全球邊緣節點、Docker-native、技術控喜歡 |
| **Vercel** | 主前端、serverless Python 也行、Next.js + Python API 完美組合 |

  
**中型（公司用）：**

| 平台 | 特色 |
|---|---|
| **AWS（ECS / Fargate / Lambda）** | 最完整最複雜、學習曲線陡 |
| **Google Cloud Run** | serverless container、按用量計費 |
| **Heroku** | 老牌、貴但穩、近年走下坡 |

  
**大型 / 不推薦 indie：**

```text
Kubernetes（K8s）：
- 自架 cluster
- 太重、學習成本爆高
- indie / 小團隊不需要
- 等公司 100 人以上再考慮
```

  
### 環境變數 + secrets 管理

**核心原則**：機密**永遠不寫進 code**、放環境變數。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📄 **寫進專案根目錄 .env**（**不要** commit 進 Git）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
DATABASE_URL=postgresql://...
SECRET_KEY=xxx
OPENAI_API_KEY=sk-...
REDIS_URL=redis://...
```

  
**`.gitignore` 一定要加：**

```text
.env
```

  
**讀 .env（2 種方式）：**

```text
# === 方式 1：python-dotenv（簡單）===
from dotenv import load_dotenv
import os

load_dotenv()
DATABASE_URL = os.environ["DATABASE_URL"]
SECRET_KEY = os.environ["SECRET_KEY"]

# === 方式 2：pydantic-settings（推薦、L26 教過）===
from pydantic_settings import BaseSettings

class Settings(BaseSettings):
    database_url: str
    secret_key: str
    openai_api_key: str
    
    class Config:
        env_file = ".env"

settings = Settings()
# 用：settings.database_url
```

> 💡 **Production 不要 .env**——在 Railway / Vercel / Render 後台**直接設環境變數**、平台會自動注入。

  
### 監控 + 日誌（production 必備）

**Sentry：抓未處理的 exception、自動 alert：**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**（裝 Sentry SDK）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
uv add sentry-sdk
```

**這行做什麼？** 裝 Sentry Python SDK、會自動抓 unhandled exception 上報 Sentry dashboard、production error 0 漏網。

```text
import sentry_sdk

sentry_sdk.init(
    dsn=os.environ["SENTRY_DSN"],             # Sentry 給你的 DSN
    traces_sample_rate=0.1,                    # 10% 請求做 performance trace
)
# 之後任何 unhandled exception 自動上報 Sentry、含 stack trace + user context
```

  
**Structured Log（用 structlog、log 變 JSON 好搜索）：**

```text
import structlog
logger = structlog.get_logger()

logger.info("user_login", user_id=123, ip="1.2.3.4")
# 輸出 JSON：{"event": "user_login", "user_id": 123, "ip": "1.2.3.4"}
# 之後可以在 log 系統（DataDog / CloudWatch）按 user_id 搜尋
```

  
**Health Check endpoint（讓 Railway / K8s 知道服務還活著）：**

```text
from sqlalchemy import text

@app.get("/health")
async def health(db: AsyncSession = Depends(get_db)):
    try:
        await db.execute(text("SELECT 1"))    # 試打一下 DB
        return {"status": "ok"}
    except Exception:
        return {"status": "unhealthy"}, 503
```

  
### Indie SaaS 完整 stack（2026 推薦組合）

```text
=== 2026 indie 最佳組合（月成本 $0-30 撐 1000 user）===

App hosting：Railway 或 Fly.io
DB：Neon（managed PostgreSQL、serverless、$0 起）
Cache：Upstash Redis（serverless、按請求數計費）
Storage：Cloudflare R2（S3 兼容、無 egress 費）
CDN：Cloudflare
Email：Resend（modern email API、3000 封/月免費）
Auth：Clerk（L29 推薦過）
Monitoring：Sentry
Domain：Cloudflare（域名 + DNS）

=== 月成本估算 ===
0-1000 user：$0-30
10k user：$50-200
100k user：$200-1000
```

  
**從本機到 production 的 8 步驟流程：**

```text
1. uv sync                       # 鎖定依賴版本
2. 把 .env 內機密設到 Railway / Vercel 後台
3. uvicorn 本機跑通測試
4. 寫 Dockerfile（如果平台需要）
5. git push → Railway / Render 自動 deploy
6. 設 Sentry 監控
7. Custom domain（Cloudflare DNS 指向平台）
8. 上線！
```

Python 後端部署 2026 心法、5 個重點：(1) **Uvicorn + Gunicorn workers**（多 worker 用滿 CPU） (2) **Docker**（環境一致、不再「我電腦能跑你電腦不能」） (3) **Railway / Fly.io**（git push 自動 deploy、indie 首選） (4) **Managed DB / Redis**（Neon / Upstash、不自己架伺服器） (5) **Sentry**（error 監控、production 必備）。**Indie 鐵則**：**不要學 K8s**——用 PaaS 月 $30 撐 1 萬用戶完全沒問題、省下時間做產品 / 找客戶。**這課是 ch26 終點、ch27 開始 Python 資料分析 / ML 應用篇**。
