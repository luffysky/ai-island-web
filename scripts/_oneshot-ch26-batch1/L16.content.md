### ML 是什麼

**傳統程式 vs ML 的根本差異：**

```text
=== 傳統程式 ===
規則 + 資料 → 答案
（你寫好規則、給資料、電腦算答案）

=== ML ===
資料 + 答案 → 規則（電腦自己學）
（你給資料 + 對應答案、電腦學出規則、之後拿來預測）
```

  
**經典例子：分辨貓 vs 狗**

```text
=== 傳統做法 ===
「if 有鬍子 and 4 腿 and 喵叫 → 貓」
→ 寫不完、且邊界一堆 case
→ 例：「沒鬍子的貓？」「會喵的狗（少見但有）？」

=== ML 做法 ===
給機器 1000 張貓 + 1000 張狗的照片
→ 機器自己學會「什麼是貓 / 狗」
→ 給新照片、機器自己判斷
```

  
### ML 三大類（學前先記、之後所有 AI 概念都從這分支）

**1. 監督學習（Supervised Learning）**——**有「標準答案」**訓練

```text
特徵：給 input 加上「正確 output」、教機器「這對 X 對應這個 Y」
                                  
分成 2 子類：
- 分類（classification）：output 是「類別」
   例：垃圾信 / 正常信、貓 / 狗 / 兔子、要 / 不要點擊廣告
- 回歸（regression）：output 是「連續數字」
   例：房價（多少萬）、明天氣溫（幾度）、股價（多少元）
```

  
**2. 非監督學習（Unsupervised Learning）**——**沒「答案」、機器自己找 pattern**

```text
特徵：給機器一堆資料、不告訴它「該怎麼分」、它自己找規律
                                  
3 大子類：
- 聚類（clustering）：把相似的湊一起
   例：1 萬個客戶 → 自動分成「年輕高頻」「中年大額」「老年穩定」3 群
- 降維（dimensionality reduction）：100 維資料壓成 2 維好畫圖
   例：PCA / t-SNE / UMAP
- 異常偵測（anomaly detection）：找「跟大家不一樣」的
   例：詐騙交易、設備即將故障
```

  
**3. 強化學習（Reinforcement Learning、RL）**——**Agent 在環境裡試錯 + 獎勵**

```text
特徵：機器「主動行動」、根據「獎勵」調整策略
                                  
例：
- AlphaGo（下圍棋、贏了得正分、輸了得負分）
- 自駕車（成功到達 +、撞車 -）
- LLM 的 RLHF（Reinforcement Learning from Human Feedback、
   用人類回饋強化學習、訓練 GPT / Claude 那種「對齊人類偏好」的階段）
```

  
### 監督學習詳解（最常用、新手先學這個）

**訓練流程：**

```text
[訓練資料]  X (特徵 features) + Y (答案 labels)
                ↓
        [訓練 model（模型）]
                ↓
[新資料]   X' → 預測 Y'
```

  
**範例：房價預測**

```text
X（特徵）：坪數、地點、屋齡、樓層、學區……
Y（答案）：實際成交房價

→ 給 model 1000 個 (X, Y) 樣本
→ model 學會「特徵」跟「價格」的關係
→ 新房 X' 進來、model 預估 Y'（這間大概多少錢）
```

  
**常見演算法（這課只介紹名字、L17 / L18 真的用）：**

| 演算法 | 用途 |
|---|---|
| **Linear Regression**（線性回歸） | 簡單回歸、找一條直線擬合資料 |
| **Logistic Regression**（邏輯回歸） | 雖然叫「回歸」、但其實是**分類**（為什麼叫 regression？歷史包袱） |
| **Decision Tree**（決策樹） | 像 if-else 樹、可解釋性強 |
| **Random Forest**（隨機森林） | 很多決策樹投票、超穩 |
| **XGBoost / LightGBM** | 業界主流、表格資料王者、Kaggle 常勝軍 |
| **Neural Network**（神經網路） | 深度學習基礎、L21 詳教 |

  
### 非監督學習詳解

**範例 1：客戶分群**

```text
1 萬個客戶資料（年齡、消費、頻率）
不知道該怎麼分類
ML 自己分：
- Group 1：年輕、高頻、小額（學生族）
- Group 2：中年、低頻、大額（家庭主購）
- Group 3：老年、高頻、大額（退休高消費）
→ 行銷團隊針對每群做不同活動
```

  
**範例 2：詐欺偵測**

```text
99% 正常交易 + 1% 詐欺
ML 學「正常」交易長什麼樣（pattern）
新交易進來、不符合 pattern → 標記「可疑」
→ 不用先告訴它「什麼是詐欺」、它自己學
```

  
**常見演算法：**

| 演算法 | 用途 |
|---|---|
| **K-Means** | 聚類最普及、簡單好懂 |
| **DBSCAN** | 找密度高的群、能識別異常點 |
| **PCA / t-SNE / UMAP** | 降維 / 視覺化（高維資料壓成 2D 畫圖） |
| **Isolation Forest** | 異常偵測常用 |

  
### 2026 ML 全景（什麼場景選什麼）

```text
=== 傳統 ML（仍是業界主流）===
- 處理「表格資料、結構化資料」（CSV、SQL 表）
- 工具：scikit-learn + XGBoost / LightGBM
- 解釋性強（可以說「為什麼判斷成這樣」）

=== 深度學習（DL、Deep Learning）===
- 處理「圖片、語音、文字」（非結構化資料）
- 工具：PyTorch / TensorFlow
- 需大量資料 + GPU、訓練成本高
- L21-L24 詳教

=== LLM（Large Language Model、大語言模型）===
- GPT-5 / Claude / Gemini 這種
- 底層：Transformer 架構
- 2026 主流：「用 prompt」而非「自己訓練」
- 改變所有 AI 工作流：客服 / 文案 / 程式 / 設計
```

  
**怎麼選？決策樹：**

```text
你的資料是什麼？
├── CSV / 表格 / SQL → 傳統 ML（XGBoost）
├── 圖 / 聲 / 文字 → 深度學習（PyTorch）
└── 對話 / 寫作 / 翻譯 → LLM API（呼叫 GPT / Claude）
```

  
### ML 工作流（CRISP-DM、業界標準流程）

**CRISP-DM**（Cross-Industry Standard Process for Data Mining、跨業界資料挖掘標準流程）：

```text
1. 業務理解（Business Understanding）
   解什麼問題？成功標準是什麼？

2. 資料理解（Data Understanding）
   有什麼資料？品質好不好？

3. 資料準備（Data Preparation）
   清洗、特徵工程、切 train / test（L18 詳教）

4. 建模（Modeling）
   選演算法、訓練、調超參數（hyperparameter）

5. 評估（Evaluation）
   準確率夠不夠？業務指標達標？（L19 詳教）

6. 部署（Deployment）
   上線、監控、優化
```

  
**實務時間分配（業界共識）：**

```text
1 個 ML 專案的時間分配：
- 60% 在資料準備（清洗、特徵工程）
- 20% 在建模（選演算法、訓練）
- 10% 在評估
- 10% 在部署

→ 「Data Cleaning is 80% of ML」（資料清理佔 80%）這句業界名言
```

  
    **☕ 用人話講**
    
ML 三大類記憶口訣：(1) **監督** = 有答案、學「X → Y 的映射」（房價、垃圾信） (2) **非監督** = 沒答案、自己找 pattern（分群、異常偵測） (3) **強化** = 試錯 + 獎勵（AlphaGo、LLM RLHF）。**2026 獨立開發者 / 學習者**：90% 場景用 **LLM API**（呼叫 GPT / Claude）直接解、10% 用 scikit-learn / XGBoost——除非你做專業 AI 公司、不太需要從頭訓練 model。**先學會用、再學會訓練**。

  

  
    
      🚀
      2026 趨勢
    
    機器學習 2026 重大改變：(1) **LLM 吃掉很多任務**——以前要訓練 model 的事（情感分析 / 分類 / 摘要）現在用 GPT/Claude 一個 API 解決 (2) **AutoML 普及**——不會調參的人也能訓練不錯的 model (3) **Fine-tune 變主流**——拿 Hugging Face 預訓練 model 用自己資料微調、L24 會教。**學習策略**：先學「**怎麼用 LLM**」（CP 值最高）、有興趣再學「**怎麼訓練自己的 model**」（要花時間 + 算力）。
