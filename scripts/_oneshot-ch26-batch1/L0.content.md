### 1. 什麼是「終端機」（Terminal）？

```text
**終端機**（英文叫 Terminal、也叫「命令列」、Command Line）
= 一個可以「打字下指令給電腦」的視窗。

長相：通常是一個黑色或白色的視窗、裡面只有文字、沒有圖示按鈕。

怎麼打開：
- Windows：按 Win 鍵 → 打「cmd」（或「PowerShell」）→ Enter
- Mac：按 Cmd + 空白 → 打「Terminal」→ Enter
- Linux：工具列通常就有「Terminal」這個 App

【比喻】終端機像「跟電腦對話的對講機」：
- 你按一下、講一句話（打指令）
- 電腦聽到、做一件事（執行）
- 做完、把結果說回給你（顯示在視窗）

終端機裡常會看到一個提示符號：
- Mac / Linux 是「$」
- Windows 是「>」

那個符號代表「終端機準備好了、等你打字」——
**不是要你打的字**。
教材後面寫到「$ uv run hello.py」、那個 $ 不要打、只打後面的 uv run hello.py。
```

  
### 2. 什麼是「程式檔」？

```text
**程式** = 寫在一個「檔案」裡的指令清單。

例如：hello.py
- 「hello」是這個檔案的名字（你自己取的）
- 「.py」叫「副檔名」、代表這是一個 Python 檔
- 整個檔名連起來、就是「一個叫 hello 的 Python 程式檔」

寫程式 = 用「編輯器」（例如 VS Code）把指令打進這個檔案、按存檔。

【比喻】程式檔像「食譜」：
- 你把步驟一條條寫下來、存成一張紙（檔案）
- 之後想吃幾次、就照這張食譜做幾次

常見副檔名對照：
- .py      Python 檔
- .js      JavaScript 檔
- .html    網頁檔
- .txt     純文字檔
- .json    設定資料檔（電腦讀的）
```

  
### 3. 「寫程式」和「執行程式」是兩件事

```text
這是零基礎最容易混淆的點、講清楚：

【寫程式】= 把指令打進 .py 檔、按存檔
- 用 VS Code、Cursor、甚至記事本——任何「編輯器」都能做
- 寫完不會自動跑、就只是「一張食譜」躺在那裡

【執行程式】= 去終端機、叫電腦「照這張食譜跑一次」
- 在終端機打：uv run hello.py
- 電腦讀那個檔、一行一行照做
- 跑完、把結果印在終端機裡給你看

【比喻】
寫程式 = 把食譜寫下來
執行程式 = 拿食譜進廚房、真的煮一次

→ 一個檔案可以「寫一次、執行很多次」
→ 改了檔案、要重新執行一次、才會看到新結果
→ 沒執行 = 程式沒跑、什麼都不會發生
```

  
### 4. 教材的四種區塊標示（之後會一直看到）

```text
為了讓你一眼分辨「這段內容要做什麼用」、
教材把程式相關區塊分成 4 種、每種有自己的標籤：

📄 寫進程式檔
   → 這段是你要打字寫進 .py 檔的程式碼

🖥️ 在終端機輸入
   → 這行是你要打在終端機（黑視窗）的指令

⌨️ 這時換你打字
   → 程式跑到一半、需要你輸入東西時、要打的字

💬 電腦會顯示
   → 這是電腦印出來給你看的結果、不是你打的

【為什麼要分這麼細？】
以前同學最常卡的就是：「咦這行到底要打哪裡？是寫進檔案？還是打在終端機？」
有了這 4 個標籤、你一眼就分得出來：
- 看到 📄 → 打開編輯器、寫進 .py 檔
- 看到 🖥️ → 切到終端機、打進去
- 看到 ⌨️ → 程式問你問題時、就照這個打
- 看到 💬 → 不用做事、那是電腦自己印的結果
```

  
### 5. 遇到看不懂的英文術語怎麼辦？

```text
程式世界很多英文術語（Terminal、REPL、docstring、type hint、shell……）。
完全沒碰過的人、看到一個不認識的詞就會卡住——這完全正常。

【教材怎麼幫你】
每個英文術語「第一次在某課出現」時、會寫成：
   英文 +（中文）+ 一句白話說明

例如：
- **REPL（讀取-執行-印出 循環）**——一個「你打一行、它立刻給你結果」的互動視窗。
- **shell（殼層、作業系統的命令列）**——終端機裡跑的那個東西、可以下系統指令。
- **docstring（文件字串）**——寫在函式或類別開頭、說明「這段程式做什麼用」的文字。
- **type hint（型別提示）**——在變數或參數旁邊標註「這裡應該放什麼型別的資料」、像貼一張標籤。

同一課再次出現、就只用英文（你已經認識它了）。

【你可以做的事】
- 看到不懂的詞、先別跳過、別緊張
- 教材沒解釋的、隨時用綠寶 AI 導師問：
   「這個詞是什麼意思？舉個例子？」
- 卡住 5 分鐘還是看不懂、就去問——
   不要硬撐 30 分鐘、那會把信心磨光

【最重要的心態】
- 第一次看不懂 = 正常
- 第二次看不太懂 = 正常
- 第五次還是不懂 = 那也正常、再多看幾次就會懂
- 「我學不會」是一個假象——所有寫程式的人剛開始都這樣
```
