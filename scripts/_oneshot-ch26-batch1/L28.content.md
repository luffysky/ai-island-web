**SQLAlchemy** 是 Python **ORM（Object-Relational Mapping、物件關聯映射）業界標準**——Stripe / Reddit / Yelp 都用。**2.0 版**加入 async + 全新 modern API。

> 💡 **ORM 是什麼？** 「**用 Python class 操作資料庫**」——不寫 SQL、用 `user.name = "Alice"` + `db.commit()` 就更新資料庫。SQLAlchemy 把 SQL 隱藏起來、你只看到 Python 物件。

  
### 定義 Model（用 Python class 描述資料庫表）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**（裝 SQLAlchemy + asyncpg + Alembic）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
uv add sqlalchemy asyncpg alembic
```

**這行做什麼？逐個解釋：**
- `sqlalchemy` —— ORM 主套件（v2.0+）
- `asyncpg` —— PostgreSQL 的 **async 驅動程式**（讓 SQLAlchemy 能 async 操作 Postgres）
- `alembic` —— DB migration（schema 版本控制、L29 後段詳教）

  
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📄 **寫進程式檔 models.py**（定義資料表 = Python class）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
from sqlalchemy.orm import DeclarativeBase, Mapped, mapped_column, relationship
from sqlalchemy import String, Integer, ForeignKey, DateTime, func
from datetime import datetime

# === 基類、所有 model 都繼承這個 ===
class Base(DeclarativeBase):
    pass

# === User 表 ===
class User(Base):
    __tablename__ = "users"                  # 對應 DB 內的 table 名稱
    
    # Mapped[型別] = 告訴 SQLAlchemy 這欄是什麼型別 + 自動生 type hint
    id: Mapped[int] = mapped_column(primary_key=True)
    name: Mapped[str] = mapped_column(String(100))
    email: Mapped[str] = mapped_column(String(255), unique=True)
    age: Mapped[int | None]                  # | None = 可為 NULL
    created_at: Mapped[datetime] = mapped_column(default=func.now())
    
    # 關聯：一個 user 有多個 order（一對多）
    orders: Mapped[list["Order"]] = relationship(back_populates="user")

# === Order 表 ===
class Order(Base):
    __tablename__ = "orders"
    
    id: Mapped[int] = mapped_column(primary_key=True)
    user_id: Mapped[int] = mapped_column(ForeignKey("users.id"))   # 外鍵
    total: Mapped[float]
    
    # 反向關聯：一個 order 屬於一個 user
    user: Mapped["User"] = relationship(back_populates="orders")
```

> 💡 **Mapped vs `mapped_column`** —— `Mapped[X]` 是 type hint（給 IDE / mypy 用）、`mapped_column(...)` 是欄位設定（給 SQLAlchemy 用、像 primary_key / unique / default）。

  
### 連線 + Session（async 版）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📄 **寫進程式檔 database.py**（連線設定）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
from sqlalchemy.ext.asyncio import create_async_engine, async_sessionmaker, AsyncSession

# === 連線字串（從 .env 讀比較安全、L8 教過）===
DATABASE_URL = "postgresql+asyncpg://user:pass@localhost/mydb"
# 格式：dialect+driver://user:pass@host/dbname

# === 建立 async engine（連線池底層）===
engine = create_async_engine(
    DATABASE_URL,
    echo=True                                # 開發時印 SQL 出來、production 關掉
)

# === Session factory（每次操作 DB 開一個 session）===
AsyncSessionLocal = async_sessionmaker(engine, expire_on_commit=False)

# === FastAPI Dependency（L27 教過 Depends）===
async def get_db() -> AsyncSession:
    async with AsyncSessionLocal() as session:
        yield session
```

  
**FastAPI endpoint 怎麼用：**

```text
from fastapi import Depends
from sqlalchemy import select

@app.get("/users")
async def list_users(db: AsyncSession = Depends(get_db)):
    result = await db.execute(select(User))   # 跑 SQL
    return result.scalars().all()              # 拿所有結果
```

  
### CRUD（Create / Read / Update / Delete）

**Create（新增）：**

```text
new_user = User(name="Alice", email="alice@x.com", age=30)
db.add(new_user)                              # 加進 session（還沒寫 DB）
await db.commit()                             # 真正寫入 DB
await db.refresh(new_user)                    # 從 DB 重抓、取得自動產生的 id
print(new_user.id)                            # 1（DB 給的）
```

  
**Read（查詢、L20 講 SELECT 的 ORM 版）：**

```text
from sqlalchemy import select

# === 一個（用主鍵）===
user = await db.get(User, 1)                  # 最簡寫法

# === 一個（複雜條件、用 select + where）===
result = await db.execute(select(User).where(User.id == 1))
user = result.scalar_one_or_none()            # 0 個 = None、1 個 = User、多個 = 例外

# === 多個 ===
result = await db.execute(select(User).where(User.age > 18))
users = result.scalars().all()

# === 排序 + 分頁（最常見組合）===
stmt = (
    select(User)
    .where(User.is_active == True)
    .order_by(User.created_at.desc())
    .limit(20)
    .offset(0)
)
result = await db.execute(stmt)
users = result.scalars().all()
```

  
**Update（更新）：**

```text
# === 方式 1：先查再改（適合單一物件）===
user = await db.get(User, 1)
user.name = "Alice Smith"
await db.commit()                             # 自動偵測改動、生 UPDATE SQL

# === 方式 2：bulk update（適合批次）===
from sqlalchemy import update
await db.execute(
    update(User)
    .where(User.id == 1)
    .values(name="Alice Smith")
)
await db.commit()
```

  
**Delete（刪除）：**

```text
# === 方式 1：先查再刪 ===
user = await db.get(User, 1)
await db.delete(user)
await db.commit()

# === 方式 2：bulk delete ===
from sqlalchemy import delete
await db.execute(delete(User).where(User.id == 1))
await db.commit()
```

  
### Relationship + Join（N+1 問題的解決）

**N+1 問題**（performance 最常見的坑）：

```text
# === ❌ Lazy load（每個 user 多打 1 個 query）===
users = (await db.execute(select(User))).scalars().all()  # 1 個 query 拿 100 個 user
for user in users:
    print(user.orders)                        # 每個 user 又打 1 個 query 拿 orders
# 結果：1 + 100 = 101 個 query！這就是 N+1
```

  
**解法：用 `selectinload` / `joinedload` eager load：**

```text
from sqlalchemy.orm import selectinload, joinedload

# === selectinload（推薦、一對多用）===
stmt = (
    select(User)
    .options(selectinload(User.orders))       # 連 orders 一起載
    .where(User.id == 1)
)
result = await db.execute(stmt)
user = result.scalar_one()
# 2 個 query：(1) 撈 user (2) 一次撈所有相關 orders
# 比 N+1 快非常多

# === joinedload（一對一用、JOIN 出來）===
stmt = select(User).options(joinedload(User.profile)).where(User.id == 1)
```

  
**JOIN（明確跨表查詢）：**

```text
from sqlalchemy import select

stmt = (
    select(User, Order)
    .join(Order)                              # JOIN users + orders
    .where(Order.total > 1000)
)
result = await db.execute(stmt)
for user, order in result:
    print(user.name, order.total)
```

  
### Alembic：DB Migration（schema 改動的版本控制）

**為什麼要 migration？** 改 model（加欄位 / 改型別）時、**不能手動進 DB 改**——隊友的 DB 會壞、production 跟 dev 不同步。用 Alembic 自動生 migration script、所有人跑同一個 script、永遠同步。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**（初始化 Alembic）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
uv run alembic init alembic
```

**這行做什麼？** 在當前資料夾建一個 `alembic/` 子目錄、含 `env.py`（設定）+ `versions/`（migration 檔案存這）+ `alembic.ini`（總設定）。

  
**設定 `alembic.ini` 內的連線字串：**

```text
sqlalchemy.url = postgresql+asyncpg://user:pass@localhost/mydb
```

  
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**（改 model 後、自動生 migration）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
uv run alembic revision --autogenerate -m "add user.age column"
```

**這行做什麼？逐個解釋：**
- `alembic revision` —— 建一個新的 migration 檔
- `--autogenerate` —— **自動比對「現在的 model」vs「DB 現狀」、生對應的 SQL**
- `-m "add user.age column"` —— migration 訊息（讓未來知道這個檔在做什麼）

跑完會在 `alembic/versions/xxx_add_user_age_column.py` 看到自動生的檔：

```text
def upgrade():
    op.add_column("users", sa.Column("age", sa.Integer))

def downgrade():                              # 回滾用、自動產生反向操作
    op.drop_column("users", "age")
```

  
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**（套用 migration 到 DB）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
uv run alembic upgrade head
```

**這行做什麼？** 跑所有「還沒套用」的 migration 到最新版（head）。隊友拉 code 也跑這行、保證 DB 同步。

  
**其他常用指令：**

```text
uv run alembic current               # 看 DB 現在是哪個版本
uv run alembic history               # 看所有 migration 紀錄
uv run alembic downgrade -1          # 回滾上一個 migration（出錯時用）
```

  
### Production 最佳實踐

**1. Connection Pool（連線池、控制資源）：**

```text
engine = create_async_engine(
    DATABASE_URL,
    pool_size=20,                             # 平常維持 20 條連線
    max_overflow=10,                          # 高峰可加開 10 條
    pool_pre_ping=True,                       # 每次用前 ping 一下、避免拿到死連線
)
```

  
**2. Transaction（原子操作、要嘛全成功要嘛全失敗）：**

```text
async def transfer(from_id, to_id, amount):
    async with db.begin():                    # 自動開 transaction
        from_user = await db.get(User, from_id)
        from_user.balance -= amount
        
        to_user = await db.get(User, to_id)
        to_user.balance += amount
        
    # 跳出 with：
    # - 沒 raise → 自動 commit
    # - 有 raise → 自動 rollback（兩個帳戶都不會被改）
```

  
**3. Repository Pattern（把 DB 操作集中、好測試 + 不洩漏 SQL）：**

```text
class UserRepository:
    def __init__(self, db: AsyncSession):
        self.db = db
    
    async def find_by_id(self, id: int) -> User | None:
        result = await self.db.execute(
            select(User).where(User.id == id)
        )
        return result.scalar_one_or_none()
    
    async def find_active(self) -> list[User]:
        result = await self.db.execute(
            select(User).where(User.is_active == True)
        )
        return result.scalars().all()
    
    async def save(self, user: User) -> User:
        self.db.add(user)
        await self.db.commit()
        await self.db.refresh(user)
        return user

# === 用 ===
@app.get("/users/{id}")
async def get_user(id: int, db: AsyncSession = Depends(get_db)):
    repo = UserRepository(db)
    user = await repo.find_by_id(id)
    if not user:
        raise HTTPException(404)
    return user
# Endpoint 不直接寫 SQL、改 SQL 只需改 Repository、好維護
```
