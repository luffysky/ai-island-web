**不同資料型態、用不同 NN 架構**：圖片用 CNN、序列用 RNN、語言用 Transformer。這課介紹 3 大架構各自的設計思路、不深入數學、重點是「**什麼時候該用哪個**」。

  
### CNN（Convolutional Neural Network、卷積神經網路、圖片之王）

**為什麼圖片不能用普通 NN？**

```text
一張 224 × 224 RGB 圖片 = 150,528 個 pixel value
如果用普通全連接 NN：
- 第 1 層就 150528 個 input
- 參數量爆炸、容易過擬合
- 沒利用「圖片的 local 性質」（相鄰像素相關）
```

  
**CNN 的解法：用「卷積層（Conv）」學 local pattern**

```text
卷積（convolution）= 一個小視窗（kernel）滑過整張圖、學「邊緣 / 紋理 / 角點」
- 第 1 層學基礎特徵（線條、邊緣）
- 第 2 層學中階特徵（眼睛、輪子）
- 第 3 層學高階特徵（臉、車）

Pooling（池化）= 降維、把 4 個 pixel 變 1 個（取 max 或平均）
- 減少參數、平移不變性

完整 CNN 結構：
[224x224x3] → Conv → Pool → Conv → Pool → Flatten → FC → Output
```

  
**PyTorch 範例：自己寫一個簡單 CNN：**

```text
class CNN(nn.Module):
    def __init__(self):
        super().__init__()
        # Conv2d(輸入通道, 輸出通道, kernel 大小, padding)
        self.conv1 = nn.Conv2d(3, 32, kernel_size=3, padding=1)
        self.pool = nn.MaxPool2d(2)         # 每 2x2 取最大
        self.conv2 = nn.Conv2d(32, 64, kernel_size=3, padding=1)
        self.fc1 = nn.Linear(64 * 56 * 56, 128)
        self.fc2 = nn.Linear(128, 10)        # 10 類分類
        self.relu = nn.ReLU()
    
    def forward(self, x):
        x = self.pool(self.relu(self.conv1(x)))   # Conv → ReLU → Pool
        x = self.pool(self.relu(self.conv2(x)))
        x = x.view(x.size(0), -1)                  # 攤平給 FC 用
        x = self.relu(self.fc1(x))
        x = self.fc2(x)
        return x
```

  
**實戰：直接用預訓練 model（推薦、不要從零訓練）**

```text
from torchvision.models import resnet50

# === 載入 ImageNet 上預訓練好的 ResNet-50 ===
model = resnet50(weights="DEFAULT")

# === 改最後一層、做自己的分類任務 ===
model.fc = nn.Linear(2048, 10)              # 自己的 10 類
# 前面所有 layer 都共用 ImageNet 學到的特徵（叫 transfer learning、L24 詳教）
```

  
**主流 CNN 家族：**

| Model | 特色 |
|---|---|
| **ResNet**（2015） | 業界 default、加 residual connection 解決深度問題 |
| **EfficientNet**（2019） | 「效能 / 參數比」最佳 |
| **ConvNeXt**（2022） | 用 Transformer 的設計思路重做 CNN |
| **ViT**（Vision Transformer、2020） | 用 Transformer 處理圖片、現在大模型主流 |

  
### RNN（Recurrent Neural Network、遞迴神經網路、處理序列資料）

**適合「有時間順序」的資料：**文字、股價、語音、時序資料。

```text
特色：
- 有「記憶」（hidden state）
- 上一個 input 的處理結果會影響下一個
- 像人「逐字讀句子、邊讀邊累積理解」
```

  
**Vanilla RNN 的問題：學不到「長距離依賴」**

```text
例：「我昨天去了 ___ 那家店、買了很多東西」
- 想填空必須記得「昨天去的店」
- 但中間隔太多字、vanilla RNN 容易忘
```

**解決方案：LSTM / GRU（有「閘門」控制記憶）**

```text
- LSTM（Long Short-Term Memory、長短期記憶）：3 個 gate（input / forget / output）
- GRU（Gated Recurrent Unit）：LSTM 簡化版、2 個 gate、更快
```

  
**PyTorch LSTM 範例：**

```text
class LSTMModel(nn.Module):
    def __init__(self, input_size, hidden_size, num_layers, output_size):
        super().__init__()
        self.lstm = nn.LSTM(
            input_size, hidden_size, num_layers,
            batch_first=True               # (batch, seq, feature) 而不是 (seq, batch, feature)
        )
        self.fc = nn.Linear(hidden_size, output_size)
    
    def forward(self, x):
        out, _ = self.lstm(x)              # out shape: (batch, seq, hidden)
        out = self.fc(out[:, -1, :])       # 取最後時間步的輸出做預測
        return out
```

  
**2026 現實：RNN / LSTM 大幅被 Transformer 取代**

```text
- 文字 / 翻譯 / 對話 → 全部用 Transformer
- LSTM 仍有用：
  - 低資源環境（手機 / 邊緣設備）
  - 序列超長（Transformer 記憶體爆）
  - 即時推論（LSTM 比 Transformer 快）
```

  
### Transformer（2017 革命、開啟現代 AI 時代）

> 論文標題：**「Attention Is All You Need」**（2017、Google）—— 一句話講完整套架構、超有自信。

**核心：Attention（注意力機制）**

```text
傳統 RNN：一個字一個字處理、後面字看不到前面（除非用 memory）
Transformer：每個字「同時看全部字」、學「自己跟誰相關」

例：「The cat sat on the mat because it was tired」
處理 "it" 時的 attention 分數：
- it ↔ cat：0.85   ← model 學會 it 指的是 cat
- it ↔ mat：0.05
- it ↔ tired：0.30
```

  
**Self-Attention 機制（QKV 三個矩陣的舞蹈）：**

```text
每個 token（字）算 3 個向量：
- Q（Query、查詢）：「我要找什麼？」
- K（Key、鍵）：「我有什麼？」
- V（Value、值）：實際內容

公式：attention = softmax(Q × K^T / sqrt(d)) × V
（不用記細節、知道是「比對相關性、加權求和」就行）
```

  
**完整 Transformer 還有：**

```text
- Multi-Head Attention（多頭注意力、平行跑多個 attention、學不同 pattern）
- Feed-Forward（每個位置獨立過 NN）
- Layer Norm（層標準化、穩定訓練）
- Residual Connection（殘差連接、L21 提過、解決深度問題）
```

  
**2026 Transformer 統治的領域：**

| 應用 | 用的模型 |
|---|---|
| 文字生成 | **GPT / Claude / Gemini**（純 decoder 架構） |
| 文字理解 / 分類 | **BERT / RoBERTa**（純 encoder） |
| 翻譯 / 摘要 | **T5**（encoder-decoder） |
| 圖片分類 | **ViT**（Vision Transformer） |
| 語音識別 | **Whisper**（OpenAI、字幕主流） |
| 蛋白質結構預測 | **AlphaFold**（DeepMind、得諾貝爾化學獎 2024） |

→ Transformer **通吃所有 AI 任務**、2026 主流地位無法撼動

3 種 NN 架構記憶口訣：(1) **CNN** = 處理**圖片**（學 local pattern、用卷積） (2) **RNN / LSTM** = 處理**序列**（有「記憶」、按順序處理） (3) **Transformer** = 2017 後**通吃所有任務**（attention 機制、平行看所有 token）。**indie 開發者不用自己實作**——會用 Hugging Face 預訓練 model（L24 詳教）就行、99% 場景夠用。**重點認知**：知道「什麼任務用什麼架構」比會寫程式重要。
