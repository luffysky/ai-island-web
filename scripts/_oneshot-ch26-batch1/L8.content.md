讀寫檔案是日常基本功——讀設定檔、存資料、處理 CSV / JSON / 文字、把結果寫進檔案。

> 💡 **核心關鍵字**：用 **`with`** 開檔（自動關檔、不會漏）。所有範例都會用 `with`。

  
### 讀檔案

**經典寫法（記得 close）：**

```text
f = open("data.txt", "r")          # "r" = read 模式
content = f.read()
print(content)
f.close()                          # ⚠️ 一定要記得關、不然檔案會被 lock 住
```

  
**Pythonic 寫法：用 `with` 自動 close（強烈推薦）：**

```text
with open("data.txt", "r") as f:
    content = f.read()
# 跳出 with 區塊、Python 自動幫你 close、就算中間出錯也會 close
```

  
**讀寫模式對照表：**

| 模式 | 意義 |
|---|---|
| `"r"` | read、讀（預設、檔案不存在會錯） |
| `"w"` | write、寫（會**蓋掉**原本內容！沒檔案就建新的） |
| `"a"` | append、追加（接在原內容後面） |
| `"r+"` | 讀 + 寫 |
| `"rb"` | read binary、讀二進位（圖片 / pdf 之類非文字檔） |
| `"wb"` | write binary、寫二進位 |

  
**3 種讀法（看檔案大小決定）：**

```text
# === 方式 1：整個檔讀進記憶體（小檔用、最簡單）===
with open("data.txt") as f:
    content = f.read()           # 整個檔變成一個 string

# === 方式 2：一行一行讀（大檔用、省記憶體）===
with open("data.txt") as f:
    for line in f:
        print(line.strip())      # strip() 去掉行尾的換行 \n

# === 方式 3：所有行讀成 list ===
with open("data.txt") as f:
    lines = f.readlines()
# ['line 1\n', 'line 2\n', ...]（每行尾還是有 \n）

# === 方式 4：讀指定 bytes（很大很大的檔分批讀）===
with open("data.txt") as f:
    chunk = f.read(1024)         # 讀 1KB
```

  
**⚠️ Windows 讀中文必加 `encoding="utf-8"`：**

```text
with open("data.txt", encoding="utf-8") as f:
    content = f.read()
# Windows 預設用 cp950 編碼（繁中 Big5 變種）、不指定會亂碼
# Mac / Linux 預設 utf-8、但養成「永遠寫 utf-8」習慣最安全
```

  
### 寫檔案

```text
# === 寫（蓋掉、檔案會被清空再寫）===
with open("output.txt", "w") as f:
    f.write("Hello\n")
    f.write("World\n")
# 結果：兩行 Hello、World

# === 寫多行（writelines）===
lines = ["line 1\n", "line 2\n", "line 3\n"]
with open("output.txt", "w") as f:
    f.writelines(lines)
# ⚠️ writelines 不會自動加換行！要自己每行加 \n

# === 追加（不蓋掉、接在後面）===
from datetime import datetime
with open("log.txt", "a") as f:
    f.write(f"{datetime.now()} - event\n")

# === 用 print 寫檔案（小技巧、print 也能寫檔）===
with open("output.txt", "w") as f:
    print("Hello", file=f)        # file= 參數指定輸出到檔
    print("World", file=f)
```

  
### JSON（資料交換格式、最常用）

```text
import json

# === 寫 JSON 進檔 ===
data = {
    "name": "Alice",
    "age": 30,
    "hobbies": ["reading", "hiking"]
}

with open("user.json", "w") as f:
    json.dump(data, f, indent=2)        # indent=2 = 排版美化、好讀

# === 從檔讀 JSON ===
with open("user.json") as f:
    data = json.load(f)

print(data["name"])

# === 字串 ↔ JSON ===
json_str = json.dumps(data)             # dict → JSON 字串（dumps、dump string）
parsed = json.loads(json_str)           # JSON 字串 → dict（loads、load string）
```

  
**⚠️ 中文預設會被 escape 成 `\uXXXX`、要加 `ensure_ascii=False`：**

```text
json.dumps({"name": "愛麗絲"})
# '{"name": "\\u611b\\u9e97\\u7d72"}'  ← 變亂碼

json.dumps({"name": "愛麗絲"}, ensure_ascii=False)
# '{"name": "愛麗絲"}'                ← 正常中文
```

  
**⚠️ 不能直接 dump 的型別（如 datetime）—— 要自訂 encoder：**

```text
from datetime import datetime

data = {"time": datetime.now()}
json.dumps(data)
# ❌ TypeError: Object of type datetime is not JSON serializable

# 解法：自訂 encoder 函式
def encoder(obj):
    if isinstance(obj, datetime):
        return obj.isoformat()      # 把 datetime 轉成 ISO 字串
    raise TypeError()

json.dumps(data, default=encoder)
# '{"time": "2026-05-26T14:30:00.123456"}'
```

  
### CSV（試算表格式、Excel 兼容）

```text
import csv

# === 寫 CSV ===
data = [
    ["name", "age", "city"],
    ["Alice", 30, "Taipei"],
    ["Bob", 25, "Tokyo"],
]

with open("users.csv", "w", newline="") as f:
    writer = csv.writer(f)
    writer.writerows(data)
# ⚠️ newline="" 是 Python 慣例、Windows 上避免多餘空行
```

  
**讀 CSV 兩種方式：**

```text
# === 方式 1：csv.reader（每行回傳 list）===
with open("users.csv") as f:
    reader = csv.reader(f)
    for row in reader:
        print(row)              # ['Alice', '30', 'Taipei']

# === 方式 2：csv.DictReader（用 header 當 key、更直觀）===
with open("users.csv") as f:
    reader = csv.DictReader(f)
    for row in reader:
        print(row["name"], row["age"])
```

  
**寫 CSV 用 DictWriter（直接給 dict）：**

```text
data = [
    {"name": "Alice", "age": 30},
    {"name": "Bob", "age": 25},
]

with open("users.csv", "w", newline="") as f:
    writer = csv.DictWriter(f, fieldnames=["name", "age"])
    writer.writeheader()         # 寫第一行 header
    writer.writerows(data)
```

  
**大資料用 pandas（L14 會詳教）：**

```text
import pandas as pd
df = pd.read_csv("users.csv")
df.to_csv("output.csv", index=False)     # index=False 不要寫額外的編號欄
```

  
### pathlib（現代檔案操作、L7 介紹過、這裡完整列）

**pathlib** 是 Python 3.4+ 的「物件導向版」檔案路徑、推薦取代老的 `os.path`。

```text
from pathlib import Path

# === 建立 Path 物件 ===
p = Path("data/users.txt")

# === 屬性（不用真的去看檔、純字串操作）===
p.name              # "users.txt"          檔名（含副檔名）
p.stem              # "users"              檔名（不含副檔名）
p.suffix            # ".txt"               副檔名
p.parent            # PosixPath('data')    父資料夾

# === 檢查 ===
p.exists()          # True / False、存不存在
p.is_file()         # 是不是一個檔案
p.is_dir()          # 是不是一個資料夾

# === 讀 / 寫（簡單版、一行解決）===
content = p.read_text(encoding="utf-8")
p.write_text("hello", encoding="utf-8")

# === Binary（圖片 / pdf 等非文字檔）===
data = p.read_bytes()
p.write_bytes(b"binary data")           # b"..." = bytes 字串

# === 路徑運算（用 / 串接、超直觀）===
data_dir = Path("data")
user_file = data_dir / "users.txt"      # 等於 "data/users.txt"
# pathlib 重載了 / 運算符、不用手動接斜線
```

  
**列資料夾 + 找檔案：**

```text
# === 列當前資料夾所有檔案 / 子資料夾 ===
for f in data_dir.iterdir():
    print(f)

# === glob（找符合 pattern 的檔、* = 任意字串）===
for f in data_dir.glob("*.json"):
    print(f)

# === rglob（recursive glob、含所有子資料夾）===
for f in data_dir.rglob("*.py"):
    print(f)

# === 建資料夾 ===
new_dir = Path("output")
new_dir.mkdir(exist_ok=True)                       # 已存在不報錯
new_dir.mkdir(parents=True, exist_ok=True)         # 連父資料夾也建（深層巢狀）

# === 刪除 ===
p.unlink()              # 刪檔
new_dir.rmdir()         # 刪空資料夾（不能刪有東西的）
```

  
### 環境變數（.env、處理機密資料）

**機密資訊不該寫進程式碼**（會 commit 進 git、推到 GitHub 就全世界看得到）。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📄 **建 .env 檔**（在專案根目錄、**不要** commit）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
DATABASE_URL=postgresql://user:pass@localhost/db
API_KEY=sk-abc123
SECRET=xyz
```

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**（裝讀 .env 的套件）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
uv add python-dotenv
```

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📄 **寫進程式檔 main.py**（從 .env 讀環境變數）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
from dotenv import load_dotenv
import os

load_dotenv()                              # 把 .env 內容載入 os.environ

db_url = os.environ["DATABASE_URL"]        # 必須有、沒有會 KeyError
api_key = os.getenv("API_KEY", "default")  # 沒有就用預設值（更安全）
```

  
**`.gitignore` 一定要排除 `.env`：**

```text
.env                # 機密、絕對不上 git
.venv/              # 虛擬環境
__pycache__/        # Python 編譯快取
*.pyc
```

  
**做給其他開發者用的範本 `.env.example`（這個要 commit）：**

```text
DATABASE_URL=postgresql://...
API_KEY=your_key_here
```

  
    **⚠ 踩雷警告**
    
新手 3 大檔案 I/O 坑：(1) **不用 `with`** → 忘記 close、檔案被 lock 住、之後動不了 (2) **讀中文沒加 `encoding="utf-8"`** → Windows 跳出一堆 `UnicodeDecodeError` 或亂碼 (3) **把 API key / 密碼直接寫進程式碼** → 推 GitHub 立刻被 GitGuardian / 機器人爬走、半夜被刷帳單。**3 個對策**：永遠用 `with` 開檔、永遠寫 `encoding="utf-8"`、機密一律放 `.env` 並加進 `.gitignore`。**SaaS 公司半夜帳單爆炸** = 99% 是 API key commit 到 GitHub 被偷。

  

  
    
      🎯
      面試常考
    
    台灣科技公司常問 Python 檔案 I/O：(1) 為什麼用 `with` 而不直接 `open` + `close`（答：with 是 context manager、出錯也保證 close） (2) 處理大檔怎麼做（答：一行一行 for line in f、不要 read 整個檔到記憶體） (3) JSON 中文亂碼怎麼解（答：`ensure_ascii=False`）。**準備時不用背 API、用做過的小專案（讀 CSV / 寫 JSON log）當例子最有說服力**。
