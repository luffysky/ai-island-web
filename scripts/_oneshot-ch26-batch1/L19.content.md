「**怎麼知道 model 好不好？**」—— 看錯指標 = model 上線後爆。**accuracy 只是一個指標、且很多場景不準**——這課教你看哪一個。

  
### 混淆矩陣（Confusion Matrix、所有指標的源頭）

**confusion matrix** = 4 個格子、把預測 vs 實際的 4 種組合算次數：

```text
                  預測：正常       預測：詐欺
實際：正常          TN              FP（誤報、False Positive）
實際：詐欺          FN（漏抓）      TP（True Positive）

4 個名詞：
- TP（True Positive、真陽性）：實際是詐欺、預測也是詐欺 ✓
- TN（True Negative、真陰性）：實際正常、預測也正常 ✓
- FP（False Positive、偽陽性、誤報）：實際正常、預測說詐欺（冤枉好人）
- FN（False Negative、偽陰性、漏抓）：實際詐欺、預測說正常（放跑壞人）
```

```text
from sklearn.metrics import confusion_matrix
cm = confusion_matrix(y_test, y_pred)
print(cm)
# [[TN  FP]
#  [FN  TP]]
```

  
### Accuracy（準確率）——直觀但常騙人

```text
accuracy = (TP + TN) / 全部
       = 「對的比例」
```

> ⚠️ **不平衡資料看 accuracy 沒意義**：
> ```
> 999 筆正常 + 1 筆詐欺
> Model「全猜正常」→ accuracy = 99.9%
> 但實際上 1 筆詐欺一個也沒抓到 → model 沒用！
> ```

  
### Precision（精準率）——「我說是、有多準」

```text
precision = TP / (TP + FP)
        = 「**預測為詐欺**的之中、真的是詐欺的比例」
        = 念作：「我說 X、有幾個真的是 X」

高 precision = 不誤報、說是就是
低 precision = 經常冤枉好人

何時優先看 precision？
→ 垃圾信過濾（誤把正常信標垃圾、使用者會抓狂）
→ 推薦系統（推給用戶的要準）
→ 信用卡盜刷（誤鎖正常交易、使用者投訴）
```

  
### Recall（召回率）——「真的是、我抓到幾個」

```text
recall = TP / (TP + FN)
      = 「**實際是詐欺**的之中、我抓到的比例」
      = 念作：「真的是 X 的、我有幾個抓到」

高 recall = 不漏抓、有就抓
低 recall = 漏掉很多

何時優先看 recall？
→ 癌症 / 重病檢測（漏抓 = 病人沒治療 = 出人命）
→ 詐欺偵測（漏抓 = 真的詐欺得逞）
→ 安全事件（漏抓 = 真的被駭）
```

  
**Precision vs Recall 是 trade-off（互相拉扯）：**

```text
=== 嚴格判斷（高門檻才說是詐欺）===
→ 高 precision（說是就是）
→ 低 recall（漏掉一些邊緣 case）

=== 寬鬆判斷（稍微像就標詐欺）===
→ 低 precision（誤報多）
→ 高 recall（不漏抓）

→ 兩者不能同時高（除非 model 神到爆）
→ 業務決定該偏哪邊
```

  
### F1 Score（precision 跟 recall 的調和平均）

當「兩個都重要、不想偏」時、看 F1：

```text
F1 = 2 * (precision * recall) / (precision + recall)

→ F1 不是普通平均（算術平均）、是「調和平均」
→ 任何一邊低、F1 就拉很低
→ 用法：迫使你「兩個都要不錯」
```

  
**程式碼一次取得 4 個指標：**

```text
from sklearn.metrics import (
    accuracy_score, precision_score, recall_score, f1_score
)

accuracy_score(y_test, y_pred)
precision_score(y_test, y_pred)
recall_score(y_test, y_pred)
f1_score(y_test, y_pred)
```

  
### ROC AUC（不分 threshold 的綜合分數）

**threshold（閾值）** 是什麼？

```text
model 通常輸出「機率」（如 0.73 表示「73% 機會是詐欺」）
你要設一個門檻（threshold）決定「機率多少以上算詐欺」
- threshold = 0.5（預設）：機率 ≥ 50% 算詐欺
- threshold = 0.3：放寬、recall ↑、precision ↓
- threshold = 0.8：收緊、precision ↑、recall ↓

→ 不同 threshold 給不同 precision / recall
→ ROC AUC 是「不分 threshold、整體有多好」的指標
```

**ROC 曲線**：在不同 threshold 下、畫「TPR（真陽率、= recall）vs FPR（偽陽率、誤報率）」。

**AUC（Area Under Curve、ROC 曲線下面積）：**

```text
AUC = 0.5  → model 跟亂猜一樣
AUC = 1.0  → model 完美
AUC > 0.8  → 算不錯
AUC > 0.9  → 很好
```

**AUC 的 3 大優勢：**

```text
✓ 跟 threshold 無關（不用先決定門檻）
✓ 不平衡資料友善（不會被多數類別主導）
✓ 比較不同 model 公平
```

  
```text
from sklearn.metrics import roc_auc_score

# === 取得「機率」（不是「類別」）===
y_pred_proba = model.predict_proba(X_test)[:, 1]
# predict_proba 回傳每個類別的機率
# [:, 1] = 取「類別 1」（如「詐欺」）的機率

auc = roc_auc_score(y_test, y_pred_proba)
print(f"AUC: {auc:.3f}")
```

  
### 完整分類報告（一行看全部）

```text
from sklearn.metrics import classification_report

print(classification_report(y_test, y_pred))
```

**典型輸出：**

```text
              precision    recall  f1-score   support
           0       0.95      0.98      0.96       900
           1       0.78      0.65      0.71       100

    accuracy                           0.94      1000
   macro avg       0.86      0.81      0.84      1000
weighted avg       0.93      0.94      0.93      1000
```

**怎麼看？**
- 每個類別都有獨立的 precision / recall / f1
- `support` 是每個類別的實際樣本數
- `macro avg` = 直接平均（每類權重相等）
- `weighted avg` = 按 support 加權（多數類別影響大）
- **不平衡資料看 macro avg**（公平）、看 weighted avg 會被主導

  
### 回歸指標（房價 / 氣溫這類「預測數字」的場景）

**MAE（Mean Absolute Error、平均絕對誤差）：**

```text
MAE = mean(|預測 - 實際|)

✓ 最直觀（誤差平均幾元 / 度）
✓ 對 outlier（極端值）不敏感

from sklearn.metrics import mean_absolute_error
mae = mean_absolute_error(y_test, y_pred)
# 例：預測房價平均差 15 萬
```

  
**MSE / RMSE（均方誤差 / 均方根誤差）：**

```text
MSE = mean((預測 - 實際)²)
RMSE = sqrt(MSE)   ← 跟原單位一致、好讀

特性：
- 把「大誤差」放大（平方）→ 懲罰偶爾爆掉的預測
- 對 outlier 敏感

from sklearn.metrics import mean_squared_error
rmse = mean_squared_error(y_test, y_pred, squared=False)
# squared=False 直接回 RMSE、不用自己 sqrt
```

  
**R² Score（決定係數）：**

```text
R² = 「model 解釋了多少變異」
   = 0 → 跟「全部用平均值預測」一樣爛
   = 1.0 → 完美預測
   > 0.7 → 算不錯
   < 0 → 比平均還爛、model 有問題

from sklearn.metrics import r2_score
r2 = r2_score(y_test, y_pred)
```

  
### 選對指標：場景對照表

| 場景 | 主要指標 |
|---|---|
| 平衡分類（類別數量差不多） | accuracy |
| **不平衡分類（一邊很少）** | **F1 / AUC**（accuracy 沒用） |
| 不要漏抓（醫療 / 詐欺 / 安全） | **recall 優先** |
| 不要誤報（垃圾信 / 推薦） | **precision 優先** |
| 房價預測 / 數值預測 | **RMSE / MAE** |
| 排名（推薦系統） | NDCG / MAP（這課沒教、L46 AI/ML 章再講） |

  
    **🚀 進階技巧**
    
評估指標 3 大新手坑：(1) **不平衡看 accuracy** —— 99% 正常時全猜正常就 99% 準確率、看似很好但完全沒抓到少數類別 (2) **只看一個指標** —— precision / recall **永遠成對看**、單看 precision 你不知道是不是漏一堆。(3) **用 train 的指標** —— train accuracy 99% 可能是 overfit（L20 詳教）、一定要看 test set 指標。**業界主流**：分類看 F1 + AUC、回歸看 RMSE + R²、報告時兩個都列。
