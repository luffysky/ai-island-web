「**Garbage in, garbage out**」（垃圾進、垃圾出）—— 資料品質決定 model 好壞。**特徵工程（feature engineering）是 ML 60% 的工作**、其實沒花時間訓練 model。

> 💡 **這課 4 大主題**：(1) 切 train/test (2) 處理缺失值 (3) 類別轉數字 (4) 數值縮放 + pipeline。

  
### train / test 切分（為什麼要切？）

**為什麼？** 你要驗證 model「**在沒看過的資料**上表現如何」、不能拿訓練用的資料考它。

```text
from sklearn.model_selection import train_test_split

X_train, X_test, y_train, y_test = train_test_split(
    X, y,
    test_size=0.2,                          # 測試集 20%（訓練集 80%）
    random_state=42,                        # 固定亂數種子、結果可重現
    stratify=y                              # 保持類別比例（重要！）
)
```

> 💡 **`stratify=y` 的意義**：如果原資料 70% 類別 A、30% 類別 B、`stratify=y` 保證 train / test 都是 70/30 比例、不會「test 全是類別 A」。

  
**進階：train / val / test 三切（大專案建議）：**

```text
# === 第一刀：分出 20% 給 test（最後考試用、不能偷看）===
X_temp, X_test, y_temp, y_test = train_test_split(X, y, test_size=0.2)

# === 第二刀：剩下的再分 25%（= 原 20%）給 val（調參用）===
X_train, X_val, y_train, y_val = train_test_split(X_temp, y_temp, test_size=0.25)

# 結果：60% train / 20% val / 20% test
# train：訓練 model
# val：調超參數
# test：最終評估（碰一次就好）
```

  
### 處理缺失值（NaN）

```text
# === 看哪裡缺 ===
df.isnull().sum()                           # 每欄缺幾個

# === 方式 1：刪 ===
df = df.dropna()                            # 任何一欄缺都刪
df = df.dropna(subset=["age"])              # 只看 age 欄

# === 方式 2：填 ===
df["age"].fillna(df["age"].median(), inplace=True)    # 填中位數（穩定、不被 outlier 影響）
df["city"].fillna("Unknown", inplace=True)             # 字串填預設

# === 方式 3：KNN 預測缺值（進階、用相似的鄰居預測）===
from sklearn.impute import KNNImputer
imputer = KNNImputer(n_neighbors=5)         # 找最近 5 個鄰居算平均
X_filled = imputer.fit_transform(X)
```

> 💡 **怎麼選？** (1) 缺得少（< 5%）→ 直接刪 (2) 缺得不多 + 重要欄位 → 填中位數 / 眾數 (3) 缺很多 + 重要 → KNN 或專屬模型預測。

  
### 類別 → 數字（model 只吃數字、字串要轉）

**Label Encoding（標籤編碼）—— 適合「有順序」的類別：**

```text
from sklearn.preprocessing import LabelEncoder

le = LabelEncoder()
df["size_encoded"] = le.fit_transform(df["size"])
# small / medium / large → 0 / 1 / 2
# ✓ 適合：T-shirt 尺寸、學歷高低、評等
# ✗ 不適合：城市 / 顏色（沒順序、變數字會被 model 誤解成有大小）
```

  
**One-Hot Encoding（單熱編碼）—— 適合「沒順序」的類別：**

```text
df = pd.get_dummies(df, columns=["city", "category"])
# city = Taipei → city_Taipei=1、city_Tokyo=0、city_Seoul=0
# 每個類別一個欄位、只有對應的是 1、其他是 0
```

> ⚠️ **One-Hot 缺點**：類別太多（如「100 個城市」）會爆欄位、用 Target Encoding 更好。

  
**Target Encoding（目標編碼）—— 高基數類別必用：**

🖥️ **在終端機輸入**（裝 category_encoders）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
uv add category-encoders
```

**這行做什麼？** 裝 `category-encoders` 套件、含多種類別變數編碼方式（Target / Catboost / James-Stein 等）、sklearn 不內建這些進階編碼。

```text
import category_encoders as ce

encoder = ce.TargetEncoder()
df["city_enc"] = encoder.fit_transform(df["city"], y)
# 每個 city 的編碼 = 該 city 對應 y 的平均
# 例：Taipei → 平均訂單金額 1500、就編碼成 1500
```

  
### 數值縮放（Scaling、讓所有特徵在同尺度）

**為什麼要 scale？**

```text
特徵 1：年齡（0-100）
特徵 2：薪資（20000-200000）

→ 不 scale 的話、薪資數字大、會「主導（dominate）」model 學習
→ 因為 model 看到的「距離」大部分來自薪資、年齡幾乎沒影響
→ scale 後兩個都在同尺度、公平比較
```

  
**3 種 scaler：**

```text
# === StandardScaler（z-score 標準化、最常用）===
# 減平均、除標準差 → 平均 0、標準差 1
from sklearn.preprocessing import StandardScaler
scaler = StandardScaler()

# === ⚠️ 重要：先 split、再 fit on train ===
scaler.fit(X_train)                         # 只用 train 算「平均跟標準差」
X_train_scaled = scaler.transform(X_train)
X_test_scaled = scaler.transform(X_test)    # 用 train 算的去 transform test
# 不能 fit 整個 X（會 data leakage、test 的資訊偷偷進 train）

# === MinMaxScaler（壓縮到 [0, 1]）===
from sklearn.preprocessing import MinMaxScaler
# 適合：神經網路、需要明確界限的演算法

# === RobustScaler（對 outlier 穩定）===
from sklearn.preprocessing import RobustScaler
# 適合：資料有極端值的情況、用中位數跟 IQR、不被極端值影響
```

  
**哪些 model 需要 scale？**

| Model | 要 scale 嗎 |
|---|---|
| Linear / Logistic Regression | ✓ 要 |
| SVM | ✓ 要 |
| KNN | ✓ 要 |
| Neural Network | ✓ 要 |
| Tree-based（DecisionTree / RandomForest / XGBoost / LightGBM） | ✗ 不用（樹分支不在乎絕對值大小） |

  
### 創建新特徵（feature creation、ML 的藝術部分）

**從日期挖出多個特徵：**

```text
df["date"] = pd.to_datetime(df["date"])     # 字串轉成 datetime 型別
df["year"] = df["date"].dt.year
df["month"] = df["date"].dt.month
df["dayofweek"] = df["date"].dt.dayofweek   # 0=週一、6=週日
df["is_weekend"] = df["dayofweek"] >= 5     # 週末旗標
# 同一個 date 欄擴展成 5 個新特徵、model 學得到「週末效應」
```

  
**比例 / 衍生 / 交互特徵：**

```text
# === 比例（兩欄相除）===
df["price_per_sqft"] = df["price"] / df["sqft"]       # 每坪單價

# === 交互（兩欄相乘、捕捉組合效應）===
df["age_x_income"] = df["age"] * df["income"]

# === 聚合特徵（groupby + agg）===
user_stats = df.groupby("user_id").agg({
    "amount": ["mean", "max", "count"],     # 每個 user 的平均、最大、總次數
})
# 帶回去主表當「該 user 的歷史行為」特徵
```

  
### Pipeline（避免 data leakage 的最好方式）

**Pipeline** = 「**把多步驟串成一條管線**」、自動處理 `fit` 跟 `transform` 順序、避免人為錯誤。

```text
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.ensemble import GradientBoostingClassifier

pipeline = Pipeline([
    ("scaler", StandardScaler()),                       # 步驟 1：標準化
    ("model", GradientBoostingClassifier())             # 步驟 2：訓練
])

pipeline.fit(X_train, y_train)              # 自動：scaler.fit_transform → model.fit
y_pred = pipeline.predict(X_test)           # 自動：scaler.transform → model.predict
# 完全避免 leakage、scaler 不會看到 test 資料
```

  
### 不平衡資料（imbalanced data、ML 常見坑）

**問題：** 99% 正常交易、1% 詐欺——**全猜「正常」就 99% 準確率、但完全沒抓到詐欺**、實際上 model 沒用。

**4 種對策：**

```text
# === 對策 1：stratify 切分（保持 train/test 比例一致）===
train_test_split(X, y, stratify=y)

# === 對策 2：SMOTE oversample（人工合成少數類別樣本）===
# Synthetic Minority Over-sampling Technique
from imblearn.over_sampling import SMOTE
smote = SMOTE()
X_resampled, y_resampled = smote.fit_resample(X_train, y_train)
# 1% 詐欺 → 變 50%、平衡資料
# ⚠️ 只能對 train 用、絕對不能對 test 用

# === 對策 3：class_weight（model 內建調權重）===
model = RandomForestClassifier(class_weight="balanced")
# model 自動把少數類別的權重調高

# === 對策 4：看對的評估指標（最重要）===
# 不要看 accuracy（會騙人）
# 看 precision / recall / F1 / AUC（L19 詳教）
```

    **🚀 進階技巧**
    
2026 特徵工程 5 大原則：(1) **切 train/test 後才做特徵工程** —— 避免 leakage、scaler / encoder 只 fit train (2) **用 Pipeline** —— 自動化、避免人為錯誤 (3) **類別 encoding 看 cardinality（基數）** —— 2-5 類用 one-hot、10+ 類用 target encoding (4) **不平衡資料看 F1 / AUC 不看 accuracy** (5) **新特徵驗證有用**——加進去看 score 變化、沒提升就拿掉。

  

  
    
      ⚠️
      新手常見錯誤
    
    特徵工程 3 大新手坑：(1) **Data leakage（資料洩漏）** —— scaler / encoder 對整個 X fit、test 資訊偷進 train、線下 95% 上線 60%。**永遠用 Pipeline + 只 fit train**。 (2) **One-hot 高基數欄位** —— 100 個城市做 one-hot → 100 個欄位、記憶體爆 + model 學不動。**改用 Target Encoding**。 (3) **不平衡資料看 accuracy** —— 99% vs 1% 不平衡、model 全猜多數就 99% 準、看起來很好但完全沒用。**看 F1 / Precision / Recall / AUC**（L19 詳教）。
