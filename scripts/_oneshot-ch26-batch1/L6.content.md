**function（函式）** 是 Python 抽象的基本單位——把一段程式打包、取個名字、之後可以重複呼叫。學熟函式 = Python 進入中階。

> 💡 **這課全部在 REPL 試最快**。函式定義完馬上呼叫看結果。

  
### 基本函式

```text
# === 定義 function ===
def greet(name):              # def = define、宣告一個叫 greet 的函式、收一個參數 name
    return f"Hello, {name}"   # return = 把結果送回去

print(greet("Alice"))         # Hello, Alice

# 名詞釐清：
# - parameter（參數）= 函式定義時的「name」這個名字
# - argument（引數）= 呼叫時實際傳進去的值、例如 "Alice"
# - 中文常混用、不用太糾結

# === 多參數 ===
def add(a, b):
    return a + b

print(add(3, 5))              # 8
```

  
**預設值（default value）：**

```text
# 給參數預設值、呼叫時可以省略
def greet(name, greeting="Hello"):
    return f"{greeting}, {name}"

print(greet("Alice"))              # "Hello, Alice"（greeting 用預設）
print(greet("Bob", "Hi"))          # "Hi, Bob"（覆蓋預設）
```

  
**關鍵字參數（keyword argument、位置不重要）：**

```text
def create_user(name, age, role="user", active=True):
    return {"name": name, "age": age, "role": role, "active": active}

# 方式 1：全靠位置（按定義順序）
create_user("Alice", 30)

# 方式 2：部分用關鍵字（指定參數名）
create_user("Alice", 30, role="admin")

# 方式 3：全部用關鍵字（順序隨便）
create_user(role="admin", name="Alice", age=30)
```

> 💡 **2026 慣例**：超過 3 個參數一律用關鍵字呼叫、可讀性差很多。`create_user(name="Alice", age=30)` 比 `create_user("Alice", 30)` 一眼就看懂。

  
**多回傳值（其實就是回傳一個 tuple）：**

```text
def divide(a, b):
    return a // b, a % b        # 回 (商, 餘)、Python 自動包成 tuple

q, r = divide(10, 3)            # 解包成 2 個變數
print(q, r)                     # 3 1

# 或回 dict、欄位名更清楚
def get_user_stats():
    return {"total": 100, "active": 80, "premium": 20}

stats = get_user_stats()
print(stats["total"])           # 100
```

  
### *args 跟 **kwargs（不定長度的參數）

當你不知道呼叫者會傳幾個參數時、用 `*args` 跟 `**kwargs`：

- **`*args`** —— 收所有「位置參數」、打包成 tuple
- **`**kwargs`** —— 收所有「關鍵字參數」、打包成 dict
- 「args / kwargs」是慣例名字、可以自取（但業界都這樣寫）

```text
# === *args 收所有位置參數 ===
def sum_all(*numbers):           # 不管傳幾個、都會塞進 numbers 這個 tuple
    return sum(numbers)

sum_all(1, 2, 3)                 # 6
sum_all(1, 2, 3, 4, 5)           # 15

# === **kwargs 收所有關鍵字參數 ===
def create(**fields):
    print(fields)

create(name="Alice", age=30, role="admin")
# {'name': 'Alice', 'age': 30, 'role': 'admin'}

# === 混合用（順序固定：一般、*args、一般帶預設、**kwargs）===
def func(a, b, *args, **kwargs):
    print(f"a={a}, b={b}, args={args}, kwargs={kwargs}")

func(1, 2, 3, 4, 5, x=10, y=20)
# a=1, b=2, args=(3, 4, 5), kwargs={'x': 10, 'y': 20}
```

  
**unpacking（解包、反過來把 list / dict 攤開傳進函式）：**

```text
# === 用 * 把 list 攤開成位置參數 ===
nums = [1, 2, 3]
sum_all(*nums)                   # 等於 sum_all(1, 2, 3)

# === 用 ** 把 dict 攤開成關鍵字參數 ===
data = {"name": "Alice", "age": 30}
create(**data)                   # 等於 create(name="Alice", age=30)
```

  
**常見 pattern：「包一層」decorator 雛形：**

```text
# 寫一個會「在原函式前後加事情」的包裝器
def log_call(func):
    def wrapper(*args, **kwargs):
        print(f"Calling {func.__name__}")   # 呼叫前印 log
        return func(*args, **kwargs)        # 原樣呼叫
    return wrapper                          # 回傳包裝過的新函式
# 這就是 decorator（裝飾器）的雛形、L11 會詳教
```

  
### Lambda（匿名函式、無名小型函式）

**lambda** = 一行寫完的「臨時小函式」、適合「只用一次、不值得取名」的場景。

```text
# === 基本寫法：「lambda 參數: 表達式」===
square = lambda x: x ** 2
print(square(5))                 # 25

# 多參數
add = lambda a, b: a + b

# === 最常用：搭配 sorted / map / filter ===
nums = [3, 1, 4, 1, 5, 9, 2, 6]

# 降冪排序（用 -x 反轉大小關係）
sorted(nums, key=lambda x: -x)
# [9, 6, 5, 4, 3, 2, 1, 1]

# 依「字典欄位」排序
users = [{"name": "Alice", "age": 30}, {"name": "Bob", "age": 25}]
sorted(users, key=lambda u: u["age"])    # 依年齡排
```

  
**對照 map / filter（內建變形 / 篩選函式）：**

```text
# map（對每個元素做變形）
squared = list(map(lambda x: x ** 2, nums))

# filter（篩選符合條件的）
evens = list(filter(lambda x: x % 2 == 0, nums))

# === 但 2026 主流：用 comprehension 更 pythonic ===
squared = [x ** 2 for x in nums]                  # 比 map 自然
evens = [x for x in nums if x % 2 == 0]           # 比 filter 自然
```

  
> ⚠️ **Lambda 限制 + 反模式：**
> - **只能放一個表達式**（不能有 `if/elif`、不能多行）
> - **不要用 lambda 取代 def**：
>   ```text
>   # ❌ 不好（沒名字、難 debug、不能寫 docstring）
>   square = lambda x: x ** 2
>   
>   # ✓ 好（有名字、可加說明、可多行擴充）
>   def square(x):
>       return x ** 2
>   ```
> - 如果你「想給 lambda 取個名字賦值」、直接用 def 就好

  
### Type Hints（型別提示、2024+ 主流）

**type hint** 之前 L2 / L3 有介紹過——標註參數跟回傳的型別。

```text
# === 沒 type hint（仍能跑、但難維護）===
def add(a, b):
    return a + b

# === 有 type hint（推薦）===
def add(a: int, b: int) -> int:    # -> int 表示「回傳 int」
    return a + b
```

  
**複雜型別：**

```text
from typing import Optional, Union   # Python 3.8 以下要 import

# Optional[X] = X 或 None
def get_user(id: int) -> Optional[dict]:
    """回傳 user dict 或 None。"""
    return user_dict.get(id)

# === Python 3.9+ 用內建型別（不用 import）===
def process(items: list[str]) -> dict[str, int]:
    return {item: len(item) for item in items}

# === Python 3.10+ 用 | 替代 Union（更簡潔）===
def parse(x: int | str) -> int:
    return int(x)

# Optional[X] 等於 X | None
def find(id: int) -> "User | None":      # 字串引號是因為 User 可能還沒定義
    ...
```

  
**Type hint 不影響執行**——只給 3 個對象看：
1. 你自己 / 同事（當文件）
2. IDE / 編輯器（VS Code 自動完成）
3. **type checker（型別檢查器）** —— mypy 或 pyright、會幫你抓出型別不對的 bug

### Docstring（文件字串、寫給人看的函式說明）

**docstring** = 寫在函式或類別**第一行**的字串、Python 認得、`help()` 可以印出來。

```text
def calculate_tax(price: float, rate: float = 0.05) -> float:
    """計算稅金。
    
    Args:
        price: 商品原價
        rate: 稅率、預設 5%
    
    Returns:
        稅金金額
    
    Examples:
        >>> calculate_tax(100)
        5.0
        >>> calculate_tax(100, 0.1)
        10.0
    """
    return price * rate

# === 看說明 ===
help(calculate_tax)
# 印出上面整段 docstring
```

**主流 docstring 格式**：Google / NumPy / Sphinx 三種。**Google 最普及**（範例就是 Google 格式）。

  
### Scope（作用域、變數的有效範圍）

**scope** = 「這個變數在哪些地方看得到」。Python 有 4 層：local（函式內）→ enclosing（外層函式）→ global（檔案層）→ built-in（內建）。

```text
x = 10                     # 全域變數（global、整個 .py 都看得到）

def func():
    y = 20                 # 區域變數（local、只在 func 內看得到）
    print(x)               # ✓ 看得到外面的 x → 10
    print(y)               # ✓ 看得到自己的 y → 20

func()
print(x)                   # 10
print(y)                   # ❌ NameError（y 出函式就消失）
```

  
**想在函式內改全域變數、要用 `global` 宣告：**

```text
x = 10

def modify():
    global x               # 告訴 Python：我要改的是全域那個 x
    x = 99

modify()
print(x)                   # 99（改了）

# 沒寫 global 的話：
def modify():
    x = 99                 # 這會建一個「新的區域 x」、不影響全域
```

  
**Closure（閉包）—— 函式內的函式、能記住外面的變數：**

```text
def make_counter():
    count = 0
    def increment():
        nonlocal count     # nonlocal = 「我要改的是外層函式裡的 count」
        count += 1
        return count
    return increment       # 回傳「會記住 count 的函式」

counter = make_counter()
counter()                  # 1
counter()                  # 2
counter()                  # 3
# count 在 make_counter 外部不可見、但 counter 內部還記得
```

  
    **🚀 進階技巧**
    
Python 函式 2026 寫法：(1) **加 type hint** —— 好讀好維護 (2) **寫 docstring** —— `help()` 看得到、給未來的自己看 (3) **少用 lambda** —— 只放「真的一行解決」的場景 (4) **關鍵字參數 > 位置參數** —— `create_user(name="Alice", age=30)` 比 `create_user("Alice", 30)` 清楚很多 (5) **一個函式做一件事** —— 超過 20 行就考慮拆。寫熟函式 = 寫好 Python 的關鍵。

  

  
    
      ⚠️
      新手常見錯誤
    
    Python 函式 3 大新手坑：(1) **不看 Traceback（錯誤訊息）** —— Python 錯誤的最後一行通常就是答案（例如 `TypeError: greet() missing 1 required positional argument: 'name'` 就是「忘了傳 name 參數」）、不要直接放棄、看完再求救 (2) **改完 .py 沒重新跑** —— 改了程式碼必須回終端機**重新** `uv run app.py`、舊的執行不會自己更新 (3) **以為函式裡改變數會影響外面** —— Python 預設「函式內賦值都是建區域變數」、想改外面要用 `global` 或 `nonlocal`、踩這坑你不是第一個。**記得：先看懂 Traceback、再求救**。
