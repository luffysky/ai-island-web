**OOP（Object-Oriented Programming、物件導向程式設計）** 在 Python——用 `class`（類別）把「**資料 + 行為**」綁在一起。

> 💡 **3 個關鍵字先記**：
> - **class（類別）** = 物件的「設計藍圖」
> - **instance（實例）** = 用藍圖造出來的「具體物件」
> - **method（方法）** = 物件「自己會做的事」（例如 `dog.bark()`）

  
### 基本 class

```text
class Dog:
    # === 類別屬性（class attribute、所有 instance 共享）===
    species = "Canis lupus familiaris"
    
    # === 建構子（constructor、創建 instance 時自動呼叫）===
    def __init__(self, name, age):
        self.name = name              # 實例屬性（instance attribute）
        self.age = age
    
    # === 方法（method）===
    def bark(self):
        print(f"{self.name} 汪汪叫")
    
    def info(self):
        return f"{self.name} ({self.age} 歲)"

# === 創建 instance（叫 class 像呼叫函式）===
lucky = Dog("Lucky", 3)
ruby = Dog("Ruby", 5)

# === 用 instance ===
lucky.bark()                  # Lucky 汪汪叫
print(lucky.info())           # Lucky (3 歲)
print(lucky.species)          # Canis lupus familiaris（從 class 拿）

# === 改屬性 ===
lucky.age = 4

# === Python 特色：可以動態加屬性（沒在 class 定義也行）===
lucky.color = "白"
# ⚠️ 不建議！容易亂、debug 困難、type checker 也抓不到
```

  
### self 是什麼

**`self`** = 「呼叫這個方法的 instance 自己」。其他語言（Java / JavaScript）叫 `this`、Python 明確要寫出來。

```text
class Calculator:
    def __init__(self, initial=0):
        self.value = initial          # 設這個 instance 自己的 value
    
    def add(self, n):
        self.value += n               # 改這個 instance 自己的 value
        return self                   # 回傳自己（讓呼叫可以「鏈式串接」）

c1 = Calculator(0)
c2 = Calculator(100)

c1.add(5).add(10)                    # c1.value = 15（不影響 c2）
c2.add(5).add(10)                    # c2.value = 115（不影響 c1）
# 兩個 instance 各有自己的 value、不共享
```

**為什麼 Python 要明寫 `self`、不像 Java 的 `this` 是隱式？**
> Python 哲學：**Explicit is better than implicit（明示勝於暗示）**。你看 `def bark(self):` 就立刻知道「這是 instance method」、不用猜。

  
### 繼承（inheritance）

「**繼承**」= 「新 class 沿用舊 class 的功能、再加自己的東西」。可以避免重複寫一樣的程式碼。

```text
class Animal:                         # 父類別（base / parent）
    def __init__(self, name):
        self.name = name
    
    def eat(self):
        print(f"{self.name} 在吃")
    
    def speak(self):
        print(f"{self.name} 發出聲音")

class Dog(Animal):                    # Dog 繼承 Animal、寫成 class Dog(Animal):
    def speak(self):                  # === override（覆寫）父的同名方法 ===
        print(f"{self.name} 汪汪叫")
    
    def fetch(self):                  # === 自己新加的方法 ===
        print(f"{self.name} 撿球")

class Cat(Animal):
    def speak(self):
        print(f"{self.name} 喵喵叫")
    
    def purr(self):
        print(f"{self.name} 呼嚕")

# === 用 ===
d = Dog("Lucky")
d.eat()                               # ✓ 繼承自 Animal（不用自己寫）
d.speak()                             # Dog 自己 override 的版本：「汪汪叫」
d.fetch()                             # Dog 獨有

c = Cat("Mimi")
c.speak()                             # 喵喵叫
c.purr()                              # 呼嚕
```

  
**`super()`：呼叫父類別的方法（不要重寫一遍）：**

```text
class Puppy(Dog):
    def __init__(self, name, age):
        super().__init__(name)        # 呼叫父（Dog）的 __init__、設好 name
        self.age = age                # 再加自己的 age
    
    def speak(self):
        super().speak()               # 先做父的「汪汪叫」
        print("(小聲版)")              # 再加自己的特色

p = Puppy("Lucky", 1)
p.speak()
# Lucky 汪汪叫
# (小聲版)
```

  
### 特殊方法（Dunder Methods、雙底線方法）

**Dunder method（dunder = double underscore）** = 名字前後都是雙底線的方法（`__xxx__`）、Python 的「**鉤子**」——讓你的物件「長得像 Python 內建型別」、可以用 `+`、`==`、`len()`、`print()` 直接操作。

```text
class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    
    # === print(v) 時呼叫（給使用者看的字串）===
    def __str__(self):
        return f"({self.x}, {self.y})"
    
    # === repr(v) 時呼叫（給開發者 debug 看的字串、要能還原）===
    def __repr__(self):
        return f"Vector({self.x}, {self.y})"
    
    # === v1 + v2 時呼叫 ===
    def __add__(self, other):
        return Vector(self.x + other.x, self.y + other.y)
    
    # === v1 == v2 時呼叫 ===
    def __eq__(self, other):
        return self.x == other.x and self.y == other.y
    
    # === len(v) 時呼叫 ===
    def __len__(self):
        import math
        return int(math.sqrt(self.x ** 2 + self.y ** 2))
    
    # === v[0]、v[1] 時呼叫（讓物件可以用 [] 取值）===
    def __getitem__(self, idx):
        return [self.x, self.y][idx]

# === 用起來像內建型別 ===
v1 = Vector(3, 4)
v2 = Vector(1, 2)

print(v1)                             # (3, 4)            ← 呼叫 __str__
print(v1 + v2)                        # (4, 6)            ← 呼叫 __add__
print(v1 == Vector(3, 4))             # True              ← 呼叫 __eq__
print(len(v1))                        # 5                 ← 呼叫 __len__
print(v1[0])                          # 3                 ← 呼叫 __getitem__
```

  
**常用 dunder 一覽表：**

| Dunder | 觸發時機 |
|---|---|
| `__init__` | 建構子、`Dog(...)` 時呼叫 |
| `__str__` | `print(obj)` / `str(obj)` |
| `__repr__` | `repr(obj)` / REPL 直接打物件名 |
| `__eq__` | `==` 比較 |
| `__lt__` / `__gt__` | `<` / `>` 比較 |
| `__add__` / `__sub__` / `__mul__` | `+` / `-` / `*` |
| `__len__` | `len(obj)` |
| `__iter__` | `for x in obj` |
| `__contains__` | `x in obj` |
| `__getitem__` | `obj[i]` |
| `__call__` | `obj()`（讓 instance 變成可呼叫、像函式） |

  
### dataclass：簡化 class（強烈推薦純資料用這個）

**dataclass** = Python 3.7+ 內建、自動幫你生 `__init__` / `__repr__` / `__eq__`、寫純資料 class 省 80% 樣板程式碼。

**傳統寫法（樣板程式碼多）：**

```text
class User:
    def __init__(self, name, age, email):
        self.name = name
        self.age = age
        self.email = email
    
    def __repr__(self):
        return f"User(name={self.name!r}, age={self.age}, email={self.email!r})"
    
    def __eq__(self, other):
        return (self.name == other.name 
                and self.age == other.age 
                and self.email == other.email)
```

**dataclass 寫法（一個 decorator 全包）：**

```text
from dataclasses import dataclass

@dataclass
class User:
    name: str                        # type hint 是必填、dataclass 靠它認欄位
    age: int
    email: str

# 自動有 __init__、__repr__、__eq__！
u = User("Alice", 30, "alice@x.com")
print(u)
# User(name='Alice', age=30, email='alice@x.com')
```

  
**帶預設值 + `__post_init__`（建構完額外處理）：**

```text
from dataclasses import dataclass, field

@dataclass
class Product:
    name: str
    price: float = 0.0
    stock: int = 0
    tags: list = field(default_factory=list)   # ⚠️ list 不能直接寫 = []
    
    def __post_init__(self):
        # 建構子跑完後額外做的事
        if self.price < 0:
            raise ValueError("價格不能負")
```

> ⚠️ **mutable default 陷阱**：dataclass 的 list / dict 預設值**不能寫 `= []`**！因為「所有 instance 會共享同一個 list」（地雷）。要用 `field(default_factory=list)`。

  
**frozen=True：建立後不可改（像 tuple、但有名字）：**

```text
@dataclass(frozen=True)
class Coord:
    x: int
    y: int

c = Coord(3, 4)
c.x = 99      # ❌ FrozenInstanceError
# 適合：當 dict key、確定不會改的設定
```

  
### Property（屬性裝飾器、讓「方法看起來像屬性」）

有時候你想「**讀的時候像屬性、但底下有檢查邏輯**」——用 `@property`。

```text
class Circle:
    def __init__(self, radius):
        self._radius = radius        # 慣例：_ 開頭表示「內部用、外部別直接碰」
    
    @property                        # 讓 radius 看起來像屬性、但其實是 method
    def radius(self):
        return self._radius
    
    @radius.setter                   # 設定時有檢查
    def radius(self, value):
        if value < 0:
            raise ValueError("半徑不能負")
        self._radius = value
    
    @property                        # 計算屬性（基於別的屬性算）
    def area(self):
        import math
        return math.pi * self._radius ** 2

c = Circle(5)
print(c.radius)        # 5         ← 看起來像屬性、其實呼叫 radius() method
print(c.area)          # 78.54...  ← 同理、即時計算
c.radius = 10          # OK
c.radius = -1          # ❌ ValueError: 半徑不能負
```

  
    **🚀 進階技巧**
    
2026 Python OOP 心法：(1) **dataclass > 手寫 class**——純資料 class 必用 (2) **type hint 必加**——`name: str, age: int` 又好讀又能讓 IDE 幫你 (3) **不要過度繼承**——超過 2 層繼承通常設計有問題 (4) **用 composition（組合）多過 inheritance（繼承）**——「has-a」比「is-a」彈性、可以隨時換零件 (5) **`@property` 取代手寫 getter/setter**——更 pythonic。**Pydantic**（後面 L26 會教）是 dataclass 進階版、含驗證、是寫 API 主流。
