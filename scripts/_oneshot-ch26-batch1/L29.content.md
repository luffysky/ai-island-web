Python 後端認證實戰——3 大件事：**密碼 hash + JWT + OAuth 第三方登入**。

> ⚠️ **認證是資安頭號漏洞**——這課每個範例都要看懂、不要抄不懂。**新手建議直接用 Clerk / Supabase Auth**（見最後段）、不要自己搞。

  
### 密碼 hash（用 bcrypt、永遠不要存明文）

**核心原則**：**永遠不要存使用者密碼的「原文」**。要存「hash 過的版本」、即使 DB 被駭、攻擊者也拿不到原密碼。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**（裝密碼 hash 套件）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
uv add passlib[bcrypt]
```

**這行做什麼？逐個解釋：**
- `passlib` —— Python 密碼 hash 標準套件（封裝多種演算法）
- `[bcrypt]` —— extras（額外依賴）、告訴 uv 多裝 bcrypt 套件（passlib 的後端）
- **bcrypt** 是 2026 主流密碼 hash 演算法（防暴力破解、計算成本可調）

  
```text
from passlib.context import CryptContext

# === 全域設定（schemes=支援的演算法、deprecated=自動處理舊版）===
pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")

# === 註冊時：hash 密碼存 DB ===
hashed = pwd_context.hash("user_password_123")
# 結果類似：$2b$12$Eix7t.../xxxxxxxxxx（含鹽 salt、無法反推）
# 存進 DB 的 users.hashed_password 欄位

# === 登入時：驗證 ===
is_valid = pwd_context.verify("user_password_123", hashed)
# True 表示密碼正確、False 錯誤
```

> 💡 **為什麼用 bcrypt 不用 MD5 / SHA-1？** MD5 / SHA-1 算太快、攻擊者每秒能猜上億組。bcrypt 故意算很慢（每次 100-300ms）、防暴力破解。

  
### JWT 完整實作（FastAPI 後端標配）

**JWT（JSON Web Token）** = 「**加密簽名過的字串、塞使用者身份**」——前端登入後拿到 token、之後每個請求帶這個 token、後端解開驗證身份、不用每次查 DB。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**（裝 JWT 套件）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
uv add python-jose[cryptography]
```

**這行做什麼？** 裝 `python-jose`、Python 處理 JWT 標準套件、`[cryptography]` extras 是底層加密庫。

  
```text
from datetime import datetime, timedelta, timezone
from jose import jwt, JWTError
from fastapi import Depends, HTTPException, status
from fastapi.security import OAuth2PasswordBearer, OAuth2PasswordRequestForm

# === 設定（SECRET_KEY 必須放 .env、不能寫在 code）===
SECRET_KEY = os.environ["JWT_SECRET_KEY"]     # 從 .env 讀
ALGORITHM = "HS256"                            # HMAC-SHA256、最常用
ACCESS_TOKEN_EXPIRE_MINUTES = 30

# === OAuth2 scheme（從 Authorization: Bearer xxx 拿 token）===
oauth2_scheme = OAuth2PasswordBearer(tokenUrl="token")

# === 生成 JWT ===
def create_access_token(data: dict, expires_delta: timedelta = None):
    to_encode = data.copy()
    expire = datetime.now(timezone.utc) + (expires_delta or timedelta(minutes=15))
    to_encode.update({"exp": expire})         # exp = 過期時間（JWT 標準欄位）
    return jwt.encode(to_encode, SECRET_KEY, algorithm=ALGORITHM)
```

  
**登入 endpoint（拿 email + 密碼、回 token）：**

```text
@app.post("/token")
async def login(
    form_data: OAuth2PasswordRequestForm = Depends(),   # 自動解析 form data
    db: AsyncSession = Depends(get_db)
):
    # === 1. 找 user ===
    result = await db.execute(
        select(User).where(User.email == form_data.username)
    )
    user = result.scalar_one_or_none()
    
    # === 2. 驗證密碼 ===
    if not user or not pwd_context.verify(form_data.password, user.hashed_password):
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail="Incorrect email or password"
            # ⚠️ 故意不告訴攻擊者「是 email 錯還是密碼錯」、防 user enumeration
        )
    
    # === 3. 生 token ===
    access_token = create_access_token(
        data={"sub": str(user.id)},           # sub = subject、JWT 標準欄位
        expires_delta=timedelta(minutes=ACCESS_TOKEN_EXPIRE_MINUTES)
    )
    
    return {"access_token": access_token, "token_type": "bearer"}
```

  
**驗證 token 的 dependency（L27 教過 Depends）：**

```text
async def get_current_user(
    token: str = Depends(oauth2_scheme),
    db: AsyncSession = Depends(get_db)
) -> User:
    credentials_exception = HTTPException(
        status_code=status.HTTP_401_UNAUTHORIZED,
        detail="Could not validate credentials"
    )
    
    # === 1. 解 JWT、過期 / 簽名錯都會 raise ===
    try:
        payload = jwt.decode(token, SECRET_KEY, algorithms=[ALGORITHM])
        user_id = payload.get("sub")
        if user_id is None:
            raise credentials_exception
    except JWTError:
        raise credentials_exception
    
    # === 2. 查 DB 確認 user 還存在 ===
    user = await db.get(User, int(user_id))
    if user is None:
        raise credentials_exception
    return user

# === 用：任何要登入才能用的 endpoint 加 current_user = Depends(get_current_user) ===
@app.get("/me")
async def read_me(current_user: User = Depends(get_current_user)):
    return current_user
```

  
### Refresh Token Pattern（短期 access + 長期 refresh）

**問題**：access token 設太長（如 30 天）→ token 被偷風險久；設太短（如 15 分鐘）→ 使用者頻繁登入很煩。

**解法**：**雙 token 設計**：
- **access token**：短期（15 分鐘）、每個 API 請求帶
- **refresh token**：長期（30 天、存 DB 可撤銷）、access 過期時用來換新 access

```text
from pydantic import BaseModel

class TokenPair(BaseModel):
    access_token: str
    refresh_token: str
    token_type: str = "bearer"

@app.post("/token", response_model=TokenPair)
async def login(form_data: OAuth2PasswordRequestForm = Depends(), db: AsyncSession = Depends(get_db)):
    # ... 驗證 user 略 ...
    
    # === 短期 access token ===
    access_token = create_access_token(
        data={"sub": str(user.id)},
        expires_delta=timedelta(minutes=15)
    )
    
    # === 長期 refresh token、加 type 區分 ===
    refresh_token = create_access_token(
        data={"sub": str(user.id), "type": "refresh"},
        expires_delta=timedelta(days=30)
    )
    
    # === 把 refresh token 存進 DB（之後可以撤銷）===
    db.add(RefreshToken(token=refresh_token, user_id=user.id))
    await db.commit()
    
    return TokenPair(access_token=access_token, refresh_token=refresh_token)
```

  
**換新 access token 的 endpoint：**

```text
@app.post("/refresh", response_model=TokenPair)
async def refresh(refresh_token: str, db: AsyncSession = Depends(get_db)):
    # === 1. 解 refresh token ===
    try:
        payload = jwt.decode(refresh_token, SECRET_KEY, algorithms=[ALGORITHM])
        if payload.get("type") != "refresh":   # 確保不是拿 access token 來偽冒
            raise HTTPException(401)
        user_id = payload.get("sub")
    except JWTError:
        raise HTTPException(401)
    
    # === 2. 檢查 refresh token 沒被撤銷（DB 還在）===
    result = await db.execute(
        select(RefreshToken).where(RefreshToken.token == refresh_token)
    )
    if not result.scalar_one_or_none():
        raise HTTPException(401, "Refresh token revoked")
    
    # === 3. 生新 access token ===
    new_access = create_access_token(
        data={"sub": user_id},
        expires_delta=timedelta(minutes=15)
    )
    return TokenPair(access_token=new_access, refresh_token=refresh_token)
```

  
### OAuth 第三方登入（用 Google / LINE / GitHub 登入）

**OAuth 流程**：

```text
1. 使用者點「用 Google 登入」
2. 後端把 user 導到 Google 授權頁
3. user 在 Google 同意授權
4. Google 把 user 導回後端的 callback URL、附 authorization code
5. 後端用 code 跟 Google 換 access token
6. 後端用 access token 拿 user info（email / name）
7. 找 / 建 user、生自己的 JWT 給前端
```

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**（裝 Authlib + httpx）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
uv add authlib httpx
```

**這 2 個套件做什麼？**
- `authlib` —— OAuth 客戶端套件、支援 Google / GitHub / Facebook / LINE 等
- `httpx` —— authlib 內部用來呼叫 OAuth provider（L27 教過）

  
**完整實作：**

```text
from authlib.integrations.starlette_client import OAuth
from starlette.requests import Request
from starlette.responses import RedirectResponse

oauth = OAuth()
oauth.register(
    name="google",
    client_id=os.environ["GOOGLE_CLIENT_ID"],        # 從 Google Cloud Console 拿
    client_secret=os.environ["GOOGLE_CLIENT_SECRET"],
    server_metadata_url="https://accounts.google.com/.well-known/openid-configuration",
    client_kwargs={"scope": "openid email profile"}
)

# === 1. 開始登入（導到 Google）===
@app.get("/auth/google")
async def google_login(request: Request):
    redirect_uri = request.url_for("google_callback")
    return await oauth.google.authorize_redirect(request, redirect_uri)

# === 2. Google 導回來的 callback ===
@app.get("/auth/google/callback")
async def google_callback(request: Request, db: AsyncSession = Depends(get_db)):
    token = await oauth.google.authorize_access_token(request)
    user_info = token.get("userinfo")          # email / name / sub（Google user id）
    
    # === 3. 找 / 建 user ===
    result = await db.execute(
        select(User).where(User.email == user_info["email"])
    )
    user = result.scalar_one_or_none()
    
    if not user:                                # 第一次 OAuth、自動建帳號
        user = User(
            email=user_info["email"],
            name=user_info["name"],
            google_id=user_info["sub"]
        )
        db.add(user)
        await db.commit()
        await db.refresh(user)
    
    # === 4. 生自己的 JWT、給前端 ===
    access_token = create_access_token(data={"sub": str(user.id)})
    return RedirectResponse(f"/?token={access_token}")
```

  
### 強烈建議 indie：直接用 Clerk / Supabase Auth / Auth.js

**自己搞 auth 是地雷區**——密碼 hash / email 驗證 / 重設 / 2FA / OAuth / Session 管理、每個都可能踩坑被駭。**用 SaaS 處理**最安全。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**（裝 Clerk Python SDK）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
uv add clerk-backend-api
```

**這行做什麼？** 裝 Clerk 後端 SDK、Clerk 是 2026 最熱的 auth-as-a-service、前端 + 後端 + DB schema 全包。

  
```text
from clerk_backend_api import Clerk

clerk = Clerk(bearer_auth=os.environ["CLERK_SECRET_KEY"])

async def get_current_user_from_clerk(
    token: str = Depends(oauth2_scheme)
):
    try:
        session = clerk.sessions.verify(token=token)
        user = clerk.users.get(user_id=session.user_id)
        return user
    except Exception:
        raise HTTPException(401)

# 用 Clerk 完全不用自己處理：
# - 密碼 hash / 重設 / 強度驗證
# - Email 驗證 / SMS 驗證
# - OAuth（Google / GitHub / Facebook / LINE / Apple 全包）
# - 2FA（Two-Factor Authentication、Authenticator App / SMS）
# - Session 管理 / 撤銷
# - 後台管理介面
```

    **⚠ 踩雷警告**
    
Auth 是頭號資安漏洞、5 大鐵則：(1) **絕對不存明文密碼** —— 用 bcrypt / argon2、不要用 MD5 / SHA-1 / SHA-256（這些都太快、防不了暴力破解） (2) **JWT SECRET_KEY 放 `.env`** —— 絕對不寫死在 code、不上 Git (3) **HTTPS only** —— 沒 HTTPS = token 被 wifi 竊聽 (4) **HttpOnly cookie 或 sessionStorage** —— 別放 localStorage（防 XSS 竊取） (5) **新手直接用 Clerk / Supabase Auth / Auth.js** —— 別自己搞、auth 的坑你想不到的多。**Stripe / OpenAI 這種大公司都不自己搞 auth、用 Auth0 或自家專業團隊**——你不是這種等級就乖乖用 SaaS。
