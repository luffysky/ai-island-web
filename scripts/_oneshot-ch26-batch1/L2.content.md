### 變數：給東西取名字

**variable（變數）** —— 給一個值取的名字、方便之後使用。像「幫一杯飲料貼標籤」、之後說「拿那杯叫 latte 的」就找得到。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📄 **寫進程式檔**（在 VS Code 編輯器裡打、任何 .py 都行）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
# 第一個變數：給值取名字
name = "Alice"
age = 30
is_student = False

# 印出來看
print(name)        # Alice
print(age)         # 30
print(is_student)  # False

# 多個變數一次解決
x, y, z = 1, 2, 3
print(x, y, z)     # 1 2 3

# 交換變數（Python 特色、別的語言要寫 3 行）
a = 5
b = 10
a, b = b, a
print(a, b)        # 10 5
# 不用 temp（臨時暫存用的）變數！

# 同時指派同一個值
x = y = z = 0
print(x, y, z)     # 0 0 0
```

**重點觀念：**
- `=` 在程式裡**不是「等於」**、是「**指派**」（assign）：
  `name = "Alice"` 念作「把 "Alice" 這個值、指派給叫 name 的盒子」。
- 上面 `#` 開頭那些句子叫 **comment（註解）** —— 寫給人看的說明文字、電腦執行時會跳過、不會當成程式跑。
- `temp` 是 temporary（暫時的）的縮寫、程式界常用來指「臨時用一下、用完就丟」的變數。

  
### Python 是動態型別

**type（型別）** —— 資料的種類（數字、文字、真假值……）。
**動態型別**（dynamic typing） —— 一個變數的型別可以隨時改變、不用事先宣告「這變數只能放數字」。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📄 寫進程式檔

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
# 同一個變數、型別會自動跟著值變
x = 5         # int（整數）
x = "hello"   # 變成 str（字串、文字）
x = [1, 2, 3] # 變成 list（清單）
x = True      # 變成 bool（布林值、真假值）

# 看當前型別
print(type(x))   # <class 'bool'>
```

  
**2024 年後主流會加 type hint（型別提示）：**

**type hint（型別提示）** —— 在變數或參數旁邊**標註「這裡應該放什麼型別的資料」**、像貼一張標籤。不強制寫、但寫了更清楚。

```text
age: int = 30          # 標註 age 是 int（整數）
name: str = "Alice"    # 標註 name 是 str（字串）
is_active: bool = True # 標註 is_active 是 bool（真假值）
```

**type hint 不影響程式執行**、只是給 3 個對象看：
1. **你自己看**（當文件、提醒這個變數放什麼）
2. **IDE / 編輯器看**（VS Code 可以靠它做自動完成）
3. **type checker（型別檢查器）看** —— 一種會幫你檢查「型別有沒有用錯」的工具、例如 mypy 或 pyright

```text
age: int = 30
age = "thirty"   # 仍然能跑（Python 動態）、但 type checker 會抗議：
                 # 「你說 age 是 int、怎麼塞字串給它？」
```

  
### 命名規則 + 慣例

**規則**（不遵守會直接錯）：

```text
1. 只能用「英文字母 / 數字 / 底線」
   ✓ user_name
   ✗ user-name（- 是減號、不是名字的一部分）
   ✗ 2nd_user（不能數字開頭）
   ✓ user2

2. 大小寫敏感
   age ≠ Age ≠ AGE  ← 這是三個不同的變數

3. 不能用「保留字」
   **保留字（reserved word）** = Python 自己預留要用的關鍵字、不能拿來當名字
   保留字有：if, else, for, while, def, class, import, return, ...
   ✗ class = "Math"        # 錯！class 是保留字
   ✓ class_name = "Math"   # 可以
```

**慣例**（依 **PEP 8（Python Enhancement Proposal #8、Python 官方推薦的程式風格指南）**）：

```text
- 變數 / 函式 → snake_case（蛇形命名、全小寫、單字之間用底線）
  user_name
  calculate_tax

- 常數 → SCREAMING_SNAKE（全大寫、單字之間用底線）
  MAX_RETRY = 5
  API_KEY = "..."

- 類別 → PascalCase（每個單字首字母大寫、不加底線）
  class UserAccount: ...
  class HttpClient: ...

- 私密成員（慣例） → 底線開頭
  _internal_var = ...
  __very_private = ...
```

→ Python **不強制**這些風格、但業界都這樣寫
→ 寫錯風格 = 同事 code review 一眼看出你是新手

  
### 輸入：input()——問使用者打字

`input()` —— Python 內建的「**問使用者一個問題、等他打字、按 Enter 後拿到他打的內容**」函式。

> 👇 **這段是整課最常被卡住的地方**、Nami 大人的回饋也是從這裡開始。仔細看每個介面切換。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📄 **寫進程式檔 hello.py**（VS Code 編輯器、寫完按存檔）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
name = input("你叫什麼名字？ ")
print(f"嗨 {name}！")
```

**這兩行在做什麼？**
- `input("你叫什麼名字？ ")` 的作用是「向使用者問問題、等他打字」：
  - 括號裡的 `"你叫什麼名字？ "` 是程式要問的**問題**、會顯示在螢幕上。
  - 使用者打完字按 Enter、他打的東西會被存進左邊那個 `name` 變數裡。
- `print(f"嗨 {name}！")` 用 **f-string（格式化字串、formatted string）** 印出回應：
  - 引號前面那個 `f` 是 f-string 的標記。
  - `{name}` 會在執行時、**自動換成 name 變數裡實際存的字**。

age = input("幾歲？ ")

age = int(input("幾歲？ "))

nums = input("輸入 3 個數字（空白隔開）：").split()

nums = [int(x) for x in input().split()]
print(sum(nums))

### 輸出：print 進階

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📄 寫進程式檔（或在 REPL 直接試）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
# 基本
print("hello")
print("hello", "world")          # hello world（多個值自動用空白隔開）
print("hello", "world", sep="-") # hello-world（sep 是分隔符號）

# 不換行（end 預設是 \n 換行、改成空字串就不換行）
print("hello", end="")
print("world")        # helloworld（兩段印在同一行）

# f-string（2026 主流的字串拼接寫法、引號前加 f）
name = "Alice"
age = 30
print(f"{name} is {age} years old")    # Alice is 30 years old

# {} 裡可以放表達式（不只變數、還可以算東西）
print(f"明年是 {age + 1} 歲")           # 明年是 31 歲

# 格式化數字（: 後面是「顯示規則」）
pi = 3.14159265
print(f"π = {pi:.2f}")               # π = 3.14    ← .2f = 小數點 2 位
print(f"π = {pi:.4f}")               # π = 3.1416  ← .4f = 小數點 4 位

# 千分位逗號
big = 1234567
print(f"{big:,}")                    # 1,234,567

# 補 0（讓數字寬度固定、適合編號）
n = 5
print(f"{n:03d}")                    # 005    ← 3 位數、不夠補 0

# 顯示成百分比
ratio = 0.756
print(f"{ratio:.1%}")                # 75.6%
```

→ **f-string** 是 Python 3.6+ 神器、2026 業界主流、必用

  
### 註解

**comment（註解）** —— 寫在程式裡、給人看的**說明文字**。電腦執行時會跳過、不會當成程式跑。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📄 寫進程式檔

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
# 單行註解：井字號開頭、整行被略過
# 這行不會執行、純粹寫給人看

x = 5  # 行尾註解：也可以放在程式碼後面

"""
多行註解（嚴格說這是個字串、放在這裡沒人接、Python 會略過）
但社群慣例：當「多行 comment」用、
或當 docstring（文件字串）用。
"""

def greet(name):
    """這是 docstring（文件字串）。
    
    寫在函式或類別**開頭**的一段說明文字、
    用來講「這段程式是做什麼用的」、給人看的。
    跟一般 comment 的差別：docstring 是 Python 認得的、
    用 help(函式名) 可以看到它印出來。
    """
    return f"Hello, {name}"

help(greet)  # 會印出上面那段 docstring
```

→ 用註解解釋「**為什麼這樣寫**」、不是「這段在做什麼」
→ Code 自己會說「做什麼」、註解負責說「為什麼」

  
    **☕ 用人話講**
    
Python 30 秒上手：(1) **print** 印東西到螢幕 (2) **變數**就是給東西取名字（`name = "Alice"`） (3) **f-string**（引號前加 f、`{變數}` 自動換成值）是 2026 主流的字串拼接寫法 (4) **input()** 問使用者打字、拿到的永遠是字串、要算數要先 `int()` 轉 (5) **type()** 看當前型別。**練習方式**：開 REPL（在終端機打 `python`）+ 隨便試。打錯不會壞電腦、放心試。
