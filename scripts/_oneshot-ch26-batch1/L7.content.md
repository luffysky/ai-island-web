把 code 拆成多個檔案、用別人寫好的 **package（套件、library）** —— 大型專案必備能力。

> 💡 **3 個關鍵字先記住**：
> - **module（模組）** = 一個 `.py` 檔
> - **package（套件）** = 一個含 `__init__.py` 的資料夾、裡面裝多個 module
> - **library（程式庫）** = 別人寫好、可下載來用的 package（口語常跟 package 混用）

  
### 什麼是 module（模組）

**module** = 一個 `.py` 檔。你寫好就能在別的 `.py` 檔用 `import` 拿來用。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📄 寫進程式檔 math_utils.py（自己的模組）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
def add(a, b):
    return a + b

def multiply(a, b):
    return a * b

PI = 3.14159
```

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📄 寫進程式檔 main.py（同個資料夾）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
# === 方式 1：整包匯入 ===
import math_utils
print(math_utils.add(2, 3))       # 5
print(math_utils.PI)              # 3.14159

# === 方式 2：匯入並改名（暱稱）===
import math_utils as mu
mu.add(2, 3)
# 業界慣例：numpy → np、pandas → pd、matplotlib.pyplot → plt

# === 方式 3：只匯入需要的（推薦、最清楚）===
from math_utils import add, PI
add(2, 3)
print(PI)

# === 方式 4：全部匯入（不推薦）===
from math_utils import *
# 為什麼不推薦：你不知道從哪來、容易跟其他套件命名衝突
```

  
### 標準函式庫（Python 自帶、不用裝）

**standard library（標準函式庫）** = Python 安裝時就自帶的 module、不用 `uv add` 就能 `import`。

```text
# === math：數學函式 ===
import math
math.pi                # 3.141592653589793
math.sqrt(16)          # 4.0
math.floor(3.7)        # 3（無條件捨去）
math.ceil(3.2)         # 4（無條件進位）

# === random：亂數 ===
import random
random.random()                    # 0 ~ 1 之間隨機浮點
random.randint(1, 10)              # 1 ~ 10 隨機整數（含頭含尾）
random.choice(["a", "b", "c"])     # 隨機選一個
random.shuffle([1, 2, 3, 4])       # 原地洗牌（會改原 list）

# === datetime：時間日期 ===
from datetime import datetime, timedelta

now = datetime.now()
print(now)             # 2026-05-26 14:30:00.123456
print(now.year)        # 2026

# 加減時間（用 timedelta）
tomorrow = now + timedelta(days=1)
last_week = now - timedelta(weeks=1)

# 格式化字串
now.strftime("%Y-%m-%d %H:%M")     # "2026-05-26 14:30"
# %Y = 4 位年、%m = 月、%d = 日、%H = 時、%M = 分

# 字串解析回 datetime
datetime.strptime("2026-05-26", "%Y-%m-%d")
```

  
**JSON（資料交換格式、最常用）：**

```text
import json

# === Python dict ↔ JSON 字串 ===
data = {"name": "Alice", "age": 30}
json_str = json.dumps(data)           # dumps = dump string、轉成 JSON 字串
parsed = json.loads(json_str)         # loads = load string、解析 JSON 字串

# === 直接讀寫 JSON 檔 ===
with open("data.json", "w") as f:
    json.dump(data, f, indent=2)      # 寫入檔案、indent 排版美

with open("data.json") as f:
    data = json.load(f)               # 從檔案讀回來
```

  
**os 跟 pathlib（操作檔案系統）：**

```text
# === os：傳統做法 ===
import os
os.environ.get("API_KEY")             # 取環境變數
os.listdir(".")                       # 列當前資料夾的檔案
os.path.exists("file.txt")            # 檢查檔案存不存在

# === pathlib：2024+ 推薦、更現代、更物件導向 ===
from pathlib import Path

p = Path("data.txt")
p.exists()                            # 檢查
p.read_text()                         # 讀整個檔（一行解決）
p.write_text("hello")                 # 寫整個檔
p.parent                              # 父資料夾
p.suffix                              # ".txt"
p.with_suffix(".md")                  # 換副檔名
```

  
**re（正規表示式、regex、字串模式比對）：**

```text
import re
# 從一段文字找出所有「連續數字」
re.findall(r"\d+", "今天 25 度、明天 28 度")
# ['25', '28']
# r"..." 是 raw string（L3 教過、反斜線不會被當 escape）
# \d+ = 一個以上的數字字元
```

  
### 第三方套件：用 uv 裝

**third-party package（第三方套件）** = 別人寫好、發佈到 PyPI（Python Package Index、官方套件倉庫）的 package。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**（裝 requests、HTTP 客戶端、最常用）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
uv add requests
```

> 💡 跑完 `pyproject.toml` 會自動加：
> ```
> [project]
> dependencies = [
>     "requests>=2.32.0",
> ]
> ```

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📄 寫進程式檔（用裝好的 requests）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
import requests

response = requests.get("https://api.github.com/users/torvalds")
data = response.json()                # 把回應當 JSON 解析
print(data["name"])                   # "Linus Torvalds"
```

  
**其他常用 uv 指令：**

```text
# === 一次裝多個（後端開發常見組合）===
uv add fastapi uvicorn sqlalchemy pydantic

# === 開發專用（測試 / lint、不會上 production）===
uv add --dev pytest ruff mypy

# === 從別人的專案同步依賴 ===
git clone someproject
cd someproject
uv sync                               # 讀 pyproject.toml + uv.lock、裝齊所有套件

# === 升級某個套件 ===
uv lock --upgrade-package requests

# === 移除套件 ===
uv remove requests
```

> 💡 **uv.lock 是什麼？** 鎖定**所有依賴的精確版本**（含次要依賴的依賴）。**一定要 commit 進 Git**、保證團隊每個人裝出來都是一樣的版本、不會因為「我電腦正常你電腦壞」吵架。

### 專案結構

**小專案（一個 .py 就解決）：**

```text
my-project/
├── .python-version       # Python 版本（uv 自動建）
├── pyproject.toml        # 專案設定、依賴清單
├── uv.lock               # 鎖定精確版本
├── README.md
├── .gitignore
└── main.py               # 你的主程式
```

  
**中型專案（拆 package）：**

```text
my-app/
├── pyproject.toml
├── uv.lock
├── src/
│   └── myapp/
│       ├── __init__.py          # 讓 myapp 變成 package
│       ├── main.py
│       ├── models.py
│       ├── api/
│       │   ├── __init__.py      # 讓 api 變成 sub-package
│       │   ├── users.py
│       │   └── products.py
│       └── utils.py
├── tests/                       # 測試獨立資料夾
│   ├── __init__.py
│   ├── test_users.py
│   └── test_api.py
├── .env                         # 環境變數（API key 之類、不上 git）
├── .gitignore
└── README.md
```

> 💡 **`__init__.py` 是什麼？** 一個空檔案、放在資料夾內就告訴 Python「這資料夾是一個 package」。沒這檔、`import myapp.api.users` 會失敗。

  
**Import 寫法：**

```text
# === 相對 import（用點點代表往上層）===
# 在 api/users.py 想用同 package 的 utils.py：
from ..utils import helper       # .. = 往上一層

# === 絕對 import（推薦、清楚）===
from myapp.utils import helper   # 從 package root 開始寫
```

  
### Virtual Environment（虛擬環境）回顧

**為什麼要 venv？**

```text
- 專案 A 用 Django 4
- 專案 B 用 Django 5
- 如果全裝在「系統 Python」= 兩個版本互相覆蓋、衝突地獄

解法：每個專案用「自己獨立的套件空間」（venv）
```

  
**uv 自動處理 venv、不用你管：**

```text
uv init                       # 建專案
uv add requests               # 自動建 .venv、裝在裡面
uv run main.py                # 自動用 .venv 內的 Python 跑
```

  
**傳統方式（仍要懂、有些公司在用）：**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ 在終端機輸入

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
python -m venv .venv               # 建 venv
source .venv/bin/activate          # 啟動（Mac / Linux）
.venv\Scripts\activate             # 啟動（Windows）
# 啟動後 prompt 前會有 (.venv) 字樣

pip install requests               # 裝
python main.py                     # 跑

deactivate                         # 退出 venv
```

  
**.gitignore 一定要排除：**

```text
.venv/             # venv 整個資料夾（裝的東西、太大）
__pycache__/       # Python 編譯後的 bytecode 快取
*.pyc              # 編譯後的單檔
.env               # 環境變數（含 API key 之類敏感資料）
```
