**Pydantic** 是 2026 Python **頭號資料驗證 library**——FastAPI / LangChain / Hugging Face / OpenAI SDK / Anthropic SDK 全用。**寫 Python 不用 Pydantic 等於沒寫**。

> 💡 **Pydantic 2 個核心能力**：(1) **資料驗證**（傳進來的 dict 對不對） (2) **型別轉換**（字串 "30" 自動轉 int 30）。

  
### 為什麼用 Pydantic

**沒驗證的痛苦：**

```text
def create_user(data: dict):
    name = data["name"]                       # ⚠️ key 不存在會 KeyError
    age = int(data["age"])                    # ⚠️ "abc" 會 ValueError
    email = data["email"]                     # ⚠️ 不知道格式對不對
# 每個欄位都要手動檢查、寫 20 行驗證、還容易漏
```

  
**用 Pydantic：3 行解決全部：**

```text
from pydantic import BaseModel, EmailStr

class User(BaseModel):
    name: str
    age: int
    email: EmailStr                            # 內建 email 格式驗證

user = User(name="Alice", age=30, email="alice@x.com")
# ✓ 自動驗證型別 + email 格式
# ✗ 任一不對自動 raise ValidationError、含詳細錯誤訊息
```

### 基本用法

```text
from pydantic import BaseModel
from typing import Optional

class Product(BaseModel):
    name: str
    price: float
    stock: int = 0                            # 預設值
    description: Optional[str] = None         # 可以是 None
    tags: list[str] = []                      # 預設空 list

# === 建立 instance（直接傳參數）===
p = Product(name="iPhone", price=999.99, stock=10)

# === 從 dict 建（用 ** 解包、API 收到 JSON 時最常用）===
data = {"name": "iPhone", "price": "999.99"}  # 注意 price 是字串
p = Product(**data)
print(p.price, type(p.price))                  # 999.99 <class 'float'>
# Pydantic 自動把 "999.99" 轉成 float 999.99（vs Python 內建會 TypeError）

# === 型別不能轉就 raise ===
Product(name="iPhone", price="not a number")
# ❌ ValidationError: price must be a number
```

  
### 進階型別 + 驗證（內建很多現成型別）

```text
from pydantic import BaseModel, EmailStr, HttpUrl, Field
from datetime import datetime

class User(BaseModel):
    # === 數值範圍限制（用 Field）===
    age: int = Field(ge=0, le=120)            # ge = 大於等於、le = 小於等於
    
    # === 字串長度限制 ===
    username: str = Field(min_length=3, max_length=20)
    
    # === Email 格式（內建驗證）===
    email: EmailStr
    
    # === URL 格式 ===
    website: HttpUrl
    
    # === 日期時間 ===
    created_at: datetime                       # 可以從 ISO 字串自動解析
    
    # === List ===
    tags: list[str] = []
    
    # === 巢狀 model ===
    address: "Address"                         # 字串引號 = forward reference

class Address(BaseModel):
    street: str
    city: str
    country: str
```

  
**Pydantic 內建超多現成型別：**

| 型別 | 用途 |
|---|---|
| `EmailStr` | Email 格式 |
| `HttpUrl` / `AnyUrl` | URL 格式 |
| `IPvAnyAddress` | IPv4 / IPv6 |
| `UUID4` | UUID v4 |
| `PositiveInt` / `NegativeInt` / `NonNegativeInt` | 數字範圍 |
| `SecretStr` / `SecretBytes` | 機密字串（log / repr 自動遮罩） |
| `Json` | 自動 parse JSON 字串 |

  
### 自訂驗證（複雜邏輯用 validator decorator）

```text
from pydantic import BaseModel, field_validator, model_validator

class User(BaseModel):
    username: str
    password: str
    password_confirm: str
    age: int
    
    # === field_validator：驗單一欄位 ===
    @field_validator("username")
    @classmethod
    def username_alphanumeric(cls, v):
        if not v.isalnum():
            raise ValueError("使用者名稱必須是字母 + 數字")
        return v                              # 一定要 return（可以順便正規化）
    
    @field_validator("age")
    @classmethod
    def age_must_be_positive(cls, v):
        if v < 0:
            raise ValueError("年齡不能負")
        return v
    
    # === model_validator：驗整個 model（多欄位互動）===
    @model_validator(mode="after")
    def passwords_match(self):
        if self.password != self.password_confirm:
            raise ValueError("兩次密碼不一致")
        return self
```

  
### 序列化（model 變 dict / JSON、API 回傳用）

```text
user = User(name="Alice", age=30, email="alice@x.com")

# === 變 dict ===
user.model_dump()
# {"name": "Alice", "age": 30, "email": "alice@x.com"}

# === 變 JSON 字串 ===
user.model_dump_json()
# '{"name": "Alice", "age": 30, "email": "alice@x.com"}'

# === 排除某些欄位（敏感資料）===
user.model_dump(exclude={"password"})

# === 只要某些欄位 ===
user.model_dump(include={"name", "email"})
```

  
**配 FastAPI（response model、自動文件生成）：**

```text
from fastapi import FastAPI

class UserResponse(BaseModel):
    id: int
    name: str
    email: str
    # 注意：沒 password 欄位、自動從 response 排除

@app.get("/users/{id}", response_model=UserResponse)
def get_user(id: int) -> UserResponse:
    return db.find(id)                        # 自動過濾掉非 UserResponse 的欄位
```

  
### Computed Field（衍生欄位）

```text
from pydantic import BaseModel, computed_field

class User(BaseModel):
    first_name: str
    last_name: str
    
    @computed_field
    @property
    def full_name(self) -> str:
        return f"{self.first_name} {self.last_name}"

u = User(first_name="Alice", last_name="Chen")
print(u.full_name)                            # "Alice Chen"
u.model_dump()
# {"first_name": "Alice", "last_name": "Chen", "full_name": "Alice Chen"}
# computed_field 也會被序列化進去
```

  
### Generic Model（泛型、寫通用 response wrapper）

```text
from typing import Generic, TypeVar
from pydantic import BaseModel

T = TypeVar("T")

# === 通用 API 回應格式（很多公司都這樣定）===
class ApiResponse(BaseModel, Generic[T]):
    data: T                                   # 實際資料
    message: str = "success"
    code: int = 200

# === 用：任何 model 都能塞進去 ===
ApiResponse[User](data=user)
# {"data": {...user...}, "message": "success", "code": 200}

ApiResponse[list[User]](data=[user1, user2])
# {"data": [{...}, {...}], "message": "success", "code": 200}
```

  
### Discriminated Union（多型、根據欄位選 model）

```text
from typing import Literal
from pydantic import BaseModel, Field

class Cat(BaseModel):
    pet_type: Literal["cat"]                  # Literal = 只能是這個值
    meow_loudness: int

class Dog(BaseModel):
    pet_type: Literal["dog"]
    bark_loudness: int

class PetOwner(BaseModel):
    pet: Cat | Dog = Field(discriminator="pet_type")
    # Pydantic 根據 pet_type 自動選用 Cat 還是 Dog

# === 用 ===
PetOwner(pet={"pet_type": "cat", "meow_loudness": 5})    # ✓ 變 Cat
PetOwner(pet={"pet_type": "dog", "bark_loudness": 7})    # ✓ 變 Dog
PetOwner(pet={"pet_type": "fish"})                       # ❌ ValidationError
```

**這行做什麼？** 裝 `pydantic-settings`、Pydantic 官方的「環境變數 → settings class」工具、取代手寫 `os.getenv()`。

```text
from pydantic_settings import BaseSettings

class Settings(BaseSettings):
    database_url: str                         # 必填、缺少會 raise
    api_key: str
    debug: bool = False                       # 預設值
    
    class Config:
        env_file = ".env"                     # 自動讀 .env

settings = Settings()
print(settings.database_url)
# 自動：(1) 讀 .env (2) 讀環境變數 (3) 驗證型別 (4) 給型別提示
```
