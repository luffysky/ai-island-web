### Pipeline：3 行用 AI（最快速入門路徑）

**`pipeline()`** = HF 最高層 API、自動處理 tokenize + model + post-process、3 行寫完一個 AI 應用。

```text
from transformers import pipeline

# === 情感分析（判斷文字是正面 / 負面）===
classifier = pipeline("sentiment-analysis")
result = classifier("I love this product!")
print(result)
# [{'label': 'POSITIVE', 'score': 0.9998}]
```

  
**10 種常用 pipeline 任務：**

```text
# === 翻譯 ===
translator = pipeline("translation_en_to_zh")
translator("Hello, how are you?")

# === 摘要（長文章變短）===
summarizer = pipeline("summarization")
summary = summarizer(long_text, max_length=50)

# === 問答（給文章 + 問題、回答案）===
qa = pipeline("question-answering")
qa(
    question="Who created Python?",
    context="Python was created by Guido van Rossum in 1991."
)
# {'answer': 'Guido van Rossum', 'score': 0.95}

# === 文字生成（給開頭、續寫）===
generator = pipeline("text-generation", model="gpt2")
generator("The future of AI is", max_length=50)

# === 語音轉文字（Whisper、最強開源語音識別）===
transcriber = pipeline(
    "automatic-speech-recognition",
    model="openai/whisper-base"
)
transcriber("audio.mp3")

# === 圖片分類 ===
classifier = pipeline("image-classification")
classifier("dog.jpg")
```

### 底層 API（更精細控制：tokenizer + model 分開操作）

當 pipeline 不夠用時、用 `AutoTokenizer` + `AutoModel*` 自己控制流程。

```text
from transformers import AutoTokenizer, AutoModelForSequenceClassification
import torch

model_name = "bert-base-uncased"
tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForSequenceClassification.from_pretrained(model_name)

# === 1. Tokenize（文字轉 model 認得的 token id）===
text = "I love this product"
inputs = tokenizer(
    text,
    return_tensors="pt",                    # 回傳 PyTorch Tensor
    padding=True,                            # 自動補 0 對齊長度
    truncation=True                          # 太長就截斷
)

# === 2. 推論 ===
with torch.no_grad():                        # 不算梯度（推論不需要）
    outputs = model(**inputs)
    logits = outputs.logits                  # raw 分數
    probs = torch.softmax(logits, dim=-1)    # 轉機率（加總 = 1）
    predicted = torch.argmax(probs, dim=-1)  # 取最大機率的類別
```

  
**AutoModel 對應不同 task：**

| Class | 適合 task |
|---|---|
| `AutoModelForSequenceClassification` | 文字分類（情感分析 / 垃圾信） |
| `AutoModelForCausalLM` | 文字生成（GPT 風格） |
| `AutoModelForQuestionAnswering` | 問答 |
| `AutoModelForTokenClassification` | 命名實體識別（NER） |
| `AutoModelForImageClassification` | 圖片分類 |

  
### Fine-tune（用自己資料微調預訓練 model）

**fine-tune** = 「**站在巨人肩膀上**」——拿 BERT / Llama 這種「已經會語言」的 model、用你的少量資料調整、就能適應你的場景。

**完整範例：用 5 筆中文評論 fine-tune 情感分析：**

```text
from transformers import (
    AutoTokenizer, AutoModelForSequenceClassification,
    Trainer, TrainingArguments
)
from datasets import Dataset

# === 1. 準備資料（label：1=正面、0=負面）===
data = {
    "text": ["這產品很棒", "服務很差", "普通", "超讚", "爛透"],
    "label": [1, 0, 1, 1, 0]
}
dataset = Dataset.from_dict(data)
dataset = dataset.train_test_split(test_size=0.2)

# === 2. Tokenize ===
model_name = "bert-base-chinese"             # 中文 BERT
tokenizer = AutoTokenizer.from_pretrained(model_name)

def tokenize(batch):
    return tokenizer(batch["text"], truncation=True, padding=True)

tokenized = dataset.map(tokenize, batched=True)

# === 3. 載 model（num_labels=2 = 二分類）===
model = AutoModelForSequenceClassification.from_pretrained(
    model_name, num_labels=2
)

# === 4. 訓練設定 ===
training_args = TrainingArguments(
    output_dir="./results",                   # model checkpoint 存哪
    num_train_epochs=3,                       # 跑 3 個 epoch
    per_device_train_batch_size=16,           # 每張 GPU 16 筆
    learning_rate=2e-5,                       # 微調用小 lr（避免破壞原 weight）
    eval_strategy="epoch",                    # 每 epoch 評估一次
    save_strategy="epoch",
    load_best_model_at_end=True,              # 訓練完自動載最佳那個
)

# === 5. Trainer（HF 包好的訓練器、不用自己寫五字訣）===
trainer = Trainer(
    model=model,
    args=training_args,
    train_dataset=tokenized["train"],
    eval_dataset=tokenized["test"],
    tokenizer=tokenizer,
)
trainer.train()

# === 6. 存 ===
trainer.save_model("./my-model")
tokenizer.save_pretrained("./my-model")
```

  
### LoRA / PEFT（高效 fine-tune、省 99% GPU 記憶體）

**問題**：完整 fine-tune Llama-70B 要 100GB GPU 記憶體、一般人玩不起。

**LoRA**（Low-Rank Adaptation、低秩適配）：**只 fine-tune 0.1% 參數、效果幾乎一樣**。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**（裝 PEFT）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
uv add peft
```

**這行做什麼？** 裝 PEFT（Parameter-Efficient Fine-Tuning、參數高效微調）套件、含 LoRA / Prefix Tuning / Prompt Tuning 等多種「省資源」微調方法。

```text
from peft import LoraConfig, get_peft_model
from transformers import AutoModelForCausalLM

model = AutoModelForCausalLM.from_pretrained("meta-llama/Llama-3-8B")

lora_config = LoraConfig(
    r=8,                                      # LoRA rank（越小越省、越大越強）
    lora_alpha=16,                            # 縮放因子
    target_modules=["q_proj", "v_proj"],      # 只 fine-tune 哪些層（attention 的 Q V）
    lora_dropout=0.05,
    bias="none",
    task_type="CAUSAL_LM"
)

model = get_peft_model(model, lora_config)
model.print_trainable_parameters()
# 輸出：trainable params: 4M / 8B（0.05%）← 只動 0.05%
```

> 💡 **2026 主流**：**LoRA + QLoRA（量化 + LoRA）**——單張消費級 GPU（如 RTX 4090）就能 fine-tune Llama-70B、五年前要超級電腦的事情、現在桌機就行。

### 2026 indie 開發者用 HF 的 5 大場景

```text
1. SaaS 內建 AI feature
   - pipeline 1 行加 AI（情感分析 / 翻譯 / 摘要）

2. 公司私有資料 fine-tune
   - 不送 OpenAI / Anthropic（敏感資料留自己手上）
   - LoRA 訓練成本低

3. Whisper 自架語音識別
   - 會議轉錄、YouTube 字幕、Podcast 文字版
   - 完全免費、不限量

4. Stable Diffusion 圖片生成
   - 不付 Midjourney（$30/月）
   - 完全控制、商用無限制

5. 自架 LLM（Llama / Mistral / Qwen）
   - 取代 OpenAI / Anthropic 訂閱
   - 隱私 + 成本可控
   - 用 vLLM 部署、QPS 高
```

  
**對應的進階套件：**

| 套件 | 用途 |
|---|---|
| **diffusers** | 圖片生成（Stable Diffusion） |
| **accelerate** | 分散式訓練 / 多 GPU |
| **peft** | LoRA / QLoRA / Prompt Tuning |
| **bitsandbytes** | 量化（INT8 / INT4、省一半 ~ 八分之一記憶體） |
| **vllm** | LLM 推論加速、production serving |

  
    **🚀 進階技巧**
    
2026 AI 工程師核心 stack：**PyTorch + Transformers + HF Hub + PEFT + Datasets**。會這 5 個 = 能做 90% modern AI 工作。**indie 必學 3 件事**：(1) HF `pipeline()`（5 分鐘加 AI feature） (2) Whisper 語音轉文字（自架免錢） (3) Stable Diffusion 圖片生成（取代 Midjourney）。**3 個月學會 fine-tune + ship 自己 model** —— 不誇張、跟著 HF 官方教學做就行。

  

  
    
      ⚠️
      新手常見錯誤
    
    Hugging Face 3 大新手坑：(1) **以為要訓練才能用 AI** —— 錯！`pipeline()` 3 行就用、99% 場景不用 fine-tune (2) **第一次跑超慢** —— model 第一次下載要時間（幾百 MB ~ 幾 GB）、之後會 cache 在 `~/.cache/huggingface/`、第二次秒開。**Colab 因為每次 session 重置、每次都要重下載**——所以大 model 用付費 Colab Pro 或自己電腦比較順 (3) **GPU 記憶體爆** —— Llama 7B 至少要 16GB GPU 記憶體、爆了用 `device_map="auto"` + bitsandbytes 量化、不然就用更小 model（Phi-3 / Gemma-2B）。
