X_temp, X_test, y_temp, y_test = train_test_split(X, y, test_size=0.2)

### 數值縮放

```text
# 為什麼 scale？
# 年齡 0-100、薪資 20000-200000
# 沒 scale → 薪資 dominate

# StandardScaler（z-score）
from sklearn.preprocessing import StandardScaler
scaler = StandardScaler()

# 重要：先 split、再 fit on train
scaler.fit(X_train)
X_train_scaled = scaler.transform(X_train)
X_test_scaled = scaler.transform(X_test)
# 不能 fit 整個 X（data leakage）

# MinMaxScaler（[0, 1]）
from sklearn.preprocessing import MinMaxScaler

# RobustScaler（對 outlier 穩定）
from sklearn.preprocessing import RobustScaler

# 哪些 model 需要 scale：
# Linear / SVM / KNN / NN → 要
# Tree-based（RF / XGBoost）→ 不用
```

  
### 創建新特徵

```text
# 日期 → 多特徵
df["date"] = pd.to_datetime(df["date"])
df["year"] = df["date"].dt.year
df["month"] = df["date"].dt.month
df["dayofweek"] = df["date"].dt.dayofweek
df["is_weekend"] = df["dayofweek"] >= 5

# 比例
df["price_per_sqft"] = df["price"] / df["sqft"]

# 交互
df["age_x_income"] = df["age"] * df["income"]

# 聚合特徵
user_stats = df.groupby("user_id").agg({
    "amount": ["mean", "max", "count"],
})
```

  
### Pipeline 避免 leakage

```text
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.ensemble import GradientBoostingClassifier

pipeline = Pipeline([
    ("scaler", StandardScaler()),
    ("model", GradientBoostingClassifier())
])

pipeline.fit(X_train, y_train)
# 自動 scale + 訓練
# 自動避免 leakage
y_pred = pipeline.predict(X_test)
```

    **🚀 進階技巧**
    
2026 特徵工程 5 原則：(1) **切 train/test 後才做特徵工程**—避免 leakage (2) **用 Pipeline**—自動化 (3) **類別 encoding 看 cardinality**—2-5 類用 one-hot、10+ 類用 target (4) **不平衡看 F1 不看 accuracy** (5) **新特徵驗證有用**—加進去看 score 變化。
