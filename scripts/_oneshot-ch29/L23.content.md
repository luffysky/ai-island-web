**AI 爬蟲 = LLM 看畫面、自己操作瀏覽器**——2024 出現、2026 漸主流。**給目標、AI 自己找資料**、不用寫 CSS selector、網站改版也不會壞。

> 💡 **代表工具**：(1) **browser-use**（Python、最熱）(2) **Magnitude**（TypeScript）(3) **Claude Computer Use**（Anthropic 官方）(4) **OpenAI Operator**。

  
### 為什麼 AI 爬蟲是未來

```text
=== 傳統爬蟲問題 ===

❌ 每個網站要寫 selector（.product .name）
❌ 網站改版（class 改名）→ 全部壞
❌ 不同網站結構不同 → 寫 N 個 scraper
❌ 反爬升級 → 要追著修
❌ 維護成本高

=== AI 爬蟲解法 ===

✅ 給目標（「找蝦皮最低 iPhone 17 價格」）
✅ AI 看畫面 + 自己決定點哪
✅ 改版？AI 仍能看到「價格」「商品名」
✅ 不同網站同方法（AI 都能讀）
✅ 維護 ≈ 0
```

  
### 1. browser-use（Python、最熱門 GitHub 50k+ star）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
uv add browser-use langchain-anthropic
```

**這行做什麼？** 裝 browser-use + LangChain Anthropic（Claude 整合）。

  
📄 **寫進程式檔**：`ai_scraper.py`

```text
import asyncio
from browser_use import Agent
from langchain_anthropic import ChatAnthropic

async def main():
    agent = Agent(
        task="去蝦皮搜尋 iPhone 17 Pro、找前 10 個最低價、回 JSON 格式：[{name, price, url}]",
        llm=ChatAnthropic(model="claude-opus-4-7")
    )
    result = await agent.run()
    print(result)

asyncio.run(main())
```

> 💡 **browser-use 內部做什麼？** (1) 開 Playwright 瀏覽器 (2) 每步給 LLM 看截圖 + DOM (3) LLM 決定點哪 / 輸入什麼 / 拿什麼 (4) 重複到任務完成。**完全像人在用**。

### 3. Claude Computer Use（Anthropic 官方）

**特色**：Claude 直接看螢幕截圖、控制滑鼠 / 鍵盤，**不只是瀏覽器、任何 GUI 都能控**。

```text
import anthropic

client = anthropic.Anthropic()

response = client.messages.create(
    model="claude-opus-4-7",
    max_tokens=4096,
    tools=[
        { "type": "computer_20250124", "name": "computer" },
        { "type": "bash_20250124", "name": "bash" },
    ],
    messages=[{
        "role": "user",
        "content": "打開蝦皮、搜尋 iPhone 17 Pro、抓前 10 個最低價"
    }]
)

# Claude 會自己決定操作：開瀏覽器 → 輸入 URL → 等載入 → 點搜尋框 → 輸入 → 截圖判斷 → 截 10 個結果
```

  
### 4. 半 AI 半傳統：LLM 結構化抽取

**情境**：用傳統爬蟲拿 HTML、再給 LLM 解析。

```text
import Anthropic from '@anthropic-ai/sdk';

const client = new Anthropic();

const html = await fetchHTML(url);          // 傳統 fetch

const prompt = `從這個網頁抽出所有商品的：name, price, link、回 JSON 陣列：

${html}`;

const response = await client.messages.create({
  model: 'claude-opus-4-7',
  max_tokens: 4096,
  messages: [{ role: 'user', content: prompt }]
});

const data = JSON.parse(response.content[0].text);
```

> 💡 **這招的優勢**：傳統 fetch 便宜 + LLM 解析智能、**最 ROI 平衡**。常見企業場景。

### 何時用 AI 爬蟲

```text
✅ 適合
- 改版頻繁的網站（傳統一直壞）
- 多種網站同一個方法（不想寫 N 個 scraper）
- 不規則資料（每頁結構不一樣）
- PoC / MVP 階段（快速驗證）
- 結構化抽取（用 LLM 解析）

❌ 不適合
- 大量同一網站（傳統便宜 100x）
- 已有官方 API
- 即時要求（每頁 LLM 思考 3-10 秒）
- 預算敏感
- 反爬嚴重（AI 仍會被擋）
```

  
### 限制

```text
1. 慢
   → 每頁 LLM 思考 3-10 秒
   → 傳統 0.5-2 秒

2. 貴
   → token 費用 $0.003-0.015 / 1k token
   → 一頁約 NT$ 0.3-3

3. 不確定性
   → LLM 偶爾理解錯
   → 拿到資料格式可能變動

4. 仍會被反爬擋
   → 反爬偵測「行為」、AI 也會被偵測

5. 不適合即時
   → 慢、不適合 user-facing 即時查詢
```
