**Curl Impersonate + TLS Fingerprint**——進階反爬技術、**模擬真實瀏覽器的 TLS 握手指紋**、過 Cloudflare / Akamai 第一道。

> 💡 **背景**：網站不只看 HTTP headers、還看你的 **TLS Client Hello** 指紋。**Python requests / Node fetch 的 TLS 指紋跟 Chrome / Firefox 不一樣**、網站一看就擋。

  
### 為什麼會被擋（TLS 指紋是什麼）

```text
=== TLS Client Hello ===
你開瀏覽器、輸入 https://example.com、
瀏覽器先送出「TLS Client Hello」訊息：

→ 我支援的加密套件清單（順序也固定）
→ 我支援的擴展（extensions）
→ 我支援的橢圓曲線
→ 我支援的簽章演算法

每個瀏覽器送的「順序 + 內容」**幾乎是指紋**：
- Chrome 131：固定一組
- Firefox：另一組
- Safari：另一組
- Python requests：超明顯、跟瀏覽器差很多
- Node fetch：跟瀏覽器也不一樣

網站工具（JA3 / JA4）可以一眼分辨、直接擋。
```

  
### 解法 1：got-scraping（Node、最簡單）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
npm install got-scraping
```

**這行做什麼？** 裝 `got-scraping`、Apify 出的 fetch 替代、**自動偽造 TLS 指紋 + headers**、過 Cloudflare 基礎防護。

```text
import { gotScraping } from 'got-scraping';

const res = await gotScraping({
  url: 'https://example.com',
  
  // === 自動生成「像真實 Chrome」的 TLS + headers ===
  headerGeneratorOptions: {
    browsers: [{ name: 'chrome', minVersion: 110 }],
    devices: ['desktop'],
    locales: ['zh-TW'],
    operatingSystems: ['windows', 'macos'],
  }
});

console.log(res.body);
```

> 💡 **got-scraping 內部做什麼？** (1) 自動生成真實 Chrome headers (2) 用 `header-generator` 套件模擬 TLS (3) 對接 fingerprint-generator。**過 Cloudflare 第一道防護成功率 90%+**。

  
### 解法 2：undici + HTTP/2（原生 Node）

```text
import { fetch, Agent } from 'undici';

const res = await fetch('https://example.com', {
  headers: {
    'User-Agent': 'Mozilla/5.0 (Macintosh) Chrome/131.0',
    'Accept': 'text/html,application/xhtml+xml,application/xml;q=0.9',
    'Accept-Language': 'zh-TW,zh;q=0.9',
    'Accept-Encoding': 'gzip, deflate, br',
  },
  
  // === 啟用 HTTP/2（真實瀏覽器都用 H2）===
  dispatcher: new Agent({ allowH2: true })
});
```

> 💡 **HTTP/2 vs HTTP/1.1**：Chrome 預設用 H2、Python requests 只支援 H1、用 H1 連 = 馬上被識破。

  
### 解法 3：curl-impersonate（最徹底、Linux）

**情境**：got-scraping 過不了、要更徹底。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**（Linux / Mac）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
# Ubuntu
sudo apt install libcurl4-openssl-dev
git clone https://github.com/lwthiker/curl-impersonate
# ... 編譯流程

# Mac
brew install curl-impersonate
```

**這 4 行做什麼？** 安裝 curl-impersonate、**重新編譯 curl 的版本**、TLS 完全模擬 Chrome / Firefox / Safari。

```text
# 模擬 Chrome 131
curl_chrome131 https://example.com

# 模擬 Firefox 122
curl_ff122 https://example.com

# 模擬 Safari 17
curl_safari17 https://example.com
```

> 💡 **curl-impersonate** = **「真的」變成 Chrome 在發 request**、TLS / 順序 / 擴展全一樣、過 99% TLS 指紋偵測。

  
### 解法 4：Playwright（最徹底、最重）

**真瀏覽器直接打、TLS 100% 對的上**：

```text
import { chromium } from 'playwright';

const browser = await chromium.launch();
const page = await browser.newPage();

// === 用 page.request 發 API、共用瀏覽器 TLS ===
const res = await page.request.get('https://example.com/api/products', {
  headers: { 'X-Custom': 'value' }
});
const data = await res.json();
```

> 💡 **`page.request` vs `fetch`**：page.request **用瀏覽器 instance 發**、共用 cookies / TLS、過反爬最強。

  
### 解法 5：FlareSolverr（過 Cloudflare 專用）

**情境**：Cloudflare 完整防護（含 challenge page）。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
docker run -d -p 8191:8191 ghcr.io/flaresolverr/flaresolverr
```

**這行做什麼？** 用 Docker 跑 FlareSolverr、**內部用 Selenium + undetected-chromedriver 過 Cloudflare**、提供 HTTP API。

```text
const res = await fetch('http://localhost:8191/v1', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    cmd: 'request.get',
    url: 'https://cloudflare-protected.com',
    maxTimeout: 60000
  })
});

const { solution } = await res.json();
console.log(solution.response);                // 過 Cloudflare 後的 HTML
console.log(solution.cookies);                 // 取得的 cookies（可重用）
```

> 💡 **FlareSolverr 流程**：(1) 你打 FlareSolverr API (2) 它開隱形 Chrome 過 Cloudflare (3) 拿到 cookies + HTML (4) 還給你。**過 Cloudflare 成功率 95%+**。
