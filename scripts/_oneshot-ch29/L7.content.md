### Node fetch 演進歷史

| Node 版本 | fetch 狀態 |
|---|---|
| Node 16 以前 | 沒有、要裝 `node-fetch` |
| Node 18 | 內建（experimental） |
| Node 20 | 內建 stable |
| **Node 22+** | 內建 + 完整 Web 標準（含 Stream / AbortController） |

> 💡 **2026 慣例**：用 Node 20 LTS 或 22 LTS、不要再裝 node-fetch。

  
### AbortController：完整 Timeout 控制

**為什麼要 AbortController？** 預設 fetch **沒 timeout**、伺服器不回會無限等。

```text
// === 方法 1：手動 AbortController（最完整）===
const controller = new AbortController();

const timeout = setTimeout(() => controller.abort(), 10_000);

try {
  const res = await fetch(url, { signal: controller.signal });
  const html = await res.text();
  return html;
} catch (err) {
  if (err.name === 'AbortError') {
    console.log('Timeout 10 秒、放棄');
  }
  throw err;
} finally {
  clearTimeout(timeout);                     // 記得清 timer
}
```

  
**方法 2：AbortSignal.timeout（Node 17.3+、最簡潔）：**

```text
const res = await fetch(url, {
  signal: AbortSignal.timeout(10_000),       // 一行搞定
});
```

> 💡 **推薦用方法 2**——簡潔、不用記得清 timer。

  
### 取消正在跑的 fetch

**情境**：使用者按「取消」按鈕、停止爬蟲。

```text
const controller = new AbortController();

// === 開始爬 ===
const promise = fetch(url, { signal: controller.signal });

// === 1 秒後使用者按取消 ===
setTimeout(() => controller.abort(), 1000);

try {
  const res = await promise;
  // ...
} catch (err) {
  if (err.name === 'AbortError') {
    console.log('使用者取消');
  }
}
```

  
### 同時取消多個 fetch

**情境**：爬 10 個 URL、一個失敗就全取消。

```text
const controller = new AbortController();

const urls = ['https://a.com', 'https://b.com', /* ... */];

const promises = urls.map(url =>
  fetch(url, { signal: controller.signal })
    .catch(err => {
      // 一個失敗、取消所有
      controller.abort();
      throw err;
    })
);

const results = await Promise.allSettled(promises);
```

  
### Stream 處理大檔（不爆記憶體）

**情境**：下載 1 GB CSV、用 `res.text()` 會把整個檔放記憶體、OOM。

```text
import fs from 'node:fs';
import { pipeline } from 'node:stream/promises';
import { Readable } from 'node:stream';

const res = await fetch('https://example.com/big-file.csv');

// === 把 fetch 的 Web Stream 轉成 Node Stream ===
const nodeStream = Readable.fromWeb(res.body);

// === pipeline 串到檔案、不過記憶體 ===
await pipeline(nodeStream, fs.createWriteStream('big-file.csv'));

console.log('下載完成、用量 < 10MB 記憶體');
```

> 💡 **何時用 stream？** 檔案 > 100MB 必用、< 10MB 直接 `res.text()` 即可。

  
### 邊下載邊處理（process while downloading）

**情境**：CSV 太大、想邊讀邊處理、每讀一行就 process。

```text
import { createInterface } from 'node:readline';

const res = await fetch('https://example.com/data.csv');
const nodeStream = Readable.fromWeb(res.body);

const rl = createInterface({
  input: nodeStream,
  crlfDelay: Infinity,
});

let count = 0;
for await (const line of rl) {
  const [name, age] = line.split(',');
  // === 處理每一行（不用等整個下載完）===
  await processRow(name, age);
  count++;
}

console.log(`處理 ${count} 行`);
```

  
### 進度條顯示下載進度

```text
const res = await fetch('https://example.com/big-file.zip');

const total = Number(res.headers.get('content-length'));
let received = 0;

const reader = res.body.getReader();
const chunks = [];

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  
  chunks.push(value);
  received += value.length;
  
  const percent = ((received / total) * 100).toFixed(1);
  process.stdout.write(`\r下載 ${percent}% (${received}/${total})`);
}

console.log('\n完成');
```

### Cloudflare Workers fetch（serverless 爬蟲）

**Workers 內也能 fetch、跑在 300+ 個資料中心、超快**：

```text
export default {
  async fetch(request) {
    const res = await fetch('https://target.com');
    const html = await res.text();
    
    // 簡單解析（cheerio 太重、Workers 不支援）
    const match = html.match(/<title>(.+?)<\/title>/);
    const title = match?.[1];
    
    return new Response(JSON.stringify({ title }), {
      headers: { 'content-type': 'application/json' },
    });
  },
};
```

> 💡 **Workers 限制**：50ms CPU 時間、不能用 cheerio（要用 regex 或 HTMLRewriter）、適合輕量爬蟲（L18 詳教）。

  
### 對照表：fetch 各情境推薦

| 情境 | 推薦 |
|---|---|
| 一般爬蟲（< 10MB 頁） | `res.text()` + cheerio |
| 大檔下載（> 100MB） | Stream pipeline |
| 邊下載邊處理 | readline + Readable.fromWeb |
| 需要 timeout | `AbortSignal.timeout(10_000)` |
| 使用者可取消 | 手動 AbortController |
| 超快 + 同樣語法 | Bun runtime |
| Serverless 部署 | Cloudflare Workers + HTMLRewriter |

  
    **☕ 用人話講**
    
fetch 進階補充：(1) **AbortController** 是 fetch timeout 唯一正解、Node 17.3+ 用 `AbortSignal.timeout` 最簡潔 (2) **Stream** 處理大檔（> 100MB）、不爆記憶體 (3) **`Readable.fromWeb`** 把 Web Stream 轉 Node Stream (4) **進度條** 用 reader + content-length (5) **Bun** 跑同樣 code、快 3 倍。**心法**：小爬蟲簡單寫、大檔 / 大量必用 stream。
