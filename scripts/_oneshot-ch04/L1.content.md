**JavaScript（JS）= 讓網頁「動起來」的程式語言**。1995 年 Netscape 工程師 Brendan Eich **10 天內**趕出來、原本叫 LiveScript、為了蹭當時最紅的 Java 才改名 JavaScript（兩個語言根本沒關係、像「雷神」跟「雷神咖啡」一樣只是名字像）。

> 💡 **這課的價值**：搞懂 JS 的「身世」會解開很多「為什麼這語言這麼怪」的疑問——這些怪不是你笨、是 30 年前趕工的歷史遺跡。

  
### 1. 為什麼學 JS？

```text
✅ 全世界使用最廣的程式語言
   → 每個瀏覽器都跑 JS、躲不掉

✅ 一個語言三條路
   → 前端（網頁畫面）
   → 後端（伺服器）
   → 桌面 App / 手機 App / 嵌入式

✅ 起薪相對高、需求大
   → 台灣 junior 月薪 NT$ 45-65K 起跳

✅ AI 時代依然關鍵
   → ChatGPT 網頁前端是 JS
   → Cursor / VS Code 本體就是 JS 寫的
```

> 💡 **重點**：學前端只能選 JS、沒有第二選項。學會 JS 後再學 TypeScript 是「JS + 型別檢查」、不是另一個語言。

  
### 2. JS 到底跑在哪？

很多人以為 JS 只能跑在瀏覽器、其實 2026 年它跑在**所有地方**。

```text
🌐 瀏覽器
   → Chrome 用 V8 引擎
   → Firefox 用 SpiderMonkey
   → Safari 用 JavaScriptCore

🖥️ 後端伺服器
   → Node.js（最老牌、最多套件）
   → Bun（2022 後新貴、超快）
   → Deno（Node 創作者的「重寫版」）

☁️ 邊緣運算（edge computing）—— 跑在離使用者最近的機房
   → Cloudflare Workers
   → Vercel Edge
   → Deno Deploy

🖼️ 桌面 App
   → Electron（VS Code / Discord / Slack 都是這個）
   → Tauri（更輕量的替代品）

📱 手機 App
   → React Native（Instagram / Facebook 用）
   → Capacitor（把網頁包成 App）

🤖 IoT 嵌入式裝置
   → Espruino / Moddable 讓小晶片也能跑 JS
```

> 💡 **白話**：JS 從「網頁特效用」變成「什麼都能寫」、這就是它今天這麼重要的原因。

  
### 3. ECMAScript vs JavaScript 差在哪？

新手最常搞混的兩個名字。

```text
📜 ECMAScript（ES）= 規範書
   → 描述「JS 語言應該長怎樣」的標準文件
   → 每年發布一版（ES2015、ES2020、ES2026...）
   → 由 TC39 委員會制定

💻 JavaScript = 實作
   → 各家瀏覽器照規範寫出來的「實際引擎」
   → V8、SpiderMonkey、JavaScriptCore 都是不同實作

類比：
  USB 是規範
  USB 線是各廠商做的實作
  兩者要對得上才能用
```

  
### 4. JS 怪在哪？看這些「歷史遺跡」

```text
🐛 typeof null === "object"
   → null 明明不是物件、卻說是物件
   → 1995 年的 bug、現在拆不掉了

🐛 0.1 + 0.2 !== 0.3
   → 浮點數運算誤差
   → 結果是 0.30000000000000004

🐛 [] + [] === ""
   → 空陣列加空陣列等於空字串
   → 隱式型別轉換的後遺症

🐛 [] + {} 跟 {} + [] 結果不同
   → 解析方式不一樣
```

> 💡 **不用背、知道有就好**——這些怪事都會在後面 25 課陸續解釋。

  
### 5. **strict mode（嚴格模式）**—— 強制 JS 用「比較不會出錯」的規則

📄 **寫進程式檔**：`app.js`

```text
"use strict";  // 寫在檔案最頂端

x = 10;  // 嚴格模式會報錯（沒先宣告就用變數）
```

```text
✅ 自動嚴格模式（不用寫 "use strict"）
   → ES Module（.mjs 或 type="module" 的 script）
   → class 內部
   → React / Vue / Next.js 等現代框架

→ 2026 寫新專案幾乎不用主動寫 "use strict"
→ 但要知道為什麼舊 code 看得到
```

  
### 6. 把 JS 塞進 HTML 的三種方法

📄 **寫進程式檔**：`index.html`

```text
<!-- 方法 1：inline（行內、寫在 HTML 屬性裡） -->
<button onclick="alert('hi')">點我</button>

<!-- 方法 2：internal（內部、寫在 <script> 標籤裡） -->
<script>
  console.log("hello");
</script>

<!-- 方法 3：external（外部、引入 .js 檔案） -->
<script src="app.js"></script>
```

```text
方式         場景                         2026 推薦
inline       絕對不要                     ✗
internal     小工具、analytics 短 code    △
external     所有正式專案                 ✓
```

> 💡 **為什麼 inline 不行？** (1) 邏輯跟畫面混在一起難維護 (2) 違反 CSP（內容安全政策）資安標準 (3) 沒辦法 minify、cache、共用。

  
### 7. `<script>` 標籤放哪裡？async / defer 差在哪？

同一個 `<script>` 標籤、放不同位置、加不同屬性、行為完全不同。

📄 **寫進程式檔**：`index.html`

```text
<!-- 1. 放 <head>、沒加任何屬性 -->
<head>
  <script src="app.js"></script>  ← 阻塞 HTML 解析、最爛
</head>

<!-- 2. 放 <body> 結尾 -->
<body>
  <h1>...</h1>
  <script src="app.js"></script>  ← 等 HTML 都解析完才執行
</body>

<!-- 3. 加 defer：HTML 解析完才執行、但下載並行 -->
<head>
  <script src="app.js" defer></script>  ← 最常用
</head>

<!-- 4. 加 async：下載完馬上執行（不管 HTML 進度） -->
<head>
  <script src="ga.js" async></script>  ← 給 analytics 用
</head>

<!-- 5. type="module"：自帶 defer + ESM -->
<head>
  <script type="module" src="app.js"></script>  ← 2026 標準
</head>
```

> 💡 **2026 建議**：(1) 應用主程式用 `type="module"`（自帶 defer + 可用 import） (2) 第三方追蹤碼用 `async` (3) 老瀏覽器 polyfill 用 `defer`。**絕對不要**放 `<head>` 裡又沒加 defer/async——會阻塞渲染、Lighthouse 評分直接掛。

  
### 5 個常見坑

```text
❌ 1. JavaScript 跟 Java 是同個東西
   → 完全沒關係、純粹當年蹭名字
   → 修：就當作兩個不同語言

❌ 2. 「不用嚴格模式比較自由」
   → 自由 = 容易出 bug
   → 修：所有新 code 都用嚴格模式（用 ES Module 就自動有）

❌ 3. <script> 放 <head> 不加 defer
   → 阻塞渲染、白畫面
   → 修：加 defer 或 type="module"

❌ 4. inline 寫法到處複製
   → 維護地獄
   → 修：第一行 code 就用 external .js 檔

❌ 5. 0.1 + 0.2 不等於 0.3 是 bug
   → 不是 bug、是浮點數本質
   → 修：金額計算用整數（以分為單位）或 bigint
```

  
### 🎯 重點回顧

- **JavaScript 是 1995 年 10 天趕出來的**、跟 Java 沒關係、卻變成全世界最常用的語言
- **JS 跑在所有地方**：瀏覽器、伺服器、桌面、手機、IoT、邊緣運算
- **ECMAScript 是規範、JavaScript 是實作**——每年一版
- **strict mode 是預設**——ES Module 自動開、不用煩
- **`<script>` 放 `<head>` 一定加 defer 或用 `type="module"`**
