**Web Worker = 在主執行緒之外、另開一條 JS 執行緒**。JS 本來是單執行緒、重運算就卡 UI——Worker 是唯一「真正並行」的方法。**Service Worker** 是特殊的 Worker、能攔截網路請求、是 **PWA（Progressive Web App、漸進式網頁應用）** 的核心。

> 💡 **這課的價值**：當網頁 lag、效能 profile 主執行緒紅一片——答案幾乎都是「把重活搬到 Worker」。**Service Worker 是 2026 PWA / 離線 App 必備**。

  
### 1. 為什麼需要 Web Worker？

```text
🧵 JS 單執行緒的痛
  ┌─────────────────────────┐
  │  main thread（主執行緒）│
  │  ├─ DOM 操作            │
  │  ├─ 事件監聽            │
  │  ├─ 渲染                │  ← 都擠在一條線
  │  └─ 重運算              │     重運算就卡死全部
  └─────────────────────────┘

🧵 Worker 救星
  ┌──────────┐   ┌──────────┐
  │  main    │   │  worker  │
  │ 點擊、UI │ ⇄ │ 重運算   │
  └──────────┘   └──────────┘
       靠 postMessage 通訊
```

  
### 2. 基本 Worker 用法

📄 **寫進程式檔**：`main.js`

```text
// 主執行緒
const worker = new Worker("worker.js");

// 送訊息給 worker
worker.postMessage({ cmd: "compute", data: bigArray });

// 收 worker 回的訊息
worker.addEventListener("message", (e) => {
  console.log("結果：", e.data);
});

// 收 worker 錯誤
worker.addEventListener("error", (e) => {
  console.error(e);
});

// 結束 worker
worker.terminate();
```

📄 **寫進程式檔**：`worker.js`

```text
// Worker 內：自己有 self（不是 window）
self.addEventListener("message", (e) => {
  const { cmd, data } = e.data;
  if (cmd === "compute") {
    const result = heavyCalculation(data);
    self.postMessage(result);
  }
});

function heavyCalculation(arr) {
  return arr.map(x => x * 2).reduce((a, b) => a + b, 0);
}
```

> 💡 **Worker 內看不到 DOM**——只能算東西、不能改畫面。要改畫面把結果 postMessage 回主執行緒。

  
### 3. **三種 Worker**

```text
🟦 Worker（Dedicated Worker）
   → 跟「某個分頁」綁定
   → 一對一通訊
   → 90% 場景用這個

🟧 SharedWorker
   → 多個分頁共用同一個 worker
   → 同網域才行
   → 適合：跨分頁共享 state、共用 WebSocket

🟩 ServiceWorker
   → 攔截網路請求 → 離線、PWA、快取、推播
   → 跟特定分頁無關、在背景跑
```

  
### 4. **postMessage 注意事項**

```text
// 資料是「複製」過去、不是共用引用
worker.postMessage({ huge: bigArray });
// → 兩邊各有一份

// 「轉移所有權」（Transferable）—— 避免複製、超快
const buffer = new ArrayBuffer(1024 * 1024);
worker.postMessage(buffer, [buffer]);
// → 主執行緒這邊 buffer 變空、所有權給 worker

// 可轉移類型：
// ArrayBuffer / MessagePort / ImageBitmap / OffscreenCanvas
```

> 💡 **白話**：`postMessage` 預設複製、大資料會慢。**用 Transferable 像「把箱子推過去」、不用「拍照給對方」**。

  
### 5. **Service Worker** —— PWA 核心

Service Worker 是個「**站在你跟 server 之間的中間人**」——所有 fetch 都先問它「要不要攔截」。

📄 **寫進程式檔**：`main.js`

```text
// 註冊
if ("serviceWorker" in navigator) {
  navigator.serviceWorker.register("/sw.js")
    .then(() => console.log("SW 已註冊"))
    .catch(err => console.error(err));
}
```

📄 **寫進程式檔**：`sw.js`

```text
// 安裝時：預先 cache 關鍵檔案
self.addEventListener("install", (e) => {
  e.waitUntil(
    caches.open("v1").then(c =>
      c.addAll(["/", "/style.css", "/app.js", "/logo.png"])
    )
  );
});

// 啟用時：清舊版 cache
self.addEventListener("activate", (e) => {
  e.waitUntil(
    caches.keys().then(keys =>
      Promise.all(keys.filter(k => k !== "v1").map(k => caches.delete(k)))
    )
  );
});

// 每個 fetch 都會問 SW
self.addEventListener("fetch", (e) => {
  e.respondWith(
    caches.match(e.request).then(r => r || fetch(e.request))
  );
});
// → 離線也能用！
```

  
### 6. Service Worker 快取策略

```text
1️⃣ Cache First（快取優先）
   先看 cache、沒有才去 server
   ✅ 適合：靜態資源（CSS、JS、字型）

2️⃣ Network First（網路優先）
   先去 server、失敗才用 cache
   ✅ 適合：API 資料

3️⃣ Stale While Revalidate（先快取後更新）
   馬上回 cache、同時去 server 更新
   ✅ 適合：圖片、頭像、新聞 feed

4️⃣ Network Only / Cache Only
   只用一種、簡單暴力
```

> 💡 **2026 寫 PWA 用 [Workbox](https://developer.chrome.com/docs/workbox)**——Google 提供的 SW 工具庫、3 行配置好快取策略。

  
### 7. **PWA（Progressive Web App）** —— 網頁變 App

PWA 三大要件：

```text
1️⃣ Service Worker（可離線）
2️⃣ manifest.json（可安裝到桌面）
3️⃣ HTTPS（資安基本）
```

📄 **寫進程式檔**：`manifest.json`

```text
{
  "name": "AI 島",
  "short_name": "AI 島",
  "start_url": "/",
  "display": "standalone",
  "theme_color": "#0f0",
  "background_color": "#000",
  "icons": [
    { "src": "/icon-192.png", "sizes": "192x192", "type": "image/png" },
    { "src": "/icon-512.png", "sizes": "512x512", "type": "image/png" }
  ]
}
```

📄 **寫進程式檔**：`index.html`

```text
<link rel="manifest" href="/manifest.json">
```

  
### 8. **SharedArrayBuffer + Atomics** —— 真正共享記憶體

普通 postMessage 是「複製」、SharedArrayBuffer 是「**真共用**」。

📄 **寫進程式檔**：`sab.js`

```text
// 主執行緒
const sab = new SharedArrayBuffer(1024);
const arr = new Int32Array(sab);
arr[0] = 42;

worker.postMessage(sab);   // 不複製、共用記憶體

// worker.js
self.addEventListener("message", (e) => {
  const sab = e.data;
  const arr = new Int32Array(sab);
  arr[0]++;   // ← 主執行緒這邊 arr[0] 也跟著變
});

// 競爭條件用 Atomics 防（多執行緒同時讀寫保護）
Atomics.add(arr, 0, 1);          // 原子加 1
Atomics.compareExchange(arr, 0, expected, newVal);
Atomics.wait(arr, 0, 0);          // 等到 arr[0] 不是 0
```

⚠️ **限制**：需要正確的 COOP / COEP HTTP headers、不然瀏覽器禁用。

  
### 9. 實戰場景：什麼時候用 Worker？

```text
✅ 適合 Worker
   - 圖片 / 影片處理（壓縮、濾鏡）
   - 大量資料計算（排序萬筆、複雜統計）
   - 加密 / 解密
   - 解析大 JSON / CSV
   - WebAssembly 重運算
   - Markdown / Code 高亮（VS Code Web 用 Worker）

❌ 不適合
   - 小邏輯（建 Worker 本身有成本）
   - DOM 操作（Worker 看不到 DOM）
   - 高頻 postMessage（通訊成本高）
```

  
### 10. Web Worker 內能用什麼？

```text
✅ 大部分標準 API
   - fetch
   - setTimeout / setInterval
   - WebSocket
   - IndexedDB
   - Cache API
   - WebAssembly
   - import（ES module worker）

🚫 不能用
   - DOM（document、window）
   - localStorage（但能用 IndexedDB）
   - alert / confirm / prompt
```

  
### 5 個常見坑

```text
❌ 1. Worker 內想動 DOM
   → 不行、Worker 沒 window / document
   → 修：postMessage 結果回主執行緒、由主執行緒改 DOM

❌ 2. postMessage 大物件超慢
   → 預設是複製、複製 100MB 要 1s+
   → 修：用 Transferable（ArrayBuffer 等）

❌ 3. Service Worker 改了沒生效
   → SW 有版本緩存、要更新版本字串才會重新註冊
   → 修：開 DevTools > Application > Service Workers > Update

❌ 4. SW 把所有東西全 cache
   → 改了 code 永遠拿到舊版
   → 修：API 用 Network First、靜態資源才 Cache First

❌ 5. localhost 沒問題、上線 SW 不能用
   → SW 強制 HTTPS（localhost 例外）
   → 修：上線網站一定要 HTTPS
```

  
### 🎯 重點回顧

- **Web Worker = JS 唯一真正多執行緒方法**——重運算搬過去、UI 不卡
- **三種 Worker**：Dedicated（一般）、Shared（多分頁共用）、Service（離線 + 攔截網路）
- **Service Worker + manifest + HTTPS = PWA**
- **postMessage 大資料用 Transferable**——避免複製
- **SharedArrayBuffer + Atomics 是真共享記憶體**——需 COOP / COEP header
