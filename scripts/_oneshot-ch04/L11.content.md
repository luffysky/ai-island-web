**Promise = 一個「未來會有結果的承諾」**。`async / await` 是 Promise 的「糖衣語法」、讓 async 程式長得像同步程式、好讀好寫。在 callback hell（回呼地獄）的年代後、Promise 是 JS 最重要的進化之一。

> 💡 **這課的價值**：fetch、檔案讀取、AI API、資料庫查詢——所有「等很久才回來」的事都是 Promise。學會它 = 學會寫現代 JS。

  
### 1. Promise 三種狀態

```text
🟡 pending（等待中）   → 還沒有結果
🟢 fulfilled（成功）   → 拿到結果（resolve）
🔴 rejected（失敗）   → 出錯（reject）

⚠️ 狀態只能變一次：pending → fulfilled 或 rejected
```

📄 **寫進程式檔**：`promise-basic.js`

```text
// 建立 Promise
const p = new Promise((resolve, reject) => {
  setTimeout(() => {
    resolve("done");        // 成功時呼叫
    // reject(new Error("..."));  // 失敗時呼叫
  }, 1000);
});

// 處理結果
p.then(value => console.log(value))    // 成功時跑
 .catch(err => console.error(err))      // 失敗時跑
 .finally(() => console.log("完成"));   // 不管成敗都跑
```

> 💡 **白話**：Promise 像「一張取餐號碼牌」——你不用站在櫃台等、做完叫號就好。`.then` = 號碼到了要做什麼、`.catch` = 餐點壞了怎麼辦。

  
### 2. **async / await** —— 寫得像同步程式

ES2017 的救星。

📄 **寫進程式檔**：`async.js`

```text
// 沒有 async / await（老寫法）
function loadUser(id) {
  return fetch(`/api/users/${id}`)
    .then(r => r.json())
    .then(user => {
      console.log(user);
      return user;
    })
    .catch(err => console.error(err));
}

// async / await（現代寫法）
async function loadUser(id) {
  try {
    const resp = await fetch(`/api/users/${id}`);
    const user = await resp.json();
    console.log(user);
    return user;
  } catch (err) {
    console.error("failed:", err);
    throw err;
  }
}

// 呼叫 async function 永遠回 Promise
loadUser(42).then(u => console.log(u));
```

**規則**：

```text
1. 用 async 標記的函式、永遠回 Promise
2. await 只能用在 async function 內（或 top-level）
3. await 後面那行會「暫停」直到 Promise 完成
4. try / catch 抓 await 失敗（取代 .catch）
```

  
### 3. **Promise 組合招式**

📄 **寫進程式檔**：`combine.js`

```text
// Promise.all —— 全部成功才繼續（任一失敗整體失敗）
const [a, b, c] = await Promise.all([
  fetch("/api/a"),
  fetch("/api/b"),
  fetch("/api/c")
]);

// Promise.allSettled —— 全部完成就回（不管成敗）
const results = await Promise.allSettled([p1, p2, p3]);
results.forEach(r => {
  if (r.status === "fulfilled") console.log(r.value);
  else console.error(r.reason);
});

// Promise.race —— 誰先完成（不管成敗）就用誰
const winner = await Promise.race([
  fetch("/api/slow"),
  new Promise((_, rej) => setTimeout(() => rej("timeout"), 3000))
]);
// 3 秒沒回視為 timeout

// Promise.any —— 任一成功就回（全失敗才整體失敗）
const first = await Promise.any([
  fetch("/api/server1"),
  fetch("/api/server2"),
  fetch("/api/server3")
]);
// 哪個 server 先回就用哪個
```

```text
方法              場景
Promise.all       多請求全要成功（缺一不可）
Promise.allSettled 多請求都要結果（不管成敗、要報告）
Promise.race      要最快的（含失敗 → 用來做 timeout）
Promise.any       要任一成功（多重備援）
```

  
### 4. **並行 vs 序列** —— 效能差很多

📄 **寫進程式檔**：`parallel.js`

```text
// ✗ 序列（慢）—— 等 A 完成才開始 B
const a = await fetchA();    // 等 1s
const b = await fetchB();    // 又等 1s
// 總時間 = 2s

// ✓ 並行（快）—— 兩個一起發、等都回來
const [a, b] = await Promise.all([fetchA(), fetchB()]);
// 總時間 = max(A, B) = 1s
```

> 💡 **新手常踩**：連續寫好幾個 `await`、不知不覺變串行。**沒有依賴關係的就用 `Promise.all` 並行**。

  
### 5. **AbortController** —— 可取消的 Promise

📄 **寫進程式檔**：`abort.js`

```text
const ctrl = new AbortController();

fetch("/api/big", { signal: ctrl.signal })
  .then(r => r.json())
  .catch(err => {
    if (err.name === "AbortError") console.log("取消");
  });

// 5 秒後取消
setTimeout(() => ctrl.abort(), 5000);

// 2024+ 簡寫：自動 timeout
fetch(url, { signal: AbortSignal.timeout(5000) });
```

**用途**：

```text
✅ 使用者切換頁面、取消上一頁的 fetch
✅ 搜尋框輸入新字、取消上一次搜尋
✅ 設 timeout
✅ React useEffect cleanup 時取消
```

  
### 6. error handling 完整模式

📄 **寫進程式檔**：`error.js`

```text
async function safeCall() {
  try {
    const r = await fetch("/api/user");

    // fetch 不會因為 HTTP 4xx / 5xx 自動 reject！
    if (!r.ok) {
      throw new Error(`HTTP ${r.status}`);
    }

    const data = await r.json();
    return { ok: true, data };

  } catch (err) {
    // 網路錯誤 / 4xx / 5xx / JSON parse 失敗都進這裡
    console.error(err);
    return { ok: false, error: err.message };
  }
}
```

> 💡 **`fetch` 的大坑**：404 / 500 **不會自動 reject**——它認為「拿到回應就算成功」。要自己檢查 `r.ok`。

  
### 7. retry + 指數退避（exponential backoff）

實戰常用模式：失敗就等更久再試。

📄 **寫進程式檔**：`retry.js`

```text
async function apiCall(url, retries = 3) {
  for (let i = 0; i < retries; i++) {
    try {
      const r = await fetch(url);
      if (!r.ok) {
        if (r.status >= 500) throw new Error("server error");
        return { ok: false, error: await r.text() };
      }
      return { ok: true, data: await r.json() };
    } catch (err) {
      if (i === retries - 1) throw err;
      // 等 1s、2s、4s ...
      await new Promise(r => setTimeout(r, 2 ** i * 1000));
    }
  }
}
```

  
### 8. forEach 不能 await —— 經典坑

📄 **寫進程式檔**：`foreach-trap.js`

```text
// ✗ 看似 OK、其實沒等
arr.forEach(async (item) => {
  await heavy(item);
});
console.log("done");   // 馬上印、heavy 還在跑

// ✓ 串行：for...of
for (const item of arr) {
  await heavy(item);
}
console.log("done");

// ✓ 並行：Promise.all + map
await Promise.all(arr.map(item => heavy(item)));
console.log("done");
```

> 💡 **記法**：`forEach` 接 callback 但**不會等 callback 回來**——它已經是 50 年前的設計。要 await 用 `for...of`。

  
### 9. top-level await（ES2022）

ESM 中、最外層可以直接 await。

📄 **寫進程式檔**：`init.js`（type: module）

```text
// 不用包 async function
const config = await fetch("/config.json").then(r => r.json());
const user = await loadUser(config.userId);

console.log(user);
```

**限制**：只能在 ESM（`type: module` 或 .mjs）用、CommonJS 不行。

  
### 5 個常見坑

```text
❌ 1. forEach 內 await 沒效果
   → 修：for...of 或 Promise.all(arr.map(...))

❌ 2. fetch 404 / 500 沒 catch
   → fetch 不會自動 reject
   → 修：判斷 if (!r.ok) throw new Error()

❌ 3. 連續 await 變串行
   → 沒依賴的應該並行
   → 修：Promise.all

❌ 4. 沒包 try / catch
   → uncaught rejection、頁面 console 紅一片
   → 修：async 函式內全包 try / catch

❌ 5. resolve / reject 呼叫兩次
   → Promise 狀態只能改一次、第二次無效
   → 修：if 判斷後 return
```

  
### 🎯 重點回顧

- **Promise 三狀態**：pending → fulfilled / rejected、只能變一次
- **async / await 是糖衣**——底層還是 Promise
- **`Promise.all / allSettled / race / any`** 各有用途
- **沒依賴關係的 await 改用 `Promise.all` 並行**
- **fetch 不會自動 reject HTTP 錯誤**——要自己檢查 `r.ok`
- **forEach 不能 await**——用 `for...of` 或 `Promise.all + map`
