**memory（記憶體）= 電腦短期存資料的地方**。JS 是「自動 garbage collection（GC、垃圾回收）」語言——程式設計師不用手動 free。**但還是會 memory leak（記憶體洩漏）**——你以為不用了、JS 卻不敢回收。網頁開久越來越慢就是這個。

> 💡 **這課的價值**：寫 SPA / 長期執行頁面（後台、儀表板、聊天室）不懂記憶體 = 「開三小時後就卡死」——找不到原因。學會用 Chrome Memory tab 抓 leak 是 senior 必備。

  
### 1. JS 怎麼決定「要不要回收」？

```text
🔍 GC 演算法：mark-and-sweep（標記-清除）

1. 從「root」開始往下找（root = window / global）
2. 找得到的物件 → 標記為「還在用」
3. 找不到的物件 → 是垃圾、回收

→ 只要從 root 找得到（reachable）、就不回收
→ 「找不到」≠「沒人用」、是「沒辦法到達」
```

**這就是為什麼會 leak**——你以為不用了、但其實還有個「鏈條」連到 root。

  
### 2. **5 大常見 memory leak**

**leak 1：沒清的 event listener**

📄 **寫進程式檔**：`leak1.js`

```text
// ✗ 元素移除了、listener 還抓著
function setup() {
  const btn = document.querySelector("#btn");
  btn.addEventListener("click", handler);
}

function teardown() {
  document.querySelector("#btn").remove();
  // ⚠️ DOM 元素移除、但 listener 還在記憶體
  // 連帶元素本身也回收不掉（detached DOM）
}

// ✓ 修
function teardown() {
  btn.removeEventListener("click", handler);
  btn.remove();
}
```

**leak 2：沒清的 setInterval**

```text
// ✗ 元件移除了、計時器還在跑
setInterval(() => heavy(), 1000);

// ✓ 存 id、unmount 時清掉
const id = setInterval(() => heavy(), 1000);
clearInterval(id);
```

**leak 3：closure 抓住大資料**

```text
// ✗ 回傳的小函式持有 huge 陣列
function attach() {
  const huge = new Array(1e7);   // 10 萬筆
  return () => console.log(huge[0]);
}

const fn = attach();
// huge 永遠不回收、即使你只用了 huge[0]

// ✓ 只持有需要的
function attach() {
  const huge = new Array(1e7);
  const first = huge[0];        // 抽出來
  return () => console.log(first);
}
```

**leak 4：全域變數**

```text
// ✗ 寫了就一直在
window.cache = bigData;
window.users = allUsers;

// ✓ 用完手動清
window.cache = null;
// 或用 Map + 定期清
```

**leak 5：detached DOM**

```text
// ✗ JS 還留著對 DOM 的 reference、DOM 已不在頁面
const ref = document.getElementById("x");
document.getElementById("x").remove();
// ref 還持有那個 element → 記憶體不釋放

// ✓ 清 reference
const ref = null;
```

  
### 3. **closure 跟 leak 的關係**

closure 是 JS 強項、也是 leak 主因。

```text
function createHandler() {
  const bigData = new Array(1e7);
  
  return {
    onClick: () => {
      console.log("clicked");
      // 雖然沒用 bigData、但 closure 還抓著
    }
  };
}

const handler = createHandler();
// → bigData 永遠不回收
```

**規則**：closure 抓的是「**整個出生環境**」、不是「**只抓你用到的變數**」。要避免就把不需要的變數明確 `= null`。

  
### 4. **WeakRef（弱引用、ES2021）** —— 「鬆鬆抓」不阻止回收

```text
const obj = { huge: new Array(1e7) };
const weakRef = new WeakRef(obj);

// 當 obj 沒「強引用」時、WeakRef 不阻止 GC
const got = weakRef.deref();
if (got) {
  console.log(got.huge);
} else {
  console.log("已被回收");
}
```

```text
✅ 用途
   - cache（物件 → 結果）想讓 cache 自動清
   - DOM 節點對應的 metadata
   - 觀察者模式內部記訂閱者

⚠️ 警告
   - 行為不可預測（不知道何時被回收）
   - 大多時候不該用、改用 WeakMap / WeakSet
```

  
### 5. **FinalizationRegistry（ES2021）** —— 物件被回收時通知

```text
const registry = new FinalizationRegistry((heldValue) => {
  console.log("被回收了：", heldValue);
});

let user = { name: "招財" };
registry.register(user, "user data");

user = null;
// 某時刻會印「被回收了：user data」
```

⚠️ 使用警告：時機不確定、不要用來做關鍵邏輯。**90% 場景用不到**——資安、低層 library 才碰。

  
### 6. **WeakMap / WeakSet vs Map / Set**

```text
// Map：強引用、key 物件不回收
const map = new Map();
let obj = { x: 1 };
map.set(obj, "data");
obj = null;
// → map 還抓著、obj 不回收

// WeakMap：弱引用、自動清
const wm = new WeakMap();
let obj2 = { x: 1 };
wm.set(obj2, "data");
obj2 = null;
// → wm 自動清掉、obj2 回收
```

**WeakMap 使用情境**：

```text
✅ DOM → 資料對應（DOM 移除自動清）
✅ private 屬性模擬（ES2022 # 之前的招）
✅ 物件 → 計算結果 cache（不阻止回收）
```

  
### 7. **Chrome DevTools Memory tab** —— 抓 leak 神器

🖥️ **操作步驟**：

```text
1. F12 → Memory tab
2. 選 "Heap snapshot"（堆快照）
3. 點 "Take snapshot"
4. 操作頁面（觸發可能 leak 的動作）
5. 再 take 一次 snapshot
6. 比較兩次差別（Comparison）

→ 找「(string)」「(closure)」「Detached HTMLElement」
→ 看哪個物件數量持續增加 = leak
```

**三種重要工具**：

```text
🔬 Heap Snapshot
   當下記憶體用了什麼物件、各占多少
   
🔬 Allocation Timeline
   一段時間內、誰在配記憶體
   找「動態增加但沒釋放」的元凶
   
🔬 Detached Elements（Chrome 內建）
   專門找 detached DOM
   點開可以看誰還抓著它
```

  
### 8. 監控線上 memory（real user monitoring）

```text
// performance.memory（Chrome only）
console.log(performance.memory);
// { jsHeapSizeLimit, totalJSHeapSize, usedJSHeapSize }

// 定期回報
setInterval(() => {
  analytics.send({
    used: performance.memory.usedJSHeapSize,
    total: performance.memory.totalJSHeapSize
  });
}, 60000);
```

⚠️ **隱私顧慮**：`performance.memory` 在跨域 isolation 環境才精準。

  
### 9. **React / Vue 常見 leak 場景**

```text
🔴 React useEffect 沒 cleanup
   useEffect(() => {
     const id = setInterval(...);
     return () => clearInterval(id);   // ← 一定要這行
   }, []);

🔴 訂閱沒退訂
   useEffect(() => {
     const unsub = bus.on("event", handler);
     return unsub;
   }, []);

🔴 closure 在 setState 持有舊資料
   → 用 useRef 替代

🔴 Context 太大、整顆樹重 render
   → 拆 context 顆粒度

🔴 大圖 / 影片 ref 不清
   → unmount 時 video.src = "" + video.load()
```

  
### 10. 預防 leak 的好習慣

```text
✅ 1. 每個 addEventListener 配對 removeEventListener
✅ 2. 每個 setInterval / setTimeout 存 id、適時 clear
✅ 3. 元件 unmount 必 cleanup（React useEffect return）
✅ 4. 大資料用完明確 = null
✅ 5. 短期物件用 WeakMap 當 key
✅ 6. 不要把整顆物件存 window
✅ 7. 用 AbortController 取消 in-flight 請求
✅ 8. 定期跑 Memory tab 檢查
```

  
### 11. JS 引擎的記憶體分區（概念）

```text
🗄️ Heap（堆）
   → 物件 / 函式 / closure 放這
   → GC 管的就是這

🗄️ Stack（堆疊）
   → 函式呼叫、原始型別
   → 函式 return 自動 pop、不用 GC

🗄️ V8 內部還分新生代 / 老生代
   → 新生代頻繁清（剛建的物件大多很快死）
   → 老生代少清（活很久的物件穩定）
   → Generational GC（分代回收）
```

  
### 5 個常見坑

```text
❌ 1. addEventListener 完忘了 remove
   → SPA 切頁多次後越來越慢
   → 修：unmount / route change 移除

❌ 2. setInterval 沒 clear
   → 元件移除、計時器還在
   → 修：存 id、適時 clearInterval

❌ 3. closure 持有大資料
   → 即使只用一個 byte、整個 huge 不回收
   → 修：抽出需要的、其他 = null

❌ 4. 全域 cache 越長越大
   → window.cache 沒上限
   → 修：用 LRU cache + 定期清

❌ 5. video / canvas 不釋放
   → 影片 ref 一直持有 buffer
   → 修：unmount 時 src = "" + load()
```

  
### 🎯 重點回顧

- **JS 自動 GC** = mark-and-sweep——只要從 root 找得到就不回收
- **5 大 leak**：listener / interval / closure / 全域 / detached DOM
- **WeakMap / WeakSet** 不阻止 GC——適合 DOM 對應、cache
- **WeakRef + FinalizationRegistry** 是 ES2021 進階工具（少用）
- **Chrome DevTools Memory tab** 是抓 leak 神器
- **React unmount 必 cleanup**——useEffect return 函式
