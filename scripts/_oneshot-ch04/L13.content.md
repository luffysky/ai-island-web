**DOM（Document Object Model、文件物件模型）= HTML 在 JS 眼中的「樹狀結構」**。每個標籤、屬性、文字都是一個節點。**DOM 操作 = 用 JS 找節點、改節點、聽事件**——這就是「讓網頁動起來」的本體。

> 💡 **這課的價值**：React / Vue 底層都是 DOM 操作、只是包了一層。學會原生 DOM、看任何框架都不會慌。

  
### 1. 選元素 —— 4 個必會方法

📄 **寫進程式檔**：`select.js`

```text
// 1. getElementById —— 找 id="main" 的元素
document.getElementById("main");

// 2. querySelector —— 用 CSS selector 找第一個
document.querySelector(".card");           // class 為 card 的第一個
document.querySelector("#main");           // id 為 main
document.querySelector("ul > li:first-child");

// 3. querySelectorAll —— 找全部、回 NodeList
document.querySelectorAll("li");           // 所有 li
document.querySelectorAll(".item.active"); // class 含 item + active

// 4. getElementsByClassName / TagName（舊）—— 回 HTMLCollection
document.getElementsByClassName("card");
document.getElementsByTagName("div");
```

> 💡 **2026 用法**：99% 場景用 `querySelector` / `querySelectorAll`——一個 API 走天下、跟 CSS 選擇器一致。

  
### 2. NodeList vs HTMLCollection —— 兩個容易搞混

```text
🟦 NodeList（querySelectorAll 回傳）
   → 「靜態快照」—— 拍下當下狀況
   → 可以 forEach
   → 不會隨 DOM 變化

🟧 HTMLCollection（getElementsByClassName 回傳）
   → 「動態 live」—— 跟 DOM 同步
   → 不能 forEach（要轉陣列）
   → 元素新增 / 刪除會自動反映
```

**兩個都不是真正的 array**——要 array 方法請 `Array.from(...)` 或 `[...nodeList]`。

  
### 3. 修改元素內容

📄 **寫進程式檔**：`modify.js`

```text
const el = document.querySelector(".title");

// 純文字（推薦、安全）
el.textContent = "Hello";

// HTML（小心 XSS！）
el.innerHTML = "<b>粗體</b>";   // 會把字串當 HTML 解析

// 屬性
el.setAttribute("data-id", "42");
el.getAttribute("data-id");      // "42"
el.removeAttribute("data-id");

// data-* 屬性簡寫
el.dataset.userId = "42";        // 等同 data-user-id
el.dataset.userId;               // "42"

// value（input / textarea / select）
input.value = "abc";
input.value;
```

> 💡 **`textContent` vs `innerHTML`**：使用者輸入永遠用 `textContent`——`innerHTML` 會執行 `<script>` 標籤、是 XSS 漏洞主因（L21 會講）。

  
### 4. **class 操作** —— classList

```text
el.classList.add("active");
el.classList.remove("active");
el.classList.toggle("active");        // 有就移除、沒有就加
el.classList.contains("active");      // 檢查
el.classList.replace("old", "new");
```

  
### 5. 樣式操作

```text
// 行內樣式
el.style.color = "red";
el.style.fontSize = "20px";       // 注意：CSS 的 font-size → fontSize
el.style.cssText = "color: red; font-size: 20px;";

// CSS 變數
el.style.setProperty("--my-color", "blue");
getComputedStyle(el).getPropertyValue("--my-color");
```

> 💡 **改樣式優先用 class**——`el.classList.add("highlight")` 比直接改 `style` 好維護。CSS 寫好變化、JS 只切換 class。

  
### 6. 新增 / 移除 / 移動元素

📄 **寫進程式檔**：`dom-tree.js`

```text
// 建立新元素
const div = document.createElement("div");
div.textContent = "新元素";
div.classList.add("card");

// 插入位置
parent.appendChild(div);          // 加到最後
parent.prepend(div);              // 加到最前
target.before(div);               // target 之前
target.after(div);                // target 之後
target.replaceWith(div);          // 取代 target

// 移除自己
el.remove();

// 複製
const clone = el.cloneNode(true);  // true = 深複製
```

  
### 7. 在 DOM 樹中走訪

```text
el.parentElement;          // 爸爸
el.children;               // 直接小孩（HTMLCollection）
el.childNodes;             // 含 text node 的小孩（NodeList）
el.firstElementChild;      // 第一個小孩
el.lastElementChild;       // 最後一個
el.nextElementSibling;     // 下一個兄弟
el.previousElementSibling; // 上一個兄弟
el.closest(".card");       // 往上找最近的祖先（包含自己）
```

  
### 8. **event（事件）監聽** —— addEventListener

📄 **寫進程式檔**：`event.js`

```text
const btn = document.querySelector("button");

btn.addEventListener("click", (e) => {
  e.preventDefault();         // 阻止預設行為（form 送出、a 跳轉）
  e.stopPropagation();        // 阻止「冒泡」
  console.log(e.target);      // 真正被點的元素
  console.log(e.currentTarget); // 綁定監聽的元素
});

// 選項：once / passive / capture
el.addEventListener("click", fn, { once: true });        // 觸發一次後自動移除
el.addEventListener("scroll", fn, { passive: true });    // 不會 preventDefault、效能好
el.addEventListener("click", fn, { capture: true });     // 捕獲階段（少用）

// 移除（必須給「同一個函式參考」）
function onClick(e) { /*...*/ }
btn.addEventListener("click", onClick);
btn.removeEventListener("click", onClick);
```

  
### 9. **bubbling / capturing / delegation** —— 事件流三階段

```text
事件從外層往內層（capture）→ 命中目標 → 從內層往外層冒泡（bubble）

document
  └─ body
      └─ ul                   ← 冒泡會經過這
          └─ li               ← 點這個
              └─ button       ← 真正被點

點 button：
  1. capture 階段：document → body → ul → li → button
  2. target 階段：button
  3. bubble 階段：button → li → ul → body → document
```

**event delegation（事件委派）**—— 在父元素監聽、靠 `e.target` 判斷。

```text
// ✗ 100 個 li 綁 100 個 listener（吃記憶體）
document.querySelectorAll("li").forEach(li => {
  li.addEventListener("click", handler);
});

// ✓ 在 ul 綁一個、靠 e.target 判斷
ul.addEventListener("click", (e) => {
  if (e.target.matches("li")) {
    handler(e.target);
  }
});
```

> 💡 **delegation 兩大好處**：(1) 一個 listener 處理所有子元素、省記憶體 (2) 新增的子元素自動「綁定」——不用重綁。

  
### 10. **自訂事件**（CustomEvent）

📄 **寫進程式檔**：`custom-event.js`

```text
// 發出自訂事件
const event = new CustomEvent("user-login", {
  detail: { name: "招財", id: 42 },
  bubbles: true
});
el.dispatchEvent(event);

// 別處監聽
document.addEventListener("user-login", (e) => {
  console.log(e.detail.name);   // "招財"
});
```

**用途**：自製元件 / 跨元件溝通 / 不想 hard-code 依賴。

  
### 11. 觀察 API（Observers）

📄 **寫進程式檔**：`observer.js`

```text
// 1. IntersectionObserver —— 元素進視窗時觸發（lazy load）
const io = new IntersectionObserver(entries => {
  entries.forEach(e => {
    if (e.isIntersecting) {
      e.target.src = e.target.dataset.src;   // 載入圖片
      io.unobserve(e.target);
    }
  });
}, { threshold: 0.5 });

document.querySelectorAll("img[data-src]").forEach(img => io.observe(img));

// 2. MutationObserver —— 監聽 DOM 變化
const mo = new MutationObserver(mutations => {
  mutations.forEach(m => console.log(m.type, m.addedNodes));
});
mo.observe(el, { childList: true, subtree: true });

// 3. ResizeObserver —— 監聽大小變化
const ro = new ResizeObserver(entries => {
  entries.forEach(e => console.log(e.contentRect));
});
ro.observe(el);
```

> 💡 **lazy load 圖片**：用 `IntersectionObserver` 是 2026 標準寫法、取代以前監聽 scroll 的爛做法。

  
### 5 個常見坑

```text
❌ 1. 用 innerHTML 塞使用者輸入
   → XSS 漏洞
   → 修：用 textContent、必要時 DOMPurify.sanitize

❌ 2. addEventListener 沒留 reference 想 remove
   → 修：把函式存起來、用同一個 reference

❌ 3. 在迴圈內綁 1000 個 listener
   → 修：用 event delegation

❌ 4. NodeList 想用 .map / .filter
   → NodeList 沒有 .map（只有 forEach）
   → 修：[...nodeList].map(...) 或 Array.from(...)

❌ 5. 元素還沒生出來就 querySelector
   → DOM 沒載完、回 null
   → 修：<script defer> 或 DOMContentLoaded 內執行
```

  
### 🎯 重點回顧

- **`querySelector` / `querySelectorAll` 是 2026 主力**——CSS 選擇器寫法
- **`textContent` 永遠優先**——`innerHTML` 是 XSS 源頭
- **classList 是好朋友**——別直接改 style
- **事件流三階段**：capture → target → bubble
- **event delegation 省記憶體 + 自動處理新元素**
- **IntersectionObserver 取代監聽 scroll**——lazy load、無限滾動
