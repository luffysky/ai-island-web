**object（物件）= 一堆「key-value（鍵值對）」的集合**。`{ name: "招財", age: 3 }` 就是一個物件—— "name" 是 key、"招財" 是 value。除了 object、JS 還有 **Map**、**Set**、**WeakMap**、**WeakSet**——它們都在裝資料、但各有用途。

> 💡 **這課的價值**：知道「什麼時候用 Map、什麼時候用 Object、什麼時候用 Set」就贏過 80% 的 JS 開發者。**這是面試愛問題**。

  
### 1. 物件基礎操作

📄 **寫進程式檔**：`obj-basic.js`

```text
const user = { name: "招財", age: 3, role: "hamster" };

// 取值
user.name;          // "招財"
user["name"];       // "招財"（key 有空格 / 特殊字元才用這個）

// 改值
user.age = 4;

// 新增
user.email = "z@z.tw";

// 刪除
delete user.role;

// 檢查 key 存在
"name" in user;     // true
user.hasOwnProperty("name");   // true（舊寫法）
Object.hasOwn(user, "name");   // true（ES2022 新寫法、推薦）
```

  
### 2. Object 的靜態方法

```text
const user = { name: "招財", age: 3 };

Object.keys(user);     // ["name", "age"]
Object.values(user);   // ["招財", 3]
Object.entries(user);  // [["name","招財"], ["age", 3]]

// 從 entries 還原回物件
Object.fromEntries([["a", 1], ["b", 2]]);   // { a: 1, b: 2 }

// 複製（淺）
const copy = { ...user };
const copy2 = Object.assign({}, user);

// 深複製（ES2022）—— 連巢狀都複製
const deep = structuredClone(user);

// 凍結（不可改）
Object.freeze(user);
user.age = 99;       // 嚴格模式報錯、否則無聲失敗
```

> 💡 **淺複製 vs 深複製**：淺只複製第一層、巢狀物件還是共用。**有巢狀就用 `structuredClone`**。

  
### 3. 動態 key + 物件簡寫

📄 **寫進程式檔**：`shorthand.js`

```text
// 動態 key（computed property names）
const key = "name";
const obj = { [key]: "招財" };   // { name: "招財" }

// 屬性簡寫（key 跟變數同名）
const name = "招財";
const age = 3;
const u = { name, age };          // 等於 { name: name, age: age }

// 方法簡寫
const obj2 = {
  greet() { return "hi"; }       // 等於 greet: function() { ... }
};
```

  
### 4. **Map** —— 比 Object 更好的「字典」容器

`Map` 跟 Object 看起來像、但 Map **強很多**。

📄 **寫進程式檔**：`map.js`

```text
const map = new Map();

map.set("name", "招財");
map.set(42, "answer");            // key 可以是任何型別！
map.set({ a: 1 }, "obj key");     // 物件當 key 也行
map.set(true, "boolean key");

map.get("name");    // "招財"
map.has("name");    // true
map.delete("name");
map.size;            // 3

// 迭代（保留「插入順序」、object 不保證）
for (const [k, v] of map) {
  console.log(k, v);
}

// 從陣列建立
const m2 = new Map([
  ["a", 1],
  ["b", 2]
]);
```

  
### 5. Map vs Object —— 什麼時候用誰？

```text
場景                            選擇
key 是字串 / Symbol、固定結構    Object
key 是任意型別（數字、物件）      Map
要保留插入順序                    Map（雖然現代 Object 也保留、但 Map 是規範）
頻繁新增 / 刪除                  Map（效能好）
要 .size 屬性                     Map
要當 JSON 序列化                  Object（Map 不行直接 JSON.stringify）
要 hash map / cache               Map（效能好、沒有 prototype 污染）
```

> 💡 **白話**：**有「結構固定」的資料用 Object（像 user、order）、有「字典 / 索引」用 Map（像 id → 物件 的對應）**。

  
### 6. **Set** —— 唯一值集合

`Set` = 「不會重複的清單」。

📄 **寫進程式檔**：`set.js`

```text
const set = new Set([1, 2, 2, 3, 3, 3]);
set.size;   // 3（自動去重）

set.add(4);
set.add(4);     // 已存在、無效果
set.has(1);     // true
set.delete(2);

// 迭代
for (const v of set) {
  console.log(v);
}

// 最常用：陣列去重
const unique = [...new Set([1, 2, 2, 3, 3, 4])];
// [1, 2, 3, 4]
```

  
### 7. **Set 集合運算（ES2025）**—— 終於有了

```text
const a = new Set([1, 2, 3]);
const b = new Set([2, 3, 4]);

a.union(b);              // {1, 2, 3, 4}（聯集）
a.intersection(b);       // {2, 3}（交集）
a.difference(b);         // {1}（差集）
a.symmetricDifference(b); // {1, 4}（對稱差）
a.isSubsetOf(b);         // false
a.isSupersetOf(b);       // false
a.isDisjointFrom(b);     // false（有沒有共同元素）
```

  
### 8. **WeakMap / WeakSet** —— 不阻止 garbage collection（垃圾回收）的版本

普通 Map 把 key 「**抓緊**」、key 對應的物件就回收不掉。WeakMap 「**鬆鬆抓**」——物件沒人用就讓它走。

📄 **寫進程式檔**：`weakmap.js`

```text
const wm = new WeakMap();
let obj = { x: 1 };
wm.set(obj, "data");

obj = null;   // obj 沒人用了
// → wm 裡那一對自動清掉

// WeakMap 限制
// 1. key 必須是 object
// 2. 不能迭代（for...of 不能用）
// 3. 沒有 size 屬性
```

**WeakMap 用途**：

```text
✅ 給物件「附加 metadata」、不阻止它被 GC
   例：DOM 節點 → 額外資料的對應
✅ 模擬「私有屬性」
✅ cache（物件 → 計算結果）
```

> 💡 **白話**：WeakMap 像「臨時貼便利貼」—— 紙條（key）丟了、便利貼自動撕掉。普通 Map 像「焊上去」、紙條都丟了還焊著。

  
### 9. 實戰範例：用 Map 做快取

📄 **寫進程式檔**：`cache.js`

```text
function memoize(fn) {
  const cache = new Map();
  return function(arg) {
    if (cache.has(arg)) {
      return cache.get(arg);
    }
    const result = fn(arg);
    cache.set(arg, result);
    return result;
  };
}

const slowSquare = n => {
  console.log("計算...");
  return n * n;
};

const fastSquare = memoize(slowSquare);
fastSquare(5);   // 計算... → 25
fastSquare(5);   // 25（直接拿快取、沒計算）
```

  
### 10. JSON —— 物件的「文字版」

📄 **寫進程式檔**：`json.js`

```text
const user = { name: "招財", age: 3 };

// 物件 → JSON 字串
const json = JSON.stringify(user);
// '{"name":"招財","age":3}'

// 美化版
JSON.stringify(user, null, 2);

// JSON 字串 → 物件
const obj = JSON.parse(json);

// 注意：Map / Set / function / undefined 不能 JSON
JSON.stringify({ a: undefined });  // '{}'（undefined 被丟掉）
JSON.stringify(new Map([["a", 1]]));   // '{}'（Map 變空物件）
```

  
### 5 個常見坑

```text
❌ 1. 想用 Object 當 hash map、踩 prototype 污染
   → const m = {}; m.toString = ... 撞到內建方法
   → 修：用 Map、或 Object.create(null)

❌ 2. 把 Map 直接 JSON.stringify
   → 變空物件 {}
   → 修：先轉 [...map] 陣列再 stringify

❌ 3. WeakMap 想 iterate
   → WeakMap 不能迭代（為了 GC 安全）
   → 修：需要迭代用普通 Map

❌ 4. Object.keys() 拿不到 Symbol key
   → Symbol key 要用 Object.getOwnPropertySymbols()
   → 修：知道有這差別就好

❌ 5. structuredClone 丟函式進去爆
   → structuredClone 不支援 function / DOM / class
   → 修：純資料用它、有函式用其他方法
```

  
### 🎯 重點回顧

- **Object**：結構固定資料、JSON 友善
- **Map**：字典 / cache、任意型別 key、保留順序、有 .size
- **Set**：唯一值集合、最常拿來「陣列去重」
- **WeakMap / WeakSet**：不阻止 GC、key 必須是物件、不能迭代
- **structuredClone 是深複製救星**（ES2022）—— 取代以前 `JSON.parse(JSON.stringify())` 的 hack
