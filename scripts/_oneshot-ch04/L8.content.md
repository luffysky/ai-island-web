**class（類別）= 物件的「模板」**。先寫一份「狗的設計圖」、再用它做出一隻隻具體的狗（instance、實例）。ES2015 之前 JS 沒有 class、要用 prototype 模擬——超痛苦。class 是「**糖衣**」、本質還是 prototype（下一課 L9 會拆穿）。

> 💡 **這課的價值**：React class component / Web Components / ORM 模型 / 遊戲開發都用 class。不會寫 class 等於少了一隻手。

  
### 1. 基本 class

📄 **寫進程式檔**：`Animal.js`

```text
class Animal {
  // 1. 公開欄位（public field）
  name;

  // 2. 私有欄位（private field、ES2022）—— # 開頭
  #secret = "hidden";

  // 3. 靜態欄位（static field）—— 屬於「class 本身」、不是 instance
  static kingdom = "Animalia";

  // 4. constructor（建構子）—— new 時自動呼叫
  constructor(name) {
    this.name = name;
  }

  // 5. 一般方法（method）
  greet() {
    return `Hello, I'm ${this.name}`;
  }

  // 6. getter / setter —— 看起來像屬性、其實是函式
  get upperName() { return this.name.toUpperCase(); }
  set upperName(v) { this.name = v.toLowerCase(); }

  // 7. 靜態方法 —— 屬於 class、不需要 instance
  static create(name) {
    return new Animal(name);
  }
}

// 使用
const a = new Animal("招財");
a.greet();           // "Hello, I'm 招財"
a.upperName;         // "招財".toUpperCase() ← getter
a.upperName = "ROCKY";   // 觸發 setter
console.log(Animal.kingdom);   // "Animalia"（從 class 取）
const b = Animal.create("小白");   // 靜態方法
```

> 💡 **`this` 在 class 內**：`this` 指向「**正在用的那個 instance**」。`a.greet()` 的 `this` 是 a、`b.greet()` 的 `this` 是 b。

  
### 2. **private fields（私有欄位）**`#` —— ES2022 才有

```text
class Counter {
  #count = 0;        // 外部無法存取

  increment() {
    this.#count++;   // 內部能用
  }

  get value() {
    return this.#count;
  }
}

const c = new Counter();
c.increment();
c.value;          // 1 ✓
c.#count;         // SyntaxError ← 外部禁止
```

> 💡 **`#` 不是「慣例 _ 開頭」**——`_name` 只是大家「假裝」是 private、其實外面還是能改。`#name` 是**真的 private**、語法層級擋住。

  
### 3. **inheritance（繼承）**`extends` —— 「繼承」父類別

📄 **寫進程式檔**：`Dog.js`

```text
class Animal {
  constructor(name) {
    this.name = name;
  }
  greet() {
    return `Hi, I'm ${this.name}`;
  }
}

// Dog 繼承 Animal
class Dog extends Animal {
  constructor(name, breed) {
    super(name);          // 必須先呼叫 super(...)、再用 this
    this.breed = breed;
  }

  bark() {
    return "汪";
  }

  // 覆寫父類方法
  greet() {
    return super.greet() + `, I'm a ${this.breed}`;
  }
}

const d = new Dog("小白", "柴犬");
d.greet();   // "Hi, I'm 小白, I'm a 柴犬"
d.bark();    // "汪"
d.name;      // "小白"（繼承自 Animal）
```

**`super` 兩種用法**：

```text
1. super(...) —— 在 constructor 內、呼叫父類的 constructor
   ⚠️ 必須在用 this 之前呼叫、不然 ReferenceError

2. super.method() —— 呼叫父類的方法
   用在子類覆寫方法後、還想用父類版
```

  
### 4. **static（靜態）**—— 屬於 class 本身

```text
class MathUtil {
  static PI = 3.14159;

  static circleArea(r) {
    return MathUtil.PI * r * r;
  }
}

// 不用 new、直接從 class 用
MathUtil.PI;              // 3.14159
MathUtil.circleArea(5);   // 78.54

// instance 拿不到靜態
const m = new MathUtil();
m.PI;        // undefined
m.circleArea(5);   // TypeError
```

**何時用 static？**

```text
✅ 工具函式：不需要 instance 狀態
✅ Factory：static create() 包 new
✅ 常數：static API_URL = "..."
✅ 計數器：static instanceCount = 0
```

  
### 5. **getter / setter** —— 看起來像屬性的方法

📄 **寫進程式檔**：`getter.js`

```text
class Temperature {
  #celsius = 0;

  get c() { return this.#celsius; }
  set c(v) { this.#celsius = v; }

  // 華氏（getter / setter 換算）
  get f() { return this.#celsius * 1.8 + 32; }
  set f(v) { this.#celsius = (v - 32) / 1.8; }
}

const t = new Temperature();
t.c = 25;    // 觸發 setter
t.f;         // 77（getter 算出來）
t.f = 100;   // 觸發 setter、c 變成約 37.78
t.c;         // 37.78
```

> 💡 **getter / setter 用途**：(1) 隱藏內部表示 (2) 加驗證邏輯 (3) 計算屬性。**但濫用會讓 debug 困難**——看起來是屬性、其實是函式。

  
### 6. class 是 expression 也是 declaration

```text
// declaration
class Foo {}

// expression（可以匿名）
const Bar = class {};
const Baz = class NamedBaz {};   // 有名字、但外面用 Baz

// 立即建 instance
const instance = new class {
  greet() { return "hi"; }
}();
```

  
### 7. **abstract class（抽象類）**—— JS 沒原生、但能模擬

```text
class Shape {
  constructor() {
    if (new.target === Shape) {
      throw new Error("Shape 是抽象類、不能直接 new");
    }
  }
  area() {
    throw new Error("子類必須實作 area()");
  }
}

class Circle extends Shape {
  constructor(r) { super(); this.r = r; }
  area() { return Math.PI * this.r ** 2; }
}

new Shape();    // ✗ 報錯
new Circle(5);  // ✓
```

  
### 8. 2026 業界 class 用得多嗎？

```text
✅ 真實大量用 class 的場景
   - Web Components（必須繼承 HTMLElement）
   - ORM 模型（Prisma、TypeORM）
   - 遊戲引擎（實體系統）
   - 狀態機 / 複雜事件處理
   - Node.js stream、error 類別

⚠️ 已經偏好用「函式 + closure」的場景
   - React 元件（hook 取代 class）
   - 純邏輯模組（API client、helper）
   - Redux / Zustand 等狀態管理
```

> 💡 **白話**：**class 沒有死、但用得比 2015-2018 少很多**。寫 React 用 function + hook、寫 Web Components 還是用 class。

  
### 9. instanceof —— 檢查 class 關係

```text
const d = new Dog("小白", "柴犬");

d instanceof Dog;       // true
d instanceof Animal;    // true（繼承也算）
d instanceof Object;    // true（最頂層）
```

> 💡 **`instanceof` 的小陷阱**：跨 iframe / window 會失敗——兩個 window 的 `Array` 不是「同一個」class。

  
### 5 個常見坑

```text
❌ 1. 沒呼叫 super(...) 就用 this
   → ReferenceError: Must call super before this
   → 修：constructor 第一行先呼叫 super(...)

❌ 2. arrow function 當 method
   → class Foo { greet = () => ... } 每個 instance 都重新建函式、吃記憶體
   → 修：用 method 簡寫 greet() { ... }（會放原型上、共用）

❌ 3. private 用 _ 開頭以為夠安全
   → _name 只是約定、外面還是能改
   → 修：真要 private 用 #name

❌ 4. extends 多重繼承
   → JS class 只能單繼承
   → 修：用 mixin 或 composition

❌ 5. instance 想叫靜態方法
   → instance.staticMethod() ← TypeError
   → 修：Class.staticMethod()
```

  
### 🎯 重點回顧

- **class 是 prototype 的糖衣**——L9 會拆穿
- **`#` 是真 private**（ES2022）—— `_` 只是慣例
- **`super(...)` 必須在 constructor 第一個 this 之前**
- **static 屬於 class 本身**、instance 拿不到
- **2026 React 已不用 class**——但 Web Components / ORM / 遊戲還是必用
