**瀏覽器儲存（browser storage）= 把資料留在使用者裝置上、不用每次重新跟 server 要**。JS 有 4 種主要儲存方式——**localStorage**、**sessionStorage**、**cookie**、**IndexedDB**——各有用途、不能亂選。

> 💡 **這課的價值**：選錯儲存方式 = 安全漏洞或效能災難。**密碼放 localStorage 是 90% 新手的錯**。

  
### 1. **localStorage** —— 永久保存（除非清快取）

📄 **寫進程式檔**：`local.js`

```text
// 存
localStorage.setItem("theme", "dark");
localStorage.setItem("token", "abc123");

// 取
localStorage.getItem("theme");        // "dark"
localStorage.getItem("notExists");    // null

// 刪
localStorage.removeItem("theme");

// 清全部
localStorage.clear();

// 數量
localStorage.length;

// 拿所有 key
Object.keys(localStorage);
```

**規格**：

```text
✅ 容量：5-10 MB（各家瀏覽器略不同）
✅ 跟著瀏覽器 + 網域永久存（user 不清就一直在）
✅ 同步 API（會卡住主執行緒、別存大資料）
⚠️ 只能存「字串」—— 物件要 JSON.stringify
🚫 任何 JS 都讀得到（XSS 就完蛋）
```

  
### 2. 只能存字串 —— 物件要轉 JSON

```text
// ✗ 直接存物件 → 變成 "[object Object]"
const user = { name: "招財", age: 3 };
localStorage.setItem("user", user);
localStorage.getItem("user");   // "[object Object]" ← 沒救了

// ✓ 用 JSON
localStorage.setItem("user", JSON.stringify(user));
const u = JSON.parse(localStorage.getItem("user"));
```

> 💡 **包成 helper 比較好用**：

```text
const storage = {
  set(key, value) {
    localStorage.setItem(key, JSON.stringify(value));
  },
  get(key, fallback = null) {
    try {
      const v = localStorage.getItem(key);
      return v === null ? fallback : JSON.parse(v);
    } catch {
      return fallback;
    }
  }
};
```

  
### 3. **sessionStorage** —— 分頁關閉就消失

```text
sessionStorage.setItem("tmp", "value");
sessionStorage.getItem("tmp");
sessionStorage.removeItem("tmp");
sessionStorage.clear();
```

```text
✅ API 跟 localStorage 完全一樣
✅ 容量 5-10 MB
🆚 差別：分頁關閉就消失（不跨分頁）

✅ 適合：
   - 結帳流程中暫存資料
   - 表單草稿
   - 一次性導引步驟
```

  
### 4. **cookie** —— HTTP 古老的儲存方式

```text
// 寫入（看起來怪、但這是規範）
document.cookie = "token=abc; max-age=3600; path=/; secure; samesite=strict";

// 讀取（讀全部、要自己 parse）
document.cookie;   // "token=abc; theme=dark"

// 刪除（把 max-age 設 0）
document.cookie = "token=; max-age=0; path=/";
```

**cookie 的特色**：

```text
✅ 容量小：4 KB（不能存大東西）
✅ 每次 HTTP 請求自動帶上 → server 認得使用者
✅ 可以設「httpOnly」→ JS 完全讀不到（防 XSS）
✅ 可以設「secure」→ 只走 HTTPS
✅ 可以設「samesite=strict」→ 防 CSRF
```

  
### 5. **httpOnly cookie** —— 存 token 的正確方法

```text
🔴 ✗ localStorage 存 session token
   const token = localStorage.getItem("token");
   → 任何 XSS 都能讀

🟢 ✓ httpOnly cookie 存 token
   server: Set-Cookie: token=xxx; HttpOnly; Secure; SameSite=Strict;
   → JS 看不到、只跟 HTTP 自動走
```

> 💡 **白話**：**敏感 token（session）一律 httpOnly cookie**——JS 摸不到、XSS 也偷不走。**localStorage 適合存「不敏感」的偏好設定**（theme、language、不重要的 cache）。

  
### 6. **IndexedDB** —— 瀏覽器內建的「資料庫」

容量大（百 MB 以上）、支援結構化資料、非同步 API。

📄 **寫進程式檔**：`idb.js`

```text
// 原生 API 超複雜、新手很痛苦
const req = indexedDB.open("myDB", 1);
req.onupgradeneeded = e => {
  const db = e.target.result;
  db.createObjectStore("users", { keyPath: "id" });
};
req.onsuccess = e => {
  const db = e.target.result;
  const tx = db.transaction("users", "readwrite");
  tx.objectStore("users").put({ id: 1, name: "招財" });
};
```

🖥️ **在終端機輸入**（用簡化套件）：

```text
npm install idb-keyval
```

📄 **寫進程式檔**：`idb-simple.js`

```text
import { set, get, del } from "idb-keyval";

await set("user", { name: "招財", age: 3 });
const u = await get("user");
await del("user");
```

```text
✅ 適合：
   - 離線 App（PWA）
   - 大量結構化資料
   - 上傳前暫存的檔案
   - AI 模型權重快取

⚠️ 不適合：
   - 小東西（用 localStorage 就好）
   - 同步邏輯（IndexedDB 全是 async）
```

  
### 7. **Cache API** —— 給 Service Worker 用

```text
// 把 Response 存起來、給離線用
const cache = await caches.open("v1");
await cache.add("/api/data");

// 取出來
const resp = await cache.match("/api/data");
if (resp) {
  const data = await resp.json();
}

// 列出所有 cache
const keys = await caches.keys();
```

> 💡 **白話**：Cache API 不是存「資料」、是存「整個 HTTP Response」——適合 PWA 離線使用（L16 詳教）。

  
### 8. 5 種儲存方式比較表

```text
方式            容量     存活        API     場景
localStorage    5-10MB   永久        同步    偏好設定、非敏感快取
sessionStorage  5-10MB   分頁關閉    同步    一次性暫存
cookie          4KB      可設        同步    session token（httpOnly）
IndexedDB       100MB+   永久        async   離線資料、大檔案
Cache API       100MB+   永久        async   PWA Response 快取
```

  
### 9. 實戰：偵測 localStorage 變化（跨分頁）

```text
// 分頁 A 改了 localStorage
localStorage.setItem("theme", "dark");

// 分頁 B 自動收到事件
window.addEventListener("storage", (e) => {
  console.log(e.key);      // "theme"
  console.log(e.oldValue); // 舊值
  console.log(e.newValue); // "dark"
});
```

**用途**：使用者在分頁 A 登出、分頁 B 自動跟著登出。

  
### 10. 儲存配額 + 檢查容量

```text
// 估計用了多少
const estimate = await navigator.storage.estimate();
console.log(estimate.usage, "/", estimate.quota);

// 申請「永久不被清」（重要資料）
await navigator.storage.persist();
```

  
### 5 個常見坑

```text
❌ 1. localStorage 存 session token
   → XSS 一爆全洩漏
   → 修：用 httpOnly cookie

❌ 2. localStorage 直接 setItem 物件
   → 變 "[object Object]"
   → 修：JSON.stringify / parse

❌ 3. JSON.parse 沒包 try / catch
   → user 改壞資料、整支爆
   → 修：包 try / catch 並 fallback

❌ 4. 把大量資料塞 localStorage
   → 同步 API 卡 UI、可能爆 5MB 配額
   → 修：超過 1MB 改用 IndexedDB

❌ 5. cookie 沒設 SameSite / Secure / HttpOnly
   → 三個都缺就是資安漏洞
   → 修：token 一定要全帶
```

  
### 🎯 重點回顧

- **localStorage** = 永久、5-10MB、同步、字串、適合偏好設定
- **sessionStorage** = 分頁關就消失、其他跟 localStorage 一樣
- **cookie** = 4KB 小、能設 HttpOnly + Secure + SameSite、適合 session token
- **IndexedDB** = 大資料、async、用 `idb-keyval` 套件最方便
- **敏感 token 永遠不放 localStorage**——用 httpOnly cookie
