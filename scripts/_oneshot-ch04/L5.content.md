**function（函式）= 一段可以重複使用的程式**。給它輸入、它回傳輸出。JS 的函式是「**第一等公民**」—— 可以當變數存、當參數傳、當回傳值。這是它最強的特性、也是 closure / callback / 高階函式的基礎。

> 💡 **這課的價值**：函式寫得好、整支程式就乾淨。寫得爛、後面 100 行全爆。**arrow function vs 一般 function 的差別、是 React Hook 必考**。

  
### 1. 三種寫法

📄 **寫進程式檔**：`functions.js`

```text
// 1. function declaration（函式宣告）—— 整個 hoist
function add(a, b) {
  return a + b;
}

// 2. function expression（函式表達式）—— 不 hoist
const add = function(a, b) {
  return a + b;
};

// 3. arrow function（箭頭函式、ES2015）—— 最常用
const add = (a, b) => a + b;

// 多行 arrow function 要加 { }
const greet = (name) => {
  const msg = `Hi ${name}`;
  return msg;
};

// 只有一個參數可以省略 ( )
const square = x => x * x;
```

> 💡 **選哪個？** 2026 慣例：(1) **頂層工具函式**用 `function`（會 hoist、stack trace 漂亮） (2) **callback / 短函式**用 arrow function (3) **class 方法**用 method 簡寫 `greet() {}`。

  
### 2. **arrow function（箭頭函式）的三大特殊性**

arrow function 不是 function 的「簡寫」——行為**不一樣**。

📄 **寫進程式檔**：`arrow.js`

```text
// 1. 沒有自己的 this（用外層的）
class Timer {
  constructor() {
    this.count = 0;

    // ✗ 一般函式：this 變成 setInterval 自己的（或 undefined）
    setInterval(function() {
      this.count++;   // 爆掉
    }, 1000);

    // ✓ arrow function：保留外層 this（class 的）
    setInterval(() => {
      this.count++;   // OK
    }, 1000);
  }
}

// 2. 沒有 arguments
const fn = () => arguments;   // ReferenceError
// 要拿全部參數用 rest：(...args) => args

// 3. 不能當 constructor（new 它會爆）
const Foo = () => {};
new Foo();   // TypeError: Foo is not a constructor
```

> 💡 **記法**：arrow function 是「**透明的**」——this / arguments 都不擋、直接看外面。一般 function 是「**會自己造 this 的**」——但 this 是什麼取決於「**怎麼呼叫**」。

  
### 3. 預設參數 + 解構參數（模擬命名參數）

📄 **寫進程式檔**：`params.js`

```text
// 預設值（ES2015）
function greet(name = "訪客", hour = new Date().getHours()) {
  return `${hour} 時、哈囉 ${name}`;
}

greet();             // 用兩個預設值
greet("招財");        // 只覆蓋 name
greet("招財", 9);     // 兩個都給

// 解構參數（模擬「命名參數」）
function createUser({ name, age = 0, role = "user" } = {}) {
  return { name, age, role };
}

createUser({ name: "招財", age: 3 });
// 比 createUser("招財", 3, "user") 清楚多了——不用記參數順序

createUser();   // 加 = {} 防止傳 undefined 爆掉
```

> 💡 **解構參數 = 命名參數**：函式呼叫者不用記順序、只要記名字。**API 設計超過 3 個參數就改用物件**。

  
### 4. **this** —— JS 最坑的概念

`this` 的值**取決於「怎麼呼叫」、不是「怎麼定義」**。

📄 **寫進程式檔**：`this.js`

```text
const obj = {
  name: "招財",
  greet() { return this.name; }
};

obj.greet();                  // "招財" ← this = obj

const g = obj.greet;          // 把方法拆出來
g();                          // undefined ← this = 全域

obj.greet.call({ name: "小白" });   // "小白" ← 手動指定 this
obj.greet.bind({ name: "小黑" })();  // "小黑" ← 綁定後新函式
```

**4 條 this 規則**：

```text
1. new 呼叫           → this = 新建的物件
2. obj.fn()           → this = obj（誰呼叫就是誰）
3. fn.call/apply/bind(x) → this = x
4. 裸呼叫 fn()         → this = undefined（嚴格模式）/ 全域
```

**arrow function 不在這 4 條內**——它「**沒有自己的 this**」、抓外層的。

  
### 5. **call / apply / bind** —— 三兄弟差別

📄 **寫進程式檔**：`bind.js`

```text
function greet(greeting, punctuation) {
  return `${greeting}, ${this.name}${punctuation}`;
}

const me = { name: "招財" };

// call —— 馬上呼叫、參數一個一個傳
greet.call(me, "Hi", "!");           // "Hi, 招財!"

// apply —— 馬上呼叫、參數打包成陣列傳
greet.apply(me, ["Hi", "!"]);        // "Hi, 招財!"

// bind —— 不呼叫、回傳新函式（this 已綁定）
const greetMe = greet.bind(me);
greetMe("Yo", "~");                  // "Yo, 招財~"
```

**記法**：**C**all = **C**omma（逗號分隔）/ **A**pply = **A**rray（陣列）/ **B**ind = **B**uild（建新函式）。

  
### 6. **IIFE（Immediately Invoked Function Expression、立即執行函式）**

定義完馬上呼叫的函式——ES6 之前用來「隔離作用域」、現在比較少用。

📄 **寫進程式檔**：`iife.js`

```text
// IIFE 基本格式
(function() {
  const secret = "藏起來";
  console.log(secret);
})();

// arrow function 版
(() => {
  console.log("立即執行");
})();

// 帶參數
(function(name) {
  console.log(`Hi ${name}`);
})("招財");
```

> 💡 **2026 還用嗎？** ESM 自帶 scope、IIFE 用得少。但 bundler 輸出的 code、library 內部還會看到。

  
### 7. **higher-order function（高階函式）**—— 函式可當參數 / 回傳值

📄 **寫進程式檔**：`hof.js`

```text
// 函式當參數
function withLog(fn) {
  return function(...args) {
    console.log("呼叫了：", fn.name, args);
    return fn(...args);
  };
}

const loggedAdd = withLog(add);
loggedAdd(1, 2);   // 印 log + 回傳 3

// 經典範例：debounce（防抖）
function debounce(fn, delay) {
  let timer;
  return function(...args) {
    clearTimeout(timer);
    timer = setTimeout(() => fn(...args), delay);
  };
}

const handleScroll = debounce(() => {
  console.log("scrolled");
}, 300);

window.addEventListener("scroll", handleScroll);
```

> 💡 **高階函式是函式式程式設計的基石**——map / filter / reduce 全是高階函式。

  
### 8. 純函式（pure function）—— 寫得好的習慣

```text
✅ 純函式
   1. 相同輸入 → 相同輸出
   2. 不改外部狀態（no side effect）

範例：
const add = (a, b) => a + b;       // 純
const double = x => x * 2;          // 純

❌ 不純
let count = 0;
const inc = () => count++;          // 改外部變數
const log = msg => console.log(msg); // side effect
```

> 💡 **盡量寫純函式**——好測試、好預測、好除錯。

  
### 5 個常見坑

```text
❌ 1. 在 callback 用 function 然後 this 不對
   → setTimeout(function() { this.x }) 的 this 是錯的
   → 修：用 arrow function、自動繼承 this

❌ 2. 函式參數太多
   → fn(a, b, c, d, e) 呼叫時記不住順序
   → 修：超過 3 個參數改用 { } 物件

❌ 3. 預設值寫在型別錯誤的地方
   → function f(opts = null) { opts.name } 給 null 就爆
   → 修：function f({ name } = {}) {}

❌ 4. 改外部變數
   → 函式內 push 外部陣列、debug 地獄
   → 修：回傳新陣列、別動原陣列

❌ 5. arrow function 當 class method
   → 不能被原型繼承、每個 instance 都重新建
   → 修：class method 用 method 簡寫
```

  
### 🎯 重點回顧

- **arrow function 沒有自己的 this / arguments / 不能 new**——這是它跟 function 最大差別
- **this 取決於怎麼呼叫**——4 條規則 + arrow function 是例外
- **call / apply / bind**：call 逗號傳、apply 陣列傳、bind 不呼叫只綁定
- **解構參數模擬命名參數**——超過 3 個參數改用 `{ }`
- **盡量寫純函式**——好測試、好預測
