**fetch API = 瀏覽器內建的「跟伺服器要資料」方法**。取代了 2010 年代的 XMLHttpRequest（XHR）、是 2026 標準。`fetch(url).then(r => r.json())` 就是現代網頁拉資料的標準寫法。

> 💡 **這課的價值**：所有「網頁顯示後端資料」都靠 fetch。學會 fetch + Promise + AbortController、就能寫出穩健的網路請求。

  
### 1. 基本 GET 請求

📄 **寫進程式檔**：`get.js`

```text
// 最簡單的版本
async function getUsers() {
  const r = await fetch("/api/users");

  // ⚠️ fetch 不會因為 404 / 500 自動 reject
  if (!r.ok) {
    throw new Error(`HTTP ${r.status}`);
  }

  const data = await r.json();
  return data;
}

// 用法
getUsers()
  .then(users => console.log(users))
  .catch(err => console.error(err));
```

> 💡 **fetch 的大坑**：404 / 500 不會自動 reject——只有「**網路斷了**」或「**CORS 擋了**」才 reject。HTTP 錯誤要自己檢查 `r.ok`。

  
### 2. **Response 物件** —— fetch 回來的東西

```text
const r = await fetch("/api/users");

r.ok;          // true / false（status 200-299 才 true）
r.status;      // 200 / 404 / 500 / ...
r.statusText;  // "OK" / "Not Found"
r.headers;     // Headers 物件
r.headers.get("content-type");

// 取出 body（只能取一次！）
await r.json();        // 解析成物件
await r.text();        // 純文字
await r.blob();        // 二進位（圖片、檔案）
await r.formData();    // form data
await r.arrayBuffer(); // 二進位 buffer
```

**⚠️ body 只能取一次**——`r.json()` 後再 `r.text()` 會爆。要重用先 `r.clone()`。

  
### 3. **POST JSON** —— 送資料給伺服器

📄 **寫進程式檔**：`post.js`

```text
async function createUser(user) {
  const r = await fetch("/api/users", {
    method: "POST",
    headers: {
      "Content-Type": "application/json",   // 告訴 server「我送 JSON」
    },
    body: JSON.stringify(user)
  });

  if (!r.ok) {
    throw new Error(`HTTP ${r.status}: ${await r.text()}`);
  }

  return r.json();
}

createUser({ name: "招財", age: 3 });
```

  
### 4. **上傳檔案** —— FormData

```text
// 從 input[type=file] 拿檔案
const fileInput = document.querySelector("input[type=file]");
const fd = new FormData();
fd.append("file", fileInput.files[0]);
fd.append("caption", "我的圖");

await fetch("/api/upload", {
  method: "POST",
  body: fd   // ⚠️ 不要自己設 Content-Type！瀏覽器會帶 multipart/form-data; boundary=...
});
```

> 💡 **`FormData` 自動處理 multipart/form-data 編碼**——手動設 Content-Type 反而會壞。

  
### 5. **超時 + 取消** —— AbortController / AbortSignal.timeout

📄 **寫進程式檔**：`abort.js`

```text
// 手動取消
const ctrl = new AbortController();
fetch(url, { signal: ctrl.signal })
  .catch(err => {
    if (err.name === "AbortError") console.log("已取消");
  });

setTimeout(() => ctrl.abort(), 5000);   // 5 秒後取消

// 2024+ 簡寫：自動 timeout
await fetch(url, { signal: AbortSignal.timeout(5000) });

// 結合多個訊號
const combined = AbortSignal.any([userCancel.signal, AbortSignal.timeout(5000)]);
```

  
### 6. **串流回應** —— 處理 AI / 大檔案

AI API（OpenAI / Claude）回應都是串流——一塊塊文字傳來、不是等全部完成。

📄 **寫進程式檔**：`stream.js`

```text
async function streamAI() {
  const r = await fetch("/api/ai-chat");
  const reader = r.body.getReader();
  const decoder = new TextDecoder();

  while (true) {
    const { value, done } = await reader.read();
    if (done) break;

    const chunk = decoder.decode(value);
    console.log(chunk);   // 一塊塊出來
    document.querySelector("#output").textContent += chunk;
  }
}
```

> 💡 **這是 ChatGPT 網頁那個「字一個個跳出來」的技術**——SSE 或 streaming response。

  
### 7. 實戰：穩健的 API client（retry + 指數退避）

📄 **寫進程式檔**：`api.js`

```text
async function apiCall(url, opts = {}, retries = 3) {
  for (let i = 0; i < retries; i++) {
    try {
      const r = await fetch(url, {
        ...opts,
        signal: AbortSignal.timeout(10000),   // 10s timeout
      });

      if (!r.ok) {
        // 5xx 才 retry、4xx 不 retry
        if (r.status >= 500) throw new Error(`Server ${r.status}`);
        return { ok: false, status: r.status, error: await r.text() };
      }

      return { ok: true, data: await r.json() };

    } catch (err) {
      if (i === retries - 1) throw err;
      // 等 1s、2s、4s...
      await new Promise(r => setTimeout(r, 2 ** i * 1000));
    }
  }
}
```

  
### 8. **JSON** —— JS 跟 server 溝通的標準格式

```text
// JS object → JSON 字串
const json = JSON.stringify({ name: "招財", age: 3 });
// '{"name":"招財","age":3}'

// 美化版
JSON.stringify(obj, null, 2);

// JSON 字串 → JS object
const obj = JSON.parse(json);

// ⚠️ 注意
JSON.stringify({ a: undefined });          // '{}' ← undefined 被丟掉
JSON.stringify({ a: NaN });                 // '{"a":null}' ← NaN 變 null
JSON.stringify(new Date());                 // '"2026-05-28T..."' ← 變字串
JSON.stringify(new Map([["a", 1]]));        // '{}' ← Map 不支援
```

> 💡 **JSON parse 失敗會 throw**——記得包 try / catch、不然 user 輸入錯資料整支爆。

  
### 9. 其他連線方式（fetch 之外）

```text
🔌 WebSocket（雙向長連線）
   const ws = new WebSocket("wss://api.example.com");
   ws.addEventListener("message", e => console.log(e.data));
   ws.send("hello");
   → 即時聊天、線上遊戲、股票報價

📡 SSE（Server-Sent Events、伺服器單向推）
   const es = new EventSource("/api/events");
   es.addEventListener("message", e => console.log(e.data));
   → 通知、AI 串流、進度推送

🎥 WebRTC（P2P 視訊 / 語音 / 資料）
   const pc = new RTCPeerConnection();
   → 視訊通話、不經過 server 直接點對點
```

  
### 10. **CORS（Cross-Origin Resource Sharing、跨來源資源共享）**

```text
場景：你的網站 https://a.com 想 fetch 到 https://b.com/api
→ 瀏覽器擋下、除非 b.com 在 response header 帶：
   Access-Control-Allow-Origin: https://a.com
   或
   Access-Control-Allow-Origin: *
```

**白話**：CORS 是瀏覽器的安全機制——預設「不同網域不能互相要資料」、要 server 明確說「OK」才放行。

```text
❌ 寫前端時常見錯誤：
   "Access to fetch at '...' from origin '...' has been blocked by CORS policy"

✅ 修法（後端要做）：
   - 加 Access-Control-Allow-Origin header
   - OPTIONS preflight 要回 200
   - 或前端透過自家 proxy 中轉
```

  
### 5 個常見坑

```text
❌ 1. fetch 沒檢查 r.ok
   → 404 / 500 都被當成功
   → 修：if (!r.ok) throw new Error()

❌ 2. body 取兩次
   → r.json() 後再 r.text() 爆
   → 修：先 r.clone() 或選一個

❌ 3. POST FormData 自己設 Content-Type
   → 缺 boundary、server 解析失敗
   → 修：不要設 Content-Type、讓瀏覽器自己處理

❌ 4. 沒處理 timeout
   → server 掛掉就一直等
   → 修：AbortSignal.timeout(5000)

❌ 5. CORS 錯誤想用前端解
   → CORS 是 server 控制、前端改不了
   → 修：請後端加 header、或用代理 server
```

  
### 🎯 重點回顧

- **fetch 不會自動 reject HTTP 錯誤**——一定要檢查 `r.ok`
- **body 只能取一次**——要重用先 `r.clone()`
- **FormData 自動處理上傳**——不要手動設 Content-Type
- **AbortController / AbortSignal.timeout 處理取消 + 超時**
- **CORS 是 server 端設定**——前端遇到只能請後端加 header
- **AI 串流用 `r.body.getReader()`**——chunk by chunk 處理
