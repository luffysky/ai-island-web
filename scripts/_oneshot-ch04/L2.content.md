**variable（變數）= 一個可以存資料的名字**。寫程式時把「招財」「3」「true」這些值存進變數、之後就能用名字叫它出來。JS 有三種宣告方式：`var`、`let`、`const`——但**三個不是同義詞**、用錯會出 bug。

> 💡 **這課的價值**：99% 的新手都被 `var` 害過。學會 `let` / `const` 的差別 + **hoisting（變數提升）**、就能避開 80% 入門踩雷。

  
### 1. 一句話差別

```text
🟥 var   （1995 老古董）  函式作用域、可重複宣告、會 hoist
🟧 let   （ES2015 新人）  區塊作用域、不可重複宣告、不可重複「在同一層」
🟩 const （ES2015 新人）  跟 let 一樣、但宣告後不可重新指派
```

**2026 規則**：**const 預設、let 偶爾、var 不用**。

  
### 2. 三種寫法的差別

📄 **寫進程式檔**：`vars.js`

```text
// var：函式作用域（function scope）、可重複宣告
var name = "招財";
var name = "小白";   // 沒報錯（這就是問題）

// let：區塊作用域（block scope）、不可重複宣告
let age = 3;
// let age = 5;   // ✗ SyntaxError: 重複宣告

// const：宣告後不能重新指派
const pet = { name: "招財" };
// pet = {};            // ✗ TypeError
pet.name = "小白";        // ✓ 物件「內容」還是可改
```

> 💡 **const 的小陷阱**：const 是「**不能換綁定**」、不是「**內容不可變**」。`const arr = []` 後你還是可以 `arr.push(1)`。要完全不可改要用 `Object.freeze()`。

  
### 3. **scope（作用域）**—— 變數能被「看到」的範圍

```text
🟦 全域作用域（global scope）
   → 寫在最外層、整個檔案都看得到

🟦 函式作用域（function scope）
   → 寫在 function 裡面、只有那個 function 看得到
   → var 用這個

🟦 區塊作用域（block scope）
   → 寫在 { } 裡面（if / for / 自訂 block）
   → let / const 用這個
```

📄 **寫進程式檔**：`scope.js`

```text
function demo() {
  if (true) {
    var a = 1;   // var 不認 { }
    let b = 2;   // let 只在 { } 內活
    const c = 3;
  }
  console.log(a);  // 1 ← var 跳出來了
  console.log(b);  // ReferenceError ← let 看不到
}
```

> 💡 **var 不認 `{}`** 是它最大的鍋——你以為 `for (var i = 0; ...)` 的 `i` 只在迴圈內、其實它「漏」到整個函式。

  
### 4. **hoisting（變數提升）**—— JS 偷偷把宣告「搬到前面」

JS 執行前會先「掃一遍」、把所有 `var` / `function` 的**宣告**搬到最上面。

📄 **寫進程式檔**：`hoist.js`

```text
console.log(a);   // undefined（不是 ReferenceError）
var a = 10;

// JS 實際上把它解讀成：
var a;            // ← 宣告被提升到最前面
console.log(a);   // undefined
a = 10;           // ← 賦值留在原地
```

`let` / `const` 也會 hoist、但有「**TDZ（temporal dead zone、暫時死區）**」—— 宣告到實際執行那行之前、變數不能用。

```text
console.log(b);   // ReferenceError: Cannot access 'b' before initialization
let b = 20;
```

> 💡 **白話**：`var` 提早出生、預設給 `undefined`。`let` / `const` 雖然也提早登記、但不准你提早用、用了就報錯。**後者比較安全**。

  
### 5. function declaration 也會 hoist

📄 **寫進程式檔**：`fn-hoist.js`

```text
sayHi();   // 可以！ ← 因為 function 整個被提升

function sayHi() {
  console.log("hi");
}

// 但 function expression / arrow 不會
sayBye();   // TypeError: sayBye is not a function
const sayBye = () => console.log("bye");
```

  
### 6. **TDZ（暫時死區）**—— let / const 的「禁區」

```text
{
  // ⛔ TDZ 開始
  console.log(x);   // ReferenceError
  // ⛔ TDZ 還在
  let x = 5;       // ✅ TDZ 結束
  console.log(x);   // 5
}
```

**白話**：你已經宣告 `let x` 在這個 block 了、但實際執行到 `let x = 5` 之前用 x 就會報錯。**這是為了強迫你「先宣告、再使用」**。

  
### 7. 經典面試題：迴圈裡的 var

📄 **寫進程式檔**：`loop.js`

```text
// ✗ var 版：3 個 setTimeout 都印 3
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 100);
}
// 印出：3 3 3

// ✓ let 版：分別印 0 1 2
for (let i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 100);
}
// 印出：0 1 2
```

> 💡 **為什麼？** `var i` 整個迴圈共用一個 `i`、setTimeout 觸發時 `i` 已經是 3。`let i` 每次迴圈都產生一個新的 `i`、各自獨立。**這就是為什麼 2026 不寫 `var`**。

  
### 8. **closure（閉包）**—— 函式記住它「出生時」周圍的變數

JS 最強也最常考的概念。

📄 **寫進程式檔**：`closure.js`

```text
function makeCounter() {
  let count = 0;          // ← 內部變數

  return function() {     // ← 回傳一個函式
    count++;
    return count;
  };
}

const counter = makeCounter();
counter();   // 1
counter();   // 2
counter();   // 3
```

`makeCounter` 跑完照理說 `count` 應該消失、但回傳的內部函式「記住」了它——這就是 closure。

> 💡 **白話**：閉包就像「便當盒裡夾帶著媽媽的紙條」——函式即使離開家、還是帶著當初媽媽寫的變數一起走。**90% 的 React Hook bug 都跟 closure 有關**。

  
### 5 個常見坑

```text
❌ 1. 還在寫 var
   → 修：const 預設、需要重新賦值才用 let

❌ 2. const 以為「整個不能改」
   → const arr = []; arr.push(1); 是合法的
   → 修：只是「不能換綁定」、要完全不可變用 Object.freeze

❌ 3. 在 if / for 外面用裡面宣告的 let 變數
   → block scope 看不到
   → 修：移到外面宣告、或用 var（不建議）

❌ 4. for (var i = ...) 加 setTimeout 印錯數字
   → 修：把 var 改 let、每次迴圈獨立 binding

❌ 5. 「我宣告了為什麼 ReferenceError」
   → 撞到 TDZ 了
   → 修：宣告寫在使用之前、不要把 let 寫在底部
```

  
### 🎯 重點回顧

- **2026 預設用 const**、需要改值才用 let、**var 退場**
- **scope 三層**：global / function / block——var 認前兩個、let/const 三個都認
- **hoisting**：JS 偷偷把宣告搬到前面、但 let/const 有 TDZ 擋著不准提早用
- **for 迴圈一律用 let**——避免 setTimeout / event handler 拿到錯誤的值
- **closure 是 JS 靈魂**——函式記住出生環境、後面 L5 / L19 / L23 都會重看
