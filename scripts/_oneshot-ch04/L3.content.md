**型別（type）= 資料的「種類」**。JS 把所有資料分成兩大類：**原始型別（primitive、簡單值）** 跟 **物件型別（object、可以裝東西的容器）**。搞清楚這 8 種型別、80% 的「為什麼結果這麼奇怪」就解開了。

> 💡 **這課的價值**：JS 的型別系統有 30 年歷史包袱（`typeof null === "object"` 就是 1995 年的 bug），看懂規則才不會被它陰。

  
### 1. 7 種 primitive（原始型別）

「原始」= **不能再拆**、像積木最小單位。

```text
1. string     文字
2. number     數字
3. boolean    真假
4. undefined  沒給值
5. null       明確說「沒有」
6. bigint     超大整數（ES2020）
7. symbol     獨一無二的識別（ES2015）
```

📄 **寫進程式檔**：`primitives.js`

```text
// 1. string（字串）—— 用 " ' 或 ` 包起來
const a = "hello";
const b = 'hello';
const c = `hello, ${name}`;   // template literal（模板字串、可插變數）

// 2. number（數字）—— 整數、小數都是 number
const d = 42;
const e = 3.14;
const f = Infinity;           // 無限大
const g = NaN;                // Not a Number（不是數字）
const h = 0xff;               // 16 進位（255）
const i = 0b101;              // 2 進位（5）
const j = 1_000_000;          // 用 _ 分隔好讀（ES2021）

// 3. boolean（布林）—— 只有兩種
const k = true;
const l = false;

// 4. undefined —— 「我還沒給你值」
let m;
console.log(m);   // undefined

// 5. null —— 「我故意給你一個空」
const n = null;

// 6. bigint（大整數）—— 數字後面加 n
const o = 9007199254740993n;  // 超過 Number 安全範圍的整數

// 7. symbol —— 每個都獨一無二
const p = Symbol("id");
Symbol("id") === Symbol("id");   // false ← 兩個明明描述一樣
```

> 💡 **undefined vs null 差別**：`undefined` = 系統說「沒值」、`null` = 你說「沒值」。**90% 場景兩者可換、但要選一個風格貫穿全專案**。

  
### 2. object（物件型別）—— 可以裝東西的容器

```text
{ ... }       一般物件
[ ... ]       array（陣列）
function      函式
new Date()    日期
new Map()     Map
new Set()     Set
/abc/         RegExp（正規表示式）
```

**重點**：**陣列、函式、Date、Map、Set 全都是 object**——它們只是「特化的物件」。

  
### 3. `typeof` —— 檢查型別的 operator

📄 **寫進程式檔**：`typeof.js`

```text
typeof "hello";    // "string"
typeof 42;         // "number"
typeof true;       // "boolean"
typeof undefined;  // "undefined"
typeof 42n;        // "bigint"
typeof Symbol();   // "symbol"

typeof {};         // "object"
typeof [];         // "object"   ← 看不出是陣列
typeof null;       // "object"   ← 傳奇 bug
typeof (() => {}); // "function" ← 函式是特例
```

> 💡 **`typeof null === "object"`？** 這是 1995 年 Brendan Eich 寫 JS 第一版的 bug——後來想修發現太多 code 依賴這行為、只能留著。**「不是 bug、是 feature」的最佳範例**。

  
### 4. 判斷陣列的正確方法

```text
typeof [];                  // "object" ← 沒用
Array.isArray([1, 2, 3]);   // true ✅
Array.isArray({});          // false ✅
```

**永遠用 `Array.isArray()`、不要用 `typeof`**。

  
### 5. 值傳遞 vs 引用傳遞

JS 變數「裝」資料有兩種模式：

```text
🟦 原始型別 → 值傳遞（複製一份）
   把資料「印一份」給對方、改一邊另一邊不變

🟧 物件型別 → 引用傳遞（共用一個東西的兩個名字）
   兩個變數指向同一個「實體」、改一邊另一邊也變
```

📄 **寫進程式檔**：`reference.js`

```text
// 原始型別：複製
let a = 5;
let b = a;     // b 是 a 的「複製」
b = 10;
console.log(a);   // 5 ← a 沒變

// 物件型別：共用
const o1 = { name: "招財" };
const o2 = o1;     // o2 跟 o1 指向「同一個物件」
o2.name = "小白";
console.log(o1.name);   // "小白" ← o1 也變了！
```

> 💡 **這就是為什麼**：你把物件丟給函式、函式內部修改、外面也跟著變——函式拿到的是「同一個物件的另一個名字」。

  
### 6. **truthy / falsy（真假值）**—— 不是 boolean 也能當條件

JS 的 `if (...)` 不只接受 boolean、任何值都會被「自動轉成 true 或 false」。

```text
🔴 7 個 falsy（會被當 false）
   false / 0 / -0 / 0n / "" / null / undefined / NaN

🟢 其他全部 truthy（包括 "0" / [] / {}）
```

📄 **寫進程式檔**：`truthy.js`

```text
if ([])    console.log("yes");   // yes ← 空陣列是 truthy！
if ({})    console.log("yes");   // yes
if ("0")   console.log("yes");   // yes ← 字串 "0" 是 truthy
if ("false") console.log("yes"); // yes ← 字串 "false" 也是 truthy
if (0)     console.log("yes");   // 不執行
if ("")    console.log("yes");   // 不執行
```

  
### 7. 新手常踩的坑：`if (count)` 跟 0

```text
// ✗ 想檢查「count 有東西」
let count = 0;
if (count) console.log("有");   // 不會印 ← 0 是 falsy 啊！

// ✓ 改寫
if (count != null) ...           // null 跟 undefined 都擋
if (count !== undefined) ...
if (typeof count === "number") ...
```

> 💡 **`!=` 跟 `!==`**：`!=` 接受隱式轉換、`null != undefined` 是 false（因為兩者相等）。**用 `x != null` 一次擋 null + undefined 是常見小技巧**。

  
### 8. **template literal（模板字串）**—— 用 ` 包字串可以插變數

📄 **寫進程式檔**：`template.js`

```text
const name = "招財";
const age = 3;

// 老寫法
const msg1 = "我叫" + name + "、" + age + " 歲";

// template literal
const msg2 = `我叫 ${name}、${age} 歲`;

// 可以跨行
const html = `
  <div>
    <h1>${name}</h1>
    <p>${age} 歲</p>
  </div>
`;
```

  
### 5 個常見坑

```text
❌ 1. typeof [] 想判斷陣列
   → 結果是 "object"、看不出來
   → 修：用 Array.isArray()

❌ 2. typeof null === "object" 以為是 bug
   → 是 1995 年遺留行為、改不掉了
   → 修：用 x === null 直接比

❌ 3. 物件丟給函式以為是複製
   → 是引用、會被改到
   → 修：要複製用 { ...obj } 淺複製 / structuredClone(obj) 深複製

❌ 4. if (count) 想檢查有沒有 count
   → count = 0 會被當 false
   → 修：用 count != null 或明確比較

❌ 5. 0.1 + 0.2 !== 0.3
   → 浮點數本質、不是 JS bug
   → 修：金額計算用整數（以分為單位）
```

  
### 🎯 重點回顧

- **7 個 primitive**：string / number / boolean / undefined / null / bigint / symbol
- **object 是萬用容器**：陣列、函式、Date、Map、Set 全是它
- **typeof 看型別、但 null 跟 array 看不出來**——分別用 `=== null` 跟 `Array.isArray()`
- **原始型別值傳遞、物件型別引用傳遞**——函式內外是「同一個物件」
- **falsy 七件事**：false / 0 / -0 / 0n / "" / null / undefined / NaN
