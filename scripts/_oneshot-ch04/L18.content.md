**module（模組）= 把程式碼拆成多個檔案、互相引用**。JS 有兩種「模組系統」—— **CommonJS（CJS）** 跟 **ESM（ES Modules）**。一個是 Node 老牌、一個是 ES2015 後的新標準。**2026 寫新專案一律用 ESM**——但 npm 老套件還很多 CJS、衝突地獄是日常。

> 💡 **這課的價值**：搞錯模組系統 = `require is not defined` 或 `Cannot use import outside a module` 報錯一片。**這是新手裝 npm 套件後最常遇到的錯**。

  
### 1. **ESM（ES Modules、ECMAScript Modules）** —— 2015 後標準

📄 **寫進程式檔**：`math.js`

```text
// named export（命名匯出、可以多個）
export function add(a, b) { return a + b; }
export const PI = 3.14;
export class Calculator {}

// default export（預設匯出、一個檔案只能一個）
export default function multiply(a, b) { return a * b; }
```

📄 **寫進程式檔**：`main.js`

```text
// 引用
import multiply, { add, PI } from "./math.js";
//     ↑ default      ↑ named

// 全部當 namespace
import * as math from "./math.js";
math.add(1, 2);
math.PI;

// 只執行不取值（side effect）
import "./polyfill.js";

// 動態 import（lazy load）—— 回傳 Promise
const { add } = await import("./math.js");

// 改名
import { add as plus } from "./math.js";
```

  
### 2. **CommonJS（CJS）** —— Node.js 老牌

📄 **寫進程式檔**：`math.cjs`

```text
function add(a, b) { return a + b; }
const PI = 3.14;

module.exports = { add, PI };
// 或單一匯出
module.exports = function multiply(a, b) { return a * b; };
```

📄 **寫進程式檔**：`main.cjs`

```text
const { add, PI } = require("./math");
const multiply = require("./math");

// 動態 require（任何時候都行）
if (cond) {
  const fs = require("fs");
}
```

  
### 3. **兩者差別 —— 規則表**

```text
項目              ESM                  CommonJS
語法              import / export      require / module.exports
載入              非同步（靜態解析）    同步
解析時機           build / parse time    runtime
Tree-shaking      ✓ 可去除沒用的       ✗ 不行
top-level await    ✓                    ✗
瀏覽器             ✓ 原生支援           ✗（要 bundler）
動態載入           import("xxx")         require("xxx")
this              undefined            module.exports
__dirname          ✗ 用 import.meta.url ✓
2026 預設         ✓                    逐漸退場
```

> 💡 **白話**：**ESM = 現代 + 標準 + 瀏覽器原生支援、CJS = Node.js 古早寫法**。新專案一律 ESM、CJS 只在維護舊 code 時碰到。

  
### 4. 怎麼讓 Node.js 用 ESM？

📄 **寫進程式檔**：`package.json`

```text
{
  "name": "my-project",
  "type": "module",        ← 加這行
  "version": "1.0.0"
}
```

**加 `"type": "module"` 後**：

```text
✅ .js 檔自動當 ESM 解析
✅ import / export 可以用
✅ require / module.exports 報錯

⚠️ 想用 CJS 要把檔名改 .cjs
```

**反過來**：沒設 `"type": "module"` 預設是 CJS、想用 ESM 要改檔名 `.mjs`。

  
### 5. 瀏覽器內用 ESM

📄 **寫進程式檔**：`index.html`

```text
<!-- 加 type="module" -->
<script type="module" src="app.js"></script>

<!-- 或內聯 -->
<script type="module">
  import { add } from "./math.js";
  console.log(add(1, 2));
</script>
```

**`type="module"` 的特色**：

```text
✅ 自動 defer（HTML 解析完才執行）
✅ 自動嚴格模式
✅ 有自己的 scope（不污染全域）
✅ 可以用 import / export
✅ 可以用 top-level await

⚠️ 必須走 HTTP/HTTPS、不能 file://
⚠️ import 路徑要完整（含 .js 副檔名）
⚠️ CORS 規則跟 fetch 一樣
```

  
### 6. **top-level await** —— ESM 才有

```text
// 在 ESM 最外層直接 await（不用包 async function）
const config = await fetch("/config.json").then(r => r.json());
const db = await openDatabase(config.dbUrl);

export { db };
```

> 💡 **CJS 完全做不到**——這是 ESM 的大優勢之一。

  
### 7. **dynamic import（動態 import）`import(...)`**

📄 **寫進程式檔**：`lazy.js`

```text
button.addEventListener("click", async () => {
  // 點下去才載入這個重模組
  const { Chart } = await import("./chart.js");
  new Chart(...);
});

// 條件載入
if (user.isAdmin) {
  const adminTools = await import("./admin.js");
}

// 程式碼分割（webpack / vite 自動拆 chunk）
const Component = lazy(() => import("./HeavyComponent.js"));
```

> 💡 **用途**：**code splitting（程式碼分割）**——首頁不載入「結帳頁」的 code、使用者真的去結帳才下載、首頁載入快很多。

  
### 8. ESM 跟 CJS 互通

```text
✅ ESM 可以 import CJS（基本能用）
   import { add } from "./math.cjs";
   ⚠️ default vs named 行為可能不一致

❌ CJS 不能直接 require ESM
   const m = require("./math.mjs");  ← TypeError
   要改用：const m = await import("./math.mjs");

✅ CJS 內可以用 dynamic import
```

**白話**：**新 → 舊 OK、舊 → 新要動態載入**。

  
### 9. **npm 套件的選擇** —— "exports" 多入口

```text
{
  "name": "my-pkg",
  "type": "module",
  "main": "./dist/cjs/index.js",       ← CJS 入口
  "module": "./dist/esm/index.js",     ← ESM 入口（老設定）
  "exports": {
    ".": {
      "import": "./dist/esm/index.js", ← import 時用這個
      "require": "./dist/cjs/index.js" ← require 時用這個
    }
  }
}
```

**用途**：**dual package**—— 同個套件、ESM 跟 CJS user 都用得到。

  
### 10. **bundler（打包工具）** —— 把 N 個檔案打成 1 個

```text
🔥 Vite（2026 最熱、Vue / React / Svelte 都推薦）
🟦 esbuild（最快、Vite 內建）
🟧 webpack（老牌、設定複雜）
🟩 rollup（library 作者愛用）
⚡ Bun bundler（隨 Bun 來、極快）
🦀 Turbopack（Next.js 自家、Rust 寫）
```

**bundler 做什麼？**

```text
✅ 把 N 個 .js 合併成 1 個（少 HTTP 請求）
✅ Tree-shaking（去掉沒用的 code）
✅ Minify（壓縮體積）
✅ 處理 import 路徑（不用寫 .js）
✅ 處理 CSS / 圖片 / 字型
✅ 熱重載（HMR、改完馬上看）
```

  
### 11. **import.meta** —— ESM 內的 metadata

```text
import.meta.url;     // 當前檔案的 URL
import.meta.env;     // Vite / 框架 注入的環境變數

// 取代 CJS 的 __dirname / __filename
import { fileURLToPath } from "url";
import { dirname } from "path";

const __filename = fileURLToPath(import.meta.url);
const __dirname = dirname(__filename);
```

  
### 5 個常見坑

```text
❌ 1. Cannot use import statement outside a module
   → 沒設 type: module 或檔名不對
   → 修：package.json 加 "type": "module" 或檔名改 .mjs

❌ 2. require is not defined
   → 在 ESM 內想用 CJS 語法
   → 修：改用 import、或用 createRequire(import.meta.url)

❌ 3. import 路徑沒寫 .js
   → 瀏覽器 ESM 必須完整路徑
   → 修：寫 ./math.js、不是 ./math

❌ 4. default 跟 named 混亂
   → import x, { y } from "..." 跟 import { x, y } from "..." 不一樣
   → 修：default 是「主角」（不用大括號）、named 用 { }

❌ 5. CJS 套件硬塞進 ESM
   → 有些套件只給 CJS、ESM import 拿到怪東西
   → 修：用 `import pkg from "pkg"; const { fn } = pkg;`
```

  
### 🎯 重點回顧

- **2026 寫新 code 一律 ESM**——CJS 是維護舊 code 時遇到
- **`"type": "module"` 在 package.json 切換**
- **瀏覽器 ESM 用 `<script type="module">`**——自帶 defer / strict mode
- **top-level await 是 ESM 殺手鐧**——CJS 做不到
- **dynamic `import("...")` 做 code splitting**——首頁加載飛快
- **import.meta.url 取代 CJS 的 \_\_dirname**
