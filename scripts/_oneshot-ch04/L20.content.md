**效能優化（performance optimization）= 讓網頁「更快、更順、不卡」的技巧**。2026 Google 把 **Core Web Vitals（核心 web 指標）** 列入 SEO 排名因子——慢的網頁不只 user 跑掉、連 Google 都不愛。這課介紹 6 大實戰招式。

> 💡 **這課的價值**：寫得會跑 ≠ 寫得快。**效能優化是 senior 工程師的標配技能**。

  
### 1. **Performance API** —— 先測量、再優化

```text
🔧 規則：「沒測量就不要優化」（Don Knuth 名言）
```

📄 **寫進程式檔**：`measure.js`

```text
// 標記 + 量
performance.mark("start");
heavyTask();
performance.mark("end");
performance.measure("task", "start", "end");

const entries = performance.getEntriesByName("task");
console.log(entries[0].duration);   // 毫秒

// Core Web Vitals 觀察
new PerformanceObserver(list => {
  list.getEntries().forEach(e => {
    console.log(e.name, e.value);
  });
}).observe({ type: "largest-contentful-paint", buffered: true });
```

**Core Web Vitals 三大指標（2026）**：

```text
🟢 LCP（Largest Contentful Paint）≤ 2.5s
   → 最大內容載入時間

🟢 INP（Interaction to Next Paint）≤ 200ms
   → 互動到下一次繪製
   → 2024 取代了舊的 FID

🟢 CLS（Cumulative Layout Shift）≤ 0.1
   → 累積版面位移（畫面跳動）
```

  
### 2. **debounce（防抖）** —— 等使用者「停手」再執行

搜尋框、輸入驗證、resize 監聽——使用者連續觸發時、只跑最後一次。

📄 **寫進程式檔**：`debounce.js`

```text
function debounce(fn, delay) {
  let timer;
  return (...args) => {
    clearTimeout(timer);
    timer = setTimeout(() => fn(...args), delay);
  };
}

// 用法：搜尋框輸入後 300ms 才打 API
const search = debounce((q) => {
  fetch(`/api/search?q=${q}`);
}, 300);

input.addEventListener("input", (e) => search(e.target.value));
```

> 💡 **白話**：debounce 像「等大家都不再說話再啟動錄音」——適合「**最後一次才有效**」的場景。

  
### 3. **throttle（節流）** —— 固定間隔最多執行一次

scroll、mousemove、resize——觸發很頻繁、但不需要每次都跑。

📄 **寫進程式檔**：`throttle.js`

```text
function throttle(fn, interval) {
  let last = 0;
  return (...args) => {
    const now = Date.now();
    if (now - last >= interval) {
      fn(...args);
      last = now;
    }
  };
}

// 用法：scroll 每 100ms 最多執行一次
const onScroll = throttle(() => {
  updateScrollProgress();
}, 100);

window.addEventListener("scroll", onScroll);
```

```text
debounce vs throttle 差別：

debounce  ▁▁▁▁▁▁▁▁▁▁▁▁____✓ ← 只跑最後一次
            連續觸發    停了才跑

throttle  ▁▁▁✓▁▁▁✓▁▁▁✓▁▁▁ ← 固定間隔跑
            每 N ms 最多 1 次
```

  
### 4. **memoize（記憶化）** —— 算過的不算第二次

```text
function memoize(fn) {
  const cache = new Map();
  return (...args) => {
    const key = JSON.stringify(args);
    if (!cache.has(key)) {
      cache.set(key, fn(...args));
    }
    return cache.get(key);
  };
}

const slowSquare = (n) => {
  console.log("計算...");
  return n * n;
};

const fastSquare = memoize(slowSquare);
fastSquare(5);   // 計算... → 25
fastSquare(5);   // 25（直接拿快取）
```

> 💡 **注意**：memoize 適合「**純函式**」（相同輸入永遠相同輸出）——對有副作用的不要用。

  
### 5. **requestAnimationFrame（RAF）** —— 跟瀏覽器繪製同步

寫動畫不要用 setInterval、要用 RAF。

📄 **寫進程式檔**：`animate.js`

```text
function animate() {
  // 更新位置
  ball.style.left = `${x++}px`;

  // 60Hz 螢幕 → 每 16.6ms 跑一次
  requestAnimationFrame(animate);
}

requestAnimationFrame(animate);
```

```text
setInterval 跟 RAF 差別：

setInterval(fn, 16)  ─ 不管螢幕、強制 16ms 跑一次
                       → 可能跟畫面渲染不同步、卡頓

requestAnimationFrame ─ 跟畫面 refresh 同步
                       → 順、省電池、tab 不可見時自動暫停
```

  
### 6. **requestIdleCallback** —— 趁瀏覽器閒著做不急的事

```text
requestIdleCallback(() => {
  analytics.flush();       // 不急的 log
  prefetchNextPage();      // 預載下一頁
  cleanupOldCache();
});

// 給 deadline 防止跑太久
requestIdleCallback((deadline) => {
  while (deadline.timeRemaining() > 0 && tasks.length) {
    tasks.shift()();
  }
});
```

> 💡 **白話**：把「**不急**」的事排到「**有空**」做——用 idle callback、不打擾用戶互動。

  
### 7. **lazy load（懶載入）** —— 看到才載入

📄 **寫進程式檔**：`lazy-img.html`

```text
<!-- 原生圖片 lazy load（瀏覽器內建） -->
<img src="big-image.jpg" loading="lazy" alt="...">

<!-- iframe 也能 -->
<iframe src="..." loading="lazy"></iframe>
```

📄 **寫進程式檔**：`lazy-component.js`

```text
// 元件 lazy load（React / Vue）
const Heavy = lazy(() => import("./HeavyComponent.js"));

// 純 JS 用 dynamic import
button.addEventListener("click", async () => {
  const { Chart } = await import("./chart.js");
  new Chart();
});
```

**用 IntersectionObserver 做進階 lazy**：

```text
const io = new IntersectionObserver(entries => {
  entries.forEach(e => {
    if (e.isIntersecting) {
      e.target.src = e.target.dataset.src;
      io.unobserve(e.target);
    }
  });
});

document.querySelectorAll("img[data-src]").forEach(img => io.observe(img));
```

  
### 8. **virtualization（虛擬列表）** —— 萬筆資料只渲染可視

1000 筆 li 全 render 會卡——**只渲染螢幕上看得到的那 10 筆**、其他用 placeholder。

🖥️ **在終端機輸入**（裝套件）：

```text
npm install @tanstack/react-virtual
```

📄 **寫進程式檔**（React 用 tanstack-virtual）：

```text
import { useVirtualizer } from "@tanstack/react-virtual";

const rowVirtualizer = useVirtualizer({
  count: 10000,
  getScrollElement: () => parentRef.current,
  estimateSize: () => 50,
});

return (
  <div ref={parentRef} style={{ height: 500, overflow: "auto" }}>
    <div style={{ height: rowVirtualizer.getTotalSize() }}>
      {rowVirtualizer.getVirtualItems().map(v => (
        <div key={v.key}>{items[v.index]}</div>
      ))}
    </div>
  </div>
);
```

```text
✅ 適合：聊天訊息、商品列表、log viewer、表格
✅ 套件：tanstack-virtual（最熱、無框架） / react-window / react-virtuoso
```

  
### 9. **code splitting（程式碼分割）** —— 不要一次載全部

📄 **寫進程式檔**：`vite.config.js` / `webpack.config.js`

```text
// dynamic import 自動分 chunk
const Admin = lazy(() => import("./AdminPanel"));

// → bundler 自動產生：
// main.js（首頁需要的）
// admin.chunk.js（點 admin 才載）
// checkout.chunk.js（點結帳才載）
```

  
### 10. **常見效能地雷**

```text
❌ 過度 DOM 操作
   → 100 次 appendChild → 100 次 reflow
   → 修：批次操作、用 DocumentFragment

❌ 沒清的 listener
   → addEventListener 但沒 removeEventListener
   → 修：componentWillUnmount / useEffect cleanup 移除

❌ 大型物件深複製
   → JSON.parse(JSON.stringify(huge)) 巨慢
   → 修：用 structuredClone 或 immer / immutable.js

❌ 同步重運算
   → 1 萬筆 sort 卡 UI
   → 修：搬到 Web Worker

❌ 大圖未壓縮
   → 載入 5MB PNG
   → 修：用 WebP / AVIF + responsive srcset
```

  
### 11. **2026 圖片優化標配**

```text
✅ 用 WebP 或 AVIF（檔案小 30-50%）
✅ <picture> + srcset（不同螢幕載不同大小）
✅ loading="lazy" + decoding="async"
✅ width / height 屬性（防 CLS）
✅ CDN（Cloudflare Images / imgix）自動最佳化
```

  
### 5 個常見坑

```text
❌ 1. 不測量就優化
   → 改半天沒效果、或改錯地方
   → 修：先用 Performance / Lighthouse 找瓶頸

❌ 2. debounce / throttle 搞混
   → 修：最後一次有效用 debounce、固定間隔用 throttle

❌ 3. memoize 純函式 + 物件參數
   → JSON.stringify 大物件超慢
   → 修：cache key 用簡單值、或用 WeakMap

❌ 4. RAF 寫成遞迴沒退出
   → 元件卸載後還在跑
   → 修：存 RAF id、cancelAnimationFrame

❌ 5. lazy load 沒設 width / height
   → 圖片載入時版面跳動、CLS 爛
   → 修：img 一定要 width / height 或 aspect-ratio
```

  
### 🎯 重點回顧

- **先測量再優化**：Performance API + Lighthouse + DevTools
- **debounce** 等使用者停手、**throttle** 固定間隔限速
- **memoize** 純函式快取、**RAF** 跟畫面同步動畫
- **lazy load** + **virtualization** 處理大量內容
- **code splitting** 用 dynamic import、首頁載入飛快
- **圖片 WebP/AVIF + width/height + lazy** 三件套
