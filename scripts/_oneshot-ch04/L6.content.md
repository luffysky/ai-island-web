**array（陣列）= 有順序的清單**。用 `[ ]` 包起來、裡面可以放任何型別的值。陣列是寫前端最常用的資料結構——使用者清單、訂單列表、留言、商品全是陣列。

> 💡 **這課的價值**：把 `map / filter / reduce` 這三件套練熟、80% 的資料處理問題都迎刃而解。**這也是面試最愛問的**。

  
### 1. 陣列基礎操作

📄 **寫進程式檔**：`array-basic.js`

```text
const arr = [1, 2, 3, 4, 5];

arr.length;       // 5（長度）
arr[0];           // 1（第一個元素）
arr[arr.length - 1];   // 5（最後一個）
arr.at(-1);       // 5（負數索引、ES2022）
arr.at(-2);       // 4

// 判斷是不是陣列
Array.isArray(arr);   // true
```

  
### 2. 「會改原陣列」的方法（mutating）

📄 **寫進程式檔**：`mutating.js`

```text
const arr = [1, 2, 3];

arr.push(4);        // 加到尾巴 → [1,2,3,4]、回傳新長度 4
arr.pop();          // 移除尾巴 → [1,2,3]、回傳被移除的 4
arr.unshift(0);     // 加到頭 → [0,1,2,3]
arr.shift();        // 移除頭 → [1,2,3]
arr.splice(1, 2);   // 從 index 1 刪 2 個 → [1]、回傳被刪的 [2,3]
arr.reverse();      // 反轉
arr.sort();         // 排序（預設按字串）
```

> 💡 **注意 sort 的坑**：`[10, 2, 1].sort()` 結果是 `[1, 10, 2]`——預設按「字串字典順序」排。要按數字要寫 `arr.sort((a, b) => a - b)`。

  
### 3. **map / filter / reduce —— 函式式三件套（不改原陣列、回新陣列）**

這三個是 JS 陣列的精華、記下來受用一生。

📄 **寫進程式檔**：`three.js`

```text
const nums = [1, 2, 3, 4, 5];

// map —— 每個元素「變身」一次
nums.map(n => n * 2);   // [2, 4, 6, 8, 10]
// 想像：每個元素都進去 fn、變成新的、組成新陣列

// filter —— 留下符合條件的
nums.filter(n => n > 2);   // [3, 4, 5]
// 想像：每個元素都問一句「你符合嗎？」true 才留

// reduce —— 把整個陣列「縮成一個值」
nums.reduce((acc, n) => acc + n, 0);   // 15（總和）
// acc 是「累積值」、初始值是 0、每次加 n
```

> 💡 **記法**：map = 變身、filter = 篩選、reduce = 收斂。**90% 資料處理用這三個搞定**。

  
### 4. 查找類方法

📄 **寫進程式檔**：`find.js`

```text
const nums = [1, 2, 3, 4, 5];

// find —— 找第一個符合的「值」
nums.find(n => n > 3);          // 4
nums.find(n => n > 100);        // undefined

// findIndex —— 找第一個符合的「索引」
nums.findIndex(n => n > 3);     // 3

// findLast / findLastIndex（ES2023）—— 從後面找
nums.findLast(n => n < 4);      // 3
nums.findLastIndex(n => n < 4); // 2

// indexOf / lastIndexOf —— 找值的索引
nums.indexOf(3);    // 2
nums.indexOf(99);   // -1（找不到）

// includes —— 有沒有這個值
nums.includes(3);   // true
nums.includes(99);  // false
```

> 💡 **includes vs indexOf**：判斷「有沒有」用 `includes`（回 boolean、好讀）。要拿位置才用 `indexOf`。

  
### 5. 判斷類方法

```text
const nums = [1, 2, 3, 4, 5];

nums.some(n => n > 4);    // true ← 至少有一個 > 4
nums.some(n => n > 100);  // false

nums.every(n => n > 0);   // true ← 全部 > 0
nums.every(n => n > 3);   // false
```

**記法**：some = **有沒有**（任一）、every = **是不是全部**（全部）。

  
### 6. **flat / flatMap** —— 攤平巢狀陣列

📄 **寫進程式檔**：`flat.js`

```text
[[1, 2], [3, 4]].flat();        // [1, 2, 3, 4]
[1, [2, [3, [4]]]].flat(2);     // [1, 2, 3, [4]]（攤 2 層）
[1, [2, [3, [4]]]].flat(Infinity);   // [1, 2, 3, 4]（全部攤）

// flatMap = map 後自動 flat 1 層
[1, 2].flatMap(n => [n, n * 2]);   // [1, 2, 2, 4]
// 等於 [1,2].map(...).flat()
```

  
### 7. **ES2023 不可變方法**—— 不改原陣列、回新陣列

```text
const arr = [3, 1, 2];

arr.toSorted();          // [1,2,3]（新陣列）—— 原 arr 不變
arr.toReversed();        // [2,1,3]
arr.toSpliced(1, 1);     // [3,2]
arr.with(0, 99);         // [99,1,2]（取代 index 0）

console.log(arr);   // [3, 1, 2] ← 原陣列沒被動
```

> 💡 **為什麼有這些？** React / Redux 推崇 immutability（不可變性）—— 改資料前先複製、原資料保持原樣、好除錯也好做時光機。**沒有 `to` 開頭的舊版會改原陣列、坑很多**。

  
### 8. 實戰：資料處理 pipeline

📄 **寫進程式檔**：`pipeline.js`

```text
const orders = [
  { id: 1, total: 100, paid: true },
  { id: 2, total: 200, paid: false },
  { id: 3, total: 300, paid: true },
  { id: 4, total: 150, paid: true }
];

// 算「已付款訂單」的總金額
const total = orders
  .filter(o => o.paid)        // 留下 paid: true → 3 筆
  .map(o => o.total)          // 取出金額 → [100, 300, 150]
  .reduce((s, n) => s + n, 0); // 加總 → 550

console.log(total);   // 550
```

> 💡 **這就是「函式式 pipeline」**——一行接一行、每一步意圖清楚、好讀好改。**這是現代 JS 的精神**。

  
### 9. 陣列建立技巧

```text
// 建立指定長度的陣列
Array.from({ length: 5 }, (_, i) => i);   // [0, 1, 2, 3, 4]
Array(5).fill(0);                          // [0, 0, 0, 0, 0]

// 用 Set 去重
const unique = [...new Set([1, 2, 2, 3, 3])];   // [1, 2, 3]

// 從字串轉陣列
[..."hello"];                  // ['h','e','l','l','o']

// 從 NodeList 轉陣列
Array.from(document.querySelectorAll("li"));

// 範圍
Array.from({ length: 10 }, (_, i) => i + 1);   // [1..10]
```

  
### 10. forEach vs for...of

📄 **寫進程式檔**：`loop.js`

```text
const arr = [1, 2, 3];

// forEach —— 簡潔、不能 break / continue
arr.forEach(n => console.log(n));

// for...of —— 可以 break / continue / await
for (const n of arr) {
  if (n === 2) break;
  console.log(n);
}

// for...in —— 遍歷物件 key（不要用在陣列！）
for (const key in arr) {
  // key 是 "0", "1", "2"（字串）
}
```

> 💡 **forEach 不能 await！** 想在迴圈裡用 `await fetch()` 必須用 `for...of` 或 `Promise.all` + `map`。**這是新手 async 必踩坑**。

```text
map / filter / reduce / find / findIndex / some / every /
includes / indexOf / slice / concat / flat / flatMap /
toSorted / toReversed / toSpliced / with

🔄 迭代
forEach / for...of / entries / keys / values
```

  
### 5 個常見坑

```text
❌ 1. sort 沒給比較函式、數字亂排
   → [10, 2, 1].sort() → [1, 10, 2]
   → 修：arr.sort((a, b) => a - b)

❌ 2. splice 跟 slice 搞混
   → splice 改原陣列、slice 不改
   → 修：要複製用 slice、要刪除用 splice

❌ 3. forEach 內用 await 沒效果
   → 修：用 for...of 或 Promise.all(arr.map(...))

❌ 4. map 不回傳值
   → arr.map(n => { n * 2 }) 得到 [undefined, ...]
   → 修：箭頭函式 { } 內要寫 return、或省略 { }

❌ 5. 用 for...in 遍歷陣列
   → 拿到字串 key、亂排序、踩 prototype 屬性
   → 修：用 for...of 或 forEach
```

  
### 🎯 重點回顧

- **map / filter / reduce 三件套**——90% 資料處理靠它們
- **`to` 開頭是 ES2023 不可變版本**（toSorted / toReversed / toSpliced / with）—— 不改原陣列
- **`some` / `every`**：有沒有任一 / 是不是全部
- **`forEach` 不能 `break` 也不會 `await`**——要用 `for...of`
- **sort 數字一定要給比較函式**：`arr.sort((a, b) => a - b)`
