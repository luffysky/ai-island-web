**regex（regular expression、正規表示式）= 用「模式」描述要找的文字**。`/^09\d{8}$/` 一行就能驗證台灣手機。寫起來像天書、但學會後處理字串效率 10 倍——驗證、搜尋、取代、解析都靠它。

> 💡 **這課的價值**：寫前端 / 後端都會碰到——表單驗證、搜尋功能、log 解析、URL 處理。regex 不會、要寫一堆 `if (str[0] === ...)`。

  
### 1. 兩種建立方式

📄 **寫進程式檔**：`regex.js`

```text
// 1. 字面量（最常用）
const re1 = /hello/i;
const re2 = /\d+/g;

// 2. 用 RegExp 建構子（要動態組合時）
const word = "招財";
const re3 = new RegExp(word, "i");

// ⚠️ 字串內反斜線要 escape 兩次
new RegExp("\\d+", "g");   // 等於 /\d+/g
```

  
### 2. **regex flags（旗標）** —— 控制行為

```text
g — global       找所有、不只第一個
i — ignoreCase   忽略大小寫
m — multiline    ^ $ 對每行有效
s — dotAll        . 可以匹配換行
u — unicode       支援 unicode 字元
y — sticky        從指定位置開始
d — hasIndices    結果帶位置（ES2022）
```

```text
"AbcAbc".match(/abc/);     // ["Abc"]（只一次、區分大小寫）
"AbcAbc".match(/abc/g);    // ["Abc", "Abc"]（找全部）
"AbcAbc".match(/abc/gi);   // ["Abc", "Abc"]（不分大小寫 + 找全部）
```

  
### 3. **必會的 metacharacters（特殊字元）**

```text
.       任意單一字元（不含換行）
\d      數字 [0-9]
\D      非數字
\w      字母 / 數字 / _ [A-Za-z0-9_]
\W      非 \w
\s      空白（空格、tab、換行）
\S      非空白

^       開頭
$       結尾
\b      字邊界（單字頭尾）

[abc]   a / b / c 其中一個
[a-z]   a 到 z
[^abc]  不是 a / b / c
```

  
### 4. **quantifier（次數）** —— 出現幾次

```text
*       0 或多次
+       1 或多次
?       0 或 1 次（可有可無）
{n}     正好 n 次
{n,}    至少 n 次
{n,m}   n 到 m 次

範例：
\d{4}            連續 4 個數字
\d{2,4}          2-4 個
a+               一個以上的 a
.*               任意字 0 次以上
\w+@\w+\.\w+     簡單 email
```

  
### 5. **常用方法**

📄 **寫進程式檔**：`methods.js`

```text
// test —— 有沒有匹配（回 true / false）
/\d+/.test("abc123");     // true
/\d+/.test("abc");        // false

// match —— 找匹配（回陣列或 null）
"hello".match(/l+/);       // ["ll", index: 2, ...]
"hello".match(/l+/g);      // ["ll"]（多個會列）
"a1b2c3".match(/\d/g);     // ["1", "2", "3"]

// matchAll —— 找全部、含 group 資訊（iterator）
const matches = [..."a1b2c3".matchAll(/(\w)(\d)/g)];
// [["a1", "a", "1"], ["b2", "b", "2"], ["c3", "c", "3"]]

// replace —— 取代
"hello".replace(/l/g, "L");        // "heLLo"
"2026-05-28".replace(/-/g, "/");   // "2026/05/28"

// replaceAll —— 直接取代全部（ES2021）
"hello".replaceAll("l", "L");

// split
"a,b,c".split(/,/);          // ["a", "b", "c"]
"one  two   three".split(/\s+/);   // ["one", "two", "three"]

// search —— 找第一個匹配的 index
"abc123".search(/\d/);   // 3
```

  
### 6. **capture group（捕獲組）** —— 把匹配的部分「分組抓出來」

```text
// 用 () 分組
const re = /(\d{4})-(\d{2})-(\d{2})/;
const m = "2026-05-28".match(re);

m[0];    // "2026-05-28"（整個匹配）
m[1];    // "2026"（第 1 組）
m[2];    // "05"
m[3];    // "28"
```

**命名分組（ES2018）**：

```text
const re = /(?<year>\d{4})-(?<month>\d{2})-(?<day>\d{2})/;
const m = "2026-05-28".match(re);

m.groups.year;    // "2026"
m.groups.month;   // "05"
m.groups.day;     // "28"
```

  
### 7. **lookahead / lookbehind** —— 「不消耗」的條件檢查

```text
(?=...)     正向 lookahead   後面要有...
(?!...)     負向 lookahead   後面不能有...
(?<=...)    正向 lookbehind  前面要有...
(?<!...)    負向 lookbehind  前面不能有...

範例：
/\d+(?= 元)/        後面是「元」的數字 → "100 元" 抓到 "100"
/(?<=NT\$)\d+/      前面是「NT$」的數字 → "NT$500" 抓到 "500"
/\d+(?! 公斤)/      後面不是「公斤」的數字
```

> 💡 **「不消耗」**：lookahead 只「看」、不算進匹配結果——這跟 capture group 不一樣。

  
### 8. 實戰範例

📄 **寫進程式檔**：`practical.js`

```text
// Email 驗證（簡化版、業界一般夠用）
const emailRe = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
emailRe.test("luffy@gmail.com");        // true
emailRe.test("invalid email");           // false

// 手機（台灣）
const phoneRe = /^09\d{8}$/;
phoneRe.test("0912345678");              // true

// 信用卡（4 組 4 位數字）
const cardRe = /^\d{4}-\d{4}-\d{4}-\d{4}$/;

// 密碼（至少 8 字、含大小寫 + 數字）
const pwdRe = /^(?=.*[a-z])(?=.*[A-Z])(?=.*\d).{8,}$/;
pwdRe.test("Abc12345");                  // true

// 從文字抓 URL
const urlRe = /https?:\/\/[^\s]+/g;
const text = "看這 https://google.com 跟 https://github.com";
text.match(urlRe);
// ["https://google.com", "https://github.com"]

// 從文字抓 hashtag
const hashRe = /#\w+/g;
"今天 #寫code #學regex".match(hashRe);
// ["#寫code", "#學regex"]

// 抓 @mention
"@luffy 早安 @雪鑰".match(/@\w+/g);
// ["@luffy", "@雪鑰"]

// 移除多餘空白
"  hello   world  ".replace(/\s+/g, " ").trim();
// "hello world"

// camelCase 轉 snake_case
"someVariableName".replace(/([A-Z])/g, "_$1").toLowerCase();
// "some_variable_name"
```

  
### 9. **動態 regex** —— 用變數組

```text
function searchHighlight(text, keyword) {
  // 把 keyword 內的特殊字 escape
  const escaped = keyword.replace(/[.*+?^${}()|[\]\\]/g, "\\$&");
  const re = new RegExp(escaped, "gi");
  return text.replace(re, `<mark>$&</mark>`);
}

searchHighlight("Hello World", "world");
// "Hello <mark>World</mark>"
```

⚠️ **動態 regex 一定要 escape user input**——不然搜尋 `.` 會變「任意字」、不對。

  
### 10. **regex 效能** —— catastrophic backtracking 災難

寫不好的 regex 可以讓 server CPU 爆 100%——叫 **ReDoS 攻擊**。

```text
// ✗ 危險：a 的數量 + a 結尾 → 指數爆炸
const bad = /^(a+)+$/;
bad.test("aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaab");   // 很久才回 false

// ✓ 改寫：避免 nested quantifier
const good = /^a+$/;
```

**避免方法**：

```text
✅ 不要 nested quantifier（()+）+
✅ 用 atomic group（Node 22+ 支援）
✅ 設超時：執行時間超過閾值就停
✅ 工具：safe-regex / vscode-regexp-preview
```

### 5 個常見坑

```text
❌ 1. .test 的 regex 有 g flag、第二次匹配失敗
   → re.lastIndex 會記住、下次從那繼續
   → 修：每次用前重設 lastIndex = 0、或重新 new RegExp

❌ 2. 沒 escape user input 進 RegExp
   → user 輸入 .* 變「任意字」
   → 修：先 escape 特殊字

❌ 3. nested quantifier 寫 catastrophic backtracking
   → server 卡死
   → 修：用工具檢查、改寫成 atomic

❌ 4. 用 regex parse HTML / JSON
   → 巢狀結構 regex 處理不來
   → 修：用 DOMParser / JSON.parse

❌ 5. greedy vs lazy 搞混
   → /<.*>/ 抓「<a><b>」整個（greedy）
   → 想抓最短用 /<.*?>/（lazy）
```

  
### 🎯 重點回顧

- **`\d \w \s` + `* + ? {n,m}` 是 regex 基礎**
- **`()` 是 capture group、`(?:)` 是不捕獲**
- **flags：`g i m s u y d`**
- **lookahead `(?=)` / `(?!)`** 不消耗、只「檢查條件」
- **動態 regex 一定 escape user input**——防 ReDoS
- **regex101.com 是寫 regex 的好朋友**——複雜 regex 一定要驗證
