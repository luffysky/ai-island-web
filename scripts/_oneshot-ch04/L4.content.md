**operator（運算符）= 對資料做操作的符號**。`+ - * /` 是算術運算符、`== === > <` 是比較運算符、`&& || ??` 是邏輯運算符。JS 的 `==` 是個惡名昭彰的陷阱——你以為它在比「相等」、其實它偷偷做型別轉換。

> 💡 **這課的價值**：學會 `===` / `??` / `?.` 這三個現代運算符、你寫的 JS 就會比 99% 老前輩寫的更安全。

  
### 1. `==` vs `===` —— 永遠用 `===`

```text
🟥 ==  「寬鬆相等」 → 先轉型別、再比較（坑很多）
🟩 === 「嚴格相等」 → 型別跟值都要一樣（推薦）
```

📄 **寫進程式檔**：`equal.js`

```text
// == 做型別轉換、結果違反直覺
0 == "";              // true   ← 字串轉數字
0 == "0";             // true
"" == "0";            // false  ← 沒邏輯吧
null == undefined;    // true
null == 0;            // false  ← 沒邏輯 ×2
[] == false;          // true   ← 陣列轉 0
[0] == false;         // true
"1" == true;          // true

// === 嚴格相等
0 === "";             // false
null === undefined;   // false
[] === [];            // false  ← 兩個不同陣列、不相等
```

> 💡 **永遠用 `===`**——唯一例外：`x == null` 是「null 或 undefined」的習慣寫法、簡潔好讀。

  
### 2. 為什麼 `[] === []` 是 false？

```text
const a = [];
const b = [];
a === b;   // false ← 雖然「看起來一樣」、但兩個不同物件

const c = a;
a === c;   // true ← 同一個物件的兩個名字
```

**規則**：物件比較的是「**是不是同一個實體**」、不是內容。要比內容用 deep equal（後面 L19 / L22 會教）。

  
### 3. 邏輯運算符：`&&` / `||` / `??`

這三個都是**短路運算**——左邊就能決定結果、右邊不執行。

📄 **寫進程式檔**：`logic.js`

```text
// && —— 兩邊都 truthy 才回右邊、否則回「第一個 falsy」
true && "yes";    // "yes"
false && "yes";   // false
0 && "yes";       // 0
"hi" && "yes";    // "yes"

// || —— 左邊 truthy 就回左邊、否則回右邊
"hello" || "x";     // "hello"
"" || "default";    // "default"
0 || "default";     // "default"  ← 但你想要 0 的話這就壞了

// ?? —— Nullish Coalescing（空值合併、ES2020）
// 只有 null 或 undefined 才用右邊
0 ?? "default";     // 0     ← 0 是「有值」、用 0
"" ?? "default";    // ""    ← "" 也算「有值」
null ?? "default";  // "default"
undefined ?? "default";   // "default"
```

> 💡 **`||` vs `??` 哪個用**：要把 0 / "" / false 當「有值」用 `??`、要把所有 falsy 都當「沒值」用 `||`。**設預設值 90% 場景用 `??`**。

  
### 4. **optional chaining（可選串連）`?.`** —— 防止「讀不存在的屬性」報錯

ES2020 的救星。

📄 **寫進程式檔**：`optional.js`

```text
const user = { profile: { name: "招財" } };

// ✗ 老寫法（痛苦）
const name = user && user.profile && user.profile.name;

// ✓ 新寫法
const name = user?.profile?.name;       // "招財"
const miss = user?.missing?.deep;       // undefined（不爆）
const noUser = nobody?.profile?.name;   // undefined（不爆）
```

`?.` 也能用在陣列、函式、方法：

```text
arr?.[0];              // 安全索引
callback?.();          // 安全呼叫
obj.method?.();         // method 不存在不爆
```

> 💡 **白話**：`?.` 像一道保險絲——前面是 null 或 undefined 就「斷電」直接回 undefined、不會繼續往下炸。

  
### 5. **spread（展開運算符）`...`** —— 把陣列／物件「攤平」

📄 **寫進程式檔**：`spread.js`

```text
// 展開陣列
const a = [1, 2, 3];
const b = [...a, 4, 5];      // [1, 2, 3, 4, 5]
const copy = [...a];          // 淺複製陣列

// 展開物件
const o1 = { a: 1, b: 2 };
const o2 = { ...o1, c: 3 };   // { a: 1, b: 2, c: 3 }
const merged = { ...o1, ...o2 };

// 函式呼叫展開
const nums = [1, 2, 3];
Math.max(...nums);   // 3 ← 等於 Math.max(1, 2, 3)

// 字串轉陣列
const arr = [..."hello"];     // ['h','e','l','l','o']
```

  
### 6. **rest（其餘參數）`...`** —— 把多個變數「收成」陣列

長得跟 spread 一樣、但用在「**接收端**」。

📄 **寫進程式檔**：`rest.js`

```text
// 函式接受任意多個參數
function sum(...nums) {
  return nums.reduce((a, b) => a + b, 0);
}

sum(1, 2, 3);         // 6
sum(1, 2, 3, 4, 5);   // 15

// 解構搭配
const [first, ...rest] = [1, 2, 3, 4];
// first = 1
// rest = [2, 3, 4]

const { name, ...others } = { name: "A", age: 3, city: "TP" };
// name = "A"
// others = { age: 3, city: "TP" }
```

> 💡 **記法**：`...` 在「攤開」就是 spread（一變多）、在「收集」就是 rest（多變一）。**位置決定用途**。

  
### 7. **destructuring（解構）**—— 一次拆好幾個值

📄 **寫進程式檔**：`destructure.js`

```text
// 陣列解構
const [a, b, c] = [1, 2, 3];
// a=1, b=2, c=3

// 物件解構
const { name, age } = { name: "招財", age: 3, city: "TP" };
// name="招財", age=3

// 改名
const { name: petName } = { name: "招財" };
// petName = "招財"

// 預設值
const { role = "user" } = {};
// role = "user"

// 巢狀
const { profile: { email } } = user;

// 解構參數（函式內最常用）
function greet({ name, age = 0 }) {
  return `${name} is ${age}`;
}
greet({ name: "招財", age: 3 });
```

  
### 8. 算術運算符的陷阱

```text
"5" + 3;     // "53"   ← 字串拼接（+ 遇到字串就拼）
"5" - 3;     // 2      ← - 強迫轉數字
"5" * 3;     // 15
"abc" * 3;   // NaN

// 確保是數字
Number("5") + 3;     // 8
+"5" + 3;            // 8（單元 + 強制轉 number）
parseInt("5px");     // 5（容忍尾巴）
parseFloat("3.14"); // 3.14
```

  
### 5 個常見坑

```text
❌ 1. 還在寫 ==
   → 隱式轉換陷阱多
   → 修：永遠 ===、唯一例外 x == null

❌ 2. 用 || 設預設值、結果 0 / "" 也被當沒值
   → const count = input || 10; 但 input = 0 也被換成 10
   → 修：const count = input ?? 10;

❌ 3. user.profile.name 沒做空值檢查
   → user 是 null 就爆
   → 修：user?.profile?.name

❌ 4. "5" + 3 = "53" 不是 8
   → + 遇到字串就拼字
   → 修：Number(x) 或 +x 強制轉數字

❌ 5. [] === [] 以為是 true
   → 物件比的是「同一個實體」
   → 修：要比內容用 JSON.stringify 或 lodash isEqual
```

  
### 🎯 重點回顧

- **永遠用 `===`**、唯一例外是 `x == null` 同時擋 null + undefined
- **`??` 比 `||` 安全**——設預設值優先 `??`
- **`?.` 是空值救星**——`user?.profile?.name` 不會爆
- **`...` 兩種用途**：攤開（spread）跟收集（rest）
- **destructuring 是現代 JS 必備**——函式參數、API response 都靠它
