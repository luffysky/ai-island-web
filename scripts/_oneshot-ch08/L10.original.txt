React 預設「過度 re-render」—useMemo / useCallback 是優化工具。但 80% 場景不該用、學完知道何時用。

  
### 什麼時候 re-render

```text
觸發 re-render 的 3 個原因：

1. State 變
   setCount(...) → component re-render

2. Props 變
   父 re-render → 子 re-render（預設）

3. Context 變
   訂閱的 component 全 re-render

預設行為：
父 re-render → 所有子 re-render
即使 props 沒變

這在小 app 沒影響
大 app 才會慢
```

  
### useMemo：快取算的值

```tsx
// 場景：expensive 計算
function FilteredList({ items, query }) {
    // ❌ 每 render 都算
    const filtered = items.filter(i => i.name.includes(query));
    
    return ...;
}

// ✓ 只在 items 或 query 變時算
function FilteredList({ items, query }) {
    const filtered = useMemo(() => {
        return items.filter(i => i.name.includes(query));
    }, [items, query]);
    
    return ...;
}

// 規則：
// 1. 第一個參數是 function
// 2. dependency array
// 3. deps 不變、回 cached 值

// 何時用 useMemo？
// 1. expensive 計算（filter / sort / 大量 reduce）
// 2. 物件 / 陣列當 dependency 給其他 hook
// 3. 傳給 memo'd component 的 props

// 不該用：
// - 簡單算（+ - * /）
// - 字串連接
// - 一般 map / filter（小 list）

// 過度用 useMemo：
// - 增加 memory
// - 增加複雜度
// - 沒明顯性能改善
```

  
### useCallback：快取 function

```tsx
// useCallback = 給 function 用的 useMemo

// 沒 useCallback：每 render 新 function
function Parent() {
    const [count, setCount] = useState(0);
    
    const handleClick = () => {     // 每 render 新 function
        console.log("clicked");
    };
    
    return ;
}

// 有 useCallback：cache function
function Parent() {
    const [count, setCount] = useState(0);
    
    const handleClick = useCallback(() => {
        console.log("clicked");
    }, []);   // 依賴空、function 永遠同一個
    
    return ;
}

// 場景：傳給 React.memo'd 子 component
const Child = React.memo(({ onClick }) => {
    return Click;
});

// 沒 useCallback：父 re-render → onClick 新 → Child re-render
// 有 useCallback：父 re-render → onClick 同 → Child 不 re-render

// 不該用 useCallback：
// - 傳給原生 element（）
// - 父 component 不會頻繁 re-render
// - Function 邏輯簡單
```

  
### React.memo

```tsx

    console.log("Child render");
    return {name} - {age};
});

// 用
function Parent() {
    const [count, setCount] = useState(0);
    
    return (
        <>
             setCount(c => c + 1)}>+

// 需要 useMemo / useCallback 配合

    return ...;

function Parent() {

    // ✓ 用 useMemo / useCallback
    const user = useMemo(() => ({ name: "Alice" }), []);
```

### 2026 React Compiler（自動 memo）

```text
React 19 開始：React Compiler

自動加 useMemo / useCallback / React.memo
你不用手動寫

裝（experimental）：
$ npm install babel-plugin-react-compiler

設定 vite.config.ts：
{
    babel: {
        plugins: ["babel-plugin-react-compiler"]
    }
}

之後寫：
function Component({ items, query }) {
    const filtered = items.filter(i => i.name.includes(query));
    return ...;
}

Compiler 自動變：
const filtered = useMemo(() => items.filter(...), [items, query]);

2026 趨勢：
- Compiler 還在 RC、不全部 stable
- 一些大公司開始用
- 1-2 年後成標配
- 「不用手動 memo 化」是未來

過渡期：仍要懂 useMemo / useCallback
即使 Compiler 解了 80%、剩 20% 要懂
```

  
### 性能優化心法

```text
1. 先量再優化
   - Profiler（React DevTools）
   - 看哪 component 慢
   - 不要無腦加 useMemo

2. 80% 場景不用優化
   - 小 app 不會卡
   - 過度優化 = 增加複雜度

3. 找瓶頸再優化
   - 大 list（1000+）→ virtual list
   - 重計算 → useMemo
   - 重 component → React.memo

4. 結構改變 > memo 化
   - 把 expensive 部分放最深層
   - 不要全部 lift up

5. 用 Profiler 驗證
   - 加 useMemo 前後對比
   - 沒改善 = 移除

範例：lift state down
❌ 父持有 state、改了所有子 re-render
✓ 子持有 state、改只影響自己
```

```tsx
function SearchableList({ items }) {

function SearchableList({ items }) {
    const [query, setQuery] = useState("");
    const debouncedQuery = useDebounce(query, 300);
    
    const filtered = useMemo(() => {
        return items.filter(i => 
            i.name.toLowerCase().includes(debouncedQuery.toLowerCase())
        );
    }, [items, debouncedQuery]);
    
    return (...);
}

// 用 useDebounce custom hook（自訂）
function useDebounce(value: T, delay: number) {
    const [debounced, setDebounced] = useState(value);
    
    useEffect(() => {
        const timer = setTimeout(() => setDebounced(value), delay);
        return () => clearTimeout(timer);
    }, [value, delay]);
    
    return debounced;
}
```

  
### 常見性能錯誤

  
- **所有 function 都 useCallback**—過度優化
- **所有計算都 useMemo**—增加複雜度
- **沒先量就優化**—瞎優化
- **React.memo 包但 props 是 new object**—沒用
- **跑 expensive 在 render**—應該 useMemo
- **大 list 沒 virtual**—1000+ 卡頓
- **沒用 Profiler**—不知哪慢

  
**🚀 進階技巧**

性能優化 5 心法：(1) **先量再優化**—Profiler 找瓶頸 (2) **不要過度用 useMemo / useCallback**—80% 場景不需要 (3) **React.memo 配 useMemo / useCallback**—單獨無效 (4) **2026 用 React Compiler**—自動 memo (5) **大 list 用 virtual scrolling**—性能級別差。
