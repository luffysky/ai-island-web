### Error Boundary

```tsx
// 抓 component tree 內的 error
// 必須是 class component（hook 版還沒）

import { Component, ErrorInfo, ReactNode } from "react";

interface Props {
    fallback: ReactNode;
    children: ReactNode;
}

interface State {
    hasError: boolean;
    error?: Error;
}

class ErrorBoundary extends Component {
    state: State = { hasError: false };
    
    static getDerivedStateFromError(error: Error): State {
        return { hasError: true, error };
    }
    
    componentDidCatch(error: Error, info: ErrorInfo) {
        // Log to Sentry / DataDog
        console.error("Error caught:", error, info);
    }
    
    render() {
        if (this.state.hasError) {
            return this.props.fallback;
        }
        return this.props.children;
    }
}

// 用

    

// 只要 UserProfile tree 內 throw error、不會炸整個 app
// 只 fallback 那塊
```

### Error Boundary 限制

```text
不會抓的：
1. Event handler 的 error
   ❌  { throw new Error() }}>
   
   要手動 try/catch：
   const handleClick = () => {
       try {
           doSomething();
       } catch (e) {
           showError(e);
       }
   };

2. Async error
   ❌ useEffect(() => {
       fetch("...").then(() => { throw new Error() });
   });
   
   要手動處理：
   useEffect(() => {
       fetch("...")
           .then(...)
           .catch(err => setError(err));
   });

3. SSR error
   - SSR 的 error 不算 component tree error
   - Next.js 有 error.tsx 處理

4. Error Boundary 自己的 error
   - 用更上層的 ErrorBoundary 包

抓得到的：
✓ Render 時 throw
✓ Lifecycle 內 throw
✓ Constructor throw
```

### 常見錯誤

  
- **沒包 Suspense**—lazy import 顯示 blank
- **整個 app 1 個 ErrorBoundary**—1 個錯炸整個
- **沒 log error**—生產環境不知道發生什麼
- **Boundary 抓 event handler error**—抓不到
- **fallback UI 跟正常 UI 跳動大**—體驗差
- **沒 reset 機制**—錯了用戶卡住
- **濫用 lazy**—每個 component lazy、慢

  
**🚀 進階技巧**

Suspense + ErrorBoundary 4 心法：(1) **Code splitting**—大 component / route 用 lazy + Suspense (2) **Multi-layer boundary**—細粒度錯誤處理 (3) **用 react-error-boundary**—業界主流 (4) **React Query + Suspense**—完美 data fetching 體驗。**2026 趨勢**：Server Components + Suspense 是 Next.js App Router 預設 pattern。
