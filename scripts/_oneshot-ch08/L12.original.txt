當 state 要傳很多層、props drilling 痛苦時—用 Context。

  
### 什麼是 Props Drilling

```tsx
// 痛苦範例：user 要傳 5 層

    
        
            
                
            
        
    

// 每層都要傳、改一個地方改 5 個
// 中間 component 根本不用 user、只是 pass through

// 解：Context
```

  
### 基本 Context

```tsx
import { createContext, useContext, useState } from "react";

// 1. 建 context
interface UserContextValue {
    user: User | null;
    setUser: (user: User | null) => void;
}

const UserContext = createContext(null);

// 2. Provider（在頂層）
function App() {
    const [user, setUser] = useState(null);
    
    return (
        
            
        
    );
}

// 3. 在子 component 用
function UserMenu() {
    const context = useContext(UserContext);
    if (!context) throw new Error("UserMenu must be in UserProvider");
    
    const { user, setUser } = context;
    
    return (
        
            {user ? user.name : "Guest"}
             setUser(null)}>Logout
        
    );
}

// 不用 props 傳 5 層！
```

  
### Custom Hook 包 Context

```tsx
// 業界 pattern：custom hook + context

// auth-context.tsx
import { createContext, useContext, useState, ReactNode } from "react";

interface AuthContextValue {
    user: User | null;
    login: (email: string, password: string) => Promise;
    logout: () => void;
}

const AuthContext = createContext(null);

export function AuthProvider({ children }: { children: ReactNode }) {
    const [user, setUser] = useState(null);
    
    const login = async (email: string, password: string) => {
        const res = await fetch("/api/login", {
            method: "POST",
            body: JSON.stringify({ email, password })
        });
        const user = await res.json();
        setUser(user);
    };
    
    const logout = () => setUser(null);
    
    return (
        
            {children}
        
    );
}

// Custom hook（隱藏 useContext 細節）
export function useAuth() {
    const context = useContext(AuthContext);
    if (!context) {
        throw new Error("useAuth must be used within AuthProvider");
    }
    return context;
}

// 用
function App() {
    return (
        
            
        
    );
}

function LoginButton() {
    const { user, login, logout } = useAuth();
    
    if (user) {
        return Logout {user.name};
    }
    return  login("...", "...")}>Login;
}
```

  
### 常用 Context 場景

```text
1. Auth / User
   - 當前登入用戶
   - login / logout

2. Theme
   - light / dark mode
   - 顏色 / 字體

3. Locale / i18n
   - 當前語言
   - 翻譯函式

4. Router
   - 當前路由
   - navigate function
   - React Router 內建 context

5. UI state
   - sidebar open / close
   - modal stack

6. Feature flags
   - A/B test
   - rollout config

不適合 Context：
✗ 頻繁更新的 state（性能問題）
✗ Server data（用 React Query）
✗ Form state（用 React Hook Form）
```

  
### Context 性能陷阱

```text
問題：Context value 變、所有訂閱的 component re-render

❌ 一個大 context
const AppContext = createContext({
    user, theme, locale, notifications, settings...
});

// theme 變、user 訂閱者也 re-render

✓ 拆多個 context
const UserContext = createContext(...);
const ThemeContext = createContext(...);
const LocaleContext = createContext(...);

// 各自獨立、只有相關 component re-render

✓ 把不變的 dispatch / setter 分開
// state context（會變）
const StateContext = createContext(state);

// dispatch context（不變）
const DispatchContext = createContext(dispatch);

// component 只訂閱 dispatch 不會 re-render

進階：
- useContextSelector（lib）—只訂閱部分 value
- 或用 Zustand / Jotai 取代
```

  
### Theme Context 完整範例

```tsx
// theme-context.tsx
import { createContext, useContext, useState, useEffect, ReactNode } from "react";

type Theme = "light" | "dark";

interface ThemeContextValue {
    theme: Theme;
    toggleTheme: () => void;
}

const ThemeContext = createContext(null);

export function ThemeProvider({ children }: { children: ReactNode }) {
    const [theme, setTheme] = useState("light");
    
    useEffect(() => {
        // 載入時讀 localStorage 或 system
        const saved = localStorage.getItem("theme") as Theme | null;
        const systemDark = window.matchMedia("(prefers-color-scheme: dark)").matches;
        setTheme(saved || (systemDark ? "dark" : "light"));
    }, []);
    
    useEffect(() => {
        // 同步到 DOM + localStorage
        document.documentElement.dataset.theme = theme;
        localStorage.setItem("theme", theme);
    }, [theme]);
    
    const toggleTheme = () => {
        setTheme(t => t === "light" ? "dark" : "light");
    };
    
    return (
        
            {children}
        
    );
}

export function useTheme() {
    const context = useContext(ThemeContext);
    if (!context) throw new Error("useTheme requires ThemeProvider");
    return context;
}

// 用
function ThemeToggle() {
    const { theme, toggleTheme } = useTheme();
    return (
        
            {theme === "light" ? "🌙" : "☀️"}
        
    );
}

// 2026 推薦：用 next-themes（Next.js）
// 不用自己寫
```

  
### Context vs Zustand vs Redux

  
|  | Context | Zustand | Redux |
| --- | --- | --- | --- |
| 學習曲線 | 低 | 低 | 高 |
| 內建 | ✓ | ✗（裝） | ✗（裝） |
| 性能 | 差（全 re-render） | 好 | 好 |
| DevTools | 有限 | 有 | 強 |
| 適合 | 低頻更新 state | 大多場景 | 大型企業 |
| 2026 主流 | Auth / Theme | App state | 少數老專案 |

  
### 常見 Context 錯誤

  
- **所有 state 用 Context**—性能差
- **沒 throw error 在 Provider 外**—難 debug
- **頻繁變的 state 用 Context**—大量 re-render
- **Value 是 new object 每 render**—訂閱者全 re-render
- **巢狀太多 Provider**—10 個 Provider hell
- **Server data 用 Context**—用 React Query
- **Form state 用 Context**—用 React Hook Form

  
**☕ 用人話講**

Context 適合「**低頻 + 全域**」state：(1) **Auth user** (2) **Theme** (3) **Locale** (4) **UI state**。**不適合**：(a) 頻繁變的 state（性能） (b) Server data（用 React Query） (c) Form state（用 React Hook Form）。**2026 業界**：Context + Zustand 組合—Context 給穩定的、Zustand 給高頻的。
