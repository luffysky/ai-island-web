### Zod schema 進階

```typescript
const SignupSchema = z.object({
    name: z.string().min(3).max(50),
    email: z.string().email(),
    
    password: z.string()
        .min(8, "At least 8 chars")
        .regex(/[A-Z]/, "Need uppercase")
        .regex(/[0-9]/, "Need number"),
    
    age: z.number().int().min(18).max(150),
    
    agreeToTerms: z.literal(true, {
        errorMap: () => ({ message: "Must agree" })
    }),
    
    role: z.enum(["user", "admin"]),
    bio: z.string().optional(),
    tags: z.array(z.string()).min(1),
    
    website: z.string().url().optional(),
    
    username: z.string()
        .refine(s => !s.includes(" "), "No spaces"),
    
    confirmPassword: z.string()
}).refine(
    (data) => data.password === data.confirmPassword,
    {
        message: "Passwords don't match",
        path: ["confirmPassword"]
    }
);

type SignupInput = z.infer;
```

### Server Actions（Next.js）

```tsx
// app/actions.ts
"use server";

import { z } from "zod";

const Schema = z.object({
    email: z.string().email(),
    password: z.string().min(8)
});

export async function loginAction(formData: FormData) {
    const parsed = Schema.safeParse({
        email: formData.get("email"),
        password: formData.get("password")
    });
    
    if (!parsed.success) {
        return { error: parsed.error.flatten() };
    }
    
    await db.login(parsed.data.email, parsed.data.password);
    redirect("/dashboard");
}
```

### 常見錯誤

  
- **用 useState 寫複雜 form**—50 個 state、地獄
- **沒 schema 驗證**—手寫 if-else 散亂
- **onChange 每次都驗**—太煩、用 onBlur
- **沒 submit disable**—點 5 次送 5 個
- **error 訊息技術化**—"ValidationError"
- **iOS auto-zoom**—input font-size 16px+

  
**☕ 用人話講**

Form 4 心法：(1) **React Hook Form + Zod**—2026 標配 (2) **schema 一處定義**—type + 驗證 + error 一條龍 (3) **onBlur 驗證**—不打擾 (4) **Next.js Server Actions**—漸進採用、無 JS 也能用。
