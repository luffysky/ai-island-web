**React 19 三大新特性**：**Actions（行動）**—— 把 form submit + loading + error + optimistic update 全打包；**`use()` Hook**—— 在 component 內直接 await Promise；**Server Components（伺服器元件）**—— 在伺服器跑的 React。**改變遊戲規則的一次升級**。

> 💡 **這課的價值**：React 19 是 React 18 之後第一次大改、業界一年內會全面採用。會這 3 招、你的 code 比同行少寫 50%。

  
### 1. Actions：async 處理革命

**Before（React 18）— 自己管 pending / error / optimistic：**

```text
📄 寫進程式檔：`Form.tsx`

function Form() {
  const [pending, setPending] = useState(false);
  const [error, setError] = useState<string | null>(null);

  async function handleSubmit(formData: FormData) {
    setPending(true);
    setError(null);
    try {
      await updateName(formData.get("name") as string);
    } catch (e: any) {
      setError(e.message);
    } finally {
      setPending(false);
    }
  }

  return (
    <form action={handleSubmit}>
      <input name="name" />
      <button disabled={pending}>送出</button>
      {error && <p>{error}</p>}
    </form>
  );
}
```

**After（React 19）— `useActionState` 一個 hook 全包：**

```text
import { useActionState } from "react";

function Form() {
  const [error, submitAction, isPending] = useActionState(
    async (prevState, formData) => {
      try {
        await updateName(formData.get("name") as string);
        return null;
      } catch (e: any) {
        return e.message;
      }
    },
    null,
  );

  return (
    <form action={submitAction}>
      <input name="name" />
      <button disabled={isPending}>送出</button>
      {error && <p>{error}</p>}
    </form>
  );
}
```

**`useActionState` 回三個東西：**

```text
[state, action, pending] = useActionState(reducer, initialState)
- state    — action 的回傳值（error / data）
- action   — 把它放 <form action={...}> 或 <button formAction={...}>
- pending  — 正在執行嗎（loading state）
```

> 💡 **Action 接 form 用 `<form action={...}>`**——不是 `onSubmit`、是 `action` 屬性。React 19 重新定義 form 元素。

  
### 2. `useOptimistic`：optimistic update（樂觀更新）

```text
import { useOptimistic } from "react";

function Likes({ count, like }: { count: number; like: () => Promise<void> }) {
  const [optimistic, addOptimistic] = useOptimistic(count, (state) => state + 1);

  async function handleLike() {
    addOptimistic(null);  // 立刻 +1（樂觀）
    await like();          // server 真的執行
  }

  return <button onClick={handleLike}>❤️ {optimistic}</button>;
}
```

**作用**：使用者按 like、UI 立刻顯示 +1（不等 server 回）、server 失敗 React 自動 revert。**iPhone 級 UX**。

  
### 3. `use()` Hook：component 內直接 await

**Before — 用 useEffect + setState：**

```text
function Post({ id }: { id: string }) {
  const [post, setPost] = useState(null);
  useEffect(() => {
    fetch(`/api/posts/${id}`).then(r => r.json()).then(setPost);
  }, [id]);
  if (!post) return <Spinner />;
  return <h1>{post.title}</h1>;
}
```

**After — `use()` 一行解決：**

```text
import { use } from "react";

// 在父元件 / loader 傳 Promise 進來
function Post({ postPromise }: { postPromise: Promise<Post> }) {
  const post = use(postPromise);  // 直接 await
  return <h1>{post.title}</h1>;
}

// 父元件
function Page() {
  const postPromise = fetch('/api/posts/1').then(r => r.json());
  return (
    <Suspense fallback={<Spinner />}>
      <Post postPromise={postPromise} />
    </Suspense>
  );
}
```

**`use()` 的特色：**

```text
✅ 可以在條件 / loop 內呼叫（不像其他 hook）
✅ 配合 <Suspense> 自動處理 loading
✅ 配合 ErrorBoundary 自動處理 error
✅ 也可以 use(Context)（取代 useContext）

→ 是「能更彈性的 await + useContext」
```

  
### 4. Server Components（伺服器元件）

**RSC（React Server Components、伺服器元件）**= 在伺服器跑、不傳 JS 到瀏覽器的 React。

```text
📄 寫進程式檔：`app/posts/page.tsx`

// 預設是 Server Component（Next.js 15 App Router）
import { db } from '@/lib/db';

async function PostsPage() {
  const posts = await db.posts.findMany();  // 直接打 DB
  return (
    <ul>
      {posts.map(p => <li key={p.id}>{p.title}</li>)}
    </ul>
  );
}
```

**Server Component 特色：**

```text
✅ 可以直接 await DB / API（不用 useEffect）
✅ 不傳 JS 到瀏覽器（bundle 變小）
✅ 不能用 useState / useEffect / 事件
✅ 不能用 useContext（用 props 傳）
✅ 可以 import Client Components
```

**需要互動 → Client Component：**

```text
📄 寫進程式檔：`components/Counter.tsx`

"use client";  // ← 這行是 Client Component 標記

import { useState } from "react";

export function Counter() {
  const [n, setN] = useState(0);
  return <button onClick={() => setN(n + 1)}>點 {n} 次</button>;
}
```

  
### 5. Server Actions：form 直接打 server

```text
📄 寫進程式檔：`app/actions.ts`

"use server";  // ← Server Action 標記

import { db } from '@/lib/db';
import { revalidatePath } from 'next/cache';

export async function createPost(formData: FormData) {
  const title = formData.get('title') as string;
  await db.posts.create({ data: { title } });
  revalidatePath('/posts');
}

📄 寫進程式檔：`app/posts/new.tsx`

import { createPost } from '../actions';

export default function NewPost() {
  return (
    <form action={createPost}>
      <input name="title" />
      <button>發布</button>
    </form>
  );
}
```

**Server Action 革命之處：**
- 不用寫 API route
- 不用寫 fetch
- form action 自動把 FormData 傳給 server function
- 自動處理 progressive enhancement（JS 沒載完也能用）

  
### 6. ref as prop（不再需要 forwardRef）

**Before：**

```text
const Input = forwardRef<HTMLInputElement, Props>((props, ref) => (
  <input ref={ref} {...props} />
));
```

**After（React 19）：**

```text
function Input({ ref, ...props }: Props & { ref: Ref<HTMLInputElement> }) {
  return <input ref={ref} {...props} />;
}
```

**`forwardRef` 正式退場**——直接把 ref 當 prop 傳。

  
### 7. `document` metadata 自動 hoist

**Before — 用 react-helmet 或 Next/Head：**

```text
<Head>
  <title>About</title>
  <meta name="description" content="..." />
</Head>
```

**After — 任何 component 內直接寫：**

```text
function AboutPage() {
  return (
    <>
      <title>About — AI 島</title>
      <meta name="description" content="關於我們" />
      <link rel="canonical" href="/about" />

      <h1>關於</h1>
    </>
  );
}
```

React 19 自動把這些 meta 移到 `<head>`。**寫 SEO 從沒這麼簡單**。

  
### 8. 升級到 React 19 checklist

```text
1. npm install react@19 react-dom@19

2. 移除（已 deprecate）：
   ❌ useEffect 內手動 cleanup setState

   ❌ propTypes（用 TS）
   ❌ legacy context（用 createContext）

3. 改寫：
   ✓ forwardRef → 直接 ref prop
   ✓ async data fetch → use() + Suspense
   ✓ form 自己管 state → useActionState

4. 新功能加：
   ✓ Server Components（如果用 Next 15）
   ✓ Server Actions（如果用 Next 15）
   ✓ useOptimistic
   ✓ document metadata
```

  
### 9. 5 大常見坑

```text
❌ 1. 在 Server Component 用 useState
   → 編譯時直接報錯
   → 修：互動部分抽出來、加 "use client"

❌ 2. 在 Client Component import server function
   → bundle 會包進去、安全洩漏
   → 修：用 Server Action（"use server"）

❌ 3. useActionState 跟 useState 混用
   → 兩套 state、容易不同步
   → 修：選一個用

❌ 4. 用 use() 但沒包 Suspense
   → render 失敗、screen 空白
   → 修：永遠配 <Suspense fallback={...}>

❌ 5. Server Action 沒 revalidate
   → DB 改了、UI 還是舊資料
   → 修：呼叫 revalidatePath() 或 revalidateTag()
```

  
### 🎯 重點回顧

- **Actions** = form submit 一個 hook 全包（pending / error / state）
- **`use()`** = component 內直接 await Promise（配 Suspense）
- **Server Components** = 在伺服器跑、不傳 JS、可直接打 DB
- **Server Actions** = form 直接打 server function、不用寫 API
- **document metadata** 任何 component 內寫、自動 hoist 到 head
