**Server Components vs Client Components**——React 19 + Next.js 15 的核心觀念。預設 Server、需要互動才標 `"use client"`。**搞清楚這個分界、你寫的 app 比同行快 3 倍**。

> 💡 **這課的價值**：80% 的新手 React 19 + Next 15 bug 出在「該 Server 卻寫 Client」或反過來。學會這課、bundle size 從 500KB 降到 100KB、首頁 LCP 從 3 秒變 0.8 秒。

  
### 1. 預設是 Server Component（Next.js 15）

```text
📄 寫進程式檔：`app/page.tsx`

// 沒寫 "use client" → 自動是 Server Component
import { db } from '@/lib/db';

async function Page() {
  const data = await db.posts.findMany();  // 直接打 DB
  return <div>{data.length} 篇文章</div>;
}

export default Page;
```

**Server Component 能做的：**

```text
✅ async / await（component 本身可以 async）
✅ 直接 import 並使用 server-only 套件（DB / fs / secret API）
✅ 讀 cookies / headers
✅ 完全不傳 JS 到瀏覽器
✅ Bundle 大小不被影響（zero impact）
```

**Server Component 不能做的：**

```text
❌ useState / useEffect / useRef / 任何 hook
❌ onClick / onChange 事件
❌ window / document / localStorage（瀏覽器 API）
❌ useContext（要在 Client 用）
❌ class components（只能 function）
```

  
### 2. 需要互動 → `"use client"`

```text
📄 寫進程式檔：`components/Counter.tsx`

"use client";  // ← 第一行、Client Component 標記

import { useState } from "react";

export function Counter() {
  const [n, setN] = useState(0);
  return <button onClick={() => setN(n + 1)}>點 {n} 次</button>;
}
```

**`"use client"` 是「邊界」標記：**

```text
- 寫了 "use client" → 這個 file 跟所有它 import 的 component 都會被打包進 client bundle
- 一個 Client Component 內 import 的 Server Component 不會自動變 Client
  → 要把 Server Component 用 props/children 傳進來才能在 Client 內 render
```

  
### 3. 該用哪個的 5 個判斷問題

```text
問 1：需要 useState / useEffect / 事件嗎？
→ 需要 → Client
→ 不需要 → Server

問 2：要存取 localStorage / window / document？
→ 需要 → Client
→ 不需要 → Server

問 3：要直接打 DB / 讀檔 / 用 server-only secret API？
→ 需要 → Server
→ 不需要 → 看其他條件

問 4：希望 bundle 變小、傳少 JS？
→ 對 → Server（能 Server 就 Server）

問 5：要 useContext / 用 React Query？
→ 對 → Client（這些 hook 不在 Server 跑）
```

> 💡 **預設思維**：「我這個 component 真的需要瀏覽器才能跑嗎？」答 NO 就 Server。

  
### 4. 真實架構 pattern

**Pattern 1：Server 抓資料、傳給 Client 互動**

```text
📄 寫進程式檔：`app/posts/page.tsx`（Server）

import { db } from '@/lib/db';
import { PostList } from '@/components/PostList';

async function PostsPage() {
  const posts = await db.posts.findMany();
  return <PostList posts={posts} />;  // 傳給 Client
}

📄 寫進程式檔：`components/PostList.tsx`（Client）

"use client";
import { useState } from "react";

export function PostList({ posts }: { posts: Post[] }) {
  const [filter, setFilter] = useState("");
  const filtered = posts.filter(p => p.title.includes(filter));
  return (
    <>
      <input value={filter} onChange={e => setFilter(e.target.value)} />
      <ul>{filtered.map(p => <li key={p.id}>{p.title}</li>)}</ul>
    </>
  );
}
```

**Pattern 2：Client 內塞 Server children**

```text
📄 寫進程式檔：`app/layout.tsx`（Server）

import { Sidebar } from '@/components/Sidebar';
import { ServerSideStats } from '@/components/ServerSideStats';

export default function Layout({ children }: { children: React.ReactNode }) {
  return (
    <Sidebar>  {/* Sidebar 是 Client */}
      <ServerSideStats />  {/* 但這個還是 Server */}
      {children}
    </Sidebar>
  );
}
```

**重點**：`<Sidebar>` 是 Client、但它的 children（`<ServerSideStats>`）還是 Server——因為是用 props 傳進去的。

  
### 5. Client 內想抓資料怎麼辦

```text
🥇 推薦：React Query（client-side fetching）
   "use client";
   import { useQuery } from '@tanstack/react-query';

   function UserList() {
     const { data } = useQuery({
       queryKey: ['users'],
       queryFn: () => fetch('/api/users').then(r => r.json()),
     });
     return <ul>{data?.map(u => <li>{u.name}</li>)}</ul>;
   }

🥈 use() Hook + Suspense（React 19）
   function Page() {
     const promise = fetch('/api/users').then(r => r.json());
     return (
       <Suspense fallback={<Spinner />}>
         <UserList promise={promise} />
       </Suspense>
     );
   }
```

  
### 6. 共享 component 怎麼跨 Server / Client 用

```text
✅ 規則：別在 Server / Client 邊界做特殊事

範例：一個 <Button>
"use client";
export function Button({ onClick, children }) {
  return <button onClick={onClick}>{children}</button>;
}

→ 在 Server 也能 import 用、但會被視為 Client component
→ 整顆 button bundle 進 client（其實是必須的、因為要 onClick）

範例：純樣式 <Card>（沒互動）
export function Card({ children }) {
  return <div className="card">{children}</div>;
}

→ 沒 "use client"、自動 Server
→ Server / Client 都能用
→ Zero bundle 影響
```

  
### 7. 傳 props 跨邊界的規則

從 Server → Client 傳 props 必須是「**可序列化（serializable）**」的：

```text
✅ 可以傳：
- 基本型別（string / number / boolean / null）
- 純物件 / 陣列
- Date / Map / Set / TypedArray
- Promise（搭配 use()）
- Server Component（當 children）

❌ 不能傳：
- function（呼叫 server function 要用 Server Action）
- class instance
- Symbol（除 well-known）
- React element（Server Component 已經處理過）
```

如果你需要在 Client 呼叫 server function，**用 Server Action**：

```text
"use server";
export async function deletePost(id: string) {
  await db.posts.delete({ where: { id } });
}

// Client 內：
import { deletePost } from './actions';
<button onClick={() => deletePost('123')}>刪除</button>
```

  
### 8. Bundle size 對比（真實數字）

```text
案例：典型 dashboard 頁

❌ 全 Client（Next 13 之前主流）：
   首頁 bundle ~ 350-500 KB（gzipped 100-150 KB）
   FCP（first contentful paint）：2-3 秒
   LCP：3-5 秒

✅ 80% Server + 20% Client（Next 15）：
   首頁 bundle ~ 80-150 KB（gzipped 30-50 KB）
   FCP：0.5-0.8 秒
   LCP：1-1.5 秒

→ 60-70% bundle size 減少
→ 50%+ 載入速度提升
```

  
### 9. 5 大常見坑

```text
❌ 1. 整個 page.tsx 加 "use client"
   → 整顆 tree 都變 Client、Server 優勢全失
   → 修：只把「真的需要互動」的小元件加 "use client"

❌ 2. Server Component 用 useState
   → 編譯時錯誤
   → 修：抽出來變 Client Component

❌ 3. 在 Client component import server-only 套件（如 fs / db client）
   → 編譯時錯誤、或執行時洩漏
   → 修：透過 Server Action / API route 隔離

❌ 4. 傳 function 當 prop 給 Server Component
   → 不能序列化、報錯
   → 修：function 在 Server Component 內定義、不要傳

❌ 5. 全在 Server 寫、互動全靠 form
   → UX 卡（每次互動都 full reload）
   → 修：互動部分用 Client Component
```

  
### 🎯 重點回顧

- **預設 Server、需要互動才 Client** —— 不是反過來
- **5 個判斷問題**：state? localStorage? DB? bundle? context?
- **Server 抓資料 → Client 互動** 是最常用 pattern
- **Server children 可以塞進 Client component**（用 props/children）
- **80% Server + 20% Client = bundle 小 70%、速度快 50%**
