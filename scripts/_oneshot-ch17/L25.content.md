**Full-Text Search（FTS、全文檢索）= 中型搜尋場景的最佳解**—— `ILIKE '%關鍵字%'` 慢又不準、PG 內建 FTS 又快又會「自然語言切詞」、相關性排序也內建。

> 💡 **這課的價值**：搜尋功能多數 Indie / 中型 SaaS 不用 Elasticsearch、PG 內建 FTS 就夠——少一個服務、省一份維運。本課把 tsvector / tsquery / GIN Index 從零講透。


### 1. 為什麼 ILIKE 不夠用？

```text
SELECT * FROM articles WHERE content ILIKE '%machine learning%';
```

**3 大缺點：**

```text
❌ 1. 慢
   左側 % 阻擋 B-tree Index、全表掃
   100 萬篇文章 = 10 秒以上

❌ 2. 不準
   搜「running」找不到「run」（不會 stemming）
   搜「the cat」要剛好那字串才到

❌ 3. 沒排序
   只能「有沒有」、不能「多相關」
```


### 2. FTS 怎麼解決？

**核心觀念：**

```text
1. 文章先「切詞 + 詞幹化」
   "running fast" → ['run', 'fast']  ← tsvector

2. 搜尋字也切詞 + 詞幹化
   "ran" → 'run'  ← tsquery

3. 用 GIN Index 加速「詞 → 文章」反向查
   像書末的「索引」一樣
```


### 3. tsvector —— 把文字切成「詞向量」

```text
SELECT to_tsvector('english', 'The quick brown fox jumps over the lazy dog');
```

💬 **電腦會顯示**：

```text
'brown':3 'dog':9 'fox':4 'jump':5 'lazi':8 'quick':2
```

**做了 3 件事：**

```text
1. 去掉 stopwords（the、over）
2. 詞幹化（jumps → jump、lazy → lazi）
3. 記錄位置（:1 :2 :3 ...）
```


### 4. tsquery —— 把搜尋字切成「查詢向量」

```text
SELECT to_tsquery('english', 'jumping & fox');
```

💬 **電腦會顯示**：

```text
'jump' & 'fox'    ← 自動把 jumping → jump、保留 & 運算
```

**tsquery 運算子：**

```text
&      AND
|      OR
!      NOT
<->    緊跟（jump <-> fox = jump 後直接接 fox）
<N>    間隔 N 個詞
( )    分組
```


### 5. @@ —— FTS 的核心運算子

`@@` 表「tsvector match tsquery」：

```text
SELECT to_tsvector('english', 'The quick brown fox')
    @@ to_tsquery('english', 'fox & quick');
-- 結果：t（true）

SELECT to_tsvector('english', 'A lazy cat')
    @@ to_tsquery('english', 'fox & quick');
-- 結果：f
```


### 6. 完整建表 + 搜尋

```text
CREATE TABLE articles (
  id        SERIAL PRIMARY KEY,
  title     TEXT NOT NULL,
  content   TEXT NOT NULL,
  -- 自動算 tsvector（generated column、PG 12+）
  search    TSVECTOR GENERATED ALWAYS AS (
    setweight(to_tsvector('english', coalesce(title, '')), 'A') ||
    setweight(to_tsvector('english', coalesce(content, '')), 'B')
  ) STORED
);

-- 建 GIN Index
CREATE INDEX idx_articles_search ON articles USING GIN (search);

-- 寫資料
INSERT INTO articles (title, content) VALUES
  ('Machine Learning 101', 'Introduction to ML and deep learning...'),
  ('Quick fox jumps', 'A quick brown fox jumps over the lazy dog'),
  ('Cooking pasta', 'How to cook pasta with tomato sauce');

-- 搜尋
SELECT id, title
FROM articles
WHERE search @@ to_tsquery('english', 'machine & learning');
```

**重點**：

```text
setweight + 'A' / 'B' / 'C' / 'D'   給不同欄位「權重」
                                     title 比 content 重要
GENERATED ALWAYS AS ... STORED       自動算、寫入時生成
GIN Index                            FTS 必備
```


### 7. plainto_tsquery / websearch_to_tsquery —— 對使用者輸入友善

`to_tsquery` 要使用者輸入 `&`、`|`——不直覺。**plainto_tsquery** 把空格自動轉 `&`：

```text
SELECT plainto_tsquery('english', 'machine learning');
-- 結果：'machine' & 'learn'

SELECT websearch_to_tsquery('english', '"machine learning" OR AI -spam');
-- 支援 Google 風格："phrase"、OR、減號
```

**Production 推薦用 `websearch_to_tsquery`**——使用者已熟悉 Google 搜尋語法。


### 8. ts_rank —— 相關性排序

```text
SELECT
  id,
  title,
  ts_rank(search, query) AS rank
FROM articles, plainto_tsquery('english', 'machine learning') query
WHERE search @@ query
ORDER BY rank DESC
LIMIT 10;
```

💬 **電腦會顯示**：

```text
 id |        title         |   rank
----+----------------------+-----------
  1 | Machine Learning 101 | 0.0865734
  3 | Deep Learning Guide  | 0.0608445
  5 | Intro to AI/ML       | 0.0303809
```

**ts_rank** 算「文章多相關」——越大越相關。


### 9. ts_headline —— 在結果裡 highlight

```text
SELECT
  id,
  title,
  ts_headline('english', content, query, 'StartSel=<b>, StopSel=</b>') AS snippet
FROM articles, plainto_tsquery('english', 'machine') query
WHERE search @@ query;
```

💬 **電腦會顯示**：

```text
snippet：Introduction to <b>ML</b> and deep <b>learning</b>...
```

**Google 搜尋結果那種「片段 + 黃色 highlight」效果**。


### 10. 中文全文搜尋

**英文 PG 內建詞幹化、但中文要外掛**：

```text
-- 中文無法切詞、英文 stemming 對中文沒用
-- 需要：
1. zhparser / pgroonga / cn_jieba 等 extension
2. 或先在應用層用 jieba 切詞、存切好的 tsvector

-- Supabase 預設沒中文 stemmer、用 simple：
SELECT to_tsvector('simple', '我愛學 SQL');
-- 結果：'sql':4 '我愛學':1-3  ← 只是空白分詞、沒切「我 / 愛 / 學」
```

**業界做法**：
- 中文輕量搜尋：應用層 jieba 切完 → 存空格分隔 → simple tsvector
- 中文重要搜尋：直接上 Elasticsearch / Meilisearch / Typesense


### 11. ILIKE vs FTS vs Elasticsearch 三選一

```text
ILIKE
   ✅ 簡單、不用設定
   ✅ 小資料（< 10 萬筆）夠快
   ❌ 不能 Index、慢、沒排序

PG FTS
   ✅ 中型場景（10 萬 - 1000 萬筆）夠用
   ✅ 跟主資料同庫、不用同步
   ✅ 免額外服務
   ❌ 中文要外掛 / 應用層處理

Elasticsearch
   ✅ 億級資料、高併發
   ✅ 中文支援完善（IK 分詞器）
   ✅ 進階功能多（aggregation、geo）
   ❌ 額外服務、要同步資料、運維成本

選擇：
   < 10 萬筆 → ILIKE 或 FTS
   10 萬 - 1000 萬 → FTS
   > 1000 萬 + 中文 + 高併發 → Elasticsearch
```


### 12. 完整實戰：部落格搜尋 API

```text
-- 1. schema
CREATE TABLE posts (
  id          SERIAL PRIMARY KEY,
  title       TEXT NOT NULL,
  content     TEXT NOT NULL,
  tags        TEXT[],
  published   BOOLEAN DEFAULT FALSE,
  created_at  TIMESTAMPTZ DEFAULT NOW(),
  search      TSVECTOR GENERATED ALWAYS AS (
    setweight(to_tsvector('english', coalesce(title, '')), 'A') ||
    setweight(to_tsvector('english', array_to_string(coalesce(tags, '{}'), ' ')), 'B') ||
    setweight(to_tsvector('english', coalesce(content, '')), 'C')
  ) STORED
);

CREATE INDEX idx_posts_search ON posts USING GIN (search);

-- 2. 搜尋 API 用的 query
SELECT
  id,
  title,
  ts_headline('english', content,
    websearch_to_tsquery('english', $1),
    'MaxWords=30, MinWords=10, StartSel=<mark>, StopSel=</mark>'
  ) AS snippet,
  ts_rank(search, websearch_to_tsquery('english', $1)) AS rank
FROM posts
WHERE published = TRUE
  AND search @@ websearch_to_tsquery('english', $1)
ORDER BY rank DESC, created_at DESC
LIMIT 20;
```


### 13. 5 大常見坑

```text
❌ 1. 沒建 GIN Index
   → FTS 一樣慢（甚至比 ILIKE 慢）
   → 修：CREATE INDEX ... USING GIN

❌ 2. 用 to_tsquery 接使用者輸入
   → 使用者輸入空格或符號 → 解析錯誤、報錯
   → 修：用 websearch_to_tsquery（容錯強）

❌ 3. 中文用 english stemmer
   → 切詞錯亂、搜不到
   → 修：用 simple、配合應用層切詞

❌ 4. tsvector 沒 setweight 排序就亂
   → title 跟 content 一視同仁、ts_rank 不準
   → 修：setweight 給 title 'A'、content 'B'

❌ 5. 沒用 generated column、要 trigger 維護
   → 寫入時忘了更新 search 欄位
   → 修：PG 12+ 用 GENERATED ALWAYS AS ... STORED 自動算
```


### 🎯 重點回顧

- **PG 內建 FTS 解決中型搜尋場景**、不用 Elasticsearch
- **tsvector（文章詞向量）+ tsquery（搜尋詞向量）+ `@@` 比對**
- **websearch_to_tsquery 對使用者輸入最友善**（支援 "phrase"、OR、減號）
- **GIN Index 是 FTS 的命**——一定要建
- **setweight + ts_rank** 做相關性排序
- **ts_headline** 自動產生 highlight snippet
- **中文要外掛 / 應用層切詞**——或直接上 Elasticsearch
