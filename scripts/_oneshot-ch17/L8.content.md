**GROUP BY（分組）+ aggregate function（聚合函式）= SQL 的統計神器**—— 寫「每個城市幾個人」、「每月營業額」、「平均訂單金額」——全靠這套。

> 💡 **這課的價值**：後台報表、儀表板、商業分析——90% 都是 GROUP BY + COUNT/SUM/AVG。會這課等於拿到「資料分析師」的入場券。


### 1. 為什麼需要 GROUP BY？

**問題**：「每個城市有幾個 user？」

```text
SELECT city FROM users;
-- 6 筆原始列：台北、台北、台中、高雄、台北、台中
-- 我們要的是：台北 3、台中 2、高雄 1
```

**人類做法**：用 Excel 樞紐分析。**SQL 做法**：GROUP BY。

```text
SELECT city, COUNT(*) AS total
FROM users
GROUP BY city;
```

💬 **電腦會顯示**：

```text
 city  | total
-------+-------
 台北  |     3
 台中  |     2
 高雄  |     1
```


### 2. 5 大 aggregate function（聚合函式）

```text
COUNT(*)      共幾筆（含 NULL）
COUNT(col)    col 不是 NULL 的筆數
COUNT(DISTINCT col)  不重複的數量
SUM(col)      加總（只能算數字）
AVG(col)      平均（只能算數字）
MIN(col)      最小（數字、文字、日期都行）
MAX(col)      最大
```

**範例：**

```text
SELECT
  COUNT(*) AS total_users,
  COUNT(age) AS has_age,            -- age 不是 NULL 的數量
  COUNT(DISTINCT city) AS cities,   -- 不重複的城市數
  AVG(age) AS avg_age,
  MAX(age) AS max_age,
  MIN(age) AS min_age,
  SUM(age) AS total_age
FROM users;
```

💬 **電腦會顯示**（1 列、整張表的統計）：

```text
 total_users | has_age | cities | avg_age | max_age | min_age | total_age
-------------+---------+--------+---------+---------+---------+-----------
           6 |       6 |      3 | 29.0000 |      32 |      26 |       174
```


### 3. GROUP BY + 聚合（黃金組合）

**範例：每個城市的人數 + 平均年齡：**

```text
SELECT
  city,
  COUNT(*) AS people,
  AVG(age)::INT AS avg_age,
  MAX(age) AS oldest
FROM users
GROUP BY city;
```

💬 **電腦會顯示**：

```text
 city  | people | avg_age | oldest
-------+--------+---------+--------
 台北  |      3 |      29 |     30
 台中  |      2 |      27 |     28
 高雄  |      1 |      32 |     32
```

**思考方式：**

```text
1. GROUP BY city   把同 city 的列「合併成一組」
2. SELECT 聚合     對每組算 COUNT / AVG / MAX
3. 每組產生 1 列
```


### 4. GROUP BY 規則（最常踩的坑）

**鐵則**：SELECT 裡只能放兩種東西：

```text
✅ GROUP BY 裡的欄位（city）
✅ aggregate function（COUNT / SUM / ...）
❌ 其他欄位（name、email...）
```

**為什麼？**因為一組裡有多個 name、SQL 不知道顯示誰。

```text
SELECT city, name, COUNT(*) FROM users GROUP BY city;
-- ❌ ERROR: column "users.name" must appear in GROUP BY clause
-- 或被用 aggregate function
```

**正解：**

```text
SELECT city, COUNT(*), STRING_AGG(name, ', ') AS names
FROM users
GROUP BY city;
```

`STRING_AGG` 把同組的 name 拼成 `'Luffy, Xueyue, Sanji'`。


### 5. 多欄位 GROUP BY

**範例：每個（城市、年齡）組合有幾人：**

```text
SELECT city, age, COUNT(*) AS cnt
FROM users
GROUP BY city, age
ORDER BY city, age;
```

💬 **電腦會顯示**：

```text
 city  | age | cnt
-------+-----+-----
 台北  |  28 |   1
 台北  |  30 |   2
 台中  |  26 |   1
 台中  |  28 |   1
 高雄  |  32 |   1
```


### 6. HAVING —— 「過濾分組結果」

WHERE 是「過濾原始列」、HAVING 是「過濾分組後的結果」。

**範例：找出「人數 ≥ 2 的城市」：**

```text
SELECT city, COUNT(*) AS people
FROM users
GROUP BY city
HAVING COUNT(*) >= 2;
```

💬 **電腦會顯示**：

```text
 city  | people
-------+--------
 台北  |      3
 台中  |      2
```

高雄被擋掉（只有 1 人）。

**WHERE vs HAVING 差別：**

```text
WHERE  在 GROUP BY 之前跑、過濾原始列
HAVING 在 GROUP BY 之後跑、過濾聚合結果

例 1：只看「年齡 ≥ 28」的人、再分組
  WHERE age >= 28
  GROUP BY city
  → 先濾人、再分組

例 2：分組後、留「人數 ≥ 2」的城市
  GROUP BY city
  HAVING COUNT(*) >= 2
  → 先分組、再濾組
```

**兩個一起用：**

```text
SELECT city, AVG(age) AS avg_age, COUNT(*) AS people
FROM users
WHERE age >= 25            -- 先過濾人
GROUP BY city
HAVING COUNT(*) >= 2       -- 再過濾組
ORDER BY avg_age DESC;
```


### 7. 完整 SELECT 執行順序（再強調）

```text
1. FROM        從哪張表
2. WHERE       過濾原始列  ← 不能用 aggregate
3. GROUP BY    分組
4. HAVING      過濾分組    ← 可以用 aggregate
5. SELECT      算欄位
6. DISTINCT    去重
7. ORDER BY    排序
8. LIMIT       取 N 筆
```

**寫的順序跟跑的順序不一樣！**這就是為什麼：

- WHERE 不能寫 SUM(amount) > 100（WHERE 跑時還沒算 SUM）
- HAVING 可以寫 SUM(amount) > 100（HAVING 在 SELECT 後）
- ORDER BY 可以用 SELECT 的別名（ORDER BY 在 SELECT 後）


### 8. 實戰範例：電商儀表板

🖥️ **準備一張訂單表**：

```text
CREATE TABLE orders (
  id          SERIAL PRIMARY KEY,
  user_id     INT NOT NULL,
  city        TEXT NOT NULL,
  amount      NUMERIC(10, 2) NOT NULL,
  created_at  DATE NOT NULL
);

INSERT INTO orders (user_id, city, amount, created_at) VALUES
  (1, '台北', 1200, '2026-05-01'),
  (1, '台北', 850, '2026-05-15'),
  (2, '台中', 3200, '2026-05-03'),
  (3, '台北', 5500, '2026-05-20'),
  (2, '台中', 750, '2026-05-22'),
  (4, '高雄', 9900, '2026-05-10'),
  (1, '台北', 2100, '2026-04-25');
```

**🔍 題目 1：每個城市的「訂單總額、訂單數、平均訂單」：**

```text
SELECT
  city,
  COUNT(*) AS order_count,
  SUM(amount) AS revenue,
  ROUND(AVG(amount), 2) AS avg_order
FROM orders
GROUP BY city
ORDER BY revenue DESC;
```

💬 **電腦會顯示**：

```text
 city  | order_count | revenue | avg_order
-------+-------------+---------+-----------
 高雄  |           1 |    9900 |   9900.00
 台北  |           4 |    9650 |   2412.50
 台中  |           2 |    3950 |   1975.00
```

**🔍 題目 2：「5 月份營業額」（過濾 + 分組）：**

```text
SELECT
  city,
  SUM(amount) AS may_revenue
FROM orders
WHERE created_at >= '2026-05-01'
  AND created_at <  '2026-06-01'
GROUP BY city
HAVING SUM(amount) > 1000
ORDER BY may_revenue DESC;
```

**🔍 題目 3：「客戶總消費排名」：**

```text
SELECT
  user_id,
  COUNT(*) AS orders,
  SUM(amount) AS total_spent
FROM orders
GROUP BY user_id
ORDER BY total_spent DESC
LIMIT 5;
```


### 9. 進階：GROUPING SETS / ROLLUP / CUBE（一個 query 多層分組）

**ROLLUP（層級總和）：**

```text
SELECT city, EXTRACT(MONTH FROM created_at) AS month, SUM(amount)
FROM orders
GROUP BY ROLLUP(city, month);
```

會額外產生：每個 city 的小計、整體總計列。**Excel 樞紐的「合計」就是這個**。


### 10. 5 大常見坑

```text
❌ 1. SELECT 出現非 GROUP BY 欄位、又不是 aggregate
   → 報錯：must appear in GROUP BY clause
   → 修：加進 GROUP BY、或用聚合函式包

❌ 2. WHERE 裡寫 aggregate
   → WHERE COUNT(*) > 5 ❌（WHERE 階段還沒算 COUNT）
   → 修：用 HAVING

❌ 3. COUNT(col) 跟 COUNT(*) 搞混
   → COUNT(*) 算全部、COUNT(col) 不算 col 是 NULL 的
   → 修：要算「全部筆數」永遠 COUNT(*)

❌ 4. 想拿「每組第一筆 row」用 GROUP BY
   → GROUP BY 是「合併」、不是「取代表」
   → 修：用 Window Function ROW_NUMBER（L9）

❌ 5. 浮點數平均誤差
   → AVG 用 FLOAT/REAL 有精度問題
   → 修：金額用 NUMERIC、AVG 結果 ROUND 一下
```


### 🎯 重點回顧

- **GROUP BY 把同類列合併成一組**、每組產生 1 列
- **5 大 aggregate**：COUNT / SUM / AVG / MIN / MAX
- **SELECT 鐵則**：只能放 GROUP BY 欄位 + aggregate
- **WHERE vs HAVING**：WHERE 過濾原始列（不能 aggregate）、HAVING 過濾分組結果
- **執行順序**：FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY → LIMIT

> 💡 **下一課（L9 Window Functions）**：學「每組的第一名」、「跟上一列比較」、「累計總額」——GROUP BY 做不到的、Window Function 全包。
