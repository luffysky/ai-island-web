**SQL（Structured Query Language、結構化查詢語言）**—— 1970 年發明、2026 年還是後端主流、學一次用一輩子的資料庫語言。發音「sequel」或「S-Q-L」都通。

> 💡 **這課的價值**：SQL 是「跟資料庫對話」的語言。網站、App、後台、AI、報表全都在用——學會 SQL 等於拿到後端世界的入場券。**而且 50 年前發明的語法跟今天 99% 一樣**、學了不會過時。


### 1. 什麼是「資料庫」、什麼又是「SQL」

先把兩個常被搞混的詞分清楚。

```text
🗄️ 資料庫（Database、DB）
   → 一個專門「存資料」的軟體
   → 像超大型的 Excel、但能裝幾億筆、不會卡
   → 例子：MySQL、PostgreSQL、SQLite、Oracle

💬 SQL（Structured Query Language）
   → 跟資料庫「下指令」用的語言
   → 例子：「給我所有 18 歲以上的會員」
   → 上面那些資料庫、全部聽得懂 SQL
```

> 💡 **比喻**：資料庫是「圖書館」、SQL 是「跟管理員講話的語言」。圖書館有很多種（MySQL、PostgreSQL...）、但講話用同一種語言（SQL）、所以學一個會全部。


### 2. SQL 的歷史（為什麼活 55 年還沒死）

```text
1970 年
   Edgar Codd（IBM 工程師）發表論文
   提出「關聯式模型（Relational Model）」
   用數學集合論處理資料

1974 年
   IBM 實作出第一版 SQL（當時叫 SEQUEL）

1986 年
   ANSI（美國國家標準協會）把 SQL 訂為國家標準
   → 從此「SQL 是什麼樣子」全世界統一

1995 年
   MySQL 開源、網站時代起飛
   全球 80% 網站後端用 MySQL

2010 年代
   NoSQL（MongoDB、Redis...）冒出來、宣告「SQL 已死」

2026 年
   SQL 還是後端主流、NoSQL 反而退潮
   PostgreSQL 變成新王者（AI 應用幾乎都用它）
```

**Edgar Codd 厲害在哪？**他把「資料」變成數學——你不用告訴電腦「怎麼找」、只要告訴它「要什麼」、電腦自己優化。這叫**宣告式語言（Declarative Language）**、跟我們平常寫的 Python / JavaScript（指令式 Imperative）完全不同。


### 3. 第一個 SQL 範例（讀起來像英文）

```text
SELECT name, email
FROM users
WHERE age >= 18
ORDER BY created_at DESC
LIMIT 10;
```

**逐句翻譯成中文：**

```text
SELECT name, email         挑出 name 跟 email 兩欄
FROM users                 從 users 這張表
WHERE age >= 18            條件：age 大於等於 18
ORDER BY created_at DESC   依 created_at 由新到舊排
LIMIT 10;                  只取前 10 筆
```

**對比同樣事情用 JavaScript 寫：**

```text
const result = users
  .filter(u => u.age >= 18)
  .sort((a, b) => b.created_at - a.created_at)
  .slice(0, 10)
  .map(u => ({ name: u.name, email: u.email }));
```

SQL 比較短、而且讀起來像英文。**這就是 Codd 的設計哲學**——讓寫的人講「要什麼」、電腦自己想「怎麼做」。


### 4. 2026 年該學哪一種 SQL 資料庫？

```text
🐬 MySQL
   優點：最熱門、教學最多、便宜的雲服務最齊
   缺點：功能比 PostgreSQL 弱一截
   用在：WordPress / 蝦皮 / Facebook / 老牌系統

🐘 PostgreSQL（簡稱 PG）
   優點：功能最強、JSON / 向量 / GIS 全內建
   缺點：學習曲線稍陡
   用在：Indie SaaS / AI 應用 / 大型企業

📦 SQLite
   優點：單一檔案、不用裝 server、手機 App 必備
   缺點：不適合多人同時寫
   用在：iOS / Android / 小型工具 / 學習

🚀 Supabase
   = PostgreSQL 雲端版 + 認證 + Realtime + Storage
   優點：10 分鐘起一個 Indie SaaS、Free tier 大方
   用在：個人專案 / MVP / Indie Hacker
```

**林董的建議**：**學 PostgreSQL**。理由：

1. 語法跟 MySQL 95% 共通、學了兩個都會
2. AI 時代 vector DB（pgvector）建在 PG 上、絕對碰得到
3. Supabase 用 PG、Indie 開發直接起飛
4. 企業也愛 PG、職缺多


### 5. SQL 50 年不死的 5 個理由

```text
✅ 1. 標準化超徹底
   ANSI SQL 標準從 1986 年到現在
   學一次、所有資料庫都通

✅ 2. 宣告式 = 對 AI 超友善
   你跟 ChatGPT 講「寫個 SQL 找出 30 天內回購的客戶」
   它幾乎都寫得對、因為 SQL 接近自然語言

✅ 3. 關聯式模型解決核心問題
   訂單 → 客戶 → 商品、用 JOIN 一秒拼起來
   NoSQL 處理「關聯」反而很痛

✅ 4. ACID 交易保證
   轉帳「扣 A 加 B」要嘛都成、要嘛都不成
   這種保證 NoSQL 補不上來

✅ 5. AI 應用反而救了 SQL
   RAG / Embedding 全往 pgvector 跑
   PostgreSQL 變 AI 時代的核心基礎設施
```


### 6. NoSQL 跟 SQL 不是二選一

很多新手以為「NoSQL 是新東西、SQL 是舊東西」——**錯**。

```text
SQL（關聯式）
   結構固定、關聯複雜、要保證一致
   → 訂單系統、會員系統、財務系統
   → 例：PostgreSQL / MySQL

NoSQL（非關聯式）
   結構彈性、單筆讀寫超快、能水平擴張
   → 聊天記錄、log、即時排行榜
   → 例：MongoDB / Redis / DynamoDB
```

**業界實務**：大公司**兩種都用**。主要資料放 SQL（會員、訂單、商品）、快取跟 log 放 Redis、搜尋放 Elasticsearch。**沒有誰取代誰、是分工**。


### 7. 學 SQL 的常見坑（5 個一次擋）

```text
❌ 1. 以為 SQL 是「程式語言」
   → 它是「查詢語言」、跟 Python / JS 思路完全不同
   → 修：練習「描述想要什麼」、不要寫 for-loop 思維

❌ 2. 一開始就學 JOIN
   → 還沒搞懂 SELECT 就跳 JOIN、會崩潰
   → 修：先把 SELECT / WHERE / ORDER BY 練熟（L5-L7）

❌ 3. 全部存 VARCHAR
   → 數字欄位也用文字、排序錯、計算也錯
   → 修：每個欄位選對型別（L3 會教）

❌ 4. 字串拼 SQL 怕 SQL Injection
   → 直接把使用者輸入塞進 SQL、駭客一秒清庫
   → 修：用 prepared statement（L26 整課講）

❌ 5. 不加 Index
   → 資料一多查詢慢到死
   → 修：常查的欄位加 Index（L19 整課講）
```


### 🎯 重點回顧

- **SQL（Structured Query Language）**= 跟資料庫對話的語言、1970 年發明、2026 年還是主流
- **資料庫** vs **SQL** 不是同一件事——資料庫是軟體、SQL 是語言
- **2026 學 PostgreSQL** 最划算（職缺多 + AI 應用 + Supabase）
- **SQL 是宣告式語言**、講「要什麼」不是「怎麼做」、對 AI 超友善
- **SQL 跟 NoSQL 不是二選一**、業界兩種一起用
