## 普通 View
```sql
CREATE VIEW active_users AS
SELECT id, name, email
FROM users
WHERE deleted_at IS NULL AND banned = FALSE;

-- 使用（跟普通表一樣）
SELECT * FROM active_users WHERE name LIKE '王%';
```

## 複雜 View（封裝 JOIN）
```sql
CREATE VIEW user_stats AS
SELECT
  u.id,
  u.name,
  COUNT(o.id) AS order_count,
  COALESCE(SUM(o.amount), 0) AS total_spent,
  MAX(o.created_at) AS last_order_at
FROM users u
LEFT JOIN orders o ON u.id = o.user_id
WHERE u.deleted_at IS NULL
GROUP BY u.id, u.name;

-- 使用
SELECT * FROM user_stats ORDER BY total_spent DESC LIMIT 10;
```

## Materialized View（快取）
普通 View 每次查都重算。Materialized View 把結果存起來、查超快、但要 refresh。

```sql
CREATE MATERIALIZED VIEW monthly_sales AS
SELECT
  DATE_TRUNC('month', created_at) AS month,
  SUM(amount) AS total
FROM orders
GROUP BY month;

-- 重新整理
REFRESH MATERIALIZED VIEW monthly_sales;

-- 排程每天凌晨 3 點 refresh
-- cron: 0 3 * * * psql -c "REFRESH MATERIALIZED VIEW monthly_sales;"
```

## View 跟 Stored Procedure 差別
- View：是查詢、SELECT 就好
- Procedure / Function：是 code、可有邏輯 / loop / 條件
