**subquery（子查詢）= SELECT 裡面又包一個 SELECT**—— SQL 的「函式裡呼叫函式」。學會它你的 SQL 表達力翻倍、但用錯效能會死。本課把三種 subquery 跟 EXISTS / IN 一次講清楚。

> 💡 **這課的價值**：subquery 是 SQL 進階分水嶺——很多人會 SELECT、會 JOIN、但 subquery 寫起來綁手綁腳。本課用「身分」分類三種 subquery、加上 EXISTS vs IN 的選擇指南、寫起來不再亂。


### 1. subquery 是什麼？

**核心**：把一個 SELECT 結果當「值」或「表」、放進另一個 SELECT 用。

**例：**

```text
-- 找出年齡比平均高的 user
SELECT name, age
FROM users
WHERE age > (SELECT AVG(age) FROM users);
--          ↑ 這個 subquery 算出平均年齡（一個數）
--          外層 WHERE 跟那個數比較
```

**子查詢能放 4 個地方：**

```text
1. WHERE 後面     最常見、過濾用
2. SELECT 後面    當計算欄位
3. FROM 後面      當「臨時 table」
4. HAVING 後面    分組後過濾
```


### 2. 三種 subquery 型態（看回傳什麼）

```text
1️⃣ Scalar subquery   回傳 1 列 1 欄（一個值）
   例：(SELECT AVG(age) FROM users)
   用：跟數字比、=、<、>、in SELECT 當欄位

2️⃣ Row subquery      回傳 1 列多欄
   例：(SELECT city, age FROM users WHERE id=1)
   用：(city, age) = (SELECT city, age FROM ...)

3️⃣ Table subquery    回傳多列多欄
   例：(SELECT * FROM orders WHERE amount > 1000)
   用：IN (...) / EXISTS (...) / FROM ... AS sub
```


### 3. Scalar subquery（最常見）

**WHERE 比較：**

```text
SELECT name FROM users
WHERE age > (SELECT AVG(age) FROM users);

SELECT * FROM products
WHERE price = (SELECT MAX(price) FROM products);
```

**SELECT 當欄位：**

```text
SELECT
  u.name,
  (SELECT COUNT(*) FROM orders WHERE user_id = u.id) AS order_count,
  (SELECT SUM(amount) FROM orders WHERE user_id = u.id) AS total_spent
FROM users u;
```

> 💡 **效能注意**：SELECT 裡的 subquery 對「每列跑一次」——大表慢、能改 JOIN 就改 JOIN。


### 4. Row subquery（不常用、但簡潔）

```text
-- 找跟「id=1」這個 user 同城市、同年齡的人
SELECT *
FROM users
WHERE (city, age) = (SELECT city, age FROM users WHERE id = 1);
```

**等同：**

```text
SELECT *
FROM users
WHERE city = (SELECT city FROM users WHERE id = 1)
  AND age = (SELECT age FROM users WHERE id = 1);
-- 寫兩次 subquery、不簡潔
```


### 5. Table subquery（FROM 後面當臨時表）

```text
-- 「每個 user 訂單總額」當作一個臨時表、再過濾出消費 > 5000 的
SELECT *
FROM (
  SELECT user_id, SUM(amount) AS total
  FROM orders
  GROUP BY user_id
) AS user_totals
WHERE total > 5000;
```

**規則**：FROM 後面的 subquery **必須加別名**（PG 嚴格、MySQL 不嚴格）。


### 6. IN 跟 EXISTS（最常用 + 最常選錯）

**IN 寫法：**

```text
SELECT name FROM users
WHERE id IN (SELECT user_id FROM orders WHERE amount > 1000);
```

**EXISTS 寫法（同樣意思）：**

```text
SELECT name FROM users u
WHERE EXISTS (
  SELECT 1 FROM orders o
  WHERE o.user_id = u.id
    AND o.amount > 1000
);
```

**邏輯一樣**——找出「有下大單的 user」。


### 7. IN vs EXISTS 哪個快？

**簡單規則：**

```text
IN
   把 subquery 結果列出來 → 跟外層比較
   → subquery 結果小、快

EXISTS
   對外層每列、check「右邊找得到不」、找到就停
   → subquery 結果大、快

PG 12+
   優化器幾乎自動轉換、兩者差不多

注意：NOT IN 撞 NULL 一定空集合（L6）
      NOT EXISTS 安全處理 NULL（推薦）
```

**業界共識**：**EXISTS 比較安全**（特別 NOT 的時候）、用習慣它。


### 8. ANY / ALL（subquery 配比較運算）

```text
-- 任何一個 order amount 都比這 user 大（= MAX）
WHERE age > ALL (SELECT age FROM users WHERE city = '台中')
-- 等同 age > (SELECT MAX(age) FROM ...)

-- 至少一個 order amount 比這個大（= MIN）
WHERE age > ANY (SELECT age FROM users WHERE city = '台中')
-- 等同 age > (SELECT MIN(age) FROM ...)

-- ANY = SOME（同義）
```

**業界少用**——通常改寫成 MAX / MIN 更清楚。


### 9. Correlated subquery（相關子查詢）

「Correlated」= subquery 內用到了外層的欄位、所以「對每列跑一次」。

```text
SELECT u.name,
  (SELECT COUNT(*) FROM orders o WHERE o.user_id = u.id) AS orders
FROM users u;
-- 內層 o.user_id = u.id 引用了外層 u
```

**效能注意**：
- 外層 1000 列 → subquery 跑 1000 次
- 通常用 LEFT JOIN + GROUP BY 改寫更快

```text
-- 改寫版本
SELECT u.name, COUNT(o.id) AS orders
FROM users u
LEFT JOIN orders o ON o.user_id = u.id
GROUP BY u.id, u.name;
```


### 10. NOT EXISTS 反向 pattern

**找出「沒下單過的 user」：**

```text
SELECT name FROM users u
WHERE NOT EXISTS (
  SELECT 1 FROM orders WHERE user_id = u.id
);
```

**為什麼這比 NOT IN 安全？**

```text
-- NOT IN 撞 NULL：
SELECT name FROM users
WHERE id NOT IN (SELECT user_id FROM orders);
-- 如果 orders.user_id 任一筆是 NULL → 全空集合 ❌

-- NOT EXISTS 沒這問題：
WHERE NOT EXISTS (SELECT 1 FROM orders WHERE user_id = u.id);
-- 正確找出沒對應的 user ✅
```

**永遠用 NOT EXISTS、不要用 NOT IN**（除非 100% 確認沒 NULL）。


### 11. WHERE EXISTS (SELECT 1 vs SELECT *)

```text
WHERE EXISTS (SELECT 1 FROM ...)    -- 寫 1
WHERE EXISTS (SELECT * FROM ...)    -- 寫 *
WHERE EXISTS (SELECT id FROM ...)   -- 寫 id
```

**3 種寫法效能一樣**——EXISTS 只關心「有沒有列」、不看 SELECT 出什麼。

**業界慣例**：寫 `SELECT 1`——表明「我不關心欄位內容」、純做存在判斷。


### 12. 複雜 subquery 拆解（CTE 救星）

寫到 3 層巢狀 subquery、可讀性炸——改成 CTE（L10）：

```text
-- subquery 3 層（難讀）
SELECT *
FROM (
  SELECT *,
    (SELECT name FROM users WHERE id = sub.user_id) AS user_name
  FROM (
    SELECT user_id, SUM(amount) AS total
    FROM orders
    GROUP BY user_id
  ) sub
  WHERE total > 5000
) final;

-- CTE 改寫（清楚多了）
WITH user_totals AS (
  SELECT user_id, SUM(amount) AS total
  FROM orders
  GROUP BY user_id
)
SELECT
  ut.user_id,
  u.name AS user_name,
  ut.total
FROM user_totals ut
JOIN users u ON u.id = ut.user_id
WHERE ut.total > 5000;
```


### 13. 完整實戰：找出「會員等級不一致」的 user

**業務邏輯**：tier 應該 = 消費總額對應的等級、找出 DB 裡 tier 跟實際消費不符的。

```text
SELECT
  u.id,
  u.name,
  u.tier AS current_tier,
  CASE
    WHEN (SELECT COALESCE(SUM(amount), 0) FROM orders WHERE user_id = u.id) >= 100000 THEN 'platinum'
    WHEN (SELECT COALESCE(SUM(amount), 0) FROM orders WHERE user_id = u.id) >= 50000  THEN 'gold'
    WHEN (SELECT COALESCE(SUM(amount), 0) FROM orders WHERE user_id = u.id) >= 10000  THEN 'silver'
    ELSE 'bronze'
  END AS expected_tier
FROM users u
WHERE u.tier <> (
  CASE
    WHEN (SELECT COALESCE(SUM(amount), 0) FROM orders WHERE user_id = u.id) >= 100000 THEN 'platinum'
    WHEN (SELECT COALESCE(SUM(amount), 0) FROM orders WHERE user_id = u.id) >= 50000  THEN 'gold'
    WHEN (SELECT COALESCE(SUM(amount), 0) FROM orders WHERE user_id = u.id) >= 10000  THEN 'silver'
    ELSE 'bronze'
  END
);
```

**寫到這種長度——就該改 CTE 了**。但範例是想讓你看到「subquery 多到爆」的樣子。


### 14. 5 大常見坑

```text
❌ 1. SELECT 裡的 correlated subquery 大表用
   → 外層 100 萬列 = 跑 100 萬次 subquery
   → 修：改成 LEFT JOIN + GROUP BY

❌ 2. NOT IN + NULL
   → 永遠空集合
   → 修：用 NOT EXISTS（最安全）

❌ 3. FROM subquery 沒給別名
   → PG 嚴格、直接報錯
   → 修：永遠 FROM (...) AS sub_name

❌ 4. Scalar subquery 回傳超過 1 列
   → 報錯「more than one row returned」
   → 修：subquery 加 LIMIT 1 或保證唯一

❌ 5. subquery 巢狀 4 層以上
   → 沒人讀得懂、debug 地獄
   → 修：改 CTE（L10）拆步驟
```


### 🎯 重點回顧

- **3 種 subquery**：Scalar（單值）、Row（單列多欄）、Table（多列多欄）
- **能放 4 個地方**：WHERE / SELECT / FROM / HAVING
- **IN vs EXISTS**：EXISTS 安全、NOT 場景永遠用 NOT EXISTS
- **Correlated subquery 對每列跑一次**——大表用 JOIN 改寫
- **巢狀 3 層以上、改 CTE**
