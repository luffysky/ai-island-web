CREATE TABLE todos (

### 4. Row Level Security（RLS）—— Supabase 的核心安全

**問題**：前端用 anon key 直接打 DB——任何人都能讀全部 todos？

**解法**：RLS——「每個 user 只能看自己的列」、由 DB 強制。

```text
-- 1. 啟用 RLS
ALTER TABLE todos ENABLE ROW LEVEL SECURITY;

-- 2. 建 policy：只允許 user 看自己的 todos
CREATE POLICY "own_todos_select" ON todos
  FOR SELECT
  USING (auth.uid() = user_id);

-- 3. 只允許 user 寫自己的 todos
CREATE POLICY "own_todos_insert" ON todos
  FOR INSERT
  WITH CHECK (auth.uid() = user_id);

-- 4. 只允許 user 改自己的
CREATE POLICY "own_todos_update" ON todos
  FOR UPDATE
  USING (auth.uid() = user_id);

-- 5. 只允許 user 刪自己的
CREATE POLICY "own_todos_delete" ON todos
  FOR DELETE
  USING (auth.uid() = user_id);
```

**或一句搞定 4 種：**

```text
CREATE POLICY "own_todos" ON todos
  FOR ALL
  USING (auth.uid() = user_id);
```

**`auth.uid()`** = 目前 session 的 user id（Supabase 從 JWT 拿）。

### 7. Realtime —— 訂閱 DB 變化

```text
supabase
  .channel('todos-changes')
  .on('postgres_changes',
    { event: '*', schema: 'public', table: 'todos' },
    (payload) => {
      console.log('變化！', payload);
      // payload.eventType: 'INSERT' / 'UPDATE' / 'DELETE'
      // payload.new: 新資料
      // payload.old: 舊資料
    }
  )
  .subscribe();
```

**應用**：聊天室、多人協作、即時通知——資料一變、所有 client 立刻知道。**用 WebSocket、不用輪詢**。

### 10. pgvector —— AI 應用必備（向量搜尋）

```text
-- 啟用 extension
CREATE EXTENSION IF NOT EXISTS vector;

-- 加向量欄位（OpenAI embedding 1536 維）
CREATE TABLE documents (
  id        SERIAL PRIMARY KEY,
  content   TEXT,
  embedding vector(1536)
);

-- 建 ivfflat index 加速
CREATE INDEX ON documents USING ivfflat (embedding vector_cosine_ops);

-- 搜尋相似文章（cosine distance）
SELECT content, 1 - (embedding <=> $1) AS similarity
FROM documents
ORDER BY embedding <=> $1
LIMIT 5;
```

**這就是 RAG（Retrieval-Augmented Generation）的核心**——AI 應用全靠這個。Supabase + pgvector = ChatGPT 風格 RAG 一條龍。


### 11. Database Functions（用 SQL function 當 API）

```text
-- 建一個 SQL function
CREATE OR REPLACE FUNCTION get_user_stats(p_user_id UUID)
RETURNS TABLE(total_todos BIGINT, completed BIGINT, pending BIGINT) AS $$
  SELECT
    COUNT(*) AS total_todos,
    COUNT(*) FILTER (WHERE completed) AS completed,
    COUNT(*) FILTER (WHERE NOT completed) AS pending
  FROM todos
  WHERE user_id = p_user_id;
$$ LANGUAGE sql;

-- 前端用 RPC 呼叫
const { data } = await supabase.rpc('get_user_stats', { p_user_id: userId });
```

**把複雜業務邏輯包成 function、前端一行呼叫**——少寫 API。

### 13. 完整實戰：30 行 code 做 Todo App

📄 **寫進程式檔**：`pages/index.tsx`

```text
'use client';
import { useEffect, useState } from 'react';
import { supabase } from '@/lib/supabase';

export default function TodoApp() {
  const [todos, setTodos] = useState<any[]>([]);
  const [title, setTitle] = useState('');

  useEffect(() => {
    const fetch = async () => {
      const { data } = await supabase
        .from('todos')
        .select('*')
        .order('created_at', { ascending: false });
      setTodos(data || []);
    };
    fetch();

    // Realtime 訂閱
    const channel = supabase.channel('todos')
      .on('postgres_changes',
        { event: '*', schema: 'public', table: 'todos' },
        fetch
      )
      .subscribe();
    return () => { supabase.removeChannel(channel); };
  }, []);

  async function addTodo() {
    const { data: { user } } = await supabase.auth.getUser();
    await supabase.from('todos').insert({ title, user_id: user!.id });
    setTitle('');
  }

  return (
    <div>
      <input value={title} onChange={e => setTitle(e.target.value)} />
      <button onClick={addTodo}>+</button>
      <ul>{todos.map(t => <li key={t.id}>{t.title}</li>)}</ul>
    </div>
  );
}
```

**30 行做完：CRUD + Realtime + Auth + RLS 保護**——後端 0 行 code。


### 14. 5 大常見坑

```text
❌ 1. service_role key 放前端
   → 任何人能繞 RLS、直接清庫
   → 修：service_role 只放後端、前端永遠用 anon

❌ 2. 沒開 RLS 就上線
   → 整張表變公開、誰都能讀寫
   → 修：每張表都 ENABLE RLS + 寫 policy

❌ 3. RLS policy 寫錯邏輯
   → 太緊（user 看不到自己的）或太鬆（看到別人的）
   → 修：用 Dashboard 的 RLS tester 驗

❌ 4. Free tier project 開太多被停權
   → 一個帳號最多 2 個 free project
   → 修：刪舊 project 才開新的

❌ 5. 沒 backup 就 schema migration
   → 改錯 → 線上資料炸
   → 修：用 supabase migration 工具、Production 一律先 dump
```
