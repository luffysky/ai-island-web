**INSERT（寫入）= SQL 寫資料三兄弟的老大**—— 從「寫 1 筆」到「批次寫 1000 筆」、再到「UPSERT（有就更新、沒有就新增）」、本課全收。

> 💡 **這課的價值**：所有「使用者註冊」、「下訂單」、「按讚」、「留言」——背後全是 INSERT。寫對 INSERT、UPSERT、RETURNING、你的後端 API 強壯一半。


### 1. 最基本：寫一筆

```text
INSERT INTO 表名 (欄位 1, 欄位 2, ...) VALUES (值 1, 值 2, ...);
```

**範例：**

```text
INSERT INTO users (name, email, age)
VALUES ('Luffy', 'luffy@ai.com', 30);
```

💬 **電腦會顯示**：

```text
INSERT 0 1
-- INSERT 0 = OID（舊版用、現在永遠 0）
-- 1 = 影響 1 列
```

**省略欄位名（不推薦）：**

```text
INSERT INTO users VALUES (1, 'Luffy', 'luffy@ai.com', 30, '台北', NOW());
-- 必須照 table 定義順序、全部欄位都要給
-- table 加新欄位後這句就壞掉、所以不推薦
```

> 💡 **永遠寫欄位名**——清楚、不怕順序錯、加新欄位不影響舊 query。


### 2. 一次寫多筆（批次 INSERT）

**慢做法（一次一筆）：**

```text
INSERT INTO users (name, email) VALUES ('A', 'a@ai.com');
INSERT INTO users (name, email) VALUES ('B', 'b@ai.com');
INSERT INTO users (name, email) VALUES ('C', 'c@ai.com');
-- 3 次來回、3 次 disk write
```

**快做法（一次多筆）：**

```text
INSERT INTO users (name, email) VALUES
  ('A', 'a@ai.com'),
  ('B', 'b@ai.com'),
  ('C', 'c@ai.com');
-- 1 次來回、1 次 disk write
-- 大量寫入快 10 倍以上
```

**實務技巧**：匯入 1 萬筆資料、用 batch insert 比 for-loop 快非常多。


### 3. INSERT ... SELECT —— 從別張表複製

```text
-- 把符合條件的 user 複製到 vip_users 表
INSERT INTO vip_users (id, name, email)
SELECT id, name, email
FROM users
WHERE total_spent > 100000;
```

**也能加計算、轉換：**

```text
INSERT INTO daily_stats (date, user_count, total_revenue)
SELECT
  CURRENT_DATE,
  COUNT(DISTINCT user_id),
  SUM(amount)
FROM orders
WHERE created_at::DATE = CURRENT_DATE;
```

**INSERT ... SELECT 應用場景：**

```text
✅ 報表生成（每日彙總）
✅ 資料搬移（搬到 archive 表）
✅ 批次 backfill
```


### 4. RETURNING —— INSERT 後立刻回傳結果

PG / Oracle 專屬、超強功能：

```text
INSERT INTO users (name, email)
VALUES ('Luffy', 'luffy@ai.com')
RETURNING id, created_at;
```

💬 **電腦會顯示**：

```text
 id |        created_at
----+----------------------------
  7 | 2026-05-28 14:30:00.123+00
```

**為什麼超有用？**——以前要寫 2 個 query（INSERT + 再 SELECT 拿 id）、現在一句搞定。

**RETURNING * 拿全部欄位：**

```text
INSERT INTO posts (title, content)
VALUES ('Hello', '...')
RETURNING *;
```

**多筆 INSERT + RETURNING：**

```text
INSERT INTO products (name, price) VALUES
  ('A', 100), ('B', 200), ('C', 300)
RETURNING id, name;
-- 一次拿到 3 個 id
```


### 5. UPSERT —— 「有就更新、沒有就新增」

**經典需求**：插入 user、如果 email 已存在、就更新 name。

**手動寫超痛**：

```text
-- 1. 先 SELECT 看在不在
-- 2. 在 → UPDATE
-- 3. 不在 → INSERT
-- 還有 race condition 問題（兩個請求同時來）
```

**ON CONFLICT 一句搞定：**

```text
INSERT INTO users (email, name)
VALUES ('luffy@ai.com', 'Luffy')
ON CONFLICT (email)
DO UPDATE SET name = EXCLUDED.name;
```

**逐句翻譯：**

```text
INSERT ...                INSERT 一筆資料
ON CONFLICT (email)       如果 email 撞到 UNIQUE 約束
DO UPDATE SET ...         就改 SET 後面這些欄位
  EXCLUDED.name           = 「本來要 INSERT 的值」
```

**EXCLUDED 是關鍵字**——指「這次 INSERT 想塞的那筆」、不是 table 原本的。


### 6. ON CONFLICT 變化

**ON CONFLICT DO NOTHING（撞到就跳過、不報錯）：**

```text
INSERT INTO users (email, name)
VALUES ('luffy@ai.com', 'Luffy')
ON CONFLICT (email) DO NOTHING;
```

**ON CONFLICT 多欄位 UNIQUE：**

```text
INSERT INTO follows (follower_id, followee_id)
VALUES (1, 2)
ON CONFLICT (follower_id, followee_id) DO NOTHING;
```

**ON CONFLICT 配 partial Index（條件唯一）：**

```text
INSERT INTO users (email, deleted_at)
VALUES ('luffy@ai.com', NULL)
ON CONFLICT (email) WHERE deleted_at IS NULL
DO UPDATE SET name = EXCLUDED.name;
```


### 7. UPSERT 結合 RETURNING（神組合）

```text
INSERT INTO users (email, name, login_count)
VALUES ('luffy@ai.com', 'Luffy', 1)
ON CONFLICT (email)
DO UPDATE SET login_count = users.login_count + 1
RETURNING id, login_count;
```

**做了 3 件事：**

1. 第一次登入 → INSERT、login_count = 1
2. 之後每次登入 → UPDATE、login_count += 1
3. 不管哪種、都回傳 id 跟最新 login_count

**這就是 Indie 開發必殺技**——一句 SQL 處理「首次 + 重複」兩種情境。


### 8. 寫日期 / JSON / Array

**日期：**

```text
INSERT INTO events (title, starts_at) VALUES
  ('Meeting', '2026-05-28 14:30:00'),       -- 字串自動轉
  ('Lunch',   NOW()),                        -- 當下時間
  ('Tea',     NOW() + INTERVAL '1 hour');    -- 1 小時後
```

**JSON / JSONB：**

```text
INSERT INTO products (name, meta) VALUES
  ('iPhone', '{"color": "black", "storage": "256GB"}'::JSONB);
```

**Array：**

```text
INSERT INTO posts (title, tags) VALUES
  ('Hello', ARRAY['sql', 'tutorial', '2026']);
-- 或
INSERT INTO posts (title, tags) VALUES
  ('Hello', '{sql, tutorial, 2026}');
```


### 9. 完整實戰：Twitter 風格按讚

📄 **寫進程式檔**：`like-post.sql`

```text
-- 一句處理：按讚（如果沒按過就 INSERT、按過就 DELETE 取消讚）
INSERT INTO likes (user_id, post_id)
VALUES ($1, $2)
ON CONFLICT (user_id, post_id) DO NOTHING
RETURNING id;
-- 回傳 id = 新增成功（按讚）
-- 回傳空 = 已經按過（取消讚則跑 DELETE）
```

📄 **寫進程式檔**：`signup.ts`

```text
// 註冊 + 自動發歡迎信
const result = await db.query(`
  INSERT INTO users (email, name, password_hash)
  VALUES ($1, $2, $3)
  ON CONFLICT (email) DO NOTHING
  RETURNING id, email, name
`, [email, name, hash]);

if (result.rows.length === 0) {
  throw new Error('Email 已被註冊');
}

const user = result.rows[0];
await sendWelcomeEmail(user.email, user.name);
```


### 10. INSERT 效能優化（大量資料）

**1000 筆以下 → batch INSERT：**

```text
INSERT INTO logs (msg) VALUES ('a'), ('b'), ..., ('z');
```

**1 萬筆以上 → COPY 指令（最快）：**

🖥️ **在 psql 輸入**：

```text
COPY logs (msg) FROM '/path/to/data.csv' WITH (FORMAT CSV);
```

**COPY 比 INSERT 快 50-100 倍**——大量 import / migration 必用。


### 11. 5 大常見坑

```text
❌ 1. INSERT 沒寫欄位名
   → table 加欄位後整段炸
   → 修：永遠 INSERT INTO 表 (欄位, ...)

❌ 2. for-loop 一筆筆 INSERT
   → 1000 筆寫 30 秒
   → 修：batch INSERT 一次寫進去

❌ 3. UPSERT 用 SELECT 先看再 INSERT
   → race condition、高併發會撞
   → 修：用 ON CONFLICT 原子操作

❌ 4. ON CONFLICT 沒指定欄位
   → 不知道哪個 UNIQUE 撞、PG 報錯
   → 修：ON CONFLICT (email) 明確指定

❌ 5. RETURNING 忘了用、又跑一次 SELECT
   → 多一次 round-trip
   → 修：INSERT ... RETURNING id, ... 一句拿完
```


### 🎯 重點回顧

- **INSERT INTO 表 (欄位...) VALUES (...)** 是基本句型
- **多筆一句**比一筆一句快 10 倍以上
- **INSERT ... SELECT** 從別表複製、報表彙總超實用
- **RETURNING** 立刻拿回新增的 id / 任何欄位、省一次 SELECT
- **ON CONFLICT DO UPDATE** = UPSERT、處理「有就更新沒就新增」
- **ON CONFLICT DO NOTHING** 撞到就跳過、不報錯
- **COPY** 大量匯入專用、比 INSERT 快 50-100 倍
