**WHERE（條件子句）= SQL 的「if」**—— 從 table 撈出符合條件的列。新手第一個踩坑的地方多半在 WHERE——尤其是 **NULL 的怪事**、本課一次說清楚。

> 💡 **這課的價值**：寫 SQL 80% 的時間都在跟 WHERE 搏鬥。學會它的 6 種運算 + NULL 規則、你已經能寫 70% 業務 query。


### 1. WHERE 基本語法

```text
SELECT 欄位 FROM 表 WHERE 條件;
```

**對照 L5 練習表（6 筆 users）：**

```text
SELECT name, age FROM users WHERE age >= 30;
```

💬 **電腦會顯示**：

```text
  name  | age
--------+-----
 Luffy  |  30
 Zoro   |  32
 Sanji  |  30
```

只留 age ≥ 30 的列。


### 2. 6 種比較運算子

```text
=        等於
<>  !=   不等於（兩種都行）
<        小於
<=       小於等於
>        大於
>=       大於等於
```

**範例：**

```text
SELECT * FROM users WHERE age = 28;     -- 28 歲
SELECT * FROM users WHERE age <> 30;    -- 不是 30 歲
SELECT * FROM users WHERE city = '台北'; -- 台北人
```

> 💡 **SQL 用 `=` 不是 `==`**——很多人從 JS/Python 過來會打成 `==` 直接報錯。SQL 的 `=` 同時是「比較」跟「賦值」（context 決定）。


### 3. AND / OR / NOT

```text
-- 同時兩個條件
SELECT * FROM users WHERE age >= 30 AND city = '台北';

-- 任一條件
SELECT * FROM users WHERE city = '台北' OR city = '台中';

-- 否定
SELECT * FROM users WHERE NOT city = '台北';
-- 等同：WHERE city <> '台北'
```

**注意 AND 比 OR 優先（跟數學一樣）：**

```text
WHERE age >= 30 AND city = '台北' OR city = '台中'
=
WHERE (age >= 30 AND city = '台北') OR city = '台中'
```

**想要先 OR、加括號：**

```text
WHERE age >= 30 AND (city = '台北' OR city = '台中')
```

> 💡 **永遠加括號**——別賴記憶優先順序、加括號是讓未來自己活下去的良心。


### 4. IN —— 「屬於這串清單」

```text
-- 又臭又長：
WHERE city = '台北' OR city = '台中' OR city = '高雄'

-- 改成 IN（推薦）：
WHERE city IN ('台北', '台中', '高雄')

-- 反過來：
WHERE city NOT IN ('東京', '大阪')
```

**IN 也能接 subquery：**

```text
SELECT * FROM posts
WHERE author_id IN (
  SELECT id FROM users WHERE city = '台北'
);
-- 找出「台北人寫的所有文章」
```


### 5. BETWEEN —— 「在這個範圍」

```text
-- 麻煩寫法：
WHERE age >= 18 AND age <= 30

-- BETWEEN（含頭含尾）：
WHERE age BETWEEN 18 AND 30
```

**日期也能用：**

```text
SELECT * FROM orders
WHERE created_at BETWEEN '2026-05-01' AND '2026-05-31';
```

> 💡 **BETWEEN 兩端都包含**（含 18 含 30）、不是「中間」。要排除端點要寫 `> AND <`。


### 6. LIKE —— 模糊比對（文字搜尋）

**兩個萬用字元：**

```text
%   任意長度的任意字元（含 0 個）
_   剛好 1 個任意字元
```

**範例：**

```text
WHERE name LIKE 'L%'        -- L 開頭：Luffy
WHERE name LIKE '%i'        -- i 結尾：Nami / Robin（沒有）
WHERE name LIKE '%a%'       -- 含 a：Nami / Sanji
WHERE name LIKE 'L___y'     -- L 開頭 3 字結尾 y 共 5 字：Luffy
WHERE email LIKE '%@ai.com' -- ai.com 結尾
```

**ILIKE（不分大小寫、PG 專屬）：**

```text
WHERE name LIKE 'l%'    -- 找 'l' 開頭、大寫 Luffy 找不到
WHERE name ILIKE 'l%'   -- 不分大小寫、Luffy / luffy 都找到
```

**搜尋字串內含 `%` 或 `_` —— 用 ESCAPE：**

```text
WHERE description LIKE '50\%off' ESCAPE '\'
-- 找「50%off」字串
```


### 7. NULL 的怪事（最大坑）

**NULL ≠ 空字串 ≠ 0**——NULL 是「沒有值 / 未知」。

```text
INSERT INTO users (name, email, age) VALUES ('Test', 'test@ai.com', NULL);

SELECT * FROM users WHERE age = NULL;   -- ❌ 永遠空集合
SELECT * FROM users WHERE age <> NULL;  -- ❌ 永遠空集合
```

**為什麼？因為 NULL 在 SQL 是「未知」、任何運算「未知 = 任何東西」都是「未知」、未知 ≠ TRUE、所以條件不成立。**

**正確寫法：**

```text
SELECT * FROM users WHERE age IS NULL;     -- 找出沒填 age 的
SELECT * FROM users WHERE age IS NOT NULL; -- 找出有填 age 的
```

**NULL 的更多陷阱：**

```text
NULL = NULL          → NULL（不是 TRUE！）
NULL + 1             → NULL
NULL || 'abc'        → NULL（字串拼空）
NULL AND TRUE        → NULL
NULL OR TRUE         → TRUE  ← 注意
NULL AND FALSE       → FALSE ← 注意
```

> 💡 **NULL 處理咒語**：永遠用 `IS NULL` / `IS NOT NULL`、不要用 `=` / `<>`。

**COALESCE —— 把 NULL 替換成預設值：**

```text
SELECT
  name,
  COALESCE(age, 0) AS age_or_zero,
  COALESCE(city, '未知') AS city_or_unknown
FROM users;
```


### 8. 完整範例：寫一個複雜過濾

🖥️ **題目**：找出「台北或台中、年齡 25-35、有填 email、名字含 a」的人。

```text
SELECT name, age, city, email
FROM users
WHERE city IN ('台北', '台中')
  AND age BETWEEN 25 AND 35
  AND email IS NOT NULL
  AND name ILIKE '%a%';
```

**好習慣**：
- 每個 AND 換行（好讀）
- 連續多個 AND/OR、加括號清楚

💬 **電腦會顯示**：

```text
  name  | age | city  |     email
--------+-----+-------+---------------
 Nami   |  26 | 台中  | nami@ai.com
 Sanji  |  30 | 台北  | sanji@ai.com
```


### 9. WHERE vs HAVING（先說一下、L8 詳講）

```text
WHERE   過濾「列」、在 GROUP BY 之前跑
HAVING  過濾「分組結果」、在 GROUP BY 之後跑
```

**簡單記**：原始列篩選用 WHERE、聚合後篩選用 HAVING。`WHERE SUM(amount) > 100` 寫不出來——因為 WHERE 階段還沒分組。


### 10. 三值邏輯（Three-Valued Logic）

SQL 的布林不是 TRUE / FALSE 兩種、是 **TRUE / FALSE / NULL** 三種、這叫 **3VL**。

```text
AND 真值表：
TRUE  AND TRUE   = TRUE
TRUE  AND FALSE  = FALSE
TRUE  AND NULL   = NULL
FALSE AND NULL   = FALSE  ← 因為已經 FALSE、不管另一邊
NULL  AND NULL   = NULL

OR 真值表：
TRUE  OR FALSE   = TRUE
TRUE  OR NULL    = TRUE   ← 因為已經 TRUE
FALSE OR NULL    = NULL
NULL  OR NULL    = NULL
```

> 💡 **3VL 是 SQL 跟其他語言最大差異**——花 5 分鐘背下 AND/OR 真值表、未來 debug 省 5 小時。


### 11. 5 大常見坑

```text
❌ 1. WHERE age = NULL
   → 永遠空集合
   → 修：WHERE age IS NULL

❌ 2. WHERE name = "Luffy"（用雙引號）
   → 雙引號 = 識別字、PG 會去找叫「Luffy」的欄位
   → 修：字串永遠單引號 'Luffy'

❌ 3. AND / OR 沒加括號
   → 業務邏輯整個錯
   → 修：永遠括號、不要賴記憶

❌ 4. LIKE 'a' 大小寫沒注意
   → 找 a 找不到 A
   → 修：用 ILIKE 不分大小寫、或先 LOWER(col) LIKE LOWER('a%')

❌ 5. NOT IN 撞 NULL 一定空
   → WHERE id NOT IN (1, 2, NULL) → 永遠 0 列
   → 因為 NOT IN 內部變 id <> NULL → 永遠 NULL
   → 修：WHERE id NOT IN (1, 2) AND id IS NOT NULL
```


### 🎯 重點回顧

- **6 種比較**：`=`、`<>`/`!=`、`<`、`<=`、`>`、`>=`、不要寫 `==`
- **AND / OR / NOT**——AND 優先於 OR、永遠加括號
- **IN / BETWEEN / LIKE** 三個高頻簡寫
- **LIKE 的 `%`（任意長）跟 `_`（1 字）**、配 `ILIKE` 不分大小寫（PG）
- **NULL 用 IS NULL / IS NOT NULL**、永遠不要 `= NULL`
- **三值邏輯（TRUE / FALSE / NULL）**是 SQL 最大坑、`NOT IN` 撞 NULL 必空
