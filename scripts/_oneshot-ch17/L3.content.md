**table（資料表）= SQL 的基本單位**—— 像 Excel 表格、每一列（row）是一筆資料、每一欄（column）是一個欄位。SQL 80% 的功夫都花在「設計 table」。

> 💡 **這課的價值**：table 設計爛、後面 query 怎麼寫都救不回來。本課把資料型別、NOT NULL、DEFAULT 三個地基打牢、你的 DB 從一開始就強壯。


### 1. table 是什麼？用 Excel 來理解

```text
                  table: users
┌────┬────────┬─────────────────┬─────┐
│ id │ name   │ email           │ age │  ← column（欄位）
├────┼────────┼─────────────────┼─────┤
│ 1  │ Luffy  │ luffy@ai.com    │ 30  │  ← row（一筆資料）
│ 2  │ Xueyue │ xueyue@ai.com   │ 28  │  ← row
│ 3  │ Nami   │ nami@ai.com     │ 26  │  ← row
└────┴────────┴─────────────────┴─────┘

📐 table     整張表（= 一個 Excel sheet）
📋 row       一筆資料（一列、英文 row 或 record）
📊 column    一個欄位（一欄、英文 column 或 field）
🆔 id        主鍵欄位、每筆資料的身分證
```

**比喻**：

- table = Excel 的 sheet（工作表）
- row = Excel 的「一列」
- column = Excel 的「一欄」
- 但 SQL 強制「同一欄型別要一致」（不能第 1 列數字、第 2 列文字）


### 2. CREATE TABLE 基本語法

```text
CREATE TABLE 表名 (
  欄位名 1   型別 1   約束 1,
  欄位名 2   型別 2   約束 2,
  ...
);
```

**最小範例：**

📄 **寫進程式檔**：`001_create_users.sql`

```text
CREATE TABLE users (
  id    SERIAL PRIMARY KEY,
  name  TEXT NOT NULL,
  email TEXT UNIQUE NOT NULL,
  age   INT
);
```

**逐行翻譯：**

```text
CREATE TABLE users          建一張叫 users 的表
  id SERIAL PRIMARY KEY     id 欄位：自動編號的整數、設為主鍵
  name TEXT NOT NULL        name 欄位：文字、不可空白
  email TEXT UNIQUE NOT NULL  email 欄位：文字、不可重複、不可空白
  age INT                   age 欄位：整數、可空白
```


### 3. PostgreSQL 12 大常用資料型別

```text
🔢 整數
   SMALLINT     -32,768 ~ 32,767（2 bytes）
   INT          ±21 億（4 bytes、最常用）
   BIGINT       ±900 京（8 bytes、超大數字用）
   SERIAL       INT + 自動 +1（適合 id）
   BIGSERIAL    BIGINT + 自動 +1

🔢 小數
   NUMERIC(p,s) 精確小數（金額必用、不會有浮點誤差）
                p = 總位數、s = 小數位
                例 NUMERIC(10,2) = 最多 10 位、其中 2 位小數
   REAL         單精度浮點（4 bytes、會有誤差）
   DOUBLE PRECISION  雙精度浮點（8 bytes、會有誤差）

📝 文字
   TEXT         任意長度文字（PG 推薦、無效能差別）
   VARCHAR(n)   最多 n 個字、超過報錯
   CHAR(n)      固定 n 個字、不足補空白（幾乎不用）

📅 日期時間
   DATE         只有日期（2026-05-28）
   TIME         只有時間（14:30:00）
   TIMESTAMP    日期 + 時間（無時區）
   TIMESTAMPTZ  日期 + 時間 + 時區（推薦）

✅ 真假
   BOOLEAN      TRUE / FALSE / NULL

🔑 識別字
   UUID         全球唯一 ID（推薦取代 SERIAL）

📦 結構化
   JSON         JSON 文字（不檢查格式）
   JSONB        JSON 二進位（推薦、可查詢、L24 整課）
   ARRAY        陣列（例 TEXT[]）
```

> 💡 **新手只要記 6 個**：`INT`、`TEXT`、`BOOLEAN`、`TIMESTAMPTZ`、`NUMERIC(10,2)`（金額）、`UUID`。其他用到再查。


### 4. 為什麼選對型別這麼重要？

**錯誤示範——全部用 TEXT：**

```text
CREATE TABLE bad_orders (
  id        TEXT,
  amount    TEXT,   -- 金額用文字
  ordered_at TEXT   -- 時間用文字
);

INSERT INTO bad_orders VALUES ('1', '99.5', '2026-05-28');
INSERT INTO bad_orders VALUES ('2', '1000', '2026-05-27');

-- 排序失敗（文字排序：'1000' < '99.5'）
SELECT * FROM bad_orders ORDER BY amount DESC;
-- 算總和失敗（文字不能加）
SELECT SUM(amount) FROM bad_orders;  -- 報錯
```

**正確示範：**

```text
CREATE TABLE good_orders (
  id          SERIAL PRIMARY KEY,
  amount      NUMERIC(10, 2),
  ordered_at  TIMESTAMPTZ
);
-- 排序正確、SUM 也能用
```

**結論**：型別選對、資料庫幫你做了一半的事。型別選錯、後面 query 全是 bug。


### 5. NOT NULL —— 強制必填

**NULL** 不是 0、不是空字串、是「沒有值」的意思（**unknown**）。

```text
✅ NOT NULL    這欄一定要填、不能空著
   email TEXT NOT NULL

❌ 沒加 NOT NULL（預設）
   age INT
   → 允許 NULL（沒填）
```

**範例：**

```text
CREATE TABLE users (
  id    SERIAL PRIMARY KEY,
  name  TEXT NOT NULL,
  email TEXT NOT NULL
);

INSERT INTO users (name) VALUES ('Luffy');
-- ❌ 報錯：null value in column "email" violates not-null constraint
```

> 💡 **新手原則**：**能 NOT NULL 就 NOT NULL**。除非真的可以沒填（例 user 的「個人簡介」）、不然全部加上。NULL 處理超痛、避免就避免。


### 6. DEFAULT —— 沒填就給預設值

```text
CREATE TABLE posts (
  id          SERIAL PRIMARY KEY,
  title       TEXT NOT NULL,
  views       INT DEFAULT 0,
  is_published BOOLEAN DEFAULT FALSE,
  created_at  TIMESTAMPTZ DEFAULT NOW()
);

INSERT INTO posts (title) VALUES ('Hello SQL');
SELECT * FROM posts;
```

💬 **電腦會顯示**：

```text
 id |   title    | views | is_published |        created_at
----+------------+-------+--------------+----------------------------
  1 | Hello SQL  |     0 | f            | 2026-05-28 14:30:00.123+00
```

**沒填 views、is_published、created_at 都自動填好**。

**常見 DEFAULT 寫法：**

```text
DEFAULT 0                  數字預設 0
DEFAULT FALSE              boolean 預設 FALSE
DEFAULT ''                 文字預設空字串
DEFAULT NOW()              預設「現在時間」
DEFAULT gen_random_uuid()  預設隨機 UUID
DEFAULT CURRENT_USER       預設「執行 SQL 的使用者名稱」
```


### 7. 完整範例：設計一個部落格 schema

📄 **寫進程式檔**：`002_create_blog.sql`

```text
-- 使用者表
CREATE TABLE users (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email       TEXT UNIQUE NOT NULL,
  name        TEXT NOT NULL,
  bio         TEXT,
  created_at  TIMESTAMPTZ DEFAULT NOW()
);

-- 文章表
CREATE TABLE posts (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  author_id     UUID NOT NULL,
  title         TEXT NOT NULL,
  content       TEXT NOT NULL,
  views         INT DEFAULT 0,
  is_published  BOOLEAN DEFAULT FALSE,
  published_at  TIMESTAMPTZ,
  created_at    TIMESTAMPTZ DEFAULT NOW(),
  updated_at    TIMESTAMPTZ DEFAULT NOW()
);

-- 留言表
CREATE TABLE comments (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  post_id     UUID NOT NULL,
  author_id   UUID NOT NULL,
  content     TEXT NOT NULL,
  created_at  TIMESTAMPTZ DEFAULT NOW()
);
```

**這個 schema 做對的事：**

- 全部用 `UUID` 當 id（取代 SERIAL、分散式系統更友善）
- 必填欄位全 `NOT NULL`
- 時間戳全 `TIMESTAMPTZ DEFAULT NOW()`
- `bio` 可空（個人簡介本來就可不填）
- `views`、`is_published` 給合理 DEFAULT


### 8. 改表、刪表

**改表（ALTER TABLE）—— 已建好的 table 還能改：**

```text
-- 加欄位
ALTER TABLE users ADD COLUMN avatar_url TEXT;

-- 改型別
ALTER TABLE users ALTER COLUMN age TYPE BIGINT;

-- 改名
ALTER TABLE users RENAME COLUMN name TO display_name;

-- 加約束
ALTER TABLE users ALTER COLUMN email SET NOT NULL;

-- 刪欄位（危險）
ALTER TABLE users DROP COLUMN bio;
```

**刪表（DROP TABLE）—— 整張表炸掉、不可復原：**

```text
DROP TABLE users;            -- 不存在會報錯
DROP TABLE IF EXISTS users;  -- 不存在也不報錯（推薦）
```

> 💡 **DROP TABLE 在 Production 慎用**——資料全沒。先 `pg_dump` 備份。


### 9. 5 大常見坑

```text
❌ 1. 金額用 REAL 或 FLOAT
   → 0.1 + 0.2 = 0.30000000000000004、財務系統爆炸
   → 修：金額永遠 NUMERIC(p, s)

❌ 2. 時間用 TIMESTAMP（沒時區）
   → 跨時區網站時間錯亂
   → 修：永遠 TIMESTAMPTZ（PG 內部存 UTC、顯示自動轉）

❌ 3. id 用 SERIAL 在分散式系統踩坑
   → 多 server 同時寫入會撞號
   → 修：用 UUID（gen_random_uuid）

❌ 4. 沒加 NOT NULL 就導致 NULL 滿地跑
   → query 處處要寫 IS NULL 判斷
   → 修：預設加 NOT NULL、真的可空才省略

❌ 5. 改表怕怕、堆一堆 ALTER TABLE
   → schema 變混亂
   → 修：用 migration 工具（Supabase migrations / Prisma / Drizzle）
```


### 🎯 重點回顧

- **table** = SQL 的 Excel sheet、row = 一筆、column = 一欄
- **CREATE TABLE 表名 (欄位 型別 約束, ...)** 是基本句型
- **12 大型別**新手記 6 個：INT / TEXT / BOOLEAN / TIMESTAMPTZ / NUMERIC(10,2) / UUID
- **NOT NULL** 強制必填、**DEFAULT** 給預設值——兩個地基
- **金額用 NUMERIC、時間用 TIMESTAMPTZ、id 用 UUID** 是 2026 業界共識
