**SELECT（讀資料）= SQL 的瑞士刀**—— 每天寫 SQL 80% 的時間都在寫 SELECT。本課從 `SELECT *` 一路講到「計算欄位」、把這把刀的每個刃磨利。

> 💡 **這課的價值**：SELECT 看似簡單、實際變化無窮。把基本 5 招（*、指定欄、AS、DISTINCT、計算欄位）練熟、後面 WHERE / JOIN / GROUP BY 都是它的延伸。


### 1. 準備練習資料

🖥️ **在 SQL Editor 輸入**：

```text
CREATE TABLE users (
  id         SERIAL PRIMARY KEY,
  name       TEXT NOT NULL,
  email      TEXT NOT NULL,
  age        INT,
  city       TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

INSERT INTO users (name, email, age, city) VALUES
  ('Luffy',  'luffy@ai.com',  30, '台北'),
  ('Xueyue', 'xueyue@ai.com', 28, '台北'),
  ('Nami',   'nami@ai.com',   26, '台中'),
  ('Zoro',   'zoro@ai.com',   32, '高雄'),
  ('Sanji',  'sanji@ai.com',  30, '台北'),
  ('Robin',  'robin@ai.com',  28, '台中');
```

之後範例都用這 6 筆。


### 2. `SELECT *` —— 拿全部欄位

```text
SELECT * FROM users;
```

💬 **電腦會顯示**：

```text
 id |  name  |     email      | age | city  |        created_at
----+--------+----------------+-----+-------+----------------------------
  1 | Luffy  | luffy@ai.com   |  30 | 台北  | 2026-05-28 14:30:00.123+00
  2 | Xueyue | xueyue@ai.com  |  28 | 台北  | 2026-05-28 14:30:00.124+00
  3 | Nami   | nami@ai.com    |  26 | 台中  | 2026-05-28 14:30:00.125+00
  4 | Zoro   | zoro@ai.com    |  32 | 高雄  | 2026-05-28 14:30:00.126+00
  5 | Sanji  | sanji@ai.com   |  30 | 台北  | 2026-05-28 14:30:00.127+00
  6 | Robin  | robin@ai.com   |  28 | 台中  | 2026-05-28 14:30:00.128+00
(6 rows)
```

`*` 是「萬用字元（wildcard）」、代表「所有欄位」。

> 💡 **`SELECT *` 在 Production 別用**：欄位多時拖效能、schema 改了應用層也要跟改、bug 難 debug。**指定要的欄位是好習慣**。


### 3. 指定欄位（最常用）

```text
SELECT name, email FROM users;
```

💬 **電腦會顯示**：

```text
  name  |     email
--------+---------------
 Luffy  | luffy@ai.com
 Xueyue | xueyue@ai.com
 Nami   | nami@ai.com
 Zoro   | zoro@ai.com
 Sanji  | sanji@ai.com
 Robin  | robin@ai.com
```

**順序自由**——你寫 `SELECT email, name` 就先顯示 email。


### 4. `AS` —— 別名（給欄位 / 表改名）

**欄位別名：**

```text
SELECT
  name AS 姓名,
  email AS 電子郵件,
  age AS 年齡
FROM users;
```

💬 **電腦會顯示**：

```text
  姓名  |   電子郵件     | 年齡
--------+----------------+------
 Luffy  | luffy@ai.com   |  30
 ...
```

**表別名（後面 JOIN 會大量用）：**

```text
SELECT u.name, u.email
FROM users AS u;
```

或省略 AS：

```text
SELECT u.name, u.email
FROM users u;        -- 這樣也行、AS 可省略
```

> 💡 **別名規則**：欄位別名要顯示中文/特殊字元、用雙引號 `"姓名"`。簡單英文不用引號。


### 5. `DISTINCT` —— 去除重複

```text
SELECT city FROM users;
-- 結果有 6 列：台北、台北、台中、高雄、台北、台中

SELECT DISTINCT city FROM users;
-- 結果 3 列：台北、台中、高雄（不重複）
```

**多欄位 DISTINCT —— 整組組合不重複：**

```text
SELECT DISTINCT age, city FROM users;
```

💬 **電腦會顯示**：

```text
 age | city
-----+------
  30 | 台北
  28 | 台北
  26 | 台中
  32 | 高雄
  28 | 台中
```

注意：`(30, 台北)` 雖然兩筆、但「30 + 台北」這組合只出現一次、所以留一列。


### 6. 計算欄位（SELECT 裡寫表達式）

SELECT 不只能撈欄位、還能「算」新欄位。

**算術運算：**

```text
SELECT
  name,
  age,
  age * 2 AS double_age,
  age + 10 AS in_10_years
FROM users;
```

💬 **電腦會顯示**：

```text
  name  | age | double_age | in_10_years
--------+-----+------------+-------------
 Luffy  |  30 |         60 |          40
 Xueyue |  28 |         56 |          38
 ...
```

**字串拼接（PG 用 `||`）：**

```text
SELECT
  name || ' (' || city || ')' AS display
FROM users;
```

💬 **電腦會顯示**：

```text
    display
---------------
 Luffy (台北)
 Xueyue (台北)
 Nami (台中)
 ...
```

**內建函式：**

```text
SELECT
  name,
  UPPER(name) AS upper_name,         -- 轉大寫
  LOWER(email) AS lower_email,       -- 轉小寫
  LENGTH(name) AS name_len,          -- 字串長度
  age,
  age::TEXT AS age_as_text           -- 型別轉換
FROM users;
```

**條件欄位（CASE WHEN）：**

```text
SELECT
  name,
  age,
  CASE
    WHEN age < 18 THEN '未成年'
    WHEN age < 30 THEN '青年'
    WHEN age < 60 THEN '中年'
    ELSE '長者'
  END AS age_group
FROM users;
```

💬 **電腦會顯示**：

```text
  name  | age | age_group
--------+-----+----------
 Luffy  |  30 | 中年
 Xueyue |  28 | 青年
 Nami   |  26 | 青年
 Zoro   |  32 | 中年
 Sanji  |  30 | 中年
 Robin  |  28 | 青年
```


### 7. 完整 SELECT 語法骨架

```text
SELECT   選哪些欄位（必）
FROM     從哪張表（必）
WHERE    篩選條件（L6 整課）
GROUP BY 分組（L8 整課）
HAVING   分組後篩選（L8）
ORDER BY 排序（L7）
LIMIT    取幾筆（L7）
OFFSET   跳過幾筆（L7）
```

**執行順序（跟寫的順序不一樣！）：**

```text
1. FROM        先決定從哪張表撈
2. WHERE       過濾掉不要的列
3. GROUP BY    分組
4. HAVING      過濾分組結果
5. SELECT      算欄位 + 取欄位
6. DISTINCT    去重
7. ORDER BY    排序
8. LIMIT       取前幾筆
```

> 💡 **執行順序對新手超重要**——這也解釋為什麼 WHERE 不能寫別名（別名是 SELECT 階段才生、WHERE 比 SELECT 早跑）。


### 8. 不用 FROM 的 SELECT

PG 允許 `SELECT` 不接 FROM、單純算一個值。

```text
SELECT 1 + 1 AS result;              -- 2
SELECT NOW() AS time_now;            -- 當前時間
SELECT version();                    -- PG 版本
SELECT 'Hello' || ' ' || 'SQL';      -- Hello SQL
```

**像把 SQL 當計算機用**——適合測試函式行為。


### 9. 註解寫法

```text
-- 單行註解（兩個減號）

/*
  多行註解
  可以寫很多行
*/

SELECT name FROM users;  -- 這也是註解
```

**好習慣**：複雜 query 一定寫註解、半年後的你會感謝現在的你。


### 10. 5 大常見坑

```text
❌ 1. SELECT * 在 Production 用
   → 欄位多時拖效能、schema 改了應用層也炸
   → 修：永遠列出要的欄位

❌ 2. WHERE 寫別名
   → SELECT age*2 AS x FROM users WHERE x > 50  ❌
   → 因為 WHERE 比 SELECT 早跑、x 還不存在
   → 修：WHERE age*2 > 50 或包成子查詢

❌ 3. 字串用雙引號
   → PG 雙引號 = 識別字、單引號 = 字串
   → 修：字串永遠單引號、'Luffy' 對、"Luffy" 錯

❌ 4. 中文欄位別名沒加引號
   → SELECT name AS 姓名 ← 大部分能用、但有些版本要 "姓名"
   → 修：中文別名用雙引號 "姓名" 最安全

❌ 5. SELECT 順序錯
   → SELECT name FROM users WHERE LIMIT 5 ❌
   → 修：FROM → WHERE → SELECT → ORDER BY → LIMIT 的順序記熟
```


### 🎯 重點回顧

- **`SELECT * FROM 表`** 撈全部欄位（debug 用、不要 Prod）
- **`SELECT col1, col2 FROM 表`** 指定欄位（建議寫法）
- **`AS`** 給欄位 / 表別名、AS 可省略
- **`DISTINCT`** 去重、可單欄或多欄
- **SELECT 裡能寫表達式**（算術、字串、函式、CASE WHEN）
- **執行順序**：FROM → WHERE → GROUP → HAVING → SELECT → DISTINCT → ORDER → LIMIT
