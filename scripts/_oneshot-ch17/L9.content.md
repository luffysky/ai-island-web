**Window Function（視窗函式）= 不合併列、但能算分組統計**—— GROUP BY 把 6 列變 3 列、Window Function 維持 6 列、每列旁邊多一個「同組的統計」。是 SQL 進階分析的核心。

> 💡 **這課的價值**：「每個城市的第 1 名業績」、「跟上個月比成長 %」、「累計總額」——這 3 個經典需求 GROUP BY 做不到、Window Function 一句搞定。會 Window Function = 上一個檔次。


### 1. 為什麼需要 Window Function？

**問題**：列出「每個 user 的訂單、同時顯示這個 user 的總消費」。

**用 GROUP BY 做不到**——GROUP BY 會把同 user 的訂單合併成 1 列、就看不到每筆訂單了。

**Window Function 做法（保留所有列）：**

```text
SELECT
  user_id,
  amount,
  SUM(amount) OVER (PARTITION BY user_id) AS user_total
FROM orders;
```

💬 **電腦會顯示**（保留全部 7 筆訂單、每筆旁邊多一個「該 user 總額」）：

```text
 user_id | amount  | user_total
---------+---------+------------
       1 | 1200.00 |    4150.00
       1 |  850.00 |    4150.00
       1 | 5500.00 |    4150.00  ← wait, user 1 還是 user 3?
       1 | 2100.00 |    4150.00
       2 | 3200.00 |    3950.00
       2 |  750.00 |    3950.00
       4 | 9900.00 |    9900.00
```


### 2. 基本語法

```text
聚合函式 OVER (PARTITION BY 分組欄位 ORDER BY 排序欄位)
                  ↑                      ↑
                 怎麼分組               怎麼排序（部分函式需要）
```

**3 種 OVER 用法：**

```text
1. OVER ()
   → 「整張表」當一組
   SUM(amount) OVER ()  -- 全表總和

2. OVER (PARTITION BY col)
   → 按 col 分組、每組算
   SUM(amount) OVER (PARTITION BY user_id)  -- 每個 user 的總和

3. OVER (PARTITION BY col ORDER BY col2)
   → 分組 + 排序、給「累計」、「排名」用
   SUM(amount) OVER (PARTITION BY user_id ORDER BY created_at)
   -- 每個 user 的「累計到當下」總和
```


### 3. 排名 3 兄弟：ROW_NUMBER / RANK / DENSE_RANK

**準備資料：**

```text
INSERT INTO scores (name, subject, score) VALUES
  ('Luffy', 'math', 90),
  ('Xueyue', 'math', 95),
  ('Nami', 'math', 95),    -- 跟 Xueyue 同分
  ('Zoro', 'math', 80);
```

**3 種「排第幾」：**

```text
SELECT
  name,
  score,
  ROW_NUMBER() OVER (ORDER BY score DESC) AS row_num,
  RANK()       OVER (ORDER BY score DESC) AS rank,
  DENSE_RANK() OVER (ORDER BY score DESC) AS dense_rank
FROM scores;
```

💬 **電腦會顯示**：

```text
  name  | score | row_num | rank | dense_rank
--------+-------+---------+------+------------
 Xueyue |    95 |       1 |    1 |          1
 Nami   |    95 |       2 |    1 |          1    ← 同分
 Luffy  |    90 |       3 |    3 |          2    ← rank 跳 2、dense 不跳
 Zoro   |    80 |       4 |    4 |          3
```

```text
ROW_NUMBER   永遠 1, 2, 3, 4（不管同分）
RANK         同分同名次、後面「跳號」（1, 1, 3, 4）
DENSE_RANK   同分同名次、後面「不跳」（1, 1, 2, 3）
```

**3 個都常用、要按業務需求選：**

- 取「每組第 1 名」用 `ROW_NUMBER() = 1`
- 排行榜顯示給使用者看用 `RANK()`（同分並列）


### 4. 經典題：每個 user 最大的訂單

```text
SELECT user_id, amount, created_at
FROM (
  SELECT
    user_id,
    amount,
    created_at,
    ROW_NUMBER() OVER (PARTITION BY user_id ORDER BY amount DESC) AS rn
  FROM orders
) sub
WHERE rn = 1;
```

**步驟拆解：**

```text
內層：每個 user 的訂單按金額由大到小編號（ROW_NUMBER）
外層：只留 rn = 1 的列（每個 user 最大那筆）
```

**這招就叫「per-group top-N pattern」**——超常用、面試也常考。


### 5. LAG / LEAD —— 跟「前一列 / 後一列」比

```text
LAG(col, N)   往「上」N 列、預設 1
LEAD(col, N)  往「下」N 列、預設 1
```

**範例：每月營收 + 跟上月比較成長：**

```text
SELECT
  month,
  revenue,
  LAG(revenue) OVER (ORDER BY month) AS last_month,
  revenue - LAG(revenue) OVER (ORDER BY month) AS diff,
  ROUND(
    (revenue - LAG(revenue) OVER (ORDER BY month))::NUMERIC
    / LAG(revenue) OVER (ORDER BY month) * 100,
    2
  ) AS growth_pct
FROM monthly_sales
ORDER BY month;
```

💬 **電腦會顯示**：

```text
  month   | revenue | last_month |  diff  | growth_pct
----------+---------+------------+--------+-----------
 2026-01  |  100000 |       NULL |   NULL |       NULL
 2026-02  |  120000 |     100000 |  20000 |      20.00
 2026-03  |  150000 |     120000 |  30000 |      25.00
 2026-04  |  140000 |     150000 | -10000 |      -6.67
```

**LAG 第一列永遠是 NULL**（沒有上一列）。


### 6. 累計計算（PARTITION BY + ORDER BY）

```text
SELECT
  user_id,
  amount,
  created_at,
  SUM(amount) OVER (
    PARTITION BY user_id
    ORDER BY created_at
  ) AS running_total
FROM orders;
```

💬 **電腦會顯示**：

```text
 user_id | amount  |  created_at  | running_total
---------+---------+--------------+---------------
       1 | 2100.00 |  2026-04-25  |       2100.00
       1 | 1200.00 |  2026-05-01  |       3300.00
       1 |  850.00 |  2026-05-15  |       4150.00
       1 | 5500.00 |  2026-05-20  |       9650.00   wait, user 1 沒這筆
```

**重點**：有 `ORDER BY` 後、`SUM` 變「從第 1 列累計到當下」。**這就叫 running total**——銀行對帳單、儀表板必用。


### 7. Window Function 完整 8 大

```text
聚合類（不需要 ORDER BY 也能用）：
  COUNT() OVER (...)
  SUM() OVER (...)
  AVG() OVER (...)
  MIN() / MAX() OVER (...)

排名類（必須 ORDER BY）：
  ROW_NUMBER() OVER (...)
  RANK() OVER (...)
  DENSE_RANK() OVER (...)
  NTILE(N) OVER (...)   -- 把資料切成 N 等分

偏移類（必須 ORDER BY）：
  LAG(col, N) / LEAD(col, N)
  FIRST_VALUE(col) / LAST_VALUE(col)
  NTH_VALUE(col, N)
```

**NTILE 範例**：把 user 按消費分 4 等分（quartile）：

```text
SELECT
  user_id,
  total_spent,
  NTILE(4) OVER (ORDER BY total_spent DESC) AS quartile
FROM (...) sub;
-- quartile = 1 是消費最高的 25%、4 是最低
```


### 8. Window Function vs GROUP BY 對比

```text
GROUP BY
   合併同組列 → 列數變少
   一組 1 列
   不能保留原始列細節

Window Function
   不合併 → 列數不變
   每列旁邊加同組統計
   能保留全部明細
```

**何時用哪個？**

- 只要看「每組統計」 → GROUP BY
- 要看「明細 + 同組統計」 → Window Function
- 要找「每組第 N 名」 → Window Function（ROW_NUMBER）


### 9. WINDOW 子句（多個 OVER 共用設定）

寫多個 Window Function 用同樣設定時、會很囉嗦：

```text
SELECT
  user_id,
  amount,
  SUM(amount) OVER (PARTITION BY user_id ORDER BY created_at),
  AVG(amount) OVER (PARTITION BY user_id ORDER BY created_at),
  COUNT(*) OVER (PARTITION BY user_id ORDER BY created_at)
FROM orders;
```

**用 WINDOW 子句省事：**

```text
SELECT
  user_id,
  amount,
  SUM(amount) OVER w,
  AVG(amount) OVER w,
  COUNT(*) OVER w
FROM orders
WINDOW w AS (PARTITION BY user_id ORDER BY created_at);
```


### 10. 5 大常見坑

```text
❌ 1. ROW_NUMBER 沒寫 ORDER BY
   → 報錯（必須有 ORDER BY 才知道誰第一）
   → 修：ROW_NUMBER() OVER (ORDER BY ...) 必加 ORDER BY

❌ 2. Window Function 寫在 WHERE
   → WHERE rn = 1 ❌（WHERE 階段還沒算 rn）
   → 修：包成 subquery、外層 WHERE rn = 1

❌ 3. 想取「每組第 1 名」用 GROUP BY MAX
   → 拿得到最大值、但其他欄位不對應
   → 修：用 ROW_NUMBER() OVER (PARTITION BY ... ORDER BY ...) = 1

❌ 4. LAG 第一列預期不是 NULL
   → 第一列沒有上一列、預設 NULL
   → 修：LAG(col, 1, 0) 第三參數給預設值

❌ 5. 大表 Window Function 沒 Index
   → PARTITION BY + ORDER BY 的欄位要有 Index、不然全表排序超慢
   → 修：(partition_col, order_col) 上 composite Index
```


### 🎯 重點回顧

- **Window Function = 保留全部列、每列旁邊多一個同組統計**
- **OVER (PARTITION BY ... ORDER BY ...)** 三段組合就能變化萬千
- **排名 3 兄弟**：ROW_NUMBER（不跳）、RANK（跳）、DENSE_RANK（不跳但連續）
- **LAG / LEAD** 跟前/後列比較、寫月成長率必用
- **累計（running total）** = `SUM() OVER (PARTITION BY ... ORDER BY ...)`
- **每組第 N 名 pattern** = `ROW_NUMBER() OVER (PARTITION BY ... ORDER BY ...) = N`
