**Schema-First（schema 優先）= 先把 GraphQL schema 寫出來、再寫 resolver**。**Code-First（code 優先）= 用 TS / Python 寫程式、自動產 schema**。兩派各有信徒、選錯會痛。

> 💡 **這課的價值**：GraphQL 大型專案的設計哲學差異。學會兩派、你能在不同 team 都跑得起來。

  
### 1. 兩派的哲學

```text
🎨 Schema-First（GraphQL 原本提倡）
   1. 先用 GraphQL SDL（Schema Definition Language）寫 schema
   2. team review schema
   3. 後端寫 resolver
   4. 前端按 schema mock

   代表：Apollo Server、GraphQL Yoga

💻 Code-First（最近流行）
   1. 寫 TS / Python class / function
   2. 用 library 自動產 schema
   3. schema = 程式碼的副產品

   代表：Pothos（TS）、NestJS、Strawberry（Python）、TypeGraphQL
```

  
### 2. Schema-First — 完整範例

📄 **寫進程式檔**：`schema.graphql`

```text
type Query {
  post(id: ID!): Post
  posts(filter: PostFilter, pagination: PaginationInput): PostConnection!
}

type Mutation {
  createPost(input: CreatePostInput!): CreatePostPayload!
  updatePost(id: ID!, input: UpdatePostInput!): UpdatePostPayload!
  deletePost(id: ID!): DeletePostPayload!
}

type Post {
  id: ID!
  title: String!
  body: String
  author: User!
  comments(limit: Int = 10): [Comment!]!
  createdAt: DateTime!
}

type PostConnection {
  edges: [PostEdge!]!
  pageInfo: PageInfo!
  totalCount: Int!
}

type PostEdge {
  cursor: String!
  node: Post!
}

type PageInfo {
  hasNextPage: Boolean!
  endCursor: String
}

input CreatePostInput {
  title: String!
  body: String
}

type CreatePostPayload {
  post: Post
  errors: [UserError!]!
}

type UserError {
  field: String!
  message: String!
}

input PostFilter {
  authorId: ID
  search: String
  createdAfter: DateTime
}

input PaginationInput {
  first: Int = 20
  after: String
}

scalar DateTime
```

📄 **寫進程式檔**：`resolvers.ts`

```text
import { GraphQLDateTime } from 'graphql-scalars';

export const resolvers = {
  DateTime: GraphQLDateTime,

  Query: {
    post: (_, { id }) => db.posts.findUnique({ where: { id } }),
    posts: async (_, { filter, pagination }) => {
      const posts = await db.posts.findMany({
        where: buildWhere(filter),
        take: pagination.first + 1,
        cursor: pagination.after ? { id: pagination.after } : undefined,
      });
      // ... build connection
    },
  },

  Mutation: {
    createPost: async (_, { input }, { user }) => {
      if (!user) return { errors: [{ field: 'auth', message: 'Not logged in' }] };
      const post = await db.posts.create({ data: { ...input, authorId: user.id } });
      return { post, errors: [] };
    },
  },

  Post: {
    author: (post) => userLoader.load(post.authorId),
    comments: (post, { limit }) =>
      db.comments.findMany({ where: { postId: post.id }, take: limit }),
  },
};
```

**啟動**：

```text
import { ApolloServer } from '@apollo/server';
import { readFileSync } from 'fs';

const typeDefs = readFileSync('./schema.graphql', 'utf-8');

const server = new ApolloServer({ typeDefs, resolvers });
```

  
### 3. Code-First — 用 Pothos（TS 推薦）

📄 **寫進程式檔**：`schema.ts`

```text
import SchemaBuilder from '@pothos/core';

const builder = new SchemaBuilder<{
  Context: { user?: User };
  DefaultFieldNullability: false;
}>({});

builder.objectType('Post', {
  fields: (t) => ({
    id: t.exposeID('id'),
    title: t.exposeString('title'),
    body: t.exposeString('body', { nullable: true }),
    author: t.field({
      type: 'User',
      resolve: (post) => userLoader.load(post.authorId),
    }),
    comments: t.field({
      type: ['Comment'],
      args: { limit: t.arg.int({ defaultValue: 10 }) },
      resolve: (post, { limit }) =>
        db.comments.findMany({ where: { postId: post.id }, take: limit }),
    }),
    createdAt: t.expose('createdAt', { type: 'DateTime' }),
  }),
});

builder.queryType({
  fields: (t) => ({
    post: t.field({
      type: 'Post',
      nullable: true,
      args: { id: t.arg.id({ required: true }) },
      resolve: (_, { id }) => db.posts.findUnique({ where: { id } }),
    }),
  }),
});

export const schema = builder.toSchema();
```

**特色**：
- 一個檔案 = type + resolver 同時定義
- TS 自動推斷型別
- IDE auto-complete

  
### 4. 完整對比

```text
                Schema-First         Code-First
─────────────────────────────────────────────────
語法            SDL（GraphQL 語法）   TS / Python class
共識來源        schema 檔             程式碼
型別安全        要工具補 (codegen)   原生
重構            大量檔案要改          IDE 一鍵改
設計討論        easy（看 SDL）        要會程式
新人上手        easy                  中
分檔            自然                  要規劃
schema 演化     看得到 diff           看不到（要 build 後 diff）

代表             Apollo / Yoga         Pothos / NestJS / TypeGraphQL
```

  
### 5. 關鍵設計模式 — Relay-style

**Facebook Relay 推的最佳實踐**：

#### a. Connection / Edge（Cursor-based pagination）

```text
posts(first: 10, after: "cursor_xxx") {
  edges {
    cursor
    node { id title }
  }
  pageInfo {
    hasNextPage
    endCursor
  }
  totalCount
}
```

**好處**：
- 比 offset pagination 穩定（不會「新增資料時跳頁」）
- 大資料集快
- 標準化

#### b. Node interface — 全域 ID

```text
interface Node {
  id: ID!
}

type Post implements Node { id: ID! ... }
type User implements Node { id: ID! ... }

# ID 全站唯一
# "post_abc" / "user_xyz"

type Query {
  node(id: ID!): Node     # 任何東西都能 fetch
}
```

#### c. Mutation Payload

```text
type CreatePostPayload {
  post: Post                  # 成功的結果
  errors: [UserError!]!       # 失敗的清單
}
```

**永遠回 Payload** —— 避免「成功是 200、失敗丟 exception」的混亂。

  
### 6. Schema 演化策略

```text
✅ 加欄位     —— 安全（不破壞舊 client）
✅ 加 type    —— 安全
✅ 加 enum 值 —— 不安全！舊 client 不認得
❌ 刪欄位     —— breaking change
❌ 改型別     —— breaking change
❌ 改 nullable → non-nullable —— breaking change

策略：
1. 加新欄位、舊欄位標 @deprecated
2. 等 6 個月（觀察 metrics）
3. 刪掉

type Post {
  oldName: String @deprecated(reason: "Use newName")
  newName: String
}
```

  
### 7. Schema Stitching / Federation — 大型架構

```text
🧩 Schema Stitching
   N 個 server 的 schema 合併成一個
   舊技術、deprecated

🌐 Apollo Federation
   每個 service 自己一個 subgraph
   Gateway 合併
   業界標準

例：
- users-service：User type
- posts-service：Post type + 引用 User
- Gateway 自動 resolve 跨 service
```

  
### 8. Client 端 — 3 大 library

```text
🚀 Apollo Client
   - 功能最全（cache / optimistic / subscription）
   - 設定多
   - 大專案首選

⚡ urql
   - 輕量、簡單
   - 中小專案

🦋 Relay
   - Facebook 自用
   - 最嚴格 + 最快
   - 學習曲線陡
```

  
### 9. 5 大常見坑

```text
❌ 1. mixed schema-first + code-first
   → 一半檔案、一半 TS、改死
   → 修：選一個徹底執行

❌ 2. ID 不全域唯一
   → "1" 可能是 post 也可能是 user
   → 修：用 "post_1" / "user_1" prefix

❌ 3. 用 offset pagination 而非 cursor
   → 新增資料時頁碼亂跳
   → 修：用 Relay-style connection

❌ 4. Mutation 直接 throw
   → client 處理錯誤超痛
   → 修：永遠回 { data, errors } payload

❌ 5. 刪 schema 欄位不 deprecate
   → 行動 app 舊版直接爆
   → 修：先 @deprecated 半年再刪
```

  
### 🎯 重點回顧

- **Schema-First**：寫 SDL → 寫 resolver、設計感強、Apollo 系
- **Code-First**：寫 TS class → 自動產 schema、Pothos / NestJS
- **Relay-style 三大模式**：Connection / Node / Payload
- **schema 演化**：加安全、改 / 刪破壞、用 @deprecated 緩衝
- **大型架構** → Apollo Federation
- **Client 端** → Apollo / urql / Relay 三選一
