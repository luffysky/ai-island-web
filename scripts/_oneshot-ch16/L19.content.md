**HTTP（HyperText Transfer Protocol、超文本傳輸協定）= 瀏覽器跟 server 對話的語言**——你按 enter、發出 HTTP request、server 回 HTTP response。**method（方法）、status code（狀態碼）、headers（標頭）、cookies、CORS**——掌握這 5 個、debug API 速度快 10 倍。

> 💡 **這課的價值**：HTTP 不熟、整個 web 開發都是黑盒子。會這課後、F12 → Network 面板每個欄位你都看得懂、99% 的 API 問題自己解決。

  
### 1. HTTP 請求長什麼樣？

📄 **一個典型的 HTTP request**：

```text
POST /api/login HTTP/1.1
Host: api.example.com
Content-Type: application/json
Authorization: Bearer eyJhbGc...
Cookie: session_id=abc123

{
  "email": "alice@example.com",
  "password": "secret"
}
```

**拆解**：

```text
1. Request Line: POST /api/login HTTP/1.1
   ↑ method    ↑ path      ↑ HTTP 版本

2. Headers（一行一個 key: value）
   - Host: 目標 server
   - Content-Type: body 的格式
   - Authorization: 認證資訊
   - Cookie: 瀏覽器自動帶

3. Body（POST/PUT/PATCH 才有）
   - JSON / form / binary
```

  
### 2. HTTP Methods（方法）—— 6 個常用

| Method | 用途 | Body | Safe | Idempotent |
|---|---|---|---|---|
| **GET** | 讀取資料 | 無 | ✓ | ✓ |
| **POST** | 建立資料 | 有 | ✗ | ✗ |
| **PUT** | 完整覆蓋 | 有 | ✗ | ✓ |
| **PATCH** | 部分更新 | 有 | ✗ | △ |
| **DELETE** | 刪除 | 通常無 | ✗ | ✓ |
| **OPTIONS** | 詢問 CORS | 無 | ✓ | ✓ |

**名詞解釋**：

```text
Safe（安全）
   → 不改變 server 狀態
   → GET / HEAD / OPTIONS

Idempotent（冪等）
   → 打 1 次跟打 100 次效果一樣
   → GET / PUT / DELETE 都是
   → POST 不是（每次都建一筆新的）
```

> 💡 **`OPTIONS` 是 CORS preflight 用的**——瀏覽器在打跨網域 API 前、會先送 OPTIONS 問「我可以打嗎」。後面 CORS 章節會講。

  
### 3. Status Code（狀態碼）—— 5 大類

#### 1xx：資訊（不常見、不用記）

```text
100 Continue           繼續送
101 Switching Protocols  升級到 WebSocket
```

#### 2xx：成功

```text
200 OK                 一般成功
201 Created            POST 建立成功
202 Accepted           收到、之後處理（async）
204 No Content         成功、無回傳（DELETE 常用）
206 Partial Content    部分回應（video 分段下載）
```

#### 3xx：重新導向

```text
301 Moved Permanently  永久搬家（SEO 友善）
302 Found              暫時重導
304 Not Modified       cache 還有效、不用重新下載
307 Temporary Redirect 暫時重導（保留 method）
308 Permanent Redirect 永久重導（保留 method）
```

#### 4xx：客戶端錯（你寫錯）

```text
400 Bad Request        參數錯
401 Unauthorized       沒登入（其實該叫 Unauthenticated）
403 Forbidden          登入但沒權限
404 Not Found          找不到
405 Method Not Allowed method 不支援（DELETE 一個只接 GET 的 URL）
408 Request Timeout    request 太慢
409 Conflict           資料衝突（email 已被註冊）
410 Gone               資源永久消失
413 Payload Too Large  body 太大
418 I'm a teapot       笑話 status（真的有）
422 Unprocessable      格式對但語意錯（驗證失敗）
429 Too Many Requests  被 rate limit 擋
```

#### 5xx：伺服器錯（server 寫錯）

```text
500 Internal Server Error  server 程式碼炸了
502 Bad Gateway            proxy 收到上游錯誤
503 Service Unavailable    server 暫時掛
504 Gateway Timeout        proxy 等不到上游回應
```

  
### 4. Headers（標頭）—— 必懂 10 個

📄 **Request headers（client → server）**：

```text
Host: api.example.com           必填、目標主機
User-Agent: Mozilla/5.0...      瀏覽器資訊
Accept: application/json         想要的回應格式
Accept-Language: zh-TW,en       語言偏好
Authorization: Bearer xxxx       認證 token
Cookie: session=abc              cookie
Content-Type: application/json   body 的格式（POST 必填）
Content-Length: 123              body 長度
Origin: https://example.com      跨網域請求的來源
Referer: https://google.com/...  從哪個頁面來的
```

📄 **Response headers（server → client）**：

```text
Content-Type: text/html; charset=utf-8   回應的格式
Content-Length: 1024                     回應大小
Cache-Control: public, max-age=300       cache 規則
Set-Cookie: session=abc; HttpOnly        設 cookie
Location: /new-url                       3xx 重導目標
ETag: "abc123"                           版本標記（cache 用）
Last-Modified: Wed, 21 Oct 2025 07:28:00 GMT 最後修改
Access-Control-Allow-Origin: *           CORS 允許網域
Strict-Transport-Security: max-age=...   強制 HTTPS
Content-Security-Policy: ...             CSP 安全規則
```

  
### 5. `Content-Type` 常見值

```text
application/json                    JSON（API 主流）
application/x-www-form-urlencoded   一般 HTML form（key=val&key2=val2）
multipart/form-data                 含檔案的 form
text/html                           HTML 頁面
text/plain                          純文字
application/xml                     XML
text/event-stream                   SSE（Server-Sent Events）
application/octet-stream            binary
image/png                           PNG 圖
```

  
### 6. Cookies（餅乾）

**Cookie = server 寫進瀏覽器、之後每次 request 自動帶上的小資料**。

📄 **server 設定 cookie**：

```text
HTTP/1.1 200 OK
Set-Cookie: session_id=abc123; Max-Age=86400; HttpOnly; Secure; SameSite=Strict; Path=/
```

**Cookie 屬性**：

```text
Max-Age=86400          1 天後過期
Expires=...            指定時間過期
HttpOnly               JS 讀不到（防 XSS 偷 token）
Secure                 只在 HTTPS 傳送
SameSite=Strict        只在同網域用（防 CSRF）
   - Strict：完全不跨站
   - Lax：top-level navigation 才跨（預設）
   - None：完全允許（要配 Secure）
Path=/                 適用路徑
Domain=.example.com    適用網域（含子網域）
```

**之後瀏覽器自動帶**：

```text
GET /api/me HTTP/1.1
Cookie: session_id=abc123
```

  
### 7. CORS（Cross-Origin Resource Sharing、跨來源資源共享）

**問題**：瀏覽器預設「同源政策」—— `https://a.com` 的 JS 不能打 `https://b.com` 的 API（不然惡意網站可以偷你登入的銀行資料）。

**解法**：server 主動「允許」哪些 origin 可以打。

📄 **server response**：

```text
Access-Control-Allow-Origin: https://app.example.com
Access-Control-Allow-Methods: GET, POST, PUT, DELETE
Access-Control-Allow-Headers: Content-Type, Authorization
Access-Control-Allow-Credentials: true
Access-Control-Max-Age: 86400
```

**Preflight（預檢請求）**：

```text
1. 瀏覽器要打跨網域 API
2. 先送 OPTIONS 詢問
3. server 回 Access-Control-Allow-* headers
4. 瀏覽器看完才允許真正的 request
5. 真正的 GET / POST / 才送出

什麼會觸發 preflight？
   - method 是 PUT / DELETE / PATCH
   - 含 Authorization / Content-Type: application/json
   - 自訂 header
```

  
### 8. 實戰：F12 看 request

📄 **打開 Chrome DevTools → Network**：

```text
Name           Status  Type    Size     Time
─────────────────────────────────────────────
/api/me        200     fetch   324 B    45 ms
/api/posts     200     fetch   12 KB    120 ms
/login         401     fetch   89 B     30 ms

點進任何一個 → 看 Headers / Payload / Response
```

**常用 debug 步驟**：

```text
1. 看 Status：4xx 是你錯、5xx 是 server 錯
2. 看 Request Headers：cookie / auth 有沒有帶
3. 看 Payload：body 對不對
4. 看 Response：server 回什麼錯
5. 看 Timing：哪裡慢（DNS / TCP / TLS / Wait / Download）
```

  
### 9. 5 大常見坑

```text
❌ 1. 401 跟 403 搞混
   → 401 = 沒登入、403 = 登入但沒權限
   → 修：認真用對

❌ 2. CORS 設成 Allow-Origin: *
   → 安全爛、不能配 credentials
   → 修：明確列出允許的 origin

❌ 3. Cookie 沒 HttpOnly + Secure
   → JS 可以偷、HTTP 明文傳
   → 修：永遠寫 HttpOnly + Secure + SameSite

❌ 4. POST body 沒寫 Content-Type
   → server 不知道怎麼 parse、收到亂碼
   → 修：JSON 就寫 application/json

❌ 5. 全部用 GET 帶 query string 傳密碼
   → URL 被 log / browser history 記
   → 修：敏感資料一律 POST + body
```

  
### 🎯 重點回顧

- **6 個 method**：GET / POST / PUT / PATCH / DELETE / OPTIONS
- **5 大 status code 分類**：2xx 成功、3xx 重導、4xx 你錯、5xx server 錯
- **10 個必懂 headers**：Authorization / Cookie / Content-Type / CORS 系列
- **Cookie 屬性**：HttpOnly + Secure + SameSite 是安全三劍客
- **CORS** preflight 機制：OPTIONS 預檢 → 真請求
- **F12 → Network** 是 debug API 的最強武器
