「**同樣資料、不要每次都重算 / 重查**」—Cache 是後端最簡單但 ROI 最高的優化。

  
### 為什麼要 Cache

```text
場景：首頁顯示「熱門商品 top 10」

爛寫法（每個 request 跑）：
def homepage():
    top_products = db.query("""
        SELECT * FROM products
        ORDER BY sales DESC
        LIMIT 10
    """)
    return render(top_products)

# DB 累
# 100 個 user 一秒 = DB 跑 100 次同個 query

好寫法（cache）：
def homepage():
    top_products = cache.get("top_products")
    if not top_products:
        top_products = db.query(...)
        cache.set("top_products", top_products, ttl=60)
        # 存 60 秒
    return render(top_products)

# 60 秒內 DB 只查 1 次
# 後續 99 個 user 直接拿 cache
# 速度 10-100x

→ Cache 是「用記憶體換速度」
```

  
### 多層 Cache 架構

```text
實戰 SaaS 通常多層：

[ Browser ] 
    ↓ 1. Browser cache（Cache-Control header）
[ CDN ] 
    ↓ 2. CDN cache（Cloudflare / Vercel）
[ Load Balancer ]
    ↓
[ Web Server ]
    ↓ 3. Application cache（in-memory）
    ↓ 4. Redis cache（共享）
[ Database ]
    ↑ 5. DB query cache（內部）

層級越往上、越快但容量小

實戰：
1. 靜態資源（圖片、CSS、JS）：browser + CDN
2. API response：CDN（短 TTL）+ Redis
3. DB query：Redis
4. 計算結果：in-memory 或 Redis
```
