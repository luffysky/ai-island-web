**ch10 最後一課**——Next.js 部署三大選擇深入比較：**Vercel / Cloudflare Pages / Self-host**。L22 講 5 大平台、這課專注 Next.js 最常用的三個、含**成本估算**跟**真實設定**。

> 💡 **這課的價值**：「**該部到哪？**」是寫完 Next.js 後最常被問。答案不是「Vercel 最好」也不是「Cloudflare 最便宜」——**看你的流量規模、預算、技術需求**。這課給你決策依據。

  
### 1. Vercel：Next.js 親兒子、最方便

🖥️ **部署指令**

```text
npm install -g vercel
vercel
```

⌨️ **首次互動**

```text
? Set up and deploy? Y
? Link to existing project? N
? What's your project's name? my-app
? In which directory? ./
```

💬 **30 秒後**

```text
✓ Production: https://my-app-xxx.vercel.app
```

🖥️ **production 重新部署**

```text
vercel --prod
```

或更推薦：**連 GitHub → push 自動部署**。

**優點**：

```text
✅ 0 設定、Next.js 完美支援（畢竟是 Vercel 做的）
✅ 全球 CDN、自動 HTTPS
✅ 免費額度大（Hobby plan：100GB / 月、無限 build）
✅ Preview Deployment（每個 PR 自動產預覽網址）
✅ Edge Functions / Image Optimization 內建
✅ Analytics + Web Vitals 內建
✅ ISR / Streaming / PPR 第一個支援
```

**缺點**：

```text
❌ 商業專案費用快漲
   • Pro $20 / 月 + 流量超出費
   • Image Optimization 用量大、超出按用量收
   • Edge Function 執行次數收費

❌ 中國境內 user 速度慢（沒大陸節點）
❌ Bandwidth 超出後一段一段收（容易爆預算）
```

**價格估算**：

```text
個人專案、月 5000 PV：免費（Hobby）
小新創、月 5 萬 PV：$20 / 月（Pro）
中型 SaaS、月 50 萬 PV：$100-300 / 月
大型 SaaS、月 500 萬 PV：$1000+ / 月
```

  
### 2. Cloudflare Pages：便宜方案、流量大

🖥️ **裝 adapter**

```text
npm install -D @cloudflare/next-on-pages
```

🖥️ **build**

```text
npx @cloudflare/next-on-pages
```

📄 **寫進程式檔**：`wrangler.toml`

```text
name = "my-app"
compatibility_date = "2025-05-01"
compatibility_flags = ["nodejs_compat"]
pages_build_output_dir = ".vercel/output/static"

[build]
command = "npx @cloudflare/next-on-pages"
```

🖥️ **部署**

```text
npx wrangler pages deploy
```

或連 GitHub 自動部署。

**優點**：

```text
✅ 免費額度爆炸（500 builds / 月、無限 流量、無限 request）
✅ 全球 270+ 邊緣節點（中國境內速度可）
✅ 接 R2（物件儲存、無出口費）/ D1 / KV / Workers AI 完整
✅ DDoS 防護內建（不另收費）
✅ Pro 才 $5 / 月（比 Vercel 便宜 4 倍）
```

**缺點**：

```text
❌ Next.js 部分功能需 adapter
   • Image Optimization 要自己接 Cloudflare Images
   • 部分 Node API 要 nodejs_compat flag

❌ Edge runtime 限制
   • bundle size 1MB
   • 某些 npm 套件不支援
   • 不能 fs / native module

❌ ISR 沒 Vercel 原生支援
   • 要自己用 Cache API 模擬
```

**價格估算**：

```text
個人專案：免費
任何規模流量：$0 額外費用
Workers 超量：$5 / 月 包 10M 請求
```

  
### 3. Self-Host：最自由、配 Docker

📄 **寫進程式檔**：`Dockerfile`

```text
FROM node:22-alpine AS deps
WORKDIR /app
COPY package*.json ./
RUN npm ci

FROM node:22-alpine AS builder
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY . .
RUN npm run build

FROM node:22-alpine AS runner
WORKDIR /app
ENV NODE_ENV=production
COPY --from=builder /app/.next/standalone ./
COPY --from=builder /app/.next/static ./.next/static
COPY --from=builder /app/public ./public
EXPOSE 3000
CMD ["node", "server.js"]
```

📄 **寫進程式檔**：`next.config.mjs`

```text
const nextConfig = {
  output: "standalone"  // self-host 必加（產獨立可跑檔）
};
export default nextConfig;
```

🖥️ **build + run**

```text
docker build -t my-app .
docker run -p 3000:3000 --env-file .env my-app
```

**self-host 平台選擇**：

```text
🟢 Zeabur（最推薦 Indie）
   • Asia 機房、跟 Vercel 體驗類似
   • NT$ 50-200 / 月
   • 一鍵接 PostgreSQL / Redis
   • 繁中介面、台灣團隊支援

🟢 Railway
   • $5 起跳、適合 prototype
   • 接 DB / cron 方便
   • UI 漂亮

🟢 Fly.io
   • 全球節點、配 Postgres / Redis 方便
   • $0 試用、按用量
   • 比較進階

🟢 AWS / GCP / Azure
   • 企業級
   • 學習曲線高、預算高
```

**何時 self-host**：

```text
✅ 要長連線（Socket.io / WebSocket）
✅ 要 cron job（每分鐘跑東西）
✅ 要 native module（bcrypt / sharp 等）
✅ 要 GPU / 機器學習
✅ 已有自家 server / Kubernetes
✅ 不想被 Vercel / Cloudflare 鎖
```

  
### 4. 對比表（最重要）

```text
| 平台             | 免費額度    | 月費起    | 適合              | 缺點          |
|------------------|------------|-----------|-------------------|---------------|
| Vercel           | 100GB / 月  | $20 USD   | Indie / Startup    | 流量大會貴     |
| Cloudflare Pages | 無限流量    | $0        | 流量大 / 預算少    | adapter 限制   |
| Zeabur           | 試用       | NT$ 50    | 台灣 Indie         | CDN 沒前兩家   |
| Railway          | $5 credit   | $5 USD    | prototype / 容器   | 不全球          |
| Fly.io           | 試用       | $0 起     | 全球容器           | 上手較難        |
| Self AWS         | $0 試用    | $20+      | 企業              | 學習曲線高     |
```

  
### 5. 決策樹

```text
你的專案是？
   ├─ 個人 portfolio / blog
   │   → Vercel（免費、最快）
   │
   ├─ 個人 side project / MVP
   │   → Vercel（免費）或 Cloudflare（更便宜）
   │
   ├─ 早期新創（< 100 用戶）
   │   → Vercel Hobby（免費）或 Pro（$20）
   │
   ├─ 成長中新創（流量爆漲）
   │   → 評估：Vercel Pro vs Cloudflare Pages
   │   → 流量 > 500GB / 月 → Cloudflare
   │   → 用大量 Edge Function → Vercel
   │
   ├─ 中型 SaaS（月 10 萬+ 用戶）
   │   → 混搭：Vercel + Cloudflare CDN
   │   → 或 self-host on AWS
   │
   ├─ 需要長連線 / cron / native module
   │   → Self-host on Zeabur / Railway / Fly
   │
   └─ 服務中國市場
       → Cloudflare（境內節點多）
       → 或國內主機（阿里雲、騰訊雲）
```

  
### 6. 環境變數設定（每個平台都要）

📄 **寫進程式檔**：`.env.local`（不進 git）

```text
DATABASE_URL=postgres://...
OPENAI_API_KEY=sk-...
AUTH_SECRET=...
NEXTAUTH_URL=https://yoursite.com

# NEXT_PUBLIC_ 開頭會曝露給 client（謹慎用）
NEXT_PUBLIC_GA_ID=G-XXX
NEXT_PUBLIC_SUPABASE_URL=https://...
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJ...
```

**部署平台設定**：

```text
Vercel：    Settings → Environment Variables
Cloudflare：Pages → Settings → Environment Variables
Zeabur：    Service → Variables
Railway：   Variables tab
Docker：    --env-file .env  /  docker-compose.yml
```

  
### 7. 自訂網域（custom domain）

```text
1. 買網域（GoDaddy / Namecheap / Cloudflare Registrar）
2. 在部署平台加網域：
   Vercel：Settings → Domains → Add
   Cloudflare：DNS records auto
3. 在 DNS 設定加：
   CNAME yoursite.com → cname.vercel-dns.com
   或 A record → Vercel IP
4. 等 DNS propagation（5 分鐘 - 24 小時）
5. HTTPS 自動啟用（Let's Encrypt）
```


### 5 大常見坑（總結）

```text
❌ 1. 部署平台選錯、帳單爆炸
   → 看流量規模、看技術需求
   → 修：對照表選

❌ 2. 環境變數沒設、production 壞
   → NEXTAUTH_URL、DATABASE_URL、API keys
   → 修：deploy 前 checklist 過一遍

❌ 3. self-host 沒 output: "standalone"
   → bundle 大、無法獨立跑
   → 修：next.config.js 加 output: "standalone"

❌ 4. 用 Vercel 跑 cron job
   → Vercel 沒原生 cron（要 Cron Jobs feature，需 Pro）
   → 修：用 cron-job.org 打 API 觸發

❌ 5. 沒設監控、出錯了不知道
   → 使用者跟你說網站掛了才知道
   → 修：Sentry + Uptime Robot 必裝
```

  
### 🎯 重點回顧

- **Vercel**：Next.js 親兒子、Hobby 免費、Pro $20、流量大會貴
- **Cloudflare Pages**：免費額度爆炸、適合流量大 / 預算少、有 adapter 限制
- **Self-host (Zeabur / Railway / Fly)**：需要長連線 / cron / native module 時選
- **決策**：看流量、看預算、看技術需求
- **每個 deploy 都要做**：環境變數 / 自訂網域 / 監控
