**部署（deploy、把網站推到網路上讓全世界看得到）**——Next.js / Nuxt 寫完只是第一步、要部署才能讓朋友打開網址看。這課講 5 大選擇：**Vercel / Cloudflare Pages / Zeabur / Railway / Self-host Docker**。

> 💡 **這課的價值**：很多新手寫完 Next / Nuxt 卡在「**怎麼讓別人看到**」——選對部署平台、5 分鐘上線；選錯、設定一週還跑不起來。

  
### 1. Vercel（Next.js 親兒子、最簡單）

**Vercel** = Next.js 開發公司、整合 0 設定、deploy 平台界的金標準。

🖥️ **方法 1：CLI**

```text
npm install -g vercel
vercel
```

⌨️ **這時換你打字**（互動式問答）

```text
? Set up and deploy "~/my-app"? [Y/n] y
? Which scope? Your Name
? Link to existing project? [y/N] n
? What's your project's name? my-app
? In which directory is your code located? ./
```

💬 **電腦會顯示**

```text
✓ Production: https://my-app-xxx.vercel.app
```

🖥️ **方法 2：GitHub 連 Vercel（推薦）**

```text
1. 推 code 上 GitHub
2. 去 vercel.com → Import Project
3. 選你的 repo
4. Vercel 自動偵測 Next.js / Nuxt、不用設定
5. 點 Deploy
6. 30 秒後上線
```

**Vercel 提供**：

```text
✅ 全球 CDN（自動）
✅ Edge Functions
✅ ISR / Streaming（內建支援）
✅ 自動 HTTPS（Let's Encrypt）
✅ Preview deployment（每個 PR 自動產預覽網址）
✅ A/B testing
✅ Analytics（流量、Web Vitals）
✅ Image Optimization（next/image 自動）
✅ Hobby plan 免費（個人 + 小新創夠用）
```

**價格**：
- Hobby：免費（100GB 流量 / 月、商業用要升級）
- Pro：$20 USD / 月起、流量超過按用量

  
### 2. Cloudflare Pages（流量大、預算少的選擇）

🖥️ **Next.js 部署**

```text
npm install -D @cloudflare/next-on-pages
npx next-on-pages
```

🖥️ **Nuxt 部署**（內建 preset）

```text
NITRO_PRESET=cloudflare-pages npm run build
```

📄 **設定**：`wrangler.toml`

```text
name = "my-app"
compatibility_date = "2025-05-01"
compatibility_flags = ["nodejs_compat"]

[build]
command = "npx @cloudflare/next-on-pages"
output = ".vercel/output/static"
```

**Cloudflare Pages 優勢**：

```text
✅ 免費額度大（500 builds / 月、無限流量）
✅ 全球 270+ 邊緣節點（速度快）
✅ R2（物件儲存、免出口費）
✅ D1（SQLite at edge）
✅ KV / Workers AI 完整生態
✅ 中國境內速度可（有節點）
```

**缺點**：

```text
❌ 部分 Node API 不支援（需 nodejs_compat flag）
❌ Next.js 部分功能要 adapter
❌ 大檔案（> 25MB）有限制
❌ Image Optimization 要自己接 Cloudflare Images
```

  
### 3. Zeabur（台灣團隊、亞洲機房）

**Zeabur** 是台灣團隊做的部署平台、繁中介面、東京 / 美西機房。

🖥️ **部署方式**

```text
1. 連 GitHub
2. 選 repo
3. Zeabur 自動偵測 Next / Nuxt
4. 選機房（東京 / 美西）
5. 點 Deploy
```

**優勢**：

```text
✅ 亞洲機房（東京）— 台灣 ping 10ms 內
✅ 繁中介面、台灣團隊支援快
✅ 便宜（NT$ 50 / 月起）
✅ 支援完整 Node 環境（沒 edge runtime 限制）
✅ 一站式：DB（PostgreSQL / Redis）/ Cron / Volume 都包
✅ AI 島本身就部署在 Zeabur
```

**缺點**：

```text
❌ 沒 Vercel ISR 自動優化（要自己設）
❌ 全球 CDN 沒 Vercel / Cloudflare 廣
```

### 5. Railway / Render / Fly.io

「**傳統容器化**」（用 Docker、不是 serverless）——適合**需要長連線 / WebSocket / cron** 的應用。

```text
🟢 Railway：$5 起跳、適合 prototype、UI 漂亮
🟢 Render：免費 tier、休眠後喚醒慢（適合 demo）
🟢 Fly.io：全球、配 Postgres / Redis 方便
```

**何時用容器化**：

```text
✅ 要長連線（Socket.io / WebSocket）
✅ 要 cron（每分鐘跑東西）
✅ Edge runtime 不夠用（要 native module）
✅ 要 GPU / 機器學習推論
```

  
### 6. 自架 Docker（自由度最高）

📄 **寫進程式檔**：`Dockerfile`

```text
FROM node:20-alpine AS deps
WORKDIR /app
COPY package*.json pnpm-lock.yaml ./
RUN npm i -g pnpm && pnpm install --frozen-lockfile

FROM node:20-alpine AS builder
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY . .
RUN npm i -g pnpm && pnpm build

FROM node:20-alpine AS runner
WORKDIR /app
ENV NODE_ENV=production
COPY --from=builder /app/.next ./.next
COPY --from=builder /app/package.json ./
COPY --from=builder /app/node_modules ./node_modules
COPY --from=builder /app/public ./public
EXPOSE 3000
CMD ["npm", "start"]
```

📄 **Next.js 需要**：`next.config.js`

```text
module.exports = {
  output: "standalone"  // self-host 必加（output 變小）
};
```

🖥️ **build + run**

```text
docker build -t my-app .
docker run -p 3000:3000 my-app
```

  
### 7. 環境變數（每個平台都要設）

📄 **寫進程式檔**：`.env.local`（**永遠 .gitignore**）

```text
DATABASE_URL=postgres://...
OPENAI_API_KEY=sk-...
AUTH_SECRET=...

# Next.js 約定
# NEXT_PUBLIC_* 開頭會曝露給 client
NEXT_PUBLIC_GA_ID=G-XXX

# Nuxt 約定
# 在 runtimeConfig.public 內的會曝露給 client
NUXT_PUBLIC_API_BASE=...
```

**部署時**：

```text
Vercel：  Project → Settings → Environment Variables
Cloudflare：Pages → Settings → Variables
Zeabur：  Service → Variables
Railway： Service → Variables
Docker：  --env-file .env  或  docker-compose
```

  
### 8. 資安：API key 絕對別放 client

```text
🚨 絕對不要：
  NEXT_PUBLIC_OPENAI_KEY=sk-...
  NUXT_PUBLIC_STRIPE_SECRET=sk-...

  → 任何 NEXT_PUBLIC_ / runtimeConfig.public 的會 bundle 進 client
  → 客戶 F12 看 source 就拿到
  → 你的 API key 被全世界用、帳單爆炸

✅ 正確：
  • OpenAI key → 只 Server Component / API Route 用
  • Stripe secret key → 只 server 用、不暴露
  • 只有 publishable key（pk_*）才能 client
```

### 10. 部署 checklist

```text
✓ .env.local 已 .gitignore（沒 push 上 GitHub）
✓ 環境變數在部署平台都設好
✓ 不該 PUBLIC_ 的 key 沒寫 PUBLIC_ 開頭
✓ DB migration 已跑（線上 schema 對齊）
✓ next.config.js / nuxt.config.ts 內 production URL 對
✓ 圖片 remotePatterns 加白名單
✓ middleware 跑得對（保護路徑、安全 header）
✓ 第三方服務（Sentry / GA / Stripe）已切 production key
✓ 自訂網域已 DNS 設定（CNAME / A record）
✓ HTTPS 已啟用（多平台自動）
✓ 部署完用 Lighthouse 跑分 + 開幾頁測試
```

  
### 5 大常見坑

```text
❌ 1. API key 寫 NEXT_PUBLIC_ 開頭、被偷
   → 整個 OpenAI 額度被燒光
   → 修：sensitive key 永遠不 PUBLIC_

❌ 2. localhost URL 寫死進 code
   → production fetch http://localhost:3000/... 失敗
   → 修：用 process.env.NEXT_PUBLIC_BASE_URL

❌ 3. .env.local 不小心 push
   → key 外洩、必須 rotate
   → 修：永遠 .gitignore、用 git-secrets 預防

❌ 4. 部署完 cookies / session 壞掉
   → 通常是 NEXTAUTH_URL 沒設 production URL
   → 修：每個環境變數確認 production 值

❌ 5. Vercel 帳單突然爆炸
   → ISR 設太短、Function 跑太多次
   → 修：能 SSG 就 SSG、加 cache header
```
