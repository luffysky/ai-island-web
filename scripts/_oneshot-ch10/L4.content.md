**Server Components（伺服器元件）vs Client Components（瀏覽器元件）** 是 App Router 最重要的觀念。**預設所有元件都是 Server Component、要互動才用 Client**。學完這課你就懂為什麼有些檔案頂端要寫 `"use client"`。

> 💡 **這課的價值**：新手最大的誤解是「**全部加 "use client" 比較簡單**」——這樣等於回到純 SPA、Next.js 所有好處都沒了。**正確分配 server / client 邊界、bundle 可以小 50%、首屏快 2 秒**。

  
### 1. Server Component（伺服器元件）

📄 **寫進程式檔**：`app/users/page.tsx`

```text
// 預設就是 Server Component（不用任何標記）
import { db } from "@/lib/db";
import fs from "node:fs";

export default async function UserList() {
  // ✓ 可以 async / await
  const users = await db.users.findMany();

  // ✓ 可以直接讀 DB（不用 API 中間層）
  // ✓ 可以用 Node.js API（fs、crypto）
  // ✓ 可以讀環境變數（API key 安全、不會到 client）
  const apiKey = process.env.SECRET_KEY;  // 不會曝露給瀏覽器

  // ✗ 不能用 useState、useEffect
  // ✗ 不能綁 onClick
  // ✗ 不能用 browser API（window、localStorage、document）

  return (
    <ul>
      {users.map(u => <li key={u.id}>{u.name}</li>)}
    </ul>
  );
}
```

**白話**：Server Component 是「**在 server 跑、跑完把結果（HTML）寄到瀏覽器**」的元件。瀏覽器看不到原始碼、API key 永遠安全。

  
### 2. Client Component（瀏覽器元件）

📄 **寫進程式檔**：`app/counter.tsx`

```text
"use client";
// ↑ 檔案頂端寫這行、這檔案的元件全部變 Client Component

import { useState, useEffect } from "react";

export default function Counter() {
  // ✓ 可以用所有 hooks
  const [count, setCount] = useState(0);

  // ✓ 可以綁事件
  // ✓ 可以用 browser API
  useEffect(() => {
    const stored = localStorage.getItem("count");
    if (stored) setCount(Number(stored));
  }, []);

  // ✗ 不能 async function 直接 await
  // ✗ 不能直接讀 DB（要透過 API / Server Action）

  return (
    <button onClick={() => setCount(c => c + 1)}>
      {count}
    </button>
  );
}
```

**白話**：Client Component 是「**程式碼會被打包送到瀏覽器、在使用者電腦上跑**」的元件。能 handle 點擊、動畫、localStorage 這些 server 做不到的事。

  
### 3. 兩者比較表

```text
| 能力                | Server Component | Client Component |
|---------------------|------------------|------------------|
| async / await        | ✓                | ✗                |
| 直接讀 DB / 檔案系統 | ✓                | ✗                |
| 讀 process.env       | ✓ 安全           | 只 NEXT_PUBLIC_  |
| useState / useEffect | ✗                | ✓                |
| onClick / onChange   | ✗                | ✓                |
| window / localStorage| ✗                | ✓                |
| 進 client bundle 嗎? | ✗ 不會           | ✓ 會（檔案變大） |
| SEO 友善             | ✓                | 部分             |
```

  
### 4. 邊界規則：Server 可以 import Client、反之不行

📄 **對的寫法**：`app/page.tsx`（Server）

```text
import Counter from "./counter";  // Client Component
import { getData } from "@/lib/db";

export default async function Page() {
  const data = await getData();  // Server 跑

  return (
    <>
      <h1>{data.title}</h1>     {/* Server 渲染 */}
      <Counter />               {/* Client 互動 */}
    </>
  );
}
```

📄 **錯的寫法**：

```text
"use client";
import ServerThing from "./ServerThing";  // ❌ 編譯錯

export default function Wrapper() {
  return <ServerThing />;
}
```

**為什麼？** Client Component 是在瀏覽器跑、不能 import 一個「需要 DB 連線」的 Server Component。

  
### 5. 用 children 模式繞過限制

雖然 Client 不能 import Server、但**可以把 Server Component 當 children 塞進 Client Component**：

📄 **寫進程式檔**：`app/client-wrapper.tsx`

```text
"use client";
import { useState } from "react";

export default function ClientWrapper({ children }) {
  const [open, setOpen] = useState(false);
  return (
    <div>
      <button onClick={() => setOpen(!open)}>切換</button>
      {open && children}  {/* children 可以是 Server Component！ */}
    </div>
  );
}
```

📄 **寫進程式檔**：`app/page.tsx`

```text
import ClientWrapper from "./client-wrapper";
import ServerData from "./server-data";  // Server Component

export default async function Page() {
  return (
    <ClientWrapper>
      <ServerData />  {/* Server 渲染好、當 children 傳 */}
    </ClientWrapper>
  );
}
```

> 💡 **重點**：children 是「**已經渲染好的 React tree**」、不是 import 關係。所以 Client 可以接受 Server 渲染的內容當 children。

  
### 6. 資料怎麼從 Server 傳到 Client？用 props

📄 **寫進程式檔**：`app/page.tsx`（Server）

```text
import Likes from "./likes";

export default async function Page() {
  // 在 server 抓資料
  const post = await db.posts.findUnique({ where: { id: 1 } });

  return (
    <article>
      <h1>{post.title}</h1>
      {/* 把初始值傳給 Client、Client 負責互動 */}
      <Likes initialCount={post.likes} postId={post.id} />
    </article>
  );
}
```

📄 **寫進程式檔**：`app/likes.tsx`（Client）

```text
"use client";
import { useState } from "react";

export default function Likes({ initialCount, postId }) {
  const [count, setCount] = useState(initialCount);

  async function like() {
    setCount(c => c + 1);
    await fetch(`/api/posts/${postId}/like`, { method: "POST" });
  }

  return <button onClick={like}>👍 {count}</button>;
}
```

**重點**：props 必須是「**可序列化**（serializable、能變字串）」的——number / string / array / object 都行、function / class 不行。

  
### 7. 實戰準則（背起來）

```text
1. 預設用 Server Component
   → 需要互動才加 "use client"

2. 把 "use client" 推到「葉子」（leaf、最小元件）
   → 不要整個 page 都 client
   → 例：button 用 client、外面 page 還是 server

3. 抓資料盡量在 server
   → 傳 props 給 client component 顯示

4. API key 永遠在 server
   → 千萬不要傳到 client（你的 OpenAI key 會被偷）

5. 表單 / 動畫 / 互動 → Client
6. 內容展示 / SEO 重要 → Server
```

  
### 8. 哪些 hooks / API 強制要 Client

```text
✓ useState / useReducer / useContext
✓ useEffect / useLayoutEffect
✓ useRef
✓ useRouter（next/navigation）
✓ usePathname / useSearchParams
✓ onClick / onChange / onSubmit
✓ window / document / localStorage / sessionStorage
✓ navigator / IntersectionObserver
```

用到其中任何一個、**那個元件就必須是 Client Component**、頂端加 `"use client"`。

  
### 5 大常見坑

```text
❌ 1. 整個 app 加 "use client"
   → 等於回到純 SPA
   → 修：只在真的要互動的元件加

❌ 2. Server Component 內用 useState
   → 編譯錯
   → 修：拆出 client 子元件

❌ 3. props 傳 function 給 Client
   → 不能序列化、編譯錯
   → 修：function 在 Client 內定義、或用 Server Action

❌ 4. 用 process.env.OPENAI_KEY 傳到 Client
   → 寫成 NEXT_PUBLIC_OPENAI_KEY、被全世界看到
   → 修：永遠在 Server Component / API Route 用

❌ 5. 在 Server Component 內 import 整個 client-only 套件
   → 例如 framer-motion 直接 import
   → 修：包成 Client wrapper、再 import
```

  
### 🎯 重點回顧

- **預設 Server**、要互動加 `"use client"`
- **Server Component**：能 async、能讀 DB、API key 安全、不進 client bundle
- **Client Component**：能用 hook / 事件 / browser API、會進 bundle
- **邊界**：Server 可以 import Client、Client 不能 import Server（但可以接 children）
- **資料傳遞**：用 props（必須可序列化）
- **策略**：把 "use client" 推到葉子、保留越多 Server 越好
