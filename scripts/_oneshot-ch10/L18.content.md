Nuxt 內建 **Nitro server**——「**檔名就是路由**、不用設 router」。同個專案前後端寫在一起、部署成一個 app、零設定。學完這課你就有完整全端能力。

> 💡 **這課的價值**：傳統 Vue + Express 要兩個專案、兩次部署、跨域要 CORS——Nuxt 用 Nitro **一個專案搞定全部**、部署一次、無 CORS 煩惱。

  
### 1. 檔名約定：method 寫在副檔名

```text
server/api/
├── users.get.ts                → GET  /api/users
├── users.post.ts               → POST /api/users
├── users/[id].get.ts           → GET  /api/users/:id
├── users/[id].patch.ts         → PATCH /api/users/:id
├── users/[id].delete.ts        → DELETE /api/users/:id
├── auth/
│   ├── login.post.ts           → POST /api/auth/login
│   └── logout.post.ts          → POST /api/auth/logout
└── webhook/
    └── stripe.post.ts           → POST /api/webhook/stripe
```

**白話**：檔名 `xxx.METHOD.ts` 自動變成那個 method 的 endpoint。不用寫 router、不用 export GET/POST、簡潔。

  
### 2. 第一個 API endpoint

📄 **寫進程式檔**：`server/api/users.get.ts`

```text
export default defineEventHandler(async (event) => {
  const query = getQuery(event);
  const page = parseInt(query.page as string) || 1;
  const limit = parseInt(query.limit as string) || 20;

  const users = await db.users.findMany({
    skip: (page - 1) * limit,
    take: limit
  });

  return {
    data: users,
    page,
    limit
  };
});
```

**重點**：
- `defineEventHandler` 把 function 包成 event handler
- `getQuery(event)` 拿 URL 查詢字串
- 直接 `return` 物件、Nitro 自動 JSON.stringify

🖥️ **測試**

```text
curl http://localhost:3000/api/users?page=1
```

💬 **電腦會顯示**

```text
{"data":[...],"page":1,"limit":20}
```

  
### 3. POST + body 驗證

📄 **寫進程式檔**：`server/api/users.post.ts`

```text
import { z } from "zod";

const CreateUserSchema = z.object({
  name: z.string().min(1),
  email: z.string().email()
});

export default defineEventHandler(async (event) => {
  const body = await readBody(event);

  const validated = CreateUserSchema.safeParse(body);
  if (!validated.success) {
    throw createError({
      statusCode: 400,
      statusMessage: "Invalid input",
      data: validated.error.flatten()
    });
  }

  const user = await db.users.create({ data: validated.data });
  return user;
});
```

**Nitro 工具函式**：
- `readBody(event)` — 讀 request body（JSON）
- `getQuery(event)` — 讀查詢字串
- `getRouterParam(event, "id")` — 讀動態路由參數
- `getHeader(event, "authorization")` — 讀 header
- `getCookie(event, "token")` — 讀 cookie
- `setCookie(event, "token", "...")` — 設 cookie
- `createError(...)` — 拋錯
- `sendRedirect(event, "/login")` — 轉址

  
### 4. 動態路由

📄 **寫進程式檔**：`server/api/users/[id].get.ts`

```text
export default defineEventHandler(async (event) => {
  const id = getRouterParam(event, "id");

  const user = await db.users.findUnique({ where: { id } });
  if (!user) {
    throw createError({
      statusCode: 404,
      statusMessage: "Not found"
    });
  }

  return user;
});
```

  
### 5. Server Middleware（每個 API 都會跑）

📄 **寫進程式檔**：`server/middleware/auth.ts`

```text
export default defineEventHandler(async (event) => {
  const url = getRequestURL(event);

  // 只對 /api/admin/* 路徑跑
  if (url.pathname.startsWith("/api/admin")) {
    const token = getCookie(event, "auth-token");
    if (!token || !await isAdmin(token)) {
      throw createError({ statusCode: 401, statusMessage: "Unauthorized" });
    }
  }
});
```

**注意**：`server/middleware/` 內每個檔案**所有 server request 都會跑**——要在 handler 內判斷路徑。

  
### 6. utils：跨 endpoint 共用的工具

📄 **寫進程式檔**：`server/utils/auth.ts`

```text
import { H3Event } from "h3";

export async function getCurrentUser(event: H3Event) {
  const token = getCookie(event, "auth-token");
  if (!token) return null;

  try {
    const payload = await verifyJWT(token);
    return await db.users.findUnique({ where: { id: payload.sub } });
  } catch {
    return null;
  }
}

export async function requireAuth(event: H3Event) {
  const user = await getCurrentUser(event);
  if (!user) {
    throw createError({ statusCode: 401 });
  }
  return user;
}
```

📄 **使用**：`server/api/profile.get.ts`

```text
export default defineEventHandler(async (event) => {
  const user = await requireAuth(event);  // 自動 import！
  return { id: user.id, email: user.email };
});
```

> 💡 **跟 Next.js 對比**：Next.js 要手動 import；Nuxt 的 `server/utils/` 跟 `composables/` 一樣**自動 import 到 server**——程式碼超簡潔。

  
### 7. cron / scheduled tasks（Nitro tasks）

📄 **寫進程式檔**：`server/tasks/cleanup.ts`

```text
export default defineTask({
  meta: {
    name: "cleanup",
    description: "清理過期 session"
  },
  async run() {
    const result = await db.sessions.deleteMany({
      where: { expiresAt: { lt: new Date() } }
    });
    return { result: `deleted ${result.count} sessions` };
  }
});
```

📄 **寫進程式檔**：`nuxt.config.ts`

```text
export default defineNuxtConfig({
  nitro: {
    experimental: { tasks: true },
    scheduledTasks: {
      "0 * * * *": ["cleanup"]   // 每小時整點
    }
  }
});
```

**注意**：scheduledTasks 在 Vercel 等 serverless 平台不會自動跑（沒有 long-running server）、要用 cron-job.org 或平台自己的 cron 觸發 `/_nitro/tasks/cleanup`。

  
### 8. 串流 streaming（AI 場景）

📄 **寫進程式檔**：`server/api/chat.post.ts`

```text
export default defineEventHandler(async (event) => {
  const { messages } = await readBody(event);

  setHeader(event, "Content-Type", "text/event-stream");

  const stream = new ReadableStream({
    async start(controller) {
      const aiStream = await openai.chat.completions.create({
        model: "gpt-4o-mini",
        messages,
        stream: true
      });

      for await (const chunk of aiStream) {
        const text = chunk.choices[0]?.delta?.content ?? "";
        controller.enqueue(new TextEncoder().encode(text));
      }

      controller.close();
    }
  });

  return sendStream(event, stream);
});
```

  
### 9. webhook 接收

📄 **寫進程式檔**：`server/api/webhook/stripe.post.ts`

```text
import Stripe from "stripe";

const stripe = new Stripe(process.env.STRIPE_SECRET_KEY!);

export default defineEventHandler(async (event) => {
  const body = await readRawBody(event);  // 注意：raw、不是 JSON parse
  const sig = getHeader(event, "stripe-signature")!;

  let stripeEvent;
  try {
    stripeEvent = stripe.webhooks.constructEvent(
      body!, sig, process.env.STRIPE_WEBHOOK_SECRET!
    );
  } catch (err) {
    throw createError({ statusCode: 400, statusMessage: "Invalid signature" });
  }

  if (stripeEvent.type === "checkout.session.completed") {
    const session = stripeEvent.data.object;
    await db.orders.update({
      where: { id: session.metadata.orderId },
      data: { paid: true }
    });
  }

  return { received: true };
});
```

  
### 10. 跟 Next.js API Route 對照

```text
| 特性          | Nuxt server/api      | Next.js app/api      |
|---------------|----------------------|----------------------|
| 路徑          | server/api/x.METHOD.ts | app/api/x/route.ts   |
| 動態          | [id].get.ts          | [id]/route.ts        |
| 工具          | defineEventHandler    | export GET / POST    |
| 讀 body       | readBody(event)        | await req.json()      |
| 讀 query      | getQuery(event)        | req.nextUrl.searchParams |
| 拋錯          | createError({...})     | NextResponse.json(..., 4xx) |
| 自動 import 工具 | ✓                    | ✗ 手動               |
| Server Action| ✗（用 $fetch 取代）   | ✓ "use server"        |
```

  
### 5 大常見坑

```text
❌ 1. 檔名寫 users.ts（沒 method 後綴）
   → 預設接 GET、但其他 method 進不來
   → 修：users.get.ts / users.post.ts

❌ 2. 在 server/api 內用 ref / reactive
   → server 沒這些（不是 Vue 環境）
   → 修：純 JS object、return 給 client

❌ 3. middleware 沒判斷路徑、全 API 都跑
   → 拖慢全部
   → 修：startsWith("/api/admin") 限制範圍

❌ 4. 直接讀 process.env.SECRET 沒驗
   → 漏設變數整個壞
   → 修：用 runtimeConfig + zod 驗證

❌ 5. webhook 用 readBody（已 JSON parse）
   → Stripe signature 驗失敗
   → 修：用 readRawBody 拿原始字串
```

  
### 🎯 重點回顧

- **檔名 = 路由**：`users.get.ts` → GET /api/users
- **defineEventHandler**：包 handler、event 用 Nitro 工具讀寫
- **server/utils/**：自動 import 跨 endpoint 用
- **server/middleware/**：每個 API 都跑、自己 startsWith 判斷
- **server/tasks/**：cron / scheduled、experimental
- **streaming**：sendStream + ReadableStream
