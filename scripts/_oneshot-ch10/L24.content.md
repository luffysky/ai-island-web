**Next.js 15** 帶來四大改變：**Partial Pre-Rendering 穩定**、**React 19 完整整合**、**Turbopack 預設**、**params / cookies 變 async**。學完這課你跟得上業界最新標準。

> 💡 **這課的價值**：Next.js 15 是 2024 年底發布、2026 年的線上專案多半已升級。**沒跟上會看不懂新 code、面試也答不出來**。這課把所有新 feature 一次過。

  
### 1. Partial Pre-Rendering（PPR、部分預渲染）

「**PPR**（Partial Pre-Rendering、部分預先渲染、Next 15 殺手鐧）」= 「**同一個頁面、靜態部分預先 build、動態部分 streaming**」。SSG 跟 SSR 不再二選一、可以混。

📄 **寫進程式檔**：`next.config.js`

```text
module.exports = {
  experimental: {
    ppr: "incremental"
  }
};
```

📄 **寫進程式檔**：`app/page.tsx`

```text
import { Suspense } from "react";

export const experimental_ppr = true;

export default function HomePage() {
  return (
    <>
      {/* ↓ 靜態：build 時預生、CDN 直出 */}
      <Hero />
      <Features />
      <Pricing />

      {/* ↓ 動態：每個請求 streaming 進來 */}
      <Suspense fallback={<UserSkeleton />}>
        <UserGreeting />  {/* 讀 cookie、個人化 */}
      </Suspense>

      <Suspense fallback={<CartSkeleton />}>
        <CartCount />     {/* 讀 DB */}
      </Suspense>

      <Footer />
    </>
  );
}
```

**結果**：
- 訪客打開首頁、**立刻**拿到 CDN 上的 HTML（含 Hero / Features / Pricing / Footer / 兩個 skeleton）
- 個人化部分（greeting / cart）**streaming** 進來
- 體感：「**全靜態頁的快**」+「**動態頁的個人化**」

> 💡 **白話**：「**頁面的 80% 是靜態 + 20% 動態、各取所長**」——這就是 PPR。

  
### 2. params / cookies / headers 變 async

Next 15 把這些 API 改成 Promise（要 await）：

📄 **舊（Next 14）**

```text
export default function Page({ params }: { params: { slug: string } }) {
  const slug = params.slug;
  // ...
}

import { cookies } from "next/headers";
const cookieStore = cookies();
const token = cookieStore.get("token");
```

📄 **新（Next 15）**

```text
export default async function Page(
  { params }: { params: Promise<{ slug: string }> }
) {
  const { slug } = await params;
  // ...
}

import { cookies } from "next/headers";
const cookieStore = await cookies();
const token = cookieStore.get("token");
```

**為什麼改？** 為了 PPR——這些 API 「讀到時才算動態」、用 await 讓 Next.js 精準決定哪段要 stream。

🖥️ **官方 codemod 幫升級**

```text
npx @next/codemod@canary next-async-request-api .
```

  
### 3. React 19 整合

Next.js 15 預設用 React 19、有這些新東西：

```text
✅ React Compiler（自動 memoization）
   → 不用手寫 useMemo / useCallback
   → 編譯器自動優化

✅ 新 hook：useFormStatus / useFormState / useOptimistic
   → Server Action 表單完整支援

✅ use() API
   → 條件式讀 Promise / Context
   → const data = use(promise);

✅ Document Metadata（在元件內寫 <title>、自動 hoist 到 head）
   → 跟 metadata 並存

✅ ref 變成普通 prop（function component）
   → 不用再 forwardRef
```

  
### 4. Turbopack 預設（dev）

「**Turbopack**（Next.js 自家的 bundler、用 Rust 寫、比 Webpack 快 10 倍）」在 Next 15 變成 dev 預設：

🖥️ **跑 dev**

```text
npm run dev
# 自動用 Turbopack（不用加 --turbo）
```

💬 **速度差異**

```text
冷啟動（cold start）：
  Webpack：8 秒
  Turbopack：1 秒

HMR（修改後 reload）：
  Webpack：500-2000ms
  Turbopack：50-200ms
```

**production build** 還是用 Webpack（Turbopack production 還在 beta）。

  
### 5. 完整 Next 15 page 範例

📄 **寫進程式檔**：`app/products/[id]/page.tsx`

```text
import { Suspense } from "react";
import { cookies } from "next/headers";

type Props = { params: Promise<{ id: string }> };

export const experimental_ppr = true;

// 動態 metadata
export async function generateMetadata({ params }: Props) {
  const { id } = await params;
  const product = await getProduct(id);
  return {
    title: product.name,
    description: product.description,
    openGraph: { images: [product.image] }
  };
}

// SSG 靜態路徑
export async function generateStaticParams() {
  const products = await getAllProducts();
  return products.map(p => ({ id: p.id }));
}

export default async function ProductPage({ params }: Props) {
  const { id } = await params;
  const product = await getProduct(id);

  return (
    <article>
      {/* 靜態部分 */}
      <h1>{product.name}</h1>
      <Image src={product.image} alt={product.name} width={800} height={600} priority />
      <p>{product.description}</p>
      <p>NT$ {product.price}</p>

      {/* 動態部分（PPR streaming） */}
      <Suspense fallback={<div>載入庫存中...</div>}>
        <StockStatus productId={id} />
      </Suspense>

      <Suspense fallback={<div>載入推薦中...</div>}>
        <RelatedProducts productId={id} />
      </Suspense>

      {/* 個人化（讀 cookie） */}
      <Suspense fallback={null}>
        <PersonalRecommendation />
      </Suspense>
    </article>
  );
}

async function PersonalRecommendation() {
  const cookieStore = await cookies();
  const userId = cookieStore.get("user-id")?.value;
  if (!userId) return null;

  const recs = await getRecommendations(userId);
  return <RelatedList recs={recs} />;
}
```

  
### 6. fetch 預設變動

Next 14：`fetch` 預設 `cache: "force-cache"`。
Next 15：`fetch` 預設 `cache: "no-store"`（要 cache 明確寫）。

📄 **新寫法**

```text
// 明確寫要 cache
await fetch(url, { cache: "force-cache" });
await fetch(url, { next: { revalidate: 3600 } });

// 預設不 cache（明確即時）
await fetch(url);  // = no-store
```

**為什麼改？** 太多新手不知道有 cache、後台改了沒更新——預設不 cache 比較直觀。

  
### 7. 新的 staleTimes（Router Cache）

Next 15 提供更細的 client-side cache 控制：

📄 **寫進程式檔**：`next.config.js`

```text

  experimental: {
    staleTimes: {
      dynamic: 30,    // 動態頁 30 秒
      static: 180     // 靜態頁 180 秒
    }
  }
};
```

  
### 8. unstable_after：回應後執行

「**unstable_after**（Next 15 新 API、回應送出後執行非緊急任務）」——例如記錄 analytics、不影響 TTFB：

📄 **寫進程式檔**：`app/api/order/route.ts`

```text
import { unstable_after as after } from "next/server";

export async function POST(req: Request) {
  const order = await createOrder(await req.json());

  // 回應「立刻」送給使用者
  // 下面這段在背景跑、不影響 response time
  after(async () => {
    await sendOrderEmail(order);
    await trackAnalytics("order_created", order.id);
    await notifyDiscord(order);
  });

  return Response.json(order);
}
```

### 11. Next 15 vs Next 14 簡表

```text
| 特性                 | Next 14         | Next 15           |
|----------------------|-----------------|-------------------|
| React 版本           | 18              | 19                |
| dev bundler          | Webpack         | Turbopack         |
| params / cookies     | sync            | async（要 await） |
| fetch 預設           | force-cache     | no-store          |
| PPR                  | experimental    | stable（incremental） |
| React Compiler       | 無              | 整合中             |
| forwardRef           | 必須            | 可省               |
| unstable_after       | 無              | 有                 |
```

  
### 5 大常見坑

```text
❌ 1. 升 Next 15 但 params 沒 await
   → TS 不報、runtime 拿到 Promise
   → 修：永遠 const { x } = await params

❌ 2. 升 Next 15 後 fetch 變慢 / API 帳單爆
   → 預設變 no-store、每次都打
   → 修：靜態的明確 force-cache 或 revalidate

❌ 3. PPR 沒包 Suspense、變整頁動態
   → 沒效果
   → 修：動態部分必包 Suspense

❌ 4. Turbopack 不支援某個 webpack plugin
   → dev 跑不起來
   → 修：dev 加回 --no-turbo、production 還是 webpack

❌ 5. React Compiler 跟手寫 useMemo 衝突
   → 雙重 memo、有 bug
   → 修：信任 compiler、移除手動 useMemo
```
