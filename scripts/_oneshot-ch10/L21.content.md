### 3. 7 個情境推薦

```text
1. 美國團隊 / FAANG / 矽谷新創求職
   → Next.js（全球職缺 5-10 倍）

2. 台灣本土 SaaS 公司 / 日本 / 中國市場
   → Nuxt 3 也是好選擇（生態豐、UI 庫多）
   → Next.js 也找得到工作

3. 個人專案、極簡上手
   → Nuxt 3（自動 import、模組生態、上手快）

4. 追求最新 RSC / Server Action 體驗
   → Next.js（領先）

5. 需要 Vercel 親兒子整合
   → Next.js（部署 0 設定）

6. 需要部署到 Cloudflare Workers
   → Nuxt 3（Nitro 內建 preset）

7. 寫部落格 / 文件站
   → Nuxt 3 + @nuxt/content（最快）
   → Next.js + MDX 也行
```

  
### 4. 兩個的「設計哲學」差異

```text
Next.js：「Convention over configuration、慣例優先」
   • 你 follow 慣例（檔名、structure）
   • 強推他們的方式（RSC、Server Action）
   • 跟 Vercel 高度整合
   • React 生態（生態最大、職缺最多）

Nuxt 3：「Convention + Auto-magic、慣例 + 自動化」
   • 自動 import（少打字）
   • 模組系統（一行加功能）
   • 部署平台中立（Nitro）
   • Vue 生態（簡潔、亞洲熱門）
```

  
### 5. 心智模型 80% 相同

雖然兩個是不同框架、但**底層心智模型 80% 相同**：

```text
✅ 檔案路由（資料夾結構 = URL）
✅ SSR / SSG / ISR / CSR 多種渲染
✅ middleware 攔截請求
✅ 內建 API endpoints
✅ 圖片字型優化
✅ SEO metadata
✅ 部署到 Vercel / Cloudflare
✅ 環境變數 / runtime config
✅ 國際化
✅ 認證模式
✅ Streaming SSR
✅ TypeScript 強支援
```

差異只在「**底層 React vs Vue、寫法不同、API 名不同**」——會其中一個轉另一個**一週內可上手**。

  
### 6. 兩個的「殺手鐧特性」對比

```text
Next.js 獨家強項：
  🚀 React Server Components（最成熟）
  🚀 Server Actions（"use server"）
  🚀 Partial Pre-Rendering（Next 15）
  🚀 Turbopack（Webpack 後繼）
  🚀 Vercel Edge Functions 一鍵
  🚀 React 19 整合（Form / useFormState）

Nuxt 3 獨家強項：
  ⚡ 自動 import（少打 100 行 import）
  ⚡ 模組生態（一行加功能）
  ⚡ routeRules 集中管理
  ⚡ Nitro 跨平台部署
  ⚡ @nuxt/content 檔案系統 CMS
  ⚡ useSeoMeta（reactive SEO）
```

### 9. 第三個選擇：Astro？

「**Astro（內容導向的 meta-framework、優先 SSG）**」是新興選擇——適合「**內容站**」（部落格、文件、行銷頁）：

```text
Astro 強項：
  ✅ 預設 0 JS（純 HTML、最快）
  ✅ 支援 React + Vue + Svelte 同時
  ✅ Content collections（type-safe md / mdx）
  ✅ Lighthouse 100 分容易

Astro 弱項：
  ❌ 不適合互動重的 SaaS
  ❌ SSR 起步晚（追不上 Next / Nuxt）
  ❌ 生態系小
```

**建議**：純內容站考慮 Astro、互動重的 SaaS 還是 Next / Nuxt。
