**Parallel Routes（平行路由、同個 layout 內同時渲染多個獨立路由）** 跟 **Intercepting Routes（攔截路由、用 modal 顯示另一個路由的內容）** 是 Next.js 14+ 兩個進階路由特性。**讓某些 UI pattern 變得超簡單**——dashboard 多區、modal as URL 都靠它。

> 💡 **這課的價值**：Instagram、Twitter 點圖開 modal（網址也變了）、複製貼別人卻是完整頁——這效果以前要寫一堆 client state、現在 Intercepting Routes 兩個檔案搞定。

  
### 1. Parallel Routes：@slot 同時渲染多個

「**Slot（插槽）**」是「**一個 layout 內可以塞多個獨立路由的位置**」。資料夾用 `@` 開頭：

```text
app/
└── dashboard/
    ├── layout.tsx
    ├── page.tsx
    ├── @analytics/        ← slot：analytics
    │   └── page.tsx
    └── @team/             ← slot：team
        └── page.tsx
```

📄 **寫進程式檔**：`app/dashboard/layout.tsx`

```text
export default function Layout({
  children,
  analytics,
  team
}: {
  children: React.ReactNode;
  analytics: React.ReactNode;
  team: React.ReactNode;
}) {
  return (
    <div className="grid grid-cols-2 gap-4">
      <div className="col-span-2">{children}</div>
      <div>{analytics}</div>
      <div>{team}</div>
    </div>
  );
}
```

📄 **寫進程式檔**：`app/dashboard/@analytics/page.tsx`

```text
export default async function Analytics() {
  const stats = await getAnalytics();
  return <div>本月訪問：{stats.visits}</div>;
}
```

📄 **寫進程式檔**：`app/dashboard/@team/page.tsx`

```text
export default async function Team() {
  const members = await getTeam();
  return <ul>{members.map(m => <li key={m.id}>{m.name}</li>)}</ul>;
}
```

**結果**：訪問 `/dashboard` 時、layout 同時收到 `children`（page.tsx）、`analytics`、`team` 三個渲染好的 React tree——三個都**獨立載入、獨立錯誤邊界**。

  
### 2. Slot 各自有自己的 loading / error

```text
app/dashboard/
├── layout.tsx
├── page.tsx
├── @analytics/
│   ├── page.tsx
│   ├── loading.tsx    ← 只給 analytics 的 loading
│   └── error.tsx      ← 只給 analytics 的 error
└── @team/
    ├── page.tsx
    ├── loading.tsx
    └── error.tsx
```

**好處**：analytics 抓資料慢、不會卡到 team 顯示。team 出錯、不會讓 analytics 整個掛。

  
### 3. Slot 條件渲染

slot 是 props、可以條件顯示：

```text
export default function Layout({ children, analytics, admin }) {
  const session = await auth();

  return (
    <>
      {children}
      <aside>{analytics}</aside>
      {session?.user.role === "admin" && admin}  {/* 只 admin 看到 */}
    </>
  );
}
```

  
### 4. Intercepting Routes：(.) (..) (...)

「**Intercept（攔截）**」是「**不離開現在頁面、顯示另一個路由的內容**」。經典用途：**modal as URL**。

```text
資料夾命名：
  (.)foo       ← 同層攔截
  (..)foo      ← 上一層攔截
  (..)(..)foo  ← 上兩層
  (...)foo     ← 從 root 攔截
```

  
### 5. 實戰：Instagram 風的照片 modal

**目標**：
- 訪問 `/feed` 看照片牆
- 點某張照片 → URL 變 `/photo/123` 但顯示為 **modal**（背景還是 feed）
- 複製網址貼給朋友 → 朋友打開看到 `/photo/123` **完整頁**（沒 modal）

```text
app/
├── layout.tsx
├── page.tsx              ← /  feed
├── @modal/
│   ├── default.tsx       ← slot 預設（沒 modal 時）
│   └── (.)photo/
│       └── [id]/
│           └── page.tsx  ← 攔截 /photo/:id 顯示為 modal
├── photo/
│   └── [id]/
│       └── page.tsx      ← 直接訪問 /photo/:id 顯示完整頁
```

📄 **寫進程式檔**：`app/layout.tsx`

```text
export default function Layout({ children, modal }) {
  return (
    <html>
      <body>
        {children}
        {modal}  {/* modal slot */}
      </body>
    </html>
  );
}
```

📄 **寫進程式檔**：`app/@modal/default.tsx`

```text
// 沒攔截時、什麼都不渲染
export default function Default() {
  return null;
}
```

📄 **寫進程式檔**：`app/@modal/(.)photo/[id]/page.tsx`

```text
import Modal from "./modal";

export default async function PhotoModal({ params }) {
  const { id } = await params;
  const photo = await getPhoto(id);
  return (
    <Modal>
      <img src={photo.url} alt={photo.alt} />
      <p>{photo.caption}</p>
    </Modal>
  );
}
```

📄 **寫進程式檔**：`app/photo/[id]/page.tsx`（完整頁版本）

```text
export default async function PhotoFull({ params }) {
  const { id } = await params;
  const photo = await getPhoto(id);
  return (
    <article>
      <img src={photo.url} alt={photo.alt} />
      <h1>{photo.caption}</h1>
      <CommentList photoId={id} />
    </article>
  );
}
```

📄 **寫進程式檔**：`app/feed/page.tsx`

```text
import Link from "next/link";

export default async function Feed() {
  const photos = await getPhotos();
  return (
    <div className="grid grid-cols-3 gap-2">
      {photos.map(p => (
        <Link key={p.id} href={`/photo/${p.id}`}>
          <img src={p.thumbUrl} alt={p.alt} />
        </Link>
      ))}
    </div>
  );
}
```

**結果**：
- 在 feed 點圖 → URL 變 `/photo/123`、但顯示 modal、背景是 feed
- 直接打開 `/photo/123` → 完整頁
- 同一個 URL、兩種顯示——這就是 Instagram、Twitter、Pinterest 的招式

  
### 6. 為什麼這比 useState 開 modal 好？

```text
傳統 client state modal：
  ❌ URL 不變、不能分享
  ❌ 上一頁鍵不能關 modal
  ❌ 重整 modal 沒了
  ❌ SEO 抓不到

Intercepting Routes modal：
  ✅ URL 變、可分享
  ✅ 上一頁鍵自動關 modal
  ✅ 重整變完整頁（SEO 友善）
  ✅ Server Component、可直接讀 DB
```

  
### 7. Parallel Routes + Intercepting 經典用途

```text
🟢 多區 dashboard
   → @analytics / @team / @sales 各區獨立載入

🟢 modal as URL
   → 照片、登入、設定 modal、結帳 modal

🟢 Tabs
   → @overview / @stats / @members 各 tab 獨立路由

🟢 條件 UI
   → @admin（只有 admin 看到）

🟢 漸進披露
   → @intro / @details / @advanced
```

  
### 5 大常見坑

```text
❌ 1. 忘記寫 default.tsx
   → 訪問 /feed 時 @modal 沒對應 page、Next.js 報錯
   → 修：每個 slot 必有 default.tsx（可 return null）

❌ 2. Intercepting 路徑層級寫錯
   → (.) (..) (...) 算錯層、不會攔截
   → 修：從 @ 資料夾算回去、不是從 app/ 算

❌ 3. Parallel slot 名稱跟現有路由衝突
   → @photo 跟 photo/ 同時存在、行為怪
   → 修：slot 名跟路由名分開

❌ 4. 在 modal 內用 useRouter.push 切換
   → modal 不會關
   → 修：用 router.back() 或 Link

❌ 5. modal 內元件用 useEffect 抓資料
   → 不需要、可用 async Server Component
   → 修：直接寫 async function、await 抓
```

  
### 🎯 重點回顧

- **Parallel Routes**：`@slot` 資料夾、layout 收 slot props、獨立載入 / 錯誤邊界
- **Intercepting Routes**：`(.) (..) (...)` 攔截別的路由、不離開現在頁
- **modal as URL**：分享網址、SEO、上一頁全友善
- **每個 slot 要 default.tsx**：沒攔截時的 fallback
- **經典用途**：多區 dashboard / 照片 modal / 結帳 modal / tabs
