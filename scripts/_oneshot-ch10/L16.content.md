**Nuxt 3** 是「**Vue 生態系的 Next.js**」——建在 Vue 3 上、補齊路由 / SSR / API / 部署所有缺。底層用 **Nitro（Nuxt 自家的 server 引擎、跨平台極快）**。**亞洲（台灣、日本、中國）市佔最高的元框架**。

> 💡 **這課的價值**：如果你會 Vue、Nuxt 是學完馬上就能用。**自動 import、檔案結構、模組生態系**——很多細節比 Next.js 更直覺。AI 島實際看的求職市場、台灣 Nuxt 職缺也很多。

### 2. 完整檔案結構（要看懂、不用背）

```text
my-app/
├── nuxt.config.ts          ← 全局設定
├── app.vue                  ← 根元件（取代 Next 的 layout.tsx）
├── pages/                   ← 自動檔案路由
│   ├── index.vue            → /
│   ├── about.vue            → /about
│   └── blog/
│       ├── index.vue        → /blog
│       └── [slug].vue       → /blog/:slug
├── layouts/                 ← 共用 layout
│   ├── default.vue
│   └── admin.vue
├── components/              ← 自動 import！（不用手寫 import）
│   ├── MyButton.vue
│   └── PostCard.vue
├── composables/             ← 自動 import！
│   ├── useCounter.ts
│   └── useAuth.ts
├── server/
│   ├── api/
│   │   ├── users.get.ts     → GET /api/users
│   │   └── users.post.ts    → POST /api/users
│   ├── middleware/
│   └── plugins/
├── middleware/              ← 路由 middleware（不是 server）
│   └── auth.global.ts
├── plugins/                 ← Nuxt 啟動時跑（Pinia、i18n 等）
├── public/                  ← 靜態檔（不處理）
├── assets/                  ← 處理過的資源（CSS、SCSS、字型）
└── stores/                  ← Pinia store（自動 import）
```

  
### 3. 自動 import：Nuxt 殺手鐧

放在 `components/`、`composables/`、`utils/` 的東西**不用 import 就能用**：

📄 **寫進程式檔**：`composables/useCounter.ts`

```text
export const useCounter = () => {
  const count = ref(0);
  const increment = () => count.value++;
  return { count, increment };
};
```

📄 **寫進程式檔**：`pages/index.vue`

```text
<script setup lang="ts">
// 不用寫 import！全部自動
const count = ref(0);                       // 自動 import vue
const route = useRoute();                   // 自動 import 路由
const { data } = await useFetch("/api/users");  // Nuxt 內建 composable
const counter = useCounter();               // 你自己寫的 composable
</script>

<template>
  <div>
    <h1>計數器：{{ counter.count }}</h1>
    <button @click="counter.increment">+1</button>
  </div>
</template>
```

> 💡 **白話**：「**Nuxt 幫你自動加 import**」——寫起來像沒 import 語法的程式。IDE 自動補全還是有、執行時也不會找不到。

  
### 4. app.vue 根元件

📄 **寫進程式檔**：`app.vue`

```text
<template>
  <div>
    <NuxtLayout>
      <NuxtPage />
    </NuxtLayout>
  </div>
</template>
```

**作用**：
- `<NuxtLayout>` = 套用 `layouts/default.vue`（或指定 layout）
- `<NuxtPage>` = 渲染當前路由的 page

跟 Next.js 對比：`app.vue` ≈ `app/layout.tsx`、`pages/index.vue` ≈ `app/page.tsx`。

### 6. 第一個 page

📄 **寫進程式檔**：`pages/blog/[slug].vue`

```text
<script setup lang="ts">
const route = useRoute();
const slug = route.params.slug;

// useFetch：SSR 友善、自動 cache（L17 詳教）
const { data: post } = await useFetch(`/api/posts/${slug}`);

// SEO metadata
useSeoMeta({
  title: post.value?.title,
  description: post.value?.excerpt,
  ogImage: post.value?.coverImage,
  twitterCard: "summary_large_image"
});
</script>

<template>
  <article>
    <h1>{{ post.title }}</h1>
    <div v-html="post.html" />
  </article>
</template>
```

  
### 7. nuxt.config.ts：全局設定

📄 **寫進程式檔**：`nuxt.config.ts`

```text
export default defineNuxtConfig({
  // SSR / SPA 切換
  ssr: true,

  // 模組（L20 詳教）
  modules: [
    "@nuxt/image",
    "@nuxtjs/tailwindcss",
    "@pinia/nuxt"
  ],

  // 環境變數（runtime 用）
  runtimeConfig: {
    // 只 server 看得到
    openaiKey: process.env.OPENAI_KEY,
    // public 開頭：client 也看得到
    public: {
      apiBase: process.env.API_BASE || "/api"
    }
  },

  // CSS
  css: ["~/assets/styles/main.css"],

  // 渲染策略（L19 詳教）
  routeRules: {
    "/": { isr: 3600 },
    "/blog/**": { prerender: true },
    "/admin/**": { ssr: false }
  },

  // SEO
  app: {
    head: {
      title: "AI 島",
      meta: [
        { name: "description", content: "..." }
      ]
    }
  }
});
```

  
### 8. Nitro：Nuxt 自家 server 引擎

「**Nitro**（拉丁文「快」、Nuxt 內建的伺服器引擎）」是 Nuxt 跑 server 的底層——同一份 code 可以部署到 **Node / Vercel / Cloudflare Workers / Netlify / Bun / Deno**、自動轉換。

```text
你寫 nuxt.config.ts → routeRules / server/api
   ↓
Nitro 自動產對應平台的部署檔
   ├── Vercel  → .vercel/output/
   ├── Cloudflare → dist/
   ├── Netlify → .netlify/
   └── Node   → .output/server/
```

**好處**：寫一次、部署到任何地方、零設定。Next.js 強綁 Vercel（其他平台要 adapter）、Nuxt 平等對待所有平台。

  
### 9. Nuxt 跟 Next.js 對照

```text
| 項目          | Nuxt 3              | Next.js              |
|---------------|---------------------|----------------------|
| 底層           | Vue 3                | React                 |
| 路由           | pages/ 檔案路由      | app/ 檔案路由         |
| Layout         | layouts/ + NuxtLayout | layout.tsx 巢狀       |
| 自動 import    | ✓                    | ✗                    |
| 資料抓取       | useFetch / useAsyncData | fetch in Server Comp |
| SEO            | useSeoMeta            | export metadata       |
| 圖片           | <NuxtImg>             | <Image>               |
| 字型           | @nuxtjs/google-fonts  | next/font             |
| API           | server/api/*.ts        | app/api/*/route.ts    |
| Hybrid 渲染    | routeRules（單一檔）   | export per route       |
| 模組系統       | ✓（@nuxt/*）          | ✗                    |
```

  
### 5 大常見坑

```text
❌ 1. 寫 import 進來、結果跟自動 import 衝突
   → 同名 component 兩個來源、報錯
   → 修：信任自動 import、不要手寫

❌ 2. server/api 跟 middleware/ 搞混
   → server/middleware = server 端的（每個 API 都跑）
   → middleware/ = 路由 middleware（換頁時跑）
   → 修：搞清楚層級

❌ 3. composables 寫成 use 開頭以外
   → 自動 import 抓 `use` 開頭
   → 修：composable 一律 useXxx

❌ 4. layouts/default.vue 漏寫 <slot />
   → 整頁空白
   → 修：layout 一定要有 <slot />

❌ 5. Pinia store 不放 stores/
   → 自動 import 找不到
   → 修：放 stores/ 並用 useXxxStore 命名
```

  
### 🎯 重點回顧

- **Nuxt 3 = Vue 生態的 Next.js**
- **檔案結構**：pages / layouts / components / composables / server / middleware / plugins / stores
- **自動 import**：components / composables / utils / stores 不用寫 import
- **Layout 系統**：layouts/default.vue + definePageMeta 切換
- **Nitro**：底層 server 引擎、跨平台部署零設定
- **nuxt.config.ts**：所有設定中心、modules / routeRules / runtimeConfig
