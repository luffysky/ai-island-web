Nuxt 抓資料有 **三兄弟**：`useFetch`、`useAsyncData`、`$fetch`。三個都能打 API、**但用錯場景會 SSR 雙打、cache 失效、或重複請求**。

> 💡 **這課的價值**：Nuxt 新手最容易卡住的就是這三個分不清。學完這課你就知道「**什麼時候用哪個**」、Nuxt SSR 自動 cache 一次到位。

  
### 1. useFetch：最常用、首選

📄 **寫進程式檔**：`pages/users.vue`

```text
<script setup lang="ts">
// SSR + Client 都會跑、自動 cache 避免 hydration 後重打
const { data, pending, error, refresh } = await useFetch("/api/users");

// 帶選項
const { data: users } = await useFetch("/api/users", {
  method: "POST",
  body: { name: "招財" },
  headers: { "X-Custom": "value" },
  query: { page: 1 },

  // 只在 client 跑（SSR 跳過、給「不想 SSR 抓」的資料用）
  server: false,

  // lazy：不等抓完就渲染（pending 顯示 loading）
  lazy: true,

  // 預設 cache key 是 url、可自訂
  key: "users-list",

  // transform 資料
  transform: (data) => data.users.map(u => ({ ...u, name: u.name.toUpperCase() })),

  // 監聽 reactive 變化、自動重抓
  watch: [page]
});
</script>

<template>
  <div>
    <p v-if="pending">載入中...</p>
    <p v-else-if="error">出錯：{{ error.message }}</p>
    <ul v-else>
      <li v-for="u in users" :key="u.id">{{ u.name }}</li>
    </ul>
    <button @click="refresh">重新整理</button>
  </div>
</template>
```

**重點特性**：
- **SSR 友善**：server 跑一次、把資料 inline 進 HTML、client hydration 直接用、不重打
- **自動 cache**：同個 key 多次呼叫共用
- **reactive**：data 是 ref、改了畫面自動更新

  
### 2. useAsyncData：更彈性的版本

當你不只是「打一個 URL」、而是「組合多個資料」：

📄 **寫進程式檔**：`pages/dashboard.vue`

```text
<script setup>
const { data, pending } = await useAsyncData("dashboard-data", async () => {
  // 任何 async 邏輯都能 cache
  const users = await $fetch("/api/users");
  const stats = await $fetch("/api/stats");
  const orders = await $fetch("/api/orders");

  return {
    users,
    stats,
    orders,
    summary: `共 ${users.length} 用戶、${orders.length} 訂單`
  };
});
</script>

<template>
  <div v-if="data">
    <p>{{ data.summary }}</p>
    <UserList :users="data.users" />
    <StatsCard :stats="data.stats" />
  </div>
</template>
```

> 💡 **白話**：「**useFetch = 簡單版（只能打單一 URL）；useAsyncData = 進階版（裡面能寫任何 async）**」。

  
### 3. $fetch：簡單版、不自動 SSR cache

`$fetch` 是底層的 HTTP client、適合「**只執行一次、不要 cache**」的操作：

📄 **寫進程式檔**：`pages/posts/new.vue`

```text
<script setup>
const title = ref("");
const content = ref("");

// 給 onClick、表單提交、mutation 用
async function handleSubmit() {
  const r = await $fetch("/api/posts", {
    method: "POST",
    body: { title: title.value, content: content.value }
  });
  // 不會自動 cache、每次都打
  navigateTo(`/posts/${r.id}`);
}
</script>

<template>
  <form @submit.prevent="handleSubmit">
    <input v-model="title" />
    <textarea v-model="content" />
    <button>發布</button>
  </form>
</template>
```

  
### 4. 三者對比表（必背）

```text
| 特性                   | useFetch       | useAsyncData    | $fetch          |
|------------------------|----------------|-----------------|-----------------|
| 用途                   | 抓一個 URL     | 組合多個資料     | 一次性 HTTP 呼叫 |
| SSR 友善（自動 hydrate）| ✓              | ✓               | ✗               |
| 自動 cache             | ✓              | ✓               | ✗               |
| 自動去重（同 key）      | ✓              | ✓               | ✗               |
| 用在 setup 外          | ✗              | ✗               | ✓               |
| 適合場景               | GET 列表 / 頁面 | 多 source 組合   | POST / DELETE / event handler |
| reactive               | data 是 ref    | data 是 ref     | 普通 promise    |
```

  
### 5. 什麼時候用哪個（決策樹）

```text
要在 setup 內、SSR 時就要抓嗎？
   ├─ 是 → 只抓一個 URL → useFetch
   │     └─ 要組合多個 → useAsyncData
   └─ 否（在 event handler、Pinia action 等地方）
         → $fetch
```

**白話**：
- 進頁面就要抓 + SSR 要包進去 → `useFetch` / `useAsyncData`
- 使用者點按鈕才抓 → `$fetch`

  
### 6. SEO：useHead + useSeoMeta

📄 **寫進程式檔**：`pages/blog/[slug].vue`

```text
<script setup>
const route = useRoute();
const { data: post } = await useFetch(`/api/posts/${route.params.slug}`);

// 基本（彈性、控制 head 全部）
useHead({
  title: "AI 島",
  meta: [
    { name: "description", content: "..." }
  ]
});

// 更方便（針對 SEO + OG + Twitter）
useSeoMeta({
  title: () => post.value?.title,
  description: () => post.value?.excerpt,
  ogImage: () => post.value?.coverImage,
  ogTitle: () => post.value?.title,
  twitterCard: "summary_large_image"
});
</script>
```

**重點**：值可以是 **function**（reactive）——post 抓完後 metadata 自動更新。

  
### 7. 錯誤處理

```text
const { data, error } = await useFetch("/api/users");

// 客製化錯誤頁
if (error.value) {
  throw createError({
    statusCode: error.value.statusCode,
    statusMessage: error.value.statusMessage,
    fatal: true
  });
}

// 客製化重試
const { data } = await useFetch("/api/flaky", {
  retry: 3,
  retryDelay: 1000

```

  
### 8. 進階：interceptor（攔截器）

```text
const { data } = await useFetch("/api/users", {
  onRequest({ options }) {
    options.headers = options.headers || {};
    options.headers["Authorization"] = `Bearer ${useCookie("token").value}`;
  },
  onResponse({ response }) {
    console.log("收到資料：", response._data);
  },
  onResponseError({ response }) {
    if (response.status === 401) {
      navigateTo("/login");
    }
  }
});
```

  
### 9. 進階：global $fetch 客製化

📄 **寫進程式檔**：`plugins/api.ts`

```text
export default defineNuxtPlugin(() => {
  const config = useRuntimeConfig();

  const api = $fetch.create({
    baseURL: config.public.apiBase,
    onRequest({ options }) {
      const token = useCookie("token").value;
      if (token) {
        options.headers = {
          ...options.headers,
          Authorization: `Bearer ${token}`
        };
      }
    },
    onResponseError({ response }) {
      if (response.status === 401) {
        navigateTo("/login");
      }
    }
  });

  return {
    provide: { api }
  };
});

// 用法：const { $api } = useNuxtApp(); const data = await $api("/users");
```

  
### 5 大常見坑

```text
❌ 1. 在 onClick 用 useFetch
   → useFetch 只能在 setup 內、會壞
   → 修：event handler 用 $fetch

❌ 2. 兩個地方 useFetch 同個 url、沒設 key
   → 預設 key 是 url、會共用 cache
   → 兩個地方都拿到第一次的資料
   → 修：不同地方要不同 key 加 { key: "users-list-1" }

❌ 3. useFetch 後直接修改 data
   → data 是 ref 但內容是「response 結果」
   → 直接改會被下次 refresh 覆蓋
   → 修：用獨立的 ref / reactive 存修改

❌ 4. 沒 await useFetch
   → SSR 時不等資料、HTML 渲染空
   → 修：永遠 await（除非 lazy: true）

❌ 5. $fetch 後忘記更新 useFetch 的 cache
   → 新增完文章、列表還是舊的
   → 修：呼叫 refresh() 或 refreshNuxtData(key)
```

  
### 🎯 重點回顧

- **useFetch**：抓單一 URL、SSR 友善、自動 cache、最常用
- **useAsyncData**：抓多 source 組合、key + async fn
- **$fetch**：底層 client、不 cache、給 event handler 用
- **決策**：setup 內 + SSR → useFetch / useAsyncData；event handler → $fetch
- **SEO**：useSeoMeta + reactive function 自動更新
- **interceptor**：onRequest / onResponse / onResponseError 全攔截
