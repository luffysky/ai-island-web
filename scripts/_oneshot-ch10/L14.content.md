**i18n（Internationalization、國際化、讓網站支援多語言）** 是 SaaS 走出台灣的必修。Next.js 14+ 的 App Router 沒內建 i18n（Pages Router 有）、要自己用 middleware + 動態路由組——或裝 `next-intl`。

> 💡 **這課的價值**：英文版多 5 倍流量、日文版多打入日本市場——i18n 不是錦上添花、是新創走國際的關鍵。Notion、Linear、Vercel 都有完整 i18n。

  
### 1. URL 策略選擇

```text
策略 ①：子路徑（subpath）— 最常見
  https://example.com/en/about
  https://example.com/zh-Hant/about
  https://example.com/ja/about

策略 ②：子網域（subdomain）
  https://en.example.com/about
  https://zh.example.com/about

策略 ③：獨立網域
  https://example.com/about      ← 英文
  https://example.com.tw/about   ← 中文

策略 ④：query string（不推）
  https://example.com/about?lang=en
  → SEO 差、不要用
```

**建議**：99% 用「**策略 ① 子路徑**」——SEO 友善、實作簡單、Google 認得。

### 3. middleware：自動偵測 + redirect

📄 **寫進程式檔**：`middleware.ts`

```text
import { NextRequest, NextResponse } from "next/server";
import { match } from "@formatjs/intl-localematcher";
import Negotiator from "negotiator";

const locales = ["en", "zh-Hant", "ja"];
const defaultLocale = "en";

function getLocale(req: NextRequest): string {
  // 從 Accept-Language header 取使用者偏好
  const headers = { "accept-language": req.headers.get("accept-language") ?? "" };
  const languages = new Negotiator({ headers }).languages();
  return match(languages, locales, defaultLocale);
}

export function middleware(req: NextRequest) {
  const pathname = req.nextUrl.pathname;

  // 已經有 locale prefix 就放行
  const hasLocale = locales.some(
    l => pathname.startsWith(`/${l}/`) || pathname === `/${l}`
  );
  if (hasLocale) return;

  // 沒 locale、偵測使用者偏好、redirect
  const locale = getLocale(req);
  req.nextUrl.pathname = `/${locale}${pathname}`;
  return NextResponse.redirect(req.nextUrl);
}

export const config = {
  matcher: [
    "/((?!_next/static|_next/image|favicon.ico|api).*)"
  ]
};
```

🖥️ **裝套件**

```text
npm install @formatjs/intl-localematcher negotiator
npm install -D @types/negotiator
```

📄 **寫進程式檔**：`app/[lang]/dictionaries.ts`

```text
const dictionaries = {
  en: () => import("@/dictionaries/en.json").then(m => m.default),
  "zh-Hant": () => import("@/dictionaries/zh-Hant.json").then(m => m.default),
  ja: () => import("@/dictionaries/ja.json").then(m => m.default)
} as const;

export type Lang = keyof typeof dictionaries;

export const getDictionary = (locale: Lang) => dictionaries[locale]();
```

```text
import { getDictionary, Lang } from "./dictionaries";

export default async function Page({ params }: { params: Promise<{ lang: Lang }> }) {

  const dict = await getDictionary(lang);

  return (
    <main>
      <h1>{dict.welcome}</h1>
      <button>{dict.cta}</button>
    </main>
  );
}
```

### 5. 多語言 metadata + hreflang

📄 **寫進程式檔**：`app/[lang]/layout.tsx`

```text
import type { Metadata } from "next";

export async function generateMetadata({ params }): Promise<Metadata> {
  const { lang } = await params;
  const dict = await getDictionary(lang);

  return {
    title: dict.siteTitle,
    description: dict.siteDescription,
    alternates: {
      canonical: `https://example.com/${lang}`,
      languages: {
        "en": "https://example.com/en",
        "zh-Hant": "https://example.com/zh-Hant",
        "ja": "https://example.com/ja",
        "x-default": "https://example.com/en"
      }
    },
    openGraph: {
      locale: lang === "zh-Hant" ? "zh_TW" : lang === "ja" ? "ja_JP" : "en_US"
    }
  };
}

export default async function Layout({ children, params }) {
  const { lang } = await params;
  return (
    <html lang={lang}>
      <body>{children}</body>
    </html>
  );
}
```

**重點**：`<html lang="...">` 跟 metadata 的 `alternates.languages` 給 Google 看「**這頁有哪些語言版本**」、避免重複內容扣分、讓使用者看到對的語言版本。

  
### 6. 語言切換器

📄 **寫進程式檔**：`app/[lang]/language-switcher.tsx`

```text
"use client";
import { usePathname, useRouter } from "next/navigation";

const locales = [
  { code: "en", label: "English" },
  { code: "zh-Hant", label: "繁體中文" },
  { code: "ja", label: "日本語" }
];

export default function LanguageSwitcher({ current }: { current: string }) {
  const pathname = usePathname();
  const router = useRouter();

  function switchLang(code: string) {
    // 把 URL 第一段 (lang) 換掉
    const newPath = pathname.replace(/^\/[^/]+/, `/${code}`);
    router.push(newPath);
  }

  return (
    <select value={current} onChange={e => switchLang(e.target.value)}>
      {locales.map(l => (
        <option key={l.code} value={l.code}>{l.label}</option>
      ))}
    </select>
  );
}
```

  
### 7. 用 next-intl 套件（推薦、省事）

手動寫太麻煩、`next-intl` 是 App Router 最完整的 i18n 套件：

🖥️ **在終端機輸入**

```text
npm install next-intl
```

📄 **寫進程式檔**：`i18n.ts`

```text
import { notFound } from "next/navigation";
import { getRequestConfig } from "next-intl/server";

const locales = ["en", "zh-Hant", "ja"];

export default getRequestConfig(async ({ locale }) => {
  if (!locales.includes(locale)) notFound();
  return {
    messages: (await import(`./messages/${locale}.json`)).default
  };
});
```

📄 **元件使用**

```text
// Server Component
import { getTranslations } from "next-intl/server";

export default async function Page() {
  const t = await getTranslations("Home");
  return <h1>{t("welcome")}</h1>;
}

// Client Component
"use client";
import { useTranslations } from "next-intl";

export function Button() {
  const t = useTranslations("Home");
  return <button>{t("cta")}</button>;
}
```

### 9. RTL（從右到左、阿拉伯文 / 希伯來文）

```text
<html lang={lang} dir={lang === "ar" ? "rtl" : "ltr"}>
```

CSS 用 `margin-inline-start` 取代 `margin-left`、自動對 RTL 友善。

  
### 5 大常見坑

```text
❌ 1. 翻譯只翻 UI、忘了 metadata
   → Google 索引拿到 en 的 title、SEO 死
   → 修：generateMetadata 也要多語言

❌ 2. 忘記 hreflang
   → Google 不知道有其他語言版本、各語言不互換流量
   → 修：alternates.languages 必設

❌ 3. <html lang> 沒跟著切
   → 螢幕閱讀器讀錯腔
   → 修：根 layout 取 params.lang 動態設

❌ 4. 字典放在 client bundle
   → 用 import 而非動態 import、所有語言都進 bundle
   → 修：用 () => import(...) 動態

❌ 5. 日期 / 數字格式沒本地化
   → 美國 12/31/2026、台灣 2026/12/31、日本 2026年12月31日
   → 修：用 Intl.DateTimeFormat / NumberFormat
```

  
### 🎯 重點回顧

- **URL 策略**：99% 用子路徑 `/en/...`
- **檔案結構**：`app/[lang]/...` + middleware 偵測 + dictionaries
- **middleware**：根據 Accept-Language redirect 預設語言
- **動態 metadata**：alternates.languages + hreflang
- **next-intl**：最完整套件、強烈推薦
- **<html lang>** + **dir** 必設、無障礙 + RTL 友善
