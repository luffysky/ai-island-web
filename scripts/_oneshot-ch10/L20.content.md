**Nuxt Modules（Nuxt 模組、一行設定加完整功能的擴充套件）** 是 Nuxt 最強的生態系——一行 config 加圖片優化、Tailwind、Pinia、SEO、Supabase、auth 等。**省幾天設定時間**。

> 💡 **這課的價值**：純 Vue 整合 Tailwind 要設 PostCSS / autoprefixer / config 一堆——Nuxt 加一行 `@nuxtjs/tailwindcss` 自動全包。模組生態系是 Nuxt 比 Next.js 直覺的地方。

  
### 1. 加模組的基本方法

🖥️ **裝套件**

```text
npm install @nuxtjs/tailwindcss
```

📄 **寫進程式檔**：`nuxt.config.ts`

```text
export default defineNuxtConfig({
  modules: [
    "@nuxtjs/tailwindcss"  // 一行搞定
  ]
});
```

就這樣——Tailwind 完整設定好、能直接寫 class。

### 4. @nuxt/content：檔案系統 CMS

```text

├── blog/
│   ├── 1.first-post.md

│   └── 3.ai-tools-2026.md
```

📄 **寫進程式檔**：`content/blog/1.first-post.md`

```text
---
title: 第一篇文章
date: 2026-05-28
cover: /covers/1.jpg
tags: [nuxt, 教程]
---

這是文章內容、用 markdown 寫。

## 小標
- 列表
- 列表
```

📄 **使用**：`pages/blog/index.vue`

```text
<script setup>
const { data: posts } = await useAsyncData("posts", () =>
  queryContent("blog").sort({ date: -1 }).find()
);
</script>

<template>
  <ul>
    <li v-for="post in posts" :key="post._path">
      <NuxtLink :to="post._path">
        <h2>{{ post.title }}</h2>
        <small>{{ post.date }}</small>
      </NuxtLink>
    </li>
  </ul>
</template>
```

📄 **文章頁**：`pages/blog/[...slug].vue`

### 6. @pinia/nuxt：狀態管理

「**Pinia（Vue 官方推薦的狀態管理庫、Vuex 後繼者）**」+ Nuxt 整合：

📄 **寫進程式檔**：`stores/auth.ts`

```text
export const useAuthStore = defineStore("auth", () => {
  const user = ref(null);
  const isLoggedIn = computed(() => !!user.value);

  async function login(email: string, password: string) {
    const data = await $fetch("/api/auth/login", {
      method: "POST",
      body: { email, password }
    });
    user.value = data.user;
  }

  async function logout() {
    await $fetch("/api/auth/logout", { method: "POST" });
    user.value = null;
  }

  return { user, isLoggedIn, login, logout };
});
```

📄 **使用**：任何 component / page

```text
<script setup>
const auth = useAuthStore();  // 自動 import
</script>

<template>
  <div>
    <p v-if="auth.isLoggedIn">你好、{{ auth.user.email }}</p>
    <button @click="auth.logout">登出</button>
  </div>
</template>
```

### 8. @nuxtjs/i18n：國際化

📄 **寫進程式檔**：`nuxt.config.ts`

```text
export default defineNuxtConfig({
  modules: ["@nuxtjs/i18n"],
  i18n: {
    locales: [
      { code: "en", language: "en-US", file: "en.json" },
      { code: "zh-Hant", language: "zh-TW", file: "zh-Hant.json" },
      { code: "ja", language: "ja-JP", file: "ja.json" }
    ],
    defaultLocale: "en",
    strategy: "prefix_except_default"  // /about, /zh-Hant/about
  }
});
```

📄 **元件用**

```text
<script setup>
const { t, locale } = useI18n();
</script>

<template>
  <div>
    <h1>{{ t("welcome") }}</h1>
    <select v-model="locale">
      <option value="en">EN</option>
      <option value="zh-Hant">繁中</option>
      <option value="ja">JP</option>
    </select>
  </div>
</template>
```

### 5 大常見坑

```text
❌ 1. 裝模組沒加進 nuxt.config.ts modules
   → 安裝了但沒啟用
   → 修：modules 陣列加進去

❌ 2. 多個 UI 模組同時用（@nuxt/ui + Naive UI）
   → 樣式衝突、bundle 變大
   → 修：選一個就好

❌ 3. Pinia store 不放 stores/
   → 自動 import 找不到
   → 修：必須在 stores/ 並用 useXxxStore 命名

❌ 4. @nuxt/content 用 useFetch 抓
   → 應該用 queryContent
   → 修：讀 @nuxt/content 文件

❌ 5. 一次裝 20 個模組
   → bundle 爆炸、build 變慢
   → 修：只裝真的會用的、需要再加
```
