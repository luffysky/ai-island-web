### 邊界：怎麼混用

  
關鍵原則：**Server Component 可以 import Client Component，反之不行**。

```tsx
// ✓ 對的
// app/page.tsx (Server)
import Counter from "./Counter"; // (Client)
```

### 實戰準則

  
1. **預設 Server**，需要互動才加 "use client"
2. **把 "use client" 推到葉子**（小元件），不要整個 page 都 client
3. **抓資料在 Server**，傳到 Client 當 props
4. **API key 永遠在 Server**，不要傳給 Client
5. **狀態管理 / 表單 / 動畫 → Client**
6. **內容展示 / SEO → Server**

  
**⚠ 踩雷警告**

「我加 use client 不就好了，全部 client 比較簡單」— 這樣等於回到純 SPA，失去 RSC 所有好處（bundle 變大、SEO 變差、首屏變慢）。**盡量保留 Server Component**，只有真的要互動才 client。
