**認證（authentication、auth、確認使用者身份）** 是每個 SaaS 必做。自己從零寫超容易出資安漏洞——用現成的方案最安全。Next.js 三大選擇：**NextAuth.js（最自由）/ Clerk（最快上手）/ Supabase Auth（搭配 Supabase DB 用）**。

> 💡 **這課的價值**：認證一搞砸、整個系統可被盜帳號、賠錢賠商譽。**用現成方案 = 站在巨人肩膀**、安全又省時。

  
### 1. 五大選擇對比

### 2. NextAuth.js v5（Auth.js）完整範例

🖥️ **在終端機輸入**

```text
npm install next-auth@beta
```

📄 **寫進程式檔**：`auth.ts`（專案根目錄）

```text
import NextAuth from "next-auth";
import GitHub from "next-auth/providers/github";
import Google from "next-auth/providers/google";

export const { auth, handlers, signIn, signOut } = NextAuth({
  providers: [
    GitHub({
      clientId: process.env.GITHUB_ID!,
      clientSecret: process.env.GITHUB_SECRET!
    }),
    Google({
      clientId: process.env.GOOGLE_ID!,
      clientSecret: process.env.GOOGLE_SECRET!
    })
  ],
  callbacks: {
    session({ session, token }) {
      session.user.id = token.sub!;
      return session;
    }
  },
  pages: {
    signIn: "/login"
  }
});
```

📄 **寫進程式檔**：`app/api/auth/[...nextauth]/route.ts`

```text
export { GET, POST } from "@/auth";
```

📄 **寫進程式檔**：`.env.local`

```text
AUTH_SECRET=（用 openssl rand -base64 32 產生）
GITHUB_ID=...
GITHUB_SECRET=...
GOOGLE_ID=...
GOOGLE_SECRET=...
NEXTAUTH_URL=http://localhost:3000
```

  
### 3. 在 Server Component 用

📄 **寫進程式檔**：`app/dashboard/page.tsx`

```text
import { auth } from "@/auth";
import { redirect } from "next/navigation";

export default async function Dashboard() {
  const session = await auth();

  if (!session) {
    redirect("/login");  // 沒登入導向登入頁
  }

  return (
    <div>
      <h1>歡迎、{session.user.email}</h1>
      <p>User ID: {session.user.id}</p>
    </div>
  );
}
```

### 7. Supabase Auth

🖥️ **裝套件**

```text
npm install @supabase/supabase-js @supabase/ssr
```

📄 **寫進程式檔**：`lib/supabase/server.ts`

```text
import { createServerClient } from "@supabase/ssr";
import { cookies } from "next/headers";

export async function createClient() {
  const cookieStore = await cookies();
  return createServerClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
    {
      cookies: {
        getAll: () => cookieStore.getAll(),
        setAll: (cookies) => {
          cookies.forEach(({ name, value, options }) =>
            cookieStore.set(name, value, options));
        }
      }
    }
  );
}
```

📄 **用法**

```text
const supabase = await createClient();
const { data: { user } } = await supabase.auth.getUser();

if (!user) redirect("/login");
```

  
### 8. JWT vs Session：核心觀念

```text
| 特性         | JWT（JSON Web Token）       | Session                        |
|--------------|------------------------------|--------------------------------|
| 儲存         | cookie / localStorage         | server DB + cookie 存 session ID |
| 失效         | 到期才失效（不能立刻撤）       | 可即時撤銷                      |
| 擴展性       | 無狀態、好 scale              | 需 sticky session 或共享 store  |
| 大小         | 較大（含資料）                 | 小（只 ID）                     |
| 建議         | 簡單 app、跨域                | 需立刻登出全裝置                |
```

  
### 9. 資安要點（必背）

```text
🚨 1. JWT 不要放敏感資訊
   → payload 是 base64、不是加密、任何人能看
   → 別放密碼 / API key

🚨 2. token 一律存 httpOnly cookie
   → 別存 localStorage
   → XSS 一中、所有 token 被偷

🚨 3. cookie 一定加 secure + sameSite
   → secure：只 HTTPS 傳
   → sameSite=lax：防 CSRF

🚨 4. 密碼用 bcrypt / argon2 hash
   → 永遠不要存明碼
   → 永遠加 salt

🚨 5. OAuth callback URL 設嚴格
   → 不要 wildcard
   → 防 redirect injection
```

  
### 10. 三大方案怎麼選

```text
🟢 個人專案、學習 → NextAuth.js
   • 通用、社群大、文件豐
   • 接 GitHub / Google / Email / Credentials 都簡單

🟢 SaaS、想 3 天內上線 → Clerk
   • UI 漂亮、UX 一流
   • 1 萬 MAU 免費
   • 缺點：付費後變貴、廠商鎖定

🟢 已用 Supabase 當 DB → Supabase Auth
   • Row Level Security 整合
   • 5 萬 MAU 免費、便宜
   • 缺點：UI 自己刻
```

  
### 5 大常見坑

```text
❌ 1. JWT 存 localStorage
   → XSS 一中、token 被偷
   → 修：永遠 httpOnly cookie

❌ 2. session callback 沒處理 user.id
   → session.user 沒 id、後面查 DB 拿不到 user
   → 修：callbacks.session 加 session.user.id = token.sub

❌ 3. AUTH_SECRET 沒設
   → JWT 無法簽、整個 auth 壞
   → 修：openssl rand -base64 32 產一個

❌ 4. middleware 沒設 matcher、保護所有路徑
   → 連 /login 都要登入、登不進去
   → 修：matcher 只列要保護的

❌ 5. OAuth redirect URL 寫死 localhost
   → 部署到 Vercel 失效
   → 修：用 NEXTAUTH_URL 環境變數
```

  
### 🎯 重點回顧

- **三大方案**：NextAuth.js（自由）/ Clerk（最快）/ Supabase Auth（DB 整合）
- **NextAuth v5**：`auth.ts` + handlers + middleware export
- **Server Component**：`await auth()` 拿 session
- **登入 / 登出**：用 Server Action 包 signIn / signOut
- **JWT vs Session**：跨域用 JWT、要即時撤銷用 Session
- **資安鐵律**：httpOnly cookie、bcrypt hash 密碼、嚴格 OAuth callback
