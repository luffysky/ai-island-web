**Streaming（串流、HTML 邊產邊送）** + **Suspense（懸停、等資料時顯示後備畫面的 React 機制）** 是 Next.js 14+ 的殺手鐧。不用等全部資料都齊、HTML **邊產邊送**到瀏覽器、使用者感知效能提升 5 倍。

> 💡 **這課的價值**：Linear、Vercel、Notion 都用這招——使用者打開頁面**立刻看到框架、慢的部分慢慢補上**、體感不卡。沒用 streaming 的 SSR 網站、慢的 API 卡 3 秒、整頁白屏。

  
### 1. 沒 streaming 的問題

```text
傳統 SSR：
  使用者打開 /dashboard
   ↓
  server 開始抓所有資料：
    - 抓 user 資料（500ms）
    - 抓 stats（1s）
    - 抓 chart 資料（3s 慢死）
   ↓
  「全部抓完」才產 HTML
   ↓
  整頁一次送出（總 3 秒白屏）
```

  
### 2. 有 streaming 的解法

```text
Next.js streaming SSR：
  使用者打開 /dashboard
   ↓
  server 立刻送出框架 HTML（標題、nav、骨架）
   ↓
  使用者「立刻」看到頁面
   ↓
  user 資料抓好 → 補進去
  stats 抓好 → 補進去
  chart 抓好 → 補進去
   ↓
  畫面從上到下漸進填滿
```

  
### 3. loading.tsx：自動 Suspense

最簡單的 streaming——在資料夾內放 `loading.tsx`、Next.js 自動把整頁包進 Suspense：

📄 **寫進程式檔**：`app/blog/loading.tsx`

```text
export default function Loading() {
  return (
    <div className="space-y-4">
      <div className="h-8 bg-gray-200 rounded animate-pulse" />
      <div className="h-32 bg-gray-200 rounded animate-pulse" />
      <div className="h-32 bg-gray-200 rounded animate-pulse" />
    </div>
  );
}
```

📄 **寫進程式檔**：`app/blog/page.tsx`

```text
export default async function BlogPage() {
  const posts = await getAllPosts();  // 假設這要 2 秒
  return (
    <ul>
      {posts.map(p => <li key={p.id}>{p.title}</li>)}
    </ul>
  );
}
```

訪問 `/blog`、Next.js 自動：
- 立刻顯示 `loading.tsx`（skeleton 骨架）
- 後台抓 posts
- 抓完用 `page.tsx` 內容替換

  
### 4. 手動 Suspense（更細粒度）

`loading.tsx` 是「**整頁 loading**」、但通常頁內有快有慢、想分開處理：

📄 **寫進程式檔**：`app/dashboard/page.tsx`

```text
import { Suspense } from "react";

// 慢的 component（3 秒）
async function SlowComponent() {
  await new Promise(r => setTimeout(r, 3000));
  const data = await fetchSlow();
  return <div>{data}</div>;
}

// 快的 component（200ms）
async function FastComponent() {
  const data = await fetchFast();
  return <div>{data}</div>;
}

export default function Page() {
  return (
    <>
      <h1>Dashboard</h1>  {/* 立刻顯示 */}

      <Suspense fallback={<div>載入快速資料...</div>}>
        <FastComponent />  {/* 200ms 後出現 */}
      </Suspense>

      <Suspense fallback={<div>載入慢資料...</div>}>
        <SlowComponent />  {/* 3 秒後出現 */}
      </Suspense>
    </>
  );
}
```

**結果**：使用者立刻看到標題、200ms 後看到快資料、3 秒後看到慢資料——而**不是等 3 秒才看到全部**。

  
### 5. 實戰：Dashboard 並行載入

📄 **寫進程式檔**：`app/dashboard/page.tsx`

```text
import { Suspense } from "react";

async function UserCard() {
  await new Promise(r => setTimeout(r, 500));
  const user = await db.users.findFirst();
  return <div className="card">{user.name}</div>;
}

async function Stats() {
  await new Promise(r => setTimeout(r, 1000));
  const stats = await getStats();
  return <div className="card">總用戶：{stats.total}</div>;
}

async function Chart() {
  await new Promise(r => setTimeout(r, 2000));
  const data = await getChartData();
  return <div className="card">圖表渲染中...</div>;
}

export default function Dashboard() {
  return (
    <div className="grid grid-cols-3 gap-4">
      <Suspense fallback={<Skeleton />}>
        <UserCard />   {/* 500ms */}
      </Suspense>

      <Suspense fallback={<Skeleton />}>
        <Stats />      {/* 1s */}
      </Suspense>

      <Suspense fallback={<Skeleton />}>
        <Chart />      {/* 2s */}
      </Suspense>
    </div>
  );
}
```

**每張卡片獨立載入、誰快誰先出現**——這就是 Vercel dashboard 跟 Linear 的感覺。

  
### 6. Suspense 的兩個用法

```text
① 等資料（搭配 Server Component + await）
   <Suspense fallback={<X />}>
     <ServerCompThatAwaits />
   </Suspense>

② 等 client component lazy load（搭配 dynamic import）
   import dynamic from "next/dynamic";
   const Chart = dynamic(() => import("./chart"), {
     loading: () => <Skeleton />,
     ssr: false
   });
```

  
### 7. error.tsx 跟 Suspense 互動

如果 Suspense 內元件失敗、會被外層 error.tsx 接住：

📄 **寫進程式檔**：`app/dashboard/error.tsx`

```text
"use client";

export default function Error({ error, reset }) {
  return (
    <div>
      <p>糟糕、出錯了：{error.message}</p>
      <button onClick={reset}>重試</button>
    </div>
  );
}
```

> 💡 **重點**：Suspense 處理「**等資料**」、error.tsx 處理「**失敗**」——兩個一起用、UX 完整。

  
### 8. Streaming 對 SEO 影響

```text
✅ Google 完全支援 streaming SSR
   → 爬蟲拿到的 HTML 跟使用者一樣完整

✅ Bing / Yandex / Baidu 也支援

⚠️ 但 Suspense 內的內容延遲到後面才送
   → 重要 SEO 內容（標題、article body）不要包 Suspense
   → 只包「不影響 SEO 的部分」（評論、相關文章、廣告）

✅ Twitter / LINE / FB 抓 OG image 不受影響
   → 因為 OG 在 <head>、Suspense 不影響 head
```

  
### 9. Loading UI 設計建議

```text
🟢 用 skeleton（骨架屏）
   → 灰色塊預示內容形狀
   → 跟最終內容尺寸相近、避免 layout shift

🟢 用 shimmer 動畫
   → 灰色塊「閃」一下
   → 暗示「載入中、不是掛了」

🟡 別用「轉圈圈 spinner」
   → 已過時、低 UX
   → 改 skeleton

🔴 別用 1 秒以上的純空白
   → 使用者以為網站掛了
   → loading.tsx 一定要寫東西
```

  
### 5 大常見坑

```text
❌ 1. Suspense 包 Client Component 但沒 throw promise
   → Suspense 對 Client 只有 React.lazy 才有用
   → 修：要 await 資料就用 Server Component

❌ 2. loading.tsx 跟 page.tsx 顯示時間差很短
   → 反而閃一下、UX 變差
   → 修：快的頁面別放 loading.tsx

❌ 3. 整頁包一個 Suspense
   → 沒分開、跟沒 stream 一樣
   → 修：每個慢區塊單獨 Suspense

❌ 4. Suspense 內元件用 useState
   → Server Component 不能 useState
   → 修：Suspense 內放 Server Component、不要 hook

❌ 5. Skeleton 尺寸跟真內容差很多
   → 載完後跳一下（layout shift）
   → 修：Skeleton 用相近高度 / 寬度
```

  
### 🎯 重點回顧

- **Streaming**：HTML 邊產邊送、不用等全部資料
- **Suspense**：包慢元件、顯示 fallback
- **loading.tsx**：整路由的自動 Suspense
- **手動 Suspense**：更細粒度、每塊獨立載入
- **設計**：用 skeleton 不用 spinner、尺寸接近真內容
- **SEO 安全**：Google / Bing / OG 全支援
