### 1. Core Web Vitals 是什麼

Google 用 **Core Web Vitals（核心網頁指標）** 算 SEO 排名跟使用者體驗：

```text
| 指標   | 全名                          | 達標     | 不達標的問題          |
|--------|-------------------------------|----------|----------------------|
| LCP    | Largest Contentful Paint      | < 2.5s   | 首屏大圖 / 標題太慢   |
| INP    | Interaction to Next Paint     | < 200ms  | 點按鈕卡             |
| CLS    | Cumulative Layout Shift       | < 0.1    | 載入後內容跳          |
| FCP    | First Contentful Paint        | < 1.8s   | 首屏顯示太慢          |
| TTFB   | Time To First Byte            | < 800ms  | 後端慢                |
```

### 3. LCP 優化（首屏大元素加速）

```text
✅ 1. next/image 加 priority
   <Image src="/hero.jpg" alt="..." priority />

✅ 2. 預載字型
   import { Inter } from "next/font/google";
   const inter = Inter({ subsets: ["latin"], display: "swap" });

✅ 3. preload hero 圖
   <link rel="preload" href="/hero.webp" as="image" fetchpriority="high">

✅ 4. 首屏關鍵 CSS inline
   Next.js / Nuxt 自動做

✅ 5. CDN（Vercel / Cloudflare 自動）

✅ 6. ISR / SSG 取代 SSR
   靜態檔比動態 server 快 5-10 倍
```

  
### 4. INP 優化（互動快速）

```text
✅ 1. 把 "use client" 推到葉子
   別整頁 client、bundle 小、hydration 快

✅ 2. 拆 client component
   一個元件不要超過 200 行

✅ 3. 重的計算包 useMemo
   const heavy = useMemo(() => calculate(data), [data]);

✅ 4. 大列表用虛擬滾動（react-virtual / @tanstack/virtual）
   1000 筆只渲染畫面內可見的 20 筆

✅ 5. debounce 搜尋輸入
   const debounced = useDebouncedValue(query, 300);

✅ 6. 動畫用 CSS transform / opacity
   不要 width / height（會 reflow）
```

  
### 5. CLS 優化（佈局不跳）

```text
✅ 1. 圖片永遠 width / height
   <Image src="..." width={800} height={600} />
   or
   <img src="..." width="800" height="600">

✅ 2. 字型用 display: swap + size-adjust
   next/font 自動做

✅ 3. 廣告 / iframe 預留空間
   <div style={{ minHeight: 250 }}><AdComponent /></div>

✅ 4. skeleton 用相近尺寸
   loading 跟真內容一樣大、不會跳

✅ 5. 避免插入內容到已渲染上方
   通知 / Toast 用 fixed position
```

### 7. 常見肥大套件 + 替代

```text
| 大肥套件          | 大小    | 替代              | 大小   |
|-------------------|---------|-------------------|--------|
| moment            | 67 KB   | date-fns / dayjs   | 7 KB   |
| lodash（全部）     | 73 KB   | lodash-es 按需 import | 1-5 KB |
| axios             | 13 KB   | 內建 fetch         | 0 KB   |
| jquery            | 30 KB   | 不要用              | 0 KB   |
| chart.js          | 60 KB   | 改 dynamic import   | lazy  |
```

  
### 8. Code Splitting（按需載入）

```text
✅ 1. dynamic import 沉重元件
   import dynamic from "next/dynamic";
   const Chart = dynamic(() => import("./chart"), {
     loading: () => <Skeleton />,
     ssr: false
   });

✅ 2. Modal / Dialog 用 dynamic
   只有點開才載

✅ 3. 第三方 widget 用 dynamic
   Crisp / Intercom / Stripe Elements

✅ 4. 大圖表 / 編輯器（Monaco / TinyMCE）lazy load
```

  
### 9. 圖片優化清單

```text
✅ 1. 永遠 next/image / NuxtImg
✅ 2. 首屏加 priority
✅ 3. 響應式 sizes 設對
   sizes="(max-width: 768px) 100vw, 50vw"
✅ 4. WebP / AVIF 自動（remotePatterns 設）
✅ 5. blur placeholder（本地圖）
✅ 6. SVG 用 inline 或 next/svgr
✅ 7. CDN cache（minimumCacheTTL: 86400）
```

  
### 10. 字型優化清單

```text
✅ 1. next/font / @nuxtjs/google-fonts
   self-host、不打 Google
✅ 2. display: swap
   先顯示 fallback、字型載完替換
✅ 3. subset（只載用到的字符）
   中文字型 10MB → 200KB
✅ 4. preload 主字型
✅ 5. font-display: optional（非關鍵字型）
✅ 6. 用 system-ui fallback
```

  
### 11. cache 策略總覽

```text
┌────────────────────────────────────────┐
│ 1. 全靜態（SSG / prerender）            │
│    Cache-Control: public, max-age=31536000│
│    → 部落格、文件、行銷頁                │
└────────────────────────────────────────┘

┌────────────────────────────────────────┐
│ 2. ISR（revalidate）                     │
│    Cache-Control: public, s-maxage=60   │
│    → 商品、新聞                          │
└────────────────────────────────────────┘

┌────────────────────────────────────────┐
│ 3. SSR + stale-while-revalidate         │
│    Cache-Control: public, s-maxage=10,  │
│      stale-while-revalidate=60          │
│    → 動態但能容忍舊資料                  │
└────────────────────────────────────────┘

┌────────────────────────────────────────┐
│ 4. 私有（個人化）                        │
│    Cache-Control: private, no-store     │
│    → dashboard、個人化、購物車           │
└────────────────────────────────────────┘
```

### 5 大常見坑

```text
❌ 1. 整站 SSR、Vercel 帳單爆炸
   → 能 SSG 就 SSG、其次 ISR
   → 修：routeRules / export const revalidate

❌ 2. 整頁 client、bundle 2MB
   → 沒用 RSC 好處
   → 修：Server Component 為主

❌ 3. 圖片用 <img> 不用 <Image />
   → 沒 WebP、沒 lazy、沒響應式
   → 修：永遠 next/image / NuxtImg

❌ 4. 載 moment.js / lodash 全部
   → bundle 50KB+
   → 修：date-fns / lodash-es 按需

❌ 5. console.log 沒清、production 滿地都是
   → 影響效能、洩漏資訊
   → 修：build 時用 babel-plugin-transform-remove-console 清掉
```
