**Server Action（伺服器動作、直接在元件內定義的 server 函式）** 是 Next.js 14+ 的革命：「**在 component 內直接寫 server function**、不用寫 API route、不用 fetch」。傳統要 30 行的事情、現在 5 行。

> 💡 **這課的價值**：寫表單、儲存資料、刪除、更新——傳統要寫「前端 fetch + 後端 API route + JSON 來回」三層、現在 Server Action 一個 function 搞定。**JS 關掉也能用**（漸進式增強）。

  
### 1. 第一個 Server Action

📄 **寫進程式檔**：`app/posts/new/page.tsx`

```text
import { revalidatePath } from "next/cache";
import { redirect } from "next/navigation";

// 在 server 跑的 function（前面加 "use server"）
async function createPost(formData: FormData) {
  "use server";

  const title = formData.get("title") as string;
  const content = formData.get("content") as string;

  // 直接呼叫 DB（不用 API route）
  await db.posts.insert({ title, content });

  revalidatePath("/posts");  // 重新驗證快取、列表會更新
  redirect("/posts");        // 導向列表頁
}

export default function NewPost() {
  return (
    <form action={createPost}>
      <input name="title" placeholder="標題" />
      <textarea name="content" placeholder="內容" />
      <button type="submit">發布</button>
    </form>
  );
}
// 厲害的點：JS 關掉也能用（Progressive Enhancement、漸進式增強）
```

> 💡 **白話**：「**form 的 action 直接指向一個 server function**」——使用者按送出、瀏覽器把 form 資料直接傳給 server function、執行完自動跳頁。中間沒有 fetch、沒有 JSON、沒有 API route。

  
### 2. 抽到獨立檔（最佳實踐）

整個檔案頂端寫 `"use server"`、裡面 export 的所有 function 都自動是 Server Action：

📄 **寫進程式檔**：`app/actions/posts.ts`

```text
"use server";  // 整個檔案的 export 都是 server action

import { z } from "zod";
import { revalidatePath } from "next/cache";

// 用 zod 做表單驗證（schema validation）
const CreatePostSchema = z.object({
  title: z.string().min(1, "標題必填").max(200),
  content: z.string().min(10, "內容至少 10 字")
});

export async function createPost(prevState: any, formData: FormData) {
  const validated = CreatePostSchema.safeParse({
    title: formData.get("title"),
    content: formData.get("content")
  });

  if (!validated.success) {
    return { error: validated.error.flatten().fieldErrors };
  }

  try {
    const post = await db.posts.create({ data: validated.data });
    revalidatePath("/posts");
    return { success: true, post };
  } catch (e) {
    return { error: { _form: ["建立失敗、請稍後再試"] } };
  }
}

export async function deletePost(id: string) {
  await db.posts.delete({ where: { id } });
  revalidatePath("/posts");
}
```

  
### 3. useFormState + useFormStatus（顯示錯誤 + loading）

React 19 提供兩個專門搭配 Server Action 的 hook：

📄 **寫進程式檔**：`app/posts/new/post-form.tsx`

```text
"use client";
import { useFormState, useFormStatus } from "react-dom";
import { createPost } from "@/app/actions/posts";

function SubmitButton() {
  const { pending } = useFormStatus();  // 自動知道送出中
  return (
    <button type="submit" disabled={pending}>
      {pending ? "送出中..." : "發布"}
    </button>
  );
}

export default function PostForm() {
  // useFormState 接收 server action 回傳值（顯示錯誤訊息）
  const [state, formAction] = useFormState(createPost, { error: null });

  return (
    <form action={formAction}>
      <input name="title" placeholder="標題" />
      {state.error?.title && (
        <p style={{ color: "red" }}>{state.error.title[0]}</p>
      )}

      <textarea name="content" placeholder="內容" />
      {state.error?.content && (
        <p style={{ color: "red" }}>{state.error.content[0]}</p>
      )}

      <SubmitButton />
    </form>
  );
}
```

**好處**：表單錯誤訊息顯示、送出中按鈕變灰、全自動、不用自己寫 `useState`。

  
### 4. 從 onClick 呼叫 Server Action

不只 `<form action={...}>`、按鈕點擊也能直接呼叫：

📄 **寫進程式檔**：`app/posts/delete-button.tsx`

```text
"use client";
import { deletePost } from "@/app/actions/posts";

export function DeleteButton({ id }: { id: string }) {
  return (
    <button onClick={async () => {
      if (confirm("確定刪除？")) {
        await deletePost(id);  // 直接呼叫 server function
      }
    }}>
      刪除
    </button>
  );
}
```

  
### 5. 樂觀更新（useOptimistic、立刻顯示變化）

UX 加分項——使用者按按鈕、**畫面立刻變**、背景才送 server。如果 server 失敗才回退：

📄 **寫進程式檔**：`app/todos/todo-list.tsx`

```text
"use client";
import { useOptimistic } from "react";
import { addTodoAction } from "@/app/actions/todos";

export default function TodoList({ todos }) {
  const [optimisticTodos, addOptimistic] = useOptimistic(
    todos,
    (state, newTodo) => [...state, newTodo]
  );

  async function handleAdd(formData: FormData) {
    const text = formData.get("text") as string;
    addOptimistic({ id: "temp-" + Date.now(), text });  // 立刻顯示
    await addTodoAction(text);                            // 真的存
  }

  return (
    <>
      <form action={handleAdd}>
        <input name="text" />
        <button>新增</button>
      </form>
      <ul>
        {optimisticTodos.map(t => <li key={t.id}>{t.text}</li>)}
      </ul>
    </>
  );
}
```

  
### 6. Server Action 安全注意（必讀）

```text
⚠️ 1. 任何標 "use server" 的 function 都會曝露成 endpoint
   → 別人可以直接呼叫
   → 必須自己驗證 auth + 輸入

⚠️ 2. 永遠驗證 input
   → 用 zod / valibot 檢查 formData
   → 不要相信前端傳什麼就用什麼

⚠️ 3. 永遠檢查 session / 權限
   → const session = await auth();
   → if (!session) throw new Error("Unauthorized");

⚠️ 4. 不要把 user input 直接拼 SQL
   → 用 ORM（Prisma / Drizzle）或 parameterized query
   → 防 SQL injection

⚠️ 5. 重要操作加 CSRF 防護
   → Next.js 已內建（同源檢查）
   → 但跨域用要設 allowedOrigins
```

📄 **正確的 Server Action 範例**

```text
"use server";
import { auth } from "@/lib/auth";

export async function deletePost(id: string) {
  // 1. 驗證登入
  const session = await auth();
  if (!session) throw new Error("Unauthorized");

  // 2. 驗證權限
  const post = await db.posts.findUnique({ where: { id } });
  if (post.authorId !== session.user.id) {
    throw new Error("Forbidden");
  }

  // 3. 才真的執行
  await db.posts.delete({ where: { id } });
  revalidatePath("/posts");
}
```

  
### 5 大常見坑

```text
❌ 1. Server Action 內忘記 revalidatePath
   → 資料改了、頁面還是舊的
   → 修：mutation 後一定 revalidatePath 或 revalidateTag

❌ 2. 在 Server Action 用 window / localStorage
   → server 沒這些東西、報錯
   → 修：browser API 只在 Client Component

❌ 3. 沒檢查 auth 就執行
   → 任何人能呼叫 deletePost("任何 id")
   → 修：每個 action 開頭驗 session

❌ 4. 從 Client 直接 import Server Action 檔
   → 看起來 ok、但檔案有 "use server" 是特殊機制
   → 修：直接 import 函式名就好（Next.js 自動處理）

❌ 5. formData.get() 沒做 type cast
   → 拿到的是 FormDataEntryValue（不是 string）
   → 修：寫 as string、或用 zod 解析
```

  
### 🎯 重點回顧

- **Server Action** = 「在元件內直接寫 server function」
- **兩種寫法**：function 內 `"use server"` 或檔案頂端 `"use server"`
- **呼叫方式**：`<form action={...}>` 或 onClick 直接呼叫
- **useFormState**：接收回傳值、顯示錯誤
- **useFormStatus**：自動拿到 pending 狀態
- **useOptimistic**：樂觀更新、UX 加分
- **安全**：自己驗 auth + 驗 input、不能省
