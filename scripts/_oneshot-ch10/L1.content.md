**Next.js / Nuxt** 是「**前端框架**（framework、把常用功能包好的工具箱）」，建在 React / Vue 上面、補了純 React / Vue 寫網站會缺的所有東西。學完這課你會懂：**為什麼 99% 的新創 SaaS 都用 Next.js / Nuxt、而不是純 React / Vue**。

> 💡 **這課的價值**：如果只會純 React、做出來的網站 Google 搜不到、首頁打開白屏 3 秒、分享到 LINE 變一片白。**Next.js / Nuxt 自動解決這些問題**——這是業界共識、不是選配。

  
### 1. 純 React / Vue 寫的東西叫 SPA

```text
SPA（Single Page Application、單頁應用）
   = 整個網站只有「一個 HTML 檔」
   = 所有內容靠 JavaScript 動態畫出來

打開網站時：
  瀏覽器拿到「空 HTML」
  ↓
  下載一大包 JS（300KB-3MB）
  ↓
  JS 跑起來、React 把內容畫出來
  ↓
  使用者終於看到畫面（2-5 秒）
```

> 💡 **白話**：純 React 就像「**請朋友來家裡幫你蓋房子**」——朋友（瀏覽器）拿著一袋工具（JS）跟一張藍圖、現場才開始蓋。慢、而且郵差（Google 爬蟲）來看時、房子還沒蓋好。

  
### 2. SPA 的 7 個致命缺點

```text
❌ SEO 差
   → Google 拿到空 HTML、看不到內容
   → 即使 Google 會跑 JS、Bing / Baidu / 其他搜尋引擎不一定

❌ 首屏（first paint）慢
   → 要下載 + 解析 JS 才有畫面
   → 白屏 2-5 秒、使用者直接關掉

❌ 社群分享卡片壞掉
   → 貼到 LINE / FB / Discord
   → 抓 Open Graph（OG、社群預覽圖）抓到空白

❌ 沒內建路由
   → 要自己裝 React Router / Vue Router

❌ 沒內建 API
   → 後端要分開寫、再用 fetch 串

❌ 沒部署最佳化
   → 圖片壓縮、字型載入、code splitting（分包）要手動

❌ 沒 SSR / SSG
   → 想做 SEO 友善要自己接 server、超複雜
```

  
### 3. Next.js / Nuxt 是什麼？解決了什麼？

**Next.js（Vercel 出品、建在 React 上）** 跟 **Nuxt 3（Vue 生態系、建在 Vue 3 上）** 是兩個最紅的「**Meta-framework**（元框架、把框架再包一層的框架）」。它們補了上面所有缺點：

```text
✅ 檔案系統路由（File-system routing）
   → 在 app/about/page.tsx 開檔
   → 自動有 /about 這條路由
   → 不用手寫 router

✅ 多種渲染模式（rendering mode）
   → SSR / SSG / ISR / CSR / RSC 自選
   → L2 會深入講

✅ 內建 API routes
   → 前端跟後端寫在同一個專案
   → 不用兩個 repo

✅ 自動優化
   → 圖片轉 WebP / AVIF（next/image / @nuxt/image）
   → 字型 self-host（next/font）
   → JS 自動 code splitting

✅ SEO metadata 一行搞定
   → generateMetadata / useSeoMeta
   → 動態 OG image（每篇文章獨立分享圖）

✅ 一鍵部署
   → Vercel / Cloudflare / Zeabur 連 GitHub 就好

✅ 國際化（i18n）內建
✅ Streaming SSR、Suspense（邊產邊送 HTML）
```

> 💡 **比喻**：React / Vue 像「**引擎**」。Next.js / Nuxt 像「**整台車**」——引擎、輪子、座椅、空調、音響全配好。你想自己組車是 OK、但 99% 的人直接買整台車。

  
### 4. 動手看一下感覺

🖥️ **在終端機輸入**（建一個 Next.js 專案）

```text
npx create-next-app@latest my-app
```

**這行做什麼？** `npx` = 跑 npm 套件、不用先安裝；`create-next-app@latest` = Next.js 官方的「**腳手架工具**（scaffolding tool、自動建專案結構）」；`my-app` = 你的專案資料夾名。

⌨️ **這時換你打字**（互動選項全部按 Enter 用預設就行）

```text
✔ Would you like to use TypeScript? → Yes
✔ Would you like to use Tailwind CSS? → Yes
✔ Would you like to use App Router? → Yes
✔ Would you like to use src/ directory? → Yes
```

💬 **電腦會顯示**

```text
Creating a new Next.js app in /Users/you/my-app
Installing dependencies...
Success! Created my-app
```

🖥️ **在終端機輸入**（啟動開發伺服器）

```text
cd my-app
npm run dev
```

💬 **電腦會顯示**

```text
▲ Next.js 15.0
- Local: http://localhost:3000
✓ Ready in 1.2s
```

打開瀏覽器到 `http://localhost:3000` 就看到首頁——**這就是 Next.js 的起點**。

  
### 5. 2026 業界共識：誰該用、誰不該用

```text
✅ 該用 Next.js / Nuxt
   • SaaS 產品（99% 用、Linear / Vercel / Notion 都是）
   • 公開行銷頁 / 部落格（必須 SSG 做 SEO）
   • 電商（ISR + RSC 是黃金組合）
   • 個人 portfolio
   • AI 產品介面（ChatGPT-like UI）

❌ 不該用 Next.js / Nuxt
   • 純桌面 App（用 Electron / Tauri）
   • 純手機 App（用 React Native / Expo / Flutter）
   • Chrome Extension（用 Vite + React）
   • 只有後台的內部工具、不在乎 SEO（純 Vite + React 更輕）
   • 極小 widget、追求 5KB bundle（用 Preact / Lit）
```

  
### 5 大常見坑

```text
❌ 1. 「我會 React 就好、不用學 Next.js」
   → 業界 8 成職缺寫 Next.js / Nuxt
   → 純 React 找工作越來越難
   → 修：學完 React 馬上學 Next.js

❌ 2. 把 Next.js 當「比較複雜的 React」
   → 它是「React + 後端 + 部署」的全套
   → 心智模型要轉變
   → 修：先讀 L2、理解 5 種渲染模式

❌ 3. 用 Next.js 但全部寫 "use client"
   → 等於回到純 SPA、失去所有好處
   → 修：L4 會深入講 Server vs Client

❌ 4. 不部署、只在 localhost 跑
   → 部署到 Vercel 30 秒就好
   → 拿不出來、面試沒作品
   → 修：每寫完一個 mini-project 馬上推 Vercel

❌ 5. 同時學 Next + Nuxt
   → 兩個概念太像、會搞混
   → 修：選一個（會 React 選 Next、會 Vue 選 Nuxt）、另一個之後一週內就能轉
```

  
### 🎯 重點回顧

- **純 React / Vue = SPA、有 7 個致命缺點**（SEO / 首屏 / 社群分享 / 路由 / API / 優化 / 渲染）
- **Next.js（React）+ Nuxt（Vue）= 補齊缺點的整套方案**
- **業界共識**：SaaS / 行銷頁 / 電商 / Portfolio 都該用
- **不該用**：桌面 / 手機 App / Extension / 小 widget
- **2026 職缺 80% 寫 Next.js / Nuxt**——這不是選配、是必修
