「**動態路由**（dynamic route、URL 帶參數的路由）」+ **SEO Metadata**（meta data、給搜尋引擎跟社群看的資料）= 一個內容網站的命脈。學完這課你的部落格每篇文章都能被 Google 找到、分享到 LINE 有專屬封面圖。

> 💡 **這課的價值**：靜態 metadata 誰都會、**動態 metadata + 每篇文章獨立 OG 圖** 是讓網站「**品牌感**」up 10 倍的關鍵。Vercel / Linear / Stripe blog 都這麼做。

  
### 1. 動態路由：`[slug]`

📄 **寫進程式檔**：`app/blog/[slug]/page.tsx`

```text
type Props = {
  params: Promise<{ slug: string }>  // Next.js 15 params 是 Promise
};

export default async function BlogPost({ params }: Props) {
  const { slug } = await params;
  const post = await getPost(slug);

  if (!post) {
    return <p>找不到文章</p>;
  }

  return (
    <article>
      <h1>{post.title}</h1>
      <div dangerouslySetInnerHTML={{ __html: post.html }} />
    </article>
  );
}
```

**白話**：資料夾名加方括號 `[slug]` 就是「**這格可以是任何值**」。訪問 `/blog/hello-world` 時、`params.slug === "hello-world"`。

  
### 2. 多參數動態路由

```text
app/shop/[category]/[id]/page.tsx
  → /shop/electronics/123
  → params = { category: "electronics", id: "123" }

app/docs/[...path]/page.tsx          ← catch-all（吃所有後段）
  → /docs/a/b/c
  → params.path = ["a", "b", "c"]

app/docs/[[...path]]/page.tsx        ← optional catch-all（可有可無）
  → /docs                (params.path = undefined)
  → /docs/intro          (params.path = ["intro"])
  → /docs/api/users      (params.path = ["api", "users"])
```

  
### 3. generateStaticParams：SSG 動態路由

如果你想「**build 時就把所有文章頁產出靜態 HTML**」（最快、最便宜）：

📄 **寫進程式檔**：`app/blog/[slug]/page.tsx`

```text
// 告訴 Next.js build 時要預先產生哪些頁面
export async function generateStaticParams() {
  const posts = await getAllPosts();
  return posts.map(p => ({ slug: p.slug }));
}

export default async function Post({ params }) {
  const { slug } = await params;
  const post = await getPost(slug);
  return <article>{post.content}</article>;
}
```

🖥️ **在終端機輸入**

```text
npm run build
```

💬 **電腦會顯示**

```text
Route                          Size      First Load JS
○ /blog/hello-world            1.2 kB    85 kB
○ /blog/why-nextjs             1.2 kB    85 kB
○ /blog/ai-tools-2026          1.2 kB    85 kB
...
✓ Generating static pages (147/147)
```

**147 篇文章 = 147 個靜態 HTML 檔案**、放 CDN 上、極快。

  
### 4. 靜態 Metadata（每頁固定的）

📄 **寫進程式檔**：`app/about/page.tsx`

```text
export const metadata = {

  description: "從入門到怪物工程師的免費 20 章教程",
  keywords: ["全端", "教學", "AI"],
  authors: [{ name: "Luffysky" }],

  openGraph: {
    title: "AI 島 - 關於我們",
    description: "從入門到怪物工程師",
    url: "https://example.com/about",
    siteName: "AI 島",
    images: [
      { url: "https://example.com/og.jpg", width: 1200, height: 630 }
    ],
    locale: "zh_TW",
    type: "website"
  },

  twitter: {
    card: "summary_large_image",
    title: "AI 島",
    images: ["https://example.com/og.jpg"]
  },

  robots: {
    index: true,
    follow: true,
    googleBot: { index: true, follow: true, "max-image-preview": "large" }
  },

  alternates: {
    canonical: "https://example.com/about",
```

> 💡 **重點**：寫 `export const metadata = {...}`、Next.js 自動產生 `<title>` / `<meta>` / `<link>` 所有 SEO 標籤、不用自己寫 `<head>`。

  
### 5. 動態 Metadata（每篇文章不同）

📄 **寫進程式檔**：`app/blog/[slug]/page.tsx`

```text
import type { Metadata } from "next";

export async function generateMetadata(
  { params }: { params: Promise<{ slug: string }> }
): Promise<Metadata> {
  const { slug } = await params;
  const post = await getPost(slug);

  return {
    title: `${post.title} | AI 島部落格`,
    description: post.excerpt,
    openGraph: {
      title: post.title,
      description: post.excerpt,
      images: [post.coverImage],
      type: "article",
      publishedTime: post.publishedAt,
      authors: [post.author]
    }
  };
}
```

**每篇文章自己的 title / description / OG**——Google 搜尋顯示每篇獨立、貼到 LINE 預覽圖每篇不同。

  
### 6. 動態 OG Image（神技、品牌感 up）

「**OG image**（Open Graph image、社群分享預覽圖）」原本要設計師畫每篇文章、現在 Next.js 自動產：

📄 **寫進程式檔**：`app/blog/[slug]/opengraph-image.tsx`

```text
import { ImageResponse } from "next/og";

export const runtime = "edge";  // 跑在 Edge runtime（邊緣執行、全球極快）
export const alt = "Blog post cover";
export const size = { width: 1200, height: 630 };
export const contentType = "image/png";

export default async function Image({ params }) {
  const { slug } = await params;
  const post = await getPost(slug);

  return new ImageResponse(
    (
      <div style={{
        background: "linear-gradient(135deg, #10b981, #059669)",
        width: "100%", height: "100%",
        display: "flex", flexDirection: "column",
        justifyContent: "center", padding: 60,
        color: "white"
      }}>
        <div style={{ fontSize: 32 }}>AI 島 / 部落格</div>
        <div style={{ fontSize: 64, fontWeight: 700, marginTop: 20 }}>
          {post.title}
        </div>
        <div style={{ fontSize: 28, marginTop: 40 }}>
          {post.author} · {post.date}
        </div>
      </div>
    ),
    { ...size }
  );
}
```

**每篇文章自動有獨特 OG image**——訪問 `https://yoursite.com/blog/hello/opengraph-image` 就看到那篇的封面。社群分享品牌感大幅提升。

  
### 7. sitemap.xml 與 robots.txt 自動產

📄 **寫進程式檔**：`app/sitemap.ts`

```text
import type { MetadataRoute } from "next";

export default async function sitemap(): Promise<MetadataRoute.Sitemap> {
  const posts = await getAllPosts();

  return [
    { url: "https://example.com", lastModified: new Date(), priority: 1.0 },
    { url: "https://example.com/about", priority: 0.8 },
    ...posts.map(p => ({
      url: `https://example.com/blog/${p.slug}`,
      lastModified: p.updatedAt,
      priority: 0.6
    }))
  ];
}
```

📄 **寫進程式檔**：`app/robots.ts`

```text
import type { MetadataRoute } from "next";

export default function robots(): MetadataRoute.Robots {
  return {
    rules: [
      { userAgent: "*", allow: "/", disallow: ["/admin/", "/api/"] }
    ],
    sitemap: "https://example.com/sitemap.xml"
  };
}
```

訪問 `/sitemap.xml` 跟 `/robots.txt` 都自動有了、Google Search Console 直接吃。

  
### 5 大常見坑

```text
❌ 1. metadata 寫在 Client Component
   → 不會生效（只 Server Component 能 export metadata）
   → 修：metadata 放 Server Component（通常是 page.tsx）

❌ 2. og:image 用相對路徑
   → FB / LINE 抓不到
   → 修：永遠寫絕對 URL（https://...）

❌ 3. generateMetadata 內又 fetch 一次資料
   → 跟 page 內 fetch 重複
   → 修：用 React cache() 包 getPost、自動去重

❌ 4. 動態路由忘記 generateStaticParams
   → 預設變 dynamic、每次請求都跑 server
   → 修：內容站一定加 generateStaticParams

❌ 5. OG image 內用一般圖
   → 必須 1200×630、其他尺寸 LINE 顯示變形
   → 修：永遠 1200×630
```

  
### 🎯 重點回顧

- **動態路由**：`[slug]` 單參數、`[...path]` catch-all、`[[...path]]` optional
- **generateStaticParams**：SSG 動態路由必加、build 時預生
- **靜態 metadata**：`export const metadata = {...}`
- **動態 metadata**：`export async function generateMetadata()`
- **動態 OG image**：`opengraph-image.tsx` + `ImageResponse`
- **sitemap.ts / robots.ts**：自動產 sitemap.xml / robots.txt
