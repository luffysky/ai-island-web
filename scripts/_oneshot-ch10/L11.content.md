**Middleware（中介層、請求進來時先攔截處理的程式）** 跑在「**請求到達 page 之前**」、可以做 redirect（轉址）、rewrite（改寫路徑）、設 cookie、加 header、A/B test。**身份驗證、地理導流、安全 header、機器人偵測都靠它**。

> 💡 **這課的價值**：Middleware 是 Next.js 「**全站攔截器**」、超強。但**跑在 Edge runtime（邊緣執行）**、有限制——不能用 fs / 大套件。寫對能加分、寫錯整站變慢。

  
### 1. 建立第一個 middleware

📄 **寫進程式檔**：`src/middleware.ts`（或專案根目錄）

```text
import { NextRequest, NextResponse } from "next/server";

export async function middleware(req: NextRequest) {
  const token = req.cookies.get("token")?.value;

  // 沒登入卻想進 /dashboard、轉去登入頁
  if (!token && req.nextUrl.pathname.startsWith("/dashboard")) {
    return NextResponse.redirect(new URL("/login", req.url));
  }

  // 加自訂 header
  const response = NextResponse.next();
  response.headers.set("x-custom", "hello");
  return response;
}

// 設定哪些路徑跑 middleware
export const config = {
  matcher: [
    "/dashboard/:path*",
    "/api/:path*",
    // 排除某些路徑（regex 寫法）
    "/((?!_next/static|_next/image|favicon.ico).*)"
  ]
};
```

> 💡 **白話**：middleware **每個請求都會被它先過一遍**。要寫 matcher 限制範圍、不然會跑在每個 request（連靜態檔都跑）、拖慢全站。

  
### 2. 常見用途 ①：身份驗證

```text
export async function middleware(req: NextRequest) {
  const token = req.cookies.get("auth-token")?.value;
  const path = req.nextUrl.pathname;

  // 後台必須登入
  if (path.startsWith("/admin")) {
    if (!token) {
      return NextResponse.redirect(new URL("/login", req.url));
    }
    // 驗 JWT
    const valid = await verifyJWT(token);
    if (!valid) {
      return NextResponse.redirect(new URL("/login", req.url));
    }
  }

  return NextResponse.next();
}
```

  
### 3. 常見用途 ②：地理區域導流

```text
export function middleware(req: NextRequest) {
  const country = req.geo?.country;  // Vercel 自動提供
  const path = req.nextUrl.pathname;

  // 台灣訪客、首頁 rewrite 到 /zh
  if (country === "TW" && path === "/") {
    return NextResponse.rewrite(new URL("/zh", req.url));
  }

  // 日本訪客、首頁 rewrite 到 /ja
  if (country === "JP" && path === "/") {
    return NextResponse.rewrite(new URL("/ja", req.url));
  }
}
```

> 💡 **redirect vs rewrite 的差別**：
> - **redirect**：URL 真的變、瀏覽器看到 `/zh`
> - **rewrite**：URL 不變（還是 `/`）、但內部當作 `/zh` 處理

  
### 4. 常見用途 ③：A/B 測試

```text
export function middleware(req: NextRequest) {
  // 已有 variant cookie 就用既有的
  let variant = req.cookies.get("variant")?.value;

  if (!variant) {
    // 第一次來、隨機分組
    variant = Math.random() > 0.5 ? "a" : "b";
  }

  // rewrite 到對應版本
  const url = req.nextUrl.clone();
  if (variant === "b") {
    url.pathname = "/variant-b" + url.pathname;
  }

  const response = NextResponse.rewrite(url);
  response.cookies.set("variant", variant);
  return response;
}
```

  
### 5. 常見用途 ④：安全 header

production 必加的安全 header：

```text
export function middleware(req: NextRequest) {
  const response = NextResponse.next();

  // 防 clickjacking
  response.headers.set("X-Frame-Options", "DENY");

  // 防 MIME sniffing
  response.headers.set("X-Content-Type-Options", "nosniff");

  // 強制 HTTPS（HSTS、HTTP Strict Transport Security）
  response.headers.set(
    "Strict-Transport-Security",
    "max-age=31536000; includeSubDomains"
  );

  // Referrer policy
  response.headers.set("Referrer-Policy", "strict-origin-when-cross-origin");

  // Permissions Policy
  response.headers.set(
    "Permissions-Policy",
    "camera=(), microphone=(), geolocation=()"
  );

  return response;
}
```

  
### 6. 常見用途 ⑤：機器人偵測

```text
export function middleware(req: NextRequest) {
  const ua = req.headers.get("user-agent") ?? "";
  const isBot = /bot|crawler|spider|crawling/i.test(ua);

  if (isBot) {
    // 給 bot 看的精簡版（不跑 JS、不個人化）
    return NextResponse.rewrite(new URL("/static-version" + req.nextUrl.pathname, req.url));
  }

  return NextResponse.next();
}
```

  
### 7. matcher 寫法詳解

```text
// 1. 簡單路徑
matcher: "/dashboard"           → 只 /dashboard

// 2. 所有子路徑
matcher: "/dashboard/:path*"   → /dashboard, /dashboard/x, /dashboard/x/y

// 3. 多個路徑
matcher: ["/dashboard/:path*", "/admin/:path*"]

// 4. 排除靜態資源（最常見）
matcher: [
  "/((?!_next/static|_next/image|favicon.ico|robots.txt).*)"
]

// 5. 條件式 matcher（進階）
matcher: [
  {
    source: "/((?!api|_next/static|_next/image|favicon.ico).*)",
    missing: [
      { type: "header", key: "next-router-prefetch" },
      { type: "header", key: "purpose", value: "prefetch" }
    ]
  }
]
```

  
### 8. Edge Runtime 限制（重要）

Middleware 跑在 **Edge Runtime（邊緣執行、輕量 V8 環境、不是 Node）**：

```text
❌ 不能用
   • fs（檔案系統）
   • net / tls / child_process
   • Node native modules（bcrypt 等）
   • 某些 npm 套件（會編譯失敗）
   • 太大的套件（> 1MB）

✅ 可以用
   • fetch
   • Web Crypto（subtle.digest 等）
   • URL / URLSearchParams
   • TextEncoder / TextDecoder
   • Headers / Request / Response
   • cookies / headers（NextRequest）

⚠️ 執行時間限制
   • Vercel：50ms（hobby）/ 1000ms（pro）
   • Cloudflare Workers：50ms
```

> 💡 **白話**：Middleware 適合「**輕量、快、純 JS 邏輯**」。重一點的（DB 查詢、複雜驗證）放 API Route。

  
### 9. 實戰：完整 production middleware

📄 **寫進程式檔**：`src/middleware.ts`

```text
import { NextRequest, NextResponse } from "next/server";

export async function middleware(req: NextRequest) {
  const response = NextResponse.next();
  const path = req.nextUrl.pathname;
  const token = req.cookies.get("auth-token")?.value;

  // ① 安全 header（全站）
  response.headers.set("X-Frame-Options", "DENY");
  response.headers.set("X-Content-Type-Options", "nosniff");
  response.headers.set("Referrer-Policy", "strict-origin-when-cross-origin");

  // ② 後台必須登入
  if (path.startsWith("/admin")) {
    if (!token) {
      return NextResponse.redirect(new URL("/login", req.url));
    }
  }

  // ③ 已登入不能進登入頁
  if (path === "/login" && token) {
    return NextResponse.redirect(new URL("/dashboard", req.url));
  }

  // ④ 記錄訪問
  response.headers.set("x-visit-time", new Date().toISOString());

  return response;
}

export const config = {
  matcher: [
    "/((?!_next/static|_next/image|favicon.ico|api/health).*)"
  ]
};
```

  
### 5 大常見坑

```text
❌ 1. 沒設 matcher、middleware 跑在每個 request
   → 連 favicon.ico、_next/static 都跑、拖慢全站
   → 修：永遠寫 matcher 排除靜態資源

❌ 2. middleware 內查 DB
   → Edge 限制 + 慢
   → 修：改在 Server Component / API Route 查

❌ 3. middleware 用 bcrypt / jsonwebtoken（CommonJS）
   → Edge 不支援
   → 修：用 jose（JWT）、或在 API Route 驗

❌ 4. 改 middleware.ts 不存在但放錯位置
   → 必須在 src/ 或專案根目錄
   → 不能放在 src/app/ 內

❌ 5. redirect 寫死 URL
   → 部署到不同網域失效
   → 修：用 new URL("/path", req.url) 相對轉址
```

  
### 🎯 重點回顧

- **middleware.ts** 放 src/ 或專案根目錄
- **Edge Runtime**：輕量、快、有限制（不能 fs / native module）
- **NextResponse**：next / redirect / rewrite / json + 設 cookie / header
- **matcher**：限制範圍、永遠排除靜態資源
- **常見用途**：身份驗證 / 地理導流 / A/B test / 安全 header / bot 偵測
- **限制**：50ms timeout、1MB size、不能複雜運算
