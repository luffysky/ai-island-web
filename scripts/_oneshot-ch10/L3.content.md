Next.js 13 之後主推 **App Router（應用程式路由器、新版檔案系統路由）**——資料夾結構就是路由結構、不用手寫 router。學完這課你能看懂任何 Next.js 專案的檔案在做什麼。

> 💡 **這課的價值**：App Router 是 Next.js 14 / 15 的標配。看 Vercel / Linear / Notion 等公司的 source code、全部都是這個結構。**懂結構 = 看任何 Next.js 專案不卡**。

  
### 1. 建專案

🖥️ **在終端機輸入**

```text
npx create-next-app@latest my-app
```

⌨️ **這時換你打字**（選項）

```text
✔ TypeScript? → Yes
✔ Tailwind CSS? → Yes
✔ App Router? → Yes
✔ src/ directory? → Yes
✔ Turbopack? → Yes
```

🖥️ **進專案啟動**

```text
cd my-app
npm run dev
```

  
### 2. 完整檔案結構（要看懂、不用背）

├── src/

### 3. App Router 慣例檔名（必背 8 個）

每個資料夾內、Next.js 「**看到特定檔名會自動處理**」：

```text
| 檔名             | 用途                                        |
|------------------|---------------------------------------------|
| page.tsx         | 這個路由的 UI（必須有才能訪問）              |
| layout.tsx       | 包裹下層所有頁面（共用 nav / footer）        |
| loading.tsx      | 自動轉成 Suspense fallback（載入中畫面）     |
| error.tsx        | 錯誤邊界（自動 catch error）                  |
| not-found.tsx    | 404 頁面（可手動 notFound() 觸發）            |
| route.ts         | API endpoint（GET / POST / PUT / DELETE）    |
| template.tsx     | 像 layout 但每次切路由重 mount               |
| default.tsx      | Parallel Routes 的後備                       |
```

> 💡 **重點**：檔名是「**慣例**（convention、約定俗成）」、不是設定。叫 `page.tsx` 就是頁、叫 `layout.tsx` 就是 layout、改名就不認。

  
### 4. 路由 group：用括號分組、不影響 URL

```text
app/
├── (marketing)/                ← 括號 = 不會出現在 URL
│   ├── layout.tsx              ← 只有行銷頁用這 layout
│   ├── about/page.tsx          → /about
│   └── pricing/page.tsx        → /pricing
└── (auth)/
    ├── layout.tsx              ← 登入註冊頁專屬 layout
    ├── login/page.tsx          → /login
    └── signup/page.tsx         → /signup
```

**為什麼要分組？** 行銷頁要置中佈局、登入頁要極簡、後台要 sidebar——不同區用不同 layout、又不想 URL 變 `/marketing/about`。括號就是這個用途。

  
### 5. 第一個 Server Component Page

📄 **寫進程式檔**：`app/page.tsx`

```text
// 預設就是 Server Component（在 server 跑）
export default async function HomePage() {
  // 直接在元件內 await（純 React 不行、RSC 才行）
  const users = await fetch("https://api.example.com/users", {
    next: { revalidate: 3600 }  // ISR：每小時重新驗證
  }).then(r => r.json());

  return (
    <main>
      <h1>使用者列表</h1>
      <ul>
        {users.map(u => <li key={u.id}>{u.name}</li>)}
      </ul>
    </main>
  );
}
```

**重點**：

- `async function`——純 React 元件不能 async、**Server Component 可以**
- 直接 `await fetch(...)`——不用 `useEffect` + `useState` 那套
- `next: { revalidate: 3600 }`——告訴 Next.js 這個 fetch 結果快取 1 小時（ISR）

  
### 6. Root Layout（根 layout）

📄 **寫進程式檔**：`app/layout.tsx`

```text
import "./globals.css";
import { Inter } from "next/font/google";

const inter = Inter({ subsets: ["latin"] });

// metadata（SEO 用、L5 詳教）
export const metadata = {
  title: "AI 島 - 全端養成班",
  description: "從入門到怪物工程師"
};

export default function RootLayout({
  children
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="zh-Hant">
      <body className={inter.className}>
        <nav>導覽列</nav>
        {children}
        <footer>底部</footer>
      </body>
    </html>
  );
}
```

> 💡 **重點**：每個 Next.js 專案有**一個** root layout、必須包 `<html>` 跟 `<body>`、不能放 `<head>`（metadata 自動產）。

  
### 7. 嵌套 Layout（巢狀 layout）

📄 **寫進程式檔**：`app/blog/layout.tsx`

```text
export default function BlogLayout({ children }) {
  return (
    <div className="blog-wrapper">
      <aside>
        <h2>分類</h2>
        <ul><li>技術</li><li>生活</li></ul>
      </aside>
      <main>{children}</main>
    </div>
  );
}
```

訪問 `/blog/post-1` 時、Next.js 自動套 **RootLayout → BlogLayout → page.tsx** 三層。

  
### 8. 跟舊版 pages/ 的差別

```text
| 特性               | App Router（app/）  | Pages Router（pages/） |
|--------------------|---------------------|------------------------|
| 預設元件型態        | Server Component     | Client Component        |
| Layout              | layout.tsx 巢狀     | _app.tsx 一個          |
| Loading            | loading.tsx 自動     | 自己寫                  |
| Error              | error.tsx 自動       | _error.tsx              |
| API                 | route.ts             | api/*.ts                |
| Streaming           | ✓（內建）            | ✗                       |
| Server Action       | ✓                   | ✗                       |
| 建議                | 新專案用這個          | 維護舊專案才用           |
```

**結論**：新專案永遠選 App Router、Pages Router 是 legacy（遺留版本）。

  
### 5 大常見坑

```text
❌ 1. 忘記寫 page.tsx、訪問 404
   → 只有 layout.tsx 不夠、要有 page.tsx
   → 修：每個路由資料夾都要 page.tsx

❌ 2. layout.tsx 裡用 useState
   → Server Component 不能 hook
   → 修：頂端加 "use client" 或拆元件

❌ 3. 在 app/ 外亂放 layout.tsx
   → 不會生效
   → 修：必須在 app/ 樹內

❌ 4. 路由 group 寫成 [marketing]
   → 變動態路由、不是 group
   → 修：用 (marketing) 括號

❌ 5. 在 app/page.tsx 裡 import 整個 lodash
   → bundle 變大
   → 修：只 import 用到的、或用 RSC 在 server 處理
```

  
### 🎯 重點回顧

- **App Router = 資料夾結構就是路由結構**
- **8 個慣例檔名**：page / layout / loading / error / not-found / route / template / default
- **路由 group**：`(name)/` 不會出現在 URL、純粹分組
- **Server Component 預設**——可以 async、直接讀 DB
- **layout 會巢狀套**——root → 區段 → page
- **新專案永遠用 App Router**、Pages Router 是 legacy
