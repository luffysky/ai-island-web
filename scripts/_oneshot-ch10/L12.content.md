**圖片**（image）跟 **字型**（font）是網站最常拖慢首屏的兩個元凶。Next.js 內建 `next/image` 跟 `next/font`——自動優化、零設定、Lighthouse 直接 90+。

> 💡 **這課的價值**：很多人 Next.js 寫得很順、但 Lighthouse 跑分只有 60-70。**99% 原因都是圖片字型沒用內建工具**。用了之後分數立刻 90+、感覺不到的快。

  
### 1. next/image：圖片自動優化

📄 **寫進程式檔**：`app/page.tsx`

```text
import Image from "next/image";

export default function Page() {
  return (
    <Image
      src="/hero.jpg"
      alt="首頁主視覺"
      width={1200}
      height={630}
      priority   // ← 首屏圖加這個、優先載
    />
  );
}
```

**自動做這些**：

```text
✅ 轉 WebP / AVIF（看瀏覽器支援的最佳格式）
✅ 響應式不同尺寸（srcset、手機載小圖、桌機載大圖）
✅ lazy load（除非加 priority）
✅ 防止 layout shift（CLS、Cumulative Layout Shift）
✅ blur placeholder（載入時模糊預覽）
✅ 自動加 width / height（瀏覽器先留空間）
```

  
### 2. fill mode（不知道尺寸時）

如果圖要鋪滿父容器、用 `fill`：

```text
import Image from "next/image";

export default function Card() {
  return (
    <div style={{ position: "relative", width: "100%", height: 300 }}>
      <Image
        src="/photo.jpg"
        alt="..."
        fill
        sizes="(max-width: 768px) 100vw, 50vw"
        style={{ objectFit: "cover" }}
      />
    </div>
  );
}
```

**重點**：
- 父容器要 `position: relative`
- `sizes` 告訴瀏覽器在不同 viewport 下圖會佔多寬（瀏覽器選對 srcset）
- `objectFit: cover` 像 CSS background-size: cover

  
### 3. 遠端圖片設定

預設 Next.js 不允許遠端 URL（防止 host 別人的圖片浪費頻寬）、要白名單：

📄 **寫進程式檔**：`next.config.js`

```text
module.exports = {
  images: {
    remotePatterns: [
      { protocol: "https", hostname: "cdn.example.com" },
      { protocol: "https", hostname: "*.supabase.co" },
      { protocol: "https", hostname: "images.unsplash.com" }
    ],
    formats: ["image/avif", "image/webp"],
    deviceSizes: [640, 750, 828, 1080, 1200, 1920, 2048, 3840],
    imageSizes: [16, 32, 48, 64, 96, 128, 256, 384],
    minimumCacheTTL: 60 * 60 * 24  // 24 小時 CDN cache
  }
};
```

  
### 4. blur placeholder（模糊預覽）

```text
// 本地圖（自動產生 blur）
import heroImg from "@/public/hero.jpg";

<Image
  src={heroImg}
  alt="..."
  placeholder="blur"  // 自動模糊預覽
  priority
/>

// 遠端圖（要自己給 blurDataURL）
<Image
  src="https://cdn.example.com/photo.jpg"
  alt="..."
  width={1200}
  height={630}
  placeholder="blur"
  blurDataURL="data:image/svg+xml;base64,..."
/>
```

  
### 5. fetchpriority（首屏圖優先）

LCP（Largest Contentful Paint、最大內容繪製、首屏最大元素）通常是首屏的大圖。加 `priority` 等於 `fetchpriority="high"`：

```text
// 首屏 hero 圖
<Image src="/hero.jpg" alt="..." width={1200} height={630} priority />

// 其他圖（自動 lazy load）
<Image src="/thumb.jpg" alt="..." width={300} height={200} />
```

> 💡 **重點**：**只有首屏 LCP 元素加 priority**——加太多反而沒用、其他圖會被擠到後面。

  
### 6. next/font：字型優化

📄 **寫進程式檔**：`app/layout.tsx`

```text
import { Inter, Noto_Sans_TC, JetBrains_Mono } from "next/font/google";

const inter = Inter({
  subsets: ["latin"],
  display: "swap",
  variable: "--font-inter"
});

const notoTC = Noto_Sans_TC({
  subsets: ["chinese-traditional"],
  display: "swap",
  weight: ["400", "500", "700"],
  variable: "--font-noto"
});

const mono = JetBrains_Mono({
  subsets: ["latin"],
  variable: "--font-mono"
});

export default function RootLayout({ children }) {
  return (
    <html lang="zh-Hant" className={`${inter.variable} ${notoTC.variable} ${mono.variable}`}>
      <body>{children}</body>
    </html>
  );
}
```

**自動做這些**：

```text
✅ self-host 字型（不用每次跟 Google 連線）
   → GDPR 友善（不傳訪客 IP 給 Google）
   → 加快載入

✅ font-display: swap（先顯示 fallback、字型載完替換）
✅ 預載字型檔（preload）
✅ 防 layout shift（FOIT / FOUT 都避免）
✅ 只載「真的用到」的字符（subset）
   → 繁中字型本來 10MB、subset 後 200KB
```

  
### 7. 自架字型檔（local font）

如果用付費字型 / 自家設計的字型：

📄 **寫進程式檔**：`app/layout.tsx`

```text
import localFont from "next/font/local";

const myFont = localFont({
  src: [
    { path: "../fonts/MyFont-Regular.woff2", weight: "400", style: "normal" },
    { path: "../fonts/MyFont-Bold.woff2", weight: "700", style: "normal" }
  ],
  variable: "--font-my",
  display: "swap"
});

export default function RootLayout({ children }) {
  return (
    <html className={myFont.variable}>
      <body>{children}</body>
    </html>
  );
}
```

  
### 8. CSS 怎麼用字型 variable

```text
/* globals.css */
:root {
  --font-sans: var(--font-inter), var(--font-noto),
               system-ui, -apple-system, "PingFang TC", sans-serif;
  --font-mono: var(--font-mono), "Menlo", monospace;
}

body {
  font-family: var(--font-sans);
}

code, pre {
  font-family: var(--font-mono);
}
```

或 Tailwind：

```text
// tailwind.config.ts
import type { Config } from "tailwindcss";

export default {
  theme: {
    extend: {
      fontFamily: {
        sans: ["var(--font-inter)", "var(--font-noto)", "sans-serif"],
        mono: ["var(--font-mono)", "monospace"]
      }
    }
  }
} satisfies Config;
```

  
### 9. next/script：第三方 JS 載入

不算圖片字型、但同類優化——載 GA / Crisp / Intercom 等第三方腳本：

```text
import Script from "next/script";

export default function Page() {
  return (
    <>
      <Script
        src="https://www.googletagmanager.com/gtag/js?id=G-XXX"
        strategy="afterInteractive"
      />
      <Script id="ga-config" strategy="afterInteractive">{`
        window.dataLayer = window.dataLayer || [];
        function gtag(){dataLayer.push(arguments);}
        gtag('js', new Date());
        gtag('config', 'G-XXX');
      `}</Script>
    </>
  );
}
```

**strategy 選項**：
- `beforeInteractive`：頁面互動前載（polyfill 才用）
- `afterInteractive`（預設）：互動後
- `lazyOnload`：全部 load 完才載（GA、Crisp）
- `worker`：跑在 Web Worker（experimental）

  
### 10. Core Web Vitals 對應

```text
| 指標   | 全名                          | 達標     | 怎麼救            |
|--------|-------------------------------|----------|-------------------|
| LCP    | Largest Contentful Paint      | < 2.5s   | next/image priority |
| FID    | First Input Delay             | < 100ms  | 減少 JS bundle     |
| CLS    | Cumulative Layout Shift       | < 0.1    | width/height 都寫  |
| INP    | Interaction to Next Paint     | < 200ms  | 拆 Client Component |
| TTFB   | Time To First Byte            | < 800ms  | Edge / CDN cache   |
```

  
### 5 大常見坑

```text
❌ 1. 用 <img> 不用 <Image />
   → 沒 WebP、沒響應式、沒 lazy load
   → 修：永遠用 next/image

❌ 2. 沒設 width / height
   → CLS 爆炸（圖載完內容跳）
   → 修：靜態圖一定要 width / height

❌ 3. 所有圖都加 priority
   → 全部優先 = 都不優先、瓜分頻寬
   → 修：只首屏 LCP 圖加

❌ 4. Google Fonts 用 <link> 載
   → 慢、不 self-host、GDPR 問題
   → 修：改用 next/font

❌ 5. 遠端圖沒設 remotePatterns
   → Next.js 不認、報錯
   → 修：next.config.js 加白名單
```

  
### 🎯 重點回顧

- **next/image**：自動 WebP / lazy / responsive、永遠用它
- **priority**：只給首屏 LCP 圖
- **fill mode**：父容器 relative + sizes + objectFit
- **remotePatterns**：遠端圖必設白名單
- **next/font**：self-host、subset、防 layout shift
- **next/script**：第三方 JS、strategy 選 afterInteractive 為主
