**Generic（泛型）= 寫一個 function 或 class，讓它能接受「任何型別」的參數、回傳對應型別**——不用為每個型別重寫一份、又能保留完整型別檢查。**TS 的招牌絕技**。

> 💡 **這課的價值**：不會 Generic、你寫的 TS 一半是 `any`；會 Generic、你寫的工具函式可以重用 10 年、IDE 自動補完到位、bug 在編譯時就被抓出來。

  
### 1. 為什麼需要 Generic？先看痛點

假設你想寫一個「回傳原值」的 function（之後會擴充功能）：

📄 **寫進程式檔**：`utils.ts`

```text
// 寫法 A：寫死 string（只能接 string）
function identityStr(value: string): string {
  return value;
}

// 寫法 B：用 any（型別檢查全失效）
function identityAny(value: any): any {
  return value;
}

const x = identityAny(42);  // x 的型別是 any、之後 .toFixed() 編譯器不會檢查
```

**問題：寫法 A 不能重用、寫法 B 等於放棄 TS**。Generic 就是來解這個問題的。

  
### 2. 基本語法：`<T>` 是什麼？

```text
function identity<T>(value: T): T {
  return value;
}

const a = identity<string>("hello");   // T = string、a 是 string
const b = identity(42);                // TS 自動推 T = number、b 是 number
const c = identity({ name: "Alice" }); // T = { name: string }
```

**`<T>` 怎麼讀？**

```text
function identity<T>(value: T): T
         ↑       ↑       ↑   ↑
         名稱   參數 T  參數型別 回傳型別

T = Type 的縮寫、是「型別變數」（type variable）
   → 像 function 的 (value) 是「值變數」一樣
   → 呼叫時才決定 T 是什麼
```

> 💡 **`<T>` 的位置很重要**：寫在 function 名稱**後面**、參數列**前面**。`function foo<T>(x: T)` 對、`function<T> foo(x: T)` 是錯的。

**為什麼叫 T？**——傳統習慣（Type）、你也可以寫 `<Item>`、`<MyType>`，但 1 個字母慣例是 T、U、V、K（key）、V（value）。

  
### 3. 多個型別參數：`<K, V>`

```text
function pair<K, V>(key: K, value: V): [K, V] {
  return [key, value];
}

const p1 = pair("age", 25);            // [string, number]
const p2 = pair("name", "Alice");      // [string, string]
const p3 = pair(1, true);              // [number, boolean]
```

**用途**：寫 Map、寫 KV store、寫任何「兩種獨立型別」的 API。

  
### 4. 約束（constraint）—— `T extends X`

純 `<T>` 太自由、有時你需要「**至少**得有某些屬性」：

```text
// ❌ 不加約束：x.length 編譯錯（T 可能沒 length）
function logLength<T>(x: T) {
  console.log(x.length);  // Error: T 不一定有 length
}

// ✅ 加約束：T 必須有 length 屬性
function logLength<T extends { length: number }>(x: T) {
  console.log(x.length);
}

logLength("hello");      // string 有 length ✓
logLength([1, 2, 3]);    // array 有 length ✓
logLength({ length: 5, name: "x" });  // 物件有 length ✓
logLength(42);           // ❌ number 沒 length、編譯錯
```

**`extends` 在這裡不是「繼承」、是「型別必須符合這個形狀」**（duck typing、鴨子型別）。

  
### 5. 預設型別參數：`<T = X>`

像 function 預設參數一樣、Generic 也可以有預設值：

```text
function createState<T = string>(initial: T) {
  let state = initial;
  return {
    get: () => state,
    set: (v: T) => { state = v; }
  };
}

const s1 = createState("hi");       // T 推成 string
const s2 = createState<number>(0);  // 明確指定 number
const s3 = createState();           // ⚠️ 預設 T = string、但沒給 initial 會錯
```

  
### 6. Generic Class：通用容器

```text
class Stack<T> {
  private items: T[] = [];

  push(item: T): void {
    this.items.push(item);
  }

  pop(): T | undefined {
    return this.items.pop();
  }

  peek(): T | undefined {
    return this.items[this.items.length - 1];
  }
}

const numStack = new Stack<number>();
numStack.push(1);
numStack.push(2);
const top = numStack.pop();  // top 的型別是 number | undefined

const strStack = new Stack<string>();
strStack.push("hi");
// strStack.push(42);  // ❌ 編譯錯：不是 string
```

**好處**：同一個 Stack class、可以裝 number、string、user object——型別還都對。

  
### 7. 實戰：useFetch hook（React + Generic）

📄 **寫進程式檔**：`useFetch.ts`

```text
import { useState, useEffect } from "react";

interface FetchState<T> {
  data: T | null;
  loading: boolean;
  error: Error | null;
}

function useFetch<T>(url: string): FetchState<T> {
  const [data, setData] = useState<T | null>(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<Error | null>(null);

  useEffect(() => {
    setLoading(true);
    fetch(url)
      .then(r => r.json() as Promise<T>)
      .then(setData)
      .catch(setError)
      .finally(() => setLoading(false));
  }, [url]);

  return { data, loading, error };
}
```

**使用時指定 T**：

```text
interface User { id: number; name: string; email: string; }

function UserPage() {
  // 告訴 useFetch：「我要的型別是 User」
  const { data, loading } = useFetch<User>("/api/me");

  if (loading) return <p>載入中...</p>;
  return <h1>{data?.name}</h1>;  // data.name IDE 自動補完
}
```

**沒有 Generic 就得 `as User` 強制轉型、Generic 讓型別流過整個 hook**。

  
### 8. Generic Function 在 React 元件

```text
interface ListProps<T> {
  items: T[];
  renderItem: (item: T) => React.ReactNode;
}

function List<T>({ items, renderItem }: ListProps<T>) {
  return (
    <ul>
      {items.map((item, i) => (
        <li key={i}>{renderItem(item)}</li>
      ))}
    </ul>
  );
}

// 使用：T 自動推成 User
<List
  items={users}
  renderItem={u => <span>{u.name}</span>}
/>
```

寫一次 List、可以渲染 user、post、product、anything。

  
### 9. 5 大常見坑

```text
❌ 1. 該用 generic 卻用 any
   → 失去型別檢查、IDE 補完爛掉
   → 修：把 any 換成 <T>

❌ 2. <T> 寫錯位置
   → function foo<T>(x: T) ✓
   → function <T> foo(x: T) ✗
   → 修：永遠寫在 function 名稱後面

❌ 3. 沒加 extends 約束、然後用 T.length
   → 編譯錯：T 不一定有 length
   → 修：<T extends { length: number }>

❌ 4. Generic class 沒指定型別、預設 unknown
   → new Stack() 的 T 是 unknown、push 什麼都進
   → 修：new Stack<number>() 明確指定

❌ 5. Generic 寫太多層、自己都看不懂
   → <T, U extends keyof T, V extends T[U]> 已經是地獄
   → 修：拆成多個 type alias、加註解
```

  
### 🎯 重點回顧

- **Generic 是「型別變數」**——讓 function / class 接受任何型別、又保留型別檢查
- **`<T>` 寫在名稱後面、參數列前面**
- **多個型別用 `<K, V>`、約束用 `extends`、預設用 `= X`**
- **Generic class + Generic Function 是 TS 工具庫的基石**——React hook、API 包裝、資料結構通通靠它
- **沒事不要寫超過 3 個型別參數**——可讀性會崩
