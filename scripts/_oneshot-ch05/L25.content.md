**TS 4.9+ 新特性**：`satisfies`（滿足）讓你「**檢查型別但不擦掉具體資訊**」、`const` 型別參數讓 Generic 推得更精確、Template Literal Types（字串模板型別）讓型別系統能玩字串——**2026 業界 TS 主流寫法**。

> 💡 **這課的價值**：很多 senior 寫的 TS 你看不懂、十之八九是這幾個 4.9+ 特性。學會、你寫的 TS 跟 2018 年的 jQuery 程式設計師立刻拉開十年距離。

  
### 1. `satisfies` 算子（4.9 引入、Game Changer）

**先看一個痛點**：

📄 **寫進程式檔**：`config.ts`

```text
// 寫法 A：用 as（型別斷言、強制當成）
const config = {
  host: "localhost",
  port: 3000,
  ssl: false
} as { host: string; port: number; ssl: boolean };

// 問題：config.host 是 string、不是 "localhost" 這個具體值
// 後續 config.port.toFixed() 沒問題
// 但失去了「精確型別」
```

```text
// 寫法 B：明確型別標註
const config: { host: string; port: number; ssl: boolean } = {
  host: "localhost",
  port: 3000,
  ssl: false
};

// 一樣的問題：失去「具體值」資訊
// config.host 是 string、不是 "localhost"
```

**`satisfies` 的妙處 —— 兩全其美**：

```text
const config = {
  host: "localhost",
  port: 3000,
  ssl: false
} satisfies { host: string; port: number; ssl: boolean };

// ✓ 編譯器檢查：欄位都符合
// ✓ 但 config.host 還是 "localhost"（字面值）！
// ✓ config.port 還是 3000（不是 number）
```

> 💡 **`satisfies` vs `as` vs `: Type`**：`as` 強制當成（可能說謊）、`: Type` 標註（會把具體值擦成型別）、`satisfies` 檢查但保留具體資訊。**2026 之後一律用 satisfies**。

  
### 2. `satisfies` 實戰：路由表

```text
// 想要：每個 key 是 page 名字、value 是路徑
const routes = {
  home: "/",
  about: "/about",
  blog: "/blog/:slug",
  user: "/u/:id"
} satisfies Record<string, string>;

// IDE 自動補完：
routes.home;   // "/"（字面值、不是 string）
routes.about;  // "/about"

// 若打錯：
routes.homee;  // ❌ 編譯錯：沒有 homee
```

**沒有 satisfies 的話**：

```text
const routes: Record<string, string> = { ... };

routes.homee;  // ⚠️ 編譯不錯（Record<string, string> 接受任何 key）
// 執行時才崩
```

  
### 3. `const` 型別參數（5.0 引入）

預設 Generic 推論會把字面值「擴展」成 broad type：

```text
function first<T>(arr: T[]): T {
  return arr[0];
}

const x = first(["a", "b", "c"]);
// x 推成 string、不是 "a" | "b" | "c"
```

**加 `const`**：

```text
function first<const T>(arr: readonly T[]): T {
  return arr[0];
}

const x = first(["a", "b", "c"]);
// x 推成 "a" | "b" | "c"（精確）
```

**用途**：寫 router、config、enum-style API 時保留具體字面值。

  
### 4. Template Literal Types（字串模板型別）

讓「型別」也可以拼字串：

```text
type Greeting = `Hello, ${string}`;

const a: Greeting = "Hello, Alice";  // ✓
const b: Greeting = "Hi, Alice";     // ❌ 不是 Hello 開頭

// 進階：把欄位名稱組起來
type EventName<T extends string> = `on${Capitalize<T>}`;

type ClickEvent = EventName<"click">;  // = "onClick"
type HoverEvent = EventName<"hover">;  // = "onHover"
```

**Capitalize / Uncapitalize / Uppercase / Lowercase** 是 TS 內建的字串型別工具。

  
### 5. 實戰：route param 自動提取

```text
type ExtractParams<Path extends string> =
  Path extends `${string}:${infer Param}/${infer Rest}`
    ? Param | ExtractParams<`/${Rest}`>
    : Path extends `${string}:${infer Param}`
      ? Param
      : never;

type UserParams = ExtractParams<"/u/:id/posts/:postId">;
// = "id" | "postId"
```

**這就是 Next.js / React Router 自動推 params 型別的魔法**。

  
### 6. `infer` 關鍵字：從型別「拆」資訊出來

`infer`（inferred、推斷）在 conditional type 內宣告型別變數：

```text
// 從陣列型別取出元素型別
type ElementOf<T> = T extends (infer U)[] ? U : never;

type A = ElementOf<string[]>;          // = string
type B = ElementOf<number[]>;          // = number
type C = ElementOf<{ id: number }[]>;  // = { id: number }
```

**內建的 ReturnType / Awaited 都是用 infer 寫成的**：

```text
type ReturnType<T> = T extends (...args: any) => infer R ? R : never;
type Awaited<T> = T extends Promise<infer U> ? U : T;
```

  
### 7. `as const` —— 全部變字面值

```text
const colors = ["red", "green", "blue"] as const;
// 型別變成 readonly ["red", "green", "blue"]
// 不是 string[]

const config = {
  env: "production",
  debug: false
} as const;
// 型別變成 { readonly env: "production"; readonly debug: false; }
```

**搭配 typeof + Generic 神搭配**：

```text
const STATUSES = ["pending", "approved", "rejected"] as const;
type Status = typeof STATUSES[number];
// = "pending" | "approved" | "rejected"

// 之後可以用：
function setStatus(s: Status) { /* ... */ }
setStatus("pending");   // ✓
setStatus("xxx");       // ❌
```

  
### 8. `NoInfer<T>` (TS 5.4+) —— 阻止推論

當你有多個參數共用 T、想用其中一個決定 T：

```text
function pick<T>(arr: T[], defaultValue: T): T {
  return arr[0] ?? defaultValue;
}

const x = pick(["a", "b"], "c");
// T 推成 "a" | "b" | "c" | string（混亂）

// 用 NoInfer 阻止 defaultValue 影響推論
function pick<T>(arr: T[], defaultValue: NoInfer<T>): T {
  return arr[0] ?? defaultValue;
}

const y = pick(["a", "b"], "c");
// T 只從 arr 推、= "a" | "b"
// defaultValue "c" 編譯錯（不是 "a" | "b"）
```

  
### 9. 實戰：寫一個型別安全的 API client

📄 **寫進程式檔**：`api.ts`

```text
const endpoints = {
  getUser: { method: "GET", path: "/u/:id" },
  createPost: { method: "POST", path: "/posts" },
  deletePost: { method: "DELETE", path: "/posts/:id" }
} as const satisfies Record<string, { method: string; path: string }>;

type Endpoints = typeof endpoints;
type EndpointKey = keyof Endpoints;
// = "getUser" | "createPost" | "deletePost"

async function call<K extends EndpointKey>(
  name: K,
  opts: { params?: Record<string, string>; body?: unknown }
) {
  const { method, path } = endpoints[name];
  // ...
}

call("getUser", { params: { id: "1" } });   // ✓
call("getUserrrr", {});                     // ❌ 編譯錯
```

**`as const satisfies X` 是 2026 最強組合**——保留字面值 + 編譯時檢查結構。

  
### 10. 5 大常見坑

```text
❌ 1. 還在用 as 強制斷言
   → const x = {...} as Config 失去字面值
   → 修：改 satisfies Config

❌ 2. 寫 const arr = [1, 2, 3]、想當 tuple
   → arr 變 number[]、不是 [1, 2, 3]
   → 修：as const

❌ 3. Generic 推論成寬型別、不是字面值
   → 修：function f<const T>(...) 加 const

❌ 4. infer 寫在錯位置
   → infer 只能在 extends 條件式內
   → 修：type X<T> = T extends Y<infer U> ? U : never

❌ 5. 巢狀 template literal type 一直跳
   → 寫 Route param 解析很容易爆炸
   → 修：拆多個 type、加註解、用測試型別驗證
```

  
### 🎯 重點回顧

- **`satisfies`** = 檢查型別但保留具體值——2026 之後完全取代 `as` 跟 `:`
- **`<const T>`** = Generic 推論保留字面值
- **Template Literal Types** = 型別系統能玩字串、拼接、提取
- **`infer`** = 從複雜型別「拆」資訊、寫 utility type 的核心
- **`as const`** = 整個物件 / 陣列變字面值 + readonly
- **`as const satisfies X`** 是 2026 業界最強組合
