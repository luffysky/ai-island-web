**表單進階：HTML5 驗證 + Constraint Validation API + accessible 錯誤提示**——L8 教過基礎、這課深入「驗證怎麼做才對 UX + a11y 都好」。

> 💡 **這課的價值**：90% 的表單錯誤訊息寫得很爛——只在按 submit 後跳 alert、a11y 慘、體驗也慘。學會這課寫的表單會跟頂級 SaaS 公司同等級。

  
### 1. HTML5 內建驗證屬性（複習）

```text
required             必填
minlength="8"        最少 8 字
maxlength="50"       最多 50 字
min="0" / max="100"  數字範圍
pattern="..."        正則驗證
type="email"         email 格式驗證
type="url"           URL 格式驗證
type="tel"           電話（不嚴格、用 pattern 補）
```

**順序：HTML 驗證 → JS 驗證 → Server 驗證**——三層都要做、前兩層體驗、第三層安全。

  
### 2. Constraint Validation API（JS 控制 HTML5 驗證）

```text
const input = document.getElementById('email');

input.validity;
  // ValidityState 物件
  // .valid          整體有效嗎
  // .valueMissing   required 沒填
  // .tooShort       < minlength
  // .tooLong        > maxlength
  // .rangeUnderflow < min
  // .rangeOverflow  > max
  // .patternMismatch 不符 pattern
  // .typeMismatch   type 不對（email 沒 @）
  // .badInput       不合法輸入（type=number 輸入字母）

input.validationMessage;
  // 預設錯誤訊息（瀏覽器中文 / 英文依系統）

input.setCustomValidity('密碼必須含大小寫字母 + 數字');
  // 自訂錯誤訊息（傳空字串清除）

input.checkValidity();
  // 檢查、回 true/false，無效時觸發 invalid event

input.reportValidity();
  // 同上 + 顯示瀏覽器預設 tooltip
```

  
### 3. 即時驗證（input event）

```text
<form>
  <label for="email">Email</label>
  <input id="email" type="email" required>
  <span id="email-err" role="alert"></span>

  <button type="submit">送出</button>
</form>

<script>
  const email = document.getElementById('email');
  const err = document.getElementById('email-err');

  email.addEventListener('input', () => {
    if (email.validity.valueMissing) {
      err.textContent = '請輸入 Email';
    } else if (email.validity.typeMismatch) {
      err.textContent = 'Email 格式不對（缺 @ 或 .）';
    } else {
      err.textContent = '';
    }
  });
</script>
```

**重點**：
- `role="alert"` → 螢幕閱讀器立刻念
- input event 而非 change（即時、不用等失焦）

  
### 4. 客製化驗證（複雜邏輯）

```text
const pwd = document.getElementById('password');
const confirmPwd = document.getElementById('confirm-password');

confirmPwd.addEventListener('input', () => {
  if (confirmPwd.value !== pwd.value) {
    confirmPwd.setCustomValidity('密碼不一致');
  } else {
    confirmPwd.setCustomValidity('');  // 清除錯誤
  }
});
```

**強密碼驗證：**

```text
function checkStrong(pwd) {
  if (pwd.length < 8) return '至少 8 字';
  if (!/[A-Z]/.test(pwd)) return '至少 1 個大寫';
  if (!/[a-z]/.test(pwd)) return '至少 1 個小寫';
  if (!/[0-9]/.test(pwd)) return '至少 1 個數字';
  if (!/[!@#$%^&*]/.test(pwd)) return '至少 1 個特殊符號';
  return '';
}

pwd.addEventListener('input', () => {
  pwd.setCustomValidity(checkStrong(pwd.value));
});
```

### 6. 完整實戰：a11y 友善的註冊表單

📄 **寫進程式檔**：`signup-form.html`

```text
<!DOCTYPE html>
<html lang="zh-Hant">
<head>
  <meta charset="UTF-8">
  <title>註冊</title>
  <style>
    .field { margin: 12px 0; }
    label { display: block; font-weight: 600; }
    input { width: 100%; padding: 8px; }
    .error { color: #ef4444; font-size: 14px; }
    .error:not(:empty)::before { content: "⚠️ "; }
    input:user-invalid { border-color: #ef4444; }  /* 只在 user 嘗試後才標紅 */
    input:user-valid { border-color: #10b981; }
  </style>
</head>
<body>
  <h1>建立帳號</h1>

  <form id="signup-form" novalidate>
    <div class="field">
      <label for="email">Email *</label>
      <input
        id="email" name="email"
        type="email" required
        autocomplete="email"
        aria-describedby="email-err"
      >
      <span id="email-err" class="error" role="alert" aria-live="polite"></span>
    </div>

    <div class="field">
      <label for="pwd">密碼 * <small>（至少 8 字、含大小寫、數字、特殊符號）</small></label>
      <input
        id="pwd" name="password"
        type="password" required minlength="8"
        autocomplete="new-password"
        aria-describedby="pwd-err pwd-hint"
      >
      <span id="pwd-hint" class="hint" style="font-size: 13px; color: #666;">
        強度：<span id="pwd-strength">—</span>
      </span>
      <span id="pwd-err" class="error" role="alert" aria-live="polite"></span>
    </div>

    <div class="field">
      <label for="confirm">確認密碼 *</label>
      <input
        id="confirm" name="confirm"
        type="password" required
        autocomplete="new-password"
        aria-describedby="confirm-err"
      >
      <span id="confirm-err" class="error" role="alert" aria-live="polite"></span>
    </div>

    <button type="submit">註冊</button>
  </form>

  <script>
    const form = document.getElementById('signup-form');
    const email = document.getElementById('email');
    const pwd = document.getElementById('pwd');
    const confirm = document.getElementById('confirm');
    const emailErr = document.getElementById('email-err');
    const pwdErr = document.getElementById('pwd-err');
    const confirmErr = document.getElementById('confirm-err');
    const strength = document.getElementById('pwd-strength');

    email.addEventListener('input', () => {
      if (email.validity.valueMissing) emailErr.textContent = '請輸入 Email';
      else if (email.validity.typeMismatch) emailErr.textContent = '格式不對';
      else emailErr.textContent = '';
    });

    pwd.addEventListener('input', () => {
      const v = pwd.value;
      let score = 0;
      if (v.length >= 8) score++;
      if (/[A-Z]/.test(v)) score++;
      if (/[a-z]/.test(v)) score++;
      if (/[0-9]/.test(v)) score++;
      if (/[!@#$%^&*]/.test(v)) score++;

      const levels = ['—', '太弱', '普通', '不錯', '強', '超強'];
      strength.textContent = levels[score];

      pwd.setCustomValidity(score < 4 ? '密碼強度不夠' : '');
      pwdErr.textContent = pwd.validity.customError ? '至少滿足 4 項條件' : '';
    });

    confirm.addEventListener('input', () => {
      if (confirm.value !== pwd.value) {
        confirm.setCustomValidity('兩次密碼不一致');
        confirmErr.textContent = '密碼不一致';
      } else {
        confirm.setCustomValidity('');
        confirmErr.textContent = '';
      }
    });

    form.addEventListener('submit', (e) => {
      e.preventDefault();
      if (!form.checkValidity()) {
        // 找第一個無效欄位 focus
        const firstInvalid = form.querySelector(':invalid');
        firstInvalid?.focus();
        return;
      }
      // 送出 → 通常 fetch('/api/signup', ...)
      console.log('OK，可送出');
    });
  </script>
</body>
</html>
```

**做對的事：**
- `novalidate` 關閉瀏覽器預設 UI、自己控制
- `:user-invalid` / `:user-valid` 只在 user 嘗試後才標色（不是一開頭就紅紅綠綠）
- `aria-describedby` 連 input 跟錯誤訊息
- `role="alert"` + `aria-live="polite"` 螢幕閱讀器念出錯誤
- 即時驗證（input event）+ submit 攔截
- submit 失敗自動 focus 第一個無效欄位

  
### 7. 5 大常見坑

```text
❌ 1. 按 submit 才驗證
   → 使用者填到第 5 欄才知道第 1 欄錯了
   → 修：input event 即時驗

❌ 2. 一開頭就標紅
   → 「Email 必填」用戶還沒輸入就紅紅紅
   → 修：用 :user-invalid 替代 :invalid

❌ 3. 錯誤訊息沒 role="alert"
   → 螢幕閱讀器看不到
   → 修：加 role="alert" + aria-live="polite"

❌ 4. error 跟 input 沒關聯
   → screen reader focus input 不知道錯誤是什麼
   → 修：aria-describedby="err-id"

❌ 5. 只依賴前端驗證
   → API 還是可以收到亂搞的資料
   → 修：Server 必須再驗一次（永遠不信任前端）
```

  
### 🎯 重點回顧

- **三層驗證**：HTML → JS → Server
- **Constraint Validation API** = JS 控制 HTML5 驗證
- **即時驗證 + `:user-invalid` 給最好的 UX**
- **a11y 錯誤訊息**：role="alert" + aria-live="polite" + aria-describedby
- **submit 失敗自動 focus 第一個無效欄位**
