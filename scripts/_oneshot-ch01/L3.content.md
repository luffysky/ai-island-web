**語意化標籤（Semantic HTML）：HTML5 的革命**——HTML5 之前、所有東西都用 `<div>`。HTML5 加入語意化標籤、**讓機器讀得懂網頁結構**。

> 💡 **核心心法**：用對標籤、Google 排名 +15-30%（Google 自己說的）。**只是換個標籤名、SEO 免費加分**。

  
### div soup vs 語意化 HTML

```text
❌ div soup（HTML5 前的寫法）：

<div class="header">
  <div class="nav">
    <div class="link"><a>Home</a></div>
  </div>
</div>
<div class="main">
  <div class="article">
    <div class="title"><h1>標題</h1></div>
  </div>
</div>

→ 機器看：一堆 div、不知道哪是 header / main
→ Google：找不到主要內容
→ 螢幕閱讀器：盲人無法跳到主內容

✅ 語意化 HTML5：

<header>
  <nav>
    <a href="/">Home</a>
  </nav>
</header>
<main>
  <article>
    <h1>標題</h1>
  </article>
</main>

→ 機器一看就知道：這是 header / main / article
→ Google：精準抓主內容
→ 螢幕閱讀器：盲人按快捷鍵跳到 <main>
```

  
### 完整語意化範本

📄 **寫進程式檔**：`index.html`

```text
<!-- ====== 頁面頂部 ====== -->
<header>
  <nav>
    <ul>
      <li><a href="/">首頁</a></li>
      <li><a href="/about">關於</a></li>
      <li><a href="/contact">聯絡</a></li>
    </ul>
  </nav>
</header>

<!-- ====== 主要內容 ====== -->
<main>
  <article>
    <header>
      <h1>文章標題</h1>
      <time datetime="2026-05-27">2026/5/27</time>
    </header>
    
    <section>
      <h2>第一段</h2>
      <p>段落內容...</p>
    </section>
    
    <section>
      <h2>第二段</h2>
      <p>段落內容...</p>
    </section>
    
    <aside>
      <p>側邊補充（不是主內容、但相關）</p>
    </aside>
  </article>
</main>

<!-- ====== 頁腳 ====== -->
<footer>
  <p>&copy; 2026 AI 島</p>
</footer>
```

  
### 8 大語意化標籤

| 標籤 | 用途 | 範例 |
|---|---|---|
| **`<header>`** | 頁首或區塊頂部 | 網站 Logo + nav / 文章標題區 |
| **`<nav>`** | 主要導覽 | 主選單 / 側邊選單 |
| **`<main>`** | 主要內容（**一頁只一個**）| 內文區 |
| **`<article>`** | 獨立、可抽出的內容 | 部落格文 / 產品卡 / 留言 |
| **`<section>`** | 主題區塊 | 文章內的 chapter |
| **`<aside>`** | 旁邊補充 | 廣告 / 相關連結 / 註解 |
| **`<footer>`** | 頁腳或區塊底部 | 版權 / 聯絡 / 連結 |
| **`<figure>` + `<figcaption>`** | 圖 + 說明 | 圖表 / 程式碼示意 |

  
### section vs article vs div：什麼時候用哪個？

**判斷法**：「**這塊內容能不能丟到 RSS 給人讀？**」

```text
✅ 能 → 用 <article>
   - 部落格文
   - 產品卡（每個產品獨立）
   - 留言（每則獨立）
   - 新聞條目

✅ 不能（但有主題）→ 用 <section>
   - 文章內的「介紹」「方法」「結論」段
   - 首頁的「Hero」「Features」「Pricing」區
   - 每個 <section> 應該有自己的 <h2>

✅ 純樣式 wrapper（無語意）→ 用 <div>
   - 純粹為了 CSS 排版
   - 沒有語意意義
```

  
### 真實案例：部落格頁面

```text
<body>
  <header>                    <!-- 網站頂部 -->
    <nav>...</nav>
  </header>
  
  <main>
    <article>                 <!-- 一篇文章（可抽出 RSS）-->
      <header>                <!-- 文章頂部 -->
        <h1>文章標題</h1>
        <time>2026-05-27</time>
        <p>作者：Luffy</p>
      </header>
      
      <section>               <!-- 文章內第 1 部分 -->
        <h2>介紹</h2>
        <p>...</p>
      </section>
      
      <section>               <!-- 文章內第 2 部分 -->
        <h2>方法</h2>
        <p>...</p>
        <figure>
          <img src="diagram.png" alt="架構圖">
          <figcaption>圖 1：系統架構</figcaption>
        </figure>
      </section>
      
      <aside>                 <!-- 文章內的補充 -->
        <h3>延伸閱讀</h3>
        <ul>...</ul>
      </aside>
      
      <footer>                <!-- 文章底部 -->
        <p>分類：HTML / CSS</p>
      </footer>
    </article>
    
    <aside>                   <!-- 整頁的補充（非文章內）-->
      <h2>熱門文章</h2>
      <ul>...</ul>
    </aside>
  </main>
  
  <footer>                    <!-- 網站頁腳 -->
    <p>&copy; 2026 我的部落格</p>
  </footer>
</body>
```

> 💡 **`<header>` / `<footer>` 可以重複用**——網站有一個、每個 article 也可以有自己的。

  
### 更多語意化標籤

```text
<time datetime="2026-05-27">2026/5/27</time>
   → 時間
   → datetime 給機器看、文字給人看

<address>
   作者：Luffy<br>
   email: luffy@example.com
</address>
   → 聯絡資訊
   → 不要拿來當地址用（雖然名字像）

<details>
  <summary>點擊展開</summary>
  <p>展開後的內容</p>
</details>
   → 可摺疊區塊
   → 不用 JS 就有展開效果

<mark>螢光標記</mark>
   → 強調 / 突顯
   → 像螢光筆畫線

<dialog>
   → 對話框 / modal
   → HTML5 內建、不用 JS lib
```

  
### 為什麼語意化重要：4 大原因

```text
🎯 SEO 加分
   → Google 用語意化判斷主內容
   → 排名 +15-30%

♿ 無障礙
   → 螢幕閱讀器讀懂結構
   → 盲人能跳到 <main>

🤖 AI 友善
   → Perplexity / ChatGPT 抓內容
   → 看 <article> 知道哪段是文章

📱 RSS / 摘要
   → <article> 直接變 RSS item
   → 自動摘要工具看 <main>
```

  
### Outline algorithm（HTML5 文件大綱）

**HTML5 規範**：每個 `<article>` / `<section>` / `<aside>` / `<nav>` 都應該有自己的 `<h1>-<h6>`、瀏覽器自動產生「outline」（像 Word 大綱）。

```text
<article>
  <h1>文章標題</h1>       <!-- 文章標題 -->
  <section>
    <h2>第一段</h2>        <!-- 文章內 h2 -->
    <section>
      <h3>細節 A</h3>      <!-- 第一段下的 h3 -->
    </section>
  </section>
</article>

→ Google / 工具自動產生大綱
→ 對長文章 SEO 特別有利
```

  
### 5 大常見坑

```text
1. ❌ 一頁多個 <main>
   ✅ 一頁只能一個 <main>

2. ❌ <section> 沒給 <h2>
   ✅ 每個 section 都應有 heading

3. ❌ 為了排版用 <article>
   ✅ <article> 是「**獨立內容**」、不是排版工具

4. ❌ <header> 只能在最頂端
   ✅ 每個 article / section 都可以有自己的 <header>

5. ❌ <nav> 用在所有連結
   ✅ <nav> 只用在「主要導覽」、不是 footer 的連結 list
```

    **☕ 用人話講**
    
語意化 HTML 5 件事：(1) **div soup → 語意標籤**：header / nav / main / article / section / aside / footer (2) **判斷 article vs section**：能不能丟 RSS → article (3) **一頁只能一個 `<main>`** (4) **`<header>` / `<footer>` 可重複用**（網站有、article 也有） (5) **SEO + 無障礙 + AI 友善** 3 合 1。**心法**：免費加分、只是換標籤名、別繼續寫 div soup。

    語意化 HTML 是「**靜默加分**」：(1) **客戶看不到差別**（畫面一樣）—— 但 SEO 排名上升 (2) **「**SEO 友善網站**」是高價賣點** —— 客戶搜不到就會找你 (3) **無障礙合規** —— 政府 / 大企業案件常要求 WCAG (4) **AI 引擎引用** —— 2026+ ChatGPT / Perplexity 引用網站、語意化內容被引用機率高。**心法**：別小看「換標籤」、語意化是「**1 小時改、10 年 SEO 受益**」工程。
