**無障礙 a11y 進階：ARIA 屬性 + 鍵盤導航**——L11 教過 a11y 基本盤、這課深入「**互動元件**」怎麼做到完全 a11y：自訂下拉、Modal、Tab、Accordion 都靠 ARIA + 鍵盤 + focus 管理。

> 💡 **這課的價值**：用 Bootstrap / Tailwind UI 寫的 modal / dropdown 表面上能動、但鍵盤使用者跟盲人完全不能用是業界常態。寫對這課讓你的網站從「能用」變「人人能用」。

  
### 1. ARIA roles vs HTML5 元素（再強調）

```text
✅ <button onclick="...">確認</button>
   → 自動 tabindex / Enter / Space 觸發 / role=button / 螢幕閱讀器念「button」

❌ <div onclick="...">確認</div>
   → 鍵盤無法 focus、按 Enter 沒反應、screen reader 無視
   → 要補：tabindex="0" + role="button" + onkeydown + aria-label

→ 結論：能用 HTML5 元素就用，ARIA 只是補丁。
```

  
### 2. ARIA 屬性五大家族

```text
🏷️ Label / Description
   aria-label          → 取代可見文字（icon 按鈕必加）
   aria-labelledby     → 連到其他元素當標籤
   aria-describedby    → 連到「補充說明」

📊 State
   aria-expanded       → 展開 / 收合（下拉、Accordion）
   aria-selected       → 選中（Tab、列表）
   aria-checked        → 勾選（自訂 checkbox）
   aria-disabled       → 禁用（替代 disabled）
   aria-hidden         → 隱藏（裝飾 icon）

📢 Live region
   aria-live="polite"      → 等 user 念完再念
   aria-live="assertive"   → 立刻打斷（緊急錯誤）
   aria-atomic="true"      → 整段一起念
   role="status"           → polite 快捷
   role="alert"            → assertive 快捷

🔗 Relationships
   aria-controls       → 「我控制這個元素」
   aria-owns           → 「我擁有這個子元素」
   aria-haspopup       → 「按下會開彈窗」

🎭 Role
   role="dialog" / "alertdialog"
   role="menu" / "menuitem"
   role="tablist" / "tab" / "tabpanel"
   role="combobox" / "listbox" / "option"
```

  
### 3. 焦點管理（focus management）

**3 大規則：**

```text
1. modal 開啟時：把焦點移進 modal 第一個可 focus 元素
2. modal 關閉時：把焦點還回原本觸發按鈕
3. 焦點絕對不能跑到 modal 外（focus trap）
```

**Focus trap 最簡實作：**

```text
function trapFocus(modal) {
  const focusable = modal.querySelectorAll(
    'a[href], button, textarea, input, select, [tabindex]:not([tabindex="-1"])'
  );
  const first = focusable[0];
  const last = focusable[focusable.length - 1];

  modal.addEventListener('keydown', (e) => {
    if (e.key !== 'Tab') return;
    if (e.shiftKey && document.activeElement === first) {
      e.preventDefault(); last.focus();
    } else if (!e.shiftKey && document.activeElement === last) {
      e.preventDefault(); first.focus();
    }
  });
}
```

  
### 4. 完整實戰：a11y Modal

📄 **寫進程式檔**：`a11y-modal.html`

```text
<!DOCTYPE html>
<html lang="zh-Hant">
<head>
  <meta charset="UTF-8">
  <title>A11y Modal</title>
  <style>
    [hidden] { display: none !important; }
    .overlay { position: fixed; inset: 0; background: rgba(0,0,0,.5); display: flex; align-items: center; justify-content: center; }
    .modal { background: white; padding: 24px; border-radius: 8px; min-width: 300px; }
    .skip-link { position: absolute; left: -9999px; }
    .skip-link:focus { left: 8px; top: 8px; background: black; color: white; padding: 8px; z-index: 999; }
    :focus-visible { outline: 2px solid #10b981; outline-offset: 2px; }
  </style>
</head>
<body>
  <a href="#main" class="skip-link">跳到主要內容</a>

  <main id="main">
    <button id="open-btn" aria-haspopup="dialog" aria-controls="signup-modal">註冊</button>
  </main>

  <div
    id="signup-modal"
    class="overlay"
    role="dialog"
    aria-modal="true"
    aria-labelledby="modal-title"
    aria-describedby="modal-desc"
    hidden
  >
    <div class="modal">
      <h2 id="modal-title">建立帳號</h2>
      <p id="modal-desc">填寫下面表單建立 AI 島帳號。</p>

      <form>
        <label for="email">Email</label>
        <input id="email" type="email" required autocomplete="email">

        <label for="pwd">密碼（至少 8 字）</label>
        <input id="pwd" type="password" required minlength="8" autocomplete="new-password">

        <button type="submit">建立</button>
        <button type="button" id="cancel-btn">取消</button>
      </form>

      <div id="status" role="status" aria-live="polite"></div>
    </div>
  </div>

  <script>
    const openBtn = document.getElementById('open-btn');
    const cancelBtn = document.getElementById('cancel-btn');
    const modal = document.getElementById('signup-modal');
    let lastFocus = null;

    function openModal() {
      lastFocus = document.activeElement;
      modal.hidden = false;
      // 焦點移進去
      document.getElementById('email').focus();
      // 焦點 trap
      modal.addEventListener('keydown', trapTab);
      document.addEventListener('keydown', escClose);
    }

    function closeModal() {
      modal.hidden = true;
      modal.removeEventListener('keydown', trapTab);
      document.removeEventListener('keydown', escClose);
      lastFocus?.focus();  // 還回原按鈕
    }

    function escClose(e) { if (e.key === 'Escape') closeModal(); }

    function trapTab(e) {
      if (e.key !== 'Tab') return;
      const focusable = modal.querySelectorAll('input, button');
      const first = focusable[0];
      const last = focusable[focusable.length - 1];
      if (e.shiftKey && document.activeElement === first) {
        e.preventDefault(); last.focus();
      } else if (!e.shiftKey && document.activeElement === last) {
        e.preventDefault(); first.focus();
      }
    }

    openBtn.addEventListener('click', openModal);
    cancelBtn.addEventListener('click', closeModal);
    modal.addEventListener('click', (e) => { if (e.target === modal) closeModal(); });
  </script>
</body>
</html>
```

**做對的 a11y 細節：**
- `role="dialog"` + `aria-modal="true"` 標明 modal
- `aria-labelledby` + `aria-describedby` 連到標題 + 說明
- Esc 可關
- 點背景可關
- 開啟 focus 進去、關閉 focus 還回去
- Tab focus trap
- `role="status" aria-live="polite"` 給動態訊息

  
### 5. 鍵盤導航必懂

```text
Tab           下一個可 focus
Shift + Tab   上一個可 focus
Enter         觸發 button / 提交 form
Space         觸發 button / 勾 checkbox
方向鍵         menu / radio / tablist / slider 內導航
Esc           關 modal / popover
Home / End    listbox / menu 首尾
```

**`tabindex` 3 種值：**

```text
tabindex="0"   → 加入 Tab 順序、用 DOM 順序
tabindex="-1"  → 不在 Tab 順序、但 JS 可 .focus()
tabindex="1"+  → 強改順序（永遠不要用！）
```

  
### 6. Skip Links（跳過導覽）

```text
<a href="#main" class="skip-link">跳到主要內容</a>
<a href="#footer" class="skip-link">跳到頁尾</a>

<style>
  .skip-link {
    position: absolute; left: -9999px;
  }
  .skip-link:focus {
    left: 8px; top: 8px;
    background: black; color: white; padding: 8px;
    z-index: 9999;
  }
</style>
```

**為什麼**：鍵盤使用者每頁都要 Tab 過 30+ 個導覽連結才到主內容、超痛苦。Skip link 一鍵跳過。

  
### 7. 5 大常見坑

```text
❌ 1. modal 開啟焦點沒移進去
   → 鍵盤使用者完全不知道 modal 開了
   → 修：自動 focus 第一個 input

❌ 2. focus trap 沒做
   → Tab 焦點跑到 modal 外
   → 修：包 trapTab + 處理 Tab/Shift+Tab

❌ 3. tabindex 用 1, 2, 3...
   → 順序混亂、永遠別這樣
   → 修：只用 0 或 -1

❌ 4. aria-label 跟可見文字衝突
   → screen reader 念 aria-label 不念可見文字
   → 修：可見文字夠清楚就不寫 aria-label（避免雙重念）

❌ 5. aria-hidden="true" 包含可 focus 元素
   → screen reader 跳過、但鍵盤還是 focus 得到（鬼焦點）
   → 修：aria-hidden 配 tabindex="-1" 用
```

  
### 🎯 重點回顧

- **ARIA roles 補 HTML5 元素的不足、能用 button 就不要 div role=button**
- **5 大家族**：Label / State / Live region / Relationships / Role
- **Modal 三件套**：focus 移進去 + trap + 還原
- **鍵盤導航**：Tab / Enter / Space / Esc / 方向鍵各得其所
- **Skip link** 是鍵盤使用者的救命稻草
