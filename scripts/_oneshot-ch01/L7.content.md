**`<table>` 表格**——專門放「兩個維度交叉」的資料：價目表、課表、比較表、財報。**絕對不要用來排版**——那是 1999 年的做法、現代用 Flexbox / Grid（Ch02 教）。

> ⚠️ **新手最大誤區**：「我想做兩欄並排、用 table 排版」——不可以。表格只用於「真的有縱橫對應關係的資料」。版面用 CSS。

  
### 1. 最基本的表格

```text
<table>
  <thead>
    <tr>
      <th>方案</th>
      <th>價格</th>
      <th>儲存空間</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>免費版</td>
      <td>NT$0</td>
      <td>5GB</td>
    </tr>
    <tr>
      <td>Pro 版</td>
      <td>NT$299/月</td>
      <td>100GB</td>
    </tr>
  </tbody>
</table>
```

**5 個核心標籤：**

```text
<table>     整個表格容器
  <thead>   表頭區（標題列）
  <tbody>   主體區（資料列）
  <tfoot>   底部區（小計/總計、選用）
  <tr>      一列（table row）
  <th>      表頭儲存格（會自動粗體置中）
  <td>      資料儲存格（table data）
</table>
```

> 💡 **為什麼分 thead / tbody / tfoot**：(1) 螢幕閱讀器念表頭給盲人聽 (2) 印表機長表格自動每頁印 thead (3) JS 排序時不會把標題列也排進去。

  
### 2. `<th>` vs `<td>` 的差別

```text
<th>  表頭儲存格
      → 自動加粗 + 置中
      → 螢幕閱讀器認得是「標題」
      → 配 scope 屬性更精確

<td>  一般資料儲存格
      → 不加樣式
      → 普通內容
```

**`<th scope>` 屬性 — 告訴瀏覽器「我是直的還是橫的標題」：**

```text
<table>
  <thead>
    <tr>
      <th scope="col">商品</th>     <!-- 直行標題 -->
      <th scope="col">價格</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th scope="row">蘋果</th>     <!-- 橫列標題 -->
      <td>30 元</td>
    </tr>
  </tbody>
</table>
```

> 💡 **`scope` 是無障礙必加**——盲人用螢幕閱讀器掃表格、有 scope 才念得對。

  
### 3. 合併儲存格：`colspan` / `rowspan`

```text
<!-- colspan="2" — 橫向合併 2 格 -->
<tr>
  <th colspan="2">2026 上半年銷售</th>
</tr>
<tr>
  <th>商品</th>
  <th>金額</th>
</tr>

<!-- rowspan="2" — 縱向合併 2 格 -->
<tr>
  <td rowspan="2">台灣區</td>
  <td>台北</td>
</tr>
<tr>
  <td>高雄</td>
</tr>
```

**視覺效果：**

```text
colspan="2"
┌───────────────┐
│ 2026 上半年   │
├───────┬───────┤
│ 商品  │ 金額  │
└───────┴───────┘

rowspan="2"
┌────────┬───────┐
│        │ 台北  │
│ 台灣區 ├───────┤
│        │ 高雄  │
└────────┴───────┘
```

  
### 4. `<caption>` 表格標題（無障礙必加）

```text
<table>
  <caption>2026 各方案功能比較</caption>
  <thead>...</thead>
  <tbody>...</tbody>
</table>
```

**特色：**
- 永遠出現在表格上方（CSS 可移到下方）
- 螢幕閱讀器先念 caption、讓盲人知道這表格在講什麼
- SEO 也吃這個

### 6. RWD 表格（手機怎麼辦）

**問題**：表格在手機螢幕一定爆寬。

**解法 1：水平捲動（最簡單）**

```text
<div style="overflow-x: auto;">
  <table>...</table>
</div>
```

**解法 2：手機改卡片排（CSS 改 display）**

```text
@media (max-width: 600px) {
  table, thead, tbody, tr, th, td {
    display: block;
  }
  thead { display: none; }  /* 隱藏原表頭 */
  tr { border: 1px solid #ddd; margin-bottom: 8px; padding: 8px; }
  td::before { content: attr(data-label) ": "; font-weight: bold; }
}
```

```text
<td data-label="價格">NT$299</td>
```

> 💡 **解法 1 最務實**——除非設計師堅持卡片版型、不然 `overflow-x: auto` 已經夠用。

  
### 7. 5 大常見坑

```text
❌ 1. 用 table 排版（兩欄並排、置中…）
   → 完全錯、無障礙爛 + RWD 痛苦
   → 修：Flexbox / Grid（Ch02 教）

❌ 2. <th> 沒寫 scope
   → 螢幕閱讀器念錯
   → 修：永遠寫 scope="col" 或 scope="row"

❌ 3. 沒寫 <caption>
   → 盲人不知道表格在講什麼
   → 修：每個表格都加 caption

❌ 4. 用 <br> 換行排「假表格」
   → 機器看不懂、Excel 複製不出來
   → 修：是表格就好好寫 table

❌ 5. 手機沒處理表格
   → 直接爆寬、橫向 scrollbar 都看不到
   → 修：包 overflow-x:auto 或 RWD media query
```

  
### 🎯 重點回顧

- **表格 = 兩維資料、不是排版工具**
- **`<thead>` + `<tbody>` + `<th scope>` + `<caption>` 是無障礙基本盤**
- **合併用 `colspan` / `rowspan`、不要用 CSS 假裝**
- **手機最簡解：包 `overflow-x: auto`**
- **動態資料量大、用 React / Vue 表格元件**（虛擬滾動、分頁）

> 💡 **下一課（L8 表單）**：HTML 最複雜也最有用的部分——登入、註冊、留言、買東西全靠它。
