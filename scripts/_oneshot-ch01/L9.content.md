**SEO meta + Open Graph + Twitter Card**——這 3 個東西決定：你的網頁在 Google 搜尋結果長什麼樣、在 LINE / FB / Discord / Twitter 分享時長什麼樣。**寫得好點擊率差 5-10 倍**。

> 💡 **這課的價值**：很多人花 100 小時寫網頁、卻不花 10 分鐘寫 meta、結果 Google 抓內容亂抓、社群分享預覽醜爆。**meta 是寫網頁最高 ROI 的 10 分鐘**。

  
### 1. SEO meta 三鐵律

放在 `<head>` 內：

```text
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">

  <!-- 🔥 SEO 必填 3 大 -->
  <title>AI 島 — 從 0 到 1 學寫程式</title>
  <meta name="description" content="75 章 1158+ 課、從 HTML / CSS / JS 到 AI 全包。台灣最完整新手 onboarding。">
  <link rel="canonical" href="https://ai-island-web.snowrealm.pet/">
</head>
```

**3 個都重要：**

```text
🏷️ <title>
   → Google 搜尋結果的「藍字標題」
   → 分頁標籤上的文字
   → 規格：50-60 字內、塞最重要關鍵字在前 30 字
   → 每頁不一樣！（不要全站同一個 title）

📝 <meta name="description">
   → Google 搜尋結果「藍字下面的描述」
   → 規格：120-160 字內、要吸引人點
   → 不寫 Google 自己抓內文亂抓

🔗 <link rel="canonical">
   → 「我這頁的正本 URL 在這裡」
   → 同樣內容有多 URL（?utm=xxx）時告訴 Google 哪個是正本
   → 防止「重複內容」扣 SEO 分
```

> 💡 **title 寫法**：`頁面主題 — 品牌名` 比 `品牌名 — 頁面主題` 好。例：✅ `Next.js 教學 — AI 島` ❌ `AI 島 — Next.js 教學`。前 30 字最重要。

  
### 2. Open Graph（FB / LINE / Discord / Slack 都吃）

```text
<!-- Open Graph 5 大必填 -->
<meta property="og:type" content="website">
<meta property="og:url" content="https://ai-island-web.snowrealm.pet/">
<meta property="og:title" content="AI 島 — 從 0 到 1 學寫程式">
<meta property="og:description" content="75 章 1158+ 課、從 HTML / CSS / JS 到 AI 全包。">
<meta property="og:image" content="https://ai-island-web.snowrealm.pet/og-cover.png">

<!-- 進階（推薦加）-->
<meta property="og:image:width" content="1200">
<meta property="og:image:height" content="630">
<meta property="og:image:alt" content="AI 島 75 章課程地圖">
<meta property="og:locale" content="zh_TW">
<meta property="og:site_name" content="AI 島">
```

**`og:image` 規格：**

```text
✅ 尺寸：1200 × 630 px（FB 官方）
✅ 比例：1.91:1
✅ 格式：PNG / JPG（不要 WebP、相容性差）
✅ 大小：< 5 MB
✅ 重要內容放中間（兩側可能被裁）
```

> 💡 **沒寫 og:image 會超慘**：FB / LINE 分享預覽會「沒圖」或「自動抓首頁第一張」（通常很醜）。OG 圖是社群推廣最重要的 conversion 元素。

  
### 3. Twitter Card（X / Twitter / 部分 Discord）

```text
<meta name="twitter:card" content="summary_large_image">
<meta name="twitter:title" content="AI 島 — 從 0 到 1 學寫程式">
<meta name="twitter:description" content="75 章 1158+ 課、從 HTML / CSS / JS 到 AI 全包。">
<meta name="twitter:image" content="https://ai-island-web.snowrealm.pet/og-cover.png">
<meta name="twitter:site" content="@aiisland_tw">  <!-- 你的 Twitter handle -->
```

**`twitter:card` 4 種值：**

```text
summary               小圖 + 文字（預設）
summary_large_image   大圖 + 文字（推薦、視覺最強）
app                   App 推廣卡
player                嵌入影片 / audio 播放器
```

> 💡 **OG + Twitter Card 寫一份就好**：實務上 `og:image` 跟 `twitter:image` 用同一張、`twitter:card` 設 `summary_large_image` 就可以。

### 6. 怎麼驗證 meta 寫對沒

**3 個工具（免費）：**

```text
🔍 OpenGraph.xyz
   https://www.opengraph.xyz/
   → 貼 URL、預覽 FB / Twitter / LinkedIn 卡片

🔍 Metatags.io
   https://metatags.io/
   → 同上 + 可在線編輯預覽

🔍 Lighthouse（Chrome DevTools）
   F12 → Lighthouse → SEO 跑分
   → 自動檢查 title / description / canonical / og 都全不全
```

**LINE 預覽除錯：**

```text
LINE 有「分享圖快取」、改了 og:image 不會立刻更新
→ 解：在 og:image URL 後加 ?v=2、強制 LINE 重抓
```

  
### 7. 5 大常見坑

```text
❌ 1. 全站同一個 title
   → Google 認為頁面沒區別、扣分
   → 修：每頁不同 title、放最重要關鍵字在前 30 字

❌ 2. 沒寫 description
   → Google 自己亂抓內文當描述、可能很醜
   → 修：每頁手寫 120-160 字 description

❌ 3. og:image 沒寫 / 連 404 / 尺寸不對
   → 分享預覽沒圖 / 醜 / 被切
   → 修：1200x630 PNG/JPG、URL 寫絕對路徑

❌ 4. og:url 寫相對路徑
   → 必須是 https:// 完整 URL
   → 修：永遠寫絕對路徑

❌ 5. 改完沒清 LINE / FB 快取
   → 自己看新預覽、別人還是看舊的
   → 修：FB Sharing Debugger / LINE 加 ?v=2 force refresh
```

  
### 🎯 重點回顧

- **twitter:card 設 `summary_large_image` 視覺最強**
- **每頁不同 meta 才有 SEO 價值**
- **改完用 OpenGraph.xyz / Metatags.io 驗證**
