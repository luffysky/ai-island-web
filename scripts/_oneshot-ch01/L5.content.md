**列表三兄弟：ul / ol / dl**——列出多個項目用、**90% 場景用 `<ul>`**、有順序用 `<ol>`、定義對照用 `<dl>`。

> 💡 **業界慣例**：導覽列 / 選單 / Tag list / Card grid 全都用 `<ul>`——Bootstrap / Material UI / Tailwind 全這樣做。

  
### 3 種列表

  
### 1. `<ul>`（無序列表、Unordered List）

```text
<ul>
  <li>蘋果</li>
  <li>香蕉</li>
  <li>橘子</li>
</ul>
```

**用途**：項目沒順序、可換位置。

  
### 2. `<ol>`（有序列表、Ordered List）

```text
<ol>
  <li>起床</li>
  <li>刷牙</li>
  <li>吃早餐</li>
</ol>
```

**用途**：步驟、排名、章節順序——**換位置會影響意義**。

  
**`<ol>` 進階屬性：**

```text
<!-- 從 5 開始 -->
<ol start="5">
  <li>項目 5</li>
  <li>項目 6</li>
</ol>

<!-- 倒序 -->
<ol reversed>
  <li>最後一名</li>
  <li>倒數第二</li>
</ol>

<!-- 改編號樣式 -->
<ol type="A">         <!-- A B C -->
<ol type="a">         <!-- a b c -->
<ol type="I">         <!-- I II III -->
<ol type="i">         <!-- i ii iii -->
<ol type="1">         <!-- 1 2 3（預設）-->
```

  
### 3. `<dl>`（定義列表、Description List）

**Description List**（之前叫 Definition List）—**對照型清單**、如「術語 + 解釋」。

```text
<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language、寫網頁結構</dd>
  
  <dt>CSS</dt>
  <dd>Cascading Style Sheets、寫網頁樣式</dd>
  
  <dt>JavaScript</dt>
  <dd>網頁動態互動的程式語言</dd>
</dl>
```

**結構：**

```text
<dl>     列表本身
  <dt>   詞（term）
  <dd>   定義（description）
</dl>
```

  
**用途：**

```text
✅ 術語表 / 字典
✅ FAQ（問題 + 答案）
✅ 規格表（屬性 + 值）
✅ Metadata（key + value）
```

  
### 巢狀列表（list 內含 list）

```text
<ul>
  <li>前端
    <ul>
      <li>HTML</li>
      <li>CSS</li>
      <li>JavaScript</li>
    </ul>
  </li>
  <li>後端
    <ul>
      <li>Node.js</li>
      <li>Python</li>
      <li>Go</li>
    </ul>
  </li>
</ul>
```

> 💡 **結構必須對**：`<li>` 內可放新 `<ul>`、但 `<ul>` 不能直接包另一個 `<ul>`、要透過 `<li>` 包。

  
### 業界主流：用 `<ul>` 寫導覽列

```text
<!-- 標準導覽列 ===
<nav>
  <ul>
    <li><a href="/">首頁</a></li>
    <li><a href="/about">關於</a></li>
    <li><a href="/products">產品</a>
      <ul>
        <li><a href="/products/a">產品 A</a></li>
        <li><a href="/products/b">產品 B</a></li>
      </ul>
    </li>
    <li><a href="/contact">聯絡</a></li>
  </ul>
</nav>
```

**為什麼用 `<ul>`：**

```text
✅ 螢幕閱讀器知道「這是一組導覽」
✅ Google 認得結構
✅ CSS 可改成水平 / 垂直 / 漢堡選單
✅ 多層 nested menu 自然支援
✅ Bootstrap / Material UI / Tailwind 都這樣
```

  
### CSS 控制列表樣式

  
**1. 移除預設 bullet：**

```text
<style>
  ul {
    list-style: none;       /* 不顯示 ●/○/■ */
    padding: 0;             /* 移除左邊縮排 */
  }
</style>

<ul>
  <li>項目 1</li>
  <li>項目 2</li>
</ul>
```

  
**2. 換 bullet 樣式：**

```text
ul { list-style-type: square; }      /* ■ */
ul { list-style-type: circle; }      /* ○ */
ul { list-style-type: disc; }        /* ● 預設 */
ul { list-style-type: "→ "; }        /* 自訂字元 */
ul { list-style-image: url(★.png); } /* 圖片 */
```

  
**3. 水平排列（導覽列常用）：**

```text
<style>
  nav ul {
    display: flex;
    gap: 1rem;
    list-style: none;
    padding: 0;
  }
</style>
```

  
### Flexbox / Grid 內用列表

**現代 CSS（flexbox / grid）配 `<ul>` 完美：**

```text
<style>
  .card-grid {
    display: grid;
    grid-template-columns: repeat(3, 1fr);
    gap: 1rem;
    list-style: none;
    padding: 0;
  }
  
  .card-grid li {
    padding: 1rem;
    border: 1px solid #ccc;
  }
</style>

<ul class="card-grid">
  <li>卡片 1</li>
  <li>卡片 2</li>
  <li>卡片 3</li>
  <li>卡片 4</li>
</ul>
```

> 💡 **業界慣例**：卡片 grid 用 `<ul>` 不用 `<div>`、有語意 + 螢幕閱讀器友善。

### 5 大常見錯誤

```text
1. ❌ 用 <div> 寫列表
   <div>項目 1</div>
   <div>項目 2</div>
   ✅ 用 <ul><li>

2. ❌ 用 <br> 分隔項目
   項目 1<br>項目 2<br>項目 3
   ✅ 用 <ul><li>

3. ❌ <li> 沒包在 <ul>/<ol> 內
   ❌ <li>項目</li>（直接放 body）
   ✅ 必須包在 <ul> 或 <ol> 內

4. ❌ 用 <ol> 表達「重要程度」
   → reordered = 意義改變
   ✅ 重要性用 CSS 或順序排版

5. ❌ <dl> 拿來排版
   ✅ 只用在「術語 + 解釋」「key + value」
```

  
### 列表 + Markdown 對應

```text
=== Markdown ===
- 無序項目 1       → <ul><li>
- 無序項目 2

1. 有序項目 1      → <ol><li>
2. 有序項目 2

Term            → <dl><dt>
: Description   → <dd>

→ Markdown 轉 HTML 自動產生
→ 教學部落格主流
```

列表 3 兄弟：(1) **`<ul>`** 無序、90% 場景（導覽 / Tag / Card grid 全用） (2) **`<ol>`** 有序、步驟 / 排名 / 章節（reordered = 意義改變） (3) **`<dl>`** 定義對照、術語 + 解釋 / FAQ / 規格。**業界慣例**：導覽列必用 `<ul>`（Bootstrap / Material UI 全這樣）。**心法**：別用 `<div>` 寫列表、用 `<ul>` 語意 + SEO + a11y 全加分。
