**圖片優化全面進階：picture / srcset / sizes / WebP / AVIF / lazy / CDN**——L6 教過基本 lazy + srcset、L15 教過效能、這課把圖片相關全部打通。**圖片是網頁最大、優化 ROI 最高的資源**。

> 💡 **數字震撼**：典型網頁圖片占總 size 60-80%。優化好圖片、頁面從 5MB 變 800KB、LCP 從 4 秒變 1.2 秒、Lighthouse 從 40 變 95。

  
### 1. 圖片格式：2026 怎麼選

```text
🥇 AVIF（最新、最小）
   - 比 JPG 小 50%、比 WebP 小 20%
   - Chrome / Firefox / Safari 16+ 支援
   - 編碼慢、壓縮慢
   - 適合：靜態 build 時轉好（Next.js 自動）

🥈 WebP（主力）
   - 比 JPG 小 30%
   - 全現代瀏覽器支援
   - 編碼快
   - 適合：90% 場景

🥉 JPG（fallback）
   - 通用、相容性最廣
   - 適合：fallback / 老瀏覽器

🎨 PNG（透明背景）
   - 比 JPG 大、但支援透明
   - 適合：logo / icon / 截圖

🎨 SVG（向量、L14 教過）
   - 適合：logo / icon / 簡單插圖
```

**做法**：用 `<picture>` 提供 AVIF + WebP + JPG 三層 fallback。

  
### 2. `<picture>` + `<source>` 完整寫法

```text
<picture>
  <!-- 1. AVIF 優先（瀏覽器支援就用）-->
  <source type="image/avif" srcset="
    hero-400.avif 400w,
    hero-800.avif 800w,
    hero-1600.avif 1600w
  ">

  <!-- 2. WebP 次選 -->
  <source type="image/webp" srcset="
    hero-400.webp 400w,
    hero-800.webp 800w,
    hero-1600.webp 1600w
  ">

  <!-- 3. JPG 最終 fallback -->
  <img
    src="hero-800.jpg"
    srcset="
      hero-400.jpg 400w,
      hero-800.jpg 800w,
      hero-1600.jpg 1600w
    "
    sizes="(max-width: 600px) 100vw, 800px"
    alt="主視覺圖、AI 島課程地圖"
    width="800"
    height="450"
    loading="lazy"
    decoding="async"
  >
</picture>
```

**`sizes` 怎麼讀：**

```text
(max-width: 600px) 100vw, 800px
↓ 拆解
螢幕寬 ≤ 600px：圖佔滿視窗（100vw）
否則：圖固定 800px 寬

→ 瀏覽器讀 sizes、配 srcset 算出該載哪個解析度
```

  
### 3. art direction：不同螢幕載「不同剪裁」的圖

```text
<picture>
  <!-- 手機：直式裁切 -->
  <source media="(max-width: 600px)" srcset="hero-portrait.webp">

  <!-- 平板：方形 -->
  <source media="(max-width: 1024px)" srcset="hero-square.webp">

  <!-- 桌面：橫式 -->
  <img src="hero-landscape.jpg" alt="...">
</picture>
```

**用途**：Hero 圖、手機顯示主體放大版、桌面顯示完整景。

  
### 4. lazy loading 細節

```text
<!-- 預設 -->
<img src="..." loading="lazy">    ← 滾到附近才載
<img src="..." loading="eager">   ← 立刻載（首屏圖用）

<!-- 進階組合 -->
<img
  src="..."
  loading="lazy"
  fetchpriority="low"    ← 給瀏覽器「這不急」hint
  decoding="async"       ← 解碼不擋 main thread
>

<!-- 首屏 Hero -->
<img
  src="..."
  loading="eager"
  fetchpriority="high"   ← 「這超重要、優先載」
  decoding="async"
>
```

> 💡 **`fetchpriority="high"` 是 2024 後的新屬性**——告訴瀏覽器 LCP 圖片優先載、實測能讓 LCP 改善 200-500ms。

  
### 5. CDN 動態圖片服務（業界主流）

**問題**：自己準備 3 種格式 × 3 種解析度 = 9 個檔、很麻煩。

**解法**：用 CDN 動態轉檔。

```text
🌐 Cloudflare Images
   https://example.com/cdn-cgi/image/width=800,format=auto/photo.jpg
   → URL 帶參數、自動轉

🌐 Vercel Image
   Next.js <Image> 自動用、Vercel host 免費額度

🌐 ImageKit / imgix / Cloudinary
   第三方付費服務、功能最強

🌐 Supabase Storage transformations
   <img src="https://xxx.supabase.co/storage/v1/render/image/public/photos/x.jpg?width=800&format=webp">
```

**Cloudflare Images 範例：**

```text
<picture>
  <source type="image/avif"
    srcset="
      /cdn-cgi/image/width=400,format=avif/photo.jpg 400w,
      /cdn-cgi/image/width=800,format=avif/photo.jpg 800w,
      /cdn-cgi/image/width=1600,format=avif/photo.jpg 1600w
    ">
  <source type="image/webp"
    srcset="
      /cdn-cgi/image/width=400,format=webp/photo.jpg 400w,
      /cdn-cgi/image/width=800,format=webp/photo.jpg 800w,
      /cdn-cgi/image/width=1600,format=webp/photo.jpg 1600w
    ">
  <img
    src="/cdn-cgi/image/width=800/photo.jpg"
    srcset="
      /cdn-cgi/image/width=400/photo.jpg 400w,
      /cdn-cgi/image/width=800/photo.jpg 800w,
      /cdn-cgi/image/width=1600/photo.jpg 1600w
    "
    sizes="(max-width: 600px) 100vw, 800px"
    alt="..."
  >
</picture>
```

### 9. 5 大常見坑

```text
❌ 1. 首屏圖也 loading="lazy"
   → LCP 變差（首屏圖延後 1-2 秒才開始載）
   → 修：首屏圖 loading="eager" + fetchpriority="high"

❌ 2. width / height 沒寫
   → CLS 爆炸
   → 修：永遠寫實際像素

❌ 3. 只給 1 種 format（只 JPG）
   → 沒利用現代瀏覽器、流量浪費
   → 修：picture + WebP + AVIF

❌ 4. srcset 都同尺寸
   → 例 400/800/1600 都載 800px 寬的圖、瀏覽器選不到合適
   → 修：真的提供不同 resolution

❌ 5. sizes 沒寫或寫錯
   → 瀏覽器不知道實際顯示寬、可能挑錯解析度
   → 修：sizes 反映實際 CSS layout
```
