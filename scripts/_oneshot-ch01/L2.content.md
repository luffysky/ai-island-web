**head 區塊：給機器看的全部秘密**——`<head>` 是 HTML 內**不會顯示在畫面上**的部分、但**決定 SEO / 分享預覽 / 編碼 / favicon / 載入順序**。

> 💡 **核心心法**：head 寫得好 = 網頁被 Google 找到 + 分享到 LINE 有預覽圖 + 載入快 + AI 認識你。**新手最容易忽略、但商業價值最高**。

  
### 完整 head 範本（生產級）

📄 **寫進程式檔**：`index.html` 的 `<head>` 區塊

```text
<!DOCTYPE html>
<html lang="zh-Hant">
<head>
  <!-- ====== 基本 ====== -->
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  
  <!-- ====== SEO ====== -->
  <title>我的網站 - 副標</title>
  <meta name="description" content="AI 島：全端養成班">
  <meta name="keywords" content="HTML, CSS, JavaScript, 教學">
  <meta name="author" content="Luffysky">
  <link rel="canonical" href="https://example.com/page">
  
  <!-- ====== 社群分享預覽（Open Graph、Facebook / LINE / Discord）====== -->
  <meta property="og:title" content="我的網站">
  <meta property="og:description" content="網站描述">
  <meta property="og:image" content="https://example.com/og.png">
  <meta property="og:type" content="website">
  <meta property="og:url" content="https://example.com">
  
  <!-- ====== Twitter / X 分享預覽 ====== -->
  <meta name="twitter:card" content="summary_large_image">
  <meta name="twitter:title" content="我的網站">
  <meta name="twitter:image" content="https://example.com/og.png">
  
  <!-- ====== Favicon（分頁小 icon）====== -->
  <link rel="icon" href="/favicon.svg" type="image/svg+xml">
  <link rel="apple-touch-icon" href="/apple-touch-icon.png">
  
  <!-- ====== 樣式 + 腳本 ====== -->
  <link rel="stylesheet" href="/style.css">
  <script src="/app.js" defer></script>
</head>
<body>
  <!-- content -->
</body>
</html>
```

  
### 4 大區塊解析

  
### 1. 基本（必有、不能少）

```text
<meta charset="UTF-8">
   → 編碼、中文 / emoji 不亂碼
   → 永遠 UTF-8、別用 Big5 / GBK

<meta name="viewport" content="width=device-width, initial-scale=1.0">
   → 手機正確縮放
   → 沒寫 = 手機看到桌面版縮小
```

  
### 2. SEO（Google 必看）

```text
<title>分頁標題</title>
   → 搜尋結果的藍字標題
   → 30-60 字最佳
   → 含關鍵字 + 品牌名

<meta name="description" content="...">
   → 搜尋結果的描述
   → 120-160 字最佳
   → 寫得吸引人 → CTR 高

<link rel="canonical" href="...">
   → 主要 URL
   → 避免「同內容多 URL」扣分
```

> 💡 **2026 SEO 重點**：title + description 是 Google 排名最大的兩個 on-page 因素、寫好 = SEO 一半成功。

  
### 3. Open Graph（社群分享預覽）

```text
=== 為什麼要 ===

❌ 沒寫 og:
   → 貼到 LINE / Facebook / Discord
   → 只顯示 URL、無圖無描述
   → 點擊率超低

✅ 有 og:
   → 顯示漂亮預覽卡
   → 圖 + 標題 + 描述
   → 點擊率 +3-5x
```

  
**5 個必加 og：**

```text
og:title       標題（90% 跟 <title> 一樣）
og:description 描述（90% 跟 meta description 一樣）
og:image       預覽圖（1200x630 最佳）
og:type        website / article / video
og:url         完整 URL
```

  
### 4. Favicon（分頁小 icon）

```text
<link rel="icon" href="/favicon.svg" type="image/svg+xml">
   → 分頁標題前面的小圖
   → SVG 最佳（不會糊）
   → 沒寫 = 顯示空白 / 預設地球

<link rel="apple-touch-icon" href="/apple-touch-icon.png">
   → iPhone 加到桌面的 icon
   → 180x180 PNG
```

  
### Script 載入：async vs defer vs 一般

**3 種載入方式對效能影響超大：**

```text
=== 一般（不推薦）===
<script src="app.js"></script>

執行流程：
1. HTML 解析到 <script>
2. 停下、下載 app.js
3. 執行 app.js
4. 才繼續解析 HTML

→ 阻擋 HTML 解析、頁面慢

=== async（不保證順序）===
<script src="app.js" async></script>

執行流程：
1. HTML 解析時、async 下載
2. 下載完立刻執行（中斷 HTML 解析）
3. HTML 繼續解析

→ 多個 async 順序不定
→ 適合：獨立 script（如 Google Analytics）

=== defer（最推薦）===
<script src="app.js" defer></script>

執行流程：
1. HTML 解析時、defer 並行下載
2. HTML 全部解析完、才依序執行 defer
3. 多個 defer 保留順序

→ 不阻擋 + 順序保證
→ 90% 場景用 defer
```

  
### CSS 載入順序

```text
<link rel="stylesheet" href="/style.css">

→ CSS 預設「阻擋渲染」
→ HTML 等 CSS 載入才畫畫面
→ 重要：CSS 寫在 <head>、不要寫在 body
```

> 💡 **進階優化**：用 `<link rel="preload">` 或 critical CSS inline 在 head。

  
### Meta 標籤完整 30+ 種

**SEO：**

```text
description      頁面描述
keywords         關鍵字（Google 不重視、但其他引擎看）
author           作者
robots           索引控制（index/noindex/follow/nofollow）
```

  
**Open Graph：**

```text
og:title
og:description
og:image
og:type
og:url
og:site_name

```

```text

twitter:site           @你的帳號
twitter:creator        作者帳號
twitter:title
twitter:description

```

```text

```

### 5 大常見坑

```text
1. ❌ 沒寫 viewport
   → 手機看到桌面版縮小

2. ❌ charset 不是 UTF-8
   → 中文亂碼

3. ❌ og:image 用小圖
   → 預覽卡糊
   → 永遠 1200x630

4. ❌ script 沒加 defer
   → 阻擋 HTML 解析、頁面慢

5. ❌ favicon 沒寫
   → 分頁無 icon、不專業
```

  
    **☕ 用人話講**
    
head 4 大區塊：(1) **基本**：DOCTYPE / charset / viewport（少 = 壞） (2) **SEO**：title / description / canonical（Google 排名核心） (3) **Open Graph**：og:title / og:image（社群分享預覽、點擊率 +3-5x） (4) **Favicon**：分頁 icon（小但專業度）。**Script 載入**：defer > async > 一般、90% 用 defer。**心法**：head 看不到但商業價值最高、SEO + 分享預覽都靠它。
