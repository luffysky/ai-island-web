**JSON-LD 進階 Schema**——L10 教過 4 種高 ROI schema（Article / Product / FAQPage / BreadcrumbList）、這課補完 7 個進階：完整 Article、Course、HowTo、Video、Person、Organization、LocalBusiness。**2026 GEO 必備**。

> 💡 **GEO 趨勢提醒**：傳統 SEO 流量 5 年內可能腰斬、AI 引擎（ChatGPT / Perplexity / Bing Copilot）會接管 30-50%。**現在多寫一份 schema = 未來多一份 AI 引用流量**。

  
### 1. Article 完整版（部落格 / 新聞）

```text
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "BlogPosting",
  "@id": "https://example.com/blog/post-1",
  "headline": "為什麼選 Next.js 而不是 Remix",
  "alternativeHeadline": "從 3 個角度分析 2026 全端框架選擇",
  "image": [
    "https://example.com/cover-16x9.jpg",
    "https://example.com/cover-4x3.jpg",
    "https://example.com/cover-1x1.jpg"
  ],
  "datePublished": "2026-05-28T08:00:00+08:00",
  "dateModified": "2026-05-28T10:00:00+08:00",
  "author": [{
    "@type": "Person",
    "name": "Luffy Lin",
    "url": "https://example.com/u/luffysky",
    "image": "https://example.com/avatars/luffysky.jpg",
    "jobTitle": "全端工程師 / 創辦人",
    "sameAs": [
      "https://twitter.com/luffysky",
      "https://github.com/luffysky"
    ]
  }],
  "publisher": {
    "@type": "Organization",
    "name": "AI 島",
    "logo": {
      "@type": "ImageObject",
      "url": "https://example.com/logo.png",
      "width": 600,
      "height": 60
    }
  },
  "mainEntityOfPage": {
    "@type": "WebPage",
    "@id": "https://example.com/blog/post-1"
  },
  "wordCount": 2500,
  "articleSection": "技術",
  "keywords": ["Next.js", "React", "全端"],
  "inLanguage": "zh-Hant"
}
</script>
```

**新加的欄位作用：**

```text
🆔 @id              → 唯一 URI（多 schema 互相 reference 用）
🖼️ image (3 種比例)  → Google News 跨 device 顯示
📝 alternativeHeadline → 副標題
👤 author.sameAs    → 連到作者社群（Knowledge Graph 抓）
📄 mainEntityOfPage → 這個 schema 主體是哪個網頁
📊 wordCount        → 字數（AI 引擎評估深度）
🏷️ articleSection / keywords → 分類 / 標籤
```

### 7. 多 schema 同一頁（最強組合）

部落格文章頁可以同時放 4 個 schema：

```text
<head>
  <!-- 1. Article 本身 -->
  <script type="application/ld+json">{ "@type": "Article", ... }</script>

  <!-- 2. 麵包屑 -->
  <script type="application/ld+json">{ "@type": "BreadcrumbList", ... }</script>

  <!-- 3. 作者 Person -->
  <script type="application/ld+json">{ "@type": "Person", ... }</script>

  <!-- 4. 網站 Organization -->
  <script type="application/ld+json">{ "@type": "Organization", ... }</script>
</head>
```

Google 全部讀、互相 reference（用 `@id`）。

  
### 8. 5 大常見坑

```text
❌ 1. duration 格式錯
   → 必須 ISO 8601（PT3M30S）、不是 "3:30"
   → 修：用 ISO 8601 duration（PnYnMnDTnHnMnS）

❌ 2. 圖片用相對路徑
   → 必須 https:// 完整 URL
   → 修：永遠絕對路徑

❌ 3. price 寫成 "NT$2990" 字串帶符號
   → 必須純數字字串 "2990" + priceCurrency 分開
   → 修：分兩欄

❌ 4. address 沒 addressCountry
   → 台灣地址沒寫 "TW" → Google 認不出國家
   → 修：永遠加 addressCountry ISO 兩碼

❌ 5. 多 schema 互相沒 @id reference
   → Google 不知道是同一個 entity
   → 修：用 @id 互相連接（"@id": "https://example.com/#org"）
```
