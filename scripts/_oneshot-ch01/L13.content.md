**PWA（Progressive Web App）基礎**——加 3 個檔案（manifest.json / service worker / icon）就讓你的網頁變成可「加到主畫面」的 App、離線也能用、能發推播。

> 💡 **PWA 的價值**：(1) 跳過 App Store 審核（直接上線）(2) 一份 code 同時是網頁 + 手機 App + 桌面 App (3) 安裝率比下載 App 高 5-10 倍。AI 島本站就是 PWA。

  
### 1. PWA 三大支柱

```text
📄 manifest.json    — App 設定檔（名稱 / icon / 主題色）
⚙️  Service Worker   — 背景跑的 JS（快取 / 離線 / 推播）
🔒 HTTPS            — 必須（localhost 例外）
```

只要這 3 個齊備、Chrome / Edge / Safari 自動顯示「安裝」按鈕。

  
### 2. manifest.json — App 設定檔

📄 **寫進程式檔**：`/manifest.json`

```text
{
  "name": "AI 島",
  "short_name": "AI 島",
  "description": "從 0 到 1 學寫程式",
  "start_url": "/",
  "display": "standalone",
  "background_color": "#0a0a0a",
  "theme_color": "#10b981",
  "lang": "zh-Hant",
  "orientation": "portrait",
  "icons": [
    {
      "src": "/icons/icon-192.png",
      "sizes": "192x192",
      "type": "image/png",
      "purpose": "any"
    },
    {
      "src": "/icons/icon-512.png",
      "sizes": "512x512",
      "type": "image/png",
      "purpose": "any"
    },
    {
      "src": "/icons/icon-maskable.png",
      "sizes": "512x512",
      "type": "image/png",
      "purpose": "maskable"
    }
  ]
}
```

**`<head>` 內連結：**

```text
<link rel="manifest" href="/manifest.json">
<meta name="theme-color" content="#10b981">
<link rel="apple-touch-icon" href="/icons/icon-192.png">
```

**5 個必填欄位：**

```text
🏷️ name / short_name
   → name 完整名（Safari Add to Home）
   → short_name ≤ 12 字（手機桌面圖示下方）

🚀 start_url
   → 開 App 進去哪頁、通常 "/"

🖥️ display
   → standalone（推薦、無瀏覽器 UI）
   → fullscreen（全螢幕、遊戲用）
   → minimal-ui（保留小 URL bar）

🎨 background_color / theme_color
   → 啟動畫面背景 / 手機 URL bar 顏色

🖼️ icons
   → 至少 192x192 + 512x512
   → 加 maskable 版本（Android 圓角適應）
```

  
### 3. Service Worker — 離線 / 快取核心

📄 **寫進程式檔**：`/sw.js`

```text
// 1. 安裝時、預載入核心檔案
const CACHE = 'aiisland-v1';
const PRECACHE = ['/', '/style.css', '/app.js', '/manifest.json'];

self.addEventListener('install', (e) => {
  e.waitUntil(caches.open(CACHE).then((c) => c.addAll(PRECACHE)));
  self.skipWaiting();
});

// 2. 啟用時、清舊版本
self.addEventListener('activate', (e) => {
  e.waitUntil(
    caches.keys().then((keys) =>
      Promise.all(keys.filter((k) => k !== CACHE).map((k) => caches.delete(k))),
    ),
  );
  self.clients.claim();
});

// 3. 攔截網路請求、快取優先 (Cache First)
self.addEventListener('fetch', (e) => {
  e.respondWith(
    caches.match(e.request).then((cached) => cached || fetch(e.request)),
  );
});
```

**註冊 SW**（HTML 內加）：

```text
<script>
  if ('serviceWorker' in navigator) {
    navigator.serviceWorker.register('/sw.js');
  }
</script>
```

  
### 4. 4 種快取策略

```text
🥇 Cache First（資源類）
   先快取、沒才打網
   → 圖片 / CSS / JS 用

🥈 Network First（API 類）
   先打網、失敗才用快取
   → 動態資料 / 即時內容

🥉 Stale While Revalidate（最佳體驗）
   立刻回快取、背景更新
   → 文章 / 商品列表

🚫 Network Only（敏感類）
   永遠打網、不快取
   → 登入 / 付款 / 個資
```

**Network First 範例：**

```text
self.addEventListener('fetch', (e) => {
  if (e.request.url.includes('/api/')) {
    e.respondWith(
      fetch(e.request)
        .then((r) => {
          const clone = r.clone();
          caches.open(CACHE).then((c) => c.put(e.request, clone));
          return r;
        })
        .catch(() => caches.match(e.request)),
    );
  }
});
```

  
### 5. 推播通知（Push API）

```text
// 1. 跟使用者要許可
await Notification.requestPermission();
// 'granted' / 'denied' / 'default'

// 2. 訂閱 push（要 VAPID public key）
const reg = await navigator.serviceWorker.ready;
const sub = await reg.pushManager.subscribe({
  userVisibleOnly: true,
  applicationServerKey: 'YOUR_VAPID_PUBLIC_KEY',
});

// 3. 送 sub 給 server 存
await fetch('/api/push-subscribe', {
  method: 'POST',
  body: JSON.stringify(sub),
});
```

**SW 內接 push：**

```text
self.addEventListener('push', (e) => {
  const data = e.data?.json() ?? {};
  e.waitUntil(
    self.registration.showNotification(data.title ?? '通知', {
      body: data.body,
      icon: '/icons/icon-192.png',
      badge: '/icons/badge.png',
    }),
  );
});
```

> ⚠️ **iOS 限制**：iOS 16.4+ 才支援 Web Push（必須加到主畫面才能用、Safari 內不行）。Android Chrome 老早支援。

  
### 6. 驗證 PWA 是否合格

**Chrome DevTools：**

```text
F12 → Application → Manifest
  → 看 manifest.json 有沒有錯
  → "Add to homescreen" 按鈕能不能按

F12 → Application → Service Workers
  → 看 SW 註冊狀態
  → Update / Unregister 按鈕

F12 → Lighthouse → PWA 跑分
  → 自動檢查所有 PWA 要求
```

**Chrome 安裝按鈕出現的條件：**

```text
✅ HTTPS（localhost 例外）
✅ manifest.json 連得到 + name + icons 含 192/512
✅ Service Worker 註冊成功
✅ display 是 standalone / fullscreen / minimal-ui
✅ 使用者跟網站有過互動（防垃圾彈窗）
```

  
### 7. Next.js / Nuxt 怎麼做 PWA

```text
Next.js：
  npm i next-pwa
  next.config.js 包 withPWA
  manifest.json 放 public/
  → 自動生成 sw.js

Nuxt:
  @vite-pwa/nuxt module
  nuxt.config.ts 加 vite-pwa 設定

不用 framework：
  手寫 manifest.json + sw.js（這課教的）
```

  
### 8. 5 大常見坑

```text
❌ 1. 沒 HTTPS
   → SW 完全不會註冊
   → 修：上 Vercel / Zeabur 自動 HTTPS

❌ 2. manifest.json 路徑錯
   → DevTools Application 顯示「No manifest」
   → 修：<link rel="manifest" href="/manifest.json">

❌ 3. icon 尺寸不齊
   → 必須有 192x192 + 512x512
   → 修：用 https://realfavicongenerator.net/ 一次生

❌ 4. SW 卡舊版
   → 改完 sw.js 使用者還用舊的
   → 修：CACHE 版本號改、activate 內 caches.delete 清舊的

❌ 5. iOS 沒 Add to Home Screen 按鈕
   → Safari 永遠沒這按鈕、要 user 從分享 menu 加
   → 修：寫 onboarding 教學說明
```

  
### 🎯 重點回顧

- **PWA = manifest.json + sw.js + HTTPS 三件套**
- **Service Worker 攔截網路請求 = 快取 / 離線核心**
- **4 種快取策略各得其所**（Cache First 資源 / Network First API）
- **推播在 iOS 16.4+ 才支援、要加到主畫面**
- **Lighthouse PWA 跑分驗證**
