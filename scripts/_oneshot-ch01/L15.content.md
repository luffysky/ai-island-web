**HTML 效能：寫法決定速度**——同一份內容、寫得對 0.8 秒載完、寫得不對 5 秒還沒完。**Google 統計**：載入超過 3 秒、53% 行動裝置使用者直接離開。電商每多 1 秒、轉換率掉 7%。

> 💡 **這課的重點**：不是教你寫 JS 優化（那是 ch04）、是教你**HTML 怎麼寫就能影響速度**——資源載入順序、預載、lazy、size hint。

  
### 1. 4 大效能指標（Web Vitals）

```text
🚀 LCP（Largest Contentful Paint）
   最大內容繪製時間
   ✅ < 2.5 秒：好
   ⚠️ 2.5-4 秒：要改
   ❌ > 4 秒：差

📏 CLS（Cumulative Layout Shift）
   累積版面晃動
   ✅ < 0.1：好
   ⚠️ 0.1-0.25：要改
   ❌ > 0.25：差

⚡ INP（Interaction to Next Paint）
   點擊到下一個畫面更新
   ✅ < 200ms：好
   ⚠️ 200-500ms：要改
   ❌ > 500ms：差

⏱️ TTFB（Time to First Byte）
   server 多快開始回 byte
   ✅ < 800ms：好
```

**驗工具**：Chrome DevTools Lighthouse / PageSpeed Insights / WebPageTest。

  
### 2. JS 載入位置：head / body / async / defer

```text
❌ <script src="app.js"></script> 放 <head> 內最上面
   → 「render-blocking」、瀏覽器停下來載 JS、頁面白屏

⚡ <script src="app.js"></script> 放 </body> 前
   → DOM 都 parse 完再載 JS、不擋 render

✅ <script src="app.js" defer></script> 放 <head>
   → 平行下載、DOM 完才執行
   → 多 script 保證執行順序
   → 2026 推薦預設

⚠️ <script src="app.js" async></script>
   → 平行下載、下載完立刻執行
   → 不保證順序
   → 適合獨立 script（analytics）

✅ <script type="module" src="app.js"></script>
   → 預設 defer 行為
   → 現代 module 用這個
```

**對比表：**

```text
              下載      執行時機         保證順序
────────────────────────────────────────────────
<script>      blocking  parse 到才下載   ✓
defer         並行      DOM 完才執行     ✓
async         並行      下載完就執行     ✗
module        並行      DOM 完才執行     ✓
```

  
### 3. CSS 載入：critical CSS + preload

```text
❌ 所有 CSS 一個檔
   → render-blocking、CSS 沒載完不畫頁

✅ 分兩塊：
   首屏（above-the-fold）→ inline 進 <head>
   其餘 → 非同步載

<head>
  <!-- Critical CSS inline -->
  <style>
    body { margin: 0; font-family: sans-serif; }
    .hero { padding: 60px 20px; background: #10b981; }
  </style>

  <!-- 其他 CSS preload + 非阻塞 -->
  <link rel="preload" href="full.css" as="style" onload="this.rel='stylesheet'">
  <noscript><link rel="stylesheet" href="full.css"></noscript>
</head>
```

> 💡 **Critical CSS 抓首屏需要的最少 CSS**（通常 5-15KB）、其他 async 載——LCP 能改善 1-2 秒。

  
### 4. 圖片優化（影響最大）

```text
✅ 寫 width / height（防 CLS）
   <img src="hero.jpg" alt="" width="800" height="400">

✅ Lazy load 首屏外
   <img src="..." loading="lazy">

✅ 響應式（srcset）
   <img
     srcset="img-400.jpg 400w, img-800.jpg 800w, img-1600.jpg 1600w"
     sizes="(max-width: 600px) 100vw, 800px"
     src="img-800.jpg"
     alt=""
   >

✅ 用 WebP / AVIF（比 JPG 小 30-50%）
   <picture>
     <source srcset="hero.avif" type="image/avif">
     <source srcset="hero.webp" type="image/webp">
     <img src="hero.jpg" alt="" loading="lazy">
   </picture>

✅ 首屏關鍵圖 fetchpriority="high"
   <img src="hero.jpg" fetchpriority="high" alt="">

✅ decode="async"（不擋 main thread）
   <img src="..." decoding="async">
```

  
### 5. Resource hints（4 種）

```text
🔗 dns-prefetch — 提早解析 DNS
   <link rel="dns-prefetch" href="https://api.example.com">

🔗 preconnect — 提早建 TCP + TLS（更積極）
   <link rel="preconnect" href="https://api.example.com" crossorigin>
   → 比 dns-prefetch 強、但只建議用 2-3 個（每個 connection 都耗資源）

🔗 prefetch — 預載低優先資源
   <link rel="prefetch" href="next-page.html">
   → 給「使用者可能會去的下一頁」用

🔗 preload — 預載高優先當前頁資源
   <link rel="preload" href="hero-font.woff2" as="font" type="font/woff2" crossorigin>
   → 字型 / Hero 圖 / Critical script
```

**完整 head 範本：**

```text
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">

  <!-- DNS / Connection -->
  <link rel="preconnect" href="https://fonts.googleapis.com" crossorigin>
  <link rel="dns-prefetch" href="https://www.google-analytics.com">

  <!-- Critical preload -->
  <link rel="preload" href="/hero.webp" as="image" fetchpriority="high">
  <link rel="preload" href="/fonts/main.woff2" as="font" type="font/woff2" crossorigin>

  <!-- Critical CSS inline -->
  <style>/* ... */</style>

  <!-- Full CSS async -->
  <link rel="preload" href="/style.css" as="style" onload="this.rel='stylesheet'">

  <!-- JS defer -->
  <script src="/app.js" defer></script>

  <!-- Prefetch next page -->
  <link rel="prefetch" href="/about">
</head>
```

  
### 6. 字型優化（最容易忽略）

```text
❌ <link href="https://fonts.googleapis.com/css?family=Noto+Sans+TC">
   → 從 Google Fonts 載 4 個 weight × 8000 字 = 1-2MB
   → FOIT（Flash of Invisible Text）卡很久

✅ 自託管 + woff2 + subset
   只載要用的字（中文 only / Latin only）
   <link rel="preload" href="/fonts/main.woff2" as="font" type="font/woff2" crossorigin>
   @font-face {
     font-family: 'Main';
     src: url('/fonts/main.woff2') format('woff2');
     font-display: swap;  /* 重要！先顯示系統字、字型載完再換 */
   }

✅ Variable Font（一檔多 weight）
   100-900 weight 一個檔、省 80% 大小
```

  
### 7. 完整 fast HTML 範本

📄 **寫進程式檔**：`fast-page.html`

```text
<!DOCTYPE html>
<html lang="zh-Hant">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>Fast Page</title>

  <link rel="preconnect" href="https://fonts.googleapis.com" crossorigin>
  <link rel="preload" href="/hero.webp" as="image" fetchpriority="high">

  <style>
    *{margin:0;padding:0;box-sizing:border-box}
    body{font-family:system-ui,sans-serif;color:#1f2937}
    .hero{min-height:60vh;background:#10b981;display:flex;align-items:center;justify-content:center;color:white}
    .hero h1{font-size:clamp(2rem,5vw,4rem)}
  </style>

  <link rel="stylesheet" href="/full.css" media="print" onload="this.media='all'">
</head>
<body>
  <header class="hero">
    <h1>從 0 到 1 學寫程式</h1>
  </header>

  <picture>
    <source srcset="/feature.avif" type="image/avif">
    <source srcset="/feature.webp" type="image/webp">
    <img src="/feature.jpg" width="1200" height="600" loading="lazy" decoding="async" alt="...">
  </picture>

  <script src="/app.js" defer></script>
</body>
</html>
```

**效能特點：**
- Critical CSS inline、其他 print → all 技巧 async 載
- Hero 圖 preload + fetchpriority="high"
- AVIF / WebP / JPG 三層 fallback
- 後續圖 lazy load
- JS defer 不擋 render

  
### 8. 5 大常見坑

```text
❌ 1. <script> 放 head 沒 defer
   → Render-blocking、白屏久
   → 修：加 defer 或移到 body 底

❌ 2. 圖片沒寫 width / height
   → CLS 爆炸、Lighthouse 直接扣
   → 修：永遠寫實際像素數字

❌ 3. 全部圖 lazy（連 hero 也 lazy）
   → 首屏圖晚出現、LCP 變差
   → 修：首屏圖不 lazy + fetchpriority="high"

❌ 4. 字型沒 font-display: swap
   → FOIT 1-3 秒看不到任何字
   → 修：永遠寫 font-display: swap

❌ 5. 太多 preload
   → preload 是「告訴瀏覽器這超重要」、寫 10 個就沒有重要的了
   → 修：最多 3-5 個 critical 資源
```

  
### 🎯 重點回顧

- **JS 用 defer / module、別 render-blocking**
- **Critical CSS inline、其他 async 載**
- **圖片：寬高 + lazy + WebP/AVIF + srcset + fetchpriority**
- **Resource hints**：preconnect / preload 是主力
- **字型：自託管 + woff2 + font-display: swap**
