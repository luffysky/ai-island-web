**無障礙（Accessibility、a11y）+ ARIA**——讓盲人 / 行動不便 / 色盲使用者也能用你的網站。在台灣是**法定義務**（政府網站必須）、在美國 ADA 法案下違反會被告。**也是基本良善**。

> 💡 **a11y 紅利**：寫對 a11y 同時順便：(1) SEO 加分（screen reader 跟 Google crawler 看同一份東西） (2) 鍵盤友善（科技狂愛） (3) 行動裝置點擊範圍大（手指族愛） (4) 法律保護。**一份工作四份收益**。

  
### 1. 無障礙的 4 大原則（WCAG）

```text
🔍 P — Perceivable（可感知）
   → 圖片有 alt、影片有字幕、顏色對比夠

⌨️ O — Operable（可操作）
   → 鍵盤能完整操作、不只能用滑鼠
   → 動畫可暫停

🧠 U — Understandable（可理解）
   → 文字清楚、表單錯誤訊息明確

🛡️ R — Robust（穩固）
   → 螢幕閱讀器 / 各種輔助科技都讀得懂
   → 用語意化 HTML（L3 教過）
```

> 💡 **WCAG**（Web Content Accessibility Guidelines）是國際標準、分 A / AA / AAA 三級。**台灣政府網站要求 AA 級**。

  
### 2. 第一招：用語意化 HTML（最重要）

> ⚠️ **80% 的 a11y 問題、靠語意化 HTML 就能解決**——不用學什麼複雜 ARIA。

```text
✅ 用對標籤：
<button> 是按鈕（不要用 <div onclick>）
<a href> 是連結（不要用 <span onclick>）
<nav> 是導覽（不要用 <div class="nav">）
<main> 是主內容
<h1> ~ <h6> 標題層次正確
<form> + <label> + <input>
```

**為什麼**：螢幕閱讀器靠這些標籤導航、給盲人「快速跳到主內容」「列出所有按鈕」「列出所有連結」等功能。`<div>` 一片無從導航。

  
### 3. 第二招：圖片 / 媒體要替代

```text
✅ <img alt="一隻睡覺的橘貓">
   → 螢幕閱讀器念出來

✅ <img alt="" role="presentation">
   → 純裝飾、不念

✅ <video> 配 <track> 字幕
   <video controls>
     <source src="video.mp4">
     <track src="captions.vtt" kind="captions" srclang="zh" label="繁中">
   </video>

✅ <audio> 配文字逐字稿（podcast 必加）

✅ 圖表 / 信息圖 要 long description
   <img src="chart.png" alt="2026 銷售圖、詳見下方文字說明">
   <p>圖表詳細：1 月 100 件、2 月 150 件…</p>
```

  
### 4. 第三招：表單一定要 label

（L8 已教、複習）

```text
✅ <label for="email">Email</label>
   <input id="email" type="email" required>

✅ 錯誤訊息要關聯到欄位
   <input id="email" aria-describedby="email-error">
   <span id="email-error" role="alert">Email 格式不對</span>

✅ checkbox / radio 包 label
   <label>
     <input type="checkbox">
     同意條款
   </label>
```

  
### 5. ARIA 屬性（語意 HTML 不夠用時補）

**ARIA = Accessible Rich Internet Applications**——當你不得不用 `<div>` 蓋按鈕、用 ARIA 補語意。

> ⚠️ **第一規則：能用 `<button>` 就不要用 `<div role="button">`**——ARIA 是補救、不是替代。

**最常用 5 個 ARIA：**

```text
🏷️ aria-label — 給沒有可見文字的元素一個名字
   <button aria-label="關閉視窗">✕</button>
   <a href="..." aria-label="前往 Twitter"><img src="twitter.svg"></a>

📝 aria-describedby — 連到「補充說明」元素
   <input aria-describedby="hint">
   <span id="hint">至少 8 個字、含大小寫</span>

🔇 aria-hidden — 跟螢幕閱讀器說「跳過這個」
   <span aria-hidden="true">👋</span>  <!-- emoji 裝飾用、不念 -->

📊 aria-live — 動態內容公告
   <div aria-live="polite">儲存成功！</div>
   <!-- polite = 等 user 念完現在的再念 -->
   <!-- assertive = 立刻打斷 user 念這個（用於緊急錯誤）-->

🎭 role — 蓋語意（最後手段）
   <div role="button" tabindex="0">假按鈕</div>
   <ul role="tablist">...</ul>
   <div role="alert">錯誤訊息</div>
```

  
### 6. 鍵盤可操作（科技狂與盲人都靠這個）

**測試法**：拔滑鼠、只用 Tab / Shift+Tab / Enter / Space / 方向鍵能完整操作網站嗎？

```text
✅ 所有互動元素都能 Tab focus
   → <button> <a href> <input> <select> 預設可以
   → <div onclick> 不行 → 加 tabindex="0"

✅ Focus 看得到（focus ring）
   → 千萬不要 CSS 寫 outline: none
   → 改：button:focus-visible { outline: 2px solid blue; }

✅ Tab 順序合理
   → 跟視覺順序一致
   → 永遠不要用 tabindex="2" 強改順序（亂)

✅ Esc 關 modal
   → 開 modal 一定要能用 Esc 關

✅ Enter / Space 都能觸發 button
   → 用 <button> 自動有
```

> 💡 **`:focus` vs `:focus-visible`**：`:focus` 滑鼠點也亮、`:focus-visible` 只在「鍵盤 focus」時亮（更好的 UX）。**2026 全部用 `:focus-visible`**。

  
### 7. 顏色對比 + 文字大小

```text
✅ 文字 vs 背景對比比例：
   一般文字：≥ 4.5:1（WCAG AA）
   大文字（18pt+）：≥ 3:1
   icon / UI 元素：≥ 3:1

✅ 不要只靠顏色傳達資訊
   ❌ 「紅色代表錯誤」← 色盲看不出來
   ✅ 紅色 + ❌ icon + "錯誤" 文字

✅ 文字可放大到 200% 不爆版
   → 用 rem / em 不要用 px 寫尺寸

✅ 不寫死字體顏色 + 背景色
   → 配合 dark mode（CSS prefers-color-scheme）
```

**驗對比工具：**

```text
🔍 Chrome DevTools
   F12 → 點任何文字 → 看 contrast ratio

🔍 WebAIM Contrast Checker
   https://webaim.org/resources/contrastchecker/
```

  
### 8. 完整實戰：a11y 友善的 modal

```text
<button type="button" aria-haspopup="dialog" aria-controls="signup-modal" id="open-btn">
  註冊
</button>

<div
  id="signup-modal"
  role="dialog"
  aria-modal="true"
  aria-labelledby="modal-title"
  hidden
>
  <h2 id="modal-title">建立帳號</h2>
  <button type="button" aria-label="關閉" id="close-btn">✕</button>

  <form>
    <label for="email">Email</label>
    <input id="email" type="email" required>

    <button type="submit">註冊</button>
  </form>
</div>

<script>
  const openBtn = document.getElementById('open-btn');
  const closeBtn = document.getElementById('close-btn');
  const modal = document.getElementById('signup-modal');

  openBtn.addEventListener('click', () => {
    modal.hidden = false;
    document.getElementById('email').focus();  // focus 進去
  });

  closeBtn.addEventListener('click', () => {
    modal.hidden = true;
    openBtn.focus();  // 還焦點回原按鈕
  });

  // Esc 關
  document.addEventListener('keydown', (e) => {
    if (e.key === 'Escape' && !modal.hidden) {
      modal.hidden = true;
      openBtn.focus();
    }
  });
</script>
```

**做對的事：**
- `role="dialog"` + `aria-modal="true"` 標明是 modal
- `aria-labelledby` 連到標題、螢幕閱讀器念「建立帳號 dialog」
- 開啟時 focus 移進 modal、關閉還回原按鈕
- Esc 可關

  
### 9. 5 大常見坑

```text
❌ 1. CSS 寫 outline: none 移掉 focus ring
   → 鍵盤使用者完全不知道焦點在哪
   → 修：保留或改 outline: 2px solid blue 醒目

❌ 2. 用 <div onclick> 當按鈕
   → 鍵盤 focus 不到、不會被列為 button
   → 修：永遠用 <button>

❌ 3. <img> 沒寫 alt
   → 盲人完全看不到圖
   → 修：永遠寫（裝飾圖 alt=""）

❌ 4. 顏色對比不夠
   → 色盲 / 弱視看不清楚
   → 修：用對比工具驗 ≥ 4.5:1

❌ 5. 只用顏色傳達狀態
   → 「紅色 = 錯誤」色盲 GG
   → 修：顏色 + icon + 文字三管齊下
```
