**HTML 全域屬性**——`data-*` / `contenteditable` / `draggable` / `hidden` / `inert` / `spellcheck` / `translate`。**全域屬性 = 任何 HTML 元素都能用的屬性**、是寫互動 UI 的重要工具。

> 💡 **全域屬性的價值**：學會這些、能少寫 50% 的 JavaScript。例 `<dialog>` 配 `inert`、`<details>` 配 `open`、`contenteditable` 直接做 inline editor。

  
### 1. `data-*` — 自訂屬性（最常用）

```text
<button
  data-user-id="123"
  data-action="delete"
  data-confirm="確定要刪除？"
>
  刪除
</button>

<script>
  const btn = document.querySelector('button');
  console.log(btn.dataset.userId);    // "123"  ← camelCase 自動轉
  console.log(btn.dataset.action);    // "delete"
  console.log(btn.dataset.confirm);   // "確定要刪除？"
</script>
```

**規則：**

```text
✅ 屬性名 data- 開頭、後面 kebab-case
   data-user-id, data-product-sku

✅ JS 讀寫用 dataset, 自動 camelCase
   element.dataset.userId

✅ 永遠是字串
   要數字自己 Number()、要 JSON 自己 JSON.parse()

❌ 不要用標準屬性名
   data-class / data-id / data-name 容易跟現有屬性混
   → 改 data-product-class / data-user-id 等
```

**典型用法：**

```text
🔘 按鈕綁資料
   <button data-user-id="123" onclick="del(this.dataset.userId)">

📊 元素間關聯
   <div data-target="#modal">點我開 modal</div>

🎨 CSS 條件樣式
   <div data-state="loading"></div>
   [data-state="loading"] { opacity: 0.5; }

📈 Analytics 追蹤
   <a data-event="click_pricing">看方案</a>
```

  
### 2. `contenteditable` — 把任何元素變編輯器

```text
<div contenteditable="true">
  這段文字可以直接編輯！
</div>

<!-- 細部控制 -->
<div contenteditable="plaintext-only">
  只能編輯純文字、不能格式化
</div>

<div contenteditable="false">
  在 contenteditable 父元素內、這塊不可編輯
</div>
```

**真實場景：**

```text
✏️ Notion 風格 inline 編輯
✏️ Google Docs 那種編輯體驗
✏️ 留言區 rich text
✏️ 即時編輯 admin 後台
```

**抓內容：**

```text
const editor = document.querySelector('[contenteditable]');
const html = editor.innerHTML;
const text = editor.textContent;

editor.addEventListener('input', (e) => {
  console.log('內容變了：', e.target.textContent);
});
```

> ⚠️ **`contenteditable` 不是完整 rich text 解決方案**——複雜功能（撤銷 / 重做 / 表格 / 媒體插入）要用 ProseMirror / TipTap / Lexical 等套件。

  
### 3. `draggable` — 拖放

```text
<div draggable="true">可以拖我</div>
<div id="dropzone">放這裡</div>

<script>
  const item = document.querySelector('[draggable]');
  const dropzone = document.getElementById('dropzone');

  item.addEventListener('dragstart', (e) => {
    e.dataTransfer.setData('text/plain', '拖動的資料');
  });

  dropzone.addEventListener('dragover', (e) => {
    e.preventDefault();  // 必須、不然 drop 不會 fire
  });

  dropzone.addEventListener('drop', (e) => {
    e.preventDefault();
    const data = e.dataTransfer.getData('text/plain');
    dropzone.textContent = `收到：${data}`;
  });
</script>
```

**7 個 drag event：**

```text
dragstart   開始拖（拖動元素觸發）
drag        拖動中（持續觸發）
dragend     結束拖

dragenter   進入目標
dragover    在目標上方（持續、要 preventDefault）
dragleave   離開目標
drop        放下（要 preventDefault）
```

> 💡 **行動裝置 drag 限制**：iOS / Android 預設不觸發 dragstart——要用 touchstart/touchmove 自己實作、或用 SortableJS 等套件。

  
### 4. `hidden` vs `inert` vs `aria-hidden`

```text
<!-- hidden — 視覺隱藏 + 不在 layout -->
<div hidden>看不到</div>
等同 display: none

<!-- inert — 視覺看得到但「整塊禁用」（2022+ 新屬性）-->
<div inert>
  看得到但 click 失效、tabindex 失效、screen reader 跳過
</div>

<!-- aria-hidden — 視覺看得到但 screen reader 跳過 -->
<span aria-hidden="true">👀</span>  ← emoji 裝飾用

組合：
✅ 視覺顯示 + 可互動：什麼都不寫
✅ 完全隱藏：hidden
✅ 看得到不可用：inert
✅ 看得到但盲人跳過：aria-hidden="true"
```

**`inert` 真實場景**：modal 開啟時、把背景內容 `inert`、防 Tab 跑出去：

```text
<main id="main" inert>...背景內容（modal 開時 inert）...</main>
<div role="dialog">...modal 內容...</div>

<script>
  // 開 modal
  document.getElementById('main').setAttribute('inert', '');
  // 關 modal
  document.getElementById('main').removeAttribute('inert');
</script>
```

  
### 5. `spellcheck` / `translate` / `lang`

```text
<!-- 拼字檢查 -->
<input spellcheck="true">    ← 預設開
<textarea spellcheck="false"> ← 關（程式碼欄位用）

<!-- 翻譯控制 -->
<p translate="no">這段不要被 Google Translate 翻</p>
<code translate="no">function helloWorld() {}</code>

<!-- 語言（影響字型 / 翻譯 / 拼字檢查）-->
<p lang="en">English content</p>
<p lang="ja">日本語コンテンツ</p>
```

**`translate="no"` 場景：**

```text
✅ 程式碼區塊
✅ 品牌名 / 人名 / 地名
✅ 數學公式
✅ 不該翻譯的專有名詞
```

  
### 6. 其他實用全域屬性

```text
🎯 id              元素唯一識別（CSS / JS / fragment URL）
🎨 class           CSS 類別（可多個、空格分隔）
🎭 style           inline CSS（少用、放外部）
🏷️ title           hover 顯示 tooltip
🔢 tabindex        Tab 順序（0 / -1 / >0）
🌐 lang            語言
↔️  dir            文字方向（ltr / rtl / auto）
🎬 autocapitalize  輸入自動大寫（off / sentences / words）
⌨️ enterkeyhint    手機 Enter 鍵長相（send / go / search / done）
📲 inputmode       手機鍵盤類型（numeric / decimal / email / url / tel）
```

**`enterkeyhint` + `inputmode` — 手機體驗神器：**

```text
<input
  type="text"
  inputmode="numeric"
  enterkeyhint="search"
  placeholder="輸入訂單編號"
>
```

→ 手機跳數字鍵盤 + Enter 鍵變「搜尋」。

  
### 7. 完整實戰：可拖排序的 TODO 列表

📄 **寫進程式檔**：`drag-todo.html`

```text
<!DOCTYPE html>
<html lang="zh-Hant">
<head>
  <meta charset="UTF-8">
  <title>Drag TODO</title>
  <style>
    .todo {
      list-style: none; padding: 0; max-width: 400px;
    }
    .todo li {
      padding: 12px; margin: 4px 0; background: #f3f4f6;
      border-radius: 4px; cursor: grab;
    }
    .todo li.dragging { opacity: 0.4; }
    .todo li[contenteditable="true"]:focus { outline: 2px solid #10b981; }
  </style>
</head>
<body>
  <h1>📝 我的待辦事項</h1>
  <p><small>提示：可拖曳排序、點兩下編輯</small></p>

  <ul class="todo">
    <li draggable="true" data-id="1" contenteditable="false" tabindex="0">寫 ch01 batch4</li>
    <li draggable="true" data-id="2" contenteditable="false" tabindex="0">commit + push</li>
    <li draggable="true" data-id="3" contenteditable="false" tabindex="0">通知林董</li>
  </ul>

  <script>
    const list = document.querySelector('.todo');
    let dragging = null;

    list.addEventListener('dragstart', (e) => {
      dragging = e.target;
      e.target.classList.add('dragging');
    });

    list.addEventListener('dragend', (e) => {
      e.target.classList.remove('dragging');
    });

    list.addEventListener('dragover', (e) => {
      e.preventDefault();
      const after = getAfterElement(list, e.clientY);
      if (after == null) list.appendChild(dragging);
      else list.insertBefore(dragging, after);
    });

    function getAfterElement(container, y) {
      const items = [...container.querySelectorAll('li:not(.dragging)')];
      return items.reduce((closest, child) => {
        const box = child.getBoundingClientRect();
        const offset = y - box.top - box.height / 2;
        if (offset < 0 && offset > closest.offset) {
          return { offset, element: child };
        }
        return closest;
      }, { offset: Number.NEGATIVE_INFINITY }).element;
    }

    // 點兩下進入編輯
    list.addEventListener('dblclick', (e) => {
      if (e.target.tagName !== 'LI') return;
      e.target.contentEditable = 'true';
      e.target.focus();
    });

    // 失焦離開編輯
    list.addEventListener('focusout', (e) => {
      if (e.target.tagName === 'LI') {
        e.target.contentEditable = 'false';
      }
    });
  </script>
</body>
</html>
```

**全域屬性用了 4 個：**
- `draggable="true"` 拖曳
- `data-id="1"` 綁資料
- `contenteditable` 雙擊編輯
- `tabindex="0"` 鍵盤可 focus

  
### 8. 5 大常見坑

```text
❌ 1. data-* 名稱大寫
   → HTML 不分大小寫、寫 data-userId 等於 data-userid
   → 修：永遠 kebab-case（data-user-id）

❌ 2. contenteditable 不處理 paste
   → 貼貼來的 rich text 含一堆樣式 / script
   → 修：preventDefault + 取 plain text 自己 insert

❌ 3. draggable 行動裝置不工作
   → iOS / Android 不觸發 drag events
   → 修：加 touch events / 用套件

❌ 4. hidden 改用 CSS display: none
   → 看起來一樣、但 hidden 是 a11y 友善寫法
   → 修：用 hidden 屬性、不寫 inline CSS

❌ 5. tabindex 用 1, 2, 3 強改順序
   → DOM 順序 + tabindex="0" 就夠
   → 修：只用 0 跟 -1
```

  
### 🎯 重點回顧

- **`data-*`** 是綁自訂資料到元素的標準作法（避開掛亂 attribute）
- **`contenteditable`** 把任何元素變編輯器、適合 inline edit
- **`draggable`** + 7 個 drag event 做拖放、行動裝置要 fallback
- **`hidden` / `inert` / `aria-hidden`** 三種「隱藏」各得其所
- **`enterkeyhint` + `inputmode`** 提升手機體驗
