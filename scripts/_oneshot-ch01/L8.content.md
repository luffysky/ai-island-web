**表單（`<form>`）= HTML 最複雜也最有用的部分**——登入、註冊、留言、買東西、上傳照片、付款、所有「使用者要打字 / 選擇 / 上傳」的場景全靠它。

> 💡 **學會表單 = 學會「跟使用者要資料」**——這是 80% 網站的核心功能。後面接 JS（驗證）+ API（送到 server），完整 onboarding 就成立。

  
### 1. 最基本的表單

```text
<form action="/api/login" method="post">
  <label for="email">Email</label>
  <input id="email" name="email" type="email" required>

  <label for="password">密碼</label>
  <input id="password" name="password" type="password" required minlength="8">

  <button type="submit">登入</button>
</form>
```

**4 個核心觀念：**

```text
🏷️ <form>  整個表單容器
   → action="..." 送去哪
   → method="post" / "get"

✏️ <input>  輸入欄
   → 最重要的元素、20+ 種 type
   → name 是傳給 server 的 key
   → id 配 <label> 用

🏷️ <label>  欄位標籤
   → for="欄位 id" 連結到對應 input
   → 點 label 也會 focus 那個 input（行動裝置超重要）

🔘 <button type="submit">  送出按鈕
```

> 💡 **`name` vs `id` 差很多**：`name` 是傳給 server 的 key（`email=xxx&password=xxx`）、`id` 是給 label 跟 JS 用的識別。兩個都要寫。

  
### 2. `<input>` 的 22 種 type（2026 完整）

```text
📝 文字類
   text         一般文字
   email        Email（手機跳對應鍵盤 + 自動驗證 @）
   tel          電話（手機跳數字鍵盤）
   url          URL（手機跳對應鍵盤）
   password     密碼（顯示星號）
   search       搜尋（多個 X 清除鍵）

🔢 數字類
   number       數字（手機跳數字鍵盤、可設 min/max/step）
   range        滑桿（音量、價格區間）

📅 日期時間類
   date         日期選擇器
   time         時間選擇器
   datetime-local  日期 + 時間
   month        月份
   week         週

🎨 視覺類
   color        顏色選擇器

📁 檔案類
   file         上傳檔案（配 accept 限類型）

🔘 選擇類
   checkbox     多選
   radio        單選（同 name 分組）

✅ 按鈕類
   submit       送出
   reset        重設
   button       一般按鈕（用 JS 處理）

🚫 隱藏類
   hidden       隱藏欄位（給 server 傳值用、使用者看不到）
```

**好處**：手機跳對應鍵盤、瀏覽器自動驗證、無障礙到位。

  
### 3. 6 個必懂屬性

```text
🏷️ name — 傳給 server 的 key（必填）
🆔 id — 配 label 用

✅ required — 必填
   <input required>

📏 minlength / maxlength — 字數限制
   <input minlength="8" maxlength="30">

🔢 min / max — 數字 / 日期範圍
   <input type="number" min="1" max="100">

🎭 placeholder — 提示文字（不是預設值！）
   <input placeholder="輸入您的 Email">

💾 value — 預設值
   <input value="預設文字">

⏸️ disabled — 整個禁用
   <input disabled>

🔒 readonly — 唯讀（不能改但能 submit）
   <input readonly value="不可改">

🔍 pattern — 正則表達式驗證
   <input pattern="[0-9]{4}-[0-9]{4}-[0-9]{4}-[0-9]{4}" placeholder="信用卡格式">

📲 autocomplete — 瀏覽器自動填
   <input autocomplete="email" / "current-password" / "new-password">
```

> 💡 **`placeholder` 不是 label**：placeholder 是「輸入後就消失」的提示、label 是「永遠顯示」的標籤。**永遠寫 label**、不要只靠 placeholder（無障礙必須）。

  
### 4. 其他常用元件

**`<textarea>` — 多行文字：**

```text
<textarea name="message" rows="5" placeholder="留言內容..."></textarea>
```

**`<select>` — 下拉選單：**

```text
<select name="country">
  <option value="tw">台灣</option>
  <option value="jp">日本</option>
  <option value="us">美國</option>
</select>
```

**`<select>` 多選：**

```text
<select name="hobbies" multiple>
  <option>程式</option>
  <option>音樂</option>
  <option>遊戲</option>
</select>
```

**`<fieldset>` + `<legend>` — 分組：**

```text
<fieldset>
  <legend>付款方式</legend>
  <label><input type="radio" name="pay" value="credit"> 信用卡</label>
  <label><input type="radio" name="pay" value="ec"> 綠界</label>
  <label><input type="radio" name="pay" value="line"> LINE Pay</label>
</fieldset>
```

  
### 5. 完整實戰：註冊頁

📄 **寫進程式檔**：`signup.html`

```text
<!DOCTYPE html>
<html lang="zh-Hant">
<head>
  <meta charset="UTF-8">
  <title>註冊 — AI 島</title>
</head>
<body>
  <h1>建立帳號</h1>

  <form action="/api/signup" method="post" autocomplete="on">
    <fieldset>
      <legend>基本資料</legend>

      <div>
        <label for="username">使用者名稱</label>
        <input
          id="username"
          name="username"
          type="text"
          required
          minlength="3"
          maxlength="20"
          pattern="[a-zA-Z0-9_]+"
          autocomplete="username"
        >
      </div>

      <div>
        <label for="email">Email</label>
        <input
          id="email"
          name="email"
          type="email"
          required
          autocomplete="email"
        >
      </div>

      <div>
        <label for="password">密碼（至少 8 字）</label>
        <input
          id="password"
          name="password"
          type="password"
          required
          minlength="8"
          autocomplete="new-password"
        >
      </div>

      <div>
        <label for="birthday">生日</label>
        <input id="birthday" name="birthday" type="date">
      </div>
    </fieldset>

    <fieldset>
      <legend>偏好</legend>
      <label>
        <input type="checkbox" name="newsletter" value="1">
        訂閱每週電子報
      </label>
    </fieldset>

    <button type="submit">建立帳號</button>
  </form>
</body>
</html>
```

**這個範例做對的事：**
- 每個 input 都有 `label`
- 用 `type="email"` / `type="password"` / `type="date"` 取得自動驗證
- `autocomplete` 讓瀏覽器幫填
- `<fieldset>` 分組視覺更清楚
- `<button type="submit">` 一個按鈕送出

  
### 6. `<form>` 的 `action` + `method`

```text
method="get"
  → 資料附在 URL（?email=xxx&q=yyy）
  → 給「搜尋 / 篩選」用、可分享 URL
  → 不要傳密碼（URL 會被記錄）

method="post"
  → 資料在 request body
  → 給「修改資料」用（登入 / 註冊 / 留言）
  → 密碼必用
```

**不寫 action 會怎樣？**——送到當前 URL（同頁）。

  
### 7. 8 大常見坑

```text
❌ 1. input 沒寫 name
   → server 收不到值
   → 修：永遠寫 name

❌ 2. input 沒配 label
   → 無障礙爛、行動裝置難點
   → 修：<label for="xxx"> + <input id="xxx">

❌ 3. 用 placeholder 取代 label
   → 輸入後提示消失、不知道在填什麼
   → 修：永遠寫 label

❌ 4. 沒寫 type
   → 預設 type="text"、手機跳錯鍵盤、無自動驗證
   → 修：email/tel/number/date 各得其所

❌ 5. 沒寫 required / minlength / pattern
   → 全靠 JS 驗證、JS 沒載完就送掉
   → 修：HTML 內建驗證先擋一層

❌ 6. 密碼欄位沒寫 autocomplete="new-password"
   → 瀏覽器密碼管理器搞混
   → 修：登入頁 current-password、註冊頁 new-password

❌ 7. checkbox / radio 沒包 label
   → 不能點文字 focus、行動裝置難用
   → 修：<label><input type="checkbox"> 文字</label>

❌ 8. button 沒寫 type
   → 預設 type="submit"、在表單內亂點就送出
   → 修：非送出按鈕一律寫 type="button"
```

  
### 🎯 重點回顧

- **`<form>` + `<input>` + `<label>` + `<button>` 是 4 大支柱**
- **永遠寫 label 配 id**——無障礙 + 行動裝置點擊範圍
- **22 種 input type 各得其所**——手機鍵盤 + 內建驗證一次到位
- **`autocomplete` 對註冊 / 登入超重要**——使用者體驗 + 密碼管理器
- **HTML 內建驗證先擋一層**（required / minlength / pattern）、JS 驗證是第二層、Server 驗證是第三層
