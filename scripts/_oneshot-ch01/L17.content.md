**語意化 HTML5：landmarks 七兄弟**——`<header>` / `<nav>` / `<main>` / `<article>` / `<section>` / `<aside>` / `<footer>`。**80% 的 div soup 都該改成這 7 個之一**。SEO + a11y + 文件結構一次到位。

> 💡 **L3 教過語意化概念、這課深入講「怎麼判斷該用哪個」**——這是新手最容易搞錯的部分。文末附「決策樹」直接看圖選擇。

  
### 1. 七兄弟一覽

```text
🏠 <header>     頁首 / section 開頭區（可重複出現）
🧭 <nav>        主要導覽（不是所有連結都要包）
📄 <main>       主內容（每頁只能有 1 個）
📰 <article>    獨立可分發的內容（可單獨 RSS 一份）
📑 <section>    主題式分區（必須有標題）
🛏️ <aside>      相關但非主內容（側欄 / 廣告 / 補充）
🦶 <footer>     頁尾 / section 結尾區（可重複出現）
```

  
### 2. `<main>` — 每頁只能 1 個

```text
✅ 包整個「主內容」（不含 header / nav / footer）
✅ 每頁只能用 1 次
✅ 螢幕閱讀器有「跳到 main」捷徑

<body>
  <header>...</header>
  <nav>...</nav>
  <main>
    這裡是這頁的核心內容
  </main>
  <footer>...</footer>
</body>
```

  
### 3. `<article>` vs `<section>` — 最常搞錯

**關鍵問題**：「這段內容**單獨拿出來放 RSS / 分享出去**還合理嗎？」

```text
✅ 合理 → <article>
   - 部落格文章
   - 新聞報導
   - 商品介紹
   - 留言（每則一個 article）
   - 論壇貼文

✅ 不合理（只是分區）→ <section>
   - 「最新消息」「熱門商品」「關於我們」這些頁面分區
   - 必須有 <h2>...<h6> 標題
```

**範例對比：**

```text
<!-- 部落格列表頁 -->
<main>
  <h1>最新文章</h1>

  <section>
    <h2>本月精選</h2>

    <article>
      <h3>為什麼 React 比 Vue 紅</h3>
      <p>...</p>
    </article>

    <article>
      <h3>2026 前端必學</h3>
      <p>...</p>
    </article>
  </section>

  <section>
    <h2>本週熱門</h2>
    <article>...</article>
    <article>...</article>
  </section>
</main>
```

> 💡 **`<article>` 可以巢狀**：文章內含留言列表、每則留言也是 `<article>`、合理且常見。

  
### 4. `<nav>` — 不是所有連結都要包

```text
✅ 該包：
   - 主導覽列（網站主功能）
   - 麵包屑
   - 文章內目錄（ToC）
   - 分頁器（pagination）

❌ 不該包：
   - footer 內一堆小連結（用 <ul> 就好）
   - 一段內文裡的相關連結
   - sidebar 內隨機連結
```

**完整導覽範例：**

```text
<header>
  <nav aria-label="主導覽">
    <ul>
      <li><a href="/">首頁</a></li>
      <li><a href="/blog">部落格</a></li>
      <li><a href="/about">關於</a></li>
    </ul>
  </nav>
</header>

<nav aria-label="麵包屑">
  <ol>
    <li><a href="/">首頁</a></li>
    <li><a href="/blog">部落格</a></li>
    <li>當前文章</li>
  </ol>
</nav>
```

> 💡 **多個 `<nav>` 一定要加 `aria-label`** 區分用途——「主導覽」「麵包屑」「分頁」。

  
### 5. `<aside>` — 補充內容

```text
✅ 適合用 <aside>：
   - 部落格側欄（作者介紹 / 訂閱 / 熱門文章）
   - 文章內補充說明（「相關閱讀」）
   - 廣告
   - 引言（pull quote）
```

**範例：**

```text
<article>
  <h1>什麼是 RAG</h1>
  <p>主要內文...</p>

  <aside aria-label="相關閱讀">
    <h3>延伸閱讀</h3>
    <ul>
      <li><a href="...">向量資料庫入門</a></li>
      <li><a href="...">Pinecone vs Weaviate</a></li>
    </ul>
  </aside>

  <p>...繼續主內文</p>
</article>
```

  
### 6. `<header>` / `<footer>` — 不只全站、每個 section 都能用

**全站 header / footer：**

```text
<body>
  <header>
    <img src="logo.svg" alt="站名">
    <nav>...</nav>
  </header>

  <main>...</main>

  <footer>
    <p>© 2026 站名</p>
  </footer>
</body>
```

**每個 article 也能有自己的 header / footer：**

```text
<article>
  <header>
    <h1>文章標題</h1>
    <p>作者 · 2026-05-28</p>
  </header>

  <p>正文...</p>

  <footer>
    <p>標籤：#React #Next.js</p>
    <p>👍 27 · 💬 5 留言</p>
  </footer>
</article>
```

### 8. 決策樹（看圖選對 tag）

```text
是「整頁主內容」嗎？
  └ 是 → <main>
  └ 否 ↓

是「導覽連結」嗎？
  └ 是 → <nav>
  └ 否 ↓

是「頁首 / section 開頭」嗎？
  └ 是 → <header>
  └ 否 ↓

是「頁尾 / section 結尾」嗎？
  └ 是 → <footer>
  └ 否 ↓

是「獨立可分發的內容」嗎？
  └ 是 → <article>
  └ 否 ↓

是「相關但非主內容」嗎？
  └ 是 → <aside>
  └ 否 ↓

是「有標題的主題分區」嗎？
  └ 是 → <section>
  └ 否 → <div>
```

  
### 9. 5 大常見坑

```text
❌ 1. 一頁多個 <main>
   → 規格不允許、a11y 工具會混亂
   → 修：每頁只能 1 個 <main>

❌ 2. <section> 沒有 <h?> 標題
   → <section> 必須有 heading、不然用 <div>
   → 修：每個 <section> 至少 <h2>

❌ 3. <article> 包「不是獨立內容」的東西
   → 例 sidebar 整個包 <article>
   → 修：sidebar 用 <aside>

❌ 4. <nav> 包所有 <a>
   → footer 一堆連結也包 <nav>、太多 nav 反而干擾
   → 修：只包「真的是導覽用途」的連結

❌ 5. 多個 <nav> 沒 aria-label
   → 螢幕閱讀器念「navigation, navigation, navigation」分不清
   → 修：每個 <nav> 加 aria-label="主導覽" / "麵包屑" / ...
```

  
### 🎯 重點回顧

- **七兄弟取代 80% div soup**：header / nav / main / article / section / aside / footer
- **`<main>` 每頁 1 個、`<section>` 必須有標題、`<article>` 必須能獨立**
- **`<nav>` 只包真正導覽用途的連結、多個 nav 一定 aria-label**
- **`<header>` / `<footer>` 不只全站、article / section 也能有自己的**
- **看不確定就用決策樹**
