CSS 全名 **Cascading Style Sheets（層疊樣式表）**——「**Cascading（串接）**」是它的靈魂。同一個元素被多條規則指到時、誰贏？這課把 CSS 三大法則 **Cascading（串接）/ Specificity（特異性）/ Inheritance（繼承）** 一次講透。

> 💡 **這課的價值**：80% 的 CSS bug 都是「樣式沒生效、想不通為什麼」。**搞懂這三條法則 = 從此 debug 不再瞎猜**、F12 一打開就知道是誰贏了。

  
### 1. Cascading（串接）—— CSS 的核心精神

「Cascading（串接）」= **多條規則疊在一起、後面蓋前面**。瀏覽器讀完所有 CSS、按優先順序合併成最終樣式。

```text
影響優先順序的 3 個因素（由低到高）：

1. 來源（origin）
   → 瀏覽器預設 < 使用者設定 < 你寫的

2. 特異性（specificity）← 最常用、下節詳教
   → 越「精準」的選擇器贏

3. 出現順序（order）
   → 同樣特異性時、後寫的贏
```

> 💡 **「Cascading」這個字怎麼來**：像瀑布（cascade）一層一層往下流、後面的覆蓋前面的。**這是 CSS 命名的由來**。

  
### 2. Specificity（特異性）—— 誰的權重大

**specificity（特異性）** = 不同 CSS 選擇器互相搶優先權的計分規則。算法用四個數字 `(行內, ID, class, 標籤)`：

| 選擇器 | 權重 | 說明 |
|---|---|---|
| `p` | 0,0,0,1 | 標籤 = 1 分 |
| `.card` | 0,0,1,0 | class = 10 分 |
| `.card p` | 0,0,1,1 | class + 標籤 = 11 分 |
| `.card .title` | 0,0,2,0 | 兩個 class = 20 分 |
| `#header` | 0,1,0,0 | ID = 100 分 |
| `style="..."` | 1,0,0,0 | 行內 = 1000 分 |
| `!important` | ∞ | 核子彈、勝過一切 |

📄 **寫進程式檔**：`style.css`

```text
/* 兩條規則都選到同個 <p class="text"> */
p { color: blue; }       /* 權重 0,0,0,1 */
.text { color: red; }    /* 權重 0,0,1,0 ← 贏、因為 10 > 1 */

/* 同權重時、後寫的贏 */
.a { color: blue; }
.a { color: red; }       /* 後寫、贏 */
```

> 💡 **快速心算**：(1) 數有幾個 `#`（×100）(2) 數有幾個 `.`（×10）(3) 數有幾個標籤（×1）(4) 行內 `style=""`（×1000）(5) `!important` 直接無敵。

  
### 3. F12 看 specificity（DevTools 神器）

不確定哪條規則贏？打開瀏覽器 F12 → Elements → 點任一元素 → 右邊 Styles 面板：

```text
✅ 沒被刪掉的 → 這條贏了
❌ 被劃刪除線的 → 這條輸了（被覆蓋）
```

**F12 是 CSS 工程師的命**。不會用 F12 = 永遠在瞎猜。

  
### 4. Inheritance（繼承）—— 自動傳給子元素

**inheritance（繼承）** = 某些屬性會「**自動傳給子元素**」、不用每個都重寫。

```text
✅ 會繼承（給子元素用）：
   color（顏色）
   font-family（字型）
   font-size（字級）
   line-height（行高）
   text-align（對齊）

❌ 不會繼承（每個元素獨立）：
   margin（外距）
   padding（內距）
   border（邊框）
   background（背景）
   width / height（寬高）
```

📄 **寫進程式檔**：`style.css`

```text
body {
  color: #333;             /* 會繼承 → 全頁文字都 #333 */
  font-family: sans-serif; /* 會繼承 → 全頁字型 */
  background: white;       /* 不繼承 → 只有 body 是白底 */
  border: 1px solid;       /* 不繼承 → 只有 body 有邊框 */
}

/* 強制繼承 */
button { color: inherit; }
/* → 按鈕的 color 強制吃父元素的值 */
```

> 💡 **為什麼這樣設計**：(1) 文字相關（顏色、字型）通常整頁一致 → 預設繼承 (2) 排版相關（margin、border）通常每個元素獨立 → 預設不繼承。**省下大量重複寫**。

  
### 5. `!important` —— 核子彈、慎用

`!important` 寫在值後面、**強制無視特異性、永遠贏**：

📄 **寫進程式檔**：`style.css`

```text
.btn { color: blue !important; }
/* → 除非另一個也 !important 且權重更高、否則絕對贏 */
```

> ⚠️ **不要濫用 !important**：一個 `!important` 會逼後人也用 `!important` 救火、越滾越大、最後 specificity 系統完全失靈。**實戰只在兩種情況用**：(1) 對抗第三方寫死的樣式（廣告、外掛、Bootstrap 元件）(2) utility class（如 Tailwind 的 `!hidden`）。

  
### 6. Cascade Layers（@layer）—— 2022 新功能

過去管理 CSS 優先級超痛苦、現在 CSS 原生支援「**分層**」：

📄 **寫進程式檔**：`style.css`

```text
/* 定義層級順序：後面的層贏 */
@layer reset, base, components, utilities;

@layer reset {
  * { margin: 0; padding: 0; }
}

@layer base {
  body { font-family: sans-serif; }
}

@layer components {
  .card { border: 1px solid gray; }
}

@layer utilities {
  .hidden { display: none; }
}
```

> 💡 **`@layer` 的威力**：不管特異性多高、**後面層的規則永遠贏前面層**。`utilities` 層的 `.hidden` 一定贏 `components` 層的 `.card`、不用 `!important`。**Tailwind v4 / shadcn/ui 都用這架構**。

  
### 7. 實戰：debug 樣式沒生效

樣式套不上去？按這個順序檢查：

```text
✅ 1. F12 看 Styles 面板
   → 規則有沒有出現？
   → 出現了但被劃刪除線？被誰蓋掉？

✅ 2. 檢查 selector 有沒有拼錯
   → .crad 應該是 .card
   → 偽類少冒號（hover 應該 :hover）

✅ 3. 檢查特異性
   → 你寫的選擇器特異性夠高嗎？
   → 不夠就加一個 class 或用更精準的選擇器

✅ 4. 檢查是否被 inherit 蓋過
   → 父元素是不是有強制設定？

✅ 5. 最後手段：暫時加 !important 驗證
   → 加了還沒效 → 不是特異性問題、是 selector 沒選到
   → 加了有效 → 確認是特異性、改 selector 別用 !important
```

  
### 8. 三大法則整合範例

📄 **寫進程式檔**：`style.css`

```text
/* 來源：你寫的 CSS（最高）*/
body {
  color: #333;            /* inheritance：全頁繼承 */
  font-family: sans-serif;
}

p { color: gray; }                          /* 權重 1 */
.warning { color: orange; }                 /* 權重 10、贏 p */
.warning.urgent { color: red; }             /* 權重 20、贏 .warning */
#alert { color: darkred; }                  /* 權重 100、贏所有 class */

/* HTML: <p id="alert" class="warning urgent">⚠️</p> */
/* → 最終顏色：darkred（#alert 贏）*/
```

  
### 5 大常見坑

```text
❌ 1. 「我明明寫了、為什麼沒效果」
   → 99% 是被另一條更高特異性蓋掉
   → 修：F12 看 Styles 面板找凶手

❌ 2. 一遇到問題就 !important
   → specificity 系統會徹底壞掉、後續維護地獄
   → 修：先想能不能用更精準 selector 解

❌ 3. id 選擇器到處用
   → 特異性 100、想覆蓋只能再用 id 或 !important
   → 修：99% 用 class、id 留給 JS

❌ 4. 以為 margin / padding 會繼承
   → 它們不繼承、要每個元素分別寫
   → 修：背好「會繼承 vs 不繼承」清單

❌ 5. 不會用 F12
   → 永遠在瞎猜「為什麼樣式不對」
   → 修：先學 F12、再寫 CSS（DevTools L24 詳教）
```

  
### 🎯 重點回顧

- **Cascading（串接）= CSS 靈魂**——多條規則疊在一起、後面蓋前面
- **Specificity（特異性）算法**：`(行內, ID, class, 標籤)`
- **Inheritance（繼承）**：color / font 會繼承、margin / border 不繼承
- **`!important` 是核子彈**——只在對抗第三方時用
- **`@layer` 分層管理**——2022 新標準、Tailwind v4 在用
- **F12 是命**——不會用 = 永遠 debug 不出來

> 💡 **下一課（L4）**：每個元素都是一個「盒子」——content / padding / border / margin。**搞懂盒模型 = 從此寬高怎麼算都不會錯**。新手最常踩的「`width: 200px` 為什麼變 240px」就是盒模型問題。
