**除錯 CSS = F12 是命**。不會用 DevTools 的 CSS 工程師 = 永遠在瞎猜。這課把 Chrome DevTools 必會的技巧一次學完、教你 **debug 從 1 小時降到 5 分鐘**。

> 💡 **這課的價值**：學完 F12 = 你看到別人的網站樣式、5 秒拆解出來。**85% 的 CSS bug 都是「特異性」或「上下文」**——用對工具一查就知道。

  
### 1. F12 必學的 7 個面板

**打開 DevTools**：F12（Windows）或 Cmd + Option + I（Mac）。

```text
1. Inspect Element（檢查元素）
   → 右鍵任何元素 → 檢查
   → 看到該元素套用了哪些 CSS

2. Styles 面板
   → 每條規則旁邊有勾選框、可即時 toggle
   → 被劃刪除線 = 那條輸了

3. Computed 面板
   → 最終計算結果（多條規則合併後）
   → 解決「為什麼這個元素是這顏色」

4. Layout 面板
   → 盒模型視覺化（margin / padding / border）
   → 點數字直接改

5. Flex / Grid 工具
   → 標出 flex 排列、grid 線
   → 排版疑難雜症的救星

6. Animation 面板
   → 時間軸可拖、慢速重播
   → 看動畫每幀變化

7. Coverage 工具
   → 看哪些 CSS / JS 沒被用到
   → 找出垃圾代碼
```

  
### 2. Inspect + Styles 面板（最常用）

**操作流程**：

🖥️ **在瀏覽器執行**

```text
1. 開網頁 → F12
2. 點左上角「箭頭 icon」（或 Ctrl+Shift+C）
3. 滑鼠移到任何元素 → 點選
4. 右邊「Styles」面板就會顯示套用的 CSS

🔍 重點觀察：
   ✅ 沒被刪除線的 → 這條贏了
   ❌ 被刪除線的 → 這條輸了（被覆蓋）
   📍 規則旁的檔名 → 點擊跳到原始檔
```

**Styles 面板的功能**：

```text
✅ 勾選框 toggle
   → 暫時關掉某條規則、看結果
   → 不用改原始檔

✏️ 直接編輯
   → 點屬性或值、直接改
   → 即時看效果（不會存進原檔）
   → 滿意了再複製到 IDE

➕ 加新規則
   → element.style 區塊（最上方）
   → 直接打新屬性、即時測試

🎨 顏色選擇器
   → 點顏色方塊
   → 視覺挑色、自動轉 HEX / RGB / HSL / OKLCH
```

  
### 3. Computed 面板（最終結果）

**Styles 顯示「所有規則」、Computed 顯示「最終計算值」**：

```text
Styles 面板（多條規則）：
  color: blue;
  color: red;   ← 贏
  color: green;

Computed 面板（最終結果）：
  color: red
```

> 💡 **Computed 解決「為什麼這個 padding 是 24px」**——可能是 `padding: 16px` + 繼承 + `!important` 互相打架的結果。Computed 一看就知道贏家。

  
### 4. Layout / Box Model 面板

**視覺化盒模型**：

```text
點任何元素 → 右邊面板 → 看到一個多層方框

     margin（橘）
     padding（綠）
     border（黃）
     content（藍）

→ 每個數字可以直接點擊修改
→ 滑鼠 hover 各層、頁面上顏色標出
```

  
### 5. Flex 工具

**找到 flex 容器、點 Styles 面板的「flex」徽章**：

```text
畫面上會浮現視覺輔助：
  → 標出 main axis 跟 cross axis 方向
  → 顯示每個子元素的對齊狀態
  → justify-content / align-items 可下拉切換
```

> 🚀 **這個工具救了我無數次**——flex 排錯方向、子元素沒對齊、`flex: 1` 沒效——F12 一看立刻知道哪裡錯。

  
### 6. Grid 工具

**Grid 容器點旁邊的「grid」徽章**：

```text
畫面上會浮現：
  → 標出所有 grid 線（垂直水平）
  → 每條線有編號（1, 2, 3, ...）
  → 看到 grid-area 怎麼排
```

  
### 7. Animation 面板

**看動畫每一幀變化**：

```text
DevTools → 三點 → More tools → Animations
→ 觸發動畫
→ 面板顯示時間軸
→ 可以拖慢速、單步看每幀
```

  
### 8. 常見問題對照表

| 症狀 | 怎麼查 |
|---|---|
| 樣式沒生效 | (1) selector 拼字 (2) 特異性（Styles 看刪除線）(3) 是否被覆蓋 |
| flex 子元素沒平均 | (1) 父元素是 flex 嗎？(2) 子元素有 flex: 1 嗎？(3) min-width: 0 試試 |
| 絕對定位沒對齊 | 父元素有 position: relative 嗎？ |
| 滾動不順 | 動畫是否動 layout 屬性？改用 transform |
| 移動裝置看起來不對 | (1) 有 viewport meta 嗎？(2) 單位用對嗎？ |
| z-index 沒效果 | (1) 有 position 嗎？(2) 父元素的 stacking context？ |
| 元素消失了 | (1) display: none (2) width / height 0 (3) overflow: hidden 父元素 |

  
### 9. 暴力除錯神技

**找不到問題時、用暴力方法視覺化所有元素**：

📄 **寫進程式檔**：`debug.css`

```text
/* 把所有元素加邊框 */
* { outline: 1px solid red !important; }

/* 父子層級用不同顏色 */
* { outline: 1px solid red; }
* * { outline: 1px solid blue; }
* * * { outline: 1px solid green; }
* * * * { outline: 1px solid yellow; }
```

> 💡 **這招看「**版面到底由哪些元素組成**」**——找看不到的 div、找撐爆容器的元素、超有效。

  
### 10. Console 神技

**在 Console 直接執行 CSS 相關 JS**：

```text
// 找所有套用某 class 的元素
$$('.card')

// 看元素的計算樣式
getComputedStyle($0).backgroundColor
// $0 = 上次在 Elements 選的元素

// 直接改樣式（不改檔案）
$0.style.background = 'red'

// 顯示所有 CSS 變數
getComputedStyle(document.documentElement)
  .cssText.split(';')
  .filter(s => s.includes('--'))
```

### 13. minimal reproducer（最小重現）

**CSS 寫不出來、解決不了 bug？做個最小重現**：

🖥️ **在瀏覽器執行**

```text
1. 開 CodePen / JSFiddle / StackBlitz
2. 把問題濃縮到最小
   → 移除無關 CSS
   → 移除無關 HTML
3. 一步一步加回來、直到問題出現
4. 那個「最後加的東西」= 兇手
```

> ✨ **接案小技巧**：85% 的問題都是「**特異性**」或「**上下文**」。**多花 5 分鐘做 minimal repro、比盲改 1 小時有效**。

  
### 14. 實戰：debug 完整流程

```text
情境：「我寫的 .card { padding: 24px } 沒效」

步驟：
1. F12 → Inspect 那張卡片
2. Styles 面板找 .card 規則
   
   情況 A：.card 不存在
   → selector 拼錯（.crad）
   → 修 selector
   
   情況 B：.card 存在但被劃刪除線
   → 被另一條更高特異性蓋掉
   → 看是誰、改 selector 或加特異性
   
   情況 C：.card 存在沒被刪、但 padding 視覺對不上
   → Computed 看實際值
   → 看是不是 box-sizing 問題
   → 看是不是繼承 inherit 的值

3. 如果還是沒解
   → 暫時加 !important 驗證
   → 加了有效 → 確認是特異性、改 selector
   → 加了沒效 → 不是這條規則的事、selector 沒選到
```

  
### 5 大常見坑

```text
❌ 1. 不用 F12、瞎改 CSS
   → 永遠不知道為什麼有效或失效
   → 修：先 F12 看、再改

❌ 2. 改了 CSS 沒重整
   → 看到舊版以為沒效果
   → 修：Ctrl+F5 強制重整、或裝 Live Server

❌ 3. 用 console.log 在 CSS（不存在）
   → CSS 沒有 log
   → 修：用 background: red 暴力標記元素

❌ 4. 沒看 Computed 就在改規則
   → 改了沒效果（其他規則贏）
   → 修：先看 Computed 確認贏家

❌ 5. minimal repro 不做、直接問
   → 別人看不懂你的問題
   → 修：濃縮到最小、放 CodePen
```
