**selector（選擇器）** = CSS 的核心、告訴瀏覽器「**你要對誰下指令**」。寫 CSS 等於 90% 時間在挑選擇器——選對人、改誰都很容易；選錯人、整個樣式失靈。

> 💡 **這課的價值**：學會選擇器 = **任何元素都選得到**。從基礎標籤到 2023 神級 `:has()`、一次全部學完、後面 23 課的所有範例你都看得懂。

  
### 1. 基礎選擇器（5 種）

📄 **寫進程式檔**：`style.css`

```text
/* 標籤選擇器：選所有這種標籤 */
p { color: gray; }
/* → 所有 <p> 都變灰 */

/* class 選擇器：選 class="card" 的元素（前面加 .）*/
.card { padding: 20px; }
/* → 所有 <div class="card"> 都加 padding */

/* id 選擇器：選 id="header" 的元素（前面加 #）*/
#header { background: black; }
/* → 整個頁面只有一個 id="header" */

/* 屬性選擇器：選有特定屬性的元素 */
a[target="_blank"] { color: blue; }
/* → 所有「新分頁開啟」的連結變藍 */

/* 通用選擇器（*）：選所有元素 */
* { box-sizing: border-box; }
/* → 每個專案開頭都寫這條（L4 詳教）*/
```

> 💡 **class（類別）vs id（識別碼）**：(1) **class** 可以重複用很多次（一個頁面 100 個 `.card` 都行）(2) **id** 整頁只能有一個（像身分證號）。**99% 場景用 class、id 留給 JS 抓特定元素**。

  
### 2. 組合選擇器（4 種關係）

選擇器之間有「**關係**」——後代、子代、相鄰、兄弟：

📄 **寫進程式檔**：`style.css`

```text
/* 後代（空格）：div 內所有 p、不管多深 */
div p { margin: 10px; }
/* 選到：<div><section><p>這個</p></section></div> */

/* 直接子代（>）：div 的「直接」p、孫子不算 */
div > p { color: red; }
/* 選到：<div><p>這個</p></div> */
/* 沒選到：<div><section><p>這個</p></section></div> */

/* 相鄰兄弟（+）：緊接在 h1 後面的「第一個」p */
h1 + p { font-weight: bold; }
/* 選到：<h1>標題</h1><p>緊接在後的這個</p> */

/* 一般兄弟（~）：h1 後面「所有」p */
h1 ~ p { color: gray; }

/* 多重（逗號）：a 或 b 或 c 都選 */
.primary, .success, .info { color: white; }
```

> 💡 **後代（空格）vs 子代（>）差很多**：`div p` 是「div 裡面所有 p、孫子也算」、`div > p` 是「只有 div 的直接小孩 p、孫子不算」。**新手最常踩這個坑**。

  
### 3. 偽類（pseudo-class）—— 狀態選擇器

**pseudo-class（偽類）** —— 不是真的選一個元素、而是選「**處於某個狀態**」的元素。前面加冒號 `:`：

📄 **寫進程式檔**：`style.css`

```text
/* 滑鼠 hover（懸停）時 */
button:hover { background: blue; }

/* 按下時 */
button:active { transform: scale(0.95); }

/* 取得焦點時（用鍵盤 Tab 過來、或點擊）*/
input:focus { outline: 2px solid orange; }

/* 只在「鍵盤」觸發 focus 時、不被滑鼠 click 觸發 */
input:focus-visible { outline: 3px solid orange; }
/* → 無障礙必備、不會在滑鼠點時也彈出醜醜的外框 */

/* 第幾個子元素 */
li:first-child { font-weight: bold; }     /* 第一個 */
li:last-child { border-bottom: 0; }       /* 最後一個 */
li:nth-child(2n) { background: #f0f0f0; } /* 雙數 */
li:nth-child(3) { color: red; }           /* 第 3 個 */

/* 沒被選中的（:not）*/
div:not(.active) { opacity: 0.5; }
/* → 除了 .active 之外的 div 都半透明 */
```

> 💡 **常見偽類速查**：`:hover`（懸停）`:active`（按下）`:focus`（焦點）`:focus-visible`（鍵盤焦點）`:first-child`（第一個）`:last-child`（最後一個）`:nth-child(n)`（第 n 個）`:not(...)`（除了…之外）`:disabled`（被停用）`:checked`（勾選）。

  
### 4. `:has()` —— 2023 神級「父選擇器」

過去 CSS 只能「**從父選子**」、不能反向「**根據子選父**」。`:has()` 改變一切：

📄 **寫進程式檔**：`style.css`

```text
/* 「有圖的文章」可以這樣選 */
article:has(img) { padding-top: 0; }
/* → 只有 <article> 裡面有 <img> 時才套用 */

/* 「有錯誤輸入的表單」按鈕變灰 */
form:has(input:invalid) button { opacity: 0.5; }

/* 「沒有任何商品的購物車」顯示空狀態 */
.cart:not(:has(.item)) .empty-state { display: block; }
```

> 💡 **`:has()` 為什麼超強**：過去要寫 JS 才能做的事（例：「有沒有某個子元素」）、現在純 CSS 就搞定。2023 後全瀏覽器支援、**現代 CSS 第一神技**。

  
### 5. 偽元素（pseudo-element）—— 插入虛擬內容

**pseudo-element（偽元素）** —— 不是選現有元素、而是「**生出**」虛擬元素來裝飾。前面加兩個冒號 `::`：

📄 **寫進程式檔**：`style.css`

```text
/* 在元素「前 / 後」插入內容 */
.quote::before { content: '"'; }
.quote::after  { content: '"'; }
/* → 自動在 .quote 文字前後加引號 */

/* 第一個字、第一行 */
p::first-letter { font-size: 3em; float: left; }
/* → 雜誌風「首字放大」*/
p::first-line { font-weight: bold; }

/* 選取的文字 */
::selection { background: yellow; }
/* → 使用者反白選字時的顏色 */

/* placeholder（輸入框預設文字）樣式 */
input::placeholder { color: #aaa; }
```

> 💡 **`:` 跟 `::` 差別**：(1) **單冒號 `:`** = 偽類（狀態：hover / focus / nth-child）(2) **雙冒號 `::`** = 偽元素（虛擬內容：before / after / first-letter）。**舊瀏覽器都用 `:`、現代寫法 `::`**。

  
### 6. 屬性選擇器進階

屬性選擇器不只能匹配「完全相等」、還能「包含 / 開頭 / 結尾」：

📄 **寫進程式檔**：`style.css`

```text
/* = 完全相等 */
input[type="email"] { border: 1px solid blue; }

/* ^= 開頭是 */
a[href^="https://"] { color: green; }
/* → 所有 https 連結 */

/* $= 結尾是 */
a[href$=".pdf"] { background: url(pdf-icon.png); }
/* → 所有指向 PDF 的連結 */

/* *= 包含 */
a[href*="youtube.com"] { color: red; }
/* → 所有 YouTube 連結 */
```

  
### 7. 實戰：經典導覽列高亮當前頁

📄 **寫進程式檔**：`style.css`

```text
/* 導覽列基本樣式 */
nav a {
  color: #555;
  padding: 8px 16px;
  text-decoration: none;
}

/* 滑鼠 hover */
nav a:hover {
  background: #f0f0f0;
}

/* 鍵盤焦點 */
nav a:focus-visible {
  outline: 2px solid blue;
}

/* 當前頁高亮（HTML 上有 class="active"）*/
nav a.active {
  color: blue;
  font-weight: bold;
}
```

  
### 5 大常見坑

```text
❌ 1. 後代選擇器（空格）跟子代（>）搞混
   → div p 選孫子、div > p 不選孫子
   → 修：不確定就用空格（後代）、99% 場景夠用

❌ 2. id 用太多
   → id 特異性超高、後面想覆蓋寫不掉
   → 修：99% 場景用 class、id 留給 JS

❌ 3. :focus 直接 outline: none 關掉
   → 鍵盤使用者（無障礙）完全找不到焦點
   → 修：永遠保留 :focus 樣式、用 :focus-visible 客製

❌ 4. 偽元素用單冒號（:before 應該 ::before）
   → 舊瀏覽器能跑、但不規範
   → 修：偽類 :、偽元素 ::

❌ 5. nth-child 跟 nth-of-type 搞混
   → :nth-child(2) 是「第 2 個子元素、且必須是這標籤」
   → :nth-of-type(2) 是「這標籤中的第 2 個」
   → 不確定先用 nth-of-type 比較不會錯
```

  
### 🎯 重點回顧

- **selector = CSS 的核心**——選對人、樣式才會套用
- **5 種基礎**：標籤 / `.class` / `#id` / `[屬性]` / `*`
- **4 種組合**：空格（後代）/ `>`（子代）/ `+`（相鄰）/ `~`（兄弟）
- **偽類 `:hover` `:focus-visible` `:nth-child`**——選「狀態」
- **`:has()` 是 2023 神技**——CSS 第一次能「根據子選父」
- **`class` 99%、`id` 留給 JS**——專業習慣

> 💡 **下一課（L3）**：兩條 CSS 規則都選到同個元素時、誰贏？這就是 **Cascading（串接）、Specificity（特異性）、Inheritance（繼承）** CSS 三大法則。沒搞懂這三個、debug 永遠卡關。
