每個 HTML 元素都有一個「**人格**」——是塊狀獨佔一行？還是內聯擠在同一行？是普通盒子還是 flex 容器？這由 **`display`** 屬性決定。**一個屬性、改變整個排版邏輯**。

> 💡 **這課的價值**：搞懂 `display` = 知道為什麼 `<div>` 自動換行、`<span>` 不會。後面 flex / grid 都是 `display` 的延伸——**這課是排版的起點**。

  
### 1. display 的 7 種人格

| 值 | 行為 | 例子 |
|---|---|---|
| `block` | **獨佔一行**、可設寬高 | `<div>` `<p>` `<h1>` `<section>` |
| `inline` | **跟其他在同一行**、寬高失效 | `<span>` `<a>` `<strong>` `<em>` |
| `inline-block` | **同行 + 可設寬高**（兩者優點都拿）| 按鈕、tag、徽章 |
| `flex` | 變成 **flex 容器**（一維排版） | 導覽列、卡片組 |
| `grid` | 變成 **grid 容器**（二維排版） | 圖庫、儀表板 |
| `none` | **完全消失**（連占位都沒）| 切換顯示 |
| `contents` | **盒子消失但子元素留** | 進階布局 |

  
### 2. block（區塊）—— 獨佔一行

📄 **寫進程式檔**：`style.css`

```text
/* HTML：<div>A</div> <div>B</div> <div>C</div> */
div {
  display: block;  /* 預設就是 block、寫不寫都一樣 */
  background: lightblue;
}

/* 結果：A / B / C 各自一行 */
/* 即使每個只有一個字、後面也會空一大片 */
```

**block 特性**：
- 自動換行（前後都換）
- 寬度預設「**撐滿父元素**」
- 可以設 `width` / `height` / `margin` / `padding`

  
### 3. inline（行內）—— 同一行

📄 **寫進程式檔**：`style.css`

```text
/* HTML：<span>A</span> <span>B</span> <span>C</span> */
span {
  display: inline;  /* 預設就是 inline */
  background: yellow;
  width: 200px;   /* ❌ 無效、inline 不吃寬高 */
  height: 100px;  /* ❌ 無效 */
}

/* 結果：A B C 排在同一行 */
```

**inline 特性**：
- **不換行**（跟下一個元素同一行）
- **寬高設了沒用**——只能容納內容大小
- **垂直 padding / margin 設了會視覺上跑掉**（但不會把後續元素推開）

> 💡 **常見 inline 元素**：`<a>` `<span>` `<strong>` `<em>` `<img>` `<input>` `<button>`。

  
### 4. inline-block —— 同行 + 可設寬高

兩者優點都拿：

📄 **寫進程式檔**：`style.css`

```text
.tag {
  display: inline-block;
  width: 80px;
  height: 30px;
  padding: 4px 8px;
  background: lightblue;
  border-radius: 4px;
}

/* HTML：<span class="tag">A</span> <span class="tag">B</span> */
/* 結果：A 跟 B 同一行、但兩個都是 80×30 的盒子 */
```

> 💡 **用在哪**：tag 標籤、徽章、自訂按鈕、不換行的 icon + 文字組合。**現在大部分場景已被 flex 取代**、但仍是基本功。

  
### 5. flex / grid（之後兩課詳教）

`display: flex` 跟 `display: grid` 把元素「**升級**」成排版容器：

```text
display: flex
  → 一維排版（一排東西怎麼排）
  → 導覽列、卡片橫排、垂直水平置中
  → L6 詳教

display: grid
  → 二維排版（行 × 列）
  → 儀表板、雜誌排版、複雜佈局
  → L7 詳教
```

  
### 6. none vs visibility: hidden vs opacity: 0（隱藏的 3+ 種方式）

「隱藏元素」其實有 4 種方式、**各自意義不同**：

📄 **寫進程式檔**：`style.css`

```text
/* 1. display: none —— 完全消失、不占位 */
.hidden-1 { display: none; }

/* 2. visibility: hidden —— 看不到但占位 */
.hidden-2 { visibility: hidden; }

/* 3. opacity: 0 —— 透明但可點、占位、可動畫 */
.hidden-3 { opacity: 0; }

/* 4. 螢幕閱讀器友善（無障礙隱藏）*/
.sr-only {
  position: absolute;
  width: 1px; height: 1px;
  overflow: hidden;
  clip-path: inset(50%);
  white-space: nowrap;
  border: 0;
}
/* → 視覺上看不到、但螢幕閱讀器仍會念給盲人聽 */
```

| 方法 | 看得見 | 占位 | 可點擊 | 可動畫 |
|---|---|---|---|---|
| `display: none` | ❌ | ❌ | ❌ | ❌ |
| `visibility: hidden` | ❌ | ✅ | ❌ | ❌ |
| `opacity: 0` | ❌ | ✅ | ✅ | ✅ |
| `.sr-only` | ❌（視覺）/ ✅（螢幕閱讀器）| ❌ | ❌ | ❌ |

> 💡 **怎麼選**：(1) **要完全消失** → `display: none` (2) **要保留位置（如切換照片）** → `visibility: hidden` (3) **要做淡入淡出動畫** → `opacity: 0` + `transition` (4) **要對螢幕閱讀器隱藏不掉文字（如「跳到主內容」連結）** → `.sr-only`。

  
### 7. block 跟 inline 互轉

每個 HTML 元素都有預設 `display` 值、但你隨時可以改：

📄 **寫進程式檔**：`style.css`

```text
/* 把 <a> 變成塊狀（整塊可點）*/
nav a {
  display: block;
  padding: 12px 16px;
  /* → 整個 padding 區域都可點、不只文字 */
}

/* 把 <li> 變成同行（橫向選單）*/
nav ul li {
  display: inline-block;
}
/* 但現在更常用 flex */
```

  
### 8. 實戰：標準導覽列

📄 **寫進程式檔**：`style.css`

```text
/* 方法 A：inline-block（傳統）*/
.nav-a li {
  display: inline-block;
  margin-right: 16px;
}

/* 方法 B：flex（現代、推薦）*/
.nav-b ul {
  display: flex;
  gap: 16px;
  list-style: none;
  padding: 0;
}

/* 方法 B 完勝：gap 精準、垂直對齊免設、響應式好做 */
```

  
### 9. display: contents —— 進階用法

`display: contents` = **盒子消失、但子元素留下**。常用在需要「打破多餘的包裝層」時：

```text
情境：
  <ul>
    <li><a>項目 A</a></li>
    <li><a>項目 B</a></li>
  </ul>

問題：希望 <li> 不影響佈局、但 HTML 結構又要保留
解法：li { display: contents; }
  → <a> 直接成為 <ul> 的子元素（佈局上）
  → 但 HTML 仍是合規結構
```

> ⚠️ **`display: contents` 有無障礙陷阱**：某些瀏覽器會把該元素從 accessibility tree 移除——使用前確認你選的標籤沒有語意（純 div / span）。

  
### 5 大常見坑

```text
❌ 1. 給 <span> 設 width / height、無效
   → inline 不吃寬高
   → 修：改成 inline-block 或 block

❌ 2. <div> 設 width 後還是整行
   → block 預設撐滿父元素
   → 修：設 width 或加 display: inline-block

❌ 3. 用 display: none 又想做淡出動畫
   → display 不能 animate
   → 修：先 opacity: 0 + transition、結束才 display: none

❌ 4. inline 元素的 padding / margin 看起來怪
   → inline 的垂直 padding / margin 不會推開其他元素
   → 修：改 inline-block 或 block

❌ 5. 用 display: none 隱藏「跳到主內容」連結
   → 螢幕閱讀器也讀不到、無障礙扣分
   → 修：用 .sr-only 視覺隱藏、保留語意
```

  
### 🎯 重點回顧

- **`display` = 元素的人格**——一個屬性決定排版邏輯
- **block 獨佔一行**（div / p / h1）
- **inline 同一行、寬高無效**（span / a / strong）
- **inline-block 同行 + 可設寬高**（傳統 tag / 按鈕）
- **flex / grid 是現代排版主力**（下兩課詳教）
- **隱藏 4 法**：`display: none` / `visibility: hidden` / `opacity: 0` / `.sr-only`
