**CSS-in-JS（把 CSS 寫在 JavaScript 裡）** = React / Vue 時代的另一條路線——**樣式跟元件綁在一起**、scope 隔離、可動態切換。**5 個主流方案**：styled-components / Emotion / CSS Modules / Vanilla Extract / Panda CSS / StyleX。

> 💡 **這課的價值**：學會 CSS-in-JS = 進入 React 生態的關鍵。**styled-components 在 2024 後逐漸退場、新專案該選哪個**——這課給你 2026 選型答案。

  
### 1. 為什麼需要 CSS-in-JS

```text
傳統 CSS 在元件框架的痛苦：
  → 全域 class 容易衝突
  → 刪元件後不知道哪些 CSS 沒用了
  → 樣式跟邏輯分散在兩個檔案
  → 動態樣式麻煩（要組合 class）

CSS-in-JS 解法：
  → 樣式 scope 到元件
  → 元件刪 = CSS 也刪
  → 樣式跟邏輯在同一個檔案
  → 可以用 JS 變數、props 動態生樣式
```

  
### 2. 五大方案分類

```text
1. Runtime CSS-in-JS（執行時計算）：
   → styled-components
   → Emotion
   優：動態靈活
   缺：runtime 開銷、SSR 複雜
   2024 後逐漸式微（不支援 React Server Components）

2. Compile-time / Atomic（編譯時）：
   → Vanilla Extract
   → Panda CSS
   → StyleX
   優：零 runtime、效能好、SSR 完美
   缺：學習曲線
   2026 新主流

3. CSS Modules（中間派）：
   → 寫純 CSS、但 scope 到元件
   優：簡單、SSR 友善、零 runtime
   缺：動態樣式麻煩
   經典穩定選擇
```

  
### 3. styled-components / Emotion（runtime）

📄 **寫進程式檔**：`Button.jsx`

```text
import styled from "styled-components";

const Button = styled.button`
  background: ${props => props.primary ? "blue" : "gray"};
  color: white;
  padding: 10px 20px;
  border-radius: 4px;
  
  &:hover {
    opacity: 0.8;
  }
`;

function App() {
  return (
    <>
      <Button>普通</Button>
      <Button primary>主要</Button>
    </>
  );
}
```

**Emotion** 語法幾乎一樣、只是套件名不同。

> ⚠️ **2024 後 styled-components 退場**：(1) 不支援 React Server Components（Next.js App Router）(2) runtime 開銷大 (3) bundle 變大。**新專案不要選 styled-components**、維護老專案可以繼續用。

  
### 4. CSS Modules（穩定派）

**寫純 CSS、檔名加 `.module.css`、import 進 JS**：

📄 **寫進程式檔**：`Button.module.css`

```text
.button {
  background: gray;
  color: white;
  padding: 10px 20px;
}

.primary {
  background: blue;
}
```

📄 **寫進程式檔**：`Button.jsx`

```text
import styles from "./Button.module.css";

function Button({ primary }) {
  return (
    <button className={primary ? styles.primary : styles.button}>
      點我
    </button>
  );
}
```

**編譯後的 class 會自動加 hash**：

```text
原始：.button → 編譯後：.Button_button__a3b9f
→ 自動 scope、不會跟其他元件撞名
```

> 💡 **CSS Modules 為什麼穩定**：(1) 寫純 CSS、學習成本零 (2) 零 runtime、效能好 (3) SSR 完美 (4) 跟設計工具能對齊。**Next.js / Vite 內建支援**。

  
### 5. Vanilla Extract（零執行時 CSS-in-TS）

**用 TS 寫 CSS、編譯成靜態 CSS**：

📄 **寫進程式檔**：`button.css.ts`

```text
import { style } from "@vanilla-extract/css";

export const button = style({
  background: "blue",
  color: "white",
  padding: "10px 20px",
  
  ":hover": {
    opacity: 0.8
  }
});

export const buttonPrimary = style([button, {
  background: "crimson"
}]);
```

📄 **寫進程式檔**：`Button.tsx`

```text
import { button, buttonPrimary } from "./button.css";

export const Button = ({ primary }) => (
  <button className={primary ? buttonPrimary : button}>
    點我
  </button>
);
```

> 🚀 **Vanilla Extract 優點**：(1) **完整 TS 型別**（CSS 屬性都有 IDE 補全）(2) **零 runtime**（編譯成純 CSS）(3) **SSR 完美**(4) **支援 design token / theme**。**強型別專案首選**。

  
### 6. Panda CSS（2024+ 新方案）

**把 CSS-in-JS 的開發體驗 + Tailwind 的效能合在一起**：

📄 **寫進程式檔**：`Button.tsx`

```text
import { css } from "styled-system/css";

const buttonStyles = css({
  bg: 'blue.500',
  color: 'white',
  px: 4,
  py: 2,
  borderRadius: 'md',
  _hover: { bg: 'blue.600' }
});

const Button = () => (
  <button className={buttonStyles}>點我</button>
);
```

**Panda 的特性**：
- 寫法像 CSS-in-JS（物件語法）
- 編譯後變 Tailwind 那樣的 atomic CSS
- 強型別 + design token
- Park UI（Panda 配套元件庫）

  
### 7. StyleX（Meta 大型專案用）

**Meta（Facebook）開發、給超大型專案用**：

📄 **寫進程式檔**：`Button.tsx`

```text
import * as stylex from '@stylexjs/stylex';

const styles = stylex.create({
  base: {
    background: 'blue',
    color: 'white',
    padding: '10px 20px',
  },
  primary: {
    background: 'crimson',
  }
});

const Button = ({ primary }) => (
  <button {...stylex.props(styles.base, primary && styles.primary)}>
    點我
  </button>
);
```

> 💡 **StyleX 特色**：(1) 強型別 (2) atomic CSS（每個屬性獨立 class、總 CSS 極小）(3) 跨平台（Web + React Native）(4) build-time。**Meta 內部 Facebook / Instagram 都用**。

  
### 8. Vue 的 scoped style（內建）

**Vue 單檔元件（`.vue`）內建 scoped CSS**：

📄 **寫進程式檔**：`Button.vue`

```text
<template>
  <button class="btn">點我</button>
</template>

<style scoped>
.btn {
  background: blue;
  color: white;
  padding: 10px 20px;
}

.btn:hover {
  opacity: 0.8;
}
</style>
```

**`scoped` 自動加 attribute selector**——`.btn` 編譯後變 `.btn[data-v-abc123]`、不影響其他元件。

  
### 9. 2026 業界選擇對照表

| 場景 | 推薦 |
|---|---|
| **新 Next.js / RSC 專案** | **Tailwind v4**（最多人選）|
| 強型別 + 設計系統 | Vanilla Extract / Panda CSS |
| 傳統 React SPA | CSS Modules / styled-components（舊專案）|
| Vue 專案 | `<style scoped>` 內建 |
| Meta / 大型 React 專案 | StyleX |
| 設計工具整合（Figma → 程式碼）| Vanilla Extract / Panda |
| 快速原型 / 接案 | Tailwind |

  
### 10. styled-components 為什麼退場

```text
2024 後 styled-components 的問題：
  1. ❌ 不支援 React Server Components
     → Next.js App Router 不能用
     → 違反 React 未來方向
  
  2. ❌ Runtime 開銷大
     → 每次 render 都計算樣式
     → 比 Tailwind / Vanilla Extract 慢
  
  3. ❌ Bundle 增加
     → 加 styled-components 套件本身 ~10KB
  
  4. ❌ SSR 配置麻煩
     → 要 server stylesheet 處理
     → Vanilla Extract / Tailwind 自動好

→ 新專案不要選 styled-components
→ 維護老專案可以繼續用、但別新引入
```

> 🚀 **業界趨勢**：**styled-components → Vanilla Extract / Panda → 最終可能都被 Tailwind v4 + shadcn/ui 取代**。CSS-in-JS 整體市占在下降、build-time 工具在升。

  
### 11. 怎麼選（給你的答案）

```text
情境 1：新專案、Next.js、不熟前端
  → 直接上 Tailwind v4 + shadcn/ui
  → 業界主流、學習資源最多

情境 2：強型別需求、設計系統重
  → Vanilla Extract 或 Panda CSS
  → TS 友善、零 runtime

情境 3：維護老 React 專案（已用 styled-components）
  → 繼續用
  → 不要為了「換掉」浪費時間

情境 4：Vue 專案
  → <style scoped>
  → 不要為了趨勢換東西

情境 5：純 CSS 派、不想學新東西
  → CSS Modules
  → 學習成本 0、效能好
```

  
### 5 大常見坑

```text
❌ 1. styled-components 用在 Next.js App Router
   → 不支援 RSC、會壞掉
   → 修：改 Tailwind / Vanilla Extract / CSS Modules

❌ 2. 動態樣式用「字串拼接」
   → ${condition ? 'a' : 'b'} 滿天飛、難維護
   → 修：用 props + variants 系統

❌ 3. CSS-in-JS 把所有元件樣式塞同一檔
   → 變得跟全域 CSS 一樣難管
   → 修：每個元件獨立檔

❌ 4. 期望 CSS-in-JS 取代 Tailwind
   → 兩者哲學不同（樣式 vs utility）
   → 修：選一個 main scheme、不要混

❌ 5. Vanilla Extract 在 vite.config 沒設定
   → 不會編譯
   → 修：必裝 @vanilla-extract/vite-plugin
```

  
### 🎯 重點回顧

- **CSS-in-JS = 樣式跟元件綁在一起**——scope 隔離
- **styled-components / Emotion** —— 退場中、新專案別選
- **CSS Modules** —— 簡單穩定、Next.js 內建
- **Vanilla Extract** —— TS 友善、零 runtime
- **Panda CSS / StyleX** —— 2024+ 新方案
- **Vue 內建 scoped style**——簡單到難以拒絕
- **2026 主流 = Tailwind v4 + shadcn/ui**——CSS-in-JS 角色縮小
