CSS 效能優化 = 讓你的網站「**快得像 Apple**」。為什麼動畫卡頓？為什麼初始載入慢？這課把瀏覽器渲染管線拆開講、教你 **Critical CSS / content-visibility / will-change** 的精準用法。

> 💡 **這課的價值**：學完效能優化 = 你的網站 Lighthouse 衝到 95+。**用戶等網頁載入超過 3 秒、53% 會離開**——效能直接決定轉換率。

  
### 1. 瀏覽器渲染管線（5 階段）

瀏覽器把 HTML + CSS 變成畫面、要經過 5 步：

```text
1. Parse（解析）
   → HTML → DOM tree
   → CSS → CSSOM tree

2. Style（計算樣式）
   → 算每個元素「最終樣式」（合併所有規則）

3. Layout（排版）
   → 算每個元素的「位置和大小」
   → 又叫 reflow（重排）

4. Paint（繪製）
   → 把元素「畫到圖層」
   → 顏色、邊框、文字

5. Composite（合成）
   → 多個圖層「合成最終畫面」
   → GPU 幫忙
```

  
### 2. 改不同屬性、觸發不同階段（最重要）

| 屬性 | 觸發階段 | 成本 |
|---|---|---|
| `width / height / top / left / margin / padding` | Layout + Paint + Composite | **最貴** |
| `color / background / box-shadow` | Paint + Composite | 中 |
| **`transform / opacity / filter`** | **只有 Composite** | **最便宜** |

**鐵律**：**動畫只動 transform 跟 opacity**——這條牢牢記住。

  
### 3. 為什麼動畫卡頓

```text
情境：捲動時卡頓
  → 99% 是有元素在「捲動時」改了 layout 屬性
  → 觸發整頁重新計算位置
  → 60fps 變 20fps、肉眼可見卡頓

解法：
  1. 用 transform 取代 top / left
  2. 用 will-change 提示瀏覽器
  3. 用 content-visibility 跳過離畫面遠的元素
```

📄 **寫進程式檔**：`style.css`

```text
/* ❌ 卡頓 */
.bad-animation {
  transition: left 0.3s, width 0.3s;
}
.bad-animation:hover {
  left: 100px;
  width: 300px;
}

/* ✅ 流暢 */
.good-animation {
  transition: transform 0.3s;
}
.good-animation:hover {
  transform: translateX(100px) scale(1.5);
}
```

  
### 4. will-change（提示瀏覽器）

`will-change` 告訴瀏覽器「**這個元素即將動畫某屬性、請先準備 GPU 圖層**」：

📄 **寫進程式檔**：`style.css`

```text
.card {
  will-change: transform;
  /* → 瀏覽器先把 .card 推到獨立 GPU 圖層 */
  /* → 之後動 transform 不用重新合成 */
}
```

> ⚠️ **不要無腦加 `will-change`**：(1) 會耗記憶體（每個元素獨立圖層）(2) 用太多反而拖慢 (3) 只在「**真的會頻繁動畫**」的元素加。**動完後最好用 JS 移除**。

  
### 5. contain（隔離元素）

`contain` 告訴瀏覽器「**這個元素變化不會影響外部**」、可以隔離計算：

📄 **寫進程式檔**：`style.css`

```text
.card {
  contain: layout style paint;
  /* 
     layout：layout 變化不影響外部
     style：樣式不影響外部
     paint：paint 不影響外部
  */
}
```

> 🚀 **`contain` 在「**列表很多元素**」時超有效**——上萬筆資料的表格、無限滾動列表。

  
### 6. content-visibility（神級優化）

`content-visibility: auto` = **離畫面遠的元素瀏覽器跳過渲染**——上萬筆列表的救星：

📄 **寫進程式檔**：`style.css`

```text
.long-list-item {
  content-visibility: auto;
  contain-intrinsic-size: 200px;
  /* 
     content-visibility: auto
       → 不在畫面時跳過渲染
     contain-intrinsic-size: 200px
       → 假裝高度 200px（捲軸不會跳）
  */
}
```

> 🚀 **`content-visibility` 效能提升範例**：1000 筆列表初始載入時間從 2 秒降到 0.3 秒（實測 Lighthouse）。**Chrome 90+ 支援、Safari 18 跟進**。

  
### 7. 降低 CSS 大小

```text
🔧 1. PurgeCSS（移除沒用到的 class）
   → Tailwind v4 預設內建
   → 從 3MB → 50KB 是常態

🔧 2. Critical CSS（首屏關鍵 CSS 內聯）
   → 把首屏需要的 CSS 直接寫在 <style> 裡
   → 不用等外部 CSS 下載完才畫
   → Next.js / Astro 自動處理

🔧 3. Code splitting（路由切分 CSS）
   → 每個路由載自己的 CSS
   → 首頁不用載商品頁的 CSS

🔧 4. 壓縮（minify）
   → CSS 壓縮 30-50%
   → 移除空格、註解
   → 自動化工具：cssnano

🔧 5. 子集化字型
   → 中文字型完整版 10MB+
   → 只取你網站用到的字 → 200KB
   → 工具：font-subset / fontmin
```

  
### 8. Critical CSS（首屏關鍵）

**把首屏需要的 CSS 直接內聯到 HTML**、瀏覽器不用等下載 CSS 才開始畫：

📄 **寫進程式檔**：`index.html`

```text
<head>
  <!-- 首屏關鍵 CSS（< 14KB、塞在第一個 HTTP packet 內）-->
  <style>
    body { font-family: -apple-system, sans-serif; margin: 0; }
    header { background: #fff; padding: 16px; }
    .hero { height: 80vh; background: #f0f0f0; }
    /* ... 只放首屏看得到的樣式 ... */
  </style>
  
  <!-- 其他 CSS 非阻塞載入 -->
  <link rel="preload" href="full.css" as="style" onload="this.rel='stylesheet'">
  <noscript><link rel="stylesheet" href="full.css"></noscript>
</head>
```

> 🚀 **Critical CSS 提升 LCP（最大內容繪製）50%+**——Google 排名因子之一。**Next.js / Astro / Nuxt 都自動處理、不用自己寫**。

  
### 9. 圖片優化（搭 CSS）

**HTML 端**：

📄 **寫進程式檔**：`index.html`

```text
<picture>
  <!-- 優先載 AVIF（比 WebP 小 30%）-->
  <source srcset="hero.avif" type="image/avif">
  <source srcset="hero.webp" type="image/webp">
  <img
    src="hero.jpg"
    alt="..."
    loading="lazy"
    width="800"
    height="400"
  >
</picture>
```

**CSS 端**：

📄 **寫進程式檔**：`style.css`

```text
img {
  /* 防 CLS（layout shift）*/
  aspect-ratio: 16 / 9;
  width: 100%;
  height: auto;
  
  /* 圖片 GPU 加速 */
  transform: translateZ(0);
}
```

> 💡 **CLS（Cumulative Layout Shift、累積版面位移）= Google 三大 Core Web Vitals 之一**——圖片載入時撐開版面、文字被推走 = CLS 高分扣分。**寫 `width` + `height` 或 `aspect-ratio` 解決**。

  
### 10. 字型載入優化

```text
🔧 1. font-display: swap
   → 載入時先顯示備用字、避免 FOIT
   
🔧 2. preload 關鍵字型
   <link rel="preload" href="inter.woff2" as="font" type="font/woff2" crossorigin>
   
🔧 3. 子集化中文字型
   → 完整版 7000+ 字 = 5MB
   → 只取常用 2000 字 = 1MB
   → 進一步只取你網站用到的字 = 50KB
   
🔧 4. 用 Variable Font
   → 一檔包所有粗細
   → 省下 7 個 HTTP request
```

  
### 11. CSS Selector 效能（其實不太重要）

```text
理論上：
  → id（#a）比 class（.a）快
  → 簡單 selector 比複雜 selector 快

實際上：
  → 現代瀏覽器都已優化
  → selector 效能影響 < 1%
  → 真正瓶頸是「layout / paint」、不是 selector

→ 99% 場景不用優化 selector
→ 把心力放在動畫優化、Critical CSS、圖片
```

### 13. 實戰：完整效能 CSS

📄 **寫進程式檔**：`style.css`

```text
/* 1. base：用系統字、不下載 */
body {
  font-family: -apple-system, BlinkMacSystemFont, sans-serif;
  font-display: swap;
}

/* 2. 圖片防 CLS */
img {
  aspect-ratio: 16 / 9;
  width: 100%;
  height: auto;
  content-visibility: auto;
  contain-intrinsic-size: auto 200px;
}

/* 3. 動畫只動 transform / opacity */
.card {
  transition: transform 0.2s, opacity 0.2s;
}

.card:hover {
  transform: translateY(-2px);
}

/* 4. 列表項目效能優化 */
.list-item {
  content-visibility: auto;
  contain-intrinsic-size: 80px;
  contain: layout style;
}

/* 5. 動畫元素事先標記 */
.modal {
  will-change: transform, opacity;
}

/* 6. 無障礙：減少動態 */
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

  
### 5 大常見坑

```text
❌ 1. 動畫用 width / left / margin
   → 觸發 layout、卡頓
   → 修：改 transform

❌ 2. 無腦加 will-change: transform 到所有元素
   → 耗記憶體、反而拖慢
   → 修：只在「真的會動畫」的元素加

❌ 3. 圖片沒寫 width / height
   → CLS 爆表、Lighthouse 扣到爆
   → 修：永遠寫 width + height 或 aspect-ratio

❌ 4. 字型沒 font-display: swap
   → 載入時整片空白（FOIT）
   → 修：永遠加 swap

❌ 5. CSS 沒 PurgeCSS、3MB 全送
   → 修：用 Tailwind / 設定 PurgeCSS
```

  
### 🎯 重點回顧

- **動畫只動 transform / opacity**——效能鐵律
- **`will-change`** —— 提示、別濫用
- **`contain` / `content-visibility`**——隔離 / 跳過渲染
- **Critical CSS 內聯**——首屏關鍵
- **圖片寫 width + height**——防 CLS
- **字型 `font-display: swap` + preload**
- **CSS selector 效能 99% 不用優化**——把心力放動畫跟圖片
