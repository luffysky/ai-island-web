**CSS Variables（CSS 變數、又叫 Custom Properties 自訂屬性）** 讓樣式變「**動態**」——一個變數改一次、整個網站跟著變。**現代 CSS 的核心**、暗黑模式 / 主題切換 / 跟 JS 互動全靠這個。

> 💡 **這課的價值**：學會 CSS 變數 = 改 1 個變數、全網站主題變更。**暗黑模式 5 行解決**、再也不用改 100 個地方。Tailwind / shadcn / Vercel 都基於變數系統。

  
### 1. 為什麼需要 CSS 變數

```text
沒變數的痛苦：
  品牌色 #3b82f6 寫死在 100 處
  → 客戶要改色 → 找 100 處改 → 漏 3 處 → debug 半天

有變數：
  --primary: #3b82f6;
  → 改 1 個變數、100 處跟著變
  → 5 秒搞定
```

  
### 2. 定義跟使用

📄 **寫進程式檔**：`style.css`

```text
/* 定義（通常在 :root、代表整個網頁根部）*/
:root {
  --primary: oklch(0.65 0.2 240);
  --gap: 16px;
  --radius: 8px;
  --shadow-sm: 0 1px 2px rgba(0, 0, 0, 0.05);
  --font-sans: 'Inter', sans-serif;
}

/* 使用（用 var() 取出）*/
.btn {
  background: var(--primary);
  padding: var(--gap);
  border-radius: var(--radius);
  box-shadow: var(--shadow-sm);
  font-family: var(--font-sans);
}

/* 帶後備值（變數沒定義時用 black）*/
.box {
  color: var(--text-color, black);
}
```

> 💡 **變數命名規則**：(1) **必須以 `--` 開頭** (2) 大小寫敏感（`--Primary` ≠ `--primary`）(3) 業界慣例：全小寫 + 連字號（`--bg-color` 而非 `--bgColor`）。

  
### 3. 巢狀作用域（區域變數）

CSS 變數有「**作用域**」——可以在某區塊內覆寫：

📄 **寫進程式檔**：`style.css`

```text
:root {
  --bg: white;
  --fg: black;
}

.dark-section {
  --bg: black;
  --fg: white;
}

/* 區域內的所有 var(--bg) 都自動變黑 */
.dark-section .card {
  background: var(--bg);  /* → black */
  color: var(--fg);        /* → white */
}
```

> 💡 **跟 Sass 變數差別**：Sass `$primary` 是**編譯時固定**、CSS `--primary` 是**runtime 動態**——可以巢狀覆寫、可以用 JS 改、可以動畫過渡。**威力差很多**。

  
### 4. 暗黑模式（一個變數搞定）

**現代暗黑模式 = 切換變數**、不是寫兩套 CSS：

📄 **寫進程式檔**：`style.css`

```text
/* 預設亮色 */
:root {
  --bg: white;
  --text: #222;
  --card-bg: #f5f5f5;
  --border: #e0e0e0;
}

/* 暗色模式（HTML 上加 data-theme="dark"）*/
[data-theme="dark"] {
  --bg: #0a0a0a;
  --text: #eee;
  --card-bg: #1a1a1a;
  --border: #333;
}

/* 一般元素只用變數、不關心主題 */
body {
  background: var(--bg);
  color: var(--text);
  transition: background 0.3s, color 0.3s;
}

.card {
  background: var(--card-bg);
  border: 1px solid var(--border);
}
```

**JS 切換主題**：

📄 **寫進程式檔**：`script.js`

```text
// 切到暗色
document.documentElement.dataset.theme = "dark";

// 切回亮色
document.documentElement.dataset.theme = "light";

// 跟著系統偏好
const prefersDark = window.matchMedia('(prefers-color-scheme: dark)').matches;
document.documentElement.dataset.theme = prefersDark ? "dark" : "light";
```

> 🚀 **這就是 Vercel / Linear / Tailwind UI 的暗黑模式做法**——一個 data-attribute 切換、所有變數同步換、CSS 一行不用改。

  
### 5. JS 動態改變數

CSS 變數可以**即時被 JS 改**、立刻反映畫面：

📄 **寫進程式檔**：`script.js`

```text
// 改全域變數
document.documentElement.style.setProperty('--primary', 'red');

// 改某元素的變數
document.querySelector('.card').style.setProperty('--card-bg', 'blue');

// 讀取變數
const primary = getComputedStyle(document.documentElement)
  .getPropertyValue('--primary');
```

**實戰：拖滑桿改主題色**：

📄 **寫進程式檔**：`script.js`

```text
const slider = document.getElementById('hue-slider');
slider.addEventListener('input', (e) => {
  const hue = e.target.value;  // 0-360
  document.documentElement.style.setProperty(
    '--primary',
    `oklch(0.65 0.2 ${hue})`
  );
});
```

  
### 6. @property（屬性註冊）—— 進階

`var(--xxx)` 預設是「**字串**」、無法 animate。**`@property`** 讓變數有「**型別**」、可以動畫：

📄 **寫進程式檔**：`style.css`

```text
@property --gradient-angle {
  syntax: "<angle>";
  initial-value: 0deg;
  inherits: false;
}

.card {
  background: linear-gradient(var(--gradient-angle), red, blue);
  animation: spin 2s linear infinite;
}

@keyframes spin {
  to { --gradient-angle: 360deg; }
}
/* → 過去不能 animate 的「漸層角度」、現在可以了 */
```

**`@property` 的 3 個欄位**：
- `syntax` —— 型別（`<angle>` / `<color>` / `<number>` / `<length>` 等）
- `initial-value` —— 預設值
- `inherits` —— 是否繼承

> 🚀 **這解鎖了很多以前 CSS 做不到的動畫**——漸層旋轉、顏色平滑過渡、自訂屬性動畫。

  
### 7. 跟 calc() 組合

CSS 變數可以塞進 `calc()` 做運算：

📄 **寫進程式檔**：`style.css`

```text
:root {
  --base: 16px;
}

.card {
  padding: calc(var(--base) * 1.5);   /* = 24px */
  margin: calc(var(--base) * 2);       /* = 32px */
  font-size: calc(var(--base) * 1.25); /* = 20px */
}
```

  
### 8. 實戰：完整變數系統

📄 **寫進程式檔**：`style.css`

```text
:root {
  /* 顏色 */
  --color-primary: oklch(0.65 0.2 240);
  --color-success: oklch(0.7 0.18 142);
  --color-danger: oklch(0.6 0.22 27);
  --color-text: oklch(0.2 0 0);
  --color-bg: oklch(0.98 0 0);
  
  /* 間距（8 點網格）*/
  --space-1: 4px;
  --space-2: 8px;
  --space-4: 16px;
  --space-6: 24px;
  --space-8: 32px;
  
  /* 圓角 */
  --radius-sm: 4px;
  --radius-md: 8px;
  --radius-lg: 16px;
  --radius-full: 999px;
  
  /* 陰影 */
  --shadow-sm: 0 1px 2px rgba(0,0,0,.04);
  --shadow-md: 0 4px 12px rgba(0,0,0,.08);
  --shadow-lg: 0 16px 32px rgba(0,0,0,.12);
  
  /* 字級 */
  --text-sm: 14px;
  --text-base: 16px;
  --text-lg: 18px;
  --text-xl: 24px;
  --text-2xl: 32px;
  
  /* 動畫 */
  --transition-fast: 150ms;
  --transition-base: 300ms;
}

[data-theme="dark"] {
  --color-text: oklch(0.95 0 0);
  --color-bg: oklch(0.15 0 0);
}

.btn {
  background: var(--color-primary);
  padding: var(--space-2) var(--space-4);
  border-radius: var(--radius-md);
  box-shadow: var(--shadow-sm);
  transition: all var(--transition-fast);
}
```

> 💡 **這就是設計系統的底層**——Tailwind 把這套工業化、shadcn 把它包成元件。**自己會寫這個 = 能造自己的設計系統**。

  
### 9. CSS 變數 vs Sass 變數

| | CSS Variables | Sass `$var` |
|---|---|---|
| **執行時機** | runtime（瀏覽器讀） | 編譯時（build 時固定）|
| **動態改變** | ✅ 可以、JS 也能改 | ❌ 編譯後就死了 |
| **巢狀作用域** | ✅ 可以區域覆寫 | ❌ 全域固定 |
| **動畫** | ✅ 可以（搭 @property）| ❌ 不行 |
| **效能** | 略慢（runtime 計算）| 較快（編譯期算好）|

> 💡 **2026 業界選擇**：**新專案直接用 CSS Variables**——Sass 變數已過時。**只有大型 Bootstrap 主題客製、舊專案維護**還在用 Sass 變數。

  
### 5 大常見坑

```text
❌ 1. 變數名忘記寫 -- 前綴
   → 寫 `primary` 變成普通屬性、不是變數
   → 修：變數一律 --xxx

❌ 2. 用 var(--xxx) 結果是空白
   → 變數沒定義、或 typo
   → 修：var(--xxx, fallback) 帶後備值

❌ 3. 在 Sass 內混用 CSS variable
   → @if $primary { ... } 跟 var(--primary) 用法不同
   → 修：選一個用、別混

❌ 4. 期望 Sass `$primary: var(--xxx)` 編譯後變值
   → Sass 不會解析 var()、會原樣輸出
   → 修：用 CSS variable 就純 CSS、別包 Sass

❌ 5. 變數寫太亂、命名不一致
   → --primary / --primary-color / --color-primary 混用
   → 修：訂規則、團隊統一（建議 --{類別}-{名稱} 例如 --color-primary）
```

  
### 🎯 重點回顧

- **CSS Variables runtime 動態**——能用 JS 改、能動畫
- **`--xxx` 定義、`var(--xxx)` 使用**——背起來
- **可區域覆寫**——`.dark-section { --bg: black; }`
- **暗黑模式 5 行解決**——切 `data-theme` 即可
- **`@property`** —— 解鎖變數動畫（漸層角度、顏色過渡）
- **設計系統底層**——Tailwind / shadcn 全靠這個
