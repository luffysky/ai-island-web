**Tailwind CSS** = **utility-first（功能優先）** CSS 框架——**不寫 CSS、只寫 class**。2020 後改變整個前端世界、開發速度提升 10 倍。爭議大、但已成 2026 新專案標配。

> 💡 **這課的價值**：學會 Tailwind = 你進入 90% 現代前端團隊的門票。**Vercel / Linear / shadcn / Next.js 範例專案全用它**。新人面試「會 Tailwind 嗎」是必問題。

  
### 1. 為什麼出現 Tailwind

```text
傳統 CSS 痛苦：
  → 命名痛苦（這個區塊叫什麼？.card-inner-wrapper-2？）
  → CSS 越寫越大、沒人敢刪
  → 改一個 class 不知道哪裡受影響
  → 設計師 / 工程師 token 不同步

Tailwind 解法：
  → 不命名、直接寫 utility class
  → 沒寫 CSS = 沒包袱可刪
  → class 就在 HTML、改一處不影響別處
  → 設計 token 統一從 config 來
```

> 💡 **utility-first（功能優先）的核心思想**：「**把 CSS 拆成最小單位、組合使用**」——`p-4` 是 padding 4、`text-lg` 是大字、`bg-blue-500` 是藍底。

  
### 2. 安裝（Tailwind v4、2024+）

🖥️ **在終端機輸入**

```text
pnpm add -D tailwindcss @tailwindcss/postcss
```

Vite 專案用專用外掛：

```text
pnpm add -D @tailwindcss/vite
```

📄 **寫進程式檔**：`src/main.css`

```text
@import "tailwindcss";
```

📄 **寫進程式檔**：`vite.config.js`

```text
import { defineConfig } from 'vite';
import tailwindcss from '@tailwindcss/vite';

export default defineConfig({
  plugins: [tailwindcss()],
});
```

> 💡 **v4 比 v3 簡化超多**——v3 要寫 `tailwind.config.js` 配置、v4 用 CSS 設定（`@theme`）、不用 JS 配置檔。

  
### 3. 第一個 Tailwind 範例

📄 **寫進程式檔**：`index.html`

```text
<!-- 傳統 CSS -->
<div class="card">
  <h2 class="card-title">標題</h2>
</div>

<style>
  .card {
    background: white;
    padding: 1rem;
    border-radius: 8px;
    box-shadow: 0 1px 3px rgba(0,0,0,.1);
  }
  .card-title {
    font-size: 1.25rem;
    font-weight: 600;
  }
</style>

<!-- Tailwind 版本 -->
<div class="bg-white p-4 rounded-lg shadow">
  <h2 class="text-xl font-semibold">標題</h2>
</div>
```

> 💡 **Tailwind 看起來像 inline CSS、但完全不同**——它是 utility class 系統、有設計 token 約束、可以開發時補全、可以壓縮成最小 CSS。

  
### 4. 常用 class 對照表

| Tailwind | 等價 CSS |
|---|---|
| `p-4` | `padding: 1rem` |
| `px-4 py-2` | `padding: 0.5rem 1rem` |
| `m-auto` | `margin: auto` |
| `flex items-center justify-center` | flex + 置中 |
| `grid grid-cols-3 gap-4` | 3 欄 grid |
| `text-lg font-bold text-gray-700` | 字級 + 粗體 + 顏色 |
| `bg-blue-500 hover:bg-blue-600` | 背景 + hover |
| `rounded-lg shadow-md` | 圓角 + 陰影 |
| `md:flex lg:grid` | 響應式 |
| `dark:bg-gray-900` | 暗黑模式 |

> 💡 **數字系統**：(1) **間距** 1 = 4px、2 = 8px、4 = 16px、8 = 32px（4 倍）(2) **顏色** 50（最淺）→ 500（中）→ 900（最深）。**背熟這套規格 = 寫起來像在打字**。

  
### 5. 修飾符（State Modifiers）

**用 `:` 前綴**加狀態：

📄 **寫進程式檔**：`index.html`

```text
<!-- hover：滑鼠懸停 -->
<button class="bg-blue-500 hover:bg-blue-600">

<!-- focus：取得焦點 -->
<input class="border focus:border-blue-500 focus:ring-2">

<!-- active：按下 -->
<button class="active:scale-95">

<!-- disabled：停用 -->
<button class="disabled:opacity-50 disabled:cursor-not-allowed">

<!-- 響應式（手機優先）-->
<div class="w-full md:w-1/2 lg:w-1/3">
<!--      手機全寬  平板 50%  桌機 33% -->

<!-- 暗色模式 -->
<div class="bg-white dark:bg-gray-900 text-gray-900 dark:text-white">

<!-- 組合：手機暗色 hover -->
<button class="hover:bg-blue-600 dark:hover:bg-blue-400">
```

  
### 6. v4 新特性：@theme（用 CSS 配置）

過去 v3 寫 `tailwind.config.js`、v4 直接在 CSS 寫：

📄 **寫進程式檔**：`main.css`

```text
@import "tailwindcss";

@theme {
  --color-brand: oklch(0.7 0.2 240);
  --color-brand-dark: oklch(0.5 0.2 240);
  --font-display: "Inter", sans-serif;
  --radius-pill: 999px;
}

/* 自動產生：
   bg-brand / text-brand
   font-display
   rounded-pill
   等所有衍生 class
*/
```

> 🚀 **`@theme` 是 v4 革命**——配置寫在 CSS、跟一般 CSS 變數一致、IDE 自動補全、構建超快。**新專案直接上 v4**。

  
### 7. @apply（複用樣式）

**抽出重複 utility 成 class**：

📄 **寫進程式檔**：`main.css`

```text
.btn-primary {
  @apply px-4 py-2 bg-blue-500 text-white rounded hover:bg-blue-600 transition;
}
```

📄 **寫進程式檔**：`index.html`

```text
<button class="btn-primary">送出</button>
```

> ⚠️ **`@apply` 慎用**——濫用 = 回到傳統 CSS 思維、失去 Tailwind 意義。**只在「真的重複 5+ 次」才抽**、且優先用 React/Vue 元件抽。

  
### 8. 動態值（Arbitrary Values）

**規格外的值用方括號**：

📄 **寫進程式檔**：`index.html`

```text
<!-- 自訂寬度 -->
<div class="w-[273px]">

<!-- 自訂顏色 -->
<div class="bg-[#1da1f2]">

<!-- 自訂位置 -->
<div class="top-[12.5%]">

<!-- 自訂 grid -->
<div class="grid-cols-[200px_1fr_100px]">
<!-- → 等同 grid-template-columns: 200px 1fr 100px -->
```

  
### 9. shadcn/ui：Tailwind 元件系統

**`shadcn/ui` 不是套件、是「複製貼上的元件」**——CLI 把 Radix UI + Tailwind 元件複製到你專案、你可以隨意改。**2024-2026 最紅的 UI 系統**。

🖥️ **在終端機輸入**

```text
pnpx shadcn@latest init
pnpx shadcn@latest add button
pnpx shadcn@latest add dialog
pnpx shadcn@latest add form
```

📄 **寫進程式檔**：你的 React 元件

```text
import { Button } from "@/components/ui/button";

<Button variant="default">點我</Button>
<Button variant="destructive">刪除</Button>
```

> 🚀 **shadcn/ui 為什麼革命**：(1) 元件「**屬於你**」（不像 npm 套件被綁定版本）(2) 可隨意改 (3) 沒鎖 framework（React / Vue / Svelte 都有版本）。**Vercel / Linear / Next.js 範例都在用**。

### 11. Tailwind 的爭議

```text
❌ 反方論點：
   1. HTML 看起來髒（一行 30+ class）
   2. 命名語意失去（看不出 .card 是卡片）
   3. 學習成本（要背 utility 名稱）

✅ 正方論點：
   1. 不用命名（最痛苦的事消失了）
   2. CSS 不會無限膨脹（沒寫 = 沒包袱）
   3. IDE 自動補全 + JIT 編譯
   4. 設計系統內建（不用自己建 token）
   5. 速度確實快
```

**解法**：
- 用 IDE 折行（VS Code → settings → Editor: Word Wrap）
- 元件化（React / Vue 把重複的包成元件）
- 善用 `@apply` 抽常用組合

> 🚀 **2026 業界共識**：Tailwind 是新專案首選、配 shadcn/ui 用更強。**不要對抗它**——違反設計哲學寫一堆傳統 CSS 就失去 Tailwind 意義。

  
### 5 大常見坑

```text
❌ 1. 期望 Tailwind 內建所有元件
   → Tailwind 只是 utility、不含元件
   → 修：搭配 shadcn/ui 或 Headless UI

❌ 2. 一行 class 寫到 50+ 個
   → 維護地獄
   → 修：(1) 元件化 (2) @apply 抽常用 (3) IDE 折行

❌ 3. 用 inline CSS 跟 Tailwind 混
   → 失去 design token 一致性
   → 修：純 Tailwind、規格外用 arbitrary [...]

❌ 4. 不會用 IntelliSense
   → 一個個記 class 名超痛苦
   → 修：VS Code 裝「Tailwind CSS IntelliSense」擴充

❌ 5. v3 的習慣寫 v4
   → tailwind.config.js 過時、改用 @theme
   → 修：v4 配置寫 CSS、不寫 JS
```

  
### 🎯 重點回顧

- **utility-first 哲學**——拆成最小單位、組合使用
- **數字系統**：間距 1=4px、顏色 500=中、字級 lg=18px
- **修飾符**：`hover:` `focus:` `dark:` `md:`（響應式）
- **v4 用 `@theme` 配置**——不再需要 tailwind.config.js
- **`@apply` 抽常用樣式**——但別濫用
- **shadcn/ui 是元件首選**——複製貼上、可隨意改
- **VS Code 裝 Tailwind CSS IntelliSense** —— 必裝
