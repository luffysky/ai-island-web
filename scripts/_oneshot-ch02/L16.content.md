**Sass（Syntactically Awesome Style Sheets）/ SCSS（Sassy CSS）** = CSS 的「長大版」——補完純 CSS 缺的功能（變數、巢狀、mixin、函式、模組）。雖然 2026 純 CSS 已經很強、但**Bootstrap 主題、大型 SaaS 後台、設計系統**還是 Sass 主場。

> 💡 **這課的價值**：學會 Sass = 看懂 90% 開源專案的樣式檔（Bootstrap、Material UI 舊版、無數企業專案）。**雖然新專案逐漸減用、但維護舊專案必備**。

  
### 1. Sass vs SCSS vs CSS

```text
Sass 有兩種語法：

1. Sass（縮排語法、舊）
   .btn
     color: blue
     padding: 10px
   → 沒有大括號、沒有分號、靠縮排

2. SCSS（兼容 CSS 語法、新）
   .btn {
     color: blue;
     padding: 10px;
   }
   → 完全像 CSS、但多功能

業界 99% 用 SCSS、Sass 縮排語法基本消失。
```

> 💡 **以下範例都用 SCSS** —— 副檔名 `.scss`。

  
### 2. 安裝 Sass

🖥️ **在終端機輸入**

```text
pnpm add -D sass
```

或用 Vite、Next.js、Webpack 等——它們都內建 Sass 支援、安裝 `sass` 套件即可。

  
### 3. 變數（$xxx）

📄 **寫進程式檔**：`style.scss`

```text
// Sass 變數用 $ 開頭
$primary: #0070f3;
$radius: 8px;
$breakpoint-md: 768px;

.btn {
  background: $primary;
  border-radius: $radius;
}

@media (min-width: $breakpoint-md) {
  .btn { padding: 16px 32px; }
}
```

> 💡 **Sass `$var` vs CSS `--var` 的差別**（L13 講過）：(1) Sass 編譯時固定 (2) CSS 變數 runtime 動態。**Sass 變數無法被 JS 改、不能 animate**。

  
### 4. 巢狀（Nesting）—— Sass 最大魅力

把相關規則寫在一起、不用一直重複 selector：

📄 **寫進程式檔**：`style.scss`

```text
.card {
  padding: 20px;
  border: 1px solid #eee;
  
  h2 {
    color: #333;
    margin-bottom: 10px;
  }
  
  // & 代表「外層 selector」
  &:hover {
    transform: scale(1.02);
  }
  
  &.primary {       // 編譯成 .card.primary
    background: blue;
  }
  
  // 後代組合
  .btn + .btn {     // 編譯成 .card .btn + .btn
    margin-left: 8px;
  }
}
```

**編譯後（純 CSS）**：

```text
.card { padding: 20px; border: 1px solid #eee; }
.card h2 { color: #333; margin-bottom: 10px; }
.card:hover { transform: scale(1.02); }
.card.primary { background: blue; }
.card .btn + .btn { margin-left: 8px; }
```

> 💡 **CSS 也有原生巢狀（2023+）**——`.card { h2 { ... } }` 直接寫純 CSS 也行。**但兼容性 Sass 巢狀已被廣泛接受**。

  
### 5. Mixin（可重用樣式塊）

**mixin（混入）** = 把一段樣式「**模板化**」、需要時 include 進來：

📄 **寫進程式檔**：`style.scss`

```text
// 定義
@mixin flex-center {
  display: flex;
  justify-content: center;
  align-items: center;
}

@mixin button($bg, $color: white) {
  // $color 有預設值 white
  background: $bg;
  color: $color;
  padding: 10px 20px;
  border-radius: 4px;
  
  &:hover {
    background: darken($bg, 10%);  // Sass 內建函式：變暗 10%
  }
}

// 使用
.center { @include flex-center; }
.btn-primary { @include button(blue); }
.btn-danger  { @include button(red, white); }
```

> 💡 **Mixin 是 Sass 最強功能**——把一組常用樣式模板化、傳參數、不重複寫 30 行。**設計系統的元件樣式都靠這個**。

  
### 6. Function（運算邏輯）

```text
@function rem($px) {
  @return ($px / 16) * 1rem;
}

h1 { font-size: rem(32); }   // 編譯成 2rem
h2 { font-size: rem(24); }   // 編譯成 1.5rem
```

  
### 7. @extend（繼承）

把一個 selector 的所有樣式「**繼承**」過來：

📄 **寫進程式檔**：`style.scss`

```text
.btn-base {
  padding: 10px 20px;
  border-radius: 4px;
  cursor: pointer;
  font-weight: 500;
}

.btn-primary {
  @extend .btn-base;
  background: blue;
  color: white;
}

.btn-danger {
  @extend .btn-base;
  background: red;
  color: white;
}
```

> ⚠️ **`@extend` 慎用**——會把 selector 合併、HTML 變大、可能產生意想不到的特異性問題。**業界比較推薦 mixin、不推薦 extend**。

  
### 8. 流程控制（@if / @each / @for）

**自動產生大量規則的神技**：

📄 **寫進程式檔**：`style.scss`

```text
// @each：跑每一項
$colors: (
  primary: blue,
  success: green,
  danger: red,
  warning: orange
);

@each $name, $color in $colors {
  .btn-#{$name} {
    background: $color;
  }
}
// → 自動產生 .btn-primary / .btn-success / .btn-danger / .btn-warning

// @for：跑迴圈
@for $i from 1 through 12 {
  .col-#{$i} {
    width: $i * 8.333%;
  }
}
// → .col-1 { width: 8.333% } ... .col-12 { width: 100% }
// （這就是 Bootstrap grid 系統的做法！）

// @if 條件
@mixin theme($mode) {
  @if $mode == dark {
    background: #000; color: #fff;
  } @else {
    background: #fff; color: #000;
  }
}
```

> 🚀 **`@each` 自動產生 utility class**——Bootstrap 的 `bg-primary` / `text-danger` 都這樣寫的。**省下幾百行手寫**。

  
### 9. @use 跟 @forward（2020+ 模組系統）

**把 SCSS 拆成多個檔案、互相 import**：

📄 **寫進程式檔**：`_variables.scss`

```text
// 底線開頭 = partial 檔（不單獨編譯）
$primary: blue;
$radius: 8px;
```

📄 **寫進程式檔**：`_buttons.scss`

```text
@use "variables" as v;
// 引入 _variables.scss、用 v 為命名空間

.btn {
  background: v.$primary;
  border-radius: v.$radius;
}
```

📄 **寫進程式檔**：`main.scss`

```text
@use "buttons";
@use "cards";
@use "navbar";
```

> 💡 **`@use` 取代舊的 `@import`**——`@import` 全域污染、`@use` 有命名空間、更安全。**新專案永遠用 `@use`**。

  
### 10. Sass 內建函式

Sass 提供大量內建函式：

```text
// 顏色處理
darken($color, 10%)       → 變暗 10%
lighten($color, 20%)       → 變亮 20%
mix($c1, $c2, 50%)         → 混合兩色
opacify($color, 0.2)       → 增加不透明度

// 數學
round($value)              → 四捨五入
ceil($value) / floor()      → 進位 / 捨去
percentage(0.5)            → 50%
abs(-5)                    → 5

// 列表
length($list)              → 長度
nth($list, 1)              → 第幾個
```

  
### 11. 實戰：完整 Sass 主題

📄 **寫進程式檔**：`_tokens.scss`

```text
$colors: (
  primary: #0070f3,
  success: #10b981,
  danger:  #ef4444,
);

$spacings: (1: 4px, 2: 8px, 3: 12px, 4: 16px);

$breakpoints: (
  sm: 640px,
  md: 768px,
  lg: 1024px,
);
```

📄 **寫進程式檔**：`_mixins.scss`

```text
@use "tokens" as t;

@mixin respond($name) {
  @media (min-width: map-get(t.$breakpoints, $name)) {
    @content;
  }
}

@mixin button-variant($color) {
  background: $color;
  color: white;
  &:hover { background: darken($color, 8%); }
}
```

📄 **寫進程式檔**：`main.scss`

```text
@use "tokens" as t;
@use "mixins" as m;

@each $name, $color in t.$colors {
  .btn-#{$name} {
    @include m.button-variant($color);
  }
}

.container {
  padding: map-get(t.$spacings, 2);
  
  @include m.respond(md) {
    padding: map-get(t.$spacings, 4);
  }
}
```

  
### 12. Sass 在 2026 還重要嗎

```text
✅ 仍主流的場景：
   → Bootstrap 客製主題（必用）
   → 大型企業 SaaS 後台
   → 設計系統 token 生成
   → CSS module 進階用法

❌ 已被取代的場景：
   → 變數 → 改用 CSS variables（動態）
   → 巢狀 → CSS 原生巢狀（2023+）
   → utility 產生 → Tailwind 一鍵搞定

→ 2026 新專案常選 Tailwind v4 而非 Sass
→ 但維護舊專案、Bootstrap 主題必懂
```

  
### 5 大常見坑

```text
❌ 1. 用 @import（舊語法）
   → 全域污染、產生重複 CSS
   → 修：永遠用 @use

❌ 2. @extend 用太多
   → selector 合併導致意外覆蓋
   → 修：80% 場景用 mixin、不用 extend

❌ 3. 巢狀深超過 3 層
   → .a .b .c .d .e { ... } 編譯後超長 selector
   → 修：扁平化、每個區塊獨立 class

❌ 4. 在 SCSS 內寫 CSS variable 期待 Sass 編譯
   → background: var(--primary) → Sass 原樣輸出
   → 修：要動態用 CSS var、要編譯用 Sass $

❌ 5. 用 Sass 變數做主題切換
   → Sass 編譯時固定、無法 runtime 換
   → 修：主題切換用 CSS variable
```

  
### 🎯 重點回顧

- **SCSS 是兼容 CSS 的語法**——業界 99% 用這個、不用舊 Sass 縮排
- **`$var` Sass 變數**——編譯時固定、跟 CSS `--var` 不同
- **巢狀（Nesting）** —— Sass 最大魅力、但 CSS 原生也有了
- **`@mixin` 模板化樣式**——Sass 最強功能
- **`@each` 自動產生大量 class**——Bootstrap 做法
- **`@use` 取代 `@import`**——新專案永遠用 @use
- **2026 Sass 角色縮小**——但 Bootstrap 主題、舊專案維護必懂

> 💡 **下一課（L17）**：**Tailwind CSS（utility-first 框架）** —— 2020 後改變前端世界的東西。**寫 class 不寫 CSS、開發速度 10 倍**。爭議大、但已成新專案標配。
