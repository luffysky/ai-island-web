**transform（變換）** = 旋轉、縮放、位移、傾斜——**動畫的引擎**。配合 transition 做出 hover 浮起、卡片翻轉、3D 翻牌、視差效果。

> 💡 **這課的價值**：transform 是「**動畫效能之王**」——只動 transform 跟 opacity、永遠 60fps。學完這課 = 你的動畫從卡頓變絲滑。

  
### 1. 為什麼用 transform 而不是 top / left

**transform 是 GPU 加速**——瀏覽器把元素丟到顯示卡處理、不重排版面：

```text
用 top / left 移動：
  → 觸發 layout（重新計算所有元素位置）
  → 觸發 paint（重畫）
  → 60 fps 很難達到
  → 手機卡頓

用 transform: translate() 移動：
  → 跳過 layout / paint
  → GPU 直接合成圖層
  → 穩定 60 fps
```

> 🚀 **動畫鐵律**：**移動用 translate、不用 top / left**。

  
### 2. translate（位移）

📄 **寫進程式檔**：`style.css`

```text
.box {
  /* 單軸 */
  transform: translateX(50px);
  transform: translateY(50px);
  transform: translateZ(20px);  /* 3D（要 perspective 才看得到）*/
  
  /* 雙軸（X, Y）*/
  transform: translate(50px, 100px);
  
  /* 3D（X, Y, Z）—— GPU 強制加速 */
  transform: translate3d(50px, 100px, 0);
  
  /* 用百分比（相對自己大小）*/
  transform: translateX(50%);
  transform: translate(-50%, -50%);  /* 經典置中技巧 */
}
```

> 💡 **百分比的 translate** 是「**相對自己的大小**」——`translate(-50%, -50%)` 表示往左上各退「自己一半」。**過去垂直水平置中靠這招**。

  
### 3. rotate（旋轉）

📄 **寫進程式檔**：`style.css`

```text
.box {
  transform: rotate(45deg);
  transform: rotate(0.25turn);  /* 0.25 turn = 90 度 */
  transform: rotate(1rad);       /* 弧度 */
  
  /* 各軸獨立 */
  transform: rotateX(45deg);     /* 繞 X 軸（上下翻）*/
  transform: rotateY(45deg);     /* 繞 Y 軸（左右翻）*/
  transform: rotateZ(45deg);     /* 繞 Z 軸（= 一般 rotate）*/
  
  /* 任意軸 */
  transform: rotate3d(1, 1, 0, 45deg);  /* x=1, y=1, z=0 軸 */
}
```

  
### 4. scale（縮放）

📄 **寫進程式檔**：`style.css`

```text
.box {
  transform: scale(1.2);          /* 放大 1.2 倍 */
  transform: scale(0.8);          /* 縮小到 0.8 倍 */
  transform: scale(2, 0.5);       /* X 軸 2 倍、Y 軸 0.5 倍 */
  
  /* 各軸獨立 */
  transform: scaleX(1.2);
  transform: scaleY(0.8);
  
  /* 鏡像（負值）*/
  transform: scaleX(-1);          /* 水平翻轉 */
}
```

> 💡 **scale 不影響 layout**——`.box` 視覺變大、但占的空間沒變。**完美做 hover 放大效果**。

  
### 5. skew（傾斜）

📄 **寫進程式檔**：`style.css`

```text
.box {
  transform: skewX(10deg);
  transform: skewY(5deg);
  transform: skew(10deg, 5deg);
  /* 想像「斜體字」效果，整個盒子被歪 */
}
```

> 💡 **skew 用得不多**——通常做特殊裝飾（傾斜條紋、平行四邊形按鈕）。不常用。

  
### 6. 組合多個 transform（順序影響結果）

📄 **寫進程式檔**：`style.css`

```text
.box {
  /* 順序：translate → rotate → scale */
  transform: translate(50px, 0) rotate(45deg) scale(1.2);
}

/* 注意：順序不同、結果不同 */
.a { transform: rotate(45deg) translate(100px, 0); }
/* → 先轉 45 度、再「往斜方向」移動 100px */

.b { transform: translate(100px, 0) rotate(45deg); }
/* → 先往右移 100px、再「在新位置」轉 45 度 */
```

> ⚠️ **transform 順序從左到右套用**——記這個、否則動畫永遠對不上預期。

  
### 7. transform-origin（變換原點）

預設所有 transform 圍繞元素「**中心點**」、用 `transform-origin` 改：

📄 **寫進程式檔**：`style.css`

```text
.box {
  transform: rotate(45deg);
  transform-origin: top left;
  /* → 從左上角為支點旋轉 */
}

/* 也可寫像素 */
.box2 {
  transform-origin: 0 0;       /* 左上 */
  transform-origin: 100% 100%; /* 右下 */
  transform-origin: 50px 30px; /* 距左 50、距上 30 */
}
```

  
### 8. 3D Transform（翻牌效果）

3D 需要父元素設 `perspective`（透視距離）：

📄 **寫進程式檔**：`style.css`

```text
.parent {
  perspective: 1000px;
  /* → 在父元素設、值越小透視越強 */
}

.card {
  transform-style: preserve-3d;
  transition: transform 0.6s;
}

.card:hover {
  transform: rotateY(180deg);
  /* → hover 時翻面（180 度 Y 軸旋轉）*/
}

/* 完整翻牌：兩面分開 */
.card .front, .card .back {
  position: absolute;
  inset: 0;
  backface-visibility: hidden;  /* 背面不顯示（翻過去看不到背面字）*/
}

.card .back {
  transform: rotateY(180deg);
}
```

> 🚀 **這就是「卡片翻面」效果**——展示產品正反面、抽卡遊戲、學習單字卡片都用這招。

  
### 9. 新語法（2023+）：可以單獨 animate

過去寫 `transform: translate(...) rotate(...) scale(...)` 三個一起、不能個別動畫。**現在可以分開寫**：

📄 **寫進程式檔**：`style.css`

```text
.box {
  translate: 50px 100px;   /* 新獨立屬性 */
  rotate: 45deg;
  scale: 1.2;
}

/* 個別動畫不互相覆蓋 */
.box:hover {
  rotate: 90deg;
  /* translate 跟 scale 不變、只動 rotate */
}
```

> 🚀 **新語法的好處**：(1) 不用每次都寫完整 `transform`(2) 個別屬性可以單獨 animate (3) 跟 Sass / 框架配合更乾淨。**2023 後全瀏覽器支援**。

  
### 10. 實戰例 1：卡片 hover 浮起

📄 **寫進程式檔**：`style.css`

```text
.card {
  background: white;
  border-radius: 12px;
  padding: 24px;
  box-shadow: 0 1px 3px rgba(0,0,0,.1);
  transition: transform 0.25s ease-out, box-shadow 0.25s ease-out;
}

.card:hover {
  transform: translateY(-4px);
  box-shadow: 0 12px 24px rgba(0,0,0,.12);
}
```

  
### 11. 實戰例 2：圖示放大

```text
.icon {
  transition: transform 0.2s;
}

.icon:hover {
  transform: scale(1.15);
}
```

  
### 12. 實戰例 3：3D 卡片翻牌

📄 **寫進程式檔**：`style.css`

```text
.flashcard {
  width: 300px;
  height: 200px;
  perspective: 1000px;
}

.flashcard-inner {
  position: relative;
  width: 100%;
  height: 100%;
  transform-style: preserve-3d;
  transition: transform 0.6s;
  cursor: pointer;
}

.flashcard:hover .flashcard-inner {
  transform: rotateY(180deg);
}

.flashcard-front,
.flashcard-back {
  position: absolute;
  inset: 0;
  backface-visibility: hidden;
  display: flex;
  align-items: center;
  justify-content: center;
  border-radius: 12px;
  font-size: 24px;
}

.flashcard-front {
  background: lightblue;
}

.flashcard-back {
  background: lightcoral;
  transform: rotateY(180deg);
}
```

📄 **寫進程式檔**：`index.html`

```text
<div class="flashcard">
  <div class="flashcard-inner">
    <div class="flashcard-front">apple</div>
    <div class="flashcard-back">蘋果</div>
  </div>
</div>
```

  
### 5 大常見坑

```text
❌ 1. 用 top / left 做動畫、卡頓
   → 觸發 layout、效能差
   → 修：改用 transform: translate()

❌ 2. translate 順序錯、動畫飄走
   → rotate 在前 vs 後、結果不同
   → 修：明確規劃順序、translate → rotate → scale

❌ 3. 3D 沒設 perspective、看起來像 2D
   → perspective 要設在「父元素」、不是自己
   → 修：父元素加 perspective: 1000px

❌ 4. 翻面時看到背面文字
   → 沒設 backface-visibility: hidden
   → 修：兩面都加這個屬性

❌ 5. transform-origin 沒設、旋轉飄離預期
   → 預設中心點、有時想用左上角
   → 修：明確設 transform-origin
```

  
### 🎯 重點回顧

- **transform 是動畫之王**——GPU 加速、60fps
- **translate / rotate / scale / skew**——4 大基本變換
- **順序從左到右**——translate → rotate → scale
- **`transform-origin`**——改變變換的「支點」
- **3D 要 `perspective`**——設在父元素
- **新語法（2023+）**：translate / rotate / scale 可獨立寫
- **永遠用 translate 代替 top / left**
