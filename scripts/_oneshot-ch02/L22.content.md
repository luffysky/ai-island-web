**a11y（accessibility、無障礙）** = 讓所有人都能用你的網站——包含視障、色盲、運動障礙、認知障礙、暫時性受傷（手扭到只能單手用滑鼠）。**10-20% 使用者有某種無障礙需求**。寫無障礙 CSS 不難、但要記住。

> 💡 **這課的價值**：學會 a11y = (1) **多 10-20% 使用者能用你的網站** (2) **政府 / 大企業案件必合規** (3) **SEO 加分** (4) **專業度的展現**。台灣身心障礙者權益保障法明文要求公部門網站無障礙。

  
### 1. 為什麼叫「a11y」

```text
accessibility 中間有 11 個字母（a + 11個字 + y）
→ 縮寫成 a11y

類似縮寫：
  i18n = internationalization（國際化）
  l10n = localization（在地化）
```

  
### 2. contrast（對比度）—— 最重要的事

**文字跟背景的對比度足夠**、才看得清楚：

📄 **寫進程式檔**：`style.css`

```text
/* ✅ 好對比 */
.good {
  color: #333;
  background: #fff;
  /* 對比度 12.6:1、超過 AA 標準 */
}

/* ❌ 差對比 */
.bad {
  color: #999;
  background: #fff;
  /* 對比度 2.8:1、低於 AA 標準（4.5:1）*/
}
```

**WCAG（Web Content Accessibility Guidelines）對比度標準**：

| 級別 | 一般文字 | 大文字（18px+ 粗體 / 24px+）|
|---|---|---|
| **AA**（最低要求）| 4.5:1 | 3:1 |
| **AAA**（最佳）| 7:1 | 4.5:1 |

**檢查工具**：
- Chrome DevTools（內建、點顏色就顯示對比度）
- WebAIM Contrast Checker（https://webaim.org/resources/contrastchecker/）
- Stark（Figma 外掛、設計階段就檢查）

> ⚠️ **新手最常犯**：用淺灰 `#aaa` 當輔助文字——看起來「優雅」、但對比 2:1 完全不及格。**輔助文字最深也只能到 `#666`**。

  
### 3. focus 樣式（鍵盤使用者必備）

**鍵盤使用者靠 Tab 鍵在元素間跳**——必須看得到「現在焦點在哪」：

📄 **寫進程式檔**：`style.css`

```text
/* ❌ 大忌：關掉 focus */
*:focus { outline: none; }
/* → 鍵盤使用者完全找不到焦點 */

/* ✅ 要客製就客製清楚 */
button:focus-visible {
  outline: 3px solid #0070f3;
  outline-offset: 2px;
}

/* :focus-visible 只在鍵盤 focus 時顯示、滑鼠 click 不會 */
```

> 🚀 **`:focus-visible` 比 `:focus` 好**——`:focus` 連滑鼠點都會出現外框（醜）、`:focus-visible` 只在鍵盤 Tab 時出現。**2026 寫法永遠用 `:focus-visible`**。

  
### 4. 螢幕閱讀器友善的隱藏（.sr-only）

**有些東西「視覺上不要顯示、但螢幕閱讀器要念」**——例：「跳到主內容」連結。

📄 **寫進程式檔**：`style.css`

```text
.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip-path: inset(50%);
  white-space: nowrap;
  border: 0;
}

/* 鍵盤 focus 時顯示（給鍵盤使用者看）*/
.sr-only:focus {
  position: static;
  width: auto;
  height: auto;
  padding: 8px 16px;
  background: yellow;
}
```

📄 **寫進程式檔**：`index.html`

```text
<!-- 視覺隱藏、但螢幕閱讀器念得到 -->
<a href="#main" class="sr-only">跳到主內容</a>

<!-- icon 按鈕加無障礙描述 -->
<button>
  <svg>...</svg>
  <span class="sr-only">關閉對話框</span>
</button>
```

> 💡 **`display: none` vs `.sr-only`**：(1) `display: none` 螢幕閱讀器**也讀不到** (2) `.sr-only` 視覺隱藏、**螢幕閱讀器仍念**。**Icon 按鈕一定要用 `.sr-only` 加文字描述**。

  
### 5. 不要只靠顏色傳達訊息

**8% 男性、0.5% 女性有色盲**——只用顏色區分會害他們看不懂：

```text
❌ 不好：
   <span style="color: red">錯誤</span>
   <span style="color: green">成功</span>
   → 紅綠色盲完全分不出
   
✅ 好：
   <span style="color: red">❌ 錯誤：請填寫名稱</span>
   <span style="color: green">✓ 成功：已儲存</span>
   → 顏色 + 圖示 + 文字、三重保險
```

📄 **寫進程式檔**：`index.html`

```text
<!-- 表單錯誤訊息 -->
<input class="error" aria-invalid="true" aria-describedby="err1">
<p id="err1" class="error-text">
  <span aria-hidden="true">⚠️</span>
  Email 格式錯誤
</p>

<!-- 狀態徽章 -->
<span class="badge success">
  ✓ 完成
</span>
<span class="badge danger">
  ✗ 失敗
</span>
```

  
### 6. 觸控目標最小 44×44px

**iOS / Material Design 都建議「**最小可觸控區域 44×44px**」**：

📄 **寫進程式檔**：`style.css`

```text
button, a {
  min-width: 44px;
  min-height: 44px;
  /* 即使 icon 很小、可點區域要大 */
}

/* icon-only 按鈕 */
.icon-btn {
  width: 44px;
  height: 44px;
  display: inline-flex;
  align-items: center;
  justify-content: center;
}
```

> 💡 **為什麼 44px**：(1) 平均成人指尖約 10mm = 44px (2) 手機操作不精準、要留容錯空間 (3) 對運動障礙者更友善。**Apple 設計指南明文規定**。

  
### 7. 表單無障礙

📄 **寫進程式檔**：`index.html`

```text
<!-- ✅ 好：label 跟 input 關聯 -->
<label for="email">Email</label>
<input id="email" type="email" required aria-required="true">

<!-- ❌ 不好：placeholder 當 label -->
<input placeholder="Email">
<!-- 螢幕閱讀器讀不到、輸入後也看不到 -->

<!-- 錯誤狀態 -->
<input
  type="email"
  aria-invalid="true"
  aria-describedby="email-error"
>
<p id="email-error">Email 格式不對</p>
```

  
### 8. prefers-reduced-motion（再強調）

```text
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

  
### 9. 字級不要寫死小於 16px

**手機預設「自動放大文字」的觸發條件 = 字級 < 16px**：

```text
❌ 不好：
  body { font-size: 12px; }
  → 手機自動放大、版面破裂

✅ 好：
  body { font-size: 16px; }
  → 正常大小、不會觸發放大
  
小字限制在「真的不重要的輔助資訊」（時間戳、版權）
```

  
### 10. 實戰：完整無障礙按鈕

📄 **寫進程式檔**：`style.css`

```text
.btn {
  /* 觸控目標 */
  min-width: 44px;
  min-height: 44px;
  padding: 8px 16px;
  
  /* 字級 */
  font-size: 16px;
  font-weight: 500;
  
  /* 對比 */
  background: #0070f3;       /* 對比 4.6:1 ✓ */
  color: white;
  
  /* 視覺反饋 */
  cursor: pointer;
  transition: all 0.2s;
}

.btn:hover {
  background: #0056c4;
}

/* 鍵盤焦點 */
.btn:focus-visible {
  outline: 3px solid #0070f3;
  outline-offset: 3px;
}

/* 禁用狀態 */
.btn:disabled {
  opacity: 0.6;
  cursor: not-allowed;
}
```

📄 **寫進程式檔**：`index.html`

```text
<!-- icon-only 按鈕 -->
<button class="btn icon-btn" aria-label="關閉">
  <svg aria-hidden="true">...</svg>
</button>

<!-- 文字 + icon -->
<button class="btn">
  <svg aria-hidden="true">...</svg>
  儲存
</button>
```

> 💡 **`aria-hidden="true"` 給 icon**——告訴螢幕閱讀器「**這個 icon 視覺裝飾、不要念**」、避免「儲存儲存」念兩次。

### 5 大常見坑

```text
❌ 1. 用 outline: none 關 focus
   → 鍵盤使用者迷路
   → 修：永遠保留、用 :focus-visible 客製

❌ 2. 用 placeholder 當 label
   → 螢幕閱讀器讀不到
   → 修：用 <label for> + <input id>

❌ 3. icon 按鈕沒文字
   → 螢幕閱讀器念出「按鈕」、不知道做什麼
   → 修：加 aria-label 或 .sr-only

❌ 4. 用 color 唯一區分狀態
   → 色盲使用者完全看不懂
   → 修：顏色 + 圖示 + 文字、三重保險

❌ 5. 觸控目標太小（< 44px）
   → 手機點不到、運動障礙者完全沒辦法
   → 修：min-width / min-height 44px
```

  
### 🎯 重點回顧

- **對比度 4.5:1 是底線**（AA 標準）
- **永遠保留 `:focus-visible`**——不要 `outline: none`
- **`.sr-only`**——視覺隱藏、螢幕閱讀器仍念
- **不只靠顏色傳達訊息**——加圖示 + 文字
- **觸控目標 ≥ 44×44px**——iOS / Material 規格
- **`aria-label` / `aria-hidden`** —— Icon 按鈕必加
- **字級 ≥ 16px**——手機不會自動放大
- **`prefers-reduced-motion`** —— 動畫無障礙
