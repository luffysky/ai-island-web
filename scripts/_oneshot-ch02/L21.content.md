**動畫不是為了炫技、是為了「溝通」**——每個動畫都在告訴使用者「發生了什麼」。這課講 **easing（緩動）/ duration（時長）/ Material Design / Apple HIG** 的動畫規範——讓你的微互動有品味、不是亂彈。

> 💡 **這課的價值**：學完動畫心理學 = 你做的網站「**感覺對**」。**Apple / Google 花了 10 年研究動畫對使用者體驗的影響**——這課把他們的精華濃縮給你。

  
### 1. 動畫的四個角色（為什麼要動畫）

動畫不是裝飾、是「**告訴使用者某件事**」：

```text
1. 狀態變化（State）
   → hover、active、disabled、selected
   → 「你現在在這、按了會這樣」
   
2. 方向暗示（Direction）
   → modal 從哪邊滑入（右 = 進階設定、下 = 通知）
   → 「這東西從哪來、會去哪」

3. 層級暗示（Hierarchy）
   → 陰影 + 浮起 = 更高層級
   → 「這比較重要、注意這個」

4. 注意力引導（Attention）
   → 脈衝、震動、發光
   → 「看這裡、別錯過」
```

**反例**：「我網站要超炫、所有按鈕都跳舞」——99% 使用者只覺得吵。**動畫服務功能、不服務虛榮**。

  
### 2. 緩動函數心理學（不同曲線傳達不同情緒）

```text
linear（直線）：
  → 機械、無感情、像機器人
  → 適合：loading 旋轉、進度條
  → 不適合：任何使用者互動
  → CSS：transition-timing-function: linear

ease-in（慢→快）：
  → 感覺「加速逃跑」、退場感
  → 適合：淡出、退場動畫
  → 例：toast 消失、modal 關閉

ease-out（快→慢）：
  → 感覺「到達」、減速感
  → 適合：進入、彈出、滑入
  → 例：modal 進場、卡片淡入
  → 「進場最常用」

ease-in-out（慢→快→慢）：
  → 兩端平緩、中間快
  → 適合：來回切換、tab 換頁
  → 例：手風琴展開

spring / overshoot（彈跳）：
  → 感覺「活的」、有質量
  → 適合：按鈕回彈、確認動作
  → cubic-bezier(0.34, 1.56, 0.64, 1)
```

  
### 3. CSS 緩動寫法

📄 **寫進程式檔**：`style.css`

```text
/* Apple 風緩動（業界最常用）*/
.card {
  transition: transform 0.3s cubic-bezier(0.4, 0, 0.2, 1);
}

/* Material Design 緩動 */
.button {
  transition: all 0.2s cubic-bezier(0.4, 0, 0.2, 1);
}

/* iOS 彈跳感（按鈕按下回彈）*/
.bounce {
  transition: transform 0.4s cubic-bezier(0.34, 1.56, 0.64, 1);
}

/* 進場（快進慢出）*/
.modal-enter {
  transition: all 0.3s ease-out;
}

/* 退場（慢進快出）*/
.modal-exit {
  transition: all 0.2s ease-in;
}
```

> 🚀 **`cubic-bezier(0.4, 0, 0.2, 1)` 是 Apple / Google 都在用的「**標準緩動**」**——背起來、新專案直接用。

  
### 4. 持續時間規範表（業界共識）

| 動畫類型 | 建議時長 | 範例 |
|---|---|---|
| **微互動** | **100-200ms** | 按鈕 hover、icon 點擊 |
| **狀態切換** | 200-300ms | tab 切換、checkbox 勾選 |
| **區塊進場** | 300-500ms | 卡片淡入、列表進入 |
| **頁面轉場** | 400-700ms | 路由切換、modal 開啟 |
| **強調動畫** | 1-2 秒 | 歡迎畫面、教學引導 |

> ⚠️ **動畫超過 500ms 就「**感覺慢**」**——Apple HIG 明文建議 200-300ms 最佳區間。**炫技動畫害死產品**。

  
### 5. Apple HIG 動畫原則

**Apple Human Interface Guidelines**（人機介面準則）動畫核心：

```text
1. Meaningful（有意義）
   → 每個動畫都該有目的
   → 沒目的的動畫 = 干擾

2. Subtle（細微）
   → 別搶戲、別讓使用者注意動畫
   → 動畫應該「感受到但不分心」

3. Consistent（一致）
   → 同類動作用同樣動畫
   → modal 永遠從同方向進

4. Performant（流暢）
   → 必須 60fps、不能卡頓
   → 只動 transform / opacity
```

  
### 6. Material Design 動畫原則

**Google Material Design** 動畫精神：

```text
1. Easing（變速感）
   → 自然世界沒有 linear 運動
   → 真實物體都有加速 / 減速

2. Duration（時長）
   → 進入比退出長（吸引注意）
   → 進場 300ms、退場 200ms

3. Choreography（編舞）
   → 多元素動畫要「協調」
   → 不要全部同時動、有先後

4. Emphasis（強調）
   → 重要元素動畫大、次要元素動畫小
```

  
### 7. 實戰：好的按鈕動畫

📄 **寫進程式檔**：`style.css`

```text
.btn {
  background: var(--primary);
  color: white;
  padding: 12px 24px;
  border-radius: 8px;
  
  /* 多屬性、各自時長 */
  transition:
    transform 0.15s cubic-bezier(0.4, 0, 0.2, 1),
    background 0.2s ease,
    box-shadow 0.2s ease;
}

/* hover：浮起 + 加陰影 */
.btn:hover {
  transform: translateY(-1px);
  background: color-mix(in oklch, var(--primary) 90%, black);
  box-shadow: 0 4px 12px rgba(0,0,0,.15);
}

/* active：按下時微縮 */
.btn:active {
  transform: scale(0.97);
  transition-duration: 0.05s;  /* 按下時立刻反應 */
}

/* focus：鍵盤焦點 */
.btn:focus-visible {
  outline: 3px solid var(--primary);
  outline-offset: 2px;
}
```

> 💡 **這就是專業按鈕**——hover 浮起（告訴你「在這、能按」）+ active 微縮（按下回饋）+ focus 外框（鍵盤友善）。**每個狀態都在跟使用者溝通**。

  
### 8. 實戰：列表進場（編舞）

**不要全部同時進、有先後才有層次**：

📄 **寫進程式檔**：`style.css`

```text
@keyframes fade-up {
  from { opacity: 0; transform: translateY(20px); }
  to   { opacity: 1; transform: translateY(0); }
}

.list-item {
  opacity: 0;
  animation: fade-up 0.5s ease-out forwards;
}

/* 每個 item 延遲 0.1 秒 */
.list-item:nth-child(1) { animation-delay: 0.0s; }
.list-item:nth-child(2) { animation-delay: 0.1s; }
.list-item:nth-child(3) { animation-delay: 0.2s; }
.list-item:nth-child(4) { animation-delay: 0.3s; }
```

> 🚀 **這叫 staggered animation（交錯動畫）**——電商商品列、Linear / Notion 開啟畫面常見技巧。

  
### 9. prefers-reduced-motion（無障礙必加）

**5% 使用者前庭功能敏感、動畫會引起頭暈**：

📄 **寫進程式檔**：`style.css`

```text
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

> ⚠️ **這段一定要加到每個專案 base CSS**——使用者在系統設定開啟「減少動態效果」、你的網站自動關閉動畫。**否則會害到前庭敏感族群**。

  
### 10. 動畫設計清單（每個動畫都問）

```text
做動畫前先問：

1. 這動畫對使用者有什麼幫助？
   ❌ 沒幫助 → 拿掉
   ✅ 有幫助 → 繼續

2. 時長合理嗎？
   微互動 < 300ms？
   超過 500ms 就太慢

3. 緩動對嗎？
   進場用 ease-out、退場用 ease-in

4. 只動 transform / opacity 嗎？
   動 top / left → 改 translate

5. 處理了 prefers-reduced-motion 嗎？
   無障礙必做

6. 多元素有編舞嗎？
   全部同時動 = 沒層次
```

  
### 11. 動畫反例（這些別做）

```text
❌ 自動播放彈跳的英文標題
   → 干擾閱讀、煩躁

❌ hover 整個放大 1.3 倍
   → 太誇張、不是「微互動」

❌ 1 秒以上的進場動畫
   → 使用者已經想點別的了

❌ 每個按鈕都旋轉跳舞
   → 喧賓奪主、沒重點

❌ 用 GIF 做 loading
   → 改用 CSS 動畫、效能 10 倍好

❌ 行銷頁全是 scroll-jacking 動畫
   → 使用者只想看內容、被綁架很煩
```

  
### 5 大常見坑

```text
❌ 1. 用 linear 做按鈕互動
   → 感覺機械、不自然
   → 修：用 ease-out 或 cubic-bezier

❌ 2. 動畫超過 500ms
   → 感覺慢、使用者已不耐
   → 修：微互動 150-300ms

❌ 3. 沒處理 prefers-reduced-motion
   → 害到前庭敏感族群
   → 修：base CSS 加 @media

❌ 4. 多元素全部同時動
   → 沒層次、不專業
   → 修：用 animation-delay 編舞

❌ 5. 動 top / left 而不是 transform
   → 卡頓、效能差
   → 修：永遠用 transform
```

  
### 🎯 重點回顧

- **動畫服務功能、不服務虛榮**——沒幫助就拿掉
- **4 個角色**：狀態 / 方向 / 層級 / 注意力
- **緩動心理學**：ease-out 進場、ease-in 退場、spring 彈跳
- **`cubic-bezier(0.4, 0, 0.2, 1)`**——Apple / Google 標準
- **時長**：微互動 150-300ms、不超過 500ms
- **編舞 staggered animation**——多元素有先後
- **`prefers-reduced-motion`**——無障礙必加

> 💡 **下一課（L22）**：**a11y（accessibility、無障礙）** —— 10-20% 使用者有某種無障礙需求（視障、色盲、運動障礙）。**對比度、focus、prefers-reduced-motion、不只靠顏色傳達訊息**——這是專業度的展現。
