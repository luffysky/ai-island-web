網頁上每個元素都是矩形盒子。盒模型 = 這個盒子怎麼計算大小。

```text
┌─────────────────────────────────────┐
│           margin（外距）              │
│  ┌───────────────────────────────┐  │
│  │       border（邊框）           │  │
│  │  ┌─────────────────────────┐  │  │
│  │  │     padding（內距）      │  │  │
│  │  │  ┌───────────────────┐  │  │  │
│  │  │  │     content        │  │  │  │
│  │  │  │   width × height   │  │  │  │
│  │  │  └───────────────────┘  │  │  │
│  │  └─────────────────────────┘  │  │
│  └───────────────────────────────┘  │
└─────────────────────────────────────┘
```

  
### 兩種計算模式

```css
/* 預設：content-box（width 只算內容） */
.box1 {
  width: 200px;
  padding: 20px;
  border: 5px solid;
  /* 實際寬度 = 200 + 20*2 + 5*2 = 250px */
}

/* border-box（width 包含 padding + border） */
.box2 {
  box-sizing: border-box;
  width: 200px;
  padding: 20px;
  border: 5px solid;
  /* 實際寬度 = 200px，內容會被擠到 200 - 20*2 - 5*2 = 150px */
}
```

  
**✨ 接案小技巧**

每個專案開頭一定寫這條：`* { box-sizing: border-box; }`。**讓所有元素都用 border-box**，width 設多少就是多少，省下無數計算地獄。所有現代 CSS 框架都這樣設。

### 單位完整對照

  
| 單位 | 意義 | 建議場景 |
| --- | --- | --- |
| `px` | 絕對像素 | 邊框、陰影 |
| `%` | 父元素百分比 | 寬度、容器 |
| `em` | 當前元素字級倍數 | 很少用，會疊乘 |
| `rem` | 根元素字級倍數 | **字級、間距首選** |
| `vw / vh` | 視窗寬/高的 1% | 滿版 hero、字級縮放 |
| `dvh` | 動態視窗高 | 手機（避免地址欄問題） |
| `svh / lvh` | 小/大視窗高 | 進階手機 |
| `ch` | 字元 "0" 的寬度 | 限制文章寬度（70ch 最舒服） |
| `fr` | Grid 的份數 | Grid 排版 |

  
**🚀 進階技巧**

2026 年的手機高度單位選擇：用 `dvh` 不用 `vh`。原因：手機瀏覽器地址欄會收合，`vh` 算的是「最高時」，會讓內容溢出。`dvh` 會跟著實際視窗動態變。`100dvh` 是滿版的正確寫法。
