### 1. 2026 現代 CSS 八大核心（速查表）

**這 8 個技術 = 2026 現代 CSS 工程師必會**：

```text
✅ 1. 用 oklch() 配色
   → 取代 HSL、感知均勻
   → Tailwind v4 / shadcn / Linear / Vercel 都在用

✅ 2. 用 :has() 取代部分 JS
   → 「根據子元素選父元素」、過去要 JS
   → 例：form:has(input:invalid) button { opacity: 0.5 }

✅ 3. 用 container queries 而非 media queries
   → 元件真正可重用
   → 不看視窗大小、看容器大小

✅ 4. 用 clamp() 做流體排版
   → 一行取代多個 media query
   → font-size: clamp(1rem, 3vw, 2rem)

✅ 5. 用 @layer 管理優先級
   → Cascade Layers
   → Tailwind v4 / shadcn 架構基礎

✅ 6. 用 dvh 而非 vh
   → 手機地址欄不會擋

✅ 7. 用 CSS 變數做主題
   → 暗黑模式一個 data-theme 切換
   → 不要用 Sass 變數做動態主題

✅ 8. animation 只動 transform / opacity
   → GPU 加速、60fps
   → 動 width / left = 卡頓
```

  
### 2. 命名慣例（選一套用）

**4 大命名流派**：

```text
1. BEM（Block-Element-Modifier）
   .card               ← Block（區塊）
   .card__title         ← Element（元素）
   .card--featured     ← Modifier（修飾）
   → 規則嚴格、團隊容易統一
   → 純 CSS 專案首選

2. OOCSS（Object-Oriented CSS）
   .btn + .btn-primary
   → 把「結構」跟「皮膚」分開
   → Bootstrap 哲學

3. SMACSS
   → 分 base / layout / module / state / theme
   → 五個資料夾、各自負責
   → 大型專案組織

4. Utility-first（Tailwind）
   → 徹底放棄命名
   → 全用 utility class
   → 2026 新主流
```

> 💡 **怎麼選**：(1) **純 CSS 專案** → BEM (2) **元件框架（Vue / Svelte）** → 自帶 scoped (3) **React / Next.js** → Tailwind（業界主流）。**選一個堅持用、不要混**。

  
### 3. BEM 完整範例

📄 **寫進程式檔**：`style.css`

```text
/* Block：獨立元件 */
.card { padding: 16px; }

/* Element：屬於 Block 的部分（用 __）*/
.card__header { font-weight: bold; }
.card__title { font-size: 18px; }
.card__body { padding: 8px 0; }
.card__footer { border-top: 1px solid #eee; }

/* Modifier：變體（用 --）*/
.card--featured { background: gold; }
.card--small { padding: 8px; }

/* 組合 */
.card.card--featured .card__title {
  color: red;
}
```

📄 **寫進程式檔**：`index.html`

```text
<article class="card card--featured">
  <header class="card__header">
    <h3 class="card__title">標題</h3>
  </header>
  <div class="card__body">內容</div>
  <footer class="card__footer">底部</footer>
</article>
```

> 💡 **BEM 的好處**：(1) **看 HTML 就知道結構** (2) **不會跟其他元件撞名** (3) **特異性扁平**（都是單 class、容易覆蓋）。

  
### 4. 檔案組織（傳統 CSS）

```text
styles/
├── reset.css           ← 重置瀏覽器預設
├── tokens.css          ← Design tokens（變數）
├── base.css            ← 全域 base（body / 字型）
├── components/
│   ├── button.css
│   ├── card.css
│   ├── form.css
│   └── modal.css
├── layout/
│   ├── header.css
│   ├── footer.css
│   └── grid.css
├── pages/
│   ├── home.css
│   └── product.css
├── utilities.css       ← 額外 utility
└── main.css            ← @import 全部
```

📄 **寫進程式檔**：`main.css`

```text
@import "reset.css";
@import "tokens.css";
@import "base.css";

@import "layout/header.css";
@import "layout/footer.css";

@import "components/button.css";
@import "components/card.css";
@import "components/form.css";

@import "utilities.css";
```

  
### 5. 檔案組織（Tailwind 專案）

**Tailwind 大大減少 CSS 檔**：

```text
src/
├── styles/
│   ├── main.css        ← @import "tailwindcss" + @theme + @layer
│   └── globals.css     ← 真的需要的全域樣式（很少）
└── components/
    ├── Button.tsx       ← className="..." 直接寫 utility
    └── Card.tsx
```

📄 **寫進程式檔**：`main.css`

```text
@import "tailwindcss";

@theme {
  --color-brand: oklch(0.65 0.2 240);
  --color-brand-dark: oklch(0.5 0.2 240);
  --font-display: "Inter", sans-serif;
}

@layer components {
  .btn-primary {
    @apply px-4 py-2 bg-brand text-white rounded;
  }
}
```

```text

   → PostCSS 外掛

   → class 自動補全 + 預覽顏色

   → 點 class 跳到定義
```

  
### 7. 跟設計師合作（接案 / 大廠都用）

**設計師端工具**：

```text
🎨 Figma（設計 + 標註 + 程式碼預覽）
🎨 Sketch（macOS 老牌）
🎨 Adobe XD（已過時）

→ 工程師工具：
   - Figma Dev Mode（直接看 CSS、複製樣式）
   - Tokens Studio for Figma（管理 design token）
   - Anima（Figma → React 程式碼）
```

**對齊工作流**：

```text
1. 設計師：在 Figma 用 Variables 管理 token
2. 用 Tokens Studio 同步到 GitHub
3. 工程師：跑 Style Dictionary 產出 CSS
4. CSS 變數連動：var(--color-primary)
5. 設計師改色：Figma 改 variable
   → 自動同步 GitHub
   → 工程師端 npm install 拿到新版
   → 整站變色、不改任何元件
```

> 🚀 **這就是大廠（Atlassian / Shopify / Vercel）的標準流程**——設計師跟工程師零誤差。

### 5 大新手到老手的差別

```text
1. 新手：把所有東西寫成 .css 一個檔
   老手：拆檔案、用 BEM 或 Tailwind 系統化

2. 新手：陰影寫 0 0 10px black
   老手：多層微妙陰影、模擬真實光線

3. 新手：每個元件複製貼上樣式
   老手：抽 token + mixin、改一處全部跟著變

4. 新手：媒體查詢寫死 5 個斷點
   老手：clamp + container query + auto-fit

5. 新手：動畫亂彈 + 沒處理 a11y
   老手：每個動畫有目的 + prefers-reduced-motion
```
