**Grid（網格）** = CSS Grid Layout、**二維排版引擎**（同時控制行 × 列）。Flexbox 解一維（一排）、Grid 解二維（行 + 列）。儀表板、雜誌排版、商品圖庫、複雜後台都靠它。

> 💡 **這課的價值**：**Grid + Flex 一起用 = 任何設計都做得出來**。整體大佈局用 Grid（行 × 列）、每個區塊內的對齊用 Flex。這是 2026 業界標準組合拳。

  
### 1. Grid 跟 Flex 的分工

```text
Flex（一維）：
  → 一排東西（橫向或縱向、但只有一個方向）
  → 導覽列、卡片橫排、按鈕組

Grid（二維）：
  → 同時控制「行 × 列」
  → 儀表板（左側欄 + 主內容 + 右側欄 + 上下 header/footer）
  → 雜誌排版（標題跨 3 欄、圖片占 1 欄）
  → 商品圖庫（4 × n 網格）

實戰：
  → 外層大架構用 Grid（行 × 列分區）
  → 每區內部對齊用 Flex
```

  
### 2. 基礎：定義行跟列

📄 **寫進程式檔**：`style.css`

```text
.grid {
  display: grid;
  
  /* 定義列（columns）*/
  grid-template-columns: 200px 1fr 100px;
  /* → 左固定 200、中彈性、右固定 100 */
  
  /* 等寬 3 欄 */
  grid-template-columns: 1fr 1fr 1fr;
  
  /* 簡寫 */
  grid-template-columns: repeat(3, 1fr);
  
  /* 列（rows）*/
  grid-template-rows: auto 1fr auto;
  /* → 頭 auto（內容多高就多高）、身彈性、尾 auto */
  
  /* 間距 */
  gap: 16px;
  row-gap: 16px;
  column-gap: 24px;
}
```

> 💡 **`fr` 單位是 Grid 專屬**——`fr` = fraction（份數）。`1fr 2fr 1fr` 表示「1 份 + 2 份 + 1 份」、總共 4 份、自動換算寬度。**完勝百分比**（百分比要自己算 gap）。

  
### 3. 響應式網格（auto-fit / auto-fill）

```text
需求：螢幕寬就多塞幾個、窄就少塞、自動換行
```

📄 **寫進程式檔**：`style.css`

```text
.gallery {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(250px, 1fr));
  gap: 16px;
}

/* 解讀：
   每欄最小 250px、最大 1fr（撐滿剩餘）
   auto-fill → 容器能塞幾欄就塞幾欄、不夠就換行
   
   螢幕 1200px → 一排 4 欄
   螢幕 800px → 一排 3 欄
   螢幕 500px → 一排 2 欄
   完全不用 media query！
*/
```

> 🚀 **這是 2026 響應式排版的「主流寫法」**——比 Flex 的 `flex: 1 1 300px` 更精準（會嚴格遵守最小寬度、不會擠在一起）。

  
### 4. auto-fit vs auto-fill 差別

```text
auto-fill：
  → 保留空欄位（即使沒內容）
  → 視覺：[A][B][C][空欄][空欄]

auto-fit：
  → 自動壓縮空欄位、讓現有內容撐滿
  → 視覺：[A----][B----][C----]

新手怎麼選：
  → 想要「卡片撐滿」 → auto-fit
  → 想要「保持欄位大小一致」 → auto-fill
```

  
### 5. grid-template-areas（區域命名）

**最易讀的 Grid 寫法**——用「區域名字」畫出版面：

📄 **寫進程式檔**：`style.css`

```text
.dashboard {
  display: grid;
  grid-template-columns: 240px 1fr;
  grid-template-rows: 60px 1fr;
  grid-template-areas:
    "sidebar header"
    "sidebar main";
  height: 100dvh;
}

.sidebar { grid-area: sidebar; }
.header  { grid-area: header; }
.main    { grid-area: main; }
```

📄 **寫進程式檔**：`index.html`

```text
<div class="dashboard">
  <aside class="sidebar">側邊欄</aside>
  <header class="header">頂部</header>
  <main class="main">主內容</main>
</div>
```

> 💡 **這寫法「畫」出版面**——`grid-template-areas` 裡的字串就是視覺位置。**設計師看 CSS 都看得懂**、極度直觀。

  
### 6. 子項目放置（grid-column / grid-row）

子元素可以「**跨欄 / 跨列**」：

📄 **寫進程式檔**：`style.css`

```text
.item {
  /* 從第 1 條線到第 3 條線（跨 2 格）*/
  grid-column: 1 / 3;
  grid-row: 1 / 2;
  
  /* 簡寫：跨幾格 */
  grid-column: span 2;
  
  /* 從這條線到最後一條 */
  grid-column: 2 / -1;
  /* → 從第 2 條到最後（-1 是反向計算）*/
}
```

> 💡 **Grid 線是 1, 2, 3... 從 1 開始數**——不是從 0。負值是從反向數（-1 = 最後一條線）。

  
### 7. 實戰：完整的後台佈局

📄 **寫進程式檔**：`style.css`

```text
.dashboard {
  display: grid;
  grid-template-columns: 240px 1fr;
  grid-template-rows: 60px 1fr 40px;
  grid-template-areas:
    "sidebar header"
    "sidebar main"
    "sidebar footer";
  height: 100dvh;
  gap: 0;
}

.sidebar { grid-area: sidebar; background: #1f2937; }
.header  { grid-area: header;  background: white; border-bottom: 1px solid #e5e7eb; }
.main    { grid-area: main;    padding: 24px; background: #f9fafb; }
.footer  { grid-area: footer;  background: white; border-top: 1px solid #e5e7eb; }
```

  
### 8. 實戰：自適應卡片牆（最常用）

📄 **寫進程式檔**：`style.css`

```text
.gallery {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));
  gap: 16px;
  padding: 16px;
}

.card {
  background: white;
  border-radius: 8px;
  padding: 16px;
  box-shadow: 0 1px 3px rgba(0,0,0,.1);
}
```

> 💡 **這 4 行 CSS 解決了 90% 的卡片排版需求**——學會這個、你就贏過一半新手。

  
### 9. Subgrid（2023 新功能）

**subgrid（子網格）** = 子元素繼承父 grid 的線、自動同步對齊：

📄 **寫進程式檔**：`style.css`

```text
.outer {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
}

.inner {
  display: grid;
  grid-template-columns: subgrid;
  /* → 繼承父 grid 的 3 欄線 */
}

/* 過去要算半天的對齊、subgrid 自動同步 */
```

> 🚀 **subgrid 解決了 Grid 的長期痛點**——「子 grid 對齊到父 grid 的線」。**2023 後所有瀏覽器支援**。

  
### 10. Grid 進階屬性速查

```text
🔧 容器（父）：
   grid-template-columns / rows
   grid-template-areas
   gap / row-gap / column-gap
   justify-items（每個子元素的主軸對齊）
   align-items（每個子元素的交叉軸對齊）
   justify-content（整個 grid 的主軸對齊）
   align-content（整個 grid 的交叉軸對齊）

📌 子項目：
   grid-column / grid-row（跨欄 / 跨列）
   grid-area（命名區域）
   justify-self（自己的主軸對齊）
   align-self（自己的交叉軸對齊）
```

  
### 5 大常見坑

```text
❌ 1. 父元素忘記 display: grid
   → grid-template-columns 完全沒效
   → 修：永遠先寫 display: grid

❌ 2. 用 grid-template-columns: 100% 100% 100%
   → 加起來 300%、爆出容器
   → 修：用 1fr 1fr 1fr 或 repeat(3, 1fr)

❌ 3. 用 padding / margin 控制間距
   → 邊緣會多一截
   → 修：永遠用 gap

❌ 4. auto-fill 跟 auto-fit 不知道差別
   → 沒內容時表現不同
   → 修：要撐滿就 auto-fit、要保留空位就 auto-fill

❌ 5. 想用 Grid 做一維排版（一排東西）
   → Grid 殺雞用牛刀、語意不對
   → 修：一維用 Flex、二維才用 Grid
```

  
### 🎯 重點回顧

- **Grid 解二維**（行 × 列）、**Flex 解一維**（一排）
- **`fr` 是 Grid 專屬單位**——按份數分配空間
- **`repeat(auto-fill, minmax(250px, 1fr))`**——響應式神咒
- **`grid-template-areas`**——畫出版面、設計師看得懂
- **`subgrid`**——子元素繼承父 grid 線、2023 新功能
- **實戰**：外層 Grid、內層 Flex、組合無敵
