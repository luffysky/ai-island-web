**position（定位）** 屬性有 5 種人格——`static` / `relative` / `absolute` / `fixed` / `sticky`。**modal、固定 header、回到頂部按鈕、毛玻璃黏性導覽**都靠這個屬性。

> 💡 **這課的價值**：搞懂 position = 知道為什麼有時 `top: 0` 沒效、為什麼絕對定位會「飄走」。**這課學完 = 任何浮動元素都定位得到**。

  
### 1. 五種人格一覽

| 值 | 行為 | 用在哪 |
|---|---|---|
| `static` | 預設、跟著文檔流 | 一般元素 |
| `relative` | 相對自己原位偏移、原位保留 | 微調位置、當絕對定位的父元素 |
| `absolute` | 脫離文檔流、相對最近的 relative 父元素 | 浮動 icon、tooltip、徽章 |
| `fixed` | 相對視窗固定、捲動不動 | 固定 header、回到頂部、聊天按鈕 |
| `sticky` | 滾到邊界後變固定 | 黏性 header、章節標題 |

  
### 2. static（預設）

每個元素預設 `position: static`——**跟著文檔流走、`top` / `left` 設了沒用**：

📄 **寫進程式檔**：`style.css`

```text
.a {
  position: static;
  top: 100px;  /* ❌ 沒效、static 不吃 top/left/right/bottom */
}
```

> 💡 **新手第一個踩坑**：寫 `top: 100px` 沒效 → 99% 是忘了寫 position（預設 static 不吃 top）。

  
### 3. relative（相對定位）

**相對自己原本位置偏移**、但**原位置保留**（不影響其他元素）：

📄 **寫進程式檔**：`style.css`

```text
.b {
  position: relative;
  top: 10px;   /* 往下 10px */
  left: 20px;  /* 往右 20px */
}
/* → 元素往右下移、但原來的空間還在（其他元素不會擠過來填補）*/
```

**relative 主要用途**：
- 微調位置（小幅偏移）
- **當絕對定位的「父元素參考點」**（下一節詳教）

  
### 4. absolute（絕對定位）

**脫離文檔流**（不占空間了）、**相對「最近的 relative / absolute / fixed 父元素」定位**：

📄 **寫進程式檔**：`style.css`

```text
.parent { position: relative; }  /* ← 一定要寫、不然會跑去 body */

.child {
  position: absolute;
  top: 0;
  right: 0;
  /* → 貼在父元素的右上角 */
}
```

📄 **寫進程式檔**：`index.html`

```text
<div class="parent">
  我是父元素
  <span class="child">🔔</span>
</div>
```

> ⚠️ **新手最常踩**：absolute 子元素「飄走了」、跑到頁面左上角 → **父元素忘了寫 `position: relative`**。**absolute 一定要配 relative 父元素**、永遠成對寫。

  
### 5. fixed（固定定位）

**相對「視窗」固定**、捲動畫面也不會動：

📄 **寫進程式檔**：`style.css`

```text
.scroll-top-btn {
  position: fixed;
  bottom: 20px;
  right: 20px;
  width: 48px;
  height: 48px;
  border-radius: 50%;
  background: blue;
  /* → 永遠在右下角、滑鼠捲多深都看得到 */
}
```

**fixed 常見用法**：
- 回到頂部按鈕
- 聊天 / 客服浮動按鈕
- 固定 header（但 sticky 更好、見下節）
- modal 對話框遮罩

  
### 6. sticky（黏性定位）—— 2026 最常用

**滾到邊界後變固定、超過範圍恢復正常**——`fixed` 跟 `relative` 的綜合體：

📄 **寫進程式檔**：`style.css`

```text
header {
  position: sticky;
  top: 0;
  /* → 一開始正常顯示、捲動到頂部時黏住、繼續捲還是黏著 */
  background: white;
  z-index: 100;
}
```

**sticky 主要用途**：
- 黏性 header
- 章節標題（在自己 section 內黏住）
- 表格的固定欄位 / 行

> 💡 **sticky 比 fixed 好的地方**：(1) **不會擋住空間**（fixed 要手動補 padding）(2) **可以「在某範圍內」黏住**（fixed 永遠黏著）。**2026 新專案幾乎不用 fixed header、都用 sticky**。

  
### 7. z-index：誰在上面

當元素疊在一起時、`z-index` 決定誰在上面：

📄 **寫進程式檔**：`style.css`

```text
.modal { position: fixed; z-index: 1000; }
.toast { position: fixed; z-index: 2000; }
/* → toast 比 modal 更上面 */

/* 注意：z-index 只在 position 不是 static 時生效 */
```

> ⚠️ **z-index 大戰是真會發生的事**：團隊裡 A 寫 999、B 寫 9999、C 不甘示弱寫 99999。**正確做法**：用 CSS 變數定義一套 z-index 系統：

📄 **寫進程式檔**：`style.css`

```text
:root {
  --z-dropdown: 100;
  --z-sticky: 200;
  --z-fixed: 300;
  --z-modal-backdrop: 1000;
  --z-modal: 1001;
  --z-toast: 1100;
  --z-tooltip: 1200;
}

.modal { z-index: var(--z-modal); }
.toast { z-index: var(--z-toast); }
/* → 全專案統一用變數、不准寫死數字 */
```

  
### 8. 實戰例 1：浮動徽章（absolute）

📄 **寫進程式檔**：`style.css`

```text
.icon-wrapper {
  position: relative;  /* ← 父元素的錨點 */
  display: inline-block;
}

.badge {
  position: absolute;
  top: -4px;
  right: -4px;
  background: red;
  color: white;
  font-size: 12px;
  padding: 2px 6px;
  border-radius: 999px;
}
```

📄 **寫進程式檔**：`index.html`

```text
<div class="icon-wrapper">
  🔔
  <span class="badge">3</span>
</div>
```

  
### 9. 實戰例 2：黏性 header + 章節標題

📄 **寫進程式檔**：`style.css`

```text
/* 整頁 header */
header {
  position: sticky;
  top: 0;
  background: rgba(255, 255, 255, 0.9);
  backdrop-filter: blur(10px);  /* 毛玻璃效果 */
  z-index: 100;
  padding: 16px;
}

/* 章節標題（在 header 下方黏住）*/
section h2 {
  position: sticky;
  top: 60px;  /* 在 header 下方 60px 處黏住 */
  background: white;
  padding: 8px;
  margin: 0;
}
```

> 💡 **毛玻璃效果**：`backdrop-filter: blur(10px)` + 半透明背景——這是 Apple / Linear / Vercel 都在用的「**現代質感**」。

  
### 10. 實戰例 3：Modal 對話框

📄 **寫進程式檔**：`style.css`

```text
/* 遮罩（蓋住整個畫面）*/
.modal-backdrop {
  position: fixed;
  inset: 0;  /* = top: 0; right: 0; bottom: 0; left: 0 */
  background: rgba(0, 0, 0, 0.5);
  z-index: 1000;
  display: flex;
  justify-content: center;
  align-items: center;
}

/* 對話框本體 */
.modal {
  background: white;
  border-radius: 12px;
  padding: 24px;
  max-width: 500px;
  width: 90%;
  z-index: 1001;
}
```

> 💡 **`inset: 0` 是 2021 後新語法**——一行頂掉 `top / right / bottom / left: 0`。**永遠用 `inset`**。

  
### 5 大常見坑

```text
❌ 1. 寫了 top: 100px 卻沒動
   → 忘了寫 position（預設 static 不吃 top）
   → 修：先寫 position: relative / absolute / fixed

❌ 2. absolute 子元素飄到頁面左上角
   → 父元素沒寫 position: relative
   → 修：永遠成對「父 relative + 子 absolute」

❌ 3. fixed header 蓋住內容
   → fixed 脫離文檔流、不占空間
   → 修：給 body 加 padding-top、或改用 sticky

❌ 4. z-index 戰爭（999 vs 9999 vs 99999）
   → 修：用 CSS 變數定義 z-index 系統

❌ 5. sticky 黏不住
   → 父元素有 overflow: hidden / auto
   → 或 sticky 元素沒寫 top / bottom
   → 修：檢查父元素 overflow + sticky 一定要寫方向
```

  
### 🎯 重點回顧

- **static**（預設、不吃 top）
- **relative**（相對自己偏移、原位保留）
- **absolute**（脫離文檔流、相對最近 relative 父元素）
- **fixed**（相對視窗、捲動不動）
- **sticky**（滾到邊界後固定、現代 header 首選）
- **`z-index` 用 CSS 變數管理**——避免戰爭
- **`inset: 0` 是 2021 新語法**——一行頂四行
