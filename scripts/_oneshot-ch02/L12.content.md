**RWD（Responsive Web Design、響應式設計）** = 同一段 HTML + CSS、在手機 / 平板 / 桌機都好看。**2026 年沒做 RWD = 等於沒做網頁**——70% 流量來自手機。

> 💡 **這課的價值**：學會 RWD = 你寫的網站在所有裝置都能用。**`@media` / `clamp()` / `container query`** 一次學完——2026 主流是「少 media query、多 clamp + container」。

  
### 1. viewport meta（一定要寫）

手機要正確縮放、必須在 HTML 的 `<head>` 加這行：

📄 **寫進程式檔**：`index.html`

```text
<head>
  <meta name="viewport" content="width=device-width, initial-scale=1">
</head>
```

> ⚠️ **沒寫 viewport meta**：手機會把網頁當「桌面版縮小顯示」——所有東西小到看不清、要放大才能讀。**新手最常忘**。

  
### 2. Media Query（媒體查詢）—— 基礎

**`@media`（媒體查詢）** = 用條件套用不同 CSS。**Mobile First（手機優先）寫法**：

📄 **寫進程式檔**：`style.css`

```text
/* 預設：手機（< 768px）*/
.card {
  width: 100%;
}

/* 平板以上（≥ 768px）*/
@media (min-width: 768px) {
  .card { width: 50%; }
}

/* 桌面（≥ 1024px）*/
@media (min-width: 1024px) {
  .card { width: 33.33%; }
}
```

> 💡 **Mobile First vs Desktop First**：(1) **Mobile First** = 預設手機、`min-width` 往上加（推薦、CSS 更少）(2) **Desktop First** = 預設桌面、`max-width` 往下減。**2026 業界主流是 Mobile First**。

  
### 3. 標準斷點（業界共識）

| 名稱 | min-width | 裝置 |
|---|---|---|
| sm | 640px | 大手機 |
| md | 768px | 平板 |
| lg | 1024px | 小筆電 |
| xl | 1280px | 桌機 |
| 2xl | 1536px | 大螢幕 |

> 💡 **這 5 個是 Tailwind 預設斷點、業界已成共識**——直接背起來用。不用自己想。

  
### 4. 進階 media 查詢

📄 **寫進程式檔**：`style.css`

```text
/* 偵測使用者偏好暗色模式 */
@media (prefers-color-scheme: dark) {
  body {
    background: #000;
    color: #fff;
  }
}

/* 偵測使用者偏好「減少動畫」（無障礙）*/
@media (prefers-reduced-motion: reduce) {
  * {
    animation: none;
    transition: none;
  }
}

/* 偵測是否真有 hover 能力（手機觸控沒 hover）*/
@media (hover: hover) {
  button:hover { background: blue; }
}

/* 列印用樣式 */
@media print {
  nav, footer { display: none; }
  body { font-size: 12pt; }
}

/* 偵測螢幕方向 */
@media (orientation: landscape) {
  .hero { height: 100vh; }
}
```

> 🚀 **`prefers-color-scheme` 是暗色模式的基石**——L13 詳教完整暗黑模式。**`prefers-reduced-motion` 是無障礙必做**——L22 詳教。

  
### 5. clamp()—— 流體排版神器

**`clamp(最小, 理想, 最大)`** = 一行解決響應式字級、不用 media query：

📄 **寫進程式檔**：`style.css`

```text
h1 {
  font-size: clamp(2rem, 5vw, 5rem);
  /* 
     最小 2rem（32px）
     理想 5vw（隨螢幕變）
     最大 5rem（80px）
  */
}

/* 解讀：
   螢幕 320px → 32px（卡在最小）
   螢幕 800px → 40px（5vw）
   螢幕 1920px → 80px（卡在最大）
*/
```

> 🚀 **`clamp()` 取代了 50% 的 media query**——字級、padding、寬度都可以用。**2026 業界主流寫法**：

📄 **寫進程式檔**：`style.css`

```text
.section {
  padding: clamp(1rem, 5vw, 4rem);
}

.container {
  width: clamp(320px, 90%, 1200px);
  margin: 0 auto;
}
```

  
### 6. Container Query（容器查詢）—— 2023 革命

**`@container`（容器查詢）** = 不看「視窗」多大、看「**容器**」多大。**讓元件真正可重用**。

```text
情境問題：
  同一個 .card 元件
  → 放在 sidebar（窄）時要直版
  → 放在 main（寬）時要橫版
  → 用 media query 做不到（同一個視窗）
```

📄 **寫進程式檔**：`style.css`

```text
/* 父容器宣告「我是查詢對象」*/
.card-wrapper {
  container-type: inline-size;
  container-name: card;
}

/* 子元素根據「父容器寬度」切換 */
@container card (min-width: 400px) {
  .card {
    display: grid;
    grid-template-columns: 1fr 2fr;
  }
}
```

> 💡 **Container Query 為什麼革命**：傳統 media query 是「全頁等寬」的思維、container query 是「**每個區域自己判斷**」——元件不管放哪都能自適應。**2023 後全瀏覽器支援**。

  
### 7. Mobile First 完整範例

📄 **寫進程式檔**：`style.css`

```text
/* 預設：手機 */
.cards {
  display: grid;
  grid-template-columns: 1fr;
  gap: 16px;
}

.card {
  padding: 16px;
  font-size: 14px;
}

/* 平板 */
@media (min-width: 768px) {
  .cards {
    grid-template-columns: 1fr 1fr;
    gap: 24px;
  }
  .card {
    padding: 24px;
    font-size: 16px;
  }
}

/* 桌面 */
@media (min-width: 1024px) {
  .cards {
    grid-template-columns: repeat(3, 1fr);
    gap: 32px;
  }
}

/* 大螢幕 */
@media (min-width: 1280px) {
  .cards {
    grid-template-columns: repeat(4, 1fr);
  }
}
```

  
### 8. 2026 新主流：less media query

**業界趨勢**：少寫 media query、多用 clamp + Grid auto-fit + container query：

📄 **寫進程式檔**：`style.css`

```text
/* 舊寫法（4 個 media query）*/
.cards { grid-template-columns: 1fr; }
@media (min-width: 640px)  { .cards { grid-template-columns: 1fr 1fr; } }
@media (min-width: 1024px) { .cards { grid-template-columns: repeat(3, 1fr); } }
@media (min-width: 1280px) { .cards { grid-template-columns: repeat(4, 1fr); } }

/* 新寫法（一行）*/
.cards {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
  gap: clamp(1rem, 3vw, 2rem);
}
/* → 自動填滿、自動換行、間距也響應式 */
```

> 🚀 **2026 響應式哲學**：「**少用 media query、多用 clamp + container query + Grid auto-fit**」——CSS 更乾淨、更穩定。可惜很多老教學還停留在「斷點切換」思維。

  
### 9. 響應式圖片

📄 **寫進程式檔**：`index.html`

```text
<picture>
  <!-- 手機載小圖 -->
  <source media="(max-width: 768px)" srcset="hero-mobile.jpg">
  <!-- 桌面載大圖 -->
  <source media="(min-width: 769px)" srcset="hero-desktop.jpg">
  <!-- 保底 -->
  <img src="hero.jpg" alt="Hero">
</picture>
```

或用 `srcset` + `sizes`：

```text
<img
  src="hero-800.jpg"
  srcset="hero-400.jpg 400w, hero-800.jpg 800w, hero-1600.jpg 1600w"
  sizes="(max-width: 600px) 100vw, 50vw"
  alt="Hero"
>
```

  
### 10. 測試 RWD

🖥️ **在瀏覽器測試**

```text
✅ Chrome / Edge → F12 → 點手機圖示（Device Toolbar）
✅ Firefox → F12 → 點螢幕圖示
✅ Safari → 偏好設定 → 進階 → 顯示開發選單 → 反應式設計模式

→ 可以模擬 iPhone / iPad / 各種尺寸
→ 改尺寸時即時看 layout 變化
```

  
### 5 大常見坑

```text
❌ 1. 沒寫 viewport meta
   → 手機把網頁當桌面版縮小顯示
   → 修：<head> 加 <meta name="viewport" ...>

❌ 2. 用 max-width 寫 desktop-first
   → CSS 重複多、覆蓋邏輯亂
   → 修：永遠 mobile-first（min-width）

❌ 3. 字級寫死 px、手機看太小
   → 修：用 rem 或 clamp() 響應式字級

❌ 4. 100vh 在手機底部被地址欄擋住
   → 修：用 100dvh 取代 100vh

❌ 5. 用 media query 做卡片網格
   → CSS 多 4-5 行
   → 修：Grid auto-fit + minmax 一行解決
```

  
### 🎯 重點回顧

- **viewport meta 必寫**——手機才會正確縮放
- **Mobile First**（min-width）——2026 業界主流
- **5 個標準斷點**——sm/md/lg/xl/2xl（Tailwind 規格）
- **`clamp()` 取代多數 media query**——一行響應式
- **`@container` 容器查詢**——元件真正可重用
- **`100dvh` 取代 `100vh`**——手機友善
- **Grid `auto-fit` + `minmax`** = 響應式神咒
