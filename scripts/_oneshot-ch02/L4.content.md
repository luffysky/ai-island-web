網頁上每個元素都是一個「**盒子**」。**box model（盒模型）** = 這個盒子怎麼計算大小、由四層構成：**content（內容）/ padding（內距）/ border（邊框）/ margin（外距）**。

> 💡 **這課的價值**：新手最常踩「`width: 200px` 為什麼實際變 240px」這種坑——都是盒模型沒搞懂。**這課學完 = 寬高算法永遠不會錯**。

  
### 1. 四層結構視覺化

```text
┌─────────────────────────────────────┐
│         margin（外距）              │
│  ┌───────────────────────────────┐  │
│  │       border（邊框）          │  │
│  │  ┌─────────────────────────┐  │  │
│  │  │     padding（內距）     │  │  │
│  │  │  ┌───────────────────┐  │  │  │
│  │  │  │     content       │  │  │  │
│  │  │  │   width × height  │  │  │  │
│  │  │  └───────────────────┘  │  │  │
│  │  └─────────────────────────┘  │  │
│  └───────────────────────────────┘  │
└─────────────────────────────────────┘
```

**4 層的意義**：
- **content（內容）**—— 真正的東西（文字、圖片）、`width` × `height` 控制
- **padding（內距）**—— content 跟 border 之間的「內部空間」
- **border（邊框）**—— 元素的外框
- **margin（外距）**—— 元素跟外面其他元素的「外部空間」

> 💡 **白話比喻**：盒子 = **手機禮盒**。content（手機本體）→ padding（盒子裡的泡棉）→ border（紙盒外殼）→ margin（這盒禮物跟下一盒之間的距離）。

  
### 2. 兩種計算模式（box-sizing）

**預設**：`width: 200px` **只算 content**——加上 padding / border 後實際更寬。
**現代寫法**：`box-sizing: border-box` 讓 `width` **包含 padding + border**。

📄 **寫進程式檔**：`style.css`

```text
/* 預設：content-box（width 只算內容）*/
.box1 {
  width: 200px;
  padding: 20px;
  border: 5px solid;
  /* 實際寬度 = 200 + 20*2 + 5*2 = 250px */
}

/* border-box（width 包含 padding + border）*/
.box2 {
  box-sizing: border-box;
  width: 200px;
  padding: 20px;
  border: 5px solid;
  /* 實際寬度 = 200px */
  /* 內容被擠到 200 - 20*2 - 5*2 = 150px */
}
```

> ✨ **接案小技巧**：每個專案開頭一定寫這條：`* { box-sizing: border-box; }`——讓所有元素都用 border-box、`width` 設多少就是多少、省下無數計算地獄。**所有現代 CSS 框架都這樣設**（Bootstrap、Tailwind、shadcn）。

  
### 3. margin / padding 速寫法

📄 **寫進程式檔**：`style.css`

```text
/* 一個值：四邊都套用 */
.a { margin: 20px; }
/* → 上下左右都 20px */

/* 兩個值：上下、左右 */
.b { margin: 10px 20px; }
/* → 上下 10px、左右 20px */

/* 三個值：上、左右、下 */
.c { margin: 10px 20px 30px; }

/* 四個值：上、右、下、左（順時針）*/
.d { margin: 10px 20px 30px 40px; }

/* 分開寫 */
.e {
  margin-top: 10px;
  margin-right: 20px;
  margin-bottom: 30px;
  margin-left: 40px;
}
```

> 💡 **背口訣**：四個值的順序「**上右下左**」（順時針從上開始）、padding 也一樣。

  
### 4. margin 折疊（Margin Collapsing）—— 經典踩雷

兩個垂直相鄰的元素 margin **會合併**、取較大值：

```text
情境：
  <div style="margin-bottom: 30px;">上面</div>
  <div style="margin-top: 20px;">下面</div>

直覺以為：30 + 20 = 50px 間距
實際：取大的 = 30px 間距（折疊了）

→ 這就是 margin collapsing
```

**解法**：用 flexbox / grid 的 `gap`、不要靠 margin：

📄 **寫進程式檔**：`style.css`

```text
.container {
  display: flex;
  flex-direction: column;
  gap: 20px;  /* ← 永遠精準、不折疊 */
}
```

  
### 5. 負 margin（重疊技巧）

margin 可以是負值、做重疊效果：

📄 **寫進程式檔**：`style.css`

```text
.card {
  margin-top: -20px;
  /* → 往上拉 20px、跟上面元素重疊 */
}

/* 例：卡片往 hero 圖上拉 */
.hero { height: 400px; }
.cards {
  margin-top: -80px;  /* 往上拉 80px、重疊在 hero 下緣 */
  position: relative;
  z-index: 1;
}
```

  
### 6. CSS 單位完整對照

CSS 有十幾種長度單位、各有用途：

| 單位 | 意義 | 建議場景 |
|---|---|---|
| `px` | 絕對像素 | 邊框、陰影、固定大小 |
| `%` | 父元素百分比 | 寬度、容器 |
| `em` | 當前元素字級倍數 | 很少用、會疊乘 |
| `rem` | 根元素（html）字級倍數 | **字級、間距首選** |
| `vw` / `vh` | 視窗寬 / 高的 1% | 滿版 hero、字級縮放 |
| `dvh` | 動態視窗高 | 手機（避免地址欄問題） |
| `svh` / `lvh` | 小 / 大視窗高 | 進階手機 |
| `ch` | 字元 "0" 的寬度 | 限制文章寬度（70ch 最舒服） |
| `fr` | Grid 的份數 | Grid 排版（L7 詳教） |

> 💡 **新手三選一就夠**：(1) **`px`** 邊框、陰影、小裝飾 (2) **`rem`** 字級、間距 (3) **`%` / `fr`** 排版寬度。

  
### 7. dvh vs vh（2024 後重要）

手機瀏覽器有「**地址欄會收合**」的問題——`vh`（視窗高）算的是「最高時」、會讓內容溢出。

```text
舊寫法：
  .hero { height: 100vh; }
  → 手機上會超出可視範圍（地址欄收合後底部被截）

新寫法（2024+ 全瀏覽器支援）：
  .hero { height: 100dvh; }
  → dvh = dynamic viewport height
  → 自動跟著實際可視高度動態變
```

> 🚀 **進階技巧**：2026 手機高度單位選擇——用 `dvh` 不用 `vh`。`100dvh` 是滿版的正確寫法。

  
### 8. 實戰：標準卡片

📄 **寫進程式檔**：`style.css`

```text
* { box-sizing: border-box; }  /* 全域 border-box */

.card {
  width: 300px;
  padding: 24px;
  border: 1px solid #e0e0e0;
  border-radius: 8px;
  margin: 16px;
  background: white;
}

.card h2 {
  margin: 0 0 12px;  /* 標題下方 12px 間距 */
}

.card p {
  margin: 0;
  line-height: 1.6;
  color: #555;
}
```

  
### 5 大常見坑

```text
❌ 1. 沒設 box-sizing: border-box
   → width: 200px 加 padding 後變 250px、版面亂掉
   → 修：專案開頭加 * { box-sizing: border-box; }

❌ 2. margin 折疊以為是 bug
   → 上 30px + 下 20px = 預期 50px、實際 30px
   → 修：用 gap 取代 margin（flex / grid）

❌ 3. 用 vh 在手機上裂開
   → 地址欄收合後內容超出
   → 修：用 dvh 取代 vh

❌ 4. em 疊乘導致字越來越大（或小）
   → 父 1.2em、子 1.2em、孫 1.2em → 1.728 倍
   → 修：字級用 rem（永遠以根元素為準）

❌ 5. 用 margin 撐高度
   → 折疊問題 + 不精準
   → 修：用 padding 或 min-height
```

  
### 🎯 重點回顧

- **盒模型 4 層**：content / padding / border / margin
- **預設 `width` 只算 content**——加 padding / border 會變更寬
- **`box-sizing: border-box` 是現代寫法**——每個專案開頭都寫
- **margin 折疊**——垂直相鄰取大值、用 `gap` 解決
- **單位三選一**：`px`（裝飾）/ `rem`（字級間距）/ `%`（排版）
- **`dvh` 取代 `vh`**——手機友善

> 💡 **下一課（L5）**：盒子是骨架、`display` 決定盒子的「人格」——是塊狀（獨佔一行）、行內（同一行）、還是 flex / grid 容器？**display 一個屬性、改變整個排版邏輯**。
