CSS 顏色不只有 `#FFFFFF`——從命名顏色到 **hex / rgb / hsl / oklch**、再加上漸層、混色。**2024 後業界改用 oklch 是有原因的**——這課把顏色系統一次講透。

> 💡 **這課的價值**：學會所有顏色寫法 = **配色變直覺**（不再瞎調 RGB 數字）+ **暗黑模式 / 主題切換變超簡單** + **跟設計師溝通有共同詞彙**。

  
### 1. 所有顏色寫法總覽

📄 **寫進程式檔**：`style.css`

```text
.a { color: red; }                       /* 1. 命名顏色（140 種）*/
.b { color: #ff0000; }                   /* 2. HEX（十六進位）*/
.c { color: #f00; }                      /* 3. HEX 簡寫 */
.d { color: #ff0000aa; }                 /* 4. HEX + alpha（透明度）*/
.e { color: rgb(255, 0, 0); }            /* 5. RGB（紅綠藍）*/
.f { color: rgba(255, 0, 0, 0.5); }      /* 6. RGB + alpha */
.g { color: rgb(255 0 0 / 50%); }        /* 7. RGB 新語法 */
.h { color: hsl(0, 100%, 50%); }         /* 8. HSL（最直覺）*/
.i { color: hwb(0 0% 0%); }              /* 9. HWB */
.j { color: lab(53% 80 67); }            /* 10. LAB */
.k { color: oklch(0.7 0.2 30); }         /* 11. OKLCH（2024 新寵）*/
```

  
### 2. HEX（十六進位）—— 最常見

**HEX（hexadecimal、十六進位）** = `#` 後面 6 位數、每兩位代表紅綠藍（00-FF）：

```text
#FF0000 = 紅 255 + 綠 0 + 藍 0 = 純紅
#00FF00 = 綠
#0000FF = 藍
#FFFFFF = 白（三色全滿）
#000000 = 黑（三色全空）
#808080 = 灰

簡寫（3 位數）：
#F00 = #FF0000
#0F0 = #00FF00
#FFF = #FFFFFF
```

**加 alpha（透明度）**：

```text
#FF0000AA
        ↑
   兩位數透明度
   FF = 100% 不透明
   80 = 50% 透明
   00 = 完全透明
```

> 💡 **設計師最常用 HEX**——Figma / Sketch 預設都是 HEX。**前端要熟到看 `#3b82f6` 就知道是藍色**。

  
### 3. RGB / RGBA

**RGB（Red Green Blue）** = 紅綠藍三色光、每個 0-255：

📄 **寫進程式檔**：`style.css`

```text
/* 舊語法 */
.a { color: rgb(255, 0, 0); }
.b { color: rgba(255, 0, 0, 0.5); }

/* 2021 新語法（推薦）*/
.c { color: rgb(255 0 0); }
.d { color: rgb(255 0 0 / 50%); }
/* → 沒逗號、用斜線分透明度 */
```

> 💡 **HEX vs RGB**：兩者完全等價、`#FF0000` = `rgb(255 0 0)`。**唯一差別**：RGB 比較好「動態運算」（例：JS 算出來的顏色塞進去）。

  
### 4. HSL —— 最直覺的配色法

**HSL（Hue Saturation Lightness、色相飽和度明度）** = **設計師思考方式的顏色系統**：

```text
H（Hue、色相）：0-360 度
  0 = 紅
  60 = 黃
  120 = 綠
  180 = 青
  240 = 藍
  300 = 紫
  360 = 紅（轉一圈回來）

S（Saturation、飽和度）：0%-100%
  0% = 完全灰
  100% = 最鮮豔

L（Lightness、明度）：0%-100%
  0% = 黑
  50% = 純色
  100% = 白
```

📄 **寫進程式檔**：`style.css`

```text
/* 想要紫色但較淡？改 L 就好、不用整個換 */
.a { color: hsl(270, 80%, 50%); }  /* 純紫 */
.b { color: hsl(270, 80%, 80%); }  /* 同色淡版 */
.c { color: hsl(270, 80%, 20%); }  /* 同色深版 */
```

> 💡 **HSL 為什麼直覺**：想做「同色不同明暗」只要改一個 L 值——做按鈕的 normal / hover / active 三態超簡單。**HEX / RGB 做不到、得整個重算**。

  
### 5. OKLCH —— 2024 業界新標準

**OKLCH（OK Lightness Chroma Hue）** = 升級版 HSL、解決 HSL 的「**亮度不一致**」問題。

```text
HSL 的問題：
  hsl(60, 100%, 50%)  → 黃色、看起來很亮
  hsl(240, 100%, 50%) → 藍色、看起來很暗
  → 同樣 L=50%、亮度差很多

OKLCH 解決：
  oklch(0.7 0.2 30)   → 橘
  oklch(0.7 0.2 270)  → 紫
  → 兩個看起來一樣亮！「感知均勻」
```

📄 **寫進程式檔**：`style.css`

```text
.brand-orange { color: oklch(0.7 0.2 30); }
.brand-purple { color: oklch(0.7 0.2 270); }
/* → L=0.7 都是 0.7、看起來真的一樣亮 */

/* 主題色階產生（從淺到深）*/
.brand-100 { color: oklch(0.95 0.05 240); }  /* 最淺 */
.brand-500 { color: oklch(0.65 0.2 240); }   /* 中間 */
.brand-900 { color: oklch(0.3 0.15 240); }   /* 最深 */
```

> 🚀 **誰在用 OKLCH**：**Tailwind v4 / Linear / Vercel / Apple HIG / shadcn 都改用 OKLCH**。2024 後新專案首選。L = 0~1（不是 0-100%）、C = chroma（飽和度）、H = hue（色相）。

  
### 6. 漸層（Gradient）

**linear-gradient / radial-gradient / conic-gradient** 三種：

📄 **寫進程式檔**：`style.css`

```text
/* 線性漸層（最常用）*/
.a { background: linear-gradient(to right, red, blue); }
.b { background: linear-gradient(45deg, red, orange, yellow); }
.c { background: linear-gradient(to bottom, #fff, transparent); }

/* 徑向漸層（圓形）*/
.d { background: radial-gradient(circle at center, white, black); }

/* 圓錐漸層（可以做圓餅圖）*/
.e { background: conic-gradient(red, yellow, green, red); }

/* 進階：硬切（做條紋）*/
.f {
  background: linear-gradient(
    90deg,
    red 0 50%,
    blue 50% 100%
  );
  /* → 左半紅、右半藍、中間沒過渡 */
}
```

  
### 7. color-mix（2023 神技）

**`color-mix()`** = 直接在 CSS 裡混合兩個顏色：

📄 **寫進程式檔**：`style.css`

```text
/* 混合 50% 紅 + 50% 藍 = 紫 */
.a {
  background: color-mix(in srgb, red 50%, blue);
}

/* 動態：主題色 + 黑 = hover 色 */
.btn:hover {
  background: color-mix(in oklch, var(--primary) 80%, black);
}
/* → 一個變數產出整套色階 */
```

> 🚀 **color-mix 的威力**：(1) **暗色版 hover** 不用手動算 (2) **半透明色** 不用算 alpha (3) **動態主題** 一個變數延伸出整套色階。**2023 後全瀏覽器支援**。

  
### 8. 透明度（opacity）

**`opacity`** = 整個元素的透明度（含子元素）、`alpha` = 只有顏色的透明度：

📄 **寫進程式檔**：`style.css`

```text
.a {
  opacity: 0.5;
  /* → 整個元素（含子元素文字）都半透明 */
}

.b {
  background: rgba(0, 0, 0, 0.5);
  color: white;
  /* → 只有背景半透明、文字完全不透明 */
}
```

> 💡 **怎麼選**：要「整個淡化」用 `opacity`、要「文字清楚但背景半透明」用 `rgba` / `oklch` 帶 alpha。**毛玻璃卡片永遠用後者**。

  
### 9. 實戰：完整顏色系統（用變數）

📄 **寫進程式檔**：`style.css`

```text
:root {
  /* 品牌色（OKLCH 色階）*/
  --brand-50:  oklch(0.97 0.02 240);
  --brand-100: oklch(0.93 0.05 240);
  --brand-500: oklch(0.65 0.2 240);
  --brand-900: oklch(0.3 0.15 240);
  
  /* 灰階 */
  --gray-50:  oklch(0.98 0 0);
  --gray-500: oklch(0.6 0 0);
  --gray-900: oklch(0.2 0 0);
  
  /* 語意色 */
  --color-primary: var(--brand-500);
  --color-bg:      var(--gray-50);
  --color-text:    var(--gray-900);
}

.btn {
  background: var(--color-primary);
  color: white;
}

.btn:hover {
  background: color-mix(in oklch, var(--color-primary) 90%, black);
}
```

  
### 5 大常見坑

```text
❌ 1. 一直用命名顏色（red / blue）
   → 不精準、跟設計稿不符
   → 修：用 HEX 或 OKLCH

❌ 2. opacity 用在背景、結果文字也半透明
   → opacity 影響整個元素含子元素
   → 修：背景半透明用 rgba 或 oklch + /alpha

❌ 3. HSL 配色時各色亮度不一致
   → HSL 的 L 不是「感知亮度」
   → 修：改用 OKLCH

❌ 4. hover 色手動算半天
   → 修：用 color-mix(in oklch, var(--primary) 80%, black)

❌ 5. 顏色寫死在每個元素
   → 改色要改 100 處
   → 修：用 CSS 變數集中管理（L13 詳教）
```

  
### 🎯 重點回顧

- **HEX**（`#FF0000`）—— 設計師標配、最常見
- **RGB**（`rgb(255 0 0)`）—— 動態運算用
- **HSL**（`hsl(0, 100%, 50%)`）—— 直覺、但亮度不均
- **OKLCH**（`oklch(0.7 0.2 30)`）—— **2024 新標準**、感知均勻
- **`linear / radial / conic-gradient`**—— 三種漸層
- **`color-mix()`**—— 2023 神技、動態混色
- **`opacity` vs `alpha`** —— 整體淡化 vs 單純顏色透明
