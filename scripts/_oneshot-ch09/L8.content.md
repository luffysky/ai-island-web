**Vue Router = Vue 官方的 routing（路由、決定哪個網址顯示哪個畫面）解法**。SPA（Single Page Application、單頁應用）只有一個 HTML、所有切頁靠 JS 操作 DOM——router 就是負責「網址變了 → 切到對應元件」的那層。學會 router、你才能做有多頁面、能分享網址、瀏覽器上一頁能用的真正應用。

> 💡 **這課的價值**：搞懂 createRouter、router-link、route guard（路由守衛）、巢狀路由、lazy load—— 這些是「能上線的 Vue 應用」必備技能。

  
### 1. 為什麼需要 router？

**沒 router 的 SPA**：
- 全部畫面用 `v-if="page === 'home'"` 切換
- 網址永遠 `/`、按重新整理畫面跳回首頁
- 不能分享「我看的這頁」給朋友
- 瀏覽器上一頁 / 下一頁不能用

**Vue Router 解法**：
- 網址 `/about` 顯示 About.vue
- 網址 `/user/123` 顯示 User.vue 並把 123 當參數
- 上一頁 / 下一頁 / 重新整理全部正常
- URL 可以分享、可以收藏

```text
傳統網站              SPA + Router
──────────           ────────────
點連結 → 切頁         點連結 → router 切元件、URL 變
請求新 HTML           不請求新 HTML（無重新整理白屏）
慢                    秒切
SEO 容易              SEO 要靠 SSR（下課 Nuxt 講）
```

  
### 2. 安裝 + 第一個 router

```text
🖥️ 在終端機輸入

pnpm add vue-router@4
```

`npm create vue@latest` 開的專案勾「Add Vue Router」就已經裝好。

```text
📄 寫進程式檔：`src/router/index.ts`

import { createRouter, createWebHistory } from "vue-router";
import Home from "@/views/Home.vue";
import About from "@/views/About.vue";

const router = createRouter({
  // history mode：用 HTML5 History API（網址漂亮、要 server 配合）
  history: createWebHistory(),

  routes: [
    { path: "/", name: "home", component: Home },
    { path: "/about", name: "about", component: About }
  ]
});

export default router;
```

```text
📄 寫進程式檔：`src/main.ts`

import { createApp } from "vue";
import router from "./router";
import App from "./App.vue";

const app = createApp(App);
app.use(router);
app.mount("#app");
```

```text
📄 寫進程式檔：`src/App.vue`

<template>
  <nav>
    <router-link to="/">首頁</router-link>
    <router-link to="/about">關於</router-link>
  </nav>

  <main>
    <router-view />              <!-- ← router 把對應元件渲染在這 -->
  </main>
</template>
```

**`<router-link to="/about">` vs `<a href="/about">`**：
- `<a>` 會請求新 HTML、整頁刷新（慢、SPA state 全失去）
- `<router-link>` 只切換 router 內部元件、URL 變但不刷新（快、爽）

  
### 3. History mode（網址漂亮）vs Hash mode

```text
createWebHistory()       → /about              (要 server 配合)
createWebHashHistory()   → /#/about            (任何 server 都能用)
```

**`createWebHistory`（推薦）**——網址正常、SEO 友善、但要 server 配合（任何路徑都吐 `index.html`）。Vercel / Netlify / Vite preview / Cloudflare Pages 預設都已設好。

**`createWebHashHistory`**——URL 有 `#`、有點醜、但靜態 server 也能跑（GitHub Pages、單純的 nginx 不設 rewrite）。

  
### 4. 動態參數（`:id` 抓 URL 變數）

```text
📄 router/index.ts

{
  path: "/user/:id",
  name: "user",
  component: () => import("@/views/User.vue")
}
```

```text
📄 src/views/User.vue

<script setup>
import { useRoute } from "vue-router";

const route = useRoute();
const userId = route.params.id;            // 抓 :id 的值
const tab = route.query.tab;               // 抓 ?tab=xxx 的值
const hash = route.hash;                   // 抓 #section 的值
</script>

<template>
  <h1>User #{{ route.params.id }}</h1>
</template>
```

**例子**：
- 網址 `/user/42` → `route.params.id === "42"`
- 網址 `/user/42?tab=info` → `route.query.tab === "info"`
- 網址 `/user/42#bio` → `route.hash === "#bio"`

  
### 5. Lazy loading（懶載入）—— SPA 必做的效能優化

**問題**：100 頁的應用、初始載入要下載 100 頁的 JS、首頁開超慢。

**解法**：用 `() => import("...")` 把每頁拆成獨立 chunk、需要才下載：

```text
{
  path: "/about",
  component: About                              // ← 馬上載入（in bundle）
},
{
  path: "/admin",
  component: () => import("@/views/Admin.vue")  // ← 用到才載入（separate chunk）
}
```

**Vite 編譯後產生**：
- `index.js` —— 主 bundle、包首頁
- `Admin-abc123.js` —— Admin 頁獨立 chunk

使用者去 `/admin` 才下載 `Admin-abc123.js`、首頁不用等。

> 💡 **2026 慣例**：除了首頁、**所有頁面都 lazy load**。寫法統一 `component: () => import("...")`。

  
### 6. 程式化導航（用 code 換頁）

不一定要 `<router-link>`——也能在 JS 內呼叫 router：

```text
<script setup>
import { useRouter, useRoute } from "vue-router";

const router = useRouter();
const route = useRoute();

function goHome() {
  router.push("/");                              // 推一個新頁、上一頁能回來
}

function goUser(id: number) {
  router.push({ name: "user", params: { id } });  // 用 named route
}

function goLogin() {
  router.push({ path: "/login", query: { redirect: route.fullPath } });
}

function goBack() {
  router.back();                                  // 上一頁
}

function replace() {
  router.replace("/login");                       // 不留 history（上一頁無法回來）
}
</script>
```

**最常用 4 招**：
- `router.push(path)` —— 換頁、留 history
- `router.replace(path)` —— 換頁、不留 history（登入後跳首頁愛用）
- `router.back()` —— 上一頁
- `router.go(-2)` —— 倒回 2 頁

  
### 7. 巢狀路由（nested route）

**場景**：後台佈局——左邊 sidebar 永遠在、右邊內容隨頁切換。

```text
📄 router/index.ts

{
  path: "/admin",
  component: () => import("@/layouts/AdminLayout.vue"),
  meta: { requiresAuth: true },
  children: [
    { path: "", component: () => import("@/views/AdminHome.vue") },      // /admin
    { path: "users", component: () => import("@/views/AdminUsers.vue") },// /admin/users
    { path: "settings", component: () => import("@/views/AdminSettings.vue") }
  ]
}
```

```text
📄 AdminLayout.vue

<template>
  <div class="admin">
    <aside>
      <router-link to="/admin">總覽</router-link>
      <router-link to="/admin/users">使用者</router-link>
      <router-link to="/admin/settings">設定</router-link>
    </aside>

    <main>
      <router-view />              <!-- ← 子路由在這渲染 -->
    </main>
  </div>
</template>
```

  
### 8. Route Guard（路由守衛）—— 擋未登入

**guard（守衛）—— 在切頁前 / 切頁後執行的攔截器**——最常用：「進這頁要先登入」。

```text
📄 router/index.ts

import { useUserStore } from "@/stores/user";

router.beforeEach((to, from, next) => {
  const userStore = useUserStore();

  if (to.meta.requiresAuth && !userStore.isLoggedIn) {
    // 未登入、踢到登入頁、帶上原本目的地
    next({ path: "/login", query: { redirect: to.fullPath } });
  } else {
    next();              // 放行
  }
});
```

```text
📄 routes 內標記要登入的頁

{ path: "/admin", meta: { requiresAuth: true }, ... }
{ path: "/profile", meta: { requiresAuth: true }, ... }
{ path: "/login", meta: { requiresAuth: false }, ... }
```

**3 種 guard**：
- `router.beforeEach` —— 全域、切頁前（最常用、擋登入）
- `router.afterEach` —— 全域、切頁後（送 analytics、改 title）
- `beforeEnter` —— 寫在 route 內、只擋這條 route

  
### 9. 404 頁（catch-all）

```text
{
  path: "/:pathMatch(.*)*",
  name: "not-found",
  component: () => import("@/views/NotFound.vue")
}
```

`:pathMatch(.*)*` 是「catch any path」、永遠放最後一條（最後 fallback）。

  
### 10. 完整 router 範例（可上線等級）

```text
📄 src/router/index.ts

import { createRouter, createWebHistory } from "vue-router";
import { useUserStore } from "@/stores/user";

const router = createRouter({
  history: createWebHistory(),
  routes: [
    { path: "/", name: "home", component: () => import("@/views/Home.vue") },
    { path: "/login", name: "login", component: () => import("@/views/Login.vue") },
    { path: "/user/:id", name: "user", component: () => import("@/views/User.vue") },
    {
      path: "/admin",
      component: () => import("@/layouts/Admin.vue"),
      meta: { requiresAuth: true, role: "admin" },
      children: [
        { path: "", component: () => import("@/views/AdminHome.vue") },
        { path: "users", component: () => import("@/views/AdminUsers.vue") }
      ]
    },
    { path: "/:pathMatch(.*)*", component: () => import("@/views/NotFound.vue") }
  ]
});

router.beforeEach((to) => {
  const user = useUserStore();
  if (to.meta.requiresAuth && !user.isLoggedIn) {
    return { path: "/login", query: { redirect: to.fullPath } };
  }
  if (to.meta.role === "admin" && user.role !== "admin") {
    return { path: "/403" };
  }
});

router.afterEach((to) => {
  document.title = `${to.meta.title || "AI 島"} — AI 島`;
});

export default router;
```

  
### 11. 5 大常見坑

```text
❌ 1. 換頁用 <a href="/about">
   → 整頁刷新、SPA state 全失去
   → 修：用 <router-link to="/about">

❌ 2. createWebHistory 部署後 /about 直接打 404
   → server 不知道要吐 index.html
   → 修：Vercel / Netlify 預設已處理、自架要設 rewrite

❌ 3. route.params.id 拿到時是 string、不是 number
   → 拿去算數會失敗
   → 修：const id = Number(route.params.id)

❌ 4. guard 內用 useUserStore() 報錯
   → Pinia 還沒掛載
   → 修：guard 必須在 createApp(...).use(pinia).use(router) 之後

❌ 5. 切回同個 route 但 params 不同、元件不重新建立
   → 點 /user/1 → /user/2、User.vue 不會重跑 onMounted
   → 修：用 watch(() => route.params.id) 或 :key="$route.fullPath"
```

  
### 🎯 重點回顧

- **Vue Router** 是 Vue 官方 routing 解法、SPA 必裝
- **`createRouter` + `createWebHistory`** 是 2026 標準寫法
- **`<router-link>` + `<router-view>`** 在 template 用、`useRouter()` + `useRoute()` 在 script 用
- **Lazy load 用 `() => import()`**、首頁外全 lazy
- **巢狀路由用 `children`** + 父 layout 內放 `<router-view>`
- **`router.beforeEach`** 擋未登入、最常用
- **404 用 `/:pathMatch(.*)*`** 放最後
