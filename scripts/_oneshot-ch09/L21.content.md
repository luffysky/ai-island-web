**Vue 樣式（styling）= 3 種主流寫法 + 1 個神功能**——Scoped CSS（內建、最常用）、CSS Modules（class 自動 hash）、Tailwind（utility-first、社群最熱）。再加上 Vue 獨有的 **`v-bind in CSS`**（CSS 內動態綁 JS 變數、其他框架沒這招）。這課幫你選對寫法、不混搭翻車。

> 💡 **這課的價值**：90% 場景用 Scoped + Tailwind 就完美——但你要知道每種怎麼寫、何時用、坑在哪。看完你能設計團隊統一的樣式策略、不再亂混 5 種寫法。

  
### 1. Scoped CSS（內建、最常用）

**Scoped CSS** = Vue SFC 原生支援、`<style scoped>` 就 enable、樣式只影響當前元件、不污染外面：

```text
📄 寫進程式檔：`src/components/MyButton.vue`

<template>
  <button class="btn">送出</button>
</template>

<style scoped>
.btn {
  background: #42b883;
  color: white;
  padding: 8px 16px;
  border: none;
  border-radius: 4px;
  cursor: pointer;
}

.btn:hover {
  background: #369870;
}
</style>
```

**Vue 編譯後變成**：

```text
<button class="btn" data-v-abc123>送出</button>

<style>
.btn[data-v-abc123] {
  background: #42b883;
  /* ... */
}
</style>
```

**原理**：Vue 給每個元件一個唯一的 `data-v-xxx` 屬性、CSS selector 自動加上 `[data-v-xxx]` 限制——**只有這個元件的 button 會套到、別元件的 `.btn` 不受影響**。

> 💡 **Scoped 是 Vue 招牌**——React 沒原生 scoped、要靠 CSS Modules / styled-components / tailwind 解決。Vue 直接內建。

  
### 2. CSS Modules（class 自動 hash）

**`<style module>`** = class name 自動 hash 化、零衝突風險：

```text
<template>
  <button :class="$style.btn">送出</button>
</template>

<style module>
.btn {
  background: #42b883;
  color: white;
}
</style>
```

**編譯後**：

```text
<button class="MyButton_btn_a8f3k">送出</button>

<style>
.MyButton_btn_a8f3k {
  background: #42b883;
}
</style>
```

**Scoped vs Modules**：

```text
Scoped
─────
class name 不變（.btn 還是 .btn）
靠 data-v 屬性 scope
✅ DevTools 看 class 名直觀
✅ 可以從外部用 :deep() 穿透改子元件樣式
適合：99% 場景

Modules
─────
class name 改成 hash
靠 import 物件 ($style.btn) 用
✅ 100% 零衝突
❌ DevTools 看到難認
適合：大型 design system / 怕同名衝突的場景
```

**社群選擇**：**Scoped 是 80% 開發者首選**——簡單夠用。

  
### 3. Tailwind CSS（2026 主流推薦）

**Tailwind** = utility-first（功能優先）的 CSS 框架——直接在 HTML 裡寫 class、不用切去 CSS 檔。**Vue 社群這 2 年大量採用**。

```text
🖥️ 安裝（Vite Vue 專案）

npm install -D tailwindcss@latest postcss autoprefixer
npx tailwindcss init -p

# Nuxt 專案
npx nuxi@latest module add tailwindcss
```

```text
📄 src/main.css（import 在 main.ts）

@tailwind base;
@tailwind components;
@tailwind utilities;
```

```text
📄 用：MyButton.vue

<template>
  <button class="bg-green-500 hover:bg-green-600 text-white py-2 px-4 rounded
                 transition-colors disabled:opacity-50">
    送出
  </button>
</template>

<!-- 完全不用寫 <style> -->
```

**Tailwind 好處**：
- 不用想 class 名（不用糾結叫 `.btn-primary` 還是 `.button-main`）
- 樣式跟 HTML 在一起、改 UI 不用切檔
- 沒用到的 class build 時自動移除（極小 CSS）
- 設計系統一致（顏色 / 間距 / 字體都從 theme 來）

**Tailwind 壞處**：
- HTML 變很長（一個 button 一堆 class）
- 學一套 class 名稱（要查文件）
- 不熟的人覺得醜

**搭配 Scoped 用最香**：

```text
<template>
  <!-- 主要用 Tailwind -->
  <button class="bg-green-500 hover:bg-green-600 px-4 py-2 rounded">
    <span class="special-icon">⭐</span> 送出
  </button>
</template>

<style scoped>
/* 少量自訂、Tailwind 沒有的東西 */
.special-icon {
  animation: spin 2s linear infinite;
}
</style>
```

  
### 4. 動態 class / style binding（不用寫 CSS、用 JS 條件決定）

```text
<script setup>
import { ref, computed } from "vue";
const isActive = ref(true);
const isDanger = ref(false);
const textColor = ref("red");
const fontSize = ref(16);
</script>

<template>
  <!-- 物件語法：true 才加上 -->
  <div :class="{ active: isActive, danger: isDanger }">
    內容
  </div>

  <!-- 陣列語法：常駐 class + 動態 class -->
  <div :class="['card', isActive && 'card-active']">內容</div>

  <!-- 三元 -->
  <div :class="isActive ? 'on' : 'off'">內容</div>

  <!-- 跟 Tailwind 搭配 -->
  <button :class="[
    'px-4 py-2 rounded',
    isActive ? 'bg-green-500' : 'bg-gray-300'
  ]">
    按鈕
  </button>

  <!-- inline style -->
  <p :style="{ color: textColor, fontSize: fontSize + 'px' }">紅字</p>
</template>
```

  
### 5. `:deep()` —— 穿透 scoped 改子元件樣式

**問題**：Scoped 不能改子元件的 class（因為 data-v 不同）。

**解法**：用 `:deep()`：

```text
<template>
  <div class="parent">
    <ChildComponent />        <!-- 想改它內部的 .child-btn -->
  </div>
</template>

<style scoped>
.parent :deep(.child-btn) {
  color: red;
}

/* 舊寫法（已 deprecate） */
/* .parent /deep/ .child-btn { color: red } */
/* .parent >>> .child-btn { color: red } */
</style>
```

> 💡 **`:deep()` 用在「父想 override 子元件樣式」**——例如套第三方 UI library（Element Plus）想改它預設色。但**不要濫用**——破壞 scoped 隔離、子元件改名你父就壞。

  
### 6. `v-bind in CSS` —— Vue 獨家神功能（Vue 3.2+）

**問題**：想根據 JS 變數動態改 CSS（顏色、大小、寬度）——傳統做法用 inline style 或寫一堆 class。

**Vue 神招**：**直接在 `<style>` 內 `v-bind(jsVar)`**：

```text
<script setup>
import { ref } from "vue";

const primaryColor = ref("#42b883");
const borderRadius = ref("8px");
</script>

<template>
  <button class="btn">動態色按鈕</button>
  <input type="color" v-model="primaryColor" />
</template>

<style scoped>
.btn {
  background: v-bind(primaryColor);          /* ← 動態！ */
  border-radius: v-bind(borderRadius);
  padding: 10px 20px;
  border: none;
  color: white;
}
</style>
```

**`v-bind` 在 CSS 內**——使用者改顏色、按鈕背景立刻變、不用寫 `:style`、不用三元、樣式檔保持乾淨。**React 沒這招**。

**v-bind 物件屬性**：

```text
<script setup>
const theme = ref({ primary: "#42b883", radius: "8px" });
</script>

<style scoped>
.btn {
  background: v-bind("theme.primary");
  border-radius: v-bind("theme.radius");
}
</style>
```

  
### 7. SCSS / PostCSS / Sass

**SCSS（Sassy CSS）** = CSS 的超集、加變數、巢狀、mixin、繼承等高階功能：

```text
🖥️ 在終端機輸入

pnpm add -D sass
```

```text
<style scoped lang="scss">
$primary: #42b883;
$radius: 8px;

.btn {
  background: $primary;
  border-radius: $radius;
  padding: 10px 20px;

  &:hover {                     /* & 是父選擇器 */
    background: darken($primary, 10%);
  }

  &.large {
    padding: 16px 32px;
  }

  .icon {
    margin-right: 8px;
  }
}
</style>
```

**現在 SCSS 大部分被 Tailwind 取代**——但傳統公司 / 老專案還很多。

  
### 8. 全域 CSS（少數場景）

有些東西「整個 App 共用」——reset、字型、CSS 變數：

```text
📄 src/assets/global.css

:root {
  --color-primary: #42b883;
  --color-text: #2c3e50;
  --color-bg: #ffffff;
}

* {
  box-sizing: border-box;
  margin: 0;
  padding: 0;
}

body {
  font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif;
}

/* 暗色模式 */
[data-theme="dark"] {
  --color-bg: #1a1a1a;
  --color-text: #f5f5f5;
}
```

```text
📄 src/main.ts

import "./assets/global.css";    // ← import 一次、整 App 生效
```

  
### 9. 推薦組合（2026 best practice）

```text
🟢 純前端 / 個人 SaaS / 接案（90% 場景）
   Tailwind + Scoped CSS（少量自訂）
   → 開發快、CSS 小、團隊一致

🟢 大型 design system / Storybook
   Tailwind + CSS variables（design tokens）
   → 主題切換、可配置

🟢 多主題（暗色 / 自訂主題）
   CSS variables + v-bind in CSS
   → 動態切換、零閃爍

🟢 企業 / 維護優先 / 不想用 utility class
   BEM 命名 + Scoped CSS + SCSS
   → 傳統穩定、新人易讀

🟡 套 UI library（Element Plus / Naive UI）
   library 預設樣式 + :deep() 微調

❌ 不推薦
   Vue 套件還是用 styled-components（CSS-in-JS）
   → 社群沒在用、效能差、踩坑沒人救
```

  
### 10. 別把樣式寫成「義大利麵」

```text
❌ 一個元件混 5 種樣式來源
   - Tailwind class
   - Scoped CSS
   - inline style
   - global CSS
   - CSS Modules

→ 維護災難、誰也不知道哪個生效

✅ 一個專案統一風格
   - 主力 Tailwind
   - 輔助 Scoped CSS（Tailwind 表達不出來的）
   - 全域 CSS 只放：reset / 字型 / CSS variables
   - 不用 inline style（除非真的動態）
```

  
### 11. 5 大常見坑

```text
❌ 1. Scoped CSS 不知道是否生效、樣式衝突
   → DevTools 看 element、看有沒有 data-v-xxx 屬性
   → 修：確認 <style scoped>、不是普通 <style>

❌ 2. 改第三方 UI 樣式改不到
   → Element Plus 的 .el-button 不在你的 scope 內
   → 修：用 :deep(.el-button) { ... }

❌ 3. Tailwind 寫了沒效
   → tailwind.config 沒包到你的 .vue 檔
   → 修：content: ["./src/**/*.{vue,ts,js}"]

❌ 4. v-bind in CSS 寫法錯
   → background: v-bind(color)  ← 對
   → background: var(--bind(color))  ← 錯
   → 修：v-bind(jsVar) 直接寫、不用 var()

❌ 5. 全域 CSS 蓋過 scoped
   → 全域 .btn 蓋過 scoped .btn（specificity 衝突）
   → 修：全域 CSS 命名加 prefix（.app-btn）、避免 generic 名
```

  
### 🎯 重點回顧

- **Scoped CSS** = Vue 招牌、`<style scoped>` 一加就 enable、99% 場景首選
- **CSS Modules**（`<style module>`）class 自動 hash、零衝突
- **Tailwind**（2026 主流）utility-first、跟 Scoped 搭配最香
- **動態 binding**：`:class="{...}"`、`:style="{...}"`
- **`:deep()`** 穿透 scoped 改子元件樣式
- **`v-bind in CSS`**（Vue 3.2+）CSS 內動態綁 JS 變數、Vue 獨家
- **推薦組合**：Tailwind + Scoped + CSS variables（90% 場景完美）
