**第一個 Vue 專案 = 用官方指令 5 分鐘生出一個能跑的網頁**——不用配 webpack、不用裝半天、用 `npm create vue@latest` 一行指令就有完整 SPA 雛形。這課跟著做、結束時你會有一個能在 `localhost:5173` 看見的 Vue 應用、看懂 SFC（Single File Component、單檔元件）三塊結構。

> 💡 **這課的價值**：從零到「螢幕上有一個會動的 Vue 按鈕」、是學前端最有成就感的 5 分鐘。看懂這課就懂了 Vue 80% 的日常寫法。

  
### 1. 開始前準備：裝 Node.js

Vue 是用 JavaScript 寫的、要跑就要先裝 **Node.js（讀「node」、伺服器端 JS 執行環境）**——不是只用在伺服器、現在前端開發也靠它跑 build tool（建置工具）。

```text
🖥️ 在終端機輸入

node --version

⌨️ 應該看到類似

v20.10.0
```

**如果看到「找不到指令」**——去 https://nodejs.org 下載 LTS 版（最穩定的長期支援版）、安裝完重開終端機。

> 💡 **「終端機」是什麼**：Mac 叫 Terminal、Windows 叫 PowerShell 或 cmd、VS Code 內建按 Ctrl+`（反引號）打開。**是讓你用文字指令操作電腦的黑窗**。

  
### 2. 一行指令建專案

```text
🖥️ 在終端機輸入

npm create vue@latest my-first-vue

💬 電腦會問你一連串問題

✔ Project name: my-first-vue
✔ Add TypeScript? (TS、強型別 JS、推薦 Yes)
✔ Add JSX Support? (要 JSX 也能寫 Vue 嗎、No)
✔ Add Vue Router? (路由、Yes)
✔ Add Pinia? (狀態管理、Yes)
✔ Add Vitest? (測試、No 先跳過)
✔ Add ESLint? (程式碼風格檢查、Yes)
✔ Add Prettier? (自動排版、Yes)
```

**`npm create vue@latest`** 拆解：
- `npm` —— **Node Package Manager（節點套件管理器）**、Node.js 內建的工具
- `create vue` —— 跑 `create-vue` 這個官方腳手架（scaffold、產生專案骨架的工具）
- `@latest` —— 拉**最新版本**

  
### 3. 進專案、裝套件、啟動

```text
🖥️ 在終端機輸入

cd my-first-vue
npm install
npm run dev

💬 終端機會印

  VITE v5.0.0  ready in 432 ms
  ➜  Local:   http://localhost:5173/
  ➜  press h + enter to show help
```

**打開瀏覽器、輸入 `http://localhost:5173`**——你就會看到一個 Vue 預設首頁。**恭喜、第一個 Vue 應用跑起來了**。

> 💡 **「localhost」是什麼**：你電腦本機的網址、外人看不到、只有你自己的瀏覽器能連。`5173` 是 Vite（讀「vit」、Vue 的 build tool）預設的 port（連接埠）。

  
### 4. 認識專案結構

打開 VS Code、看左邊檔案樹：

```text
my-first-vue/
├── public/              ← 靜態檔案（圖片、favicon）
├── src/                 ← 你寫程式的地方
│   ├── assets/          ← 圖片、CSS、字型
│   ├── components/      ← 可重用的小元件（HelloWorld.vue）
│   ├── router/          ← Vue Router 設定
│   ├── stores/          ← Pinia 狀態管理
│   ├── views/           ← 整頁的元件（HomeView.vue、AboutView.vue）
│   ├── App.vue          ← 根元件（root component、最外層）
│   └── main.ts          ← 程式進入點
├── index.html           ← 唯一的 HTML 檔
├── package.json         ← 專案設定 + 套件清單
├── tsconfig.json        ← TypeScript 設定
└── vite.config.ts       ← Vite build 設定
```

**5 個你最常碰的檔案**：
- `src/App.vue` —— 整個應用的最外層
- `src/views/*.vue` —— 每個「頁面」一個檔案
- `src/components/*.vue` —— 可複用的小積木（按鈕、卡片、表單）
- `src/router/index.ts` —— 哪個網址對應哪個頁面
- `src/stores/*.ts` —— 跨頁共享的資料

  
### 5. 認識 SFC（Single File Component、單檔元件）

**SFC** 是 Vue 的招牌、一個 `.vue` 檔包**三塊**：

```text
📄 寫進程式檔：`src/components/HelloWorld.vue`

<script setup lang="ts">
import { ref } from "vue";

const count = ref(0);

function increment() {
  count.value++;
}
</script>

<template>
  <div class="hello">
    <h1>Hello Vue!</h1>
    <p>Count: {{ count }}</p>
    <button @click="increment">+1</button>
  </div>
</template>

<style scoped>
.hello {
  text-align: center;
  padding: 20px;
}
button {
  background: #42b883;
  color: white;
  border: none;
  padding: 8px 16px;
  border-radius: 4px;
  cursor: pointer;
}
</style>
```

**三塊各做什麼**：

```text
<script setup>        ← JS / TS 邏輯（Composition API）
   import { ref } from "vue"
   ref(0) = 「會自動讓畫面跟著變」的數字
   .value 是它的內部值（template 內不用寫）

<template>            ← HTML 結構（含 Vue 指令）
   {{ count }}        = interpolation（插值、把資料塞進 HTML）
   @click="..."       = v-on:click 簡寫、綁定點擊事件
   :class="..."       = v-bind:class 簡寫、動態綁屬性

<style scoped>        ← CSS（scoped 表示只影響這個元件）
   scoped 編譯後會自動加 data-v-xxx 屬性
   不會污染其他元件、超安心
```

> 💡 **`<script setup>` 是什麼**：Vue 3.2+ 的語法糖、讓 Composition API 寫起來像「直接寫 script」、不用 export 一個 setup function。**現代 Vue 99% 都用這個**。

  
### 6. 進入點 `main.ts` —— 應用怎麼啟動的

```text
📄 寫進程式檔：`src/main.ts`

import { createApp } from "vue";
import App from "./App.vue";
import router from "./router";
import { createPinia } from "pinia";

const app = createApp(App);     // 建一個 Vue 應用實例
app.use(createPinia());          // 接上狀態管理
app.use(router);                 // 接上路由
app.mount("#app");               // 掛到 index.html 的 <div id="app">
```

**4 步驟**：
1. `createApp(App)` —— 用 `App.vue` 當根元件、建一個應用
2. `app.use(...)` —— 接「plugin（外掛）」、像 Pinia / Router
3. `app.mount("#app")` —— 把整個 Vue 應用塞進 `index.html` 裡的 `<div id="app"></div>`

打開 `index.html` 你會看到：

```text
<body>
  <div id="app"></div>           ← Vue 會把元件塞這裡
  <script type="module" src="/src/main.ts"></script>
</body>
```

**整個應用只有一個 HTML 檔**——這就是 **SPA（Single Page Application、單頁應用）**的意思、所有切頁都是 JS 操作 DOM 完成、不會真的換 HTML 檔。

  
### 7. 改一行 code 試試 Hot Module Replacement

```text
📄 改 src/App.vue 第幾行的文字（隨便改）

把「Welcome to Vue」改成「林董的第一個 Vue」

💬 不用刷新瀏覽器
   → 畫面瞬間更新
   → 而且 state（你按過幾次的計數）還在！
```

**HMR（Hot Module Replacement、模組熱替換）**—— Vite 的招牌特性、改檔不重新整理、保留 state 直接 patch 上去。**寫起來爽到飛起來**、是 Vue/React 現代開發的基本功。

  
### 8. 5 大常見坑

```text
❌ 1. 用了舊版指令 vue create my-app（Vue 2 的 Vue CLI）
   → 跑出 webpack-based 老專案、不是 Vite
   → 修：永遠用 npm create vue@latest

❌ 2. 忘記 cd 進專案就跑 npm run dev
   → 報錯「找不到 package.json」
   → 修：cd my-first-vue 先

❌ 3. ref(0) 在 script 內忘記寫 .value
   → console.log(count) 印 [object Ref]、不是 0
   → 修：console.log(count.value)、template 內才不用

❌ 4. 同時跑兩個 npm run dev（兩個專案、兩個視窗）
   → port 5173 被佔、第二個自動跳 5174
   → 修：注意終端機印的網址、不要連錯

❌ 5. 改 .vue 檔忘記 import 進 App.vue
   → 寫了元件畫面卻沒出現
   → 修：在 App.vue script setup 內 import + template 用 <YourComp />
```

  
### 🎯 重點回顧

- **`npm create vue@latest`** 一行建專案、5 分鐘上線
- **Vite** 是 Vue 預設 build tool、啟動快、HMR 爽
- **SFC（單檔元件）** = `<script setup>` + `<template>` + `<style scoped>` 三塊
- **`ref(0)`** 是「會自動更新畫面的變數」、script 內用 `.value`、template 內直接寫
- **整個 SPA 只有一個 `index.html`**、Vue 把元件塞進 `<div id="app">`
