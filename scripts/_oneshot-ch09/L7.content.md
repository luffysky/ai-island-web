**Pinia（讀「皮尼亞」、西班牙文「鳳梨」）= Vue 3 官方推薦的 state management（狀態管理）**——讓你把「跨整個 App 共享的資料（user、cart、theme）」放在一個地方、任何元件都能拿、任何元件都能改、自動同步。**取代舊的 Vuex**、API 簡單 10 倍、TypeScript 支援滿分。

> 💡 **這課的價值**：搞懂為什麼需要 store（資料倉庫）、寫好第一個 Pinia store、知道 state / getters / actions 三角架構。看完你就能做有登入、購物車、跨頁同步的完整應用。

  
### 1. 為什麼需要 Pinia？

**沒 Pinia 的痛**：
- 使用者登入後、20 個元件都要顯示「你好、林董」——用 props 傳 20 層？
- 購物車加了一件東西、Header 的徽章要 +1、Footer 的「結帳」按鈕要解鎖、Cart 頁要更新——三個分散元件怎麼同步？
- 暗色模式切換、整個 App 100 個元件樣式要跟著變

**Pinia 解法**：建一個「store（倉庫）」、把這些 state 放裡面、任何元件都能直接讀寫、變了會自動推播給所有訂閱者。

```text
傳統 props 傳法
─────────────────
App
 ├─ Header
 │   ├─ Logo
 │   └─ UserMenu ← 要顯示 user.name、props 傳 3 層
 └─ Page
     └─ Section
         └─ Card
             └─ ... 又是一條 props 鏈

用 Pinia
─────────────────
App
 ├─ Header → useUserStore() 直接拿 user
 ├─ Page   → useUserStore() 直接拿 user
 └─ Card   → useUserStore() 直接拿 user
   (任何元件、不用 props 傳)
```

### 3. 定義第一個 Store

**Pinia 有兩種寫法**——Composition 風格（推薦）跟 Options 風格。新專案 99% 用 Composition：

```text
📄 寫進程式檔：`src/stores/counter.ts`

import { defineStore } from "pinia";
import { ref, computed } from "vue";

// store id "counter" 是這個 store 的全域識別名
export const useCounterStore = defineStore("counter", () => {
  // state（資料）
  const count = ref(0);
  const history = ref<number[]>([]);

  // getters（衍生值、像 computed）
  const doubled = computed(() => count.value * 2);
  const isEven = computed(() => count.value % 2 === 0);

  // actions（修改方法）
  function increment() {
    count.value++;
    history.value.push(count.value);
  }

  async function fetchFromAPI() {
    const res = await fetch("/api/count");
    const data = await res.json();
    count.value = data.count;
  }

  function reset() {
    count.value = 0;
    history.value = [];
  }

  return { count, history, doubled, isEven, increment, fetchFromAPI, reset };
});
```

**3 大成員（對齊 Vuex 概念）**：
- **state（狀態）** = 資料本身（`ref` / `reactive`）
- **getters（取值器）** = 從 state 算出來的值（`computed`）
- **actions（動作）** = 改 state 的函式（可以 async）

> 💡 **store id（第一個參數 "counter"）很重要**：是這個 store 的全域 ID、Vue DevTools 用它識別、不能跟其他 store 撞名。

  
### 4. 在元件用 store

```text
📄 寫進程式檔：`src/components/Counter.vue`

<script setup lang="ts">
import { useCounterStore } from "@/stores/counter";
import { storeToRefs } from "pinia";

// 拿到 store instance
const counter = useCounterStore();

// 解構 state / getters 要用 storeToRefs 保持響應性
const { count, doubled, isEven } = storeToRefs(counter);

// actions 直接解構就行（不用 storeToRefs）
const { increment, reset } = counter;
</script>

<template>
  <p>Count: {{ count }}</p>
  <p>Doubled: {{ doubled }}</p>
  <p>偶數嗎：{{ isEven ? "是" : "否" }}</p>

  <button @click="increment">+1</button>
  <button @click="reset">重設</button>
</template>
```

**⚠️ 必懂：`storeToRefs` 為什麼一定要用**：

```text
❌ const { count } = useCounterStore();
   → count 被「拆出來」、失去 reactive
   → store 更新、畫面不會跟著變

✅ const { count } = storeToRefs(useCounterStore());
   → 解構出來還是 ref、保留響應性
   → store 更新、畫面自動更新

🔧 actions（function）不用 storeToRefs
   → const { increment } = counter; 這樣就可以
```

  
### 5. 實戰：使用者登入 store

```text
📄 寫進程式檔：`src/stores/user.ts`

import { defineStore } from "pinia";
import { ref, computed } from "vue";

interface User {
  id: string;
  name: string;
  email: string;
  isVip: boolean;
}

export const useUserStore = defineStore("user", () => {
  const user = ref<User | null>(null);
  const loading = ref(false);

  const isLoggedIn = computed(() => user.value !== null);
  const displayName = computed(() => user.value?.name || "訪客");

  async function login(email: string, password: string) {
    loading.value = true;
    try {
      const res = await fetch("/api/login", {
        method: "POST",
        body: JSON.stringify({ email, password })
      });
      user.value = await res.json();
    } finally {
      loading.value = false;
    }
  }

  function logout() {
    user.value = null;
  }

  return { user, loading, isLoggedIn, displayName, login, logout };
});
```

```text
📄 用：`src/components/Header.vue`

<script setup>
import { useUserStore } from "@/stores/user";
import { storeToRefs } from "pinia";

const userStore = useUserStore();
const { displayName, isLoggedIn } = storeToRefs(userStore);
const { logout } = userStore;
</script>

<template>
  <header>
    <span>你好、{{ displayName }}</span>
    <button v-if="isLoggedIn" @click="logout">登出</button>
    <router-link v-else to="/login">登入</router-link>
  </header>
</template>
```

**任何元件 import 一次 `useUserStore()` 就能拿同一份資料**——這就是 store 的威力。

  
### 6. 持久化（重新整理也不消失）

預設 Pinia 資料只在記憶體、重新整理就沒了。要存進 localStorage 用 plugin：

```text
🖥️ 在終端機輸入

pnpm add pinia-plugin-persistedstate
```

```text
📄 main.ts

import { createPinia } from "pinia";
import piniaPluginPersistedstate from "pinia-plugin-persistedstate";

const pinia = createPinia();
pinia.use(piniaPluginPersistedstate);
app.use(pinia);
```

```text
📄 store 內加 persist: true（Options 寫法才有）
或 Composition 寫法用 setup option：

export const useUserStore = defineStore("user", () => {
  // ...
}, {
  persist: true                      // ← 自動存 localStorage
});

// 只存特定欄位
{
  persist: {
    paths: ["user.id", "user.name"]  // ← email 不存
  }
}
```

  
### 7. Pinia vs Redux 對比

| 項目 | Pinia（Vue） | Redux（React） |
| --- | --- | --- |
| 寫 store | 1 個 file、3 個區（state/getters/actions） | 4-5 個 file（slice、reducer、action、selector） |
| boilerplate（樣板碼） | 少 | 多（即使用 Redux Toolkit 仍然多） |
| TypeScript | 滿分、零設定 | 中等、要寫 type |
| async（API） | actions 直接 async/await | 要用 thunk / saga |
| DevTools | 內建 | 內建 |
| 學習曲線 | 1 天 | 1 週 |

> 💡 **Pinia 是 Redux 的「省去 80% 樣板」版**——資深 React 工程師第一次寫 Pinia 通常震驚於「就這樣？也太簡單了吧」。

  
### 8. Vuex（舊版）怎麼辦？

**Vuex 是 Vue 2 / Vue 3 早期的官方 state 解法、現在已被 Pinia 取代**。新專案不要用 Vuex。

```text
舊（Vuex）          新（Pinia）
─────────         ─────────
state             state
mutations         （直接改、不用）
actions           actions
getters           getters
modules           多個 defineStore
this.$store.commit("...") → useStore().action()
mapState/mapGetters → storeToRefs
```

舊專案用 Vuex 沒關係、能跑就繼續用、新功能新 store 用 Pinia。Pinia 跟 Vuex 可以共存。

  
### 9. 5 大常見坑

```text
❌ 1. 解構 store 沒用 storeToRefs
   → const { count } = useStore() → 失去響應性
   → 修：const { count } = storeToRefs(useStore())

❌ 2. 在 setup 外 call useStore()
   → 報錯「getActivePinia was called with no active Pinia」
   → 修：useStore() 必須在 script setup 內 / setup function 內

❌ 3. 直接 mutate store 的 array / object 屬性
   → store.list.push(...) 在嚴格模式可能無效
   → 修：在 action 內改、或用 store.list = [...store.list, newItem]

❌ 4. 同個 store id 重複定義
   → defineStore("user", ...) 寫了兩次 → 資料覆蓋
   → 修：每個 store 用獨立 id（"user" / "cart" / "ui"）

❌ 5. 拿 store 當「全域變數倉庫」亂塞
   → store 內塞 50 個無關 state、變肥大
   → 修：按功能拆 store（useUserStore / useCartStore / useUIStore）
```

  
### 🎯 重點回顧

- **Pinia** = Vue 3 官方推薦的 state 管理、取代 Vuex
- **3 大成員**：state（ref/reactive）+ getters（computed）+ actions（function）
- **`defineStore("id", () => {...})`** 定義 store、`useXxxStore()` 用 store
- **解構 state / getters 要用 `storeToRefs`**、actions 直接解構
- **持久化** 裝 `pinia-plugin-persistedstate`、`persist: true` 一鍵存 localStorage
- **vs Redux** 樣板少 80%、學習 1 天 vs 1 週
