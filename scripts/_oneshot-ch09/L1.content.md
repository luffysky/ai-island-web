**Vue（讀作 view、念「優」）= 2014 年由前 Google 工程師尤雨溪（Evan You）打造的前端框架**，主打「**漸進式採用**—— 你不必一次重寫全部、可以只在一頁先試試水溫」。它是中國、日本、東南亞市場前端三巨頭之一（React、Vue、Angular），亞洲新創有近一半選 Vue。

> 💡 **這課的價值**：搞懂 Vue 的歷史、為什麼亞洲市場特愛它、2026 還值不值得學。讀完你就能在「React 還是 Vue」的抉擇前不再迷路。

  
### 1. Vue 是什麼？一句話講完

```text
React 像「樂高散裝箱」
   → 給你積木自己拼
   → 自由、但要選很多（router 選哪個？state 選哪個？form 選哪個？）

Vue 像「樂高套裝盒」
   → 官方幫你配好路由（Vue Router）、狀態（Pinia）、build tool（Vite）
   → 按指引走、半天就能跑起來
```

**Vue 的核心理念**：「**progressive framework（漸進式框架）**—— 你可以只用一點點、也可以全部用」。意思是你可以只在現有網站的一個頁面塞一段 Vue、不用打掉重來。React 也能做、但 Vue 是「**設計目標**就是這樣」。

> 💡 **「漸進式」白話翻譯**：你今天可以只用它做「一個聊天按鈕」、明天升級成「整個前端 SPA（Single Page Application、單頁應用）」、不用換工具。React 比較傾向「要嘛全用、要嘛不用」。

  
### 2. Vue 的歷史—— 一個人寫出來的框架

```text
🕐 2013   尤雨溪在 Google 寫 AngularJS 專案、覺得「太重」
🚀 2014   他自己抽出最喜歡的部分、發布 Vue 0.6
📈 2016   Vue 2 發布、開始爆紅（中國市場為主）
🔧 2020   Vue 3 + Composition API（組合式 API）重大改寫
⚡ 2021   尤雨溪自己做 Vite（讀「vit」、法文「快」）
✨ 2025+  Vue 3.5 + Pinia + Nuxt 3 成為亞洲事實標準
```

**冷知識**：尤雨溪是**靠 Patreon 贊助全職寫 Vue**的——沒大公司撐腰、純社群養他。這也是為什麼 Vue 看起來「比較沒商業味」、更貼近開發者體驗。

  
### 3. Vue 3 的革命—— 跟舊版差在哪

```text
舊 Vue 2（Options API、選項式 API）
─────────────────────────────────
   data: { count: 0 },
   methods: { add() { ... } },
   computed: { ... },
   ← 一個元件分成「data/methods/computed」幾個區
   ← 邏輯散在各區、難以重用

新 Vue 3（Composition API、組合式 API）
─────────────────────────────────
   const count = ref(0);
   const add = () => count.value++;
   ← 像 React Hooks、邏輯寫一起
   ← 可以抽成 composable（複用函式）共用
```

**Composition API（組合式 API）**—— Vue 3 的新寫法、用 setup function（設定函式）把 state（狀態）+ 邏輯組合在一起、跨元件共用很方便。**這跟 React Hooks 在解的是同一個問題**——讓邏輯能跨元件複用。

**reactive（響應式）**—— Vue 自動追蹤資料變化、改 data 畫面就更新。Vue 3 把背後的機制從 `Object.defineProperty` 換成 `Proxy（代理物件）`、變更快也變聰明（可以追蹤新增的屬性、Vue 2 做不到）。

  
### 4. Vue vs React 一句話對比

| 比較項 | Vue | React |
| --- | --- | --- |
| 寫法 | template + script（兩塊分開） | JSX（HTML 寫在 JS 裡） |
| 學習曲線 | 緩 | 中等 |
| 中文資源 | 最完整（中國市場大） | 完整、但要英文文件 |
| 工作機會（台灣） | 中 | 多 |
| 工作機會（中國） | 多 | 中 |
| 工作機會（美國） | 少 | 巨多 |
| 大公司採用 | 阿里、Bilibili、GitLab、Nintendo | Meta、Netflix、Airbnb |
| 跨平台 | 有（NativeScript-Vue、較弱） | 強（React Native） |
| 內建功能 | 多（router/state 官方都有） | 少（要自己選） |

  
### 5. 為什麼亞洲（中日韓）特愛 Vue？

**3 個主要原因**：

```text
1. 中國市場帶頭
   → 阿里、Bilibili、京東、騰訊大量採用
   → 大型開源 UI（Element Plus、Ant Design Vue）都中國團隊維護
   → 中文文件 + 中文社群超級活躍

2. 學習門檻低
   → template 就是 HTML 改一點點、新人 1 週上手
   → React 的 JSX 對純 HTML/CSS 出身的人很彆扭

3. 設計師友好
   → SFC（Single File Component、單檔元件）三塊（template/script/style）分清楚
   → 設計師看 template 區可以直接認字、不用學 JSX
```

> 💡 **接案族的觀察**：台灣中小企業 / 接案市場 Vue 比例其實不低（約 30-40%）、客戶很多看到「Vue」覺得「跟 HTML 比較像、我可以自己維護」就選了。React 反而是大公司 / 美商比例高。

  
### 6. 2026 還能學 Vue 嗎？

**答案：能、但要看你的目標**。

```text
✅ 適合學 Vue
   - 想接案（中小客戶愛 Vue）
   - 想去中國 / 香港 / 東南亞工作
   - 純 HTML/CSS 出身、不想跳太大步
   - 想做 Indie SaaS（單人開發、Vue 開發速度快）
   - 已經會 React、想多會一個

❌ 別優先學 Vue
   - 目標是進 FAANG / 美商（React 機會多 5 倍）
   - 想做 React Native（跨手機 App）
   - 公司用 React、學 Vue 沒地方練
```

**業界共識（2026）**：**會一個夠用、會兩個薪水高**。資深工程師應該兩個都會（雖然只精通一個）。看到 Vue 專案不要說「我只會 React 不接」、看 Vue 文件 2-3 天就能 PR（pull request、提交合併請求）。

### 8. 5 大常見誤解

```text
❌ 1. 「Vue 是過氣的舊框架」
   → 2025 全球下載量第 2（僅次 React）、亞洲第 1
   → Vue 3 + Vite + Nuxt 比 React 一些 stack 還新潮

❌ 2. 「Vue 都中國人在用、英文資源差」
   → 官方文件多語齊全、英文社群也活躍
   → GitLab / Nintendo 美國分公司也大量用

❌ 3. 「Vue 跟 jQuery 一樣是『加在現有頁面』的」
   → 那是 Vue 2 早期印象、Vue 3 + Vite 可以做完整 SPA / SSR
   → Nuxt 3 是全端框架、跟 Next.js 同級

❌ 4. 「Vue 跟 React 差超多、學一個要重學」
   → 心智模型 80% 一樣（元件、props、state、生命週期）
   → 會 React 學 Vue 通常 3-5 天可以上手

❌ 5. 「Vue 不能做大專案、效能比較差」
   → 阿里整個淘寶後台是 Vue
   → 效能跟 React 差別 <5%、瓶頸永遠在你的 code 不在框架
```

  
### 🎯 重點回顧

- **Vue = 漸進式前端框架**、由尤雨溪 2014 發布、社群驅動
- **Vue 3 + Composition API** 是 2026 主流、舊的 Options API 仍能用但不推薦
- **亞洲市場（中日韓）特愛 Vue**、台灣中小企業 / 接案約 30-40% 用 Vue
- **vs React** 心智模型 80% 一樣、學 Vue 不會浪費（會兩個薪水高）
- **2026 適合**：接案族、亞洲市場、純 HTML 出身、Indie 開發者
