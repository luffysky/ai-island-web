**Vue 跟 React 全面深度對比—— 心智模型、學習曲線、性能、生態、社群、業界採用 6 個維度**。L10 給你「該選哪個」的速答、這課給你「為什麼」的完整理由。看完你能在面試 / 團隊技術選型討論時、講出有理有據的論點、不再只說「我覺得」。

### 1. 心智模型（Mental Model）

**心智模型 = 你腦中怎麼想這個框架在做什麼**——這決定你 debug 跟設計元件的方式。

**React 心智模型**：

```text
「component 是純函式」
─────────────────
function Component(props) {
  const [state, setState] = useState(...);

  // 每次 render 整個 function 重跑一次
  // useState 用內部 fiber 記住值

  return <JSX />;
}

🧠 核心觀念
- render = 跑一次 function
- state 變 → 整個 component 重跑
- useEffect dependencies 顯式宣告
- 心智負擔：「這個 hook 該放哪、dep array 要寫什麼」
```

**Vue 心智模型**：

```text
「component 是 reactive object」
─────────────────
<script setup>
const count = ref(0);
// setup 只跑一次、count 是「會自己叫畫面更新的容器」
</script>

🧠 核心觀念
- setup = 註冊一堆 reactive 變數 + 函式（只跑一次）
- count.value 變 → Vue 自動知道誰用了 count、只更新那些地方
- 不用宣告依賴、自動追蹤
- 心智負擔：「.value 要不要寫、ref 還是 reactive」
```

**比喻**：
- React = 你寫食譜（recipe）、每次煮飯就重做整道菜
- Vue = 你準備食材（reactive）、廚師（Vue）看你動了什麼食材自動煮

  
### 2. API 風格對照

| 功能 | React | Vue |
| --- | --- | --- |
| state（單值） | `useState` | `ref` |
| state（物件） | `useState` | `reactive` / `ref` |
| 衍生值 | `useMemo` | `computed` |
| 副作用 | `useEffect` | `watchEffect` / `watch` / `onMounted` |
| 自訂 hook | Custom Hook | Composable |
| 全域狀態 | Zustand / Redux / Jotai | Pinia |
| context | `useContext` | `provide` / `inject` |
| ref（DOM） | `useRef` | `ref` + `<div ref="el">` |
| 條件渲染 | `{cond && <X />}` | `v-if="cond"` |
| 列表渲染 | `{arr.map(x => <X key=...>)}` | `v-for="x in arr" :key="..."` |
| 事件 | `onClick={fn}` | `@click="fn"` |
| 表單雙向 | `value + onChange` | `v-model` |
| 樣式 scope | CSS Modules / styled-comp | `<style scoped>` 內建 |

**規律**：React 統一用 hook（function call）、Vue 統一用 ref + 自動追蹤。

  
### 3. 學習曲線

```text
React 學習路徑（典型 3-6 個月）
─────────────────
1. JS 基礎（特別是 async / closure / array methods）
2. JSX 語法（HTML 寫在 JS）
3. Component + Props
4. useState
5. useEffect + dependency array ← 最大坑、新人在這卡 1-2 週
6. Context / Custom Hook
7. React Router
8. 選 state 庫（Zustand / Redux）
9. Next.js / SSR / Server Components ← 新坑

🔻 卡點：useEffect dep / re-render / memo / Server vs Client Component
```

```text
Vue 學習路徑（典型 1-3 個月）
─────────────────
1. HTML / CSS / JS 基礎
2. SFC 三塊（template / script / style）
3. ref / reactive
4. Template 指令（v-if / v-for / v-on）
5. Component + Props + Emit
6. computed / watch
7. Vue Router（官方教程順手帶）
8. Pinia
9. Nuxt / SSR

🔻 卡點：ref vs reactive / .value 何時要寫 / Composition vs Options API
```

**為什麼 Vue 比較緩**：
- template 接近原生 HTML、不用學 JSX
- 響應式自動、不用手動宣告 dep
- 官方包好 Router / Pinia、不用選

**為什麼 React 比較陡**：
- JSX 對純 HTML 出身的人陌生
- useEffect dep 是有名的踩坑王
- 生態太大、新人選擇困難

  
### 4. 性能（performance）

**真相：兩個都很快、差別小於 5%**。

**React 性能特點**：
- VDOM diff → 整個 component 重 render、Vue 是精確更新
- 需要手動 `useMemo` / `useCallback` 優化
- React Compiler（2025 推出）開始自動 memo、未來會大幅減少手動優化
- 巨大列表 + Concurrent + Suspense 強

**Vue 性能特點**：
- Proxy reactive → 精確追蹤、只更新真正變的 DOM
- 不用手動 memo（編譯器自動優化）
- 中小元件 / 頻繁 update 場景略勝
- bundle size 略小

```text
性能差別實測（2025 benchmark）
─────────────────
js-framework-benchmark：兩者差 < 5%
真實 App：差別 < 1%（CPU / 網路才是瓶頸）

→ 寫得爛的 React app vs 寫得好的 Vue app
   後者快 10 倍
→ 「框架性能」這件事 99% 場景無感
```

**結論**：**選框架不要看性能**——挑你寫起來爽的。

### 10. 「會兩個」薪資 +20%—— 真的嗎？

```text
理由
─────────────────
1. 接案不挑食
   - 客戶說 Vue 你也能接、客戶說 React 你也能接
   - 月接案量 +30%

2. 換工作選擇大
   - 看到任何前端職缺都能投
   - 不會「我只會 X、這缺要 Y」

3. 跨團隊有優勢
   - 一個公司多產品線（A 用 Vue、B 用 React）你都能支援
   - 升 senior / tech lead 機會大

4. 心智模型轉換能力
   - 學第三個（如 Svelte）只要 3 天
   - 寫履歷加分項
```

**現實**：**精通一個 + 熟悉一個** > 兩個都半吊子。**先深後廣**。

### 12. 5 個常見錯誤觀念

```text
❌ 1. 「Vue 是過氣的、React 才是未來」
   → Vue 仍在前端 Top 3、亞洲市場第一
   → 兩個都未來 5-10 年不會死

❌ 2. 「React 比 Vue 難太多、新人別學」
   → 難 30%、但生態大 3 倍
   → 走完那 30% 痛苦、市場機會是 3 倍

❌ 3. 「Vue 只能做小專案」
   → 阿里整個淘寶後台是 Vue
   → 大專案能不能撐看架構、不是看框架

❌ 4. 「會 React 看 Vue 一眼就懂、不用學」
   → 80% 是、20% 不是（v-model / Composition / SFC 概念要學）
   → 不正式學會踩坑 1 個月

❌ 5. 「Vue 中文資源多、所以中國以外沒人用」
   → GitLab / Nintendo / Behance 是西方公司
   → 中文資源多是優勢、不是缺點
```
