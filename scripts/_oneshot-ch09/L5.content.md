**元件通訊（component communication）= 讓父元件、子元件、兄弟元件互相傳資料 / 觸發動作**。Vue 給你 4 種武器：**props（父→子）**、**emit（子→父）**、**v-model（雙向）**、**provide/inject（跨多層）**。學會這 4 招、任何 UI 拆分都能優雅串起來。

> 💡 **這課的價值**：元件拆得好、code 才好維護。看懂這 4 招、你就不會「所有 state 全擠在 App.vue」、也不會「props 一層層傳到第 7 層快瘋掉」。

  
### 1. Props（父→子）—— 父元件傳資料給子元件

**props（properties、屬性）**—— 父元件透過「**像 HTML 屬性**」的方式把資料傳給子元件。子元件 readonly（唯讀）、不能改。

```text
📄 子元件：`src/components/UserCard.vue`

<script setup lang="ts">
// 定義 props 的型別 + 預設值
interface Props {
  name: string;
  age?: number;          // ? 表示可以不傳
  tags?: string[];
}

const props = withDefaults(defineProps<Props>(), {
  age: 0,
  tags: () => []         // 物件 / 陣列預設值要用 function 包
});
</script>

<template>
  <div class="card">
    <h3>{{ props.name }}</h3>
    <p>年齡：{{ props.age }}</p>
    <p>標籤：{{ props.tags.join(", ") }}</p>
  </div>
</template>
```

```text
📄 父元件：`src/views/Home.vue`

<script setup>
import UserCard from "@/components/UserCard.vue";
</script>

<template>
  <!-- 靜態 prop -->
  <UserCard name="林董" />

  <!-- 動態 prop（用 :） -->
  <UserCard :name="userName" :age="30" :tags="['Vue', 'TypeScript']" />
</template>
```

**3 個關鍵**：
- **`defineProps`** 是 Vue 3 的 compiler macro（編譯時宏）、不用 import
- **TS 範型 `<Props>`** 給你型別檢查、IDE 補全
- **`withDefaults`** 設預設值、避免父沒傳就 undefined

> 💡 **props 是「單向」的**：父改 → 子跟著變、子不能改 props。要改值請用 emit 通知父。

  
### 2. Emit（子→父）—— 子元件通知父元件

**emit（發送）—— 子元件「對外廣播」一個事件、父元件接到就執行對應動作**。

```text
📄 子元件：`src/components/MyButton.vue`

<script setup lang="ts">
// 宣告會 emit 哪些事件 + 參數型別
const emit = defineEmits<{
  (e: "click", payload: { time: number }): void;
  (e: "dblclick"): void;
}>();

function handleClick() {
  emit("click", { time: Date.now() });
}
</script>

<template>
  <button @click="handleClick" @dblclick="emit('dblclick')">
    按我
  </button>
</template>
```

```text
📄 父元件：`src/views/Home.vue`

<script setup>
import MyButton from "@/components/MyButton.vue";

function onChildClick(payload) {
  console.log("子元件被點了、時間：", payload.time);
}
</script>

<template>
  <!-- 用 @ 接子元件 emit 的事件 -->
  <MyButton @click="onChildClick" @dblclick="() => alert('連點兩下！')" />
</template>
```

**React 對比**：React 沒有 emit、要把「函式當 prop 傳下去」（`<MyButton onClick={handleClick} />`）。Vue 兩個寫法都能用、但 emit 更語意化（「我發送一個事件給外面」）。

  
### 3. v-model on Component（雙向綁定到子元件）

把「父的 state」跟「子的 input」雙向綁——一句 `v-model` 搞定：

```text
📄 子元件：`src/components/SearchBox.vue`（Vue 3.4+）

<script setup lang="ts">
// defineModel() 一行搞定雙向綁定
const model = defineModel<string>();
</script>

<template>
  <input :value="model" @input="model = $event.target.value" />
</template>
```

```text
📄 父元件

<script setup>
import { ref } from "vue";
import SearchBox from "@/components/SearchBox.vue";
const keyword = ref("");
</script>

<template>
  <SearchBox v-model="keyword" />
  <p>正在搜尋：{{ keyword }}</p>
</template>
```

**等價於**：`<SearchBox :modelValue="keyword" @update:modelValue="keyword = $event" />`、Vue 自動展開的語法糖。

**多個 v-model**：

```text
<!-- 子元件 -->
const firstName = defineModel<string>("firstName");
const lastName = defineModel<string>("lastName");

<!-- 父元件 -->
<NameInput v-model:firstName="fn" v-model:lastName="ln" />
```

  
### 4. Slot（內容插槽）—— 父決定子內部塞什麼

**slot（插槽）—— 子元件挖個「洞」、讓父元件決定洞裡塞什麼 HTML**。比 props 強——可以塞整段 UI 不只是字串。

```text
📄 子元件：`src/components/Card.vue`

<template>
  <div class="card">
    <header>
      <slot name="header">預設標題</slot>
    </header>

    <main>
      <slot>預設內容</slot>            <!-- 預設 slot、不用 name -->
    </main>

    <footer>
      <slot name="footer" />
    </footer>
  </div>
</template>
```

```text
📄 父元件

<Card>
  <template #header>
    <h2>📦 訂單 #1234</h2>
  </template>

  <!-- 不寫 #xxx 就塞進預設 slot -->
  <p>數量：3 件</p>
  <p>金額：NT$ 1,200</p>

  <template #footer>
    <button>結帳</button>
  </template>
</Card>
```

**`#header` 是 `v-slot:header` 簡寫**——把這段 HTML 塞進子元件的 `<slot name="header">` 位置。

  
### 5. Scoped Slot（範圍插槽）—— 子元件把資料回傳給父

**scoped slot** = 子元件提供「插槽」、但同時把它**內部的資料**回傳給父、讓父決定怎麼渲染：

```text
📄 子元件：`src/components/UserList.vue`

<script setup>
const users = [
  { id: 1, name: "林董", vip: true },
  { id: 2, name: "Alice", vip: false }
];
</script>

<template>
  <ul>
    <li v-for="u in users" :key="u.id">
      <!-- 把 user 傳給父 -->
      <slot :user="u" />
    </li>
  </ul>
</template>
```

```text
📄 父元件

<UserList>
  <!-- 接住子傳來的 user -->
  <template #default="{ user }">
    <strong>{{ user.name }}</strong>
    <span v-if="user.vip">⭐ VIP</span>
  </template>
</UserList>
```

**用途**：寫 reusable 元件時、子負責邏輯（資料、loop）、父負責長相。Vue 版的 render prop。

  
### 6. Provide / Inject（跨多層傳值）

**問題**：A → B → C → D → E 5 層 component、A 想傳資料給 E、用 props 要傳 5 層、累死。

**解法**：用 `provide`（A 提供）+ `inject`（E 直接拿）跳過中間層。

```text
📄 祖先元件：`src/views/App.vue`

<script setup lang="ts">
import { provide, ref } from "vue";

const theme = ref<"light" | "dark">("light");
provide("theme", theme);             // ← 把 theme 提供出去

function toggleTheme() {
  theme.value = theme.value === "light" ? "dark" : "light";
}
provide("toggleTheme", toggleTheme);
</script>
```

```text
📄 深層子孫元件：`src/components/DeepChild.vue`

<script setup lang="ts">
import { inject, type Ref } from "vue";

// inject 拿祖先 provide 的東西
const theme = inject<Ref<string>>("theme");
const toggleTheme = inject<() => void>("toggleTheme");
</script>

<template>
  <p>當前主題：{{ theme }}</p>
  <button @click="toggleTheme()">切換主題</button>
</template>
```

**TypeScript 版（推薦、用 InjectionKey）**：

```text
// keys.ts
import type { InjectionKey, Ref } from "vue";
export const themeKey = Symbol() as InjectionKey<Ref<string>>;

// 祖先
provide(themeKey, theme);

// 後代（自動推導型別）
const theme = inject(themeKey);
```

> 💡 **provide/inject vs Pinia**：跨層共享狀態小範圍用 provide/inject（一個元件樹內）、跨整個 App 用 Pinia（下一課講）。**provide/inject 不會出現在 Vue DevTools 的「全域 state」**、適合 UI 主題、表單 context 這種「跟某棵子樹綁定」的資料。

  
### 7. 4 招速查表

```text
場景                            用什麼
────────────────────────────────────────
父→子 傳資料                    props
子→父 通知事件                  emit
父⇄子 雙向綁定（表單元件）       v-model + defineModel
父→子 傳一整段 UI               slot
兄弟元件互通                    抬升 state 到共同父（用 props + emit）
跨 5+ 層共享                    provide / inject
全 App 共享（user / cart）      Pinia（下一課）
```

  
### 8. 5 大常見坑

```text
❌ 1. 在子元件直接改 props
   → Vue 警告「Avoid mutating prop」
   → 修：emit 通知父改、或子內 ref(props.x) 複製一份

❌ 2. defineProps 沒加 <ts type>
   → JS 寫法收到的全是 any、IDE 沒補全
   → 修：defineProps<Props>() 或加 generic

❌ 3. emit 名稱跟原生 HTML event 撞
   → 子 emit("click")、父 @click 收不到（被原生 click 吃了）
   → 修：emit 用語意名稱（"submit"、"select"、"close"）

❌ 4. slot 沒寫 default 內容
   → 父沒塞東西、版面空一塊
   → 修：<slot>預設文字</slot> 給 fallback

❌ 5. provide 傳 non-reactive 值（純物件）
   → 後代收到不會跟著更新
   → 修：永遠 provide ref / reactive 包過的值
```

  
### 🎯 重點回顧

- **Props** 父→子、defineProps + TS 型別、子不能改
- **Emit** 子→父、defineEmits + payload 型別、用 `@event` 接
- **v-model** 雙向、defineModel() 一行搞定（Vue 3.4+）
- **Slot** 父決定子內部塞什麼、scoped slot 還能反向傳資料
- **Provide/Inject** 跨多層共享、適合 UI context、要傳 ref / reactive
