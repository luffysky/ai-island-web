**Nuxt（讀「那克斯特」）= Vue 的全端框架**——把 Vue + Router + Pinia + SSR（伺服器端渲染）+ SSG（靜態生成）+ API routes 全打包、開箱即用。**Nuxt 之於 Vue、等於 Next.js 之於 React**——它解決「光有 Vue 不夠、還要 server 怎辦」的問題。

> 💡 **這課的價值**：Vue 自己只是「前端框架」、做全端網站要自己拼 Express + 部署。Nuxt 一個 npm 指令給你「完整全端 stack」、單人 SaaS 從這開始最快。

  
### 1. Vue 跟 Nuxt 差在哪？

```text
Vue（純前端框架）
─────────────────────
✅ component / template / reactive
✅ Vue Router（要自己接）
✅ Pinia（要自己接）
❌ SSR（自己用 Node 搭、超麻煩）
❌ SEO（爬蟲看到空 HTML）
❌ API（要另開後端專案）
❌ 檔案路由（路由要手寫）

Nuxt（Vue 的全端框架）
─────────────────────
✅ 全部 Vue 功能
✅ Router 自動（檔案 = 路由）
✅ Pinia 預設整合
✅ SSR 內建（爬蟲看得到內容）
✅ SSG 內建（產靜態 HTML）
✅ API routes（server/api/*.ts 自動變 API）
✅ Middleware / Auth / SEO meta 全包
```

**一句話**：**Vue 是引擎、Nuxt 是整台車**。引擎能跑、但車有方向盤、座椅、輪子、冷氣——直接能上路。

  
### 2. 為什麼要 SSR / SSG？

**問題：純 Vue SPA 的硬傷**

```text
使用者連網址
  → server 吐 index.html（裡面 <div id="app"></div> 是空的）
  → 瀏覽器下載 JS bundle（500KB-2MB）
  → JS 跑起來、Vue 渲染、畫面才出現

問題：
1. 慢（網路差時白屏 5 秒）
2. 爬蟲（Google / Facebook / LINE）看到空 HTML、SEO 爛、分享預覽糟
3. 第一次互動延遲（JS 還沒載完就點不到按鈕）
```

**SSR（Server-Side Rendering、伺服器端渲染）—— 在 server 把 HTML 先算好、瀏覽器收到的 HTML 就有內容**。

```text
使用者連網址
  → server 跑 Vue、把首頁 HTML 算好
  → 直接吐含內容的 HTML（爬蟲看得到、分享有預覽）
  → 同時下載 JS、做「hydration」（讓 HTML 變成可互動的 Vue）

優點：SEO 好、首屏快、分享預覽漂亮
缺點：server 負擔大
```

**SSG（Static Site Generation、靜態生成）—— build 時就把所有頁面預先算成 HTML、丟 CDN**。

```text
build 時跑一次：
  → 把每個 route 渲染成 HTML 檔（index.html / about.html / blog/post-1.html）
  → 全部丟 Cloudflare / Vercel CDN

優點：最快（CDN cache）、最便宜（不用 server）、SEO 滿分
缺點：內容變了要重 build（適合不常變的：部落格、產品頁、文件）
```

**hydration（水合）**—— SSR 把純 HTML「澆水」變成可互動的 Vue（綁定事件、reactive 接上）的過程。聽起來玄、就是「把死 HTML 變活」。

### 4. 檔案路由（File-based routing）—— 不用寫 router

**Nuxt 看 `pages/` 資料夾自動建路由**——你只要建檔案、router 不用寫：

```text
pages/
├── index.vue              → /
├── about.vue              → /about
├── login.vue              → /login
├── blog/
│   ├── index.vue          → /blog
│   └── [slug].vue         → /blog/:slug（[slug] 是動態參數）
├── user-[id].vue          → /user-:id
└── admin/
    └── [...catchAll].vue  → /admin/* 全部接（catch-all）
```

```text
📄 pages/blog/[slug].vue

<script setup>
const route = useRoute();
const slug = route.params.slug;
</script>

<template>
  <h1>文章：{{ slug }}</h1>
</template>
```

**好處**：團隊任何人看 `pages/` 就知道有哪些路由、不用翻 router config 檔。

  
### 5. `useFetch` —— SSR 友善的資料抓取

`fetch` 在純 Vue 寫 `onMounted(async () => ...)` ——但 SSR 場景下 server 跟 client 都要抓、容易抓兩次。Nuxt 給你 `useFetch`：

```text
📄 pages/users.vue

<script setup>
// SSR 時 server 抓、client hydration 時不重複抓
const { data: users, pending, error, refresh } = await useFetch("/api/users");
</script>

<template>
  <p v-if="pending">載入中...</p>
  <p v-else-if="error">錯誤：{{ error.message }}</p>
  <ul v-else>
    <li v-for="u in users" :key="u.id">{{ u.name }}</li>
  </ul>

  <button @click="refresh">重新抓</button>
</template>
```

**`useFetch` 自動處理**：
- SSR：server 抓 → 把資料塞進 HTML → client 不重複抓
- cache：同個 key 不重複請求
- loading state：`pending`
- error state：`error`
- refetch：`refresh()`

  
### 6. Server API（檔案系統路由的後端）

**Nuxt 把 `server/api/` 內的檔案自動變 API endpoint**——前後端同一個專案：

```text
server/api/
├── users.get.ts         → GET /api/users
├── users.post.ts        → POST /api/users
├── users/[id].get.ts    → GET /api/users/:id
└── login.post.ts        → POST /api/login
```

```text
📄 server/api/users.get.ts

export default defineEventHandler(async (event) => {
  const db = useDB();             // 你的 DB 連線
  const users = await db.query("SELECT * FROM users");
  return users;                    // 自動轉 JSON 回傳
});
```

```text
📄 server/api/users/[id].get.ts

export default defineEventHandler(async (event) => {
  const id = getRouterParam(event, "id");
  const user = await db.query("SELECT * FROM users WHERE id = ?", id);
  if (!user) throw createError({ statusCode: 404, message: "Not found" });
  return user;
});
```

```text
📄 server/api/login.post.ts

export default defineEventHandler(async (event) => {
  const body = await readBody(event);
  const user = await login(body.email, body.password);
  return { user, token: signJWT(user) };
});
```

**前端 `useFetch("/api/users")` 直接打到 server/api/users.get.ts**——同個專案、不用跨 origin、不用 CORS。

  
### 7. SSR / SSG / Hybrid（混合模式）

Nuxt 給你細粒度控制——**每條路由可以選不同渲染方式**：

```text
📄 nuxt.config.ts

export default defineNuxtConfig({
  // 全站預設 SSR
  ssr: true,

  // 但分頁不同策略
  routeRules: {
    "/blog/**":      { static: true },        // 部落格用 SSG（build 時生成）
    "/dashboard":    { ssr: false },          // 後台用純 SPA（不需要 SEO）
    "/":             { isr: 3600 },           // 首頁 ISR（每小時重新生成）
    "/api/**":       { cors: true },          // API 加 CORS
    "/admin/**":     { robots: false }        // admin 不給 Google 爬
  }
});
```

**4 種策略**：
- **SSR（預設）**—— 每次 request 都跑、最新但耗 server
- **SSG（`static: true`）**—— build 時生成 HTML、最快最便宜
- **ISR（`isr: 秒數`）**—— Incremental Static Regeneration、定期重新生成
- **SPA（`ssr: false`）**—— 純前端、不 SEO、適合後台

  
### 8. Nuxt vs Next.js 對比

| 項目 | Nuxt（Vue） | Next.js（React） |
| --- | --- | --- |
| 檔案路由 | `pages/` | `pages/` 或 `app/` |
| 資料抓取 | `useFetch` | `fetch` in Server Component |
| API routes | `server/api/*.ts` | `pages/api/` 或 `app/api/route.ts` |
| SSR / SSG | routeRules 控制 | Server Components / generateStaticParams |
| 部署 | Vercel / Cloudflare / Netlify | Vercel（親爸）/ 其他 |
| 社群規模 | 中（亞洲為主） | 大（全球） |
| Server Components | 跟進中 | 領先 |
| 學習曲線 | 中等 | 中等 |

**核心理念都一樣**——只是 Vue 跟 React 的語法差。


### 11. 5 大常見坑

```text
❌ 1. useFetch 沒 await
   → SSR 時資料還沒到、HTML 沒內容
   → 修：永遠 const { data } = await useFetch(...)

❌ 2. 在 server/api 內 import vue / 瀏覽器 API
   → server 沒有 window / document、報錯
   → 修：server 端只用 Node API（不 import vue）

❌ 3. SSR 時用 window / localStorage
   → server 沒這些、報錯
   → 修：if (process.client) { ... } 或用 useState（SSR 安全）

❌ 4. 用 useFetch 在 onMounted 內
   → onMounted 只在 client 跑、SSR 沒抓到、SEO 失效
   → 修：直接在 script setup 頂層 await useFetch

❌ 5. 大量 API call 用 $fetch 而不是 useFetch
   → $fetch 不 cache、SSR 期間每次 render 都重 call
   → 修：頁面初始抓資料用 useFetch、事件觸發用 $fetch
```

  
### 🎯 重點回顧

- **Nuxt = Vue 的全端框架**、相當於 React 的 Next.js
- **SSR / SSG / ISR** 三種渲染策略、routeRules 一條條設
- **檔案路由**：`pages/about.vue` → `/about`、`[id].vue` → `/:id`
- **`useFetch`** 是 SSR 友善的抓資料工具、頁面初始用它
- **`server/api/*.ts`** 自動變 API endpoint、前後端同專案
- **Hydration（水合）** = server HTML 變 client 可互動 Vue 的過程
- **Vercel / Cloudflare Pages** 部署最簡單
