**lifecycle（生命週期）= 一個元件「出生 → 顯示 → 更新 → 死掉」的各個時間點**——Vue 給你 hook（鉤子函式）讓你在每個時間點插入自己的程式碼。最常用的兩個：**`onMounted`（畫面顯示後立刻跑）**、**`onUnmounted`（畫面被銷毀前跑、清理用）**。配合 **Composition API（組合式 API）** 把這些 hook 包成 **composable（可複用函式）**，是現代 Vue 寫法的核心。

> 💡 **這課的價值**：搞懂生命週期才能正確地「抓資料、設 timer、訂閱事件、清理資源」——不會抓完資料 component 卻已經消失、不會 timer 沒清造成記憶體洩漏。

  
### 1. 生命週期 7 個時間點

```text
🥚 setup                  ← script setup 整段在這跑（最早）
  ↓
🐣 onBeforeMount          ← DOM 還沒掛上去
  ↓
🐤 onMounted              ← DOM 已掛上、瀏覽器看得到了 ★★★ 最常用
  ↓
🐔 onBeforeUpdate         ← reactive data 變了、DOM 還沒更新
  ↓
🦅 onUpdated              ← DOM 已重新渲染
  ↓
💀 onBeforeUnmount        ← 元件即將被銷毀
  ↓
👻 onUnmounted            ← 元件已銷毀（清理 timer / listener）★★ 第二常用
```

  
### 2. 最常用：`onMounted`（畫面顯示後抓資料）

```text
📄 寫進程式檔：`src/components/UserList.vue`

<script setup lang="ts">
import { ref, onMounted } from "vue";

interface User { id: number; name: string }

const users = ref<User[]>([]);
const loading = ref(true);
const error = ref<string | null>(null);

// onMounted 在元件「掛上 DOM 後立刻」執行一次
onMounted(async () => {
  try {
    const res = await fetch("/api/users");
    users.value = await res.json();
  } catch (e: any) {
    error.value = e.message;
  } finally {
    loading.value = false;
  }
});
</script>

<template>
  <p v-if="loading">載入中...</p>
  <p v-else-if="error">錯誤：{{ error }}</p>
  <ul v-else>
    <li v-for="u in users" :key="u.id">{{ u.name }}</li>
  </ul>
</template>
```

**用法總結**：抓 API、操作 DOM（如取得 element 寬高）、設 setInterval、訂閱 WebSocket / EventBus。

  
### 3. 第二常用：`onUnmounted`（清理資源）

**沒清理會怎樣**：使用者切到別頁、元件消失、但 setInterval 還在跑、WebSocket 還在連、記憶體洩漏（leak）越積越多、瀏覽器越來越慢。

```text
<script setup>
import { onMounted, onUnmounted } from "vue";

let timer: ReturnType<typeof setInterval>;

onMounted(() => {
  timer = setInterval(() => {
    console.log("每秒一次");
  }, 1000);

  window.addEventListener("resize", handleResize);
});

onUnmounted(() => {
  clearInterval(timer);                                // ← 清 timer
  window.removeEventListener("resize", handleResize);   // ← 清 listener
});

function handleResize() {
  console.log("視窗大小變了");
}
</script>
```

> 💡 **規則**：**任何在 onMounted「訂閱」的東西**（timer、listener、subscription）、**必須在 onUnmounted 對應「取消」**。這是寫前端最容易漏的地方。

  
### 4. 7 個 hook 一覽

```text
🟢 onBeforeMount    DOM 還沒掛、看不到畫面     罕用
🟢 onMounted        DOM 已掛、可以操作 element  ★★★ 抓資料、初始化
🟡 onBeforeUpdate   reactive data 變了未更新   除錯用
🟡 onUpdated        DOM 已重新渲染              罕用（容易誤用變無限迴圈）
🔴 onBeforeUnmount  即將銷毀、還能訪問 DOM      罕用
🔴 onUnmounted      已銷毀、要清資源            ★★ 清 timer / listener
⚠️ onErrorCaptured  子元件錯誤捕捉              做 ErrorBoundary
🔄 onActivated      <KeepAlive> 切回來時觸發    cache 元件用
🔄 onDeactivated    <KeepAlive> 切走時觸發       cache 元件用
```

**setup 本身 = beforeCreate + created（Vue 2 的概念）**——script setup 整段就是「最早」。

  
### 5. Composable（組合式函式）—— Vue 版的 Custom Hook

**痛點**：5 個元件都要「計數 + 增 + 減 + 重設」、複製貼上 5 次 code、難維護。

**解法**：抽成一個 `useCounter()` 函式、5 個元件 import 即可。

```text
📄 寫進程式檔：`src/composables/useCounter.ts`

import { ref } from "vue";

export function useCounter(initial = 0) {
  const count = ref(initial);

  const increment = () => count.value++;
  const decrement = () => count.value--;
  const reset = () => (count.value = initial);

  return { count, increment, decrement, reset };
}
```

```text
📄 用：`src/components/CounterA.vue`

<script setup>
import { useCounter } from "@/composables/useCounter";

const { count, increment, decrement, reset } = useCounter(10);
</script>

<template>
  <p>{{ count }}</p>
  <button @click="increment">+1</button>
  <button @click="decrement">-1</button>
  <button @click="reset">重設</button>
</template>
```

**3 個 composable 慣例**：
1. **名字以 `use` 開頭**（useXxx）—— 跟 React Hook 一樣
2. **回傳物件**（讓使用者用解構拿）
3. **要 reactive 的值用 `ref`**（包進去才會自動更新）

  
### 6. 實戰 composable：`useMouse`（追滑鼠位置）

```text
📄 寫進程式檔：`src/composables/useMouse.ts`

import { ref, onMounted, onUnmounted } from "vue";

export function useMouse() {
  const x = ref(0);
  const y = ref(0);

  function update(e: MouseEvent) {
    x.value = e.clientX;
    y.value = e.clientY;
  }

  onMounted(() => window.addEventListener("mousemove", update));
  onUnmounted(() => window.removeEventListener("mousemove", update));

  return { x, y };
}
```

```text
📄 用：

<script setup>
import { useMouse } from "@/composables/useMouse";
const { x, y } = useMouse();
</script>

<template>
  <p>滑鼠在：({{ x }}, {{ y }})</p>
</template>
```

**重點**：composable 內部可以用所有 Vue API（ref / onMounted / watch / computed）、相當於把「component 的一段邏輯」打包成函式。

  
### 7. VueUse —— 200+ composables 大全

**VueUse** 是 Vue 生態的瑞士刀——把所有常見邏輯（localStorage、debounce、暗色模式、剪貼簿、IntersectionObserver）都包好了：

```text
🖥️ 在終端機輸入

pnpm add @vueuse/core
```

```text
<script setup>
import {
  useLocalStorage,        // localStorage 自動同步
  useDebounce,            // 防抖
  useDark,                // 暗色模式
  useTitle,               // 改 document.title
  useClipboard,           // 剪貼簿
  useIntersectionObserver,// 元素進視窗
  useMouse,               // 滑鼠位置
  useFetch                // fetch with reactive

const isDark = useDark();                    // ← 一行搞定暗色模式
const theme = useLocalStorage("theme", "light"); // ← 自動存 localStorage

</script>
```

> 💡 **VueUse 是 2026 新專案必裝**——React 沒有對等的官方庫、要自己拼很多 npm 包。Vue 在這方面贏一截。

  
### 8. Composition API vs Options API（舊寫法）

**舊：Options API（選項式 API、Vue 2 時代）**

```text
export default {
  data() {
    return { count: 0 };
  },
  methods: {
    increment() { this.count++; }
  },
  mounted() {
    console.log("mounted");
  }
};
```

**新：Composition API（組合式 API、Vue 3 推薦）**

```text
<script setup>
import { ref, onMounted } from "vue";
const count = ref(0);
function increment() { count.value++; }
onMounted(() => console.log("mounted"));
</script>
```

**為什麼新的好**：
- 邏輯**可以抽成 composable** 跨元件複用
- 沒有 `this` 指向問題
- TypeScript 推導更精準
- 大元件不會被 `data/methods/computed` 切碎、相關邏輯能寫一起

**2026 新專案 99% 用 Composition API**——遇到 Options API 的 code 大多是 Vue 2 留下來的。

  
### 9. 5 大常見坑

```text
❌ 1. onMounted 內忘記清資源
   → setInterval / EventListener 不停跑、記憶體洩漏
   → 修：對應 onUnmounted 寫清理

❌ 2. composable 名字不是 use 開頭
   → 看起來像普通 function、團隊看不出來
   → 修：永遠 useXxx 命名

❌ 3. composable 回傳「拆出來的 reactive 屬性」
   → const { count } = reactive({count:0}) → 失去響應性
   → 修：用 ref + storeToRefs / toRefs 解構

❌ 4. onMounted 內用 ref(component) 取 DOM 卻拿到 null
   → ref 寫了但 template 沒綁、或寫錯名稱
   → 修：const el = ref(null) + <div ref="el"> + 在 onMounted 訪問 el.value

❌ 5. 在 onUpdated 內改 reactive data
   → 觸發新的 update、再進 onUpdated、無限迴圈
   → 修：用 watch 監聽特定值、不要無腦改
```

  
### 🎯 重點回顧

- **lifecycle 7 個時間點**、常用 `onMounted`（抓資料）+ `onUnmounted`（清資源）
- **「訂閱」一定要對應「取消」**——timer、listener、subscription 都要
- **Composable** = 把 ref/onMounted/watch 包成 `useXxx()` 函式、跨元件複用
- **VueUse** = 200+ composables 大全、2026 必裝
- **Composition API > Options API**——新專案統一用 `<script setup>`
