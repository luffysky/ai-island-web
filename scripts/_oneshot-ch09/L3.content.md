**Template（模板）是 Vue 的招牌**——它看起來就是 HTML、但塞了 Vue 特殊語法（叫 **directive、指令**、用 `v-` 開頭）讓資料能跑進畫面、條件能切換、列表能渲染。學會 7 個指令 + 3 種綁定、你就能寫出 90% 的 Vue 畫面。

> 💡 **這課的價值**：React 用 JSX（HTML 寫在 JS 裡）、Vue 用 template（HTML 寫在 HTML 裡）。**Vue 的 template 更接近原生 HTML、設計師也看得懂**——這就是亞洲市場特愛 Vue 的核心理由。

  
### 1. Interpolation（插值）`{{ }}` —— 把資料塞進 HTML

```text
📄 寫進程式檔：`src/components/Greet.vue`

<script setup>
import { ref } from "vue";
const name = ref("林董");
const year = ref(2026);
</script>

<template>
  <h1>哈囉、{{ name }}！</h1>
  <p>現在是 {{ year }} 年</p>
  <p>明年是 {{ year + 1 }} 年</p>
  <p>大寫：{{ name.toUpperCase() }}</p>
</template>
```

**interpolation（插值、念「英特爾波雷遜」）**—— 把 JS 變數 / 算式塞進 HTML 裡的雙大括號 `{{ }}`。可以放：
- 變數：`{{ name }}`
- 算式：`{{ year + 1 }}`
- 函式呼叫：`{{ formatDate(date) }}`
- 三元運算：`{{ isOK ? "✅" : "❌" }}`

**不能放**：if 語句、for 迴圈、宣告變數（那些用指令解決）。

  
### 2. `v-bind`（綁屬性）—— 動態 class / src / href

要把資料塞進 HTML **屬性**裡（不是內容）、用 `v-bind:屬性` 或簡寫 `:屬性`：

```text
<script setup>
import { ref } from "vue";
const imgUrl = ref("/cat.jpg");
const link = ref("https://ai-island.tw");
const isActive = ref(true);
</script>

<template>
  <!-- 完整寫法 -->
  <img v-bind:src="imgUrl" />

  <!-- 簡寫（99% 都用這個） -->
  <img :src="imgUrl" />
  <a :href="link">前往</a>

  <!-- class 動態切換 -->
  <button :class="{ active: isActive }">按我</button>

  <!-- class 多個條件 -->
  <button :class="['btn', isActive ? 'active' : 'disabled']">按我</button>

  <!-- inline style -->
  <div :style="{ color: 'red', fontSize: '20px' }">紅字</div>
</template>
```

> 💡 **`:class` 跟 `class` 可以並存**：`<button class="btn" :class="{ active: isActive }">` —— Vue 會自動合併、靜態跟動態都保留。

  
### 3. `v-on`（綁事件）—— 處理點擊、輸入、滑鼠

要綁事件、用 `v-on:event` 或簡寫 `@event`：

```text
<script setup>
import { ref } from "vue";
const count = ref(0);

function increment() {
  count.value++;
}

function logKey(e) {
  console.log("按了：", e.key);
}
</script>

<template>
  <!-- 完整寫法 -->
  <button v-on:click="increment">+1</button>

  <!-- 簡寫（99% 都用這個） -->
  <button @click="increment">+1</button>

  <!-- inline 函式 -->
  <button @click="count++">+1（直接寫）</button>

  <!-- 傳參數 -->
  <button @click="increment(5)">+5</button>

  <!-- 拿 event 物件 -->
  <input @keydown="logKey" />

  <!-- 事件修飾符（Vue 強項） -->
  <form @submit.prevent="handleSubmit">...</form>
  <a @click.stop="onClick">不冒泡</a>
  <input @keyup.enter="onEnter" />
</template>
```

**事件修飾符（modifier、用 `.` 接）—— Vue 比 React 簡潔的地方**：

```text
.prevent      → e.preventDefault()（阻止預設、表單最常用）
.stop         → e.stopPropagation()（阻止冒泡）
.once         → 只觸發一次
.self         → 只有點到自己才觸發（不算冒泡來的）
.capture      → 用捕獲模式
.enter / .esc / .tab / .space → 按特定鍵才觸發
```

> 💡 **`@submit.prevent` 一行幹掉 React 的 `e.preventDefault()`**—— 這就是亞洲團隊喊「Vue 寫起來舒服」的具體例子。

  
### 4. `v-if` / `v-else-if` / `v-else`（條件渲染）

```text
<script setup>
import { ref } from "vue";
const score = ref(85);
const user = ref(null);   // null = 沒登入
</script>

<template>
  <p v-if="score >= 90">A 等</p>
  <p v-else-if="score >= 80">B 等</p>
  <p v-else-if="score >= 70">C 等</p>
  <p v-else>需要加油</p>

  <!-- 用 <template> 包多個元素（不會渲染額外 wrapper） -->
  <template v-if="user">
    <h1>歡迎、{{ user.name }}</h1>
    <p>VIP 會員</p>
  </template>
  <p v-else>請先登入</p>
</template>
```

**`v-if` vs `v-show` 差在哪**：

```text
v-if="cond"     → cond 為 false 時、元素「完全不渲染」（DOM 內沒有）
   → 適合：很少切換 / 條件失敗就不該存在的東西

v-show="cond"   → cond 為 false 時、加 display: none 隱藏（DOM 內還在）
   → 適合：常常切換的東西（不用一直建立/銷毀、效能好）
```

  
### 5. `v-for`（列表渲染）—— 一定要寫 `:key`

```text
<script setup>
import { ref } from "vue";
const todos = ref([
  { id: 1, title: "學 Vue" },
  { id: 2, title: "寫 SaaS" },
  { id: 3, title: "上線賺錢" }
]);
</script>

<template>
  <!-- 渲染陣列 -->
  <ul>
    <li v-for="todo in todos" :key="todo.id">
      {{ todo.title }}
    </li>
  </ul>

  <!-- 同時拿 index -->
  <ul>
    <li v-for="(todo, i) in todos" :key="todo.id">
      第 {{ i + 1 }} 件：{{ todo.title }}
    </li>
  </ul>

  <!-- 渲染物件 -->
  <ul>
    <li v-for="(value, key) in { name: '林董', age: 30 }" :key="key">
      {{ key }}: {{ value }}
    </li>
  </ul>

  <!-- 渲染數字（1-5） -->
  <span v-for="n in 5" :key="n">{{ n }} </span>
</template>
```

> 💡 **`:key` 是什麼、為什麼必須寫**：Vue 用 key 來識別「哪個 item 是哪個」、更新時才能正確地新增 / 刪除 / 移動而不是整列重建。**不寫 key 列表會出 bug**（順序錯、表單值跳掉）。**key 要用「穩定唯一的 id」**、不要用陣列 index（重新排序就壞了）。

  
### 6. `v-model`（雙向綁定）—— Vue 的招牌

`v-model` 把表單元素跟資料**雙向同步**——使用者輸入就更新資料、資料變了畫面也跟著變：

```text
<script setup>
import { ref } from "vue";
const name = ref("");
const checked = ref(false);
const selected = ref("tw");
const message = ref("");
</script>

<template>
  <!-- text input -->
  <input v-model="name" placeholder="輸入名字" />
  <p>你好、{{ name || "陌生人" }}</p>

  <!-- checkbox -->
  <input type="checkbox" v-model="checked" />
  <p>勾了嗎：{{ checked }}</p>

  <!-- radio -->
  <input type="radio" v-model="selected" value="tw" /> 台灣
  <input type="radio" v-model="selected" value="jp" /> 日本

  <!-- select -->
  <select v-model="selected">
    <option value="tw">台灣</option>
    <option value="jp">日本</option>
  </select>

  <!-- textarea -->
  <textarea v-model="message"></textarea>

  <!-- v-model 修飾符 -->
  <input v-model.trim="name" />       <!-- 自動去頭尾空白 -->
  <input v-model.number="age" />      <!-- 自動轉數字 -->
  <input v-model.lazy="search" />     <!-- 失焦才更新（不是即時） -->
</template>
```

**React 對比**：React 沒有 v-model、要自己寫 `value={x}` + `onChange={e => setX(e.target.value)}` 兩行。`v-model` 一個就解決——這也是 Vue 簡潔的代表特性。

  
### 7. 完整實戰：可勾選 Todo List

```text
📄 寫進程式檔：`src/components/Todo.vue`

<script setup>
import { ref } from "vue";

const newTitle = ref("");
const todos = ref([
  { id: 1, title: "學 Vue template", done: false }
]);

function add() {
  if (!newTitle.value.trim()) return;
  todos.value.push({
    id: Date.now(),
    title: newTitle.value,
    done: false
  });
  newTitle.value = "";
}

function remove(id) {
  todos.value = todos.value.filter(t => t.id !== id);
}
</script>

<template>
  <h2>Todo List</h2>

  <form @submit.prevent="add">
    <input v-model.trim="newTitle" placeholder="今天要做什麼？" />
    <button type="submit">新增</button>
  </form>

  <p v-if="todos.length === 0">尚無待辦</p>

  <ul v-else>
    <li v-for="t in todos" :key="t.id">
      <input type="checkbox" v-model="t.done" />
      <span :class="{ done: t.done }">{{ t.title }}</span>
      <button @click="remove(t.id)">刪除</button>
    </li>
  </ul>
</template>

<style scoped>
.done {
  text-decoration: line-through;
  color: #999;
}
</style>
```

**這個範例用到所有指令**：`v-model`（input + checkbox）、`@submit.prevent`（form）、`v-if/v-else`（空狀態）、`v-for + :key`（列表）、`:class`（動態樣式）、`@click`（事件）。

  
### 8. 5 大常見坑

```text
❌ 1. v-for 忘記寫 :key
   → 列表更新會出怪 bug（表單值跳到別 row）
   → 修：永遠寫 :key="item.id"

❌ 2. :key 用 index
   → :key="i" 排序 / 刪除中間項時整列重建
   → 修：用穩定 id、不要用 index

❌ 3. v-if 跟 v-for 同層
   → Vue 3 直接報錯（v-for 跟 v-if 不要混）
   → 修：先 computed 過濾陣列、再 v-for

❌ 4. {{ }} 寫在屬性裡
   → <img src="{{ url }}"> 不會生效
   → 修：用 :src="url"（屬性要 v-bind）

❌ 5. v-model 綁 prop（從父元件來的值）
   → Vue 警告「不能改 prop」
   → 修：用 defineModel()（Vue 3.4+）或 emit 給父元件改
```

  
### 🎯 重點回顧

- **`{{ }}` 插值** 把資料塞進 HTML 內容
- **`:屬性`（v-bind）** 把資料塞進 HTML 屬性、`:class` / `:style` 最常用
- **`@事件`（v-on）** 綁事件、`.prevent / .stop / .enter` 修飾符超好用
- **`v-if / v-else`** 條件渲染、**`v-show`** 是 display:none 切換（常切換用）
- **`v-for` 必配 `:key`**、key 要穩定唯一 id
- **`v-model`** 雙向綁定、Vue 招牌、React 沒有
