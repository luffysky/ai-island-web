**Composition API（組合式 API）= Vue 3 的核心新寫法**——把 state、邏輯、生命週期全部「組合」在一個 `<script setup>` 區塊內、邏輯能抽成 composable 跨元件複用。這課把 `ref` / `reactive` / `computed` / `watch` / `watchEffect` 五大 API 一次講完、外加自訂 composable 實戰。

> 💡 **這課的價值**：搞懂 ref vs reactive 該用哪個、computed vs watch 該用哪個、watchEffect 跟 watch 差在哪—— 寫 Vue 3 不卡關的關鍵知識。

  
### 1. `<script setup>` 是什麼

**舊寫法（不要學、認得就好）**：

```text
<script>
import { ref } from "vue";

export default {
  setup() {
    const count = ref(0);
    return { count };
  }
};
</script>
```

**新寫法（2026 99% 用這個）**：

```text
<script setup>
import { ref } from "vue";

const count = ref(0);
// 不用寫 export、不用 return、頂層宣告自動暴露給 template
</script>

<template>
  <p>{{ count }}</p>
</template>
```

**`<script setup>`** 是 Vue 3.2+ 的 syntax sugar（語法糖）—— 編譯器幫你生 setup function、把所有 import / const / function 自動 expose 給 template 用。**少寫 80% 樣板碼**。

  
### 2. `ref` —— 萬用響應式變數

`ref` 把任何值（數字、字串、物件、陣列）變成「會自動讓畫面跟著變」的響應式變數：

```text
<script setup>
import { ref } from "vue";

const count = ref(0);
const name = ref("林董");
const user = ref({ id: 1, name: "林董" });
const list = ref<string[]>([]);
</script>
```

**規則**：
- 在 script 內存取 / 修改 → 用 `.value`：`count.value++`、`user.value.name = "..."`
- 在 template 內 → **不用** `.value`（Vue 自動解包）：`{{ count }}`

```text
<script setup>
const count = ref(0);

function add() {
  count.value++;            // ← 必須 .value
  console.log(count);       // 印 RefImpl { ... }
  console.log(count.value); // 印 1
}
</script>

<template>
  <p>{{ count }}</p>          <!-- 不用 .value -->
  <button @click="count++">+1</button>  <!-- template 內也不用 -->
</template>
```

  
### 3. `reactive` —— 只能用在物件 / 陣列

`reactive` 也是響應式、但**只接受物件 / 陣列、不能用在原始值**（數字、字串、布林）：

```text
<script setup>
import { reactive } from "vue";

const user = reactive({ name: "林董", age: 30 });
const list = reactive<string[]>([]);

// ❌ 錯：reactive 不能用原始值
// const count = reactive(0);

function update() {
  user.name = "Alice";      // ← 不用 .value
  user.age++;
  list.push("新項目");
}
</script>
```

  
### 4. `ref` vs `reactive` 怎麼選？

**官方建議：永遠用 `ref`、不要混**——理由：

```text
✅ ref 的優點
   - 數字 / 字串 / 物件 / 陣列 全部都能用、寫法統一
   - 整包能傳給 composable / Pinia
   - 解構不會失去響應性（用 toRefs）

❌ reactive 的坑
   - 只能用在物件、原始值不行
   - 解構就死：const { name } = reactive(user) → name 失去響應性
   - 重新賦值就死：user = newUser → 失去響應（必須一個一個改屬性）
   - 跟 ref 混用容易搞混

→ 結論：99% 場景用 ref、reactive 留給「絕對不會解構也不會重新賦值的大物件」（如 form state）
```

```text
✅ 推薦寫法

<script setup>
import { ref } from "vue";

const count = ref(0);                                    // 數字
const name = ref("");                                    // 字串
const user = ref({ id: 1, name: "林董" });                // 物件
const list = ref<Todo[]>([]);                            // 陣列

count.value++;
user.value.name = "Alice";
list.value.push(newTodo);
</script>
```

  
### 5. `computed` —— 衍生值（從其他 reactive 算出來）

**場景**：你有 `firstName` + `lastName`、想要 `fullName`——不要寫成普通 function、用 `computed`：

```text
<script setup>
import { ref, computed } from "vue";

const firstName = ref("林");
const lastName = ref("Luffy");
const cart = ref([
  { name: "雞排", price: 65, qty: 2 },
  { name: "珍奶", price: 50, qty: 1 }
]);

// 衍生：完整名字
const fullName = computed(() => `${firstName.value}${lastName.value}`);

// 衍生：購物車總價
const total = computed(() =>
  cart.value.reduce((sum, item) => sum + item.price * item.qty, 0)
);

// 衍生：購物車是否空的
const isEmpty = computed(() => cart.value.length === 0);
</script>

<template>
  <p>{{ fullName }} — NT$ {{ total }}</p>
  <p v-if="isEmpty">購物車空了</p>
</template>
```

**`computed` 的特性**：
- **自動追蹤依賴**：firstName / lastName / cart 變了、自動重算
- **cache**：值沒變不重算（呼叫 100 次只算一次）
- **lazy**：沒人讀就不算
- **readonly**（預設）：不能直接改、但可以寫 getter + setter

  
### 6. `computed` vs 普通 function 差在哪

```text
普通 function
─────────────
function totalFn() {
  return cart.value.reduce(...);
}

每次呼叫 totalFn() 都重新算
template 內 {{ totalFn() }} 每次 render 都算一次

computed
─────────────
const total = computed(() => cart.value.reduce(...));

第一次讀 total.value 時算一次
之後讀 100 次都用 cache、不重算
cart 變了才重算
```

> 💡 **規則**：**任何「依賴 reactive 算出來的值」都用 computed**——比 function 快、自動追蹤、寫起來簡潔。

  
### 7. `watch` —— 監聽變化、執行副作用

`watch` 跟 `computed` 不同——`computed` 是「算出新值」、`watch` 是「值變了去做某件事」（API 請求、localStorage 存、route 換頁）：

```text
<script setup>
import { ref, watch } from "vue";

const search = ref("");

// 監聽單一值
watch(search, (newVal, oldVal) => {
  console.log(`從 "${oldVal}" 變 "${newVal}"`);
  fetchResults(newVal);
});

// 監聽多個值（用陣列）
const page = ref(1);
const size = ref(20);
watch([page, size], ([newPage, newSize], [oldPage, oldSize]) => {
  fetchPage(newPage, newSize);
});

// 監聽物件「裡面的」變化（要 deep）
const user = ref({ name: "", age: 0 });
watch(user, () => {
  console.log("user 內部某個屬性變了");
}, { deep: true });

// 監聽特定屬性（用 getter）
watch(() => user.value.name, (newName) => {
  console.log("name 變了：", newName);
});

// 立刻執行一次（不等第一次變化）
watch(search, (val) => {
  fetchResults(val);
}, { immediate: true });
</script>
```

**`watch` 的常用 options**：
- **`deep: true`** —— 監聽物件深層變化（預設只比較頂層 reference）
- **`immediate: true`** —— 註冊時立刻跑一次（不等變化）
- **`flush: "post"`** —— 等 DOM 更新後才跑（罕用）

  
### 8. `watchEffect` —— 自動追蹤、不用宣告依賴

`watch` 要明寫「監聽什麼」、`watchEffect` 自動偵測「函式內用到的 reactive」並追蹤：

```text
<script setup>
import { ref, watchEffect } from "vue";

const search = ref("");
const page = ref(1);

watchEffect(() => {
  // 自動追蹤：用到 search.value + page.value
  // search 或 page 變了就重跑
  fetchResults(search.value, page.value);
});

// 第一次註冊時也會跑一次（內建 immediate）
</script>
```

**`watch` vs `watchEffect`**：

```text
watch
─────
✅ 明確指定要監聽什麼（安全、可預測）
✅ 拿得到 newVal / oldVal
✅ 預設不立刻跑（要 immediate: true）
❌ 寫法略長

watchEffect
─────
✅ 寫法簡潔、自動追蹤
✅ 預設立刻跑一次
❌ 拿不到 newVal / oldVal
❌ 容易「不知不覺多追蹤了東西」造成多次重跑
```

**選擇**：
- **要 oldVal / 要 immediate 控制 → watch**
- **單純「值變了重跑」→ watchEffect**

  
### 9. 自訂 composable 完整實戰：`useTodos`

```text
📄 寫進程式檔：`src/composables/useTodos.ts`

import { ref, computed, watch } from "vue";

interface Todo {
  id: number;
  title: string;
  done: boolean;
}

export function useTodos() {
  const todos = ref<Todo[]>(
    JSON.parse(localStorage.getItem("todos") || "[]")
  );
  const filter = ref<"all" | "active" | "done">("all");

  // 自動存 localStorage
  watch(todos, (val) => {
    localStorage.setItem("todos", JSON.stringify(val));
  }, { deep: true });

  // 衍生：過濾後的列表
  const filtered = computed(() => {
    if (filter.value === "active") return todos.value.filter(t => !t.done);
    if (filter.value === "done") return todos.value.filter(t => t.done);
    return todos.value;
  });

  // 衍生：未完成數量
  const activeCount = computed(() =>
    todos.value.filter(t => !t.done).length
  );

  function add(title: string) {
    todos.value.push({ id: Date.now(), title, done: false });
  }

  function toggle(id: number) {
    const t = todos.value.find(t => t.id === id);
    if (t) t.done = !t.done;
  }

  function remove(id: number) {
    todos.value = todos.value.filter(t => t.id !== id);
  }

  return { todos, filter, filtered, activeCount, add, toggle, remove };
}
```

```text
📄 用：`src/views/TodoView.vue`

<script setup>
import { useTodos } from "@/composables/useTodos";

const { todos, filter, filtered, activeCount, add, toggle, remove } = useTodos();
</script>

<template>
  <h1>Todo（剩 {{ activeCount }} 件）</h1>

  <select v-model="filter">
    <option value="all">全部</option>
    <option value="active">未完成</option>
    <option value="done">已完成</option>
  </select>

  <ul>
    <li v-for="t in filtered" :key="t.id">
      <input type="checkbox" :checked="t.done" @change="toggle(t.id)" />
      <span :class="{ done: t.done }">{{ t.title }}</span>
      <button @click="remove(t.id)">刪</button>
    </li>
  </ul>
</template>
```

**這個 composable 用到所有 API**：ref、computed、watch、localStorage 整合——120 行內做出可上線的 todo app。

  
### 10. 5 大常見坑

```text
❌ 1. ref 忘記 .value
   → count + 1 而不是 count.value + 1、結果是 "[object]1"
   → 修：script 內永遠 .value、template 內不用

❌ 2. 解構 reactive 物件
   → const { name } = reactive(user) → name 失去響應性
   → 修：用 toRefs：const { name } = toRefs(user)、或統一用 ref

❌ 3. computed 內塞副作用（fetch、改別的 ref）
   → computed 應該只「算」、不該「做」
   → 修：副作用用 watch / watchEffect

❌ 4. watch 物件沒加 deep
   → user.name 變了 watch 沒觸發
   → 修：watch(user, fn, { deep: true })、或用 getter watch(() => user.name, fn)

❌ 5. watchEffect 內條件式讀取 ref
   → if (cond) { ... ref.value } → cond 為 false 時不追蹤 ref
   → 修：把所有依賴讀在外面、或用 watch 明確指定
```

  
### 🎯 重點回顧

- **`<script setup>`** 是 Vue 3 標準寫法、頂層自動 expose 給 template
- **`ref` 99% 場景用它**、script 內 `.value`、template 內不用
- **`reactive`** 只能用在物件、容易解構 / 重新賦值翻車、能不用就不用
- **`computed`** 衍生值、自動追蹤 + cache、優於普通 function
- **`watch`** 明確指定、有 newVal/oldVal；**`watchEffect`** 自動追蹤、立刻跑
- **composable** = 把 ref+computed+watch+lifecycle 包成 `useXxx()` 函式、跨元件複用
