**GitHub Actions = GitHub 內建的 CI/CD 系統**——push code 自動跑 test、build、deploy。**寫一個 YAML 檔、推上去就行**。**業界最普及、免費額度爆炸大**。

> 💡 **這課的價值**：手動 deploy 是上世紀的事。學會 GitHub Actions = 你的 PR 自動測試、合 main 自動上線、不用記任何指令。

  
### 1. 核心概念

```text
📋 Workflow（工作流）
   一個 YAML 檔、在 .github/workflows/ 下
   例：ci.yml、deploy.yml

⚡ Event（事件）
   什麼時候觸發 workflow
   例：push、pull_request、schedule

🧱 Job（任務）
   workflow 內的「平行單位」
   - 各自有自己的 runner
   - 互不影響（除非 needs）

🪜 Step（步驟）
   job 內的「依序動作」
   - 跑 shell command
   - 或用別人的 Action

🔌 Action（行動）
   別人寫好的 reusable 步驟
   - actions/checkout
   - actions/setup-node
   - 用 - uses: name@version 呼叫
```

  
### 2. 第一個 workflow

📄 **寫進程式檔**：`.github/workflows/ci.yml`

```text
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Lint
        run: npm run lint

      - name: Typecheck
        run: npm run typecheck

      - name: Test
        run: npm test

      - name: Build
        run: npm run build
```

**push 上去後**：GitHub 自動跑、UI 看綠勾 / 紅叉。

  
### 3. Workflow 結構詳解

```text
name: ...                # workflow 名稱（UI 顯示）

on:                      # 觸發事件
  push:                  # push 時
    branches: [main]
  pull_request:          # PR 時
  schedule:              # 排程
    - cron: '0 3 * * *'  # 每天 3am
  workflow_dispatch:     # 手動觸發按鈕
  release:               # 發 release 時

jobs:
  job_id:                # 你取的 id
    runs-on: ubuntu-latest   # 用什麼機器
    needs: [other_job]   # 等其他 job 跑完
    if: github.ref == 'refs/heads/main'   # 條件
    timeout-minutes: 10
    
    strategy:            # 矩陣（同 job 跑多版本）
      matrix:
        node: [18, 20, 22]
        os: [ubuntu-latest, windows-latest]
    
    steps:
      - uses: ...        # 用 action
      - run: ...         # 跑 shell
      - name: ...        # 顯示名稱
```

  
### 4. Secrets — 密鑰管理

**設定路徑**：repo → Settings → Secrets and variables → Actions

```text
DATABASE_URL = postgres://...
DISCORD_WEBHOOK = https://discord.com/...
RAILWAY_TOKEN = xxxxx
VERCEL_TOKEN = yyyyy
```

**在 workflow 用**：

```text
- name: Deploy
  env:
    DATABASE_URL: ${{ secrets.DATABASE_URL }}
  run: npm run deploy
```

**重要**：
- Secrets 在 log 自動 mask（顯示為 `***`）
- PR from fork 拿不到 secrets（安全）

  
### 5. 完整 example：test + deploy

📄 **寫進程式檔**：`.github/workflows/deploy.yml`

```text
name: Deploy

on:
  push:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
      - run: npm ci
      - run: npm test

  deploy-staging:
    needs: test
    runs-on: ubuntu-latest
    environment: staging
    steps:
      - uses: actions/checkout@v4
      - name: Deploy to Railway
        uses: bervProject/railway-deploy@v3
        with:
          railway_token: ${{ secrets.RAILWAY_TOKEN }}
          service: my-app-staging

  e2e:
    needs: deploy-staging
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm ci
      - run: npx playwright test
        env:
          BASE_URL: https://staging.my-app.com

  deploy-prod:
    needs: e2e
    runs-on: ubuntu-latest
    environment:
      name: production
      url: https://my-app.com
    steps:
      - uses: actions/checkout@v4
      - name: Deploy to Railway
        uses: bervProject/railway-deploy@v3
        with:
          railway_token: ${{ secrets.RAILWAY_TOKEN }}
          service: my-app-prod

      - name: Notify Slack
        uses: slackapi/slack-github-action@v1
        with:
          webhook: ${{ secrets.SLACK_WEBHOOK }}
          payload: '{"text":"deployed v${{ github.sha }} to prod"}'
```

**流程**：
```text
push main
  ↓
test ─→ deploy-staging ─→ e2e ─→ deploy-prod ─→ Slack
```

  
### 6. Matrix — 同一 job 跑多版本

```text
jobs:
  test:
    runs-on: ${{ matrix.os }}
    strategy:
      matrix:
        os: [ubuntu-latest, macos-latest, windows-latest]
        node: [18, 20, 22]
        # 3 × 3 = 9 個 job 平行跑
    steps:
      - uses: actions/setup-node@v4
        with:
          node-version: ${{ matrix.node }}
      - run: npm test
```

### 8. Build & Push Docker image

📄 **寫進程式檔**：`.github/workflows/docker.yml`

```text
name: Docker

on:
  push:
    branches: [main]
    tags: ['v*']

jobs:
  build:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      packages: write
    steps:
      - uses: actions/checkout@v4

      - name: Login to GHCR
        uses: docker/login-action@v3
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: Setup Buildx
        uses: docker/setup-buildx-action@v3

      - name: Build & push
        uses: docker/build-push-action@v5
        with:
          context: .
          push: true
          tags: |
            ghcr.io/${{ github.repository }}:latest
            ghcr.io/${{ github.repository }}:${{ github.sha }}
          cache-from: type=gha
          cache-to: type=gha,mode=max
```

  
### 9. 進階：Reusable Workflow

**避免重複**：把常用 workflow 抽出來。

📄 **寫進程式檔**：`.github/workflows/_test.yml`

```text
name: Reusable Test

on:
  workflow_call:
    inputs:
      node-version:
        type: string
        default: '20'

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: ${{ inputs.node-version }}
      - run: npm ci
      - run: npm test
```

**用它**：

```text
jobs:
  call-test:
    uses: ./.github/workflows/_test.yml
    with:
      node-version: '22'
```

  
### 10. Environment（環境）

```text
jobs:
  deploy-prod:
    environment:
      name: production
      url: https://my-app.com   # PR 顯示
```

**設定**：repo → Settings → Environments → New

- 加 **required reviewer**（合人才能 deploy）
- 加 **wait timer**（合後 5 分鐘才跑）
- 設環境特定 secrets

### 12. 5 大常見坑

```text
❌ 1. secrets 寫進 yaml
   → push 上去 = 洩漏
   → 修：永遠用 ${{ secrets.X }}

❌ 2. 沒設 timeout-minutes
   → 卡住的 job 跑 6 小時、燒額度
   → 修：每個 job 加 timeout-minutes: 30

❌ 3. 沒 cache
   → 每次 npm ci 重抓、慢 5 倍
   → 修：actions/setup-node 加 cache: 'npm'

❌ 4. PR from fork 拿不到 secrets 嚇到
   → 修：用 pull_request_target（小心安全）
       或讓 deploy 只跑在 push main

❌ 5. 一個 workflow 100 行混雜
   → 修：拆 reusable workflow
```

  
### 🎯 重點回顧

- **GitHub Actions = .github/workflows/*.yml**
- **核心**：Event → Job → Step → Action
- **Marketplace** 有上萬 action 直接用
- **Secrets** 在 repo settings 設、用 `${{ secrets.X }}`
- **Matrix** 同 job 跑多版本
- **Reusable workflow** 避免重複
- **Environment** 加 reviewer 防 deploy 出包
