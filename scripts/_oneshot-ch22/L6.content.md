**Docker = 把「整個 OS + 程式 + 設定」打包成一個檔案**——叫做 image。在任何電腦跑這個 image 結果都一樣。**解決「我電腦跑得起來、你電腦跑不起來」這個千古難題**。

> 💡 **這課的價值**：2026 年所有部署 90% 都用 container。學會 Docker = 你可以把 app deploy 到任何雲、不被綁死。

  
### 1. 核心三個概念

```text
🧊 Image（映像）
   - 「打包好的程式 + OS」
   - 一個唯讀檔
   - 例：nginx:1.27、postgres:16

📦 Container（容器）
   - image 跑起來的「實例」
   - 像 class vs object 的關係
   - 一個 image 可以跑很多 container

📋 Dockerfile（建構檔）
   - 一份文字食譜
   - 告訴 docker「怎麼做出這個 image」
```

  
### 2. 跟 VM 比較

```text
🖥️ Virtual Machine（虛擬機）
   - 整個 OS（GB 等級）
   - 開機要分鐘
   - 隔離強

📦 Container
   - 只裝差異層（MB 等級）
   - 啟動秒級
   - 共享 host kernel
   - 隔離夠用

→ Container 是「輕量版 VM」
```

  
### 3. 第一個 Dockerfile

📄 **寫進程式檔**：`Dockerfile`

```text
# 從哪個 image 開始（base）
FROM node:20-alpine

# 設工作目錄
WORKDIR /app

# 複製檔案進 image
COPY package*.json ./
RUN npm ci

COPY . .

# 開放 port
EXPOSE 3000

# 啟動指令
CMD ["node", "server.js"]
```

🖥️ **在終端機輸入**：

```text
# Build image
docker build -t my-app:1.0 .

# 跑 container
docker run -p 3000:3000 my-app:1.0
```

💬 **電腦會顯示**：

```text
Server listening on http://localhost:3000
```

  
### 4. Dockerfile 12 個 instruction

```text
FROM         base image（必須）
WORKDIR      設工作目錄
COPY         複製本機檔案到 image
ADD          像 COPY 但能解壓 / 抓 URL（少用）
RUN          build 階段執行指令（裝套件）
ENV          設環境變數
ARG          build-time 參數
EXPOSE       宣告 port（純文件）
VOLUME       宣告掛載點
USER         切換 user（安全）
CMD          預設啟動指令（可被覆蓋）
ENTRYPOINT   進入點（不被覆蓋）
HEALTHCHECK  健康檢查
```

**CMD vs ENTRYPOINT**：

```text
CMD ["node", "server.js"]
  → docker run image            跑 node server.js
  → docker run image bash       跑 bash（覆蓋）

ENTRYPOINT ["node"]
CMD ["server.js"]
  → docker run image            跑 node server.js
  → docker run image other.js   跑 node other.js（CMD 被覆蓋）
```

  
### 5. Layer（層）— 為什麼順序重要

每個指令產生一層、cache 起來。**沒變的 layer 不重 build**。

❌ **錯誤順序**：

```text
COPY . .
RUN npm install
```

→ 改一個 source code、整個 npm install 重跑（慢）。

✅ **正確順序**：

```text
COPY package*.json ./
RUN npm ci
COPY . .
```

→ 只改 source 不改 package.json → npm ci 用 cache（快 10 倍）。

**口訣**：**「越穩定的放前面、越常變的放後面」**。

  
### 6. Multi-stage Build（多階段建置）

**問題**：build 需要 1.2GB 的工具（compiler、devDeps）、跑只需要 100MB。

**解法**：分兩階段、最後只留 runtime。

📄 **寫進程式檔**：`Dockerfile`

```text
# Stage 1: builder
FROM node:20 AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

# Stage 2: runtime（只留必要的）
FROM node:20-alpine
WORKDIR /app
COPY --from=builder /app/dist ./dist
COPY --from=builder /app/node_modules ./node_modules
COPY package.json .
CMD ["node", "dist/server.js"]
```

**結果**：image 從 1.2GB → 180MB。

  
### 7. .dockerignore（必寫）

📄 **寫進程式檔**：`.dockerignore`

```text
node_modules
.git
.env
.env.local
*.log
coverage/
.next/
dist/
README.md
.vscode/
```

**作用**：build context 上傳不會把這些丟進去（更快、更安全）。

  
### 8. docker-compose — 多 service 編排

📄 **寫進程式檔**：`docker-compose.yml`

```text
services:
  web:
    build: .
    ports:
      - "3000:3000"
    environment:
      DATABASE_URL: postgres://user:pass@db:5432/mydb
      REDIS_URL: redis://cache:6379
    depends_on:
      - db
      - cache

  db:
    image: postgres:16-alpine
    environment:
      POSTGRES_USER: user
      POSTGRES_PASSWORD: pass
      POSTGRES_DB: mydb
    volumes:
      - pg_data:/var/lib/postgresql/data
    ports:
      - "5432:5432"

  cache:
    image: redis:7-alpine
    ports:
      - "6379:6379"

volumes:
  pg_data:
```

🖥️ **在終端機輸入**：

```text
docker compose up         # 一次跑全部
docker compose up -d      # 背景跑
docker compose logs -f web   # 看 log
docker compose down       # 全停
```

  
### 9. 常用 base image 推薦

```text
🐳 Node.js
   node:20-alpine        最小（80MB）
   node:20-slim          中等
   node:20               完整（1GB+）

🐍 Python
   python:3.12-slim      推薦
   python:3.12-alpine    更小但 wheel 可能不相容

🐹 Go
   golang:1.22-alpine    build
   scratch / distroless   runtime（< 10MB）

🦀 Rust
   rust:alpine           build
   debian:bookworm-slim  runtime

🌐 全能
   ubuntu:24.04          穩、大
   debian:bookworm-slim  穩、小
   alpine:3.20           最小但 musl 可能踩雷
```

  
### 10. 安全最佳實踐

```text
✅ 不以 root 跑
   USER node
   
✅ 用具體版本（不要用 latest）
   FROM node:20.11.0-alpine  ✅
   FROM node:latest          ❌（會偷偷升）

✅ 不要 COPY secrets 進去
   用 BuildKit secrets、或 runtime env

✅ 掃漏洞
   docker scout cves my-app:1.0
   trivy image my-app:1.0

✅ multi-stage 減少 attack surface

✅ HEALTHCHECK 加上
   HEALTHCHECK --interval=30s CMD curl -f http://localhost:3000/health || exit 1
```

  
### 11. 常用指令清單

🖥️ **在終端機輸入**：

```text
# Image
docker build -t name:tag .       建構
docker images                    列出
docker rmi image_id              刪除
docker pull nginx                拉取
docker push my/image:tag         推送

# Container
docker run -p 3000:3000 -d name  跑
docker ps                        看在跑的
docker ps -a                     全部（含停的）
docker logs container_id         看 log
docker logs -f container_id      跟著看
docker exec -it container bash   進去
docker stop container_id         停
docker rm container_id           刪
docker restart container_id      重啟

# 清理
docker system prune              清沒用的
docker system prune -a --volumes 全清（小心）

# Compose
docker compose up -d             跑
docker compose ps                看狀態
docker compose logs -f service   看 log
docker compose down              停
```

  
### 12. 5 大常見坑

```text
❌ 1. 把 node_modules COPY 進去
   → image 巨大、不同 OS 模組爆
   → 修：.dockerignore + RUN npm ci

❌ 2. 用 latest tag
   → 哪天 base image 升 → 你的 app 突然壞
   → 修：用具體版本（node:20.11.0-alpine）

❌ 3. ENV 寫死 secrets
   → image push 出去 = 洩漏
   → 修：runtime 才注入

❌ 4. 沒 multi-stage
   → image 1.2GB、pull 半天
   → 修：build / runtime 分階段

❌ 5. 沒 .dockerignore
   → 把 .git / node_modules 都打包進去
   → 修：第一天就寫 .dockerignore
```

  
### 🎯 重點回顧

- **Image** 是模板、**Container** 是實例、**Dockerfile** 是食譜
- **Layer 順序**：穩定的放前面、常變的放後面（cache 友善）
- **Multi-stage build** 把 image 縮小 5 ~ 10 倍
- **.dockerignore** 第一天就寫
- **docker-compose** 編排多 service（web + db + cache）
- **不要用 latest**、不要 root 跑、不要 COPY secrets
