**Railway / Fly.io / Render = 新世代 PaaS（Platform as a Service、平台即服務）**。**比 Heroku 便宜、比 AWS 簡單、push code 就跑**。各有強項——選對能省 70% 成本 + 90% 設定時間。

> 💡 **這課的價值**：2026 年的 backend 部署不應該還在 SSH 進 EC2 自己裝 nginx。學會這 3 大平台 = 你的 side project / startup 可以 5 分鐘上線。

  
### 1. PaaS 的演進

```text
🟢 第一代：Heroku（2007）
   - PaaS 開山始祖
   - 簡單到爆
   - 2022 取消免費方案、貴到不行

🟡 第二代：Vercel / Netlify（2015 ~）
   - 為前端 / Jamstack 設計
   - 不適合 long-running backend / DB

🔵 第三代：Railway / Fly.io / Render（2020 ~）
   - 為「全棧」設計
   - backend / DB / cron / worker 全包
   - 比 Heroku 便宜 5 倍
```

  
### 2. 三大平台速覽

```text
┌─────────────┬─────────────────────────────────┐
│ Railway     │ 最簡單、UI 漂亮、DB 整合超好    │
│ Fly.io      │ 全球 edge、低延遲、技術控最愛   │
│ Render      │ 平衡、像新版 Heroku             │
└─────────────┴─────────────────────────────────┘
```

  
### 3. Railway（最簡單）

🌐 **網址**：railway.app

**特色**：
```text
✅ 連 GitHub → push 即部署
✅ 內建 PostgreSQL / MySQL / Redis / MongoDB
✅ 環境變數 UI 編輯
✅ 服務間自動 DNS（DATABASE_URL 自動注入）
✅ Preview environment per PR
✅ pricing：$5/月起、按用量
```

**最爽用法**：

🖥️ **在終端機輸入**：

```text
# 安裝 CLI
npm i -g @railway/cli
railway login

# 在你的 project
railway init
railway up    # ← 整個 deploy
```

💬 **電腦會顯示**：

```text
✓ Building...
✓ Deploying...
✓ https://your-app.up.railway.app
```

**加 PostgreSQL**：
- UI：點 `+ New → Database → PostgreSQL`
- DATABASE_URL 自動注入到你的 app
- 完成

**何時用**：
- side project / MVP
- 不想學 Docker
- 要 backend + DB 一起

**注意**：
- 不是 serverless、有 idle cost
- 流量大時不算便宜

  
### 4. Fly.io（全球 edge）

🌐 **網址**：fly.io

**特色**：
```text
✅ 真正全球部署（30+ 城市）
✅ 你的 app 跑在 multiple region 同步
✅ 自動就近 routing（台灣用戶打東京）
✅ 支援 stateful（持久磁碟）
✅ 跑 Docker container
✅ pricing：$5/月免費額度、超出按用量
```

📄 **寫進程式檔**：`fly.toml`

```text
app = "my-app"
primary_region = "nrt"   # 東京

[build]
  dockerfile = "Dockerfile"

[http_service]
  internal_port = 3000
  force_https = true
  auto_stop_machines = true     # 沒流量自動關
  auto_start_machines = true    # 有流量自動開
  min_machines_running = 0

[[vm]]
  cpu_kind = "shared"
  cpus = 1
  memory_mb = 512
```

🖥️ **在終端機輸入**：

```text
# 安裝
brew install flyctl

# 一次部署
fly launch         # 自動偵測 + 產生 fly.toml
fly deploy

# 加 region
fly regions add sin   # 新加坡
fly scale count 3 --region sin
```

**何時用**：
- 全球用戶（東南亞 + 美國 + 歐洲都有）
- 對延遲敏感
- 想跑特殊 binary（不只 Node.js）

  
### 5. Render（最平衡）

🌐 **網址**：render.com

**特色**：
```text
✅ 像新版 Heroku、最像「正常 PaaS」
✅ 內建 PostgreSQL / Redis
✅ Web Service / Cron Job / Static Site / Worker 都有
✅ 自動 HTTPS + 自訂 domain
✅ Preview environment
✅ pricing：$7/月起
```

📄 **寫進程式檔**：`render.yaml`

```text
services:
  - type: web
    name: my-api
    env: node
    buildCommand: npm install && npm run build
    startCommand: npm start
    envVars:
      - key: DATABASE_URL
        fromDatabase:
          name: my-db
          property: connectionString

databases:
  - name: my-db
    plan: starter
```

**何時用**：
- 從 Heroku 搬家
- 中型 production app
- 要穩定不要花俏

### 7. 完整 deploy 範例 — Next.js + PostgreSQL

#### a. Vercel + Supabase（最熱組合）

```text
1. Vercel deploy Next.js（前端）
2. Supabase 託管 PostgreSQL
3. Vercel env：DATABASE_URL = supabase 給的
4. 完成
```

#### b. Railway 一鍵全包

```text
1. Railway → New Project
2. Deploy from GitHub
3. + Database → PostgreSQL
4. 環境變數自動注入
5. 完成
```

#### c. Fly.io（要全球）

```text
1. fly launch
2. fly postgres create
3. fly postgres attach
4. fly deploy
5. fly regions add ams（阿姆斯特丹）
```

  
### 8. 進階：multi-region 配置

**Fly.io 的拿手好戲**：

```text
# 在多個 region 跑 web service
fly scale count 3 --region nrt   # 東京 3 台
fly scale count 2 --region iad   # 維吉尼亞 2 台
fly scale count 2 --region ams   # 阿姆斯特丹 2 台

# DB 主機 + read replica
fly postgres create --region nrt    # primary
fly postgres replica create --region iad
fly postgres replica create --region ams
```

**結果**：
- 台灣用戶 → 東京 server（30ms）
- 美國用戶 → 維吉尼亞 server（20ms）
- 歐洲用戶 → 阿姆斯特丹 server（10ms）

  
### 9. 成本估算（小型 production）

```text
情境：1 個 backend + 1 個 PostgreSQL + 月 100 萬 request

Vercel + Supabase：~$45/月（Pro plans）
Railway：         ~$15/月（hobby + DB）
Fly.io：          ~$10/月（小規模免費額度大）
Render：          ~$20/月（Starter + DB）
AWS（EC2+RDS）：  ~$60/月（要懂 ops）
```

  
### 10. 移植性 — 不要被鎖死

```text
🟢 標準化（容易搬家）：
   - 用 Docker
   - 用 PostgreSQL（任何雲都有）
   - 環境變數讀 config
   - 12-Factor App

🔴 平台鎖死（搬家痛）：
   - Vercel 專屬：Edge Functions、ISR
   - Railway 專屬：很少（基本標準）
   - Fly.io 專屬：很少
   - Lambda 專屬：API Gateway integration
```

  
### 11. 5 大常見坑

```text
❌ 1. 把 backend 塞進 Vercel
   → cold start + 10 秒超時
   → 修：backend 用 Railway / Fly / Render

❌ 2. Railway free tier 跑 production
   → 每月用完就停
   → 修：升 hobby ($5) 起跳

❌ 3. Fly.io 沒設 auto-stop
   → 沒流量也燒錢
   → 修：auto_stop_machines = true

❌ 4. 沒設健康檢查
   → app crash 平台不知道、不重啟
   → 修：加 /health endpoint + healthcheck 設定

❌ 5. secrets 寫進 code commit
   → GitHub 一 push 全洩漏
   → 修：用平台 env 管理、加 .gitignore
```

  
### 🎯 重點回顧

- **Railway** 最簡單、適合 MVP / side project
- **Fly.io** 全球 edge、適合對延遲敏感的全球 app
- **Render** 最平衡、像新版 Heroku
- **Vercel** 前端王、backend 不適合
- **Multi-region** 是 Fly.io 殺手鐧
- **永遠用 Docker + 環境變數** 保持移植性
