**IaC（Infrastructure as Code、基礎設施即代碼）= 用程式碼描述「我要什麼伺服器 / DB / 網路」、跑一下自動建好**。**告別「手動在 AWS console 點來點去」**——所有資源像 git 一樣 review / version。

> 💡 **這課的價值**：手動點 cloud console 是 2015 年的事。學會 IaC = 環境可重現、PR review 基礎設施變更、災難時 5 分鐘重建整套。

  
### 1. 沒 IaC 的痛點

```text
❌ 手動建：
   - 開 EC2 → 點 30 次按鈕
   - 設 VPC → 又 50 次
   - 接 RDS → 再 30 次
   - 半年後同事問「這個 security group 為什麼開」→ 沒人記得

❌ 沒法 review：
   - 改一個 setting 沒人知道
   - 改錯沒法 rollback

❌ 災難不可恢復：
   - 整個 region 倒了
   - 要在另一個 region 重建 → 一週

✅ IaC：
   - 寫 terraform 檔
   - PR review
   - terraform apply 自動建好
   - git 歷史就是基礎設施歷史
   - 換 region 改一個變數 → 30 分鐘重建
```

  
### 2. 5 大工具

```text
🔷 Terraform（HashiCorp、業界 #1）
   - HCL 語法（HashiCorp Configuration Language）
   - 多 cloud（AWS / GCP / Azure / Cloudflare ...）
   - 2023 改 BSL license（社群分叉 OpenTofu）

🍴 OpenTofu（社群分叉）
   - Terraform 1.5 的 fork
   - 100% 開源（MPL）
   - 跟 Terraform 99% 相容

🟪 Pulumi
   - 用 TS / Python / Go 寫
   - 適合工程師（不愛 HCL 的）

🔶 AWS CloudFormation
   - 只支援 AWS
   - YAML / JSON
   - AWS 內服務深度整合

☁️ AWS CDK（Cloud Development Kit）
   - 寫 TS / Python、生成 CloudFormation
   - 最舒服的 AWS-only
```

**業界選擇**：
- 跨雲 / 大廠 → Terraform / OpenTofu
- AWS 重度 → CDK
- 喜歡寫程式 → Pulumi

  
### 3. Terraform 核心概念

```text
📋 Provider（提供者）
   要管哪個 cloud
   例：aws / google / azurerm / cloudflare

📦 Resource（資源）
   實際的東西
   例：aws_instance、aws_s3_bucket

📊 State（狀態）
   Terraform 記錄「目前實際長什麼樣」
   存在 .tfstate 檔（要存遠端 S3 + lock）

🔄 Plan & Apply
   plan：看 diff（要建什麼、改什麼、刪什麼）
   apply：實際執行
```

  
### 4. 第一個 Terraform 檔

📄 **寫進程式檔**：`main.tf`

```text
terraform {
  required_version = ">= 1.6"
  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.0"
    }
  }

  # State 存遠端
  backend "s3" {
    bucket         = "my-tf-state"
    key            = "prod/terraform.tfstate"
    region         = "ap-northeast-1"
    dynamodb_table = "tf-lock"
    encrypt        = true
  }
}

provider "aws" {
  region = var.region
}

variable "region" {
  type    = string
  default = "ap-northeast-1"
}

variable "env" {
  type    = string
  default = "production"
}

# VPC
resource "aws_vpc" "main" {
  cidr_block = "10.0.0.0/16"

  tags = {
    Name = "${var.env}-vpc"
    Env  = var.env
  }
}

# RDS
resource "aws_db_instance" "main" {
  identifier        = "${var.env}-db"
  engine            = "postgres"
  engine_version    = "16.2"
  instance_class    = "db.t3.medium"
  allocated_storage = 50
  storage_encrypted = true

  db_name  = "myapp"
  username = "postgres"
  password = var.db_password   # 從 var 帶

  backup_retention_period = 7
  multi_az                = true   # HA

  tags = {
    Env = var.env
  }
}

variable "db_password" {
  type      = string
  sensitive = true
}

# Output 給其他模組 / 人用
output "db_endpoint" {
  value = aws_db_instance.main.endpoint
}
```

🖥️ **在終端機輸入**：

```text
# 第一次
terraform init           裝 providers

# 看計畫
terraform plan -var="db_password=..."

# 套用
terraform apply -var="db_password=..."

# 看當前狀態
terraform show

# 刪一切
terraform destroy
```

💬 **電腦會顯示（plan）**：

```text
Terraform will perform the following actions:

  # aws_vpc.main will be created
  + resource "aws_vpc" "main" {
      + cidr_block = "10.0.0.0/16"
      ...
    }

  # aws_db_instance.main will be created
  + resource "aws_db_instance" "main" {
      ...
    }

Plan: 2 to add, 0 to change, 0 to destroy.
```

  
### 5. Module — 重用基礎設施

📄 **寫進程式檔**：`modules/web/main.tf`

```text
variable "name" {}
variable "image" {}
variable "replicas" {
  default = 2
}

resource "aws_ecs_service" "web" {
  name            = var.name
  task_definition = aws_ecs_task_definition.web.arn
  desired_count   = var.replicas
  # ...
}

output "url" {
  value = aws_lb.web.dns_name
}
```

**使用模組**：

```text
module "api" {
  source   = "./modules/web"
  name     = "api"
  image    = "ghcr.io/me/api:v1"
  replicas = 5
}

module "worker" {
  source   = "./modules/web"
  name     = "worker"
  image    = "ghcr.io/me/worker:v1"
  replicas = 2
}
```

**Public module registry**：registry.terraform.io ——找 AWS / GCP 常用 pattern 直接拿。

  
### 6. Workspace — 多環境

```text
terraform workspace new dev
terraform workspace new staging
terraform workspace new production

terraform workspace select dev
terraform apply       # 只改 dev 環境

# 在 code 內

  is_prod = terraform.workspace == "production"
  
  instance_type = local.is_prod ? "t3.large" : "t3.micro"
}
```

### 8. State 管理 — 最重要

```text
⚠️ State 是 IaC 的核心
   - 記錄當前實際狀態
   - 不能丟、不能多人同時改

✅ 必做：
   1. 存遠端（S3 + DynamoDB lock / Terraform Cloud）
   2. 加密
   3. 加 lock（防併發寫）
   4. backup state

❌ 不可：
   - state 存本機（一個人改 = 衝突）
   - 手動改 state（搞壞超痛）
   - 多人同時 apply（無 lock 會壞）
```

  
### 9. GitOps 工作流

```text
1. 工程師改 main.tf
2. 開 PR → CI 跑 terraform plan
3. plan 結果貼到 PR comment（diff 可 review）
4. team review
5. merge 後 CI 自動 terraform apply
6. Slack 通知
```

**工具**：
- **Atlantis**（最熱）
- **Terraform Cloud / HCP**
- **Spacelift**
- **env0**

  
### 10. IaC 適用範圍

```text
✅ 適合 IaC：
   - VPC / subnet / security group
   - RDS / S3 / ECS
   - Load balancer / Route 53
   - IAM role / policy
   - k8s cluster

⚠️ 不適合 IaC（用其他工具）：
   - app 部署（用 k8s yaml / Helm）
   - DB schema migration（用 Prisma / Flyway）
   - 真實資料（用備份還原）
   - secrets 內容（用 Vault / Secret Manager）
```

  
### 11. 5 大常見坑

```text
❌ 1. state 存本機
   → 同事一改、衝突地獄
   → 修：第一天就放 S3 + lock

❌ 2. 把 secrets 寫死在 .tf
   → git push = 洩漏
   → 修：用 var + sensitive = true、或 Vault

❌ 3. 沒 review 直接 apply
   → 改錯把 production DB 刪了
   → 修：永遠先 plan、永遠 PR review

❌ 4. 手動改 console、不更新 .tf
   → 下次 apply 把你的手動改回去
   → 修：所有變更走 Terraform、或 import 進來

❌ 5. 不寫 module、複製貼上
   → dev / staging / prod 三份相似 code
   → 修：抽 module、用 workspace
```

  
### 🎯 重點回顧

- **IaC = 寫程式碼描述基礎設施**——可 review、可 version、可重建

- **核心**：Provider → Resource → State → Plan & Apply

- **Module** 重用、**Workspace** 多環境
