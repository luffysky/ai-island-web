**Transformer：為什麼改變一切**——2017 Google 「**Attention is All You Need**」論文發明。**這篇 paper 改變了整個 AI 領域**。2026 所有主流 LLM、大部分圖像 / 語音 / 影片 AI **都是 Transformer**。

> 💡 **2026 真相**：你聽到的 GPT / Claude / Gemini / Llama / Stable Diffusion / Sora 全是 Transformer 架構。**學會 Transformer = 看懂整個 AI 業界**。

  
### Transformer 之前：RNN / LSTM 的問題

2017 前處理「序列資料」（文字 / 語音）主流是 **RNN（Recurrent Neural Network、遞迴神經網路）** 跟 **LSTM（Long Short-Term Memory、長短期記憶）**。

  
**問題：**

```text
=== 句子：「我喜歡咖啡、特別是肯亞 AA、每天早上必喝一杯。」===

RNN 一個字一個字讀：
  我 → 喜 → 歡 → 咖 → 啡 → ， → 特 → 別 → ... → 一 → 杯

問題：
1. 慢 — 必須順序處理、無法平行（GPU 浪費）
2. 健忘 — 讀到「一杯」時已經忘了「我」（gradient vanishing 梯度消失）
3. 短記憶 — 長句子 100 字後就模糊
4. 不能雙向看 — 標準 RNN 只看前面、看不到後面
```

**結果：** 2017 前的 AI 翻譯品質差、文章生成沒邏輯。

  
### Transformer 的解法：Attention（注意力機制）

**核心思想**：「**每個字都看所有其他字、自己算跟誰最相關**」。

```text
=== 句子：「我喜歡咖啡、特別是肯亞 AA、每天早上必喝一杯。」===

當處理「一杯」時：
  - 看「我」：關聯度 0.3
  - 看「喜歡」：關聯度 0.1
  - 看「咖啡」：關聯度 0.9 ← 最相關！
  - 看「肯亞」：關聯度 0.7
  - 看「AA」：關聯度 0.6
  - 看「早上」：關聯度 0.2
  ...

機器自己決定「一杯」最該跟「咖啡」連在一起
不管字數差距多遠都能連起來
```

這就是 **Self-Attention（自注意力機制）**——自己跟自己的關聯。

  
### Attention 三大優勢

```text
⚡ 平行處理（Parallelization）
   → 所有字同時算 attention
   → GPU 友善
   → 訓練速度快 50-100 倍

🔗 長距離關聯（Long-range Dependency）
   → 第 1 字跟第 1000 字直接相連
   → 沒有距離衰減

📈 可擴展（Scaling）
   → 加更多參數、效果繼續好
   → Scaling Laws（後面教）
```

  
### Q K V：Attention 怎麼算？

不用懂數學細節、但要知道概念：

```text
=== 每個字產生 3 個向量 ===

🔍 Query（Q）：「我在找什麼」
🗝️ Key（K）：「我能提供什麼」
📦 Value（V）：「我的具體內容」

=== 對每個字 ===

1. 用我的 Q 跟所有字的 K 算相似度
   → 得到 attention score

2. score 越高 = 越相關

3. softmax 把 score 變機率（加起來 = 1）

4. 用機率對 V 加權平均
   → 得到我的「context」

→ 我的新表示 = 我自己 + 跟我相關的人的資訊
```

> 💡 **像搜尋引擎**：你輸入 query「咖啡推薦」、系統有一堆網頁的 key（標題 / 關鍵字）、算相似度、把最相關的網頁內容（value）給你。**Self-Attention 就是每個字當自己是搜尋者、從句子裡找跟自己相關的其他字**。

  
### Transformer 完整結構

```text
輸入文字
  ↓
Tokenize（切成 token）
  ↓
每個 token 變成 Embedding 向量（512 或 1024 維）
  ↓
+ Position Encoding（位置資訊、Attention 本身不知道字的順序）
  ↓
┌────────────────────────────┐
│ Multi-Head Attention       │ ← 多個 attention 平行
├────────────────────────────┤
│ Add & Norm（殘差 + 正規化）│
├────────────────────────────┤
│ Feed Forward（一般 NN）    │
├────────────────────────────┤
│ Add & Norm                 │
└────────────────────────────┘
  ↓
重複 N 層（GPT-4 約 96 層）
  ↓
輸出層 → 預測下一個 token（或分類等）
```

  
### Multi-Head Attention（多頭注意力）

不只一個 attention、**同時跑多個**（GPT-4 約 96 個 head）。每個 head 學不同的關聯：

```text
🎯 Head 1：學「主詞-動詞」關聯
🎯 Head 2：學「形容詞-名詞」
🎯 Head 3：學「代詞指誰」（「他」指誰）
🎯 Head 4：學「時間順序」
🎯 ...

多 head 結果合併、同時考慮多種關聯
```

  
### Encoder vs Decoder

原始 Transformer 有兩部分（用於翻譯）：

| | Encoder | Decoder |
|---|---|---|
| **看** | 整個輸入句子 | 一個字一個字生成 |
| **方向** | 雙向（看前後） | 單向（只看前面） |
| **適合** | 理解 / 分類 / 摘要 | 生成 / 翻譯 / 對話 |
| **代表** | BERT | GPT / Claude |

  
### GPT 系列：Decoder-only

2018 OpenAI 發現「**只用 Decoder + 夠大就強**」。

```text
=== 自回歸生成（autoregressive）===

輸入：「今天天氣很」
  ↓
Decoder（Transformer 內部運算）
  ↓
輸出：「好」（預測下一個 token）

把「好」加到輸入：
「今天天氣很好」→「，」
「今天天氣很好、」→「我」
「今天天氣很好、我」→「想」
...

→ 一個 token 一個 token 往後生
→ 直到生成「結束符號」或達到 max length
```

  
### BERT vs GPT

| | BERT | GPT |
|---|---|---|
| 架構 | Encoder-only | Decoder-only |
| 能看前後 | 能（雙向） | 只能看前面 |
| 擅長 | 分類 / 搜尋 / 嵌入 | 生成 / 對話 / 創作 |
| 2026 用途 | Embedding / 語義搜尋 | 幾乎所有 LLM |

  
### Scaling Laws：規模 = 效果

2020 OpenAI 發現「**Scaling Laws**」：

```text
模型效果 ∝ (參數 × 訓練資料 × 算力)

三個因素同步加 → 效果穩定上升
少加一個 → 沒效（資料夠多但模型太小 = 學不完整）
```

  
**這個發現引爆「越大越好」軍備競賽：**

```text
2019：GPT-2     15 億參數
2020：GPT-3     1750 億參數
2023：GPT-4     估 1-2 兆參數
2024-2026：朝 10 兆走
```

  
**但 2024 後出現「Scaling 飽和」討論：**

```text
單純加大已邊際遞減
重心轉到：
✅ 更好的資料（質 > 量）
✅ 更聰明的訓練（RLHF / Constitutional AI）
✅ 推理時間更長（o1 / Claude 思考模式）
✅ 多模態（圖 / 聲 / 影一起）
```

  
### Transformer 的演化

```text
🔀 MoE（Mixture of Experts、混合專家）
   → 不是所有參數都用
   → 依輸入「路由到專家」
   → Mixtral / GPT-4 都用

🐍 Mamba / State Space Models
   → 嘗試取代 Transformer
   → 2024 興起但仍非主流

📏 長 context（Long Context）
   → 從 2k → 8k → 32k → 200k → 2M（Gemini）

🧠 推理模型（Reasoning Models）
   → o1 / o3 / Claude Opus 思考
   → 先想再答
   → 解數學 / 寫 code 強

🎨 多模態（Multimodal）
   → 把圖 / 聲音都當 token 處理
   → GPT-4V / Claude / Gemini
```

  
### 為什麼這節重要

2026 你跟業界人講 AI、大家都會丟出「**Transformer / Attention / Q K V / Multi-head**」這些詞。

  
**懂結構 = 能討論深度問題：**

```text
❓ 為什麼 GPT 慢？
✅ 因為自回歸生成必須一個個算

❓ 為什麼 context 變長變慢？
✅ 因為 Attention 計算量是 O(n²)、不是 O(n)

❓ 為什麼模型有 context limit？
✅ 因為位置編碼有上限

❓ 為什麼 Llama 70B 比 8B 強？
✅ 參數多、能學更複雜的 attention pattern
```

Transformer 5 件事：(1) **Attention** = 每個字看所有其他字、自己算關聯（取代 RNN）(2) **Q K V** = Query 找、Key 提供、Value 內容（像搜尋引擎）(3) **Multi-Head** = 多個 attention 同時跑、學不同關聯 (4) **GPT = Decoder-only**、自回歸生成（一個字一個字往後生）(5) **Scaling Laws** = 參數 × 資料 × 算力 = 效果。**心法**：2026 所有主流 LLM / 多模態 AI 都是 Transformer、學會 = 看懂業界。
