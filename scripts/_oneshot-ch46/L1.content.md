2024-2026 滿街講「AI」、但**大部分人混用 4 個層次的概念**——新聞講「AI」可能指 ChatGPT、可能指人臉辨識、可能指規則引擎。先把這幾個詞釐清才能討論其他。

> 💡 **這課的價值**：學會分辨「AI / ML / DL / GenAI」、聽客戶 / 同事說「AI」時不會被話術。

> 💡 你聽到的「**AI 應用**」99% 是**最內層的 GenAI**、但被用「AI」這個最外層的名字。

  
### AI（Artificial Intelligence、人工智慧）

**最廣的概念** —「**讓機器表現得有智慧**」。1956 達特茅斯會議命名。

它包含**所有讓電腦看起來會思考的技術**、不一定要用 ML：

```text
🤖 規則引擎（Rule Engine）
   → if 使用者輸入 X、回 Y
   → 1980 年代主流

📚 專家系統（Expert System）
   → 1980 年代醫療診斷
   → 醫生寫好規則、電腦判斷

🌳 決策樹（Decision Tree）
   → if-else 巢狀
   → 不用 ML 也可

🗺️ 圖搜尋（Graph Search）
   → A* 演算法
   → 棋類 AI 用

🚗 路徑規劃（Pathfinding）
   → Google Maps 最短路徑
   → 算 graph 的算法
```

這些**都是 AI**、但都不是 ML。你的 iPhone 計算機**嚴格說也是 AI**（會算數）—但顯然不是現在大家講的「AI」。

  
### ML（Machine Learning、機器學習）

1959 年 Arthur Samuel 定義：「**讓電腦不用明確程式設計就能學習**」。

**核心觀念**：「**機器從資料學規則**」、不用人寫規則。

  
**傳統程式 vs ML 對比：**

```text
=== 傳統程式（你寫規則）===
if 照片有四隻腳 + 毛 + 鬍鬚:
    return "貓"

問題：
- 玩具貓有四腳（不是貓）
- 狗也有毛（不是貓）
- 橫躺的貓只有兩腳可見（漏判）
→ 規則寫不完

=== ML（給範例、機器自己學）===
給機器 10,000 張 "這是貓 / 這不是貓" 的照片
機器自己找出「貓的特徵」

你不用解釋什麼是貓
→ 就像不用解釋什麼是「美」一樣
→ 給範例就好
```

  
### ML 不一定要 neural network

很多其他演算法：

| 演算法 | 中文 | 用途 |
|---|---|---|
| **Linear Regression** | 線性迴歸 | 找一條最 fit 資料的直線 |
| **Decision Tree** | 決策樹 | 學「if-else」的條件 |
| **Random Forest** | 隨機森林 | 很多棵決策樹投票 |
| **SVM**（Support Vector Machine） | 支援向量機 | 找最佳分隔線 |
| **K-Means** | K-平均分群 | 自動分群 |
| **XGBoost** | 梯度提升樹 | 表格資料王者 |
| **Neural Network** | 神經網路 | DL 用這個 |

> 💡 **2010 年前**、ML 主流是非神經網路（Random Forest / XGBoost / SVM）。**2012 後** DL 興起、但其他 ML 仍很重要——金融 / 製造 / 推薦系統很多不用 DL。

  
### DL（Deep Learning、深度學習）

ML 的一個子集——**用很多層神經網路**。「Deep」就是「**很多層**」。

  
**關鍵時間點：**

```text
1958：Perceptron 提出（最早的神經元模型）
1986：Backpropagation（反向傳播、讓多層神經網路能訓練）
2006：Hinton 重新推動「Deep Learning」這個名字
2012：AlexNet 在 ImageNet 大勝 → DL 引爆點
2017：Transformer 論文（Attention is All You Need）
2022：ChatGPT 發布 → 公眾意識到 AI
2024-2026：多模態 LLM / 推理模型 / Agent
```

  
**DL 需要：**

```text
🔢 大量資料（百萬-億級）
💻 GPU 算力（CPU 跑 DL 慢 100 倍）
⏰ 長時間訓練（GPT-4 估訓練 100 天 × 數萬張 GPU）
💰 巨大成本（GPT-4 訓練估 $100M）
```

  
**2026 主流 DL 模型：**

```text
CNN（Convolutional Neural Network、卷積神經網路）
  → 影像識別（看 X 光 / 人臉辨識）

RNN / LSTM（Recurrent / Long Short-Term Memory）
  → 序列資料（2017 前語音辨識主流、被 Transformer 取代）

Transformer
  → LLM 的核心架構（L4 詳教）

GAN（Generative Adversarial Network、生成對抗網路）
  → 早期 deepfake 用

Diffusion（擴散模型）
  → Midjourney / Stable Diffusion / Flux
```

  
### GenAI（Generative AI、生成式 AI）

2022-2024 起飛的新領域。能**「產生新東西」**、不只是分類 / 預測。

  
**對比：**

```text
=== 傳統 ML（Discriminative AI、判別式）===
輸入：一張圖
輸出：「這是貓」（分類）

=== GenAI（Generative AI、生成式）===
輸入：「畫一隻穿太空裝的貓」
輸出：一張全新的圖
```

  
**2026 主要 GenAI 類型：**

| 類型 | 中文 | 代表 |
|---|---|---|
| **LLM**（Large Language Model） | 大型語言模型 | GPT / Claude / Gemini / Llama |
| **Image Diffusion** | 圖像擴散模型 | Midjourney / Stable Diffusion / Flux / DALL-E |
| **Video** | 影片生成 | Sora / Veo / Runway / Kling |
| **Audio TTS** | 語音合成 | ElevenLabs / OpenAI TTS |
| **Audio Music** | 音樂生成 | Suno / Udio |
| **Code** | 程式碼生成 | GitHub Copilot / Cursor / Claude Code |
| **3D** | 3D 模型 | Tripo / Meshy（還不成熟） |

  
### 常見混淆與正確說法

| 常見錯誤 | 正確說 |
|---|---|
| 「AI 就是 ChatGPT」 | ChatGPT 是 LLM 的應用、LLM 是 GenAI 的一種、GenAI 是 DL 的一種、DL 是 ML 的一種、ML 是 AI 的一種 |
| 「Machine Learning 就是 AI」 | ML 是 AI 的一部分、AI 還包括規則引擎 |
| 「所有 AI 都需要訓練」 | 傳統規則 AI 不用、ML 才需要 |
| 「GenAI 跟 LLM 一樣」 | LLM 是 GenAI 的一種、GenAI 還包括圖像 / 影片 / 音樂 |
| 「AI 會思考」 | LLM 是 pattern matching、不是「思考」（哲學爭議大） |
| 「ChatGPT 知道一切」 | 有知識截止日、且會編造（hallucination） |

  
### AGI 跟 ASI

新聞會出現的兩個詞：

```text
🤖 AGI（Artificial General Intelligence、通用人工智慧）
   → 能做任何認知任務
   → 像人類那樣靈活遷移
   → OpenAI / Anthropic 的公司目標

🚀 ASI（Artificial Super Intelligence、超人工智慧）
   → 在所有領域超越人類
   → 哲學爭議大
   → 有人認為永遠達不到
```

  
**2026 真相：**

```text
✅ 現在的 LLM 不是 AGI
   → 某些 benchmark 超人
   → 但「常識推理」「跨領域遷移」差很多

✅ 業界對「2030 前能否達到 AGI」分裂
   → 樂觀派：5-10 年
   → 悲觀派：50+ 年

✅ ASI 更遠
   → 甚至有人認為永遠達不到
```

  
### 為什麼這節重要

2026 跟人講「AI」、**每個人腦中浮現的東西不一樣**。

  
**同事說「我們用 AI 做這個」可能是：**

```text
1. 用 ChatGPT API（LLM）
2. 用 OpenCV 做影像處理（傳統 CV）
3. 用 XGBoost 做預測（ML 但非 DL）
4. 用 if-else 寫規則（連 ML 都不是）

→ 三個成本差 100 倍
→ 三個技術棧完全不同
→ 三個維護方式不同
```

  
**會釐清 = 不被話術 = 能問對問題**。

客戶說「**給我做 AI 客服**」、你要問：

```text
1. 「你要 LLM 對話？」
   → 月費高（API call 多）、效果最好

2. 「規則式 chatbot？」
   → 月費低（無 API）、回答死板

3. 「帶 RAG 的知識庫問答？」
   → 中等成本、最適合企業內部
   → 介於 1 跟 2 之間
```

> 💡 **三個方案成本差 100 倍**——不問清楚、做錯方向、客戶不爽、你白做。
