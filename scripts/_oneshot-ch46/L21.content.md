**開源模型部署：Ollama / vLLM / llama.cpp**——不想付 API 錢？想要隱私？要在 GPU 上跑？**開源模型來。**

> 💡 **2026 真相**：閉源 API 跟開源模型差距越來越小、自架成本越來越低、indie hacker 也能跑 Llama 70B。

  
### 為什麼用開源？

```text
💰 沒 API 費
   → 但要硬體 / 雲端 GPU

🔒 隱私
   → 資料不出公司

🎨 可 fine-tune
   → 客製化更深

   → API 漲價 / 限制 / 出包不怕

🛠️ 邊緣裝置
   → 手機 / IoT / 路由器
```

### 三大部署方案

  
### 1. Ollama（最簡單、推薦個人）

**本機跑 LLM 的「Docker」：**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
# Mac / Linux
curl -fsSL https://ollama.com/install.sh | sh

# Windows
# 去 ollama.com 下載 installer

# 跑模型
ollama run llama3.3
ollama run qwen2.5:7b
ollama run mistral
ollama run phi4
```

**這 5 行做什麼？** (1-2) 裝 Ollama (3-6) 直接跑各種開源模型、第一次會自動下載。

  
**API 一樣是 OpenAI 格式：**

```text
from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:11434/v1",   # ⭐ 本機 Ollama
    api_key="ollama"                          # 任意字串
)

response = client.chat.completions.create(
    model="llama3.3",
    messages=[{"role": "user", "content": "哈囉"}]
)
```

> 💡 **Ollama 殺手特性**：OpenAI 相容 API、換 model name 就好、本機跑、免費。

  
**適合**：個人 Mac / Linux 開發、prototype、隱私敏感本機處理。

  
### 2. vLLM（生產級、推薦企業）

**高效能 LLM 推論引擎：**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
uv add vllm

vllm serve meta-llama/Meta-Llama-3-70B-Instruct \
    --port 8000 \
    --gpu-memory-utilization 0.9
```

**這 2 行做什麼？**
- `pip install vllm` —— 裝 vLLM
- `vllm serve` —— 啟動 LLM service、port 8000、用 90% GPU 記憶體

  
**vLLM 殺手特性：**

```text
✅ PagedAttention
   → 高效記憶體管理

✅ Continuous Batching
   → 吞吐量翻倍

✅ 分散式部署
   → 多 GPU 支援

✅ OpenAI 相容 API
   → 程式碼不用改

✅ Throughput
   → 比 transformers 快 10-20x
```

**適合**：自架 LLM 服務、有 GPU、需要高吞吐。

  
### 3. llama.cpp（CPU / 量化）

**把 LLM 跑在沒 GPU 的機器：**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
git clone https://github.com/ggerganov/llama.cpp
cd llama.cpp && make

./llama-cli -m llama-3-8b-Q4_K_M.gguf -p "哈囉"
```

**這 4 行做什麼？**
- `git clone` —— clone llama.cpp source
- `make` —— 編譯
- `./llama-cli` —— 跑模型、`-m` 指定 model 檔、`-p` prompt

  
**透過量化（4-bit / 8-bit）大模型也能在 CPU 跑：**

```text
- Llama 7B Q4：4GB RAM 能跑
- Llama 70B Q4：40GB RAM 能跑
- 速度慢、但能跑
```

**適合**：個人實驗、邊緣裝置、學術研究。

### OpenRouter：統一介面（推薦）

**一個 API key 接 100+ 模型**：

```text
from openai import OpenAI

client = OpenAI(
    base_url="https://openrouter.ai/api/v1",
    api_key="sk-or-..."
)

# === 換模型一行 ===
response = client.chat.completions.create(
    model="meta-llama/llama-3.3-70b-instruct",
    messages=[...]
)

response = client.chat.completions.create(
    model="anthropic/claude-opus-4-7",
    messages=[...]
)

response = client.chat.completions.create(
    model="deepseek/deepseek-chat",
    messages=[...]
)
```

> 💡 **OpenRouter 殺手特性**：(1) 一個 key 接所有 (2) 統一 OpenAI API (3) 自動 fallback (4) 用量分析。

  
### 什麼時候自架？

```text
✅ 每月 LLM 花費 > $1000
✅ 強烈隱私需求（醫療 / 金融 / 政府）
✅ 需要極致客製（fine-tune）
✅ 規模大（> 1000 req/sec）
✅ 合規要求（資料不能出國）
```

  
### 什麼時候別自架？

```text
❌ 規模小（API 反而便宜）
❌ 沒 GPU 維運經驗
❌ 需要最新最強模型（GPT-4.5 / Claude Opus）
❌ 追求穩定性（自架要自己管 uptime）
```

### 完整實戰：本機 Llama + RAG

```text
import requests
import json

# === 1. 跑 Ollama（背景）===
# terminal: ollama serve

# === 2. 用 Llama 3.3 + RAG ===
from llama_index.core import VectorStoreIndex, SimpleDirectoryReader
from llama_index.llms.ollama import Ollama
from llama_index.embeddings.ollama import OllamaEmbedding

# === 用本機 Ollama 跑 Llama + Embedding ===
llm = Ollama(model="llama3.3", request_timeout=120)
embed_model = OllamaEmbedding(model_name="nomic-embed-text")

# === 載入文件 + 建索引 ===
documents = SimpleDirectoryReader("./data").load_data()
index = VectorStoreIndex.from_documents(documents, embed_model=embed_model)

# === 查詢 ===
query_engine = index.as_query_engine(llm=llm)
response = query_engine.query("公司請假規定？")
print(response)

# → 完全本機跑、$0 API 費、資料不出機器
```
