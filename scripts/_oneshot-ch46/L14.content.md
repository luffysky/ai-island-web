**Embedding：文字變向量的魔法**——把任何東西變向量。**RAG / 搜尋 / 推薦 / 分類的基礎**。

> 💡 **核心觀念**：Embedding = **「把意思變成數字」**。意思相近 → 數字接近、跨語言也通。

  
### 什麼是 Embedding？

```text
"咖啡"   → [0.12, -0.45, 0.78, ..., 0.23]  (1536 維)
"咖啡因" → [0.15, -0.42, 0.76, ..., 0.21]
"拿鐵"   → [0.18, -0.40, 0.74, ..., 0.25]
"汽車"   → [0.89, 0.32, -0.21, ..., 0.65]

關鍵：語義相近 → 向量接近
「咖啡」「拿鐵」相似度 0.9
「咖啡」「汽車」相似度 0.2
```

> 💡 **每個 token / 字 / 句子變成一個 1024-3072 維的向量**——維度多但「**距離**」能算「**語意相似度**」。

  
### 怎麼用？算相似度（Cosine Similarity）

```text
import numpy as np

def cosine_similarity(a, b):
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

e1 = get_embedding("我喜歡咖啡")
e2 = get_embedding("我愛喝咖啡")
e3 = get_embedding("今天天氣很好")
e4 = get_embedding("I love coffee")

cosine_similarity(e1, e2)  # 0.92（很像）
cosine_similarity(e1, e3)  # 0.21（不像）
cosine_similarity(e1, e4)  # 0.85（跨語言也行！）
```

  
### 7 大應用

```text
🔍 語意搜尋（Semantic Search）
   → 找含義相似的內容

📊 分類（Classification）
   → 算跟各類別中心距離

🎬 推薦（Recommendation）
   → 找相似商品

🔁 去重（Deduplication）
   → 找重複內容（措辭不同）

🚨 異常檢測（Anomaly Detection）
   → 跟正常資料 embedding 差太遠

👥 聚類（Clustering）
   → 自動分群

📚 RAG（L13）
   → 找相關文件給 LLM
```

  
### 關鍵字 vs Embedding

| 查詢 | 關鍵字（BM25） | Embedding |
|---|---|---|
| 「咖啡」 | 只找含「咖啡」 | 找咖啡 / 拿鐵 / espresso |
| 「能解渴的飲料」 | 找不到水 | 找到水 / 果汁 / 茶 |
| 跨語言「coffee」 | 找不到中文 | 能找到「咖啡」 |
| 精確「2024」 | ⭐ 強 | 弱（可能混進 2023） |

> 💡 **2026 主流**：**兩個都用（Hybrid）**——L13 教過。

  
### 取得 Embedding API

  
**OpenAI（業界標準）：**

```text
from openai import OpenAI

client = OpenAI()

response = client.embeddings.create(
    input="我喜歡咖啡",
    model="text-embedding-3-small"
)

embedding = response.data[0].embedding
print(len(embedding))   # 1536
```

  
**Cohere：**

```text
import cohere

co = cohere.Client()

response = co.embed(
    texts=["我喜歡咖啡"],
    model="embed-multilingual-v3.0",
    input_type="search_document"
)

embedding = response.embeddings[0]
```

  
**自架開源（BGE-M3、jina）：**

```text
from sentence_transformers import SentenceTransformer

model = SentenceTransformer("BAAI/bge-m3")
embedding = model.encode("我喜歡咖啡")
# → 1024 維、本機跑、免費
```

  
### 多模態 Embedding（圖 + 文）

| 模型 | 公司 | 特色 |
|---|---|---|
| **CLIP** | OpenAI 2021 | 圖 + 文一起、能搜「貓的照片」找到貓圖 |
| **SigLIP** | Google 2023 | CLIP 改進、效果更好 |
| **Voyage Multimodal** | Voyage AI | 圖文檢索 |
| **Cohere Embed v3 Multilingual** | Cohere | 100+ 語言 |

  
### 維度對比

| 模型 | 維度 |
|---|---|
| **OpenAI text-embedding-3-small** | 1536 |
| **OpenAI text-embedding-3-large** | 3072 |
| **Cohere embed v3** | 1024 |
| **BGE-M3 / Voyage-3** | 1024 |
| **jina-embeddings-v3** | 1024 |

> 💡 **維度越高**：精度略高、儲存大、計算慢、成本高。**1024-1536 維足夠 90% 場景**。

  
### Matryoshka Embeddings（2024 新技術）

**OpenAI text-embedding-3 可截短維度：**

```text
# === 原 3072 維 ===
emb = client.embeddings.create(
    input="hello",
    model="text-embedding-3-large",
    dimensions=256   # ⭐ 截短到 256 維
)

# → 一樣有意義
# → 儲存 / 計算便宜 12 倍
```

> 💡 **用途**：先 256 維粗搜、再完整維度精搜——速度 + 精準兼顧。

### 實戰：語意搜尋

```text
from openai import OpenAI
import numpy as np

client = OpenAI()

def embed(text: str):
    """取 embedding"""
    res = client.embeddings.create(
        input=text,
        model="text-embedding-3-small"
    )
    return np.array(res.data[0].embedding)

def cosine_sim(a, b):
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

# === 建文件庫 ===
docs = [
    "我們的咖啡採用衣索比亞單品豆",
    "今天下午有產品 demo",
    "新員工 onboarding 在週一",
    "拿鐵買 10 送 1",
    "請假規定：年假 14 天",
]

doc_embeddings = [embed(d) for d in docs]

# === 搜尋 ===
query = "咖啡優惠"
q_emb = embed(query)

# 算每個 doc 跟 query 的相似度
scores = [(d, cosine_sim(q_emb, e)) for d, e in zip(docs, doc_embeddings)]

# 排序
scores.sort(key=lambda x: x[1], reverse=True)

# 印 top 3
for doc, score in scores[:3]:
    print(f"{score:.3f}: {doc}")

# 輸出：
# 0.823: 拿鐵買 10 送 1
# 0.781: 我們的咖啡採用衣索比亞單品豆
# 0.234: 請假規定：年假 14 天
```
