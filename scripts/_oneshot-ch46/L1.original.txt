2024-2026 滿街講「AI」，但**大部分人混用 4 個層次的概念**。新聞講「AI」可能指 ChatGPT、可能指人臉辨識、可能指規則引擎。先把這幾個詞釐清才有辦法討論其他。

  
### 四個詞的關係：俄羅斯娃娃

```text
┌─────────────────────────────────────────┐
│  AI（人工智慧）— 最廣                     │
│  ┌──────────────────────────────────┐  │
│  │  ML（機器學習）                    │  │
│  │  ┌─────────────────────────────┐ │  │
│  │  │  DL（深度學習）               │ │  │
│  │  │  ┌────────────────────────┐ │ │  │
│  │  │  │  GenAI（生成式 AI）     │ │ │  │
│  │  │  │  - LLM（GPT、Claude）   │ │ │  │
│  │  │  │  - Diffusion（Midjourney）│ │ │  │
│  │  │  │  - Audio（Suno、ElevenLabs）│ │ │  │

│  └──────────────────────────────────┘  │

```

  
你聽到的「AI 應用」99% 是**最內層的 GenAI**，但用「AI」這個最外層的名字。

  
### AI（Artificial Intelligence）

  
最廣的概念—「**讓機器表現得有智慧**」。1956 達特茅斯會議命名。

  
它包含**所有讓電腦看起來會思考的技術**，不一定要用 ML：

  
- **規則引擎**：「如果使用者輸入 X，就回 Y」
- **專家系統**：1980 年代醫療診斷系統，醫生寫規則
- **決策樹**：if-else 巢狀的判斷
- **圖搜尋**：A* 演算法、棋類 AI 用的
- **路徑規劃**：Google Maps 的「最短路徑」

  
這些**都是 AI**，但都不是 ML。你的 iPhone 計算機**嚴格說也是 AI**（會算數）—但顯然不是現在大家講的「AI」。

  
### ML（Machine Learning）

  
1959 年 Arthur Samuel 定義：「**讓電腦不用明確程式設計就能學習**」。

  
核心觀念：「**機器從資料學規則**」，不用人寫規則。

  

**傳統程式**：你寫規則—「如果照片裡有四隻腳 + 毛 + 鬍鬚 → 是貓」。但這超難寫對：玩具貓有四腳、狗也有毛、橫躺的貓只有兩腳。


**ML**：你給 1000 張「這是貓 / 這不是貓」的照片，機器自己找出「貓的特徵」。你不用解釋什麼是貓—就像不用解釋什麼是「美」一樣，給範例就好。

  

  
ML 不一定要 neural network。有很多其他演算法：

  
- **線性回歸**：找一條最 fit 資料的直線
- **決策樹**：學「if-else」的條件
- **隨機森林**：很多棵決策樹投票
- **SVM**（支援向量機）：找最佳分隔線
- **K-Means**：自動分群
- **神經網路**：DL 用這個

  
**2010 年前**，ML 主流是非神經網路的演算法（隨機森林、XGBoost、SVM）。**2012 後** DL 興起，但其他 ML 演算法**仍然很重要**—很多金融、製造、推薦系統不用 DL。

  
### DL（Deep Learning）

  
ML 的一個子集—**用很多層神經網路**。「Deep」就是「很多層」。

  
關鍵時間點：

  
- **1958 年**：Perceptron 提出（最早的神經元模型）
- **1986 年**：反向傳播演算法（讓多層神經網路能訓練）
- **2006 年**：Hinton 重新推動「Deep Learning」這個名字
- **2012 年**：AlexNet 在 ImageNet 大勝 — DL 引爆點
- **2017 年**：Transformer 論文發表
- **2022 年**：ChatGPT 發布 — 公眾意識到 AI
- **2024-2026**：多模態 LLM、推理模型、Agent

  
DL 需要：

  
- **大量資料**（百萬到億級）
- **GPU 算力**（CPU 跑 DL 慢 100 倍）
- **長時間訓練**（GPT-4 訓練估 100 天 × 數萬張 GPU）

  
2026 你聽到的 AI 模型幾乎都是 DL：

  
- CNN：影像識別（看 X 光、人臉辨識）
- RNN / LSTM：序列資料（語音辨識的 2017 主流，現被 Transformer 取代）
- Transformer：LLM 的核心架構
- GAN：生成式對抗網路（早期 deepfake 用）
- Diffusion：擴散模型（Midjourney、Stable Diffusion）

  
### GenAI（Generative AI）

  
2022-2024 起飛的新領域。能**「產生新東西」**而不只是分類 / 預測。

  
對比：

```text
傳統 ML（Discriminative AI 判別式）：
  輸入：一張圖
  輸出：「這是貓」（分類）

GenAI（Generative AI 生成式）：
  輸入：「畫一隻穿太空裝的貓」
  輸出：一張全新的圖
```

  
2026 主要 GenAI 類型：

  
- **LLM**（Large Language Model）：產生文字 — GPT、Claude、Gemini、Llama
- **Image Diffusion**：產生圖片 — Midjourney、Stable Diffusion、Flux、DALL-E
- **Video**：產生影片 — Sora、Veo、Runway、Kling
- **Audio (TTS)**：產生語音 — ElevenLabs、OpenAI TTS
- **Audio (Music)**：產生音樂 — Suno、Udio
- **Code**：產生程式碼 — GitHub Copilot、Cursor、Claude Code
- **3D**：產生 3D 模型 — Tripo、Meshy（還不成熟）

  
### 常見混淆與正確說法

  
| 常見錯誤說法 | 實際應該說 |
| --- | --- |
| 「AI 就是 ChatGPT」 | ChatGPT 是 LLM 的一個應用、LLM 是 GenAI 的一種、GenAI 是 DL 的一種、DL 是 ML 的一種、ML 是 AI 的一種 |
| 「Machine Learning 就是 AI」 | ML 是 AI 的一部分，AI 還包括規則引擎等非 ML 技術 |
| 「所有 AI 都需要訓練」 | 傳統規則 AI 不用，ML 才需要 |
| 「GenAI 跟 LLM 一樣」 | LLM 是 GenAI 的一種，GenAI 還包括圖像、影片、音樂生成 |
| 「AI 會思考」 | LLM 是 pattern matching，不是「思考」（哲學爭議大） |
| 「ChatGPT 知道一切」 | 有知識截止日，且會編造（hallucination） |

  
### AGI 跟 ASI

  
新聞會出現的兩個詞：

  
- **AGI**（Artificial General Intelligence）：通用人工智慧 — 能做任何認知任務、像人類那樣靈活遷移
- **ASI**（Artificial Super Intelligence）：超人工智慧 — 在所有領域都超越人類

  
2026 真相：

  
- 現在的 LLM **不是 AGI**—在某些 benchmark 超人，但在「常識推理」「跨領域遷移」差很多
- OpenAI / Anthropic 都把 AGI 當公司目標
- 業界對「**2030 前能否達到 AGI**」分裂—樂觀派認為快、悲觀派認為要 50+ 年
- ASI 更遠—甚至有人認為永遠達不到

  
### 為什麼這節重要

  
2026 跟人講「AI」，**每個人腦中浮現的東西不一樣**。同事說「我們用 AI 做這個」可能是：

  
- 用 ChatGPT API（LLM）
- 用 OpenCV 做影像處理（傳統 CV）
- 用 XGBoost 做預測（ML 但非 DL）
- 用 if-else 寫規則（連 ML 都不是）

  
**會釐清 = 不被話術 = 能問對問題**。客戶說「給我做 AI 客服」—你要問「你要：(1) LLM 對話？(2) 規則式 chatbot？(3) 帶 RAG 的知識庫問答？」三個成本差 100 倍。
