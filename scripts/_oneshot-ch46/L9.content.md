**Few-shot / Chain-of-Thought / ReAct**——三個經典 prompt 技巧、**每個都能顯著提升效果**。

> 💡 **2026 真相**：好 prompt 跟壞 prompt 答案差 5-10 倍。學會這三個技巧 = LLM 使用上手。

  
### Zero-shot：直接問

```text
「翻譯成英文：我喜歡咖啡」
→ AI 回：「I like coffee」
```

不給範例。**LLM 強大時直接能用、但複雜 / 新任務效果差**。

  
### Few-shot：給幾個範例（最強技巧）

```text
「請依範例做情緒分析：

「今天好開心」→ 正面
「考試又考砸了」→ 負面
「天氣還行」→ 中性
「不予置評」→ 中性
「我終於買到了！」→ 正面

「老闆又畫大餅了」→ ?」

→ AI 學範例風格：「負面」
```

  
### Few-shot 的力量

Few-shot 是 LLM 最強大的能力——**不用 fine-tune、用幾個範例就學新任務**。

**應用場景：**

```text
🆕 新任務 LLM 沒見過
   → 「把產品評論轉成行銷文案」

📋 需要特定格式
   → 你的 JSON schema

🏥 領域專業
   → 法律、醫療術語

🎨 風格控制
   → 學品牌語氣

⚠️ 處理 edge case
   → 給範例教 AI 怎麼處理
```

  
### Few-shot 5 大注意事項

```text
1. ⭐ 2-5 個範例最佳
   → 太少不夠、太多浪費 token

2. ⭐ 範例要涵蓋多樣性
   → 包含 edge case

3. ⭐ 範例品質很關鍵
   → 差範例會讓 AI 學壞

4. ⭐ 順序有影響
   → 最近 / 最後的範例影響最大

5. ⭐ 用 XML / 分隔符
   → 清楚劃分範例跟實際輸入
```

  
**建議格式（用 XML 分隔、Claude 推薦）：**

```text
以下是範例：

<example>
input: 我喜歡咖啡
output: 正面
</example>

<example>
input: 今天好爛
output: 負面
</example>

現在請處理：
<input>
天氣不錯
</input>
output: ?
```

  
### Chain-of-Thought（CoT、思維鏈）

**2022 年 Google 論文發現**：讓 AI「**先想再答**」效果大幅提升、尤其是推理 / 數學題。

  
**對比：**

```text
❌ 直接問：
Q：小明買 3 個蘋果每個 10 元、4 個橘子每個 8 元、給 100 元找多少？

AI 可能直接答（可能算錯）：「找 60 元」

✅ 加 CoT：
Q：[同上題目]

請逐步計算：

AI 會寫：
步驟 1：蘋果 3 × 10 = 30 元
步驟 2：橘子 4 × 8 = 32 元
步驟 3：總共 30 + 32 = 62 元
步驟 4：100 - 62 = 38 元
答案：找 38 元

→ 準確率大幅提升
→ GSM8K（數學題 benchmark）從 17% 飆到 78%
```

  
### CoT 的關鍵句

加在 prompt 結尾的「**魔法句**」：

```text
🇹🇼 中文：
- 「請逐步思考」
- 「先列出步驟」
- 「先分析、再判斷」
- 「請說明推理過程」

🇺🇸 英文：
- "Let's think step by step"
- "Show your reasoning"
- "Think carefully"
```

> 💡 **實測 Claude / GPT 對英文 "Let's think step by step" 最有效**——這句話是 CoT 業界標準。

  
### Few-shot CoT（結合兩者）

**給含步驟的範例：**

```text
請依範例計算：

Q：3 + 5 等於多少？
A：3 加 5 等於 8。答案：8

Q：12 - 4 + 7 等於多少？
A：12 減 4 等於 8。8 加 7 等於 15。答案：15

Q：(15 + 8) × 2 等於多少？
A：先算括號內 15 + 8 = 23。再 23 × 2 = 46。答案：46

Q：100 / 4 + 6 × 3 等於多少？
A：

→ AI 學會「先寫過程、再給結果」的模式
```

  
### ReAct：Reason + Act（思考 + 行動）

**2022 Yao 等人提出**。讓 AI **結合「思考」跟「使用工具」**：

```text
使用者：「台北今天天氣怎樣？」

AI 內部運作：

Thought（思考）: 我需要查台北今天的天氣
Action（行動）: search_web("台北今天天氣")
Observation（觀察）: 「台北 5/27 多雲時晴、22-28°C、降雨機率 20%」

Thought: 我有答案了
Final Answer: 台北今天多雲時晴、氣溫 22-28°C、降雨機率 20%。

→ 使用者只看到 Final Answer
→ 但 AI 實際透過 search_web 工具拿到即時資料
```

  
### ReAct 是 Agent 的基礎

所有 AI Agent（L16 詳教）都用 **ReAct 或類似架構**：

```text
1. 看到使用者問題
2. Thought：思考要做什麼
3. Action：呼叫工具（搜尋 / API / 計算）
4. Observation：看工具結果
5. 繼續 Thought / Action（可以多輪）
6. 最終 Final Answer
```

  
### 什麼時候 CoT 不夠？

```text
❌ 需要外部資料
   → CoT 只想、不查
   → 天氣 / 股票 / 新聞 → 用 ReAct + Tool

❌ 需要計算
   → LLM 算數差
   → 要呼叫 calculator

❌ 需要即時資訊
   → 訓練資料截止後的事
   → 要呼叫 search

❌ 需要操作
   → 寄信 / 訂位 / 寫檔案
   → 要呼叫對應 API

→ 這些都該用 ReAct + Tool Use（L10）
```

  
### 2026 推理模型：內建 CoT

**2024 後出現「內建 CoT 的模型」**（o1 / o3 / Claude Opus 思考模式）：

```text
✅ 不用你寫「請逐步思考」
✅ 模型自己花更長時間想（10 秒到幾分鐘）
✅ 內部產生長 reasoning trace
✅ 對數學 / 程式 / 邏輯 / 規劃題大幅提升

⚠️ 延遲較長
⚠️ 價格較貴
⚠️ 不適合即時對話
```

  
**什麼時候用推理模型：**

```text
✅ 數學競賽題、奧林匹亞
✅ 需要嚴謹推理（醫療 / 法律分析）
✅ 程式 debug 困難案例
✅ 系統設計、架構規劃
✅ 不需要即時回應的場景
```

  
**什麼時候不用：**

```text
❌ 聊天、創意寫作
❌ 簡單問答
❌ 要求即時回應（< 5 秒）
❌ 預算敏感（推理模型貴 2-10x）
```

  
### 三個技巧對比

| 技巧 | 用途 | 何時用 | 範例難度 |
|---|---|---|---|
| **Zero-shot** | 直接問 | 簡單通用任務 | ⭐ 最簡 |
| **Few-shot** | 給範例學 | 特定格式 / 領域 | ⭐⭐ |
| **CoT** | 先想再答 | 推理 / 數學 / 邏輯 | ⭐⭐ |
| **ReAct** | 想 + 用工具 | 需要外部資料 / 操作 | ⭐⭐⭐⭐ |

  
### 2026 業界趨勢

```text
=== 讓模型自己決定要不要 CoT ===

Claude Sonnet 4.6 / Gemini 2.5 / o3 都有「自適應思考」：
- 內建 extended_thinking 模式
- 可以設「think budget」（思考預算）
- 模型自己判斷簡單題快答、難題慢想
- 不用你手動加「請逐步思考」
- 但 API call 還是要明確開啟
```
