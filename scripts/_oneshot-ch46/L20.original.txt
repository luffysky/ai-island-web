「**怎麼讓 LLM 學會我們公司的東西？**」三種方案。

  
### 三種方案差異

  
|  | Prompt | RAG | Fine-tune |
| --- | --- | --- | --- |
| 原理 | 每次塞 context | 動態查再塞 | 改變模型權重 |
| 準備時間 | 分鐘 | 幾小時 | 幾天 |
| 成本 | 低 | 中 | 高 |
| 更新資料 | 立即 | 立即 | 要重訓 |
| 大量資料 | 不行 | 行 | 行 |
| 學風格 | 少量可 | 不太行 | 強 |
| 學新知識 | 當下 | 當下 | 內化 |

  
### Prompt（系統提示 + Few-shot）

  
適合：

  
- 規則 / 風格不多（ 100k 字就難）。

  
### RAG

  
適合：

  
- 知識量大（千、萬篇文件）
- 需要引用來源
- 知識常更新
- 需要多租戶（不同使用者看不同資料）

  
限制：學不會「**風格**」，只能基於資料回答。

  
### Fine-tune

  
適合：

  
- 需要學特殊風格 / 語氣（醫生口吻、律師措辭）
- 輸出格式很固定
- 專業領域術語
- 要省 token（fine-tune 後 prompt 可變短）
- 有 10000+ 高質量訓練資料

  
限制：

  
- 準備資料困難
- 訓練要錢
- 更新麻煩
- 容易 overfit（過擬合）
- 可能讓模型變笨（catastrophic forgetting）

  
### 實戰流程

```text
1. 先試 Prompt（最快、最便宜）
   ↓ 效果不夠
2. 加 Few-shot（給範例）
   ↓ 還不夠
3. 上 RAG（大量資料）
   ↓ 學風格不行
4. 才考慮 Fine-tune

90% 場景到 RAG 就夠
真的需要 fine-tune 的  數量
```

  
### Cost 對比範例

```text
客服 SaaS 每月 100 萬次 query：

純 Prompt（GPT-4o）：
- 每次 2k input + 500 output tokens
- 月成本：~$750

純 RAG（GPT-4o + pgvector）：
- 每次多 1k context token
- 月成本：~$1000 + $50 vector DB

Fine-tune GPT-4o-mini：
- 訓練：$50 一次
- 每次 only 500 token input（風格內化）
- 月成本：~$75

→ Fine-tune 後成本最低、但前期投資
```

  
### 什麼時候 Fine-tune 才真值得

  
- 大量重複 prompt → fine-tune 省 token
- 特殊風格 / 領域 → prompt 怎麼寫都不對
- 固定格式輸出 → 100% 穩定
- 已驗證產品有 PMF、要降成本 / 提準確

  
### 什麼時候別 Fine-tune

  
- 還在 prototype 階段
- 規則 / 知識常變
- 沒幾個高質量 example
- 需要立刻 deploy
- 用得不多（
**🚀 進階技巧**

2026 indie hacker 黃金路徑：**Prompt → Few-shot → RAG → Prompt Caching → Fine-tune（最後手段）**。前三個解決 90% 問題、第四個解決成本問題、Fine-tune 通常 overkill。
