不想付 API 錢？想要隱私？要在 GPU 上跑？開源模型來。

  
### 為什麼用開源？

  
- **沒 API 費**（但要硬體 / 雲端 GPU）
- **隱私**：資料不出公司
- **可 fine-tune**
- **不依賴一家公司**
- **客製化更深**
- **邊緣裝置**（手機、IoT）

### 三大部署方案

  
#### 1. Ollama（最簡單）

  
本機跑 LLM 的「Docker」：

```bash
# 裝
curl -fsSL https://ollama.com/install.sh | sh

# 跑模型
ollama run llama3.2
ollama run qwen2.5:7b
ollama run mistral
ollama run phi3
```

```python
# API 一樣是 OpenAI 格式
from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:11434/v1",
    api_key="ollama"
)

response = client.chat.completions.create(
    model="llama3.2",
    messages=[{"role": "user", "content": "哈囉"}]
)
```

  
適合：個人 Mac / Linux 開發。

  
#### 2. vLLM（生產級）

  
高效能 LLM 推論引擎：

```bash
pip install vllm

vllm serve meta-llama/Meta-Llama-3-70B-Instruct \
    --port 8000 \
    --gpu-memory-utilization 0.9
```

  
- **PagedAttention**：高效記憶體管理
- **Continuous Batching**：吞吐量翻倍
- **分散式部署**：多 GPU
- **OpenAI 相容 API**

  
適合：自架 LLM 服務、有 GPU。

  
#### 3. llama.cpp（CPU / 量化）

  
把 LLM 跑在**沒 GPU 的機器**：

```bash
git clone https://github.com/ggerganov/llama.cpp
cd llama.cpp && make
./llama-cli -m llama-3-8b-Q4_K_M.gguf -p "哈囉"
```

  
透過量化（4-bit / 8-bit）**大模型也能在 CPU 跑**：

  
- Llama 7B Q4：4GB RAM 能跑
- Llama 70B Q4：40GB RAM 能跑

### OpenRouter：統一介面

```python
from openai import OpenAI

client = OpenAI(
    base_url="https://openrouter.ai/api/v1",
    api_key="sk-or-..."

# 換模型一行
response = client.chat.completions.create(
    model="meta-llama/llama-3.3-70b-instruct",
    messages=[...]

```

  
### 什麼時候自架？

  
- 每月 LLM 花費 > $1000
- 強烈隱私需求（醫療、金融、政府）
- 需要極致客製（fine-tune）
- 規模大（> 1000 req/sec）
- 合規要求（資料不能出國）

  
### 什麼時候別自架？

  
- 規模小（API 反而便宜）
- 沒 GPU 維運經驗
- 需要最新最強模型
- 追求穩定性

  
### 個人開發者實戰

```text
Mac / 本機開發：Ollama
測試 / Prototype：Ollama 或 OpenRouter
小規模 production：API（OpenAI / Anthropic）
中規模 production：OpenRouter / Groq
大規模 / 隱私：自架 vLLM
邊緣 / 手機：llama.cpp / ggml
```
