**結構化輸出：JSON Mode / Pydantic / Zod**——LLM 寫程式碼用、要**能 parse 的結構化資料**、不是自然語言。**2026 標配**、不會就落伍。

> 💡 **核心痛點**：傳統靠 prompt 「請回 JSON」、LLM 每次格式不一樣 → 維護地獄。新做法強制保證 schema。

  
### 傳統做法的痛苦

```text
Prompt：「請列 3 個 Python 框架、用 JSON 格式」

LLM 可能回（每次不一樣）：

版本 1（OK）：
{"frameworks": ["Django", "Flask", "FastAPI"]}

版本 2（多了 markdown 包覆）：
```json
{"frameworks": ["Django", "Flask", "FastAPI"]}
```

版本 3（多解釋）：
這裡是 3 個框架：
{"frameworks": ["Django", "Flask", "FastAPI"]}
希望對你有幫助！

版本 4（純文字）：
[Django, Flask, FastAPI]

版本 5（缺括號）：
{"frameworks": ["Django", "Flask", "FastAPI"]

→ 你的程式碼要處理各種變形 = 維護地獄
```

  
### 2024+ 結構化輸出（強制 JSON）

**各家 API 都加了「保證 JSON 輸出」功能。**

  
### OpenAI Structured Outputs（2024.8+）

```text
from pydantic import BaseModel
from openai import OpenAI

class Framework(BaseModel):
    name: str
    pros: list[str]
    cons: list[str]
    stars: int
    license: str

class Result(BaseModel):
    frameworks: list[Framework]
    summary: str

client = OpenAI()

response = client.beta.chat.completions.parse(
    model="gpt-4o",
    messages=[{
        "role": "user",
        "content": "列 3 個 Python web 框架"
    }],
    response_format=Result   # ⭐ 直接傳 Pydantic schema
)

# === 直接拿到 typed object ===
result: Result = response.choices[0].message.parsed

for fw in result.frameworks:
    print(fw.name, fw.stars)

# → 100% 保證 schema 正確
# → OpenAI 用 constrained decoding（強制每個 token 都符合 schema）
```

  
### Anthropic：用 Tool Use 達成

**Claude 沒有專門的「JSON mode」、但用 Tool Use 可達成同樣效果：**

```text
tools = [{
    "name": "output_result",
    "description": "輸出結果（請務必使用此工具）",
    "input_schema": {
        "type": "object",
        "properties": {
            "frameworks": {
                "type": "array",
                "items": {
                    "type": "object",
                    "properties": {
                        "name": {"type": "string"},
                        "stars": {"type": "integer"}
                    },
                    "required": ["name", "stars"]
                }
            }
        },
        "required": ["frameworks"]
    }
}]

response = client.messages.create(
    model="claude-opus-4-7",
    max_tokens=1024,
    tools=tools,
    tool_choice={"type": "tool", "name": "output_result"},   # ⭐ 強制用這個 tool
    messages=[{"role": "user", "content": "列 3 個 Python 框架"}]
)

result = response.content[0].input
# → 拿到結構化 dict、schema 跟你定義的一樣
```

### TypeScript：Zod + Vercel AI SDK（前端首選）

```text
import { z } from "zod";
import { generateObject } from "ai";
import { openai } from "@ai-sdk/openai";

const { object } = await generateObject({
  model: openai("gpt-4o"),
  schema: z.object({
    frameworks: z.array(z.object({
      name: z.string(),
      pros: z.array(z.string()),
      stars: z.number(),
      license: z.enum(["MIT", "Apache", "GPL", "Other"])
    }))
  }),
  prompt: "列 3 個 Python web 框架"
});

// === 完全 type-safe ===
object.frameworks[0].name;       // TypeScript 知道是 string
object.frameworks[0].stars;      // number
object.frameworks[0].license;    // "MIT" | "Apache" | "GPL" | "Other"
```

  
### 為什麼結構化輸出重要？

```text
✅ 跟程式碼整合
   → 直接 `.name` 而非 regex 抓

✅ Type safety
   → 用 Pydantic / Zod 驗證
   → IDE 自動補完

✅ 產品化
   → 穩定 = 可信賴
   → 上線不會 500 error

✅ 多步驟流程
   → 上一步 JSON 給下一步當輸入

✅ Tool Use 基礎
   → tool 的參數本質就是結構化

✅ Eval 容易
   → 能寫測試（schema 對 = 對）
```

### Schema 設計 5 大原則

```text
1. ⭐ Schema 簡單
   → 超過 3 層巢狀 LLM 容易亂

2. ⭐ 給 Optional 預設值
   → 避免空值

3. ⭐ 用 enum / Literal 比 string 描述更可靠
   → 強制限制

4. ⭐ 每個欄位寫 description
   → LLM 知道意義

5. ⭐ 用 default 給預設值
   → LLM 不確定時填預設
```

  
### 常見坑（5 大）

```text
1. ❌ 太複雜 schema
   → 超過 20 個欄位 LLM 容易混亂
   ✅ 拆成多次調用

2. ❌ 欄位名衝突
   → 「name」可能指公司 / 人 / 產品
   ✅ 改具體：company_name / person_name

3. ❌ 沒寫 description
   → LLM 不知道欄位意義
   ✅ 每個欄位都寫一句

4. ❌ 沒測 edge case
   → 空陣列、特殊字元、超長文字
   ✅ 寫 test case

5. ❌ LLM 偷懶
   → 陣列回 1 個就停
   ✅ 明確「至少 3 個」+ description 強調
```

  
### 真實案例：發票 OCR + 結構化

```text
import anthropic
import base64
from pydantic import BaseModel
from datetime import date

class InvoiceItem(BaseModel):
    name: str
    quantity: int
    unit_price: float
    total: float

class Invoice(BaseModel):
    invoice_number: str
    date: date
    merchant: str
    items: list[InvoiceItem]
    subtotal: float
    tax: float
    total: float

client = anthropic.Anthropic()

with open("invoice.png", "rb") as f:
    img = base64.b64encode(f.read()).decode()

# === 用 tool use 強制 schema ===
tools = [{
    "name": "extract_invoice",
    "description": "從發票圖片抽出結構化資料",
    "input_schema": Invoice.model_json_schema()
}]

response = client.messages.create(
    model="claude-opus-4-7",
    max_tokens=2048,
    tools=tools,
    tool_choice={"type": "tool", "name": "extract_invoice"},
    messages=[{
        "role": "user",
        "content": [
            {"type": "image", "source": {"type": "base64", "media_type": "image/png", "data": img}},
            {"type": "text", "text": "請抽取這張發票"}
        ]
    }]
)

# === 直接得到 typed Invoice ===
invoice = Invoice(**response.content[0].input)
print(f"商家：{invoice.merchant}")
print(f"總金額：{invoice.total}")
for item in invoice.items:
    print(f"  - {item.name} × {item.quantity} = {item.total}")
```
