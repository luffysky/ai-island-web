**機器學習三大類**——ML 演算法可分三類、看「**訓練資料長什麼樣**」。理解這三類是看懂 AI 的基礎。

> 💡 **2026 重點**：訓練 LLM **三類都用上**——pre-training（非監督）+ SFT（監督）+ RLHF（增強）= ChatGPT。

  
### 1. 監督式學習（Supervised Learning）

**機器看「X 跟正確答案 Y」的資料對**、學完能對新 X 預測 Y。

```text
=== 訓練資料（含答案、已標籤）===
{ 圖片1, "貓" }
{ 圖片2, "狗" }
{ 圖片3, "貓" }
...10 萬筆

=== 訓練後 ===
給新圖片 → 預測「貓 / 狗」
```

> 💡 **像老師教學生考試**：先給你 1000 題練習題（含答案）、考試時遇到新題目你能答。**「練習題」= 訓練資料、「答案」= label**。

  
**實際應用（占 ML 應用 80%）：**

```text
🖼️ 影像分類（image classification）
   → 這是貓 / 狗 / 鳥

📧 垃圾信偵測（spam detection）
   → spam / 正常

🏠 房價預測（regression）
   → 給房子特徵 → 預測價格

🌐 機器翻譯（machine translation）
   → 給「I love you」→ 預測「我愛你」

🎤 語音辨識（speech recognition）
   → 給音檔 + 文字轉錄

📷 OCR（Optical Character Recognition、光學文字辨識）
   → 圖片 + 對應的文字

🏥 醫療診斷
   → X 光片 + 醫生標記

💳 信用評分（credit scoring）
   → 客戶資料 + 是否違約
```

  
**關鍵限制：** **需要大量標籤資料**。標 100 萬張圖很貴（雇外包打標、**ImageNet 花了好幾年**）。

  
### 2. 非監督式學習（Unsupervised Learning）

**只給 X、沒答案**、機器自己找資料的結構。

```text
=== 訓練資料（無標籤）===
[資料1, 資料2, 資料3, ...10000 筆]

=== 機器自己找 ===
- 「這 10000 筆可分成 5 群」
- 「這些資料的關鍵特徵是 A, B, C」
- 「這筆跟其他都不一樣（異常）」
```

  
**實際應用：**

```text
👥 客戶分群（Customer Segmentation）
   → 給客戶資料 → 自動找出「重度使用者」「流失危險群」「新手」

🚨 異常偵測（Anomaly Detection）
   → 信用卡盜刷
   → 伺服器異常流量

📉 降維（Dimensionality Reduction）
   → 1000 維資料壓縮成 10 維
   → PCA / t-SNE / UMAP

🔢 Embedding（嵌入向量）
   → 把文字 / 圖片變成向量
   → RAG 用這個（L14 教）

🎬 推薦系統（Recommendation）
   → 找「跟你看過電影類似的人」

📰 Topic Modeling（主題建模）
   → 自動找出文件主題
   → LDA 演算法
```

  
**難度：** 比監督式難——**沒有「正確答案」可以打分數**。「分成 5 群是對還是錯」沒有標準。

  
### 3. 增強式學習（Reinforcement Learning、RL）

**機器試錯、做對給獎勵、做錯給懲罰**、逐漸學會最佳策略。

```text
=== 訓練流程 ===

1. Agent 在環境中觀察狀態（state）
2. 做動作（action）
3. 環境給獎勵（reward）或懲罰
4. Agent 學「在某 state 下做哪 action 能得最多 reward」
5. 重複幾百萬次

→ 學會最佳策略
```

> 💡 **像訓練狗**：狗坐下 → 給零食、狗亂叫 → 不理。久了狗學會「坐下能得零食」。**機器人學走路、AlphaGo 下圍棋都是這套**。

  
**實際應用：**

```text
🎮 AlphaGo（DeepMind）
   → 下圍棋打敗世界冠軍（2016）

🎮 遊戲 AI
   → 星海爭霸（AlphaStar）/ Dota（OpenAI Five）

🚗 自動駕駛
   → 怎麼判斷該轉彎 / 煞車

🤖 機器人控制
   → 學走路 / 抓物品

📊 廣告投放
   → 自動找最佳 bid 策略

🎬 推薦系統
   → YouTube 推下一支影片

🤖 RLHF（Reinforcement Learning from Human Feedback）
   → 訓練 ChatGPT 的關鍵！
```

  
### RLHF：訓練 ChatGPT 的秘密

2022 OpenAI 用 RLHF 把 GPT-3 升級成 ChatGPT。**三步驟：**

  
**Step 1：Pretrain（pre-training、非監督）**

```text
- 餵 GPT-3 整個網路的文字（45 TB）
- 學會「預測下一個字」
- 但回答可能不友善、不準確、不安全
```

  
**Step 2：SFT（Supervised Fine-Tuning、監督）**

```text
- 雇人寫範例對話（「使用者問 X、最理想答案是 Y」）
- 約 10000 筆高質量範例
- 教 GPT 怎麼當「助手」
```

  
**Step 3：RLHF（人類回饋強化學習、增強）**

```text
- 對同一個問題、讓 GPT 生 4 個答案
- 雇人排序（A > B > C > D）
- 訓練「Reward Model」
- 用 RL 讓 GPT 學會產生高分答案

結果：ChatGPT 比原始 GPT-3 友善、安全、有用
```

> 💡 **2026 所有主流 LLM 都用類似流程**——OpenAI / Anthropic / Google / Meta 全用。

  
**各家有變種：**

```text
🔵 RLHF（OpenAI 原版）

🟣 DPO（Direct Preference Optimization、直接偏好優化）
   → 更簡單、不用 reward model
   → 2024 後主流

🟠 Constitutional AI（Anthropic）
   → 用 AI 自己當裁判
   → 減少人力標註

🟢 RLAIF（Reinforcement Learning from AI Feedback）
   → AI 互相回饋
   → Scaling 友善
```

  
### 三類比較

| 類型 | 資料 | 學什麼 | 難度 | 應用占比 |
|---|---|---|---|---|
| **監督式** | X + Y（答案） | 從 X 預測 Y | ⭐⭐ | 80% |
| **非監督** | 只有 X | 找 X 的結構 | ⭐⭐⭐ | 15% |
| **增強** | 狀態 + 動作 + 獎勵 | 最佳策略 | ⭐⭐⭐⭐⭐ | 5%（但很關鍵） |

  
### 還有其他變種

```text
🔵 半監督（Semi-supervised Learning）
   → 少部分有標籤 + 大部分沒
   → 中間路線

🟣 Self-supervised
   → 機器自己造標籤
   → LLM pre-training 用這個（把文字下個字當「答案」）

🟢 Transfer Learning（遷移學習）
   → 把一個任務學的搬到另一個
   → 在 ImageNet 學過的網路、加少量資料就能識別新類別

🟡 Few-shot / Zero-shot Learning
   → 很少 / 沒有例子就會做
   → LLM 強項

🟠 Meta-learning（學怎麼學）
   → 模型學「如何學習新任務」

🔴 Continual Learning（持續學習）
   → 邊用邊學、不忘舊知識
   → 很難（catastrophic forgetting）
```

  
### 2026 LLM 怎麼結合三類？

訓練 LLM **三類都用上：**

```text
=== 完整 LLM 訓練流程 ===

Step 1：Pre-training（self-supervised、本質無監督）
  → 餵整個網路 45TB 文字
  → 學會語言基礎
  → 占訓練時間 99%

Step 2：Fine-tuning / SFT（supervised）
  → 給範例對話 10000 筆
  → 學怎麼回答
  → 占訓練時間 1%

Step 3：RLHF / DPO（reinforcement）
  → 人類比較答案
  → 學偏好
  → 占訓練時間 0.1%

→ 三種都用上才有現在的 Claude / GPT
```

  
### 怎麼判斷一個 AI 應用屬於哪類？

```text
🔍 看資料有沒有「答案」
   → 有 → 監督式
   → 沒 → 非監督

🔍 看是不是「邊試邊學」
   → 是 → RL
   → 不是 → 監督 / 非監督

🔍 看是不是「產生新東西」
   → 是 → 通常用監督 + 大量資料
   → LLM / Diffusion 都這樣
```

  
### 實務上 indie 怎麼用？

```text
✅ 大部分 indie hacker 不用訓練模型
   → 直接用 OpenAI / Anthropic / Gemini API
   → 1 行 code 解決 90% 場景

✅ 但理解三類能幫你
   → 看懂業界討論
   → 選對工具
   → 不被話術
   → 跟 ML 工程師溝通

❌ 別跳進「自己訓練模型」坑
   → 99% indie 不需要
   → 用 API 便宜 + 效果好
   → 訓練超貴（GPU 燒錢）
```

  
    **☕ 用人話講**
    
ML 三大類：(1) **監督式** 給 X+答案 Y、學從 X 預測 Y（占 80%、用最多） (2) **非監督** 只給 X、自己找結構（分群 / 異常偵測 / Embedding） (3) **增強** 試錯學最佳策略（AlphaGo / RLHF）。**ChatGPT 三類都用**：pre-training（非監督）+ SFT（監督）+ RLHF（增強）。**心法**：indie 不用自己訓練、用 API 就好、但理解原理能跟業界溝通。
