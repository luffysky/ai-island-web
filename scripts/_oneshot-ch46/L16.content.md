**Agent 架構：ReAct / Plan-Execute / Multi-agent**——Agent = **「能自己思考 + 用工具 + 達成目標」的 LLM 系統**。從「**chatbot**」進化到「**doer**」。

> 💡 **2026 真相**：Agent 從炒作（2023 AutoGPT）變成業界標配（Cursor / Claude Code / Devin）。但有限制——不是萬能。

  
### 三大架構

  
### 架構 1：ReAct（最簡單、最常用）

**Think → Act → Observe 不斷循環：**

```text
=== 使用者目標：「找出最新 iPhone 跟前一代的差別」===

Agent 內部：

Thought: 我需要先查最新 iPhone 是哪代
Action: web_search("最新 iPhone 2026")
Observation: iPhone 17 是 2025 發布的最新...

Thought: 那前一代是 iPhone 16、我需要查兩者規格
Action: web_search("iPhone 17 vs iPhone 16 規格")
Observation: [搜尋結果]

Thought: 我已有足夠資訊
Final Answer: iPhone 17 跟 16 主要差別...

→ 直到達標
```

> 💡 **ReAct = Reasoning + Acting**——L9 教過、是所有 Agent 的基礎。

  
### 架構 2：Plan-and-Execute（適合複雜任務）

```text
=== 目標：「準備明天的會議簡報」===

Planner LLM 先列計畫：
1. 看會議邀請知道主題
2. 找相關歷史文件
3. 統整重點
4. 產出 10 張 slides
5. 加圖表
6. 寄給參與者

Executor 一步一步做
每步可能再生子計畫
```

**vs ReAct：**

```text
ReAct：邊想邊做、每步看結果再決定下步
Plan-Execute：先把計畫全列出、再一步步執行

→ Plan-Execute 適合：
  - 任務複雜、需要全局觀
  - 有明確目標、可預先規劃
  - 步驟多（10+）

→ ReAct 適合：
  - 探索型、不確定下步要什麼
  - 步驟少（3-5）
  - 需要快速反應
```

  
### 架構 3：Multi-Agent（多個 agent 協作）

```text
=== 範例：寫文章團隊 ===

🔍 Researcher Agent
   → 搜資料

✍️ Writer Agent
   → 寫初稿

📝 Editor Agent
   → 改進

🎯 Critic Agent
   → 找問題

每個 agent 專精一件事
互相溝通完成複雜任務
```

> 💡 **Multi-Agent 優勢**：(1) 分工明確、效果好 (2) 可以平行跑 (3) 每個 agent 用不同模型（便宜的做搜尋、貴的做寫作） (4) 容易 debug。

  
### Agent vs Tool Use 差別

| | Tool Use | Agent |
|---|---|---|
| **互動** | 使用者問一次、LLM 用工具 1-2 次、回答 | 使用者給目標、LLM 多輪用工具、規劃、執行 |
| **複雜度** | 簡單 | 複雜 |
| **時間** | 秒 | 分鐘-小時 |
| **適合** | 查天氣 / 算數 / 簡單 API | 寫程式 / 研究 / 完整任務 |

  
### Agent Loop 本質

```text
def agent_loop(goal: str, tools: list, max_steps: int = 10):
    """所有 Agent 框架的本質"""
    history = [{"role": "user", "content": goal}]
    
    for step in range(max_steps):
        # === 1. LLM 決定下一步 ===
        response = llm.complete(history, tools=tools)
        
        # === 2. 如果要用 tool、執行 ===
        if response.is_tool_call():
            result = execute_tool(response.tool_call)
            history.append({"role": "tool", "content": result})
        else:
            # === 3. 沒要用 tool、給最終答案 ===
            return response.text
    
    return "超過最大步驟"
```

### Agent 的 5 大挑戰

```text
1. ⚠️ 失控（無限迴圈）
   → Agent 一直想用 tool、永遠不停
   ✅ 設 max_steps（10-20）

2. ⚠️ 誤判（用錯工具 / 做錯事）
   → LLM 對任務理解錯
   ✅ 限制工具範圍 + 重要操作 confirm

3. ⚠️ 成本（100 個 API call）
   → 複雜任務跑很久、燒錢
   ✅ Budget 上限 + 模型路由

4. ⚠️ 幻覺累積
   → 每輪都可能編造、錯誤滾雪球
   ✅ Critic agent 檢查

5. ⚠️ 沒法回頭
   → 寄出去的信收不回
   ✅ 高風險操作 human-in-the-loop
```

  
### 設計安全的 Agent（6 原則）

```text
1. ⭐ 限制步驟數（max_steps = 10-20）
2. ⭐ 限制工具範圍（最小權限）
3. ⭐ 關鍵操作要使用者確認
4. ⭐ 有 budget 上限（每次最多花 $X）
5. ⭐ 能 audit 每步（log）
6. ⭐ 能中斷 / rollback
```

### 完整實作（Vercel AI SDK Agent）

📄 **寫進程式檔**：`agent.ts`

```text
import { generateText, tool } from 'ai';
import { anthropic } from '@ai-sdk/anthropic';
import { z } from 'zod';

const result = await generateText({
  model: anthropic('claude-opus-4-7'),
  maxSteps: 10,                            // ⭐ 限制步驟數
  tools: {
    search: tool({
      description: 'Search the web',
      parameters: z.object({ query: z.string() }),
      execute: async ({ query }) => {
        return await webSearch(query);
      }
    }),
    calculate: tool({
      description: 'Calculate math',
      parameters: z.object({ expression: z.string() }),
      execute: async ({ expression }) => {
        return eval(expression);
      }
    }),
  },
  prompt: '研究 AI 晶片市場、寫一份 500 字報告',
});

console.log(result.text);
console.log(`用了 ${result.steps.length} 步`);
```
