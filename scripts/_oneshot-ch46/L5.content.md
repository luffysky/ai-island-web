**LLM 是什麼：GPT / Claude / Gemini 怎麼運作**——**LLM = Large Language Model**（大型語言模型）。2026 你會遇到的「AI」幾乎都是 LLM。

> 💡 **核心真相**：LLM 本質是「**超大型猜下一個字機器**」——聽起來蠢、但這就是 ChatGPT 革命的全部。

  
### LLM 的本質：超大型「猜下一個字」機器

```text
=== 輸入：「台灣的首都是」===

LLM 內部運算：
1. Tokenize（切 token）：[台灣, 的, 首都, 是]
2. 每個 token 變 embedding（向量）
3. 96 層 Transformer Decoder 跑運算
4. 最後一層輸出 50,000 維機率分布
   （每個可能的 next token 的機率）

=== 輸出（next token 的機率）===
  "台北"  78%
  "高雄"  3%
  "新北"  2%
  "台中"  1.5%
  ...其他 50000+ token

選機率最高 → "台北"
加到輸入：「台灣的首都是台北」

繼續預測：
  "。"    65%
  "市"    15%
  ","    8%
  ...

迴圈直到產生「結束 token」或達 max length
```

  
### 為什麼這樣有用？

因為訓練資料是**整個網路**——書、論文、程式碼、對話、新聞、Wikipedia、Reddit、GitHub。

  
**能預測「下一個字」等於能：**

```text
🤖 回答問題
   → 接著問題後面的「答」

💻 寫程式
   → 接著 function signature 寫實作

🌐 翻譯
   → 接著「翻譯成英文：[原文]」

📝 摘要
   → 接著「請摘要這段：[長文]\n摘要：」

💬 對話
   → 接著對話歷史

🎭 角色扮演
   → 接著「你是莎士比亞、請以他風格回答」
```

> 💡 **所有 NLP 任務都變成「預測下一個 token」**——這是 LLM 的天才之處。**「One model to rule them all」**（一個模型統治所有）。

### 各家 API 接入（3 行 code 跑通）

📄 **寫進程式檔**：`openai_demo.py`

```text
from openai import OpenAI

client = OpenAI(api_key="sk-...")
res = client.chat.completions.create(
    model="gpt-4o",
    messages=[
        {"role": "system", "content": "你是助手"},
        {"role": "user", "content": "哈囉"}
    ]
)
print(res.choices[0].message.content)
```

  
📄 **寫進程式檔**：`anthropic_demo.py`

```text
from anthropic import Anthropic

client = Anthropic(api_key="sk-ant-...")
res = client.messages.create(
    model="claude-opus-4-7",
    max_tokens=1024,
    system="你是助手",
    messages=[
        {"role": "user", "content": "哈囉"}
    ]
)
print(res.content[0].text)
```

  
📄 **寫進程式檔**：`gemini_demo.py`

```text
from google import genai

client = genai.Client(api_key="...")
res = client.models.generate_content(
    model="gemini-2.5-pro",
    contents="哈囉"
)
print(res.text)
```

### 抽象 API client（重要！）

**2026 indie hacker 建議**：**不要綁死一家**。

**原因：**

```text
✅ 各家強項不同、混用 CP 最高
✅ 某家漲價 / 限制 / 出包 → 隨時換
✅ 各家都有出包紀錄（OpenAI 2024 多次大當機）
```

  
**用統一介面：**

| 工具 | 語言 | 特色 |
|---|---|---|
| **Vercel AI SDK** | TypeScript | 換 provider 改一行 |
| **LiteLLM** | Python | 統一介面接 100+ 模型 |
| **OpenRouter** | 任何 | 一個 API key 接所有模型 |
| **LangChain** | TS / Python | 也有抽象但較重 |

  
**LiteLLM 範例（換模型 = 改一行）：**

```text
from litellm import completion

# === 用 OpenAI ===
res = completion(
    model="gpt-4o",
    messages=[{"role": "user", "content": "哈囉"}]
)

# === 換 Anthropic、只改 model 名字 ===
res = completion(
    model="anthropic/claude-opus-4-7",
    messages=[{"role": "user", "content": "哈囉"}]
)

# === 換 Llama 開源（透過 OpenRouter）===
res = completion(
    model="openrouter/meta-llama/llama-3.3-70b-instruct",
    messages=[{"role": "user", "content": "哈囉"}]
)
```

def route_model(query: str, context: dict) -> str:
    # === 簡單 FAQ → 便宜模型 ===
    if is_simple_faq(query):

    return "gpt-4o-mini"

### LLM 的真實局限（必知）

2026 LLM 強到很多人覺得「無所不能」、但有幾個本質局限要知道：

```text
🕰️ 知識截止（Knowledge Cutoff）
   → 訓練資料到某時間、不知道之後的事
   → 補救：RAG / search（L13）

😶 會編造（Hallucination）
   → 不知道時瞎掰
   → 特別是數字、引文、URL
   → 補救：tool use 接 search

🧮 不會真的「算數」
   → 複雜算術會錯
   → 補救：tool use 接計算機

📏 context 有限
   → 再長也有 limit
   → 不能塞整個公司資料
   → 補救：RAG + chunking

💭 沒有持久記憶
   → 每次對話獨立
   → 補救：要自己管 memory

🚫 不能執行動作
   → 本身只生文字
   → 補救：Function Calling / Tool Use（L10）

📝 對 prompt 敏感
   → 同樣意思不同寫法效果差很多
   → 補救：好好學 Prompt Engineering（L8）

🤔 不會真的「推理」
   → 是 pattern matching、不是邏輯推導
   → 補救：用推理模型（o1 / o3 / Claude Opus 思考）
```
