RAG = **「先查資料、再生成」**。讓 LLM 能回答「不在它訓練資料裡」的問題。2026 indie hacker 做 AI 產品的核心技巧。

  
### 為什麼需要 RAG？

  
LLM 本身的限制：

  
- **知識有截止日期**：訓練資料截到某時間
- **不知道你的私有資料**：公司文件、客戶資料、內部 wiki
- **會編造（hallucinate）**：不知道時瞎掰
- **context 有限**：不能把 10GB 文件塞 prompt
- **更新困難**：要更新知識要 fine-tune（貴）

  
RAG 解這些：「**每次問問題前先去你的資料庫找相關資訊、再讓 LLM 回答**」。

  
### RAG 完整流程

```text
=== 準備階段（建立 index，做一次）===

1. 載入文件（PDF / Word / Markdown / 網頁 / DB）
2. 切段（Chunking）—每段 300-800 字 + overlap 50-100
3. 每段轉 embedding（向量）
4. 存進向量資料庫（含原文、向量、metadata）

=== 查詢階段（每次使用者問）===

1. 使用者問題："請假規定是什麼？"
2. 問題轉 embedding
3. 向量 DB 找最相似的 5-10 段
4. 把找到的段落 + 問題 + system prompt 塞給 LLM
5. LLM 根據 context 回答 + 引用來源

→ 答案基於你的資料、不會編造、有來源、不需重訓 LLM
```

  
### RAG 真實範例

```text
使用者問：「公司請假規定是什麼？」

向量搜尋找到 3 段：

段落 1（員工手冊 p.45）：
「全職員工每年享有 14 天年假...」

段落 2（規章 §3.2）：
「請假需提前 3 天告知主管...」

段落 3（HR FAQ）：
「病假超過 3 天需附醫生證明...」

塞給 LLM 的 prompt：
「根據以下文件回答使用者問題。
不要根據文件以外的知識回答。
回答時引用來源。

文件：[段落 1] [段落 2] [段落 3]

問題：公司請假規定是什麼？」

LLM 回答：
「公司請假規定如下：
1. 年假：每年 14 天[1]
2. 申請：需提前 3 天告知主管[2]
3. 病假：超過 3 天需醫生證明[3]

來源：
[1] 員工手冊 p.45
[2] 規章 §3.2
[3] HR FAQ」
```
