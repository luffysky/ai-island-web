### 終極心法：爬蟲不是「能爬」

```text
爬蟲新手：能爬 = 成功
爬蟲老手：能爬 ≠ 成功

真正的成功要 4 件事：
1. ⭐ 法律安全
2. ⭐ 客戶滿意
3. ⭐ 長期維護
4. ⭐ 商業可行
```

  
### 5 大事業心法

  
### 心法 1：領域知識 > 技術深度

```text
❌ 純技術爬蟲工程師
   → 每個案件從 0 開始
   → 客戶想找便宜的人
   → 報價被壓低

✅ 「金融 + 爬蟲」混合型
   → 知道客戶在乎什麼（哪些股價、哪些指標）
   → 客戶找你不是因為「會爬」、是「懂他的領域」
   → 報價權在你

✅ 「電商 + 爬蟲」混合型
   → 知道蝦皮 / momo / PChome 怎麼運作
   → 知道哪些資料商業價值最高
   → 直接提供「**商業洞察**」、不只「資料」

→ 找你熟悉的領域（電商 / 金融 / 房地產 / 醫療）深耕
```

  
### 心法 2：法律意識（保命）

```text
=== 台灣法律重點 ===

⚠️ 妨害電腦使用罪（刑法 358-360 條）
   → 未經授權存取電腦
   → 大量爬蟲可能觸法

⚠️ 著作權法
   → 爬內容 ≠ 重製
   → 用於商業要授權

⚠️ 個資法
   → 個人資料嚴禁
   → 含姓名 / 電話 / Email / IP

⚠️ GDPR（歐盟 / 全球）
   → 爬歐盟使用者資料要遵守
   → 違反罰金 4% 全球營收

⚠️ 公平交易法
   → 不正當競爭
   → 攻擊 / 摧毀競爭對手系統

=== 心法 ===
✅ 商業爬蟲先諮詢律師
✅ 不確定 → 不做
✅ 個資相關 → 別碰
✅ 詐騙相關 → 拒接
✅ ToS 明文禁止 → 評估
```

  
### 心法 3：永遠先 API

```text
=== 為什麼 API > 爬蟲 ===

✅ 穩定
   → API 改版有公告
   → 爬蟲改版直接壞

✅ 合法
   → 官方授權使用
   → 不會被告

✅ 快
   → 結構化 JSON
   → 不用解 HTML

✅ 不會被擋
   → API 是邀請你用
   → 爬蟲是反邀請

✅ 維護成本低
   → 一勞永逸
   → 爬蟲要持續修

=== 心法 ===
先查官方 API → 沒有再考慮爬蟲
80% 場景 API 滿足
別跳過 API 直接爬
```

  
### 心法 4：客戶導向（賺錢核心）

```text
=== 客戶要的不是「資料」、是「解決方案」 ===

❌ 給 CSV
   → 客戶要自己開、自己分析
   → 體驗差、續約率低

✅ 給 Dashboard
   → 客戶一目了然
   → 主動洞察、不用思考
   → 續約率高 5-10x

❌ 一次性交付
   → 賣完就結束
   → 不能複利

✅ 月費維護
   → 自動更新、自動報表
   → 客戶愛、續約 1-3 年
   → 收入可預測

=== 心法 ===
Dashboard > Raw data
自動更新 > 手動寄
月費合約 > 一次性
主動洞察 > 被動查詢
```

  
### 心法 5：持續學新

```text
=== 爬蟲技術 2026 在變 ===

🆕 AI 爬蟲
   → browser-use / Magnitude / Claude Computer Use
   → 改版頻繁的網站新解法

🆕 反偵測升級
   → ML 偵測（行為模式）
   → 對策：模擬真人行為

🆕 TLS impersonate
   → curl-impersonate / got-scraping
   → 過 Cloudflare 主流

🆕 Bun runtime
   → 比 Node 快 3x
   → 2026 新標配

🆕 Cloudflare Workers Browser Rendering
   → Serverless 動態爬蟲
   → 邊緣節點超快

=== 心法 ===
每月學 1 個新技術
追隨 Indie Hackers / Hacker News / X
別停在「以前學的就夠」
3-5 年後落伍 = 工作沒了
```
