**什麼時候要動用 Go / Rust 寫爬蟲？**——99% 的爬蟲 Python / Node 就夠、**1% 規模超大才動 Go / Rust**。這課讓你**不過度工程、也不錯失規模化機會**。

> 💡 **2026 答案**：(1) 1 萬筆/天以下 → Python / Node (2) 10 萬筆/天 → Node + 框架 (3) 100 萬筆/天 → Go (4) 1000 萬+ → Rust + 分散式。

  
### 規模 → 語言選擇表

| 日爬頁數 | 推薦語言 | 原因 |
|---|---|---|
| **< 10K** | **Python / Node** | 寫快、生態完整、規模夠 |
| **10K - 100K** | **Python（Scrapy）/ Node（Crawlee）** | 框架成熟、async 支援 |
| **100K - 1M** | **Go（colly）** | 並發 5 萬+、編譯快、單機強 |
| **1M - 10M** | **Go + 分散式** | colly + Redis queue、多機 |
| **> 10M** | **Rust + 分散式** | 記憶體最省、效能極致 |

### 商業考量（4 個面向）

```text
💰 Cloud 成本
   → Rust 用 1/10 機器、年省幾十萬
   → 規模小無感、規模大有感

⏱️ 開發速度
   → Python（最快）> Node > Go > Rust
   → MVP 階段 Python / Node 優先

👥 找工程師
   → Python（最多）> Node > Go > Rust
   → Rust 工程師薪資高 + 難找

🛠️ 維護成本
   → Go / Rust 編譯型、type safety、bug 少
   → 長期維護 Go / Rust 爽
   → Python 動態型別、bug 容易 runtime 才爆
```

### 何時「不要」用 Go / Rust

```text
❌ 不要過度工程
- 規模 < 1 萬筆 / 天用 Rust = 浪費
- MVP 階段用 Go = 開發慢 3x
- 接案用 Rust = 客戶找不到人維護

❌ 不要為了「炫技」
- 客戶要 1 天 demo、用 Rust 寫一週
- 學技術很爽、但商業不划算

✅ 用 Go / Rust 的時機
- 已經被效能 / 雲端費用打臉
- 規模超大、Python 撐不住
- 大公司、有人才庫
- 你想轉職、學 Go / Rust 提薪
```

  
### 完整爬蟲技術選型流程

```text
=== Step 1：估算規模 ===

人力（Rust 工程師 NT$ 150K+）

=== Step 3：考慮人才 ===
誰維護？團隊現有技能？

=== Step 4：選對工具 ===
< 10K/天：Python（最簡單）
10K-100K：Python Scrapy / Node Crawlee
100K-1M：Go colly
1M-10M：Go + Redis 分散式
> 10M：Rust + 分散式
```
