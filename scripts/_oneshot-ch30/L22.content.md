**儲存策略：CSV vs Parquet vs DB**——爬到的資料**存哪裡決定能不能用**。選錯儲存 = 查詢慢 10-100x、佔空間多 10x。

> 💡 **2026 慣例**：**抓中用 JSONL、處理後存 Parquet、Production 用 PostgreSQL、分析用 DuckDB**。

  
### 6 大儲存格式對比

| 格式 | 大小 | 速度 | Schema | 適合 |
|---|---|---|---|---|
| **CSV** | 大（基準） | 慢 | ❌ | 給人看 / Excel |
| **JSON / JSONL** | 大 1.5x | 中 | 彈性 | 爬蟲中、API 交換 |
| **Parquet** | **小 5-10x** | **快 10x** | ⭐ 強 | 大資料分析 |
| **SQLite** | 中 | 快 | ⭐ SQL | 嵌入式、單機 |
| **PostgreSQL** | 中 | 快 | ⭐ SQL | Production API |
| **DuckDB / ClickHouse** | 小 5x | **快 100x 分析** | ⭐ SQL | OLAP / 分析 |

  
### 1. CSV（人類可讀）

```text
import pandas as pd

df.to_csv("data.csv", encoding="utf-8-sig", index=False)
#                              ↑
#                  utf-8-sig 加 BOM、Excel 開不會亂碼
```

**優點：**
- ✅ 人類可讀
- ✅ Excel / Google Sheets 直接開
- ✅ 跨工具相容

**缺點：**
- ❌ 大（沒壓縮）
- ❌ 慢（純文字 parse）
- ❌ 無 schema（型別都是字串）
- ❌ 不適合大資料（百萬筆爬蟲別用）

**何時用：** 給客戶 / 業務看、< 10 萬筆。

  
### 2. JSONL（爬蟲首選）

```text
df.to_json("data.jsonl", orient="records", lines=True, force_ascii=False)
```

**優點：**
- ✅ 彈性 schema（可巢狀）
- ✅ Streaming 友善（append）
- ✅ jq / Spark / pandas 都支援

**缺點：**
- ❌ 大（比 Parquet 大 5-10x）
- ❌ 慢（純文字 parse）

**何時用：** 爬蟲中暫存、API 交換。

  
### 3. Parquet（大資料推薦）

```text
df.to_parquet("data.parquet", compression="snappy")
df = pd.read_parquet("data.parquet")
```

**Parquet 殺手特性：**

```text
✅ 壓縮（小 5-10x）
   100 MB CSV → 10 MB Parquet

✅ 列式儲存
   只讀「price」欄、不讀整列、超快

✅ 有 schema
   每欄型別固定、不會猜錯

✅ 業界標準
   Spark / Hadoop / DuckDB / BigQuery 全支援

❌ 不能直接 Excel 開
   要寫 code 或用工具
```

**何時用：** 大資料分析、Data Lake、ETL 中間層。

  
### 4. SQLite（嵌入式）

```text
import sqlite3

conn = sqlite3.connect("data.db")
df.to_sql("products", conn, if_exists="replace", index=False)

# === 查 ===
result = pd.read_sql("SELECT * FROM products WHERE price < 1000", conn)
```

**優點：**
- ✅ 嵌入、無需 server
- ✅ SQL 查詢
- ✅ 單檔備份方便

**缺點：**
- ❌ 單一寫入（多 worker 寫會 lock）
- ❌ 不適合 > 100 GB

**何時用：** 單機爬蟲、Bun 內建 SQLite。

  
### 5. PostgreSQL（Production 首選）

```text
from sqlalchemy import create_engine

engine = create_engine("postgresql://user:pass@localhost/mydb")
df.to_sql("products", engine, if_exists="append", index=False)
```

**優點：**
- ✅ ACID 可靠
- ✅ SQL 強大
- ✅ 並發寫入
- ✅ 強大 index
- ✅ 業界主流

**缺點：**
- ❌ 要 server
- ❌ 維護成本

**何時用：** Production API、SaaS 後端、大部分場景。

  
### 6. DuckDB / ClickHouse（分析超快）

**DuckDB = 「**嵌入式 OLAP**」**：

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
uv add duckdb
```

**這行做什麼？** 裝 DuckDB、嵌入式分析資料庫、SQLite 的分析版。

```text
import duckdb

# === 直接查 Parquet 檔（不用 import）===
result = duckdb.sql("""
    SELECT category, AVG(price) AS avg_price, COUNT(*) AS count
    FROM 'data.parquet'
    GROUP BY category
    ORDER BY avg_price DESC
""").df()

print(result)
```

> 💡 **DuckDB 殺手特性**：(1) **直接查 Parquet / CSV / JSON、不用 import** (2) 列式儲存、分析快 100x (3) 嵌入式、無 server (4) 完整 SQL 支援。**取代「**Excel + pandas**」分析工作流**。

  
### S3 / R2（cold storage）

**情境**：歷史 raw 資料、便宜長期存。

```text
import boto3
import io

s3 = boto3.client("s3")

# === 把 DataFrame 存成 Parquet 上傳 ===
buf = io.BytesIO()
df.to_parquet(buf)

s3.put_object(
    Bucket="my-bucket",
    Key=f"data/{date}.parquet",
    Body=buf.getvalue()
)
```

**價格對比（2026）：**

```text
S3 Standard：$0.023/GB/月
Cloudflare R2：$0.015/GB/月  ⭐ 便宜 + 免 egress
Backblaze B2：$0.006/GB/月    最便宜
Wasabi：$0.0059/GB/月         最便宜 + 免 egress

→ 1 TB 資料 R2 = $15/月、隨便存
```

  
### 分層儲存架構（推薦）

```text
=== 完整 production 架構 ===

[Raw（S3 / R2 Parquet）]
  ← 爬蟲直接寫
  → 永遠保留
  → 便宜
  ↓ 每天 ETL
[Staging（PG）]
  ← 清理後
  → 待驗證
  ↓ 驗證通過
[Production（PG + indexed）]
  ← 給 API / dashboard 用
  → 嚴格 schema
  → 高效能 query
  ↓ 每週聚合
[Analytics（DuckDB / ClickHouse / BigQuery）]
  ← 統計報表
  → 列式儲存
  → 分析快 100x
```

  
### 場景選擇表

| 階段 | 推薦 |
|---|---|
| **爬蟲中**（streaming append） | JSONL |
| **處理後存** | Parquet（如果之後要分析） |
| **Production API** | PostgreSQL |
| **單機 / 嵌入式** | SQLite |
| **分析 / OLAP** | DuckDB（嵌入）/ ClickHouse / BigQuery |
| **歸檔 / 備份** | S3 / R2 Parquet |
| **給人看 / 報表** | CSV（小資料） |

  
### Partition（按時間分檔、查詢加速）

**情境**：1 億筆資料、每次只查某天、傻傻全讀慢死。

```text
# === 按年月分檔 ===
for date, group in df.groupby(df['scraped_at'].dt.strftime('%Y-%m')):
    year, month = date.split('-')
    group.to_parquet(
        f"data/year={year}/month={month}/data.parquet"
    )
```

**結構：**

```text
data/
├── year=2026/
│   ├── month=01/data.parquet
│   ├── month=02/data.parquet
│   └── month=03/data.parquet
└── year=2025/
    └── month=12/data.parquet
```

**查詢時 DuckDB / Spark 自動跳過：**

```text
SELECT * FROM 'data/year=2026/month=03/data.parquet'
-- 只讀 3 月、不讀其他、快 100x
```

  
### 3 大心法

```text
1. ⭐ 不要存重複（每階段不同 storage）
   → Raw 在 S3、Production 在 PG、別重複
   → 用 Parquet 不用同時存 CSV

2. ⭐ 列式（Parquet）比行式（CSV）快 10x
   → 大資料一定用 Parquet
   → 不要堅持 CSV 給人看（給 dashboard）

3. ⭐ 巨量資料用 S3 + Parquet + DuckDB
   → S3 便宜（$5-15/TB）
   → Parquet 小 + 快
   → DuckDB 嵌入式分析、無 server
```
