**PHP / Ruby 爬蟲**——2 個「**老牌 web 語言**」、爬蟲生態縮小但**WordPress / Laravel / Rails 整合場景仍有**。

> 💡 **2026 狀態**：(1) PHP 用於 WordPress / Laravel 內整合爬蟲 (2) Ruby 用於 Rails 內或 Mechanize 自動化 (3) 新爬蟲專案**不推薦**用這兩個。

  
### PHP 爬蟲 3 大套件

| 套件 | 用途 | 對應 Python |
|---|---|---|
| **Guzzle** | HTTP client | requests |
| **Symfony DomCrawler** | HTML 解析 + DOM | BeautifulSoup |
| **Goutte（已 deprecated）** | Guzzle + DomCrawler 合體 | requests + BS4 |

  
### PHP - Guzzle（HTTP client）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
composer require guzzlehttp/guzzle
```

**這行做什麼？** 用 Composer（PHP 的 npm）裝 Guzzle。

📄 **寫進程式檔**：`scrape.php`

```text
<?php
require 'vendor/autoload.php';

use GuzzleHttp\Client;

$client = new Client([
    'headers' => [
        'User-Agent' => 'Mozilla/5.0 (Macintosh) Chrome/131.0',
    ],
    'timeout' => 10,
]);

$res = $client->request('GET', 'https://example.com');
$html = $res->getBody()->getContents();

echo $html;
```

  
### PHP - Symfony DomCrawler（HTML 解析）


### Laravel 排程爬蟲

📄 **寫進程式檔**：`app/Console/Commands/Scrape.php`

```text
namespace App\Console\Commands;

use Illuminate\Console\Command;
use GuzzleHttp\Client;
use Symfony\Component\DomCrawler\Crawler;
use App\Models\Product;

class Scrape extends Command {
    protected $signature = 'scrape:daily';
    
    public function handle() {
        $client = new Client();
        $res = $client->get('https://example.com');
        $html = $res->getBody()->getContents();
        
        $crawler = new Crawler($html);
        $crawler->filter('div.product')->each(function ($node) {
            Product::updateOrCreate(
                ['name' => $node->filter('.name')->text()],
                ['price' => $node->filter('.price')->text()]
            );
        });
    }
}
```

📄 **寫進程式檔**：`app/Console/Kernel.php`

```text
protected function schedule(Schedule $schedule) {
    $schedule->command('scrape:daily')->dailyAt('03:00');
}
```

### Ruby - Nokogiri

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
gem install nokogiri
```

**這行做什麼？** 用 gem（Ruby 的 npm）裝 Nokogiri、Ruby 最強 HTML/XML 解析套件。

📄 **寫進程式檔**：`scrape.rb`

```text
require 'nokogiri'
require 'open-uri'

doc = Nokogiri::HTML(URI.open('https://example.com'))

doc.css('div.product').each do |product|
  name = product.css('.name').text
  price = product.css('.price').text
  puts "#{name} - #{price}"
end
```

> 💡 **Nokogiri 強項**：(1) C 編譯、超快 (2) 同時支援 CSS selector + XPath (3) 是 Ruby 業界標準。

  
### Ruby - Mechanize（自動化登入 / 表單）

```text
require 'mechanize'

agent = Mechanize.new

# === 開登入頁 ===
page = agent.get('https://example.com/login')

# === 填表單 ===
form = page.forms.first
form.email = 'user@example.com'
form.password = 'secret'

# === 送出、自動 follow redirect ===
page = agent.submit(form)

# === 已登入、繼續爬 ===
dashboard = agent.get('https://example.com/dashboard')
puts dashboard.search('h1').text
```

> 💡 **Mechanize** = Ruby 的 requests + cookie jar + form handler 合體、自動化登入超方便。

  
### Rails 內整合爬蟲

📄 **寫進程式檔**：`app/jobs/scrape_job.rb`

```text
class ScrapeJob < ApplicationJob
  queue_as :default
  
  def perform
    doc = Nokogiri::HTML(URI.open('https://example.com'))
    
    doc.css('div.product').each do |p|
      Product.find_or_create_by(name: p.css('.name').text) do |product|
        product.price = p.css('.price').text.to_f
      end
    end
  end
end

# === 用 Whenever gem 排程 ===
# config/schedule.rb
every 1.day, at: '3:00 am' do
  runner 'ScrapeJob.perform_later'
end
```

  
### 為什麼這 2 個語言爬蟲少人用

```text
1. 生態縮小
   → PHP 主要做 web、爬蟲場景萎縮
   → Ruby 整體市占下降

2. 沒 Scrapy 等級框架
   → 都要自己組（HTTP + 解析 + queue）
   → Scrapy / Crawlee 開箱即用、差距大

3. Async 支援弱
   → PHP Reactphp / Ruby Async 都不流行
   → 並發要靠 thread / process

4. 招募市場小
   → PHP / Ruby 工程師越來越少
   → 客戶找不到人維護

5. 中文資源少
   → 比不過 Python / Node
```

  
### 何時用 PHP / Ruby 爬蟲

```text
✅ 用 PHP
- 公司是 WordPress / Laravel shop
- 已有 PHP 後端、爬蟲整合進去
- WP 外掛賣家（爬蟲是其中一個功能）

✅ 用 Ruby
- 公司是 Rails shop
- Shopify 生態（Shopify 是 Rails 出身）
- Mechanize 做自動化測試 / 表單

❌ 不用 PHP / Ruby
- 新爬蟲專案
- 純爬蟲（不整合其他系統）
- 個人 / Indie 起步
- MVP / PoC
- 規模化
```

  
### 招募市場（2026 台灣）

```text
=== PHP / Laravel ===
新手：NT$ 45-65k
中堅：NT$ 65-100k
資深：NT$ 100-150k

=== Ruby on Rails ===
新手：NT$ 50-70k
中堅：NT$ 70-110k
資深：NT$ 110-160k

→ 兩個都是「**穩定下降**」中
→ 但會的人少、現有公司還是要養
→ 適合「**接特定客戶 case**」
```

  
### Indie 建議

```text
=== 不要學 PHP / Ruby 爬蟲、除非 ===

✅ 你已經會 PHP / Ruby
✅ 你的客戶用 Laravel / Rails
✅ 你要寫 WordPress 外掛
✅ 你的接案範圍受限

=== 推薦學習順序（一般） ===

1. Python（必學、生態最強）
2. Node（補、Next.js 全端）
3. Go（規模化時）
4. PHP / Ruby（特定客戶才學）
```

  
### 心法

```text
✅ PHP / Ruby 不是不能用、是不主流
✅ 已有 PHP / Ruby 系統時用、不要為了爬蟲學
✅ 接 WordPress / Laravel / Rails 客戶用、其他不用
✅ 新專案直接 Python / Node、別走回頭路

❌ 別跟風學「老語言」、市場 ROI 低
❌ 別為了「精通多語言」學 5 個、深入 2 個夠用
```
