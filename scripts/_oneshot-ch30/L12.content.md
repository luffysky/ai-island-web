**cURL = 命令列爬蟲**——所有作業系統內建、**1 行就抓網頁**、適合一次性 / CI/CD / Docker 內 / 簡單腳本。

> 💡 **cURL 不是程式語言**——它是命令列工具、配 jq / xargs / bash 可以做很多事。**「shell scripting + cURL」是 sysadmin / DevOps 必備技能**。

  
### cURL 基本

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
curl https://example.com
```

**這行做什麼？** 抓網頁、印到 stdout。

  
**5 個常用選項：**

```text
# === 存到檔案 ===
curl -o output.html https://example.com

# === Follow redirect（重要！否則跳轉抓不到）===
curl -L https://example.com

# === 加 header ===
curl -H "User-Agent: Mozilla/5.0 (Macintosh) Chrome/131.0" https://example.com

# === Silent mode（不顯 progress bar）===
curl -s https://example.com

# === Show only headers ===
curl -I https://example.com
```

  
### POST 請求

```text
# === JSON POST ===
curl -X POST https://api.example.com/login \
  -H "Content-Type: application/json" \
  -d '{"email": "a@b.com", "password": "secret"}'

# === Form data POST ===
curl -X POST https://example.com/submit \
  -d "email=a@b.com" \
  -d "name=Alice"

# === 上傳檔案 ===
curl -X POST https://api.example.com/upload \
  -F "file=@photo.jpg" \
  -F "description=My photo"
```

  
### 並發爬（xargs）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
cat urls.txt | xargs -P 10 -I{} curl -s {} -o {}.html
```

**這行做什麼？**
- `cat urls.txt` —— 讀 URL 清單檔（每行一個 URL）
- `xargs -P 10` —— **並行 10 個 process** 跑
- `-I{}` —— 把 URL 替換成 `{}`
- `curl -s {} -o {}.html` —— 抓 URL、存成檔

> 💡 **`xargs -P` 是 unix 的「免費並發工具」**——不用學 Python asyncio、bash 就能並發爬。

  
### 解析 JSON（jq）

**jq = JSON 的 sed / awk、命令列 JSON 處理神器**：

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
# Mac
brew install jq

# Linux
sudo apt install jq

# Windows（PowerShell）
winget install jqlang.jq
```

**這 3 種做什麼？** 在不同 OS 裝 jq。

  
**用法：**

```text
# === 取所有商品 ===
curl https://api.example.com/products | jq '.products[]'

# === 取 name 和 price ===
curl https://api.example.com/products | jq '.products[] | {name, price}'

# === 過濾：price < 1000 ===
curl https://api.example.com/products | jq '.products[] | select(.price < 1000)'

# === 轉 CSV ===
curl https://api.example.com/products | jq -r '.products[] | [.name, .price] | @csv'

# === 取陣列長度 ===
curl https://api.example.com/products | jq '.products | length'
```

> 💡 **jq syntax** 像 SQL + functional language、學會超強。**craplab.com/jqplay** 線上玩 jq。

  
### bash 完整爬蟲腳本

📄 **寫進程式檔**：`scrape.sh`

```text
#!/bin/bash

# === 確保 bash strict mode ===
set -euo pipefail

# === 每頁爬一次、間隔 1 秒 ===
for page in {1..10}; do
    echo "Scraping page $page..."
    
    curl -s "https://example.com/products?page=$page" \
        -H "User-Agent: Mozilla/5.0 (Macintosh) Chrome/131.0" \
        > "data/page_$page.html"
    
    sleep 1
done

echo "Done!"
```

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
chmod +x scrape.sh
./scrape.sh
```

**這 2 行做什麼？**
- `chmod +x` —— 加執行權限
- `./scrape.sh` —— 跑腳本

  
### 命令列 HTML 解析（htmlq / pup）

**htmlq = jq 的 HTML 版（Rust 寫的、超快）：**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
cargo install htmlq
```

**這行做什麼？** 用 Cargo（Rust 套件管理）裝 htmlq、命令列 HTML 解析。

```text
# === 取所有 div.product .name 的文字 ===
curl -s https://example.com | htmlq 'div.product .name' --text

# === 取所有 a[href] 的連結 ===
curl -s https://example.com | htmlq 'a[href]' --attribute href
```

  
**pup = Go 寫的 HTML 解析、語法不同：**

```text
brew install pup

# === 取商品名 ===
curl https://example.com | pup 'div.product .name text{}'

# === 取所有連結 ===
curl https://example.com | pup 'a attr{href}'
```

  
### 完整 bash + jq + curl 爬蟲（API 版）

📄 **寫進程式檔**：`scrape-api.sh`

```text
#!/bin/bash
set -euo pipefail

mkdir -p data

for keyword in "iphone" "macbook" "ipad"; do
    # === URL encode ===
    encoded=$(echo "$keyword" | jq -sRr @uri)
    
    echo "Scraping $keyword..."
    
    # === fetch + 解析 + 存 CSV ===
    curl -s "https://api.example.com/search?q=$encoded" \
        -H "User-Agent: Mozilla/5.0 (Macintosh) Chrome/131.0" \
        | jq -r '.products[] | [.name, .price] | @csv' \
        > "data/$keyword.csv"
    
    sleep 2
done

echo "Done! Files in data/"
```

  
### cron 排程

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在 server 終端機輸入**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
crontab -e
```

**加排程：**

```text
0 3 * * * /home/user/scrape.sh >> /var/log/scrape.log 2>&1
```

**這行做什麼？** 每天凌晨 3 點跑 `scrape.sh`、log 寫到 `/var/log/scrape.log`。

  
### cURL vs 其他工具對比

| 場景 | cURL + bash | Python | Node |
|---|---|---|---|
| 簡單一次性抓 | ⭐ 最簡 | 過度 | 過度 |
| 抓 100 個 URL | OK | 較好 | 較好 |
| 解 HTML | 弱（要 htmlq） | ⭐ BS4 強 | ⭐ cheerio 強 |
| 動態 JS | ❌ | Playwright | Playwright |
| 錯誤處理 | 弱 | ⭐ | ⭐ |
| 寫 DB | 弱 | ⭐ | ⭐ |
| CI/CD 內 | ⭐ 內建 | 要裝 | 要裝 |
| Docker 內 | ⭐ alpine 內建 | 要裝 | 要裝 |

  
### 何時用 cURL 爬蟲

```text
✅ 用 cURL
- 一次性簡單抓（< 100 個 URL）
- CI/CD 內（不裝 Python / Node）
- Docker alpine 內（極小 image）
- 健康檢查（curl https://api/health）
- sysadmin / DevOps 自動化
- shell script 內

❌ 不用 cURL
- 複雜邏輯（用 Python）
- 解 HTML（用 BS4 / cheerio）
- 大量 URL（用 Scrapy / Crawlee）
- 動態 JS（用 Playwright）
- 錯誤處理重要（用 Python）
```
