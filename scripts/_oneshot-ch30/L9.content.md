### 為什麼 Rust 快

```text
⚡ 編譯型
   → 沒有 Python interpreter overhead
   → CPU 直接執行 native code

🔒 記憶體安全
   → ownership 系統避免 memory leak / use-after-free
   → 適合 long-running service（爬蟲跑幾個月不會炸）

🚀 zero-cost abstractions
   → 高階語法、runtime 開銷 0
   → for / iterator 跟手寫 C 一樣快

📦 編譯 binary 小
   → 5-10 MB（Python 500MB、Node 100MB）
   → Docker image 5MB、K8s 部署超快
```

  
### 最簡 reqwest + scraper

📄 **寫進程式檔**：`src/main.rs`

```text
use reqwest;
use scraper::{Html, Selector};

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    // === 1. 抓 HTML ===
    let res = reqwest::get("https://example.com").await?.text().await?;
    
    // === 2. 解析 ===
    let doc = Html::parse_document(&res);
    let selector = Selector::parse("div.product").unwrap();
    
    // === 3. 迭代 ===
    for el in doc.select(&selector) {
        let name = el.select(&Selector::parse(".name").unwrap())
            .next()
            .map(|e| e.text().collect::<String>());
        
        println!("{:?}", name);
    }
    
    Ok(())
}
```

  
📄 **寫進程式檔**：`Cargo.toml`

```text
[dependencies]
reqwest = { version = "0.12", features = ["json"] }
scraper = "0.18"
tokio = { version = "1", features = ["full"] }
```

  
### 並發（join_all）

```text
use futures::future::join_all;

let tasks: Vec<_> = urls.into_iter().map(|url| async move {
    reqwest::get(url).await?.text().await
}).collect();

let results: Vec<_> = join_all(tasks).await;
```

> 💡 **vs `buffer_unordered`**：(1) `join_all` 全部一起跑（沒並發限制） (2) `buffer_unordered(n)` 限制並發 n。**爬蟲用後者**、別把對方打爆。

  
### 為什麼少人用 Rust 寫爬蟲

```text
1. ❌ 學習超陡
   ownership / borrow / lifetime / async
   學 Python 1 週、學 Rust 2-3 個月

2. ❌ 生態小
   爬蟲套件少、沒 Scrapy / Crawlee 等級框架
   每個小功能都要自己組

3. ❌ 爬蟲是 IO bound
   主要時間花在「等網路」、不是「跑 CPU」
   Python 等 vs Rust 等、差不多

4. ❌ 開發速度慢
   Python 1 天、Rust 1 週
   接案 / Indie / MVP 不划算

5. ❌ 找工程師難 / 貴
   Rust 工程師台灣稀缺、薪資 NT$ 150-300K
   接案客戶找不到人維護
```

  
### 何時用 Rust 爬蟲（真實場景）

```text
✅ 巨大 scale（每天億級 req）
   → Cloudflare 內部監控爬蟲
   → 搜尋引擎索引

✅ 已是 Rust 團隊
   → Discord backend
   → Figma / Linear / 1Password 工程

✅ Long-running service
   → 爬蟲跑幾個月不能崩
   → 記憶體要超穩

✅ 嵌入式 / 邊緣
   → 路由器 / IoT / 區塊鏈節點
   → 小記憶體環境

✅ 高頻 / 量化交易
   → 微秒級反應
   → 不能有 GC pause

❌ 個人 / Indie / 接案 / MVP / 99% 場景
```

### Indie / 接案 建議

```text
=== 不要學 Rust 爬蟲、除非 ===

✅ 你已經會 Python / Node 爬蟲
✅ 你做的爬蟲 scale 真的超大
✅ 你想跳大廠（Cloudflare / Discord）
✅ 你享受學習痛苦過程

=== 否則 ===

學 Python（簡單、生態強、80% 場景）
學 Node（已會 JS、Next.js 全端友善）
學 Go（要規模化時）
別碰 Rust（時間花在學語言、不在解決問題）
```

  
### 心法總結

```text
1. Python 慢但夠用
   → 99% 場景不會遇到瓶頸
   → 慢 5x 但對方反應 1 秒、你慢 0.2 秒沒差

2. Rust 快但累
   → 寫起來 3-5x 時間
   → 但跑 50x 快、單機 10x 並發
   → 大規模才有 ROI

3. 別過度工程
   → 100 個 URL 用 Rust = 浪費生命
   → 1000 萬 URL 用 Python = 燒錢

4. 看 ROI 選工具
   → 時間 vs 雲端費用
   → 1 個 indie 算自己工資 vs proxy 月費
```
