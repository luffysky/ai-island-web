### 第一個 colly 爬蟲

📄 **寫進程式檔**：`main.go`

```text
package main

import (
    "fmt"
    "github.com/gocolly/colly/v2"
)

func main() {
    // === 1. 建 Collector（colly 的核心）===
    c := colly.NewCollector(
        colly.AllowedDomains("books.toscrape.com"),    // 限定爬這個 domain
        colly.MaxDepth(3),                              // 最多遞迴 3 層
    )
    
    // === 2. 註冊 callback：抓到 article.product_pod ===
    c.OnHTML("article.product_pod", func(e *colly.HTMLElement) {
        title := e.ChildAttr("h3 a", "title")
        price := e.ChildText(".price_color")
        fmt.Printf("%s | %s\n", title, price)
    })
    
    // === 3. 找下一頁、自動繼續爬 ===
    c.OnHTML("li.next a", func(e *colly.HTMLElement) {
        e.Request.Visit(e.Attr("href"))
    })
    
    // === 4. 錯誤處理 ===
    c.OnError(func(r *colly.Response, err error) {
        fmt.Printf("Error: %s\n", err)
    })
    
    // === 5. 起跑 ===
    c.Visit("http://books.toscrape.com")
}
```

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
go run main.go
```

**這行做什麼？** 編譯 + 立刻跑 `main.go`（不存編譯檔、適合開發）。

> 💡 **colly 設計哲學**：(1) `OnHTML(selector, fn)` 註冊「**看到這個 selector 就跑 fn**」 (2) `e.Request.Visit(url)` 加 URL 進 queue (3) colly 自動 BFS 爬。

  
### 並發控制（colly 強項）

```text
import "time"

// === 限制並發 + 加 delay ===
c.Limit(&colly.LimitRule{
    DomainGlob:  "*",                          // 對所有 domain
    Parallelism: 10,                           // 同時 10 個
    Delay:       1 * time.Second,              // 每次間隔 1 秒
    RandomDelay: 2 * time.Second,              // 額外隨機 0-2 秒
})

// === Async 模式（用 goroutine 並行）===
c2 := c.Clone()
c2.Async = true                                // 開啟非同步
c2.Visit("https://example.com")
c2.Wait()                                      // ⚠️ 一定要 Wait、不然主程式提早退出
```

> 💡 **goroutine 是 Go 的「輕量級 thread」**——創 1 萬個沒壓力（Python thread 創 100 就慢了）。colly Async 模式 = 自動用 goroutine 並行。

### 儲存到 CSV（goroutine 安全）

```text
import (
    "encoding/csv"
    "os"
    "sync"
)

type Book struct {
    Title string
    Price string
}

var (
    books []Book
    mu    sync.Mutex                           // 多 goroutine 寫 slice 要 lock
)

c.OnHTML("article.product_pod", func(e *colly.HTMLElement) {
    book := Book{
        Title: e.ChildAttr("h3 a", "title"),
        Price: e.ChildText(".price_color"),
    }
    
    // === Lock 保護 ===
    mu.Lock()
    books = append(books, book)
    mu.Unlock()
})

c.Visit(url)
c.Wait()

// === 寫 CSV ===
file, _ := os.Create("books.csv")
defer file.Close()

w := csv.NewWriter(file)
w.Write([]string{"Title", "Price"})            // header
for _, b := range books {
    w.Write([]string{b.Title, b.Price})
}
w.Flush()
```

> 💡 **`sync.Mutex` 是 Go 的鎖**——多 goroutine 同時寫 slice 會出錯、要 lock。Python GIL 自動保護 list、Go 沒有、要手動。

  
### 分散式 colly（Redis backend）

**情境**：5 台機器同時跑同一個爬蟲、共用 queue + 去重。

```text
import "github.com/gocolly/redisstorage"

storage := &redisstorage.Storage{
    Address:  "localhost:6379",
    Password: "",
    DB:       0,
    Prefix:   "my-scraper",
}

c.SetStorage(storage)

// → 5 台機器都這樣設、自動共用 queue
// → A 機器加 URL 到 queue、B 機器拿來爬
// → Redis 自動去重（同 URL 不重爬）
```

> 💡 **這就是 1000 萬+/天爬蟲架構基礎**——多機 + Redis queue + colly。
