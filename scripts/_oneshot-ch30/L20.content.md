**資料品質：去重 + 驗證**——爬蟲不只要「**抓到**」、要「**抓對**」+「**不重複**」+「**格式對**」。**這課決定爬蟲商業價值**。

> 💡 **核心心法**：「**抓的時候就驗證、不放髒資料進 DB**」。垃圾資料 = 垃圾 SaaS、客戶會跑。

  
### 4 種去重技術（依規模選）

| 規模 | 推薦 | 記憶體 |
|---|---|---|
| **< 10 萬** | Python `set()` | URL × 100 bytes |
| **10 萬-100 萬** | Redis SET（多 worker 共用） | 同 set |
| **100 萬-1 億** | Bloom Filter（機率） | < 100MB |
| **任何規模** | DB UNIQUE constraint（最穩） | DB index |

  
### 1. URL 去重（最簡單）

```text
seen = set()

for url in urls:
    if url in seen:                            # O(1) 查詢
        continue
    seen.add(url)
    scrape(url)
```

> 💡 **set 不是 list**——`if x in list` 是 O(n)、`if x in set` 是 O(1)。**百萬筆 list 會慢 10000x**、用 set。

  
### 2. Bloom Filter（百萬級）

**問題**：1 億個 URL 用 set 要 10GB 記憶體。

**解法**：Bloom Filter = **機率性資料結構**、用 hash 判斷「**可能存在 / 一定不存在**」、記憶體只要 100MB。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
uv add pybloom-live
```

**這行做什麼？** 裝 pybloom-live、Python 最常用 Bloom Filter 套件。

```text
from pybloom_live import BloomFilter

# === 容量 100 萬、誤判率 0.1% ===
bf = BloomFilter(capacity=1_000_000, error_rate=0.001)

for url in urls:
    if url in bf:                              # 可能爬過、跳過
        continue
    bf.add(url)
    scrape(url)
```

**注意：Bloom Filter 有 false positive（誤判）：**

```text
- 「不存在」→ 100% 確定
- 「可能存在」→ 有 0.1% 機率是新的
- 結論：可能漏爬 0.1%、可接受就用
```

  
### 3. Content 去重（內容 hash）

**情境**：同一商品在不同 URL、要去重 by 內容。

```text
import hashlib

def content_hash(item):
    """把 item 字串化後 MD5"""
    s = f"{item['name']}|{item['price']}|{item['link']}"
    return hashlib.md5(s.encode()).hexdigest()

seen_hashes = set()

for item in items:
    h = content_hash(item)
    if h in seen_hashes:
        continue
    seen_hashes.add(h)
    save(item)
```

### 5. DB UNIQUE constraint（最穩）

**SQL 層級保證不重複、應用層不用想**：

```text
CREATE TABLE products (
    id UUID PRIMARY KEY,
    link TEXT UNIQUE,                          -- ⭐ link 不能重複
    name TEXT,
    price NUMERIC,
    scraped_at TIMESTAMP DEFAULT NOW()
);

-- === INSERT 重複時自動忽略 ===
INSERT INTO products (link, name, price)
VALUES (...)
ON CONFLICT (link) DO NOTHING;

-- === 或 UPDATE 既有 ===
INSERT INTO products (link, name, price)
VALUES (...)
ON CONFLICT (link) DO UPDATE
SET name = EXCLUDED.name, price = EXCLUDED.price;
```

> 💡 **`ON CONFLICT DO NOTHING` / `DO UPDATE`** 是 PostgreSQL 神技、UPSERT 寫法、爬蟲 idempotent 的關鍵。

  
### 資料驗證（Pydantic）

**為什麼用 Pydantic**：(1) runtime 驗證 (2) 自動 type cast (3) 自訂規則 (4) error 訊息清楚。

```text
from pydantic import BaseModel, validator, HttpUrl
from pydantic import ValidationError
import logging

logger = logging.getLogger(__name__)

class Product(BaseModel):
    name: str
    price: float
    link: HttpUrl                              # 自動驗 URL 格式
    
    @validator('name')
    def name_not_empty(cls, v):
        if not v.strip():
            raise ValueError('name 不能空')
        return v.strip()
    
    @validator('price')
    def price_reasonable(cls, v):
        if v < 0:
            raise ValueError('price 不能負')
        if v > 1_000_000:
            raise ValueError('price 異常（> 100 萬）')
        return v

# === 用 ===
try:
    p = Product(name='AirPods', price=5990, link='https://shopee.tw/...')
    save(p)
except ValidationError as e:
    logger.error(f'invalid: {e}')
    skip_count += 1
```

  
### 資料品質監控（必加）

```text
total = 0
valid = 0
duplicates = 0
errors = []

for raw in scrape_all():
    total += 1
    
    try:
        # === 1. 驗證 ===
```

### 異常偵測（價格 outlier）

```text
import numpy as np

prices = [p.price for p in products]
mean = np.mean(prices)
std = np.std(prices)

# === 3 sigma 法則：超出 3 個標準差 = 異常 ===
for p in products:
    if abs(p.price - mean) > 3 * std:
        print(f"⚠️ 異常價格：{p.name} NT$ {p.price}（平均 NT$ {mean:.0f}）")
```

> 💡 **常見異常**：(1) 蝦皮價格沒除 100000、應為 5990 卻是 5990 萬 (2) 漏掉小數點 (3) 抓到「2 件」價格、不是單價。

  
### 完整品質檢查 pipeline

```text
def quality_check(items):
    """完整品質檢查、回傳統計"""
    stats = {
        'total': 0,
        'valid': 0,
        'duplicates': 0,
        'invalid': 0,
        'outliers': 0,
    }
    
    seen = set()
    valid_items = []
    
    for raw in items:
        stats['total'] += 1
        
        # === 1. Pydantic 驗證 ===
        try:
            p = Product(**raw)
        except ValidationError:
            stats['invalid'] += 1
            continue
        
        # === 2. 去重 ===
        if p.link in seen:
            stats['duplicates'] += 1
            continue
        seen.add(p.link)
        
        valid_items.append(p)
        stats['valid'] += 1
    
    # === 3. Outlier 偵測 ===
    if valid_items:
        prices = [p.price for p in valid_items]
        mean = np.mean(prices)
        std = np.std(prices)
        
        for p in valid_items:
            if abs(p.price - mean) > 3 * std:
                stats['outliers'] += 1
                logger.warning(f"outlier: {p}")
    
    return valid_items, stats
```

  
### 5 大心法

```text
1. ⭐ 抓的時候就驗證
   → 不要先存、之後再清
   → 髒資料進 DB 後悔莫及

2. ⭐ 不放髒資料進 DB
   → Pydantic / Zod 驗證
   → DB UNIQUE constraint

3. ⭐ 監控有效率（< 80% 必有問題）
   → 網站改版？反爬？格式變？
   → 立刻告警

4. ⭐ 異常 alert
   → outlier 偵測
   → 抓量異常

5. ⭐ 定期清舊 / 無效資料
   → 30 天前的可能過時
   → 重複的合併
```
