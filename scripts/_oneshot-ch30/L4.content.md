**爬蟲排程 4 大方案**——Linux cron / GitHub Actions / Vercel Cron / Temporal-Inngest。**選對工具省時間、不掉資料**。

> 💡 **2026 選擇**：個人用 cron 或 GH Actions、Next.js 用 Vercel Cron、商業級用 Inngest。

  
### 4 種排程對比

| 方案 | 月費 | 適合 | 複雜度 |
|---|---|---|---|
| **Linux cron** | $0（已有 server） | 個人 / 學習 | ⭐ 最簡 |
| **GitHub Actions** | $0 | 小型 SaaS / Indie | ⭐⭐ 簡單 |
| **Vercel Cron** | $0（Hobby 1 個） | Next.js 專案 | ⭐⭐ 簡單 |
| **Inngest / Temporal** | $0（免費額度） | Production / 複雜 workflow | ⭐⭐⭐⭐ 進階 |

  
### 1. Linux cron（最傳統、最簡單）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在 server 終端機輸入**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
crontab -e
```

**這行做什麼？** 編輯目前使用者的 crontab（cron 設定檔）、會開預設編輯器（通常是 nano / vi）。

  
**寫排程（5 個範例）：**

```text
# 每天早上 8 點跑 Python 爬蟲
0 8 * * * /usr/bin/python3 /home/user/scraper.py >> /var/log/scraper.log 2>&1

# 每小時跑 Node 爬蟲
0 * * * * /usr/bin/node /home/user/scraper.js

# 每 5 分鐘
*/5 * * * * /scrape.sh

# 每週一早上 9 點
0 9 * * 1 /weekly-report.py

# 每月 1 號跑備份
0 0 1 * * /backup.sh
```

> 💡 **`>> /var/log/scraper.log 2>&1`** = (1) `>>` 把 stdout 追加到 log (2) `2>&1` 把 stderr 也導到 stdout (3) 不寫 = log 消失。**爬蟲一定要寫**、否則 debug 無路可走。

  
**cron 語法（5 個欄位）：**

```text
* * * * * command
│ │ │ │ │
│ │ │ │ └─ 星期幾 (0-6, 0=Sunday)
│ │ │ └─── 月份 (1-12)
│ │ └───── 日 (1-31)
│ └─────── 小時 (0-23)
└───────── 分鐘 (0-59)

特殊：
*/5  每 5 個單位（每 5 分 / 每 5 小時...）
1,5,10  指定 1、5、10
1-10  1 到 10
*  任何
```

  
**檢查 cron 是否在跑：**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在 server 終端機輸入**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
sudo systemctl status cron
tail -f /var/log/cron
crontab -l
```

**這 3 行做什麼？**
- `systemctl status cron` —— 看 cron service 是不是活著
- `tail -f /var/log/cron` —— 即時看 cron log（誰跑了什麼）
- `crontab -l` —— 看你目前的 crontab 設定

  
### 2. GitHub Actions（免費、推薦小型）

**情境**：你不想自己架 server、但要定時跑爬蟲。

📄 **寫進程式檔**：`.github/workflows/scrape.yml`

```text
name: Daily Scraper

on:
  schedule:
    - cron: '0 0 * * *'                       # UTC 0:00 = 台北 8:00
  workflow_dispatch:                           # 也能手動觸發

jobs:
  scrape:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - uses: actions/setup-python@v5
        with:
          python-version: '3.12'
      
      - name: Install deps
        run: pip install -r requirements.txt
      
      - name: Run scraper
        run: python scraper.py
        env:
          API_KEY: ${{ secrets.API_KEY }}      # 從 GitHub Secrets 拿
      
      - name: Commit results
        run: |
          git config user.name github-actions
          git config user.email actions@github.com
          git add data/
          git commit -m "chore: daily scrape $(date +%Y-%m-%d)" || echo "Nothing to commit"
          git push
```

  
**GitHub Actions 優勢：**

```text
✅ 免費（公開 repo 無限、私 repo 每月 2000 分鐘）
✅ 版本控制（commit 進 repo）
✅ 不用自己架 server
✅ secret 管理內建
✅ matrix build（多版本 / 多平台）

❌ 限制
- 排程不準（可能延遲 5-30 分鐘）
- 每次最長 6 小時
- 不適合超頻繁排程（每分鐘）
```

  
### 3. Vercel Cron（Next.js 友善）

📄 **寫進程式檔**：`vercel.json`

```text
{
  "crons": [
    {
      "path": "/api/cron/scrape",
      "schedule": "0 0 * * *"
    }
  ]
}
```

📄 **寫進程式檔**：`app/api/cron/scrape/route.ts`

```text
export async function GET(req: Request) {
  // === 驗證是 Vercel 來的（不是有人偷打）===
  const auth = req.headers.get('authorization');
  if (auth !== `Bearer ${process.env.CRON_SECRET}`) {
    return new Response('Unauthorized', { status: 401 });
  }
  
  const data = await scrapeWebsite();
  await saveToDatabase(data);
  
  return new Response('OK');
}
```

> 💡 **Vercel Cron 限制**：(1) Hobby 1 個 cron / Pro $20/月 40 個 (2) Function timeout 10s（Hobby）/ 60s（Pro） (3) **長任務分批做**（L20 教過）。

  
### 4. Inngest / Temporal（Production 級工作流）

**情境**：爬蟲流程複雜（多步驟、要 retry、要狀態）、cron 不夠用。

📄 **寫進程式檔**：`src/inngest/scrape.ts`

```text
import { inngest } from './client';

export const dailyScrape = inngest.createFunction(
  { id: 'daily-scrape' },
  { cron: '0 0 * * *' },
  async ({ step }) => {
    // === step 1：抓商品列表（失敗自動重試）===
    const products = await step.run('fetch-products', async () => {
      return await scrapeProducts();
    });
    
    // === step 2：併發抓細節（fan-out、Inngest 自動分散執行）===
    await step.run('fetch-details', async () => {
      await Promise.all(
        products.map(p => fetchDetail(p.id))
      );
    });
    
    // === step 3：寫 DB ===
    await step.run('save-to-db', async () => {
      await db.products.bulkInsert(products);
    });
    
    // === step 4：通知 ===
    await step.run('notify', async () => {
      await sendLineMessage(`今天爬了 ${products.length} 個商品`);
    });
  }
);
```

  
**Inngest 為什麼強：**

```text
✅ 失敗自動 retry（不用自己寫）
✅ 每 step 獨立狀態（中途失敗從失敗點繼續）
✅ Web UI 看執行紀錄（每 step 多久、什麼錯）
✅ 並行控制（fan-out / fan-in）
✅ 跨 step 傳資料（自動 serialize）
✅ Dead letter queue（失敗 N 次推到別處）

→ 「serverless 版 Airflow」
```

  
### 排程選擇指南

| 場景 | 推薦 |
|---|---|
| **個人 / 學習** | Linux cron（最簡） |
| **小型 SaaS / Indie** | GitHub Actions（免費 + 版本控制） |
| **Next.js / Vercel 專案** | Vercel Cron（整合好） |
| **複雜 workflow / 商業** | Inngest（最強） |
| **企業 production** | Temporal（自架）/ Inngest（託管） |
| **AWS 環境** | EventBridge + Lambda |
| **Kubernetes** | CronJob resource |
| **Cloudflare 環境** | Workers Cron（免費） |

  
### Cron 表達式速查（GitHub / Vercel / 多數平台）

```text
0 0 * * *       每天 UTC 00:00
0 8 * * *       每天 UTC 08:00（台灣 16:00）
0 22 * * *      每天 UTC 22:00（台灣 06:00、適合早爬）
0 * * * *       每小時整點
*/15 * * * *    每 15 分鐘
0 0 * * 0       每週日 00:00
0 0 1 * *       每月 1 號 00:00
0 0 1 1 *       每年 1/1 00:00（過年）
```
