### 傳統 vs AI 爬蟲

```text
=== 傳統 ===
你寫：「.product .name」
網站改版「class 改成 .item-title」
→ 全部壞、要修
→ 每月維護成本高

=== AI ===
你寫：「找商品的名字和價格」
網站改版（HTML 結構改）
→ AI 仍能看到「商品」「名字」「價格」
→ 維護成本 ≈ 0
```

  
### 1. browser-use（Python 生態最熱）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
uv add browser-use langchain-anthropic
```

**這行做什麼？** 裝 browser-use（AI 控瀏覽器）+ LangChain Anthropic（接 Claude）。

  
📄 **寫進程式檔**：`ai_scrape.py`

```text
import asyncio
from browser_use import Agent
from langchain_anthropic import ChatAnthropic

async def main():
    # === 1. 建 Agent + LLM ===
    agent = Agent(
        task="去蝦皮搜尋 iPhone 17 Pro、找前 10 個最低價、回 JSON 格式 [{name, price, url}]",
        llm=ChatAnthropic(
            model="claude-opus-4-7",
            anthropic_api_key="你的_anthropic_key"
        )
    )
    
    # === 2. 跑（AI 開瀏覽器自己操作）===
    result = await agent.run()
    print(result)

asyncio.run(main())
```

  
**browser-use 內部做什麼？**

```text
1. 自動開 Playwright Chromium
2. 載入網頁、截圖
3. 把截圖 + DOM 給 Claude
4. Claude 決定：「點搜尋框」「輸入 iPhone 17」「按搜尋」
5. 重複到任務完成
6. 回傳結果

→ 完全像人在用、不用寫 selector
```

  
### 2. Claude Computer Use（Anthropic 官方）

**特色**：不只控瀏覽器、控**整個電腦**（滑鼠 / 鍵盤 / 任何 app）。

```text
import anthropic

client = anthropic.Anthropic()

response = client.messages.create(
    model="claude-opus-4-7",
    max_tokens=4096,
    tools=[
        {"type": "computer_20250124", "name": "computer"},
        {"type": "bash_20250124", "name": "bash"},
    ],
    messages=[{
        "role": "user",
        "content": "打開蝦皮、搜尋 iPhone 17 Pro、抓前 10 個最低價、存 CSV"
    }]
)

# Claude 會自己決定：
# → 開 Chrome
# → 輸入 URL
# → 點搜尋框
# → 輸入文字
# → 截圖判斷結果
# → 抓資料寫 CSV
```

### 4. 半 AI 半傳統（最 ROI 平衡）

**情境**：傳統 fetch 拿 HTML 便宜、LLM 解析強。

```text
import anthropic
import requests
from bs4 import BeautifulSoup
import json

# === 傳統 fetch ===
res = requests.get('https://example.com/products')
html = res.text

# === 給 LLM 解析 ===
client = anthropic.Anthropic()

prompt = f"""從這個網頁抽出所有商品的 name, price, link、回 JSON 陣列：

{html[:5000]}"""

response = client.messages.create(
    model="claude-opus-4-7",
    max_tokens=2048,
    messages=[{"role": "user", "content": prompt}]
)

data = json.loads(response.content[0].text)
```

**為什麼最 ROI：**

```text
✅ fetch 免費、快
✅ LLM 只解析、便宜（一頁 NT$ 0.3-3）
✅ 不用寫 selector、不用維護
✅ 改版 LLM 仍能解析

→ 比傳統便宜（不用寫 selector）
→ 比全 AI 便宜（不用 LLM 操作瀏覽器）
```

  
### 結構化抽取進階：用 Pydantic 確保格式

```text
import anthropic
import json
from pydantic import BaseModel
from typing import List

class Product(BaseModel):
    name: str
    price: float
    link: str
    in_stock: bool

class ProductList(BaseModel):
    products: List[Product]

client = anthropic.Anthropic()

prompt = f"""從這個網頁抽出商品、回傳符合此 schema 的 JSON：

Schema:
{ProductList.model_json_schema()}

網頁：
{html}"""

response = client.messages.create(
    model="claude-opus-4-7",
    max_tokens=2048,
    messages=[{"role": "user", "content": prompt}]
)

# === 用 Pydantic 驗證 ===
data = ProductList.model_validate_json(response.content[0].text)

for p in data.products:
    print(f"{p.name}: NT$ {p.price}")
```

> 💡 **Pydantic schema 自動轉 JSON Schema**——LLM 看了知道要回什麼格式、輸出格式準確度提升。

### 何時用 AI 爬蟲

```text
✅ 適合
- 改版頻繁的網站（傳統一直壞）
- 多種網站同方法（不想寫 N 個 scraper）
- 不規則資料（每頁結構不一樣）
- PoC / MVP 階段（快速驗證）
- 結構化抽取（用 LLM 解析）

❌ 不適合
- 大量同一網站（傳統便宜 100x）
- 已有官方 API
- 即時要求（每頁 LLM 思考 3-10 秒）
- 預算敏感
- 反爬嚴重（AI 仍會被擋）
```

  
### 限制

```text
1. 慢
   每頁 LLM 思考 3-10 秒（傳統 0.5-2 秒）

2. 貴
   $0.05-0.20 / 頁（傳統 $0）

3. 不確定性
   LLM 偶爾理解錯、輸出格式飄

4. 仍會被反爬擋
   AI 看畫面操作仍會被偵測

5. 不適合即時
   user-facing 即時查詢別用
```

  
### 完整實戰：多網站同一個方法

```text
import asyncio
from browser_use import Agent
from langchain_anthropic import ChatAnthropic

WEBSITES = [
    {"name": "shopee", "url": "https://shopee.tw"},
    {"name": "momo", "url": "https://momoshop.com.tw"},
    {"name": "pchome", "url": "https://24h.pchome.com.tw"},
]

llm = ChatAnthropic(model="claude-opus-4-7")

async def scrape_site(site, keyword):
    """同一個方法、適用任何網站"""
    agent = Agent(
        task=f"去 {site['name']} 搜尋 '{keyword}'、找前 10 個最低價、回 JSON",
        url=site['url'],
        llm=llm
    )
    return await agent.run()

async def main():
    keyword = "iPhone 17 Pro"
    
    # === 三個網站並行爬 ===
    results = await asyncio.gather(*[
        scrape_site(site, keyword)
        for site in WEBSITES
    ])
    
    # === 整合所有結果 ===
    all_products = []
    for site, products in zip(WEBSITES, results):
        for p in products:
            p['source'] = site['name']
        all_products.extend(products)
    
    # === 排序按價格 ===
    all_products.sort(key=lambda x: x['price'])
    
    print(f"找到 {len(all_products)} 個商品、最低 NT$ {all_products[0]['price']}")

asyncio.run(main())
```

> 💡 **這就是 AI 爬蟲的價值**——3 個網站、1 個 task 描述、不用寫 3 個 scraper。
