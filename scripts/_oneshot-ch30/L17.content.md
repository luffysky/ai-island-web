**分散式爬蟲（跨多台機器）**——L5 教過架構、這課是**完整工具大全**：Celery / scrapy-redis / BullMQ / K8s / Lambda + SQS。

> 💡 **規模建議速查**：< 1 萬/天用單台、1-10 萬用 Celery、10-100 萬用 Celery + 雲、100 萬-千萬用 scrapy-redis、> 千萬用 Kafka + Spark。

  
### 為什麼分散式

```text
=== 單台爬蟲瓶頸 ===

單台 1 萬 req / 小時 ≈ 24 萬 req / 天

→ 100 萬筆需求？要 4 天
→ 1000 萬筆？要 40 天
→ 太慢、客戶受不了

=== 分散式 ===

10 台機器 = 10 倍速
100 台機器 = 100 倍速

→ 100 萬筆 10 小時搞定
→ 1000 萬筆 1 天搞定
```

  
### 標準分散式架構

```text
[Master / Scheduler]
       ↓ 分派 task
[Queue（Redis / RabbitMQ / Kafka）]
       ↓
[Worker 1] [Worker 2] [Worker N]  ← 多台機器
       ↓
[Storage（PG / MongoDB / S3）]
```

  
### Celery（Python 業界標準）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
uv add celery redis
```

**這行做什麼？** 裝 Celery（Python 最常用 distributed task queue）+ redis client（當 broker）。

  
📄 **寫進程式檔**：`tasks.py`

```text
from celery import Celery

# === 建 app、broker 用 Redis ===
app = Celery("scraper", broker="redis://localhost:6379")

@app.task
def scrape_url(url):
    """這個 function 會被多個 worker 共同處理"""
    from scrape import scrape
    return scrape(url)
```

  
📄 **寫進程式檔**：`producer.py`（派 task）

```text
from tasks import scrape_url

urls = [f"https://example.com/page/{i}" for i in range(10000)]

for url in urls:
    # === .delay() 把 task 丟進 queue ===
    scrape_url.delay(url)

print(f"丟了 {len(urls)} 個任務進 queue")
```

  
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在多台機器跑**（每台都這樣跑）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
celery -A tasks worker -l info --concurrency=10
```

**這行做什麼？**
- `-A tasks` —— 用 `tasks.py` 內的 app
- `worker` —— 啟動 worker
- `-l info` —— log level info
- `--concurrency=10` —— 這個 worker process 內並行 10 個 task

> 💡 **這就是分散式**——5 台機器 × 10 並發 = 同時跑 50 個 task。

  
### Scrapy + scrapy-redis（最強 Python 分散式爬蟲）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
uv add scrapy scrapy-redis
```

**這行做什麼？** 裝 Scrapy + scrapy-redis 套件、把 Scrapy 內建 scheduler 換成 Redis-based。

  
📄 **寫進程式檔**：`scrapy_project/settings.py`

```text
# === 用 Redis 當 scheduler（不再單機）===
SCHEDULER = "scrapy_redis.scheduler.Scheduler"

# === 自動去重（多台共用 seen set）===
DUPEFILTER_CLASS = "scrapy_redis.dupefilter.RFPDupeFilter"

# === 不要每次清空 queue（多台共用、不清）===
SCHEDULER_PERSIST = True

# === Redis 設定 ===
REDIS_URL = "redis://redis-server:6379"
```

  
📄 **寫進程式檔**：`spiders/my_spider.py`

```text
from scrapy_redis.spiders import RedisSpider

class MySpider(RedisSpider):
    name = "my_spider"
    redis_key = "my_spider:start_urls"            # 從 Redis 拿起始 URL
    
    def parse(self, response):
        yield {
            'url': response.url,
            'title': response.css('h1::text').get(),
        }
```

**起始 URL 進 Redis：**

```text
redis-cli LPUSH my_spider:start_urls "https://example.com/page/1"
redis-cli LPUSH my_spider:start_urls "https://example.com/page/2"
```

**多台跑：**

```text
# 機器 A：scrapy crawl my_spider
# 機器 B：scrapy crawl my_spider
# 機器 C：scrapy crawl my_spider

→ 三台自動分配 task、Redis 去重、不重爬同 URL
```

  
### BullMQ（Node 版分散式）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
npm install bullmq
```

**這行做什麼？** 裝 BullMQ、Node 生態最強 Redis-based task queue（前身 Bull）。

  
📄 **寫進程式檔**：`producer.ts`

```text
import { Queue } from 'bullmq';

const queue = new Queue('scrape', {
  connection: { host: 'localhost', port: 6379 }
});

// === 派 task ===
await queue.add('scrape-url', {
  url: 'https://example.com'
});
```

  
📄 **寫進程式檔**：`worker.ts`（多台跑）

```text
import { Worker } from 'bullmq';

const worker = new Worker('scrape', async (job) => {
  const { url } = job.data;
  const res = await fetch(url);
  const html = await res.text();
  
  // 寫 DB
  await db.results.insert({ url, html });
  
  return { success: true };
}, {
  connection: { host: 'localhost', port: 6379 },
  concurrency: 10,
});
```

  
### Kubernetes Jobs（雲端原生）

📄 **寫進程式檔**：`k8s/scrape-job.yaml`

```text
apiVersion: batch/v1
kind: Job
metadata:
  name: scrape-batch
spec:
  parallelism: 10                                 # 10 個 pod 並行
  completions: 10                                 # 跑完 10 個就結束
  template:
    spec:
      restartPolicy: OnFailure
      containers:
      - name: scraper
        image: my-scraper:latest
        env:
        - name: REDIS_URL
          value: "redis://redis-service:6379"
        resources:
          limits:
            memory: "512Mi"
            cpu: "500m"
```

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
kubectl apply -f k8s/scrape-job.yaml
kubectl get pods -w
```

**這 2 行做什麼？**
- `kubectl apply` —— 部署 Job

### AWS Lambda + SQS（無 server、極端 scale）

**架構：**

```text
[Cron / EventBridge]
        ↓
[SQS（task queue）]
        ↓
[Lambda × auto-scale 到 1000]
        ↓
[DynamoDB / S3]
```

**為什麼用 Lambda + SQS：**

```text
✅ 無 server、不用管機器
✅ 自動 scale 到 1000 並發
✅ 按 invocation 計費（便宜）
✅ AWS 內整合好

❌ 限制
- Lambda 15 分鐘上限
- 冷啟動 100-500ms
- 不能跑超大 Playwright
```

  
### 4 大分散式 queue 對比

| Queue | 適合 | 學習曲線 | 規模 |
|---|---|---|---|
| **Redis** | < 100 萬 / 天 | ⭐ 簡單 | 中 |
| **RabbitMQ** | 100 萬-1000 萬 | ⭐⭐⭐ 中 | 大 |
| **Kafka** | > 1000 萬 | ⭐⭐⭐⭐ 難 | 巨大 |
| **AWS SQS** | 雲端原生 | ⭐⭐ 簡單 | 巨大（managed） |

  
### 規模建議

| 規模 | 推薦 |
|---|---|
| < 1 萬 req/天 | ⭐ 單台（別過度工程） |
| 1 萬-10 萬 | Celery 多台 |
| 10 萬-100 萬 | Celery + 雲端 worker（K8s / Lambda） |
| 100 萬-千萬 | Scrapy + scrapy-redis 或 BullMQ |
| > 千萬 | Kafka + Spark / Apache Beam |

  
### 6 大分散式心法

```text
1. ⭐ 多 worker > 單台多執行緒
   → GIL / event loop 限制
   → 多 process / 多機器避開

2. ⭐ Redis / Queue 協調
   → 共用 task queue
   → 避免重複爬

3. ⭐ 去重要
   → 不同 worker 別爬同 URL
   → Bloom filter / Redis SET

4. ⭐ Idempotent
   → 同 task 跑 2 次結果一樣
   → 用 UPSERT 不用 INSERT

5. ⭐ 監控每台 worker
   → 哪台死了立刻換
   → Prometheus + Grafana

6. ⭐ Dead letter queue
   → 失敗超過 N 次推走
   → 不要無限 retry 卡 queue
```

  
### Indie 不需要分散式（重要！）

```text
99% 個人 / Indie 一台機器就夠

❌ 別「為了學技術」上分散式
❌ 別 1000 個 URL 用 K8s

✅ 規模真的撞到瓶頸再分散式
✅ 先優化單機（async / streaming）
✅ 再分散式（多 worker）
```
