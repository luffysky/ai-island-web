**分散式爬蟲 = 多台機器同時爬同一個任務**——用 Redis 當「**共用 task queue**」、多個 worker 競爭拿任務、橫向擴展不受單機限制。

> 💡 **何時需要分散式？** (1) 單機跑不完（時間 / 量太大） (2) 要 24/7 高可用（一台掛還能跑） (3) 不同地區 IP（多機分散）。**規模 100 萬+/天的標配**。

  
### 分散式爬蟲架構

```text
┌─────────────┐
│  Scheduler  │ ← 產生 URL 任務丟進 Redis
└──────┬──────┘
       ↓
┌─────────────────────┐
│   Redis Queue       │ ← URLs 待爬清單
└──────┬──────────────┘
       ↓ (多 worker 競爭拿任務)
   ┌───┴───┬─────┬─────┐
   ↓       ↓     ↓     ↓
┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐
│Worker│ │Worker│ │Worker│ │Worker│ ← 10 台機器（K8s pod）
└──┬───┘ └──┬───┘ └──┬───┘ └──┬───┘
   ↓       ↓        ↓        ↓
┌──────────────────────────────┐
│      PostgreSQL              │ ← 結果寫入
└──────────────────────────────┘
```

  
### 3 大角色

```text
1. Scheduler（排程器）
   → 產生要爬的 URL、丟進 queue
   → 通常 1 台、cron 觸發

2. Queue（佇列、用 Redis）
   → 暫存 URL 任務
   → 自動去重
   → 支援 retry / 優先級

3. Worker（工人）
   → 從 queue 拿任務、爬、寫 DB
   → 通常多台（10-100）
   → 一台掛、其他繼續跑
```

  
### Python 版（用 RQ / Redis Queue）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
uv add rq redis
```

**這行做什麼？** 裝 RQ（Redis Queue、Python 最常用的分散式 task queue）+ redis client。

  
📄 **寫進程式檔**：`tasks.py`

```text
import requests
from bs4 import BeautifulSoup

def scrape_url(url):
    """單個 URL 爬蟲、會被 worker 呼叫"""
    res = requests.get(url, timeout=30)
    soup = BeautifulSoup(res.text, 'lxml')
    return {
        'url': url,
        'title': soup.select_one('h1').text,
    }
```

  
📄 **寫進程式檔**：`scheduler.py`（產生任務）

```text
from redis import Redis
from rq import Queue
from tasks import scrape_url

q = Queue(connection=Redis())

# === 丟 1000 個 task 進 queue ===
urls = [f'https://example.com/page/{i}' for i in range(1000)]
for url in urls:
    q.enqueue(scrape_url, url)

print(f'已加入 {len(urls)} 個任務到 queue')
```

  
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在多台機器終端機輸入**（每台機器都跑這行）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
rq worker
```

**這行做什麼？** 啟動一個 worker process、開始從 Redis queue 拿任務跑。

> 💡 **這就是分散式**——10 台機器都跑 `rq worker`、10 台同時搶 queue 裡的 task、效率 10x。

  
### Node 版（用 BullMQ）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
npm install bullmq
```

**這行做什麼？** 裝 BullMQ、Node 生態最強的 Redis-based queue（前身 Bull）、支援 priority / delay / cron。

  
📄 **寫進程式檔**：`scheduler.ts`

```text
import { Queue } from 'bullmq';

const queue = new Queue('scraper', {
  connection: { host: 'localhost', port: 6379 }
});

// === 產生 1000 個任務 ===
for (let i = 0; i < 1000; i++) {
  await queue.add('scrape-url', {
    url: `https://example.com/page/${i}`
  });
}
```

  
📄 **寫進程式檔**：`worker.ts`

```text
import { Worker } from 'bullmq';
import * as cheerio from 'cheerio';

const worker = new Worker('scraper', async (job) => {
  const { url } = job.data;
  
  const res = await fetch(url);
  const html = await res.text();
  const $ = cheerio.load(html);
  
  const result = {
    url,
    title: $('h1').text(),
  };
  
  // 寫 DB
  await db.results.insert(result);
  
  return result;
}, {
  connection: { host: 'localhost', port: 6379 },
  concurrency: 10                            // 每個 worker process 並發 10
});

worker.on('completed', (job) => {
  console.log(`Done: ${job.data.url}`);
});
```

  
### 部署到多台（Docker Compose）

📄 **寫進程式檔**：`docker-compose.yml`

```text
version: '3.8'
services:
  # === Redis（task queue）===
  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
  
  # === Scheduler（產生任務）===
  scheduler:
    build: .
    command: node scheduler.js
    depends_on:
      - redis
  
  # === Worker × 10（多 instance 並行爬）===
  worker:
    build: .
    command: node worker.js
    depends_on:
      - redis
    deploy:
      replicas: 10                            # ⭐ 10 個 worker container
```

### Kubernetes（雲端自動 scale）

**情境**：流量大時自動加 worker、流量小時減 worker、省錢。

📄 **寫進程式檔**：`k8s/deployment.yaml`

```text
apiVersion: apps/v1
kind: Deployment
metadata:
  name: scraper-worker
spec:
  replicas: 50                                # 預設 50 個
  template:
    spec:
      containers:
      - name: worker
        image: my-scraper:latest
        env:
        - name: REDIS_URL
          value: "redis://redis-service:6379"
        resources:
          limits:
            memory: "512Mi"
            cpu: "500m"
---
# === HPA（Horizontal Pod Autoscaler）自動 scale ===
apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: scraper-worker-hpa
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: scraper-worker
  minReplicas: 10                             # 最少 10 個
  maxReplicas: 100                            # 最多 100 個
  metrics:
  - type: Resource
    resource:
      name: cpu
      target:
        type: Utilization
        averageUtilization: 70                # CPU > 70% 就加 worker
```

> 💡 **K8s HPA**：CPU 滿了自動加 worker、空閒減 worker、**完全免維護的彈性 scale**。

  
### Bull Board（監控 dashboard）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
npm install @bull-board/express @bull-board/api
```

**這 2 個套件做什麼？**
- `@bull-board/express` —— Express adapter（也支援 Fastify / NestJS）
- `@bull-board/api` —— Bull Board 核心 API

```text
import { ExpressAdapter } from '@bull-board/express';
import { createBullBoard } from '@bull-board/api';
import { BullMQAdapter } from '@bull-board/api/bullMQAdapter';
import express from 'express';
import { Queue } from 'bullmq';

const app = express();
const queue = new Queue('scraper');

const serverAdapter = new ExpressAdapter();
createBullBoard({
  queues: [new BullMQAdapter(queue)],
  serverAdapter,
});

app.use('/admin/queues', serverAdapter.getRouter());

app.listen(3000);
// → 開瀏覽器 localhost:3000/admin/queues 看任務進度
```

**Bull Board 顯示：** 待跑 / 處理中 / 完成 / 失敗 task 數量、每個 task 的 input / output / 失敗原因。

  
### 進階：優先級 + 延遲執行

```text
// === 優先級（低數字 = 高優先級）===
await queue.add('scrape', { url }, { priority: 1 });   // 緊急
await queue.add('scrape', { url }, { priority: 100 }); // 一般

// === 延遲執行（n 毫秒後跑）===
await queue.add('scrape', { url }, { delay: 60_000 }); // 1 分鐘後

// === 重複任務（cron 風格）===
await queue.add('daily-scrape', {}, {
  repeat: { pattern: '0 8 * * *' }           // 每天 08:00
});

// === Rate limit（每秒最多 5 個）===
const worker = new Worker('scraper', handler, {
  connection: redisConfig,
  limiter: {
    max: 5,
    duration: 1000,                          // 每秒 5 個
  }
});
```

  
### 失敗處理 + Dead Letter Queue

```text
const worker = new Worker('scraper', async (job) => {
  const res = await fetch(job.data.url);
  if (!res.ok) throw new Error(`HTTP ${res.status}`);
  return await res.text();
}, {
  connection: redisConfig,
  attempts: 3,                               // 失敗重試 3 次
  backoff: {
    type: 'exponential',                     // 指數退避
    delay: 2000,                             // 第一次 2s、第二次 4s、第三次 8s
  },
});

// === 全失敗的 task 推到 Dead Letter Queue ===
worker.on('failed', async (job, err) => {
  if (job.attemptsMade >= 3) {
    await deadLetterQueue.add('dead', {
      original: job.data,
      error: err.message,
    });
  }
});
```
