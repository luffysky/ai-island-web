**終端機 + GitHub + PowerShell 入門**——L0 講完歡迎、L1 要裝 Python、**但你連終端機是什麼都不知道、怎麼裝？** 這課給你最最基本的「**會用終端機 + 會推 GitHub**」、所有後面課的基礎。

> 💡 **這課的價值**：不會終端機 = 連 Python 都裝不起來。**會 cd / ls / git push 3 個指令、能解 80% onboarding 卡關**。

  
### 終端機（Terminal / 命令列）是什麼

```text
=== 圖形介面 vs 終端機 ===

🖼️ 圖形介面（GUI）
   → 用滑鼠點按鈕
   → 桌面 / 視窗 / 圖示
   → 你平常用的 macOS / Windows / iPhone

⌨️ 終端機（CLI、Command Line Interface）
   → 用打字下指令
   → 黑底白字 / 看起來像駭客
   → 工程師主要工作介面

→ 為什麼用終端機：
   ✅ 快（不用點來點去）
   ✅ 自動化（寫 script 重複跑）
   ✅ 遠端 server 沒桌面只有終端機
   ✅ 業界標準、一定要會
```

  
### 三大作業系統終端機

| OS | 預設終端機 | 推薦 |
|---|---|---|
| **macOS** | Terminal.app | iTerm2（更強） |
| **Windows** | PowerShell（內建） | Windows Terminal（微軟出、推薦） |
| **Linux** | bash / zsh | 預設 |

  
### Mac 開終端機

```text
方法 1：Spotlight 搜尋
  - 按 Cmd + Space
  - 打「terminal」
  - 按 Enter

方法 2：Launchpad
  - 開 Launchpad
  - 找「其他」資料夾 → 終端機

方法 3：Finder
  - Finder → 應用程式 → 工具程式 → 終端機

打開會看到：
luffy@MacBook ~ %
  ↑    ↑     ↑ ↑
 user 機器  路徑 提示符
```

> 💡 **推薦升級 iTerm2**（免費）：分頁、分屏、主題、搜尋、快捷鍵——比預設 Terminal 強 10 倍。

  
### Windows 開 PowerShell

```text
方法 1：開始選單
  - 按 Win 鍵
  - 打「powershell」
  - 按 Enter

方法 2：右鍵桌面
  - 按 Shift + 右鍵
  - 選「在這裡開啟 PowerShell 視窗」

方法 3：Windows Terminal（推薦）
  - 從 Microsoft Store 裝 Windows Terminal
  - 開啟、預設就是 PowerShell

打開會看到：
PS C:\Users\luffy>
↑↑    ↑
PS    當前路徑
```

> 💡 **PowerShell vs Command Prompt（cmd）**：(1) PowerShell 是新版、功能多 (2) cmd 是 Windows 95 時代老的 (3) **永遠用 PowerShell**、別用 cmd。

  
### 學生入門必會 10 個指令

  
**1. pwd（Print Working Directory、印當前路徑）**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
pwd
```

**這行做什麼？** 印你「**現在在哪個資料夾**」。所有指令都基於這個路徑。

  
**2. ls / dir（List、列出資料夾內容）**

```text
# Mac / Linux
ls

# Windows PowerShell（兩個都能用）
ls
dir
```

**這行做什麼？** 列出當前資料夾所有檔案 + 子資料夾。

  
**進階：**

```text
ls -la       # 詳細模式（含隱藏檔）
ls -lh       # 人類可讀大小（KB / MB）
ls *.py      # 只列 .py 結尾的
```

  
**3. cd（Change Directory、切換資料夾）**

```text
cd Documents          # 進入 Documents 資料夾
cd ..                 # 回上一層
cd ~                  # 回家目錄（你的使用者資料夾）
cd /                  # 回根目錄
cd -                  # 回上次的資料夾
cd "My Folder"        # 含空格用雙引號
```

> 💡 **最容易卡的**：用 `cd` 切到某資料夾後、`ls` 確認自己在哪。**永遠 pwd / ls 確認位置**。

  
**4. mkdir（Make Directory、建資料夾）**

```text
mkdir my-project              # 建單一資料夾
mkdir -p src/components/ui    # 連續建多層（-p = parents）
```

  
**5. touch / New-Item（建空檔）**

```text
# Mac / Linux
touch index.html

# Windows PowerShell
New-Item index.html

# 或更簡單（PowerShell）
ni index.html
```

  
**6. cat / Get-Content（看檔內容）**

```text
# Mac / Linux
cat package.json

# Windows PowerShell
cat package.json          # 也能用
Get-Content package.json  # 原生 PowerShell 寫法
```

  
**7. mv / rm（移動 / 刪除）**

```text
# === 移動 / 重新命名 ===
mv old.txt new.txt           # 改名
mv file.txt Documents/       # 搬到 Documents

# === 刪除 ===
rm file.txt                  # 刪檔案
rm -rf folder                # ⚠️ 刪整個資料夾（含內容、不會問）
                              # -r recursive、-f force
```

> ⚠️ **`rm -rf` 是危險指令**——刪了就沒了、沒有垃圾桶。**永遠 pwd 確認位置才刪**。

  
**8. clear（清空螢幕）**

```text
clear       # Mac / Linux / PowerShell 都行
cls         # Windows 老指令、PowerShell 也接受
```

或按 **Ctrl + L** 快捷鍵。

  
**9. 上下鍵（翻歷史指令）**

```text
按 ↑ 鍵 → 上次打的指令
按 ↓ 鍵 → 下次
```

**省 80% 打字時間**——終端機最重要快捷鍵。

  
**10. Tab（自動補完）**

```text
打 cd Doc
按 Tab → 自動補完成「cd Documents/」

打 cd D
按 Tab Tab → 列出所有 D 開頭的選項
```

**會用 Tab = 速度 3x**。

  
### Mac / Linux vs PowerShell 指令對照

| 用途 | Mac/Linux | PowerShell（都支援）| PowerShell 原生 |
|---|---|---|---|
| 列檔 | `ls` | `ls` | `Get-ChildItem` |
| 看檔 | `cat` | `cat` | `Get-Content` |
| 切資料夾 | `cd` | `cd` | `Set-Location` |
| 印路徑 | `pwd` | `pwd` | `Get-Location` |
| 複製 | `cp` | `cp` | `Copy-Item` |
| 移動 | `mv` | `mv` | `Move-Item` |
| 刪除 | `rm` | `rm` | `Remove-Item` |
| 建檔 | `touch` | `New-Item` | `New-Item` |

> 💡 **2026 推薦**：用 alias（`ls` / `cd` / `pwd` 那組）—跨平台、肌肉記憶通用。

  
### Git 基礎（必會 5 招）

**Git = 版本控制系統**、追蹤 code 變更歷史。**GitHub = 把 Git repo 放雲端的服務**。

  
**第一次設定（只要設一次）：**

```text
git config --global user.name "你的名字"
git config --global user.email "your@email.com"
```

**這 2 行做什麼？** 告訴 Git 你是誰、之後 commit 才會顯示你的名字。

  
**Git 核心 5 招：**

```text
1. git init          # 在當前資料夾建立 git repo
2. git status        # 看哪些檔案改了
3. git add .         # 把所有改動加入「待 commit」區
4. git commit -m "說明"   # 確認改動、寫一句說明
5. git push           # 推到 GitHub（要先設 remote）
```

  
**完整第一次流程：**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**（建專案 + 推 GitHub）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
# === 1. 建專案資料夾 ===
mkdir my-first-project
cd my-first-project

# === 2. 建一個檔案 ===
echo "# My First Project" > README.md

# === 3. 初始化 git ===
git init

# === 4. 加 + commit ===
git add .
git commit -m "first commit"

# === 5. 連 GitHub 遠端 ===
git remote add origin https://github.com/你的帳號/my-first-project.git

# === 6. 推上去 ===
git branch -M main
git push -u origin main
```

**這 8 行做什麼？**
- `mkdir + cd` —— 建專案 + 進入
- `echo > README.md` —— 寫一個 README（GitHub 慣例）
- `git init` —— 開始 git 追蹤
- `git add + commit` —— 第一次 commit
- `git remote add` —— 告訴 git「我的 GitHub repo 在這裡」
- `git branch -M main` —— 主 branch 改名 main（GitHub 預設）
- `git push -u origin main` —— 推、`-u` 設 upstream（之後 git push 不用再加參數）

  
### GitHub 註冊 + 建 repo（5 分鐘）

```text
1. 去 https://github.com 註冊（用 email + 密碼）

2. 驗證 email

3. 建 repo：
   - 右上角 + → New repository
   - 名稱：my-first-project
   - Public（公開）或 Private（私人、限免費 3 個）
   - ☐ Add README（不要勾、我們本機有了）
   - Create repository

4. 看到 URL：
   https://github.com/你的帳號/my-first-project.git
   → 貼到 git remote add 後面
```

  
### GitHub 重要概念

```text
📦 Repository（repo、倉庫）
   → 一個專案 = 一個 repo

🌿 Branch（分支）
   → 平行的開發線
   → main = 主 branch、其他是 feature branch

📤 Commit
   → 一個改動的 snapshot
   → 寫清楚做什麼的

🔄 Pull Request（PR）
   → 「我這個 branch 改好了、可以合進 main 嗎？」
   → review + merge

🍴 Fork
   → 把別人的 repo 複製一份到你帳號

⭐ Star
   → 把 repo 加進「我喜歡」
   → GitHub 履歷的「likes」
```

  
### Git 進階常用（必會 5 招）

```text
git pull              # 從 GitHub 拉最新版下來
git clone <url>       # 把 GitHub 別人的 repo 複製到本機
git branch            # 看所有 branch
git checkout -b feat  # 建新 branch 並切過去
git log               # 看歷史 commit
git diff              # 看跟上次 commit 差異
```

  
**完整工作流（每天用）：**

```text
# 每天開始 ===
git pull                          # 拉最新

# 工作中 ===
# ... 改 code ...

# 結束時 ===
git status                        # 看改了什麼
git add .                         # 加進待 commit
git commit -m "fix: 修登入 bug"  # commit
git push                          # 推上去
```

  
### Commit Message 慣例（業界標準）

```text
=== Conventional Commits ===

feat: 新功能
   → feat: 加 dark mode

fix: 修 bug
   → fix: 登入按鈕失靈

docs: 文件
   → docs: 更新 README

style: 排版 / 樣式
   → style: 縮排修正

refactor: 重構（功能不變）
   → refactor: 簡化 login function

test: 測試
   → test: 加 login unit test

chore: 雜事（build / config）
   → chore: 升級 React 19

→ 寫得好 = 履歷 + 團隊愛你
```

  
### PowerShell 特有功能（Windows 用戶必看）

  
**1. PowerShell 用 `$env:VAR` 不是 `$VAR`：**

```text
# Linux / Mac
echo $PATH

# PowerShell
$env:PATH         # 印 PATH
$env:NAME = "luffy"   # 設環境變數
```

  
**2. 執行 `.ps1` script 要先解鎖：**

```text
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

**這行做什麼？** 允許跑本機 PowerShell script（預設禁止、避免病毒）。

  
**3. PowerShell 物件導向（vs Linux 純文字）：**

```text
# 列檔案 + 過濾 > 1MB
Get-ChildItem | Where-Object { $_.Length -gt 1MB }

# 跟 Linux 對應：
# ls -la | awk '$5 > 1024*1024'
```

  
### 5 大新手卡關 + 解法

```text
1. ❌ 「找不到指令 command not found」
   → 套件沒裝、或 PATH 沒設
   ✅ 重開終端機、確認套件裝對
   ✅ which python（找指令在哪）

2. ❌ 「Permission denied」
   → 沒權限
   ✅ Mac / Linux：前面加 sudo
   ✅ Windows：右鍵 → 以系統管理員執行

3. ❌ cd 到不存在的資料夾
   → 拼錯 / 路徑錯
   ✅ pwd 確認位置、ls 看有什麼

4. ❌ 路徑含空格 / 中文
   → 容易出錯
   ✅ 用雙引號 cd "我的 專案"
   ✅ 推薦資料夾用純英文 + 連字號

5. ❌ Git push 要密碼
   → 2021 後 GitHub 廢密碼、要用 Personal Access Token（PAT）
   ✅ 去 GitHub Settings → Developer settings → PAT
   ✅ 或裝 GitHub CLI：gh auth login
```
