**為什麼學資料分析要選 Python？** 不選 R / SAS / SPSS / Excel？這課給你 2026 的答案。

> 💡 **如果你還沒讀過 ch26 Python 基礎**——建議先去看。這章假設你會 Python 基本語法、直接進「資料分析角度」的工具用法。

  
### Python vs R / SAS / SPSS / Excel（5 個對手對比）

| 對手 | 強項 | 弱項 | 適合 |
|---|---|---|---|
| **R** | 統計學深、ggplot2 畫圖一流 | 工程整合差、不易部署 | 學術研究、純統計 |
| **SAS** | 企業級穩定、金融業常用 | 一年 USD 10 萬授權、超貴 | 金融 / 製藥 |
| **SPSS** | UI 友善、不用寫 code | 拓展性差、生態小 | 學術問卷統計 |
| **Excel** | 人人會、看得到資料 | 10 萬列就慢、不能自動化 | 小型臨時分析 |
| **Python** ⭐ | **全方位**（分析 + ML + Web + 自動化） | 學習曲線中等 | **2026 主流選擇** |

  
### Python 在資料分析的 4 大優勢

```text
=== 1. 生態最豐富 ===
- NumPy / Pandas / Scikit-learn / TensorFlow / PyTorch 全在 Python
- 100 萬+ 套件、PyPI 倉庫最大
- AI / ML 框架 90% Python first

=== 2. 通用語言 ===
- 分析完直接接 API（FastAPI L25 教過）
- 可寫爬蟲抓資料（ch28 詳教）
- 可自動化排程（Celery L30）
- 一個語言整合所有事、不用切換工具

=== 3. 開源免費 ===
- 跟 SAS（USD 10 萬/年）比起來、Python $0
- 公司省授權費、個人完全免費

=== 4. AI 時代主場 ===
- 2025-2026 LLM / Agent 全部用 Python（OpenAI / Anthropic / Google SDK）
- 學 Python = 同時學會用 AI
```

  
### 核心 4 大套件（這章重點）

**任何資料分析 task 都離不開這 4 個：**

| 套件 | 中文角色 | 一句話 |
|---|---|---|
| **NumPy** | 數值計算底層 | 所有其他套件的「地基」、處理多維陣列 |
| **Pandas** | 表格資料處理 | 90% 時間在用、像 Excel 的程式版 |
| **matplotlib** | 畫圖底層 | 所有 Python 圖的祖師爺、能畫一切 |
| **seaborn** | 統計圖美化 | 基於 matplotlib、預設超美、寫法更簡潔 |

> 💡 **學習順序**：NumPy（L3）→ Pandas（L4、L8）→ matplotlib（L5、L10）→ seaborn → 進階套件。

  
### 2026 進階套件（讓你超越 90% 同行）

```text
=== 新興主流（不只「會」、要會「為什麼選它」）===

1. Polars（L18 詳教）
   - Rust 寫的 Pandas 替代
   - 速度快 Pandas 5-10 倍
   - 大資料時 must have
   - 2026 越來越多公司轉用

2. Plotly（L11 詳教）
   - 互動式圖表（hover / zoom）
   - 適合做 dashboard、嵌入網頁
   - Dash 框架（Plotly 出的）= Python 版 Tableau

3. Streamlit（L12 詳教）
   - 1 小時把分析變網頁 dashboard
   - 純 Python、不用學前端
   - indie 出 SaaS 神器

4. DuckDB（L19 詳教）
   - Python 內跑 SQL、超快
   - 嵌入式 OLAP、單檔分析 GB 級資料
   - SQLite 的「分析版本」

5. PySpark（L17 詳教）
   - 處理 TB 級超大資料
   - 大公司、雲端必備（GCP Dataproc / AWS EMR）

6. Prophet（L15 詳教）
   - Facebook 出的時序預測
   - 1 行預測股價 / 銷量 / 流量
```
