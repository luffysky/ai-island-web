### Pandas 時間索引（L6 回顧 + 速查）

```text
import pandas as pd

# === 解析日期欄 ===
df = pd.read_csv("sales.csv", parse_dates=["date"])
df = df.set_index("date").sort_index()

# === 切片（時間 index 超直覺）===
df["2026"]                                   # 2026 全年
df["2026-01"]                                # 1 月
df["2026-01":"2026-03"]                      # 範圍（Q1）

# === Resample（重採樣聚合）===
daily = df["sales"].resample("D").sum()       # 每日總和
weekly = df["sales"].resample("W").sum()      # 每週
monthly = df["sales"].resample("ME").sum()    # 每月末（month end）

# === Rolling（滑動窗口、均線）===
df["ma7"] = df["sales"].rolling(7).mean()
df["ma30"] = df["sales"].rolling(30).mean()
df["std30"] = df["sales"].rolling(30).std()

# === Diff（差分、看「跟前一天差多少」）===
df["change"] = df["sales"].diff()
df["pct_change"] = df["sales"].pct_change() * 100   # 變化率 %

# === Shift（位移、拿前 N 期值）===
df["yesterday"] = df["sales"].shift(1)
df["last_week"] = df["sales"].shift(7)
```

**這行做什麼？** 裝 `statsmodels`、Python 統計學標準套件（時序 / 回歸 / ANOVA / hypothesis testing 都用）。

  
```text
from statsmodels.tsa.seasonal import seasonal_decompose

result = seasonal_decompose(
    df["sales"],
    model="additive",                        # additive 加法 / multiplicative 乘法
    period=7                                 # 週期長度（7 = 週）
)
result.plot()
```

**4 張圖一次看：**
- **Observed** —— 原始資料
- **Trend** —— 長期趨勢（去掉季節 / 雜訊）
- **Seasonal** —— 週期性（每 7 天 / 12 月重複的 pattern）
- **Residual** —— 剩下的「噪音」（觀察點剩下解釋不了的）

> 💡 **何時用 additive vs multiplicative？** 季節振幅**固定**用 additive、隨資料規模**放大**用 multiplicative（例如電商銷售年成長、年底季節振幅也越來越大）。

  
### Stationarity（平穩性、預測前必檢查）

**「平穩」= 平均 / 方差不隨時間改變**。時序預測模型大多假設資料平穩、不平穩要先 diff。

```text
from statsmodels.tsa.stattools import adfuller

result = adfuller(df["sales"])
p_value = result[1]

if p_value < 0.05:
    print("✓ 平穩、可直接建模")
else:
    print("✗ 不平穩、需要 diff（df.diff()）")
```

> 💡 **不用過度糾結 stationarity**——用 Prophet 就不用管這個、Prophet 內部會處理。

  
### ARIMA（傳統統計模型、新手可跳過）

**ARIMA**（AutoRegressive Integrated Moving Average）= 經典時序模型、學起來累、Prophet 出現後新手很少用。

```text
from statsmodels.tsa.arima.model import ARIMA

model = ARIMA(df["sales"], order=(1, 1, 1))  # p, d, q 三個超參數要調
fitted = model.fit()
forecast = fitted.forecast(steps=30)
```

> 💡 **新手建議直接跳 Prophet**——ARIMA 學完發現效果差不多、不如花時間學 Prophet + 神經網路。

**完整範例：**

```text
from prophet import Prophet
import pandas as pd

# === 1. 整理資料：Prophet 要求 ds（日期）/ y（值）兩欄 ===
data = df.reset_index().rename(columns={"date": "ds", "sales": "y"})
# Prophet 規定欄位名一定是 ds, y、不能改

# === 2. 訓練（設定季節）===
m = Prophet(
    yearly_seasonality=True,                 # 年季節（每年重複的 pattern）
    weekly_seasonality=True,                 # 週季節（週末效應）
    daily_seasonality=False,                 # 日季節（小時級資料才開）
    seasonality_mode="multiplicative"        # 季節振幅隨規模放大（電商常用）
)
m.fit(data)

# === 3. 預測未來 90 天 ===
future = m.make_future_dataframe(periods=90)
forecast = m.predict(future)
# forecast 含 yhat（預測值）/ yhat_lower / yhat_upper（信賴區間）

# === 4. 視覺化 ===
fig = m.plot(forecast)                       # 主圖：實際值 + 預測 + 信賴區間
fig2 = m.plot_components(forecast)           # 拆解：trend / weekly / yearly / 假日
```

```text
holidays = pd.DataFrame({

```

  
### 異常偵測（找出資料的怪事）

**方式 1：用 Prophet 的信賴區間**

```text
forecast["actual"] = data["y"]
forecast["is_anomaly"] = (
    (forecast["actual"] < forecast["yhat_lower"]) |
    (forecast["actual"] > forecast["yhat_upper"])
)
# 實際值跑出信賴區間 = 異常
```

  
**方式 2：傳統 z-score（3-sigma rule）**

```text
mean = df["sales"].mean()
std = df["sales"].std()
df["is_anomaly"] = abs(df["sales"] - mean) > 3 * std
# 偏離平均 3 個標準差 = 異常（99.7% 信賴）
```

### 常見坑（新手 4 大陷阱）

```text
1. ❌ 未來資訊洩漏（data leakage）
   訓練時用了未來的資料（例如：用整年 mean 當特徵、但「整年」包含 test 期間）
   → 表現超好、上線爆掉

2. ❌ 過度擬合
   model 完美對訓練資料、預測未來爛掉
   → 永遠在 holdout test set 驗證

3. ❌ 沒看 baseline
   做了複雜 model、結果簡單 moving average 也差不多
   → 永遠先跑 baseline（用 rolling mean / 上一期值）對比

4. ❌ 忽略外部變數
   只看歷史數字、沒考慮行銷活動 / 新聞事件 / 競品促銷
   → Prophet 支援 extra regressors（外部變數）、記得用
```
