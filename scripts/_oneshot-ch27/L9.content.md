### 為什麼 NumPy 比 Python list 快？實測對比

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📄 **在 Jupyter cell 跑**（用 `%%time` 計時）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
import numpy as np
import time

# === 純 Python list（慢）===
lst = list(range(1_000_000))
start = time.time()
result = [x * 2 for x in lst]
print(f"list: {time.time() - start:.3f}s")    # 約 0.1s

# === NumPy array（快 100 倍）===
arr = np.arange(1_000_000)
start = time.time()
result = arr * 2
print(f"numpy: {time.time() - start:.3f}s")   # 約 0.001s
```

**為什麼快？**

```text
1. C 語言實作（不是 Python）
2. 連續記憶體（CPU cache 命中率高）
3. SIMD 指令（CPU 一次處理多個數字）
4. 無 Python overhead（不用每個元素包成 Python 物件）

→ 結論：能用 NumPy 就用 NumPy、不要用 Python list 處理大量數字
```

### Broadcasting（廣播、神功能、不同形狀也能算）

**核心規則**：「**從右邊對齊、缺的維度補 1、長度為 1 的維度可以擴展**」。

```text
m = np.array([[1, 2, 3], [4, 5, 6]])        # shape (2, 3)
v = np.array([10, 20, 30])                  # shape (3,) = (1, 3)

m + v
# v 自動擴展成 [[10, 20, 30], [10, 20, 30]]
# [[11, 22, 33],
#  [14, 25, 36]]
```

  
**實戰：每筆資料減去該欄平均（centering）：**

```text
data = np.random.rand(100, 5)               # 100 筆、5 個特徵
centered = data - data.mean(axis=0)         # 每欄獨立 center
# data.mean(axis=0) shape = (5,)、自動廣播到 (100, 5)
```

  
### 矩陣運算（線性代數、ML 基礎）

```text
A = np.array([[1, 2], [3, 4]])
B = np.array([[5, 6], [7, 8]])

A @ B                                       # 矩陣乘法
A.T                                         # 轉置
np.linalg.inv(A)                            # 反矩陣
np.linalg.det(A)                            # 行列式
np.linalg.eig(A)                            # 特徵值 + 特徵向量（PCA 底層）

# === 解線性方程組 Ax = b ===
b = np.array([5, 11])
x = np.linalg.solve(A, b)
# 比 np.linalg.inv(A) @ b 快且穩定
```

### 應用：用 NumPy 替代 Python loop

**核心心法**：**看到 for 處理數字、想想能不能用 NumPy 一行解決**。

```text
# === ❌ 慢 Python loop ===
result = []
for x in data:
    if x > 0:
        result.append(x ** 2)

# === ✅ 快 100 倍 NumPy ===
arr = np.array(data)
result = arr[arr > 0] ** 2

# === ❌ 慢 Python loop（雙層）===
result = []
for i in range(len(a)):
    result.append(a[i] * b[i] + c[i])

# === ✅ 快 NumPy ===
result = a * b + c                          # 向量化、一行解決
```

  
    **☕ 用人話講**
    
NumPy 5 大心法背起來：(1) **向量化** —— `arr * 2` 取代 for 迴圈 (2) **Broadcasting** —— 不同 shape 自動對齊（標準化 / centering 用） (3) **Boolean indexing** —— `arr[arr > 0]` 一行篩選 (4) **`axis` 軸方向** —— axis=0 沿列、axis=1 沿行 (5) **`seed(42)` 保證可重現**。**速度對比實測**：100 萬元素、`* 2` 操作、Python list 約 100ms、NumPy 約 1ms。**怎麼練習**：把你寫過的任何 for 迴圈處理數字、試著用 NumPy 改寫、比較速度。
