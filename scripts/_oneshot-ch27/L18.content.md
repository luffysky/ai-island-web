**這行做什麼？** 裝 Polars 主套件。Polars 是 Rust 寫的、純函式庫、不需要額外依賴。

  
```text
import polars as pl                            # 慣例暱稱 pl（跟 plotly 衝突要注意）

df = pl.read_csv("data.csv")
df.head()                                      # 前 5 列
df.shape                                       # (列數, 欄數)
df.schema                                      # 欄位 + 型別
```

  
### 操作（比 Pandas 簡潔、用 expression）

```text
# === Filter（過濾）===
df.filter(pl.col("age") >= 18)
# pandas: df[df["age"] >= 18]

# === Select（選欄）===
df.select(["name", "age", "email"])

# === With column（加新欄、超直觀）===
df.with_columns(
    (pl.col("price") * pl.col("qty")).alias("total")
)

# === GroupBy + Agg（最重要的操作）===
df.group_by("country").agg([
    pl.col("amount").sum().alias("total"),
    pl.col("amount").mean().alias("avg"),
    pl.count().alias("orders")
])

# === Sort ===
df.sort("amount", descending=True)
```

> 💡 **Polars 用 `pl.col("欄名")` expression、Pandas 用 `df["欄名"]`** —— Polars 更明確、IDE 能 autocomplete。

  
### Lazy（query optimization、Polars 最強功能）

跟 Spark 一樣、**lazy 寫法可以讓 Polars「整體優化」**、跑超大資料超快。

```text
result = (
    pl.scan_csv("huge.csv")                   # scan = lazy 不馬上讀
    .filter(pl.col("country") == "TW")        # 不執行、加進 query plan
    .group_by("category")
    .agg(pl.col("amount").sum())
    .sort("amount", descending=True)
    .collect()                                # ← 這時才執行、優化過的 plan
)
```

**Polars 自動優化 3 件事：**
1. **Predicate pushdown** —— filter 推到讀檔階段、少讀資料
2. **Projection pushdown** —— 只讀需要的欄
3. **Multi-core parallelization** —— 自動分多核跑

  
### Pandas ↔ Polars 互通（無痛遷移）

```text
import pandas as pd
import polars as pl

# === pandas → Polars ===
df_pl = pl.from_pandas(df_pd)

# === Polars → pandas ===
df_pd = df_pl.to_pandas()
```

> 💡 **遷移策略**：新功能用 Polars、舊 code 慢慢轉。**完全不衝突、可以混用**。

  
### 實戰範例：各國月銷售 Top 5

```text
df = pl.read_parquet("orders.parquet")

result = (
    df.filter(pl.col("year") == 2026)
    .group_by(["country", "month"])
    .agg([
        pl.col("amount").sum().alias("revenue"),
        pl.col("order_id").count().alias("orders"),
    ])
    .sort("revenue", descending=True)
    .group_by("country")                       # 再按國家分組
    .head(5)                                   # 每國取 top 5
)
```

> 💡 **Polars expression 可以鏈式組合**、寫起來像流程、比 Pandas 直觀。

  
### Time Series（時序、語法跟 Pandas 不同但更乾淨）

```text
df = pl.read_csv("sales.csv")
df = df.with_columns(pl.col("date").str.to_date())   # 字串轉日期

# === Resample（用 group_by_dynamic）===
df.group_by_dynamic(
    "date",
    every="1mo",                              # 每月聚合
    closed="left"                             # 左閉右開
).agg(pl.col("sales").sum())

# === Rolling（滑動視窗）===
df.with_columns(
    pl.col("sales").rolling_mean(7).alias("ma7")
)
```

  
### 何時用 Polars / Pandas / Spark（決策樹）

```text
你的資料量？
├── < 100GB
│   ├── 新專案 → Polars ⭐
│   ├── 已有 pandas codebase → pandas（漸進遷移）
│   └── 需要 SQL → DuckDB（L19）
├── 100GB - 1TB
│   └── DuckDB / Polars lazy（單機極限）
└── > 1TB
    └── Spark（L17、真的需要分散式）
```

  
### 何時還是 Pandas（不要 100% 換 Polars）

```text
=== ✅ Pandas 仍贏的場景 ===
- 生態整合（scikit-learn / statsmodels / Prophet 等）
  → 這些套件 input 預期是 pandas DataFrame
  → Polars 要先 .to_pandas() 給它們
- 教學 / 入門
  → 學習資源 Pandas 多 100 倍
- 老 codebase
  → 不要為改而改、漸進遷移即可
```

  
### Polars vs Pandas 語法對照表

| 操作 | Pandas | Polars |
|---|---|---|
| Filter | `df[df["a"] > 5]` | `df.filter(pl.col("a") > 5)` |
| Select | `df[["a", "b"]]` | `df.select(["a", "b"])` |
| 新欄 | `df["c"] = df["a"] + df["b"]` | `df.with_columns((pl.col("a") + pl.col("b")).alias("c"))` |
| GroupBy | `df.groupby("country").sum()` | `df.group_by("country").agg(pl.col("amount").sum())` |
| Sort | `df.sort_values("a")` | `df.sort("a")` |
| Read CSV | `pd.read_csv(...)` | `pl.read_csv(...)` |
| Lazy | ❌ 沒有 | `pl.scan_csv(...).collect()` |
