**PySpark** = 「**處理 TB 級超大資料**」用——Pandas 撐到 10GB 就死、Spark 跑 100 TB 都行。大公司 / 雲端必備。

> 💡 **新手 95% 不需要 Spark**。Pandas / Polars / DuckDB 涵蓋 99% indie 場景。**這課當「**認識存在 + 知道何時該用**」**。

  
### 為什麼需要 Spark

```text
=== 資料規模 vs 工具選擇 ===
< 1GB        → pandas（一台機器、純 Python、最簡單）
1GB - 100GB  → Polars（單機、Rust 快、L18 詳教）
100GB - 1TB  → DuckDB（單機 OLAP、L19 詳教）
> 1TB        → Spark（分散式、多台機器、業界標準）
```

**Spark 賣點：**
- 跑在「**集群（cluster）**」上、多台機器並行
- 速度比 Hadoop MapReduce 快 100 倍
- DataFrame API 跟 Pandas 像、學習曲線低
- 內建 SQL、MLlib、Streaming

**啟動 SparkSession：**

```text
from pyspark.sql import SparkSession

spark = SparkSession.builder \
    .appName("analysis") \
    .config("spark.executor.memory", "4g") \
    .getOrCreate()
# 拿到 spark 物件、所有操作從這開始
```

  
### DataFrame API（跟 Pandas 像、寫起來無痛）

```text
# === 讀資料（Parquet 比 CSV 快 10 倍、L13 教過）===
df = spark.read.parquet("s3://bucket/data.parquet")
df = spark.read.csv("data.csv", header=True, inferSchema=True)
df = spark.read.json("data.json")

# === 看 schema + 樣本 ===
df.printSchema()                              # 印欄位 + 型別
df.show(5)                                    # 看前 5 列
df.count()                                    # 算總列數
```

  
### 基本操作（跟 Pandas 對照學）

```text
from pyspark.sql.functions import col, sum as _sum, avg, count

# === Filter（過濾）===
df.filter(col("age") >= 18)

# === Select + 計算（衍生欄）===
df.select("name", (col("price") * col("qty")).alias("total"))
# alias 給欄位取別名

# === GroupBy + Agg（最常用）===
df.groupBy("country").agg(
    _sum("amount").alias("total"),
    avg("amount").alias("avg"),
    count("*").alias("orders")
)

# === JOIN ===
result = df1.join(df2, on="user_id", how="inner")
# how: inner / left / right / outer

# === Sort ===
df.orderBy(col("amount").desc())

# === Limit ===
df.limit(100)
```

> 💡 **`from pyspark.sql.functions import sum as _sum`** —— Spark 內 `sum` 跟 Python 內建 `sum` 衝突、慣例改名。

  
### Spark SQL（混 SQL 跟 DataFrame）

**Spark 強項**：可以「**對 DataFrame 直接跑 SQL**」、SQL 跟 Python 隨意切換。

```text
# === 把 DataFrame 註冊成「臨時 view」===
df.createOrReplaceTempView("orders")

# === 用 SQL 查 ===
result = spark.sql("""
    SELECT country, SUM(amount) AS total
    FROM orders
    WHERE year = 2026
    GROUP BY country
    ORDER BY total DESC
""")
result.show()
```

  
### UDF（User-Defined Function、自訂函式）

當 Spark 內建函式不夠用時、寫 Python 自訂：

```text
from pyspark.sql.functions import udf
from pyspark.sql.types import StringType

@udf(returnType=StringType())
def categorize(amount):
    if amount > 1000:
        return "high"
    elif amount > 100:
        return "mid"
    else:
        return "low"

df.withColumn("tier", categorize(col("amount")))
# 新增 tier 欄、跑自訂函式
```

> ⚠️ **UDF 是效能殺手**——Spark 沒法優化 Python code、能用內建就用內建。

  
### Lazy Evaluation（Spark 核心特性）

**Spark 跟 Pandas 最大差別**：Pandas 是「**eager**」（每步馬上算）、Spark 是「**lazy**」（組 query plan、最後一次算）。

```text
df1 = spark.read.parquet("...")               # 沒讀（lazy）
df2 = df1.filter(...)                          # 沒過濾（lazy）
df3 = df2.groupBy(...).agg(...)                # 沒聚合（lazy）
df3.show()                                     # ← 這時才真正執行整條 pipeline
```

**為什麼 lazy 好？** Spark 看到完整 plan、可以「**整體優化**」（如把 filter 推到讀取階段、少讀資料）。

  
### 寫出結果

```text
df.write.parquet("s3://output/result.parquet")

# === 覆蓋模式 ===
df.write.mode("overwrite").parquet("...")

# === 寫 CSV ===
df.write.csv("output.csv", header=True)

# === Partition 寫（按欄位分資料夾、後續查詢只讀需要的）===
df.write.partitionBy("year", "month").parquet("...")
# 結果：output/year=2026/month=1/xxx.parquet
# 之後查 2026 年 1 月、只讀那個資料夾、快 100 倍
```

  
### Pandas API on Spark（pyspark.pandas、無痛遷移）

**最強的功能**：原本 pandas code 直接跑大資料、改 1 行 import 就行。

```text
import pyspark.pandas as ps

df = ps.read_csv("huge.csv")
df["new"] = df["a"] + df["b"]
df.groupby("country").mean()
# 寫法跟 pandas 一模一樣、但跑在 Spark 上
# 適合：團隊已會 pandas、要遷移到大資料時
```

### 何時用 Spark、何時不用

```text
=== ✅ 用 Spark ===
- 資料 > 50GB
- 需要分散式運算
- 結合 MLlib 做大資料 ML
- 已有 Hadoop / Databricks 生態

=== ❌ 不用 Spark ===
- < 10GB（用 pandas / Polars / DuckDB 更快）
- 一次性分析（Spark 啟動就要幾秒）
- 沒有集群（單機跑 Spark 沒意義）
```

  
### Spark 替代品（2026 趨勢）

| 工具 | 特色 | 適合 |
|---|---|---|
| **Polars** ⭐ | Rust 寫、單機快、L18 詳教 | < 100GB |
| **DuckDB** ⭐ | in-process OLAP、L19 詳教 | < 1TB |
| **Dask** | Python 分散式、跟 Pandas 親 | 中大資料 |

> 💡 **2026 趨勢**：**Spark 退潮、Polars / DuckDB 興起**。新專案 < 1TB 直接跳過 Spark。

  
### 效能心法（避免常見坑）

```text
1. ❌ 不要 .collect() 到 driver
   .collect() 把所有資料拉回單台機器、超大資料 OOM
   → 用 .show() 看樣本就好

2. ✅ 善用 .cache() 重複用的 DataFrame
   df.cache() 之後重用不會重算
   記得用完 df.unpersist() 釋放

3. ✅ Partition 適中
   太大 = 並行差、太小 = overhead 高
   調 spark.sql.shuffle.partitions（預設 200）

4. ✅ 看 Spark UI 找瓶頸
   http://localhost:4040、看哪步驟慢
```
