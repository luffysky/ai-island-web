### 中文分詞（jieba、英文不用、中文必做）

**為什麼中文要分詞？** 英文 `"I love python"` 用空白切就好、中文 `"我愛 Python"` 要先「**斷詞**」分成 `["我", "愛", "Python"]`。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**（裝 jieba）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
uv add jieba
```

**這行做什麼？** 裝 jieba（結巴）、最熱門的中文分詞套件、輕量 + 快。

  
```text
import jieba

text = "我愛吃台灣牛肉麵和滷肉飯"
words = list(jieba.cut(text))
print(words)
# ['我', '愛', '吃', '台灣', '牛肉麵', '和', '滷肉飯']

# === 自訂詞典（讓 jieba 認得專有名詞）===
jieba.add_word("AI 島")                       # 之後「AI 島」會被當一個詞
jieba.del_word("不要的詞")

# === 載入外部詞典檔 ===
jieba.load_userdict("custom_dict.txt")
# 格式：每行「詞 詞頻 詞性」

# === 詞性標註（POS tagging）===
import jieba.posseg

words = jieba.posseg.cut(text)
for w in words:
    print(w.word, w.flag)
# 我 r（代名詞）、愛 v（動詞）、吃 v、台灣 ns（地名）...
```

### TF-IDF（重要詞抽取、L21 教過、這裡實戰）

```text
from sklearn.feature_extraction.text import TfidfVectorizer
import jieba

docs = [
    "我愛 Python",
    "Python 是程式語言",
    "Python 適合資料分析"
]

# === 中文要先斷詞、定義 tokenizer ===
def tokenize(text):
    return list(jieba.cut(text))

vec = TfidfVectorizer(tokenizer=tokenize)
tfidf = vec.fit_transform(docs)

print(vec.get_feature_names_out())
# ['Python', '我', '愛', '是', '程式語言', '適合', '資料分析']
```

### 主題建模（LDA、找文章主題）

**LDA**（Latent Dirichlet Allocation）= 從大量文章自動找出「**N 個主題**」+「**每篇文章的主題比例**」。

```text
from sklearn.decomposition import LatentDirichletAllocation

# === 先用 TF-IDF 轉成數字矩陣 ===
# tfidf = ...（前面建好）

lda = LatentDirichletAllocation(
    n_components=5,                          # 找 5 個主題
    random_state=42
)
lda.fit(tfidf)

# === 看每個主題的關鍵詞 ===
for i, topic in enumerate(lda.components_):
    top_words = [
        vec.get_feature_names_out()[j]
        for j in topic.argsort()[-10:]       # 取每個主題前 10 個詞
    ]
    print(f"Topic {i}: {top_words}")
# Topic 0: ['服務', '態度', '客氣', '貼心', ...] ← 服務主題
# Topic 1: ['好吃', '美味', '推薦', '香', ...]   ← 餐點主題
```

  
### Embedding + 相似度（語意搜尋、RAG 基礎）

**Embedding** = 「**把文字轉成向量、語意相似的向量也相近**」、ChatGPT / Claude 的 RAG 都用這個。

```text
from sentence_transformers import SentenceTransformer
from sklearn.metrics.pairwise import cosine_similarity

# === 中文 Embedding 模型 ===
model = SentenceTransformer("uer/sbert-base-chinese-nli")

texts = ["我愛吃牛肉麵", "我喜歡牛肉麵", "Python 很棒"]
embeddings = model.encode(texts)
# 每筆 → 768 維向量

# === 兩兩相似度（cosine similarity 0-1）===
similarities = cosine_similarity(embeddings)
print(similarities)
# [[1.00, 0.92, 0.10],   ← "我愛吃" 跟 "我喜歡" 超像
#  [0.92, 1.00, 0.08],
#  [0.10, 0.08, 1.00]]
```

### 中文 NLP 套件對比（什麼時候用什麼）

| 套件 | 強項 | 適合 |
|---|---|---|
| **jieba** | 分詞、最快、輕量 | 詞頻 / 簡單斷詞 |
| **CKIP** | 中研院出、最準 | 學術研究 |
| **spaCy** | 工程友善、英文強 | 多語言 production |
| **Transformers (HF)** | 最新 SOTA、有預訓練 model | 分類 / 情感 / 摘要 |
| **Anthropic / OpenAI API** | 最強、無痛、彈性大 | 複雜理解 / 開放 task |

  
### 何時用哪個（決策樹）

```text
你的需求？
├── 純分詞 / 詞頻             → jieba
├── 情感分析 / 文字分類         → Transformers（HF）
├── 複雜理解 / 摘要 / 對話      → Claude / GPT API
├── 學術研究 / 高準度斷詞       → CKIP
└── 英文 / 多語言 production    → spaCy
```

NLP 2026 心法：(1) **中文先分詞**（jieba 或 CKIP） (2) **詞頻分析最簡單**（Counter + jieba 一行解決） (3) **情感分析 / 分類**用 Hugging Face 預訓練 model（不用自己訓練） (4) **複雜任務直接 LLM API**（Claude / GPT、prompt 即解） (5) **語意搜尋用 embedding + 相似度**（RAG 基礎）。**新手起手**：jieba 分詞 → wordcloud 視覺化 → HF pipeline 情感分析、3 個小時涵蓋 80% 需求。
