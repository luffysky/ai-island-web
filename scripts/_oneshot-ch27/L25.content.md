### 3 大資料職涯（看你想做哪個）

**1. Data Analyst（資料分析師）：**

```text
工作內容：
- 寫 SQL 撈資料、做報表、做 dashboard
- 跟商業團隊溝通需求、出 insight
- A/B test 分析、產品成效追蹤

技能：
- SQL（必、最重要）
- Excel + Tableau / Power BI（報表工具）
- Python pandas + matplotlib
- 商業思維 + 溝通能力

台灣月薪：NT$ 50-90k（依公司規模）
入門難度：⭐⭐⭐
適合誰：想接觸業務、會講話、邏輯好
```

  
**2. Data Scientist（資料科學家）：**

```text
工作內容：
- 建 ML model、A/B test 設計
- 預測 / 推薦 / 分類問題
- 把分析結果變成產品 feature

技能：
- Python（pandas / sklearn / pytorch）
- 統計學（必、不能只會調 API）
- ML / DL 演算法理解
- 領域知識（金融 / 電商 / 醫療...）

台灣月薪：NT$ 80-150k
入門難度：⭐⭐⭐⭐
適合誰：數學底子好、喜歡建模、研究型
```

  
**3. Data Engineer（資料工程師）：**

```text
工作內容：
- 建 data pipeline（ETL）
- 維護資料倉儲（BigQuery / Snowflake）
- 確保資料品質 + 可用性

技能：
- SQL（深入）
- Python（重工程、寫 production code）
- Spark / Airflow / dbt
- 雲端：AWS / GCP / Azure

台灣月薪：NT$ 90-180k
入門難度：⭐⭐⭐⭐⭐
適合誰：工程背景、喜歡系統設計、不怕運維
```

  
**進階：ML Engineer / MLOps（DS + DE 結合）：**

```text
工作內容：
- 把 ML 模型部署 production
- 監控 model 表現、定期 retrain
- 處理 model serving、A/B 上線

技能：DS + Engineering + DevOps
台灣月薪：NT$ 100-200k（市場供不應求）
```

  
### 完整學習路線（4 階段、12+ 個月）

**0-3 個月：基礎（這章 + ch26 已涵蓋大部分）**

```text
[必做 checklist]
[ ] Python 基本（ch26 已學）
[ ] pandas（這章 L4 / L8）
[ ] SQL（ch17、必修）
[ ] 統計（平均 / 中位數 / 分布 / p-value）
[ ] 視覺化（matplotlib / seaborn / Plotly）
```

  
**3-6 個月：能交付（做出能 show 的東西）**

```text
[ ] 完整資料專案 1 個（從 raw 資料 → 結論報告）
[ ] Streamlit / Tableau dashboard 1 個
[ ] Portfolio project 至少 1 個（放 GitHub）
[ ] Kaggle 比賽試 1-2 個（不用得名、有提交就行）
```

  
**6-12 個月：找工作**

```text
[ ] 寫履歷 + 整理 LinkedIn
[ ] 投 10-30 個職位（量大、別怕被拒）
[ ] 練 SQL leetcode（StrataScratch 是 SQL 版）
[ ] 練面試題（行為 + 技術 + case study）
```

  
**12+ 個月：進階 / 跳槽**

```text
- 選一個主方向（DS / DE / MLE）深入
- 持續做 side project（保持手感）
- 累積 1-2 年跳槽（薪水 +20-30%）
```

  
### Portfolio 準備（找工作必備）

**做 3 個經典 project：**

```text
1. EDA（Exploratory Data Analysis）
   - 開放資料集（政府 data.gov.tw / Kaggle）
   - 完整 Jupyter Notebook + 結論
   - 寫一篇 blog 解釋

2. 預測（時間序列 / 分類 / 回歸）
   - 用 sklearn / Prophet
   - 比較多個 model
   - 評估指標 + 商業 insight

3. 互動 Dashboard
   - Streamlit / Plotly Dash / Tableau
   - 部署到 Streamlit Cloud（免費）
   - URL 直接放履歷
```

**怎麼讓 portfolio 加分：**
- 放 **GitHub**（README 寫清楚）
- 寫 **blog 文章** 解釋你的思路
- 加 **量化成果**（「分析 10 萬筆資料、發現 X 趨勢、建議節省 Y%」）

### 履歷必有元素（1 頁原則）

```text
✓ 1-2 個有完整故事的專案（不是一堆關鍵字）
✓ GitHub 連結（整理過、README 清楚）
✓ 量化成果（「提升 X%」「節省 Y 小時」「處理 Z 萬筆」）
✓ 簡潔（1 頁、不要超過 2 頁）

❌ 不要寫的：
- 「精通 50 個技術」（不可能）
- 「擅長團隊合作」（廢話）
- 興趣寫「閱讀、跑步、看電影」（沒人在乎）
```

### 面試準備重點

```text
=== SQL（90% 公司會考）===
- LeetCode SQL（150 題）
- StrataScratch（更貼近實務）
- 重點：JOIN / Window function / 子查詢

=== Python（基礎演算法）===
- LeetCode Easy 100 題
- 不用刷 Hard、會基本就行

=== Statistics ===
- p-value、信心區間、A/B test
- 中心極限定理、Bayes 定理

=== ML（DS / MLE）===
- 各模型優缺點、何時用哪個
- 評估指標（accuracy / F1 / AUC）
- Overfitting 怎麼防

=== Case Study（高階職位必考）===
- 「如何衡量 X 產品的成功？」
- 「如果 metric 下降 20%、怎麼 debug？」
- 「設計一個推薦系統 / A/B test」
```

### 6 大心法（資深 vs 新手差別）

```text
1. ✅ SQL 是命根
   寫不好 SQL = 別想做資料職位
   每天練、每週寫真實 query

2. ✅ 溝通 > 技術深度
   能解釋給老闆聽 = 升職關鍵
   不會講話的工程師永遠是執行者

3. ✅ 領域知識是護城河
   懂業務的資料人 > 純技術的資料人
   選一個產業深入（金融 / 電商 / 遊戲）

4. ✅ 持續輸出（blog / GitHub）
   1 年 12 篇 blog = 履歷大加分
   長期競爭力

5. ✅ 永遠保持手感
   不要只在公司寫、私下也要做 side project
   保持「面試體質」

6. ✅ 跟上 AI 趨勢
   2026 LLM / Agent 改變所有工作
   不會用 AI 的資料人 = 慢慢被取代
```

資料職涯 3 大方向：(1) **Data Analyst** 適合溝通型、(2) **Data Scientist** 適合建模型、(3) **Data Engineer** 適合工程型。**入行 4 步驟**：基礎（0-3M）→ 能交付（3-6M）→ 找工作（6-12M）→ 進階（12+ M）。**履歷核心**：1-2 個完整 portfolio 專案、量化成果、GitHub 整理。**面試重點**：SQL（90% 公司考）+ ML 概念 + case study。**心法**：SQL 是命根、溝通 > 技術、領域知識是護城河、持續輸出長期競爭力。
