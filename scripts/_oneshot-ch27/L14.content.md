### Magic Commands 完整版

**Jupyter 兩種 magic：`%` 行 magic（單行）+ `%%` cell magic（整個 cell）。**

  
**`%` 行 magic（最常用 6 個）：**

```text
%timeit df.sort_values("amount")             # 跑很多次取平均、精確計時
%pwd                                          # 當前資料夾路徑
%ls                                           # 列檔案
%who                                          # 列目前定義的變數
%matplotlib inline                            # 圖內嵌（新版預設不用寫）
%load_ext autoreload                          # 啟用 autoreload
%autoreload 2                                 # 改 .py 自動重 import（開發必用）
```

  
**`%%` cell magic（整個 cell 作用）：**

```text
# === %%time：計時整個 cell 跑一次 ===
%%time
result = expensive_computation()

# === %%writefile：把 cell 內容寫到檔案（不執行）===
%%writefile config.py
SECRET = "..."

# === %%sql：直接寫 SQL（需 ipython-sql）===
%%sql
SELECT * FROM users LIMIT 10

# === %%bash：跑 bash 指令 ===
%%bash
ls -la
git status
```

### 互動 Widget（讓 notebook 變 mini app）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**（裝 ipywidgets）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
uv add ipywidgets
```

**這行做什麼？** 裝 `ipywidgets`、Jupyter 互動 widget（slider / button / select）的標準套件。

  
**`@interact` decorator（一行做互動 dashboard）：**

```text
from ipywidgets import interact

@interact(
    country=df["country"].unique(),          # 自動變 dropdown
    year=(2020, 2026)                        # tuple 自動變 slider
)
def show(country, year):
    filtered = df[(df["country"] == country) & (df["year"] == year)]
    print(filtered)
# 拉 dropdown / slider 自動 rerun 函式
```

  
**手動 widget（細控）：**

```text
import ipywidgets as widgets

slider = widgets.IntSlider(min=0, max=100)
display(slider)
print(slider.value)
```

### 匯出 Notebook（給人 / 部署用）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**（用 nbconvert 匯出）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
jupyter nbconvert --to html notebook.ipynb
```

**這行做什麼？** 把 `.ipynb` 轉成 `.html`、可以直接寄給客戶 / 老闆看。

  
**其他格式：**

```text
jupyter nbconvert --to pdf notebook.ipynb     # PDF（需先裝 LaTeX）
jupyter nbconvert --to script notebook.ipynb  # 變 .py 純 script
jupyter nbconvert --to slides notebook.ipynb --post serve  # 變 reveal.js 簡報
```

  
### Notebook 5 大好習慣（資深分析師 vs 新手差別）

**1. 標題清楚（第一個 cell 寫 Markdown）：**

```text
# 2026 Q1 銷售分析
- 作者：你的名字
- 日期：2026-05-27
- 目標：找出成長最快的品類

## 1. 載入資料
## 2. 清理
## 3. 分析
## 4. 結論
```

**2. Import 集中在第一個 cell**——之後改套件好找。

**3. 用 markdown cell 解釋**——每個 section 加 1-2 段說明、不要只貼 code、3 個月後自己也忘了在幹嘛。

**4. 最後寫結論 cell**——不要讓人讀 100 個 cell 後問「**所以呢？**」。

**5. 用 nbstripout 清 output（commit 前）：**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**（裝 + 啟用 nbstripout）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
uv add --dev nbstripout
nbstripout --install
```

**這 2 行做什麼？**
- `uv add --dev nbstripout` —— 開發專用套件、不上 production
- `nbstripout --install` —— 設定 git pre-commit hook、commit 前自動清掉 notebook 的 output

> 💡 **為什麼要清 output？** Notebook 輸出（圖片 / 表格）會塞進 .ipynb 檔、檔案變超大 + git diff 一團亂。清掉 output 之後、git diff 只看 code 變動、commit 紀錄乾淨。
