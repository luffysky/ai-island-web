> 💡 **3 件神器**：(1) **In-process** —— 不用裝 DB server (2) **直接 query 檔案** —— `SELECT * FROM 'data.parquet'` 不用先 import (3) **跟 Pandas / Polars 互通**、自然融入 Python 工作流。

**這行做什麼？** 裝 DuckDB Python binding、純函式庫、不需要安裝 DB server（in-process）。

  
```text
import duckdb

# === 1. 直接 SQL（in-memory、跑完不留資料）===
result = duckdb.sql("SELECT 42").fetchall()

# === 2. 連到檔案（持久化、像 SQLite）===
conn = duckdb.connect("mydata.duckdb")
conn.execute("CREATE TABLE users (id INT, name TEXT)")
conn.execute("INSERT INTO users VALUES (1, 'Alice')")
```

  
### 殺手特色：直接 query 檔案（不用先 import）

**這是 DuckDB 最強的地方**——把檔案當 table 用、SQL 一行解決。

```text
# === 直接 query CSV ===
duckdb.sql("SELECT * FROM 'data.csv' WHERE age > 18")

# === 直接 query Parquet（最常用、快）===
duckdb.sql("SELECT * FROM 'data.parquet'")

# === 一次 query 多個檔案（用 glob pattern）===
duckdb.sql("SELECT * FROM 'data/*.parquet'")

# === 直接 query S3（雲端、不用先下載）===
duckdb.sql("SELECT * FROM 's3://bucket/*.parquet'")
```

> 💡 **這 4 行就能做掉 80% ETL 工作**——傳統做法要先寫 Python 讀檔、轉 Pandas、再 query；DuckDB 一行 SQL 解決。

  
### Pandas 整合（query DataFrame 像 query 表）

**DuckDB 可以「**直接對 Pandas DataFrame 跑 SQL**」、不用先寫進 DB：**

```text
import pandas as pd
import duckdb

df = pd.read_csv("data.csv")

# === 直接 query 變數名稱 df（DuckDB 自動偵測）===
result = duckdb.sql("""
    SELECT category, SUM(amount) AS total
    FROM df
    WHERE year = 2026
    GROUP BY category
""").to_df()
```

> 💡 **這超猛**——熟 SQL 的人不用學 pandas 語法、直接寫 SQL 操作 DataFrame、回傳還是 DataFrame。

### 進階：JOIN 多個檔案（不用 import）

```text
result = duckdb.sql("""
    SELECT o.id, u.name, o.amount
    FROM 'orders.parquet' o
    JOIN 'users.csv' u ON o.user_id = u.id
    WHERE o.amount > 1000
""").to_df()
# 一個 Parquet + 一個 CSV、直接 JOIN、寫進 .to_df()
```

  
### Window Functions（Pandas 沒有、DuckDB 標配）

**Window function** = SQL 強大功能、算「**每組內排名 / 累積 / 移動平均**」、Pandas 寫起來超複雜、SQL 一行解決。

```text
duckdb.sql("""
    SELECT
        name,
        amount,
        RANK() OVER (PARTITION BY country ORDER BY amount DESC) as rank,
        SUM(amount) OVER (PARTITION BY country) as country_total
    FROM 'orders.parquet'
""")
# 每筆訂單：自己金額 + 在該國排名 + 該國總金額
```

  
### 寫出結果（COPY 指令、快又省）

```text
duckdb.sql("""
    COPY (
        SELECT * FROM 'data.csv' WHERE active = true
    ) TO 'output.parquet' (FORMAT PARQUET, COMPRESSION ZSTD)
""")
# 一行：query + 寫 Parquet + ZSTD 壓縮
# ZSTD = 2026 最強通用壓縮、比 gzip 快 + 小
```

  
### Python API 3 種寫法

```text
# === 1. 直接 SQL（最簡、in-memory）===
duckdb.sql("...")

# === 2. 用 connection（持久化）===
conn = duckdb.connect("db.duckdb")
conn.execute("...")
df = conn.execute("SELECT * FROM users").df()

# === 3. 從 pandas（流式 API）===
duckdb.from_df(pandas_df).query(...)
```

  
### Web 用：DuckDB-Wasm（瀏覽器內跑 SQL）

**DuckDB 編譯成 WebAssembly**、可以在**瀏覽器內**跑 SQL、不用後端。

```text
// JavaScript / TypeScript
import * as duckdb from '@duckdb/duckdb-wasm';

const db = await duckdb.AsyncDuckDB.create(...);
await db.query("SELECT * FROM 'data.parquet'");
```

**適合場景**：互動式 dashboard、不想自架後端、靜態網站也能有資料 query 能力。

  
### 何時用 DuckDB

```text
=== ✅ 用 DuckDB ===
- 分析查詢（OLAP）
- 處理 Parquet / CSV / JSON
```

### 案例：取代 pandas（同樣需求、DuckDB 快 10 倍）

**Pandas 寫法：**

```text
df = pd.read_parquet("big.parquet")            # 全載入記憶體
filtered = df[df["country"] == "TW"]
result = filtered.groupby("category")["amount"].sum()
```

  
**DuckDB 寫法（快 10 倍 + 不爆記憶體）：**

```text
result = duckdb.sql("""
    SELECT category, SUM(amount) AS total
    FROM 'big.parquet'
    WHERE country = 'TW'
    GROUP BY category
""").to_df()
# DuckDB 只讀需要的欄 + filter 推到讀檔階段
```
