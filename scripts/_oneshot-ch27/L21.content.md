**Feature Engineering（特徵工程）= 「**把原始資料變成 model 能用的特徵**」**——這是資料科學家「**真功夫**」、決定 model 好壞的關鍵。**經驗法則：好的特徵 > 複雜的 model**。

> 💡 **業界鐵則**：「**好的特徵 + 簡單 model > 爛特徵 + XGBoost**」。新手都想學最炫 model、實際上 80% 提升來自特徵工程。

  
### 為什麼需要

```text
原始資料：「Alice / 25 歲 / 台灣 / 2026-05-27 / 1500 元」
↓
Model 看不懂、要轉成：
- 25                                          # age（數字）
- 1, 0, 0                                     # one-hot 國家（台灣=1）
- 0.5                                         # 年齡 z-score
- 5                                           # month
- 1, 0, 0, 0, 0, 0, 0                         # one-hot weekday
- 7.31                                        # log(1500)
- ...

→ 這個轉換過程 = Feature Engineering
```

  
### 1. 類別變數編碼（categorical encoding）

**Label Encoding（有序、用一個整數）：**

```text
from sklearn.preprocessing import LabelEncoder

le = LabelEncoder()
df["education_encoded"] = le.fit_transform(df["education"])
# 大學=0、碩士=1、博士=2（有「順序」關係）

# ⚠️ 不要用在「無序」類別（如國家）！
# country=Taiwan=0、Japan=1、US=2 → model 會以為 US > Japan > Taiwan
```

  
**One-Hot Encoding（無序、每個類別獨立欄）：**

```text
pd.get_dummies(df["country"], prefix="country")
# country_TW、country_JP、country_US 三欄
# 是哪國 = 1、其他 = 0
```

  
**Target Encoding（用 target 平均、進階）：**

```text
df["category_target"] = df.groupby("category")["amount"].transform("mean")
# 每個類別 → 該類別的目標變數平均值
# 例：「電子產品」→ 平均訂單金額 1500
```

> ⚠️ **Target Encoding 易 data leakage**——一定要用 cross-validation 算（避免用到 test 資料）。

  
### 2. 數值變換

**Scaling（範圍統一、L18 詳教）：**

```text
from sklearn.preprocessing import StandardScaler, MinMaxScaler

# === Standard：平均 0、std 1 ===
scaler = StandardScaler()
df_scaled = scaler.fit_transform(df[["age", "income"]])

# === MinMax：壓到 0-1 ===
mm = MinMaxScaler()
df_norm = mm.fit_transform(df[["age", "income"]])
```

  
**Log 轉換（處理偏態分布、金額 / 人數常用）：**

```text
import numpy as np

df["log_income"] = np.log1p(df["income"])     # log(1+x)、handle 0
# 收入分布常「右偏」（少數人超高、多數低）
# Log 後變正常分布、model 學得更好
```

  
**Binning（連續變類別、寫死門檻）：**

```text
df["age_group"] = pd.cut(
    df["age"],
    bins=[0, 18, 35, 60, 120],
    labels=["minor", "young", "mid", "senior"]
)

# === 等頻 binning（每組樣本數一樣多）===
df["income_quintile"] = pd.qcut(df["income"], q=5, labels=False)
# 0-4 五個級距、每級 20%
```

  
### 3. 文字特徵

**長度 / 字數：**

```text
df["text_length"] = df["text"].str.len()
df["word_count"] = df["text"].str.split().str.len()
```

  
**TF-IDF（傳統 NLP、L24 詳教）：**

```text
from sklearn.feature_extraction.text import TfidfVectorizer

vec = TfidfVectorizer(max_features=100)
tfidf = vec.fit_transform(df["text"])
# 每筆文字變成 100 維稀疏向量
```

  
**Embedding（深度學習、2026 主流）：**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**（裝 sentence-transformers）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
uv add sentence-transformers
```

**這行做什麼？** 裝 Hugging Face 的 sentence-transformers、把文字轉成「**語意向量**」、是 RAG / 語意搜尋的核心。

  
```text
from sentence_transformers import SentenceTransformer

model = SentenceTransformer("all-MiniLM-L6-v2")
embeddings = model.encode(df["text"].tolist())
# 每筆文字 → 384 維向量（語意相似的向量也相近）
```

  
### 4. 時間特徵（時序資料必做）

```text
df["hour"] = df["timestamp"].dt.hour
df["day_of_week"] = df["timestamp"].dt.dayofweek
df["is_weekend"] = df["day_of_week"] >= 5
df["is_holiday"] = df["date"].isin(holidays)

# === 週期性特徵（神技巧）===
df["month_sin"] = np.sin(2 * np.pi * df["month"] / 12)
df["month_cos"] = np.cos(2 * np.pi * df["month"] / 12)
# 月份是循環的（12 月接 1 月）、用 sin/cos 表達循環關係
# 比單純 month=1-12 更好、model 學得到「12 月跟 1 月很近」
```

  
### 5. 交互特徵（兩欄組合）

```text
df["price_per_qty"] = df["price"] / df["quantity"]    # 單價
df["bmi"] = df["weight"] / (df["height"] / 100) ** 2  # BMI
```

> 💡 **領域知識 > 自動化**——你最懂你的資料、知道哪兩欄相乘 / 相除有意義。

  
### 6. Lag / Rolling Features（時序預測必用）

```text
df["sales_yesterday"] = df["sales"].shift(1)
df["sales_last_week"] = df["sales"].shift(7)
df["sales_ma7"] = df["sales"].rolling(7).mean()
df["sales_std30"] = df["sales"].rolling(30).std()
```

  
### 7. Group Features（每個 user 的歷史行為）

```text
df["user_avg_amount"] = df.groupby("user_id")["amount"].transform("mean")
df["user_count"] = df.groupby("user_id")["amount"].transform("count")
df["user_max_amount"] = df.groupby("user_id")["amount"].transform("max")
# 每筆訂單 + 該 user 的歷史平均 / 次數 / 最大
```

  
### 8. PCA（降維、處理高維資料）

```text
from sklearn.decomposition import PCA

pca = PCA(n_components=5)                     # 壓到 5 維
df_reduced = pca.fit_transform(df_scaled)
# 100 維 → 5 維、保留 90% 資訊
# 適合：高維資料 + 線性 model（避免 curse of dimensionality）
```

  
### Pipeline（推薦做法、避免 data leakage）

**Pipeline** = 把所有預處理 + model 串成一條、保證 train / test 用同樣轉換。

```text
from sklearn.pipeline import Pipeline
from sklearn.compose import ColumnTransformer
from sklearn.preprocessing import StandardScaler, OneHotEncoder
from sklearn.ensemble import RandomForestClassifier

# === 分欄處理（數值 vs 類別不同 transformer）===
numeric_features = ["age", "income"]
categorical_features = ["country", "education"]

preprocessor = ColumnTransformer([
    ("num", StandardScaler(), numeric_features),
    ("cat", OneHotEncoder(handle_unknown="ignore"), categorical_features),
])

# === 串成 pipeline ===
pipeline = Pipeline([
    ("preprocessor", preprocessor),
    ("model", RandomForestClassifier()),
])

# === 一行 fit ===
pipeline.fit(X_train, y_train)
# train 跟 test 自動用同樣的 scaler / encoder、不會 data leakage
```

  
### 特徵選擇（不是越多越好、避免過擬合）

**1. 用相關性（最快）：**

```text
df.corr()["target"].abs().sort_values(ascending=False)
# 跟 target 最相關的特徵排序
```

  
**2. 用 Random Forest 重要度：**

```text
from sklearn.ensemble import RandomForestRegressor

rf = RandomForestRegressor().fit(X, y)
importance = pd.Series(rf.feature_importances_, index=X.columns)
importance.sort_values(ascending=False).plot.bar()
# 視覺化看哪些特徵最重要
```

  
**3. Recursive Feature Elimination（RFE、自動選）：**

```text
from sklearn.feature_selection import RFE

rfe = RFE(estimator=model, n_features_to_select=10)
rfe.fit(X, y)
selected = X.columns[rfe.support_]
# 自動選出最重要的 10 個特徵
```

  
### Feature Engineering 5 大心法

```text
1. ✅ 領域知識 > 自動化
   → 你最懂業務、知道什麼特徵有意義
   → 自動化工具（autofeat / featuretools）是輔助

2. ❌ 不要為加特徵而加
   → 太多無關特徵 = 過擬合 + 速度慢

3. ✅ Train / Test 用同樣 preprocessor
   → 用 Pipeline 自動處理

4. ❌ 避免 data leakage
   → Target encoding 用 CV、不要用 test 算 mean
   → 時序預測：不要用未來資訊（如「整年 mean」當特徵）

5. ✅ 衡量「加了有沒有用」
   → 加新特徵 → 看 CV 分數有沒有提升
   → 沒提升就拿掉、KISS 原則
```

  
    **☕ 用人話講**
    
Feature Engineering 是「**資料科學家的真功夫**」、決定 model 好壞 80%。**8 大技巧**：(1) **類別編碼**（Label / One-Hot / Target） (2) **數值變換**（Scale / Log / Binning） (3) **文字**（length / TF-IDF / Embedding） (4) **時間**（hour / weekday / sin-cos 週期） (5) **交互**（price/qty、bmi） (6) **Lag / Rolling**（時序） (7) **Group**（user 歷史） (8) **PCA**（降維）。**永遠用 Pipeline** 避免 leakage。**心法**：好特徵 + 簡單 model > 爛特徵 + XGBoost。
