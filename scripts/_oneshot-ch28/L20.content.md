### 進階：ProxyRotator class（業界做法、自動處理失敗）

**核心功能**：
- 多個 proxy 輪流用
- 失敗的自動跳過
- 全失敗時 reset 重試

```text
import random
import requests

class ProxyRotator:
    def __init__(self, proxies):
        self.proxies = proxies
        self.failed = set()
    
    def get(self):
        """拿一個沒失敗過的 proxy"""
        available = [p for p in self.proxies if p not in self.failed]
        if not available:
            # 全失敗 → reset、給「至少能再試一次」的機會
            self.failed.clear()
            available = self.proxies
        return random.choice(available)
    
    def mark_failed(self, proxy):
        self.failed.add(proxy)
    
    def get_with_retry(self, url, max_retries=3):
        """完整 retry 邏輯：拿 proxy → 失敗換下一個"""
        for _ in range(max_retries):
            proxy = self.get()
            try:
                res = requests.get(
                    url,
                    proxies={'http': proxy, 'https': proxy},
                    timeout=10
                )
                if res.status_code == 200:
                    return res
            except Exception:
                self.mark_failed(proxy)
        return None

# === 用 ===
PROXIES = [
    'http://user:pass@proxy1.com:8080',
    'http://user:pass@proxy2.com:8080',
    'http://user:pass@proxy3.com:8080',
]
rotator = ProxyRotator(PROXIES)

for url in urls:
    res = rotator.get_with_retry(url)
    if res:
        # 處理...
        pass
```

### 失敗檢測 + 健康度追蹤（production 用）

```text
from collections import Counter
import time

class HealthyProxyRotator:
    def __init__(self, proxies):
        self.proxies = proxies
        self.stats = {p: {'success': 0, 'fail': 0, 'last_fail': 0} for p in proxies}
    
    def get(self):
        # === 1 小時內失敗超過 5 次 → cooldown ===
        now = time.time()
        healthy = [
            p for p in self.proxies
            if not (
                self.stats[p]['fail'] > 5 and
                now - self.stats[p]['last_fail'] < 3600
            )
        ]
        if not healthy:
            healthy = self.proxies                # 全死、reset
            for p in healthy:
                self.stats[p]['fail'] = 0
        return random.choice(healthy)
    
    def mark_success(self, proxy):
        self.stats[proxy]['success'] += 1
    
    def mark_failed(self, proxy):
        self.stats[proxy]['fail'] += 1
        self.stats[proxy]['last_fail'] = time.time()
    
    def report(self):
        """看每個 proxy 健康度"""
        for p, s in self.stats.items():
            rate = s['success'] / max(s['success'] + s['fail'], 1)
            print(f'{p}: 成功 {s["success"]} 失敗 {s["fail"]} 成功率 {rate:.1%}')
```

### Proxy 安全 + 法律

```text
=== ⚠️ 警告 ===
1. 不要用免費 proxy
   - 99% 失效 / 慢 / 可能被監聽
   - 你的 cookie / token 被偷
   - 你的爬蟲行為被記錄

2. Residential proxy 道德爭議
   - 真實住宅 IP 的「**主人**」可能不知道自己的 IP 被用來爬蟲
   - 部分供應商透過免費 VPN / 免費 App 換用戶授權
   - 大量爬蟲打到對方、可能影響「房間住戶」上網

3. 法律
   - 用 proxy 繞 IP 限制本身合法
   - 但「用 proxy 爬付費 / 登入後內容」= 違反 ToS、可能被告
```

  
### 4 大反 pattern（不要做）

```text
1. ❌ 用免費 proxy
   → 99% 失效、慢、可能被監聽

2. ❌ 一個 proxy 打 10000 次
   → 跟沒用 proxy 一樣會被封

3. ❌ 沒 retry 機制
   → proxy 暫時掛掉、整個爬蟲失敗

4. ❌ 沒健康度追蹤
   → 壞掉的 proxy 一直 retry、浪費時間
```

Proxy 商業實戰 4 件事：(1) **ProxyRotator class** 自動輪換 + retry (2) **HealthyProxyRotator** 追蹤健康度、壞掉自動 cooldown (3) **Scrapy ProxyMiddleware** 整合 Scrapy (4) **成本估算** 100 萬 request 用 ScraperAPI $149/月最划算。**選擇心法**：indie 用 **ScraperAPI**（不用管 proxy）、規模大用 Smartproxy（自己管）、超大規模用 Bright Data。**永遠不用免費 proxy**——慢 / 不穩 / 可能被監聽。
