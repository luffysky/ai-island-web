**Scrapy = Python 大型爬蟲框架**——爬上萬頁、做產品級爬蟲必用。Requests + BS4 是手工作坊、Scrapy 是工廠生產線。

> 💡 **何時用 Scrapy？** 要爬上千 / 上萬個頁面、需要 retry / pipeline / 排程、要 production 級可靠性。**爬幾十頁直接用 Requests + BS4** 就好、Scrapy 學習曲線陡。

  
### 安裝 + 建專案

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**（裝 Scrapy + 建專案 + 建 spider）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
uv add scrapy
scrapy startproject myspider
cd myspider
scrapy genspider books books.toscrape.com
```

**這 4 行做什麼？逐個解釋：**
- `uv add scrapy` —— 裝 Scrapy 主套件
- `scrapy startproject myspider` —— 建一個叫 `myspider` 的 Scrapy 專案資料夾（含 settings.py / pipelines.py 等）
- `cd myspider` —— 進專案資料夾
- `scrapy genspider books books.toscrape.com` —— 建一個叫 `books` 的 spider、目標網站是 `books.toscrape.com`（Scrapy 官方練習網站）

  
### 第一個 Spider

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📄 **寫進 myspider/spiders/books.py**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
import scrapy

class BooksSpider(scrapy.Spider):
    name = 'books'                            # 跑時用這名字
    start_urls = ['http://books.toscrape.com']
    
    def parse(self, response):
        """每個頁面跑這個 method、抓資料 + 找下一頁"""
        
        # === 抓書名 + 價格 + 庫存 ===
        for book in response.css('article.product_pod'):
            yield {
                'title': book.css('h3 a::attr(title)').get(),   # ::attr(title) 取屬性
                'price': book.css('.price_color::text').get(),  # ::text 取文字
                'stock': book.css('.availability::text').get().strip(),
            }
        
        # === 找「下一頁」連結、繼續爬 ===
        next_page = response.css('li.next a::attr(href)').get()
        if next_page:
            yield response.follow(next_page, callback=self.parse)
            # 自動處理相對路徑、自動發 request、回來再跑 self.parse
```

> 💡 **Scrapy CSS selector** 跟 BS4 / 前端類似、但**多了 `::text` 跟 `::attr()`** 兩個偽選擇器、超方便。

  
### 執行 spider

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
scrapy crawl books -o books.json
```

**這行做什麼？逐個解釋：**
- `scrapy crawl books` —— 跑名字叫 `books` 的 spider
- `-o books.json` —— 輸出結果到 `books.json`（自動格式化）

  
**其他輸出格式：**

```text
scrapy crawl books -o books.csv               # CSV
scrapy crawl books -o books.xml               # XML
scrapy crawl books -s CLOSESPIDER_ITEMCOUNT=20  # 爬 20 個就停（測試用）
```

  
### settings.py 重要設定（必懂、不然會被網站封）

```text
# === 自動遵守 robots.txt（L1 教過、道德守則）===
ROBOTSTXT_OBEY = True

# === 並發控制（同時幾個 request）===
CONCURRENT_REQUESTS = 16                      # 全域同時 16 個
CONCURRENT_REQUESTS_PER_DOMAIN = 8            # 同一網站同時 8 個

# === 請求間隔（防 IP 被封）===
DOWNLOAD_DELAY = 1                            # 每個 request 間隔 1 秒
RANDOMIZE_DOWNLOAD_DELAY = True               # 0.5-1.5 倍隨機（看起來像人）

# === 自動 throttle（智慧調速、推薦）===
AUTOTHROTTLE_ENABLED = True
AUTOTHROTTLE_START_DELAY = 1
AUTOTHROTTLE_MAX_DELAY = 10
# Scrapy 自動依伺服器回應速度調整間隔、避免打爆

# === User-Agent ===
USER_AGENT = 'Mozilla/5.0 (compatible; MyBot/1.0; +https://yoursite.com/bot)'
# 慣例：寫網址 / email、讓對方能聯絡你

# === 自動 retry（失敗自動重試）===
RETRY_ENABLED = True
RETRY_TIMES = 3
RETRY_HTTP_CODES = [500, 502, 503, 504, 408, 429]
# 429 = Too Many Requests（被擋）、自動 retry 救命
```

  
### Item Pipeline（資料處理 / 儲存層）

**Pipeline 概念**：spider 抓到的 item 經過一條「**處理管線**」——去重 / 清洗 / 寫入 DB / 寄 email。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📄 **寫進 myspider/pipelines.py**（MongoDB 儲存範例）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
import pymongo

class MongoDBPipeline:
    def open_spider(self, spider):
        """spider 啟動時跑（開 DB 連線）"""
        self.client = pymongo.MongoClient('mongodb://localhost:27017')
        self.db = self.client['scrapy']
    
    def process_item(self, item, spider):
        """每個 item 經過這裡"""
        # === 1. 去重 ===
        if self.db.books.find_one({'title': item['title']}):
            raise scrapy.exceptions.DropItem('重複')
        
        # === 2. 清洗（價格 '£51.77' → 51.77）===
        item['price'] = float(item['price'].replace('£', ''))
        
        # === 3. 存 DB ===
        self.db.books.insert_one(dict(item))
        return item
    
    def close_spider(self, spider):
        """spider 結束時跑（關連線）"""
        self.client.close()
```

  
**啟用 pipeline（在 settings.py 加）：**

```text
ITEM_PIPELINES = {
    'myspider.pipelines.MongoDBPipeline': 300,  # 數字 = 優先序、越小越前
}
```

  
### Scrapy + Playwright（最強組合、處理 JS 渲染）

**Scrapy 本身不跑 JS**——用 `scrapy-playwright` plugin 整合 L3 的 Playwright。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**（裝 scrapy-playwright）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
uv add scrapy-playwright
playwright install chromium
```

**這 2 行做什麼？**
- `scrapy-playwright` —— Scrapy 跟 Playwright 整合 plugin
- `playwright install chromium` —— 裝 Chromium 瀏覽器（L3 提過）

  
**設定（settings.py）：**

```text
DOWNLOAD_HANDLERS = {
    'http': 'scrapy_playwright.handler.ScrapyPlaywrightDownloadHandler',
    'https': 'scrapy_playwright.handler.ScrapyPlaywrightDownloadHandler',
}
TWISTED_REACTOR = 'twisted.internet.asyncioreactor.AsyncioSelectorReactor'
```

**Spider 內用：**

```text
def start_requests(self):
    yield scrapy.Request(
        url,
        meta={'playwright': True}             # 這個 request 用 Playwright 跑
    )
```

  
### 分散式 Scrapy（Scrapy-Redis、爬全網規模時用）

**多台機器一起爬、共用 queue + 去重**——爬百萬商品時用這個架構。中型公司爬蟲常見配置。

```text
架構：
- 1 台 Redis（中央 queue + 去重）
- N 台 Scrapy worker（從 queue 拉 URL、爬、結果寫 DB）
- 失敗自動 retry、queue 持久化、機器掛了不丟資料
```

> 💡 **indie 通常用不到**——上千頁用單機 Scrapy 就夠、百萬頁才考慮分散式。

  
    **☕ 用人話講**
    
Scrapy 是 **「爬蟲工廠生產線」**——適合上萬頁、需要 retry / pipeline / 排程的大專案。**5 大組件**：(1) **Spider**（爬蟲邏輯） (2) **Pipeline**（資料處理） (3) **settings.py**（throttle / retry / UA） (4) **Item Loader**（清洗、進階） (5) **Middleware**（請求攔截、進階）。**怎麼決定**：< 100 頁用 Requests + BS4、100-10000 頁用 Scrapy、10000+ 頁用 Scrapy + Redis。
