**Async 爬蟲 = 「**同時打 100 個 request、不一個個等**」**——用 `httpx` + `asyncio`、速度比 requests 快 10-100 倍。

> 💡 **何時用 async？** 一次要爬 > 50 個 URL、且不需要 JS 渲染。**Playwright 也支援 async**（L10 教過）、但 httpx 更輕量、適合純抓 HTML / API。

  
### httpx vs requests（為什麼選 httpx）

| 特性 | requests | **httpx** ⭐ |
|---|---|---|
| 同步 | ✓ | ✓ |
| **Async** | ❌ | ✓ |
| HTTP/2 | ❌ | ✓ |
| API | 經典 | 跟 requests 一致 |
| 速度 | 中 | 快（async 模式快 10-100 倍） |

> 💡 **2026 趨勢**：新專案直接用 httpx、API 跟 requests 一樣、又可以 async。

```text
uv add httpx
```

**這行做什麼？** 裝 httpx、modern HTTP 客戶端、支援 async / HTTP/2 / sync 三模式。

  
**基本 async 爬蟲：**

```text
import asyncio
import httpx

async def fetch(client, url):
    """爬一個 URL、回傳 text"""
    response = await client.get(url, timeout=10)
    return response.text

async def main():
    async with httpx.AsyncClient() as client:
        # === 同時打 100 個 request ===
        urls = [f'https://example.com/page/{i}' for i in range(100)]
        results = await asyncio.gather(*[fetch(client, url) for url in urls])
    return results

# === 跑 ===
asyncio.run(main())
# 速度：requests 同步 = 100 秒、httpx async = 5 秒（20 倍快）
```

  
### 限制並發（避免打爆對方）

**`asyncio.Semaphore`** = 「**同時最多 N 個跑**」、防止一次打 100 個被擋。

```text
async def main():
    sem = asyncio.Semaphore(10)               # 同時最多 10 個
    
    async def fetch_limited(client, url):
        async with sem:                       # 進來前要拿 sem 鎖
            return await fetch(client, url)
        # 跳出 with 自動放開 sem 鎖
    
    async with httpx.AsyncClient() as client:
        urls = [f'https://example.com/page/{i}' for i in range(100)]
        results = await asyncio.gather(*[fetch_limited(client, url) for url in urls])
    return results
```

> 💡 **業界經驗值**：對方網站不抗議 = 並發 5-10、有反爬 = 並發 1-3、API 服務 = 看對方 rate limit。


**這行做什麼？** 裝 `tenacity`、Python 最強 retry 函式庫、有 decorator + 多種策略（指數退避 / 隨機 jitter / 條件 retry）。

  
```text
from tenacity import retry, stop_after_attempt, wait_exponential

@retry(
    stop=stop_after_attempt(3),               # 最多 3 次
    wait=wait_exponential(multiplier=1, min=2, max=10)   # 指數退避 2s / 4s / 8s
)
async def fetch_with_retry(client, url):
    response = await client.get(url, timeout=10)
    response.raise_for_status()               # 4xx / 5xx raise
    return response.text
# 失敗自動重試 3 次、每次等更久
```

### 完整 production 範本（並發 + retry + 進度 + 寫檔）

```text
import asyncio
import httpx
import json
from tenacity import retry, stop_after_attempt, wait_exponential
from tqdm.asyncio import tqdm

HEADERS = {
    'User-Agent': 'Mozilla/5.0 (Macintosh) Chrome/131.0',
    'Accept-Language': 'zh-TW,zh;q=0.9',
}

@retry(
    stop=stop_after_attempt(3),
    wait=wait_exponential(multiplier=1, min=2, max=10)
)
async def fetch(client, url):
    response = await client.get(url, headers=HEADERS, timeout=15)
    response.raise_for_status()
    return response.json()                    # 假設爬 API、回傳 JSON

async def main(urls):
    sem = asyncio.Semaphore(10)               # 限制並發 10
    
    async def limited_fetch(url):
        async with sem:
            try:
                return await fetch(client, url)
            except Exception as e:
                print(f'失敗 {url}: {e}')
                return None
    
    async with httpx.AsyncClient(http2=True) as client:
        # http2=True 啟用 HTTP/2、單一連線多路復用、更快
        results = await tqdm.gather(*[limited_fetch(u) for u in urls])
    
    # 過濾失敗的、寫 JSONL
    valid = [r for r in results if r]
    with open('results.jsonl', 'w', encoding='utf-8') as f:
        for item in valid:
            f.write(json.dumps(item, ensure_ascii=False) + '\n')
    
    print(f'✓ 完成、成功 {len(valid)}/{len(urls)}')

# === 跑 ===
urls = [f'https://api.example.com/item/{i}' for i in range(1, 1001)]
asyncio.run(main(urls))
```

  
### Async 常見坑（新手必踩）

```text
1. ❌ 忘記 await
   response = client.get(url)            # 沒 await、回傳 coroutine
   text = response.text                  # AttributeError
   
   ✓ 加 await：response = await client.get(url)

2. ❌ 在 sync 函式內呼叫 async
   def main():
       result = fetch(url)               # 沒用、回 coroutine
   
   ✓ async def main(): result = await fetch(url)
   ✓ 或 asyncio.run(fetch(url))（top-level）

3. ❌ 用 time.sleep 阻塞 event loop
   await some_task()
   time.sleep(1)                         # 阻塞整個 event loop
   
   ✓ 用 await asyncio.sleep(1)

4. ❌ 並發太高被封
   await asyncio.gather(*[fetch(u) for u in 10000_urls])
   # 同時 1 萬個 request 對方瞬間擋你
   
   ✓ 用 Semaphore 限制：Semaphore(10)

5. ❌ AsyncClient 每個 request 重建（慢）
   for url in urls:
       async with httpx.AsyncClient() as c:   # 每次都 new
           await c.get(url)
   
   ✓ 共用一個 client：
   async with httpx.AsyncClient() as c:
       for url in urls:
           await c.get(url)
```

  
### 何時用 async vs sync

```text
=== ✅ 用 async ===
- 大量 I/O 操作（爬多個 URL）
- 每個 request 等對方時間久
- 不需要 CPU 重運算

=== ❌ 不用 async ===
- 只爬 1-10 個 URL（async overhead 不划算）
- CPU 密集（用 multiprocessing）
- 簡單 script 一次性
```
