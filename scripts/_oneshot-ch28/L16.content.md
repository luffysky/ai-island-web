**Schedule = 「**爬蟲自動定時跑、不用手動每天執行**」**——4 種主流方案：cron / APScheduler / Airflow / Prefect、依複雜度選。

> 💡 **業界選擇心法**：1-3 個爬蟲用 **cron** 就夠、5-20 個用 **APScheduler / Prefect**、20+ 個或多步驟 pipeline 才上 **Airflow**。

  
### 方案 1：cron（最簡單、Linux / Mac 內建、零成本）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**（編輯 crontab）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
crontab -e
```

**這行做什麼？** 開啟 cron 編輯器、加排程任務。

  
**加這行（每天凌晨 3 點跑、log 寫到 /var/log）：**

```text
0 3 * * * cd /app && python3 scraper.py >> /var/log/scraper.log 2>&1
```

**cron 5 格式速記**：`分 時 日 月 週幾`、`*` 代表「任何」。

| 格式 | 意思 |
|---|---|
| `0 3 * * *` | 每天 3:00 |
| `*/15 * * * *` | 每 15 分鐘 |
| `0 9 * * 1-5` | 週一到週五 9:00 |
| `0 0 1 * *` | 每月 1 號 0:00 |

> 💡 **cron 語法產生器**：`crontab.guru` 線上工具、輸入需求自動產 cron 字串、超實用。

  
### 方案 2：APScheduler（Python 內嵌排程、適合單機）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
uv add apscheduler
```

**這行做什麼？** 裝 APScheduler、Python 純內嵌排程器、不用外部服務、適合「**爬蟲本身就是常駐 process**」場景。

  
```text
from apscheduler.schedulers.blocking import BlockingScheduler

sched = BlockingScheduler()

# === Decorator 排程：每天 3 點跑 ===
@sched.scheduled_job('cron', hour=3)
def scheduled_scrape():
    scrape_all()

# === interval 排程：每 30 分鐘跑 ===
@sched.scheduled_job('interval', minutes=30)
def quick_check():
    check_prices()

sched.start()                                 # 阻塞、進入排程 loop
```

> 💡 **APScheduler 優勢**：不用學 cron 語法、純 Python、可以動態加排程（從 DB 讀任務）。

  
### 方案 3：Airflow（業界標準、多步驟 pipeline）

**Airflow** = Airbnb 出的「**工作流編排平台**」、業界 ETL / 爬蟲 #1 工具、有 web UI 看執行狀況。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
uv add apache-airflow
```

**這行做什麼？** 裝 Airflow、複雜（要設 DB / scheduler / webserver）、適合公司級 ETL。

  
```text
from airflow import DAG
from airflow.operators.python import PythonOperator
from datetime import datetime

# === DAG = Directed Acyclic Graph（有向無環圖）===
dag = DAG(
    'daily_scrape',
    schedule='0 3 * * *',                     # cron 格式
    start_date=datetime(2026, 1, 1),
    catchup=False                             # 不補跑歷史
)

# === 多步驟 task 串聯（A → B → C）===
scrape_task = PythonOperator(
    task_id='scrape',
    python_callable=scrape_all,
    dag=dag
)
clean_task = PythonOperator(
    task_id='clean',
    python_callable=clean_data,
    dag=dag
)
notify_task = PythonOperator(
    task_id='notify',
    python_callable=send_report,
    dag=dag
)

scrape_task >> clean_task >> notify_task     # 串聯
```

**Airflow Web UI** 功能：
- 看每個 task 跑成功 / 失敗
- 手動 retry 失敗的 task
- 看歷史執行紀錄
- 看 task log

> ⚠️ **Airflow 啟動成本高**——需要 PostgreSQL + Redis + Scheduler + Webserver 4 個服務。**indie 不要用、過度工程**。

  
### 方案 4：Prefect（modern Airflow 替代、輕量友善）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
uv add prefect
```

**這行做什麼？** 裝 Prefect、2020+ 興起的 Airflow 替代、API 友善、支援 cloud + local。

  
```text
from prefect import flow, task
import requests

@task(retries=3, retry_delay_seconds=10)      # 內建 retry
def scrape_url(url):
    return requests.get(url).text

@flow
def daily_scrape():
    urls = get_urls()
    results = [scrape_url(u) for u in urls]   # 自動平行、自動 retry
    save_to_db(results)

if __name__ == '__main__':
    # === 直接 serve、用 cron 排程 ===
    daily_scrape.serve(cron='0 3 * * *')
```

**Prefect 優勢**：
- 比 Airflow 簡單 10 倍
- 內建 retry / cache / parallel
- Cloud 版（Prefect Cloud）免費 tier 含 dashboard
- 純 Python decorator、學習曲線低

  
### 4 種方案對比表

| 方案 | 適合 | 學習曲線 | 啟動成本 |
|---|---|---|---|
| **cron** ⭐ | 1-3 個爬蟲、簡單排程 | 低 | $0 |
| **APScheduler** | 5-20 個、單機 Python | 低 | $0 |
| **Airflow** | 20+ 個、多步驟 pipeline、公司用 | 高 | 高（要架 PG + Redis） |
| **Prefect** ⭐ | 5-50 個、modern stack | 中 | 低（有 Cloud） |

  
### 監控 + 通知（排程必加、不然壞了你不知道）

```text
import requests, traceback

def safe_scrape():
    try:
        scrape_all()
    except Exception as e:
        # === 失敗自動發 Telegram / Discord / Slack 通知 ===
        msg = f'❌ 爬蟲失敗：{e}\n{traceback.format_exc()}'
        requests.post(
            f'https://api.telegram.org/bot{TOKEN}/sendMessage',
            json={'chat_id': CHAT_ID, 'text': msg}
        )
```

> 💡 **新手 #1 坑**：排程跑爬蟲、壞了 1 個月才發現（沒監控）。**永遠加失敗通知**。

  
### Docker 化（讓爬蟲在任何環境都能跑）

```text
# Dockerfile
FROM python:3.12-slim
WORKDIR /app
COPY pyproject.toml uv.lock ./
RUN pip install uv && uv sync --frozen
COPY . .
CMD ["python", "scraper.py"]
```

**部署到雲端**：Railway / Fly.io 直接連 GitHub、push code 自動 deploy + cron 排程（Railway cron jobs / Fly.io machines）。

排程 4 大方案：(1) **cron** ⭐ 1-3 個爬蟲就夠 (2) **APScheduler** Python 內嵌、適合 5-20 個 (3) **Airflow** 公司級 ETL、過度工程 (4) **Prefect** ⭐ modern 替代 Airflow、indie 推薦。**核心心法**：(1) 簡單 cron 解決 80% 場景 (2) **永遠加失敗通知**（Telegram / Discord） (3) **永遠加 retry**（網路抖動很常見） (4) Docker 化、雲端部署。**接案 indie 起手 cron**、規模大才升級。
