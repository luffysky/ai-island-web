**解析 HTML：CSS Selector + XPath + 6 大抓資料招數**——爬蟲的核心技能。會看 HTML 找路徑、什麼網站都能爬。

> 💡 **業界比例**：CSS Selector 80%、XPath 15%、特殊抓法（JSON-LD / data-* / 內嵌 JSON）5% —— 但**特殊抓法是接案高 CP 值技巧**、會的人少。

  
### CSS Selector 速查（最常用、跟前端一致）

| Selector | 意思 |
|---|---|
| `.class` | class 是 X |
| `#id` | id 是 X |
| `div > p` | div 的**直接子**是 p |
| `div p` | div **底下任何後代** p |
| `[attr^="abc"]` | 屬性**開頭**是 abc |
| `[attr$="abc"]` | 屬性**結尾**是 abc |
| `[attr*="abc"]` | 屬性**包含** abc |
| `:nth-child(2)` | 第 2 個子元素 |
| `:not(.x)` | 不是 class x 的 |
| `:has(> span)` | 含直接子 span 的（CSS4） |

  
### XPath 速查（CSS 做不到的時候用）

```text
//div[@class="x"]                            # 全網 div、class=x
//div[contains(@class, "abc")]               # class 包含 abc
//div[contains(text(), '優惠')]              # 文字含「優惠」
//div[contains(text(), '優惠')]/following-sibling::div[1]
                                              # 「含優惠的 div」之後第 1 個兄弟 div
//a[starts-with(@href, '/product/')]/@href   # href 開頭 /product/ 的所有 href 屬性值
```

**XPath 優勢**：能用「**文字內容**」找元素（CSS 做不到）、能往「**父 / 兄弟**」走（CSS 也較弱）。

  
**用 lxml 跑 XPath（比 BS4 快、L8 教過）：**

```text
from lxml import html

tree = html.fromstring(response.text)
prices = tree.xpath('//div[@class="product"]/span[@class="price"]/text()')
```

  
### 6 大抓資料招數

**1. JSON-LD（金礦、現代電商 90% 都有）：**

```text
import json

# === 從 <script type="application/ld+json"> 直接拿結構化資料 ===
script = soup.select_one('script[type="application/ld+json"]')
data = json.loads(script.string)
print(data)
# {
#   "@type": "Product",
#   "name": "iPhone 17",
#   "offers": {"price": 30000, "availability": "InStock"},
#   ...
# }
```

> 💡 **為什麼 e-commerce 一定要有 JSON-LD？** Google 搜尋結果要顯示「**價格 / 評分 / 庫存**」豐富片段（Rich Snippets）必須有 JSON-LD。所以**所有大電商都有**——直接拿、比解 HTML 穩 100 倍。

  
**2. `data-*` 屬性（前端框架愛用）：**

```text
div = soup.select_one('div[data-product-id]')
print(div['data-product-id'])                # '12345'
print(div['data-price'])                     # '1299'
print(div['data-stock'])                     # 'in_stock'

# === 取所有 data-* 屬性 ===
data_attrs = {k: v for k, v in div.attrs.items() if k.startswith('data-')}
```

  
**3. 內嵌 JSON（SPA 常見、最強技巧之一）：**

```text
import re, json

# === 很多 SPA 把資料塞 window.__INITIAL_DATA__ ===
match = re.search(
    r'window\.__INITIAL_DATA__\s*=\s*({.*?});',
    response.text,
    re.DOTALL
)
data = json.loads(match.group(1))
# 拿到整個頁面初始 state、不用解 HTML
```

**常見的 JS 全域變數名**：`__INITIAL_DATA__` / `__NUXT__` / `__NEXT_DATA__` / `window.appData`。

> 💡 **怎麼找？** Chrome F12 → Console → 打 `window` 看屬性、找你要的資料藏在哪個變數。

  
**4. 屬性值：**

```text
img_url = soup.select_one('img')['src']
img_alt = soup.select_one('img')['alt']
href = soup.select_one('a')['href']

# === 如果可能沒這個屬性、用 .get() ===
img_url = soup.select_one('img').get('src', 'default.png')
```

  
**5. 多個元素一次取：**

```text
tags = [t.get_text(strip=True) for t in soup.select('.tag')]
# ['Python', 'Web', 'AI']

links = [a['href'] for a in soup.select('a[href]')]
# 所有有 href 的連結
```

  
**6. 巢狀（典型電商列表 pattern）：**

```text
products = []
for card in soup.select('div.product'):
    products.append({
        'name': card.select_one('.name').get_text(strip=True),
        'price': card.select_one('.price').get_text(strip=True),
        'image': card.select_one('img')['src'],
        'link': card.select_one('a')['href'],
    })
```

  
### 進階：Next.js 網站（`__NEXT_DATA__`）

**Next.js（前端框架）的網站、HTML 內一定有 `<script id="__NEXT_DATA__">` 塞所有 server-side props——不用爬、直接 parse！**

```text
import re, json

next_data = soup.select_one('script#__NEXT_DATA__')
if next_data:
    data = json.loads(next_data.string)
    # data['props']['pageProps'] 通常含完整商品資料
    products = data['props']['pageProps']['products']
```

> 💡 **看 HTML view source、Ctrl+F 搜「__NEXT_DATA__」、有就是 Next.js 寫的、就有金礦**。Vercel / Netlify / 大電商超多。

  
### 怎麼找 selector（Chrome F12 流程）

```text
1. 在網頁上 右鍵想抓的元素 → 檢查（Inspect）
2. F12 開發者工具自動跳出、藍色 highlight 該元素
3. 右鍵該 HTML → Copy → Copy selector
4. 貼出來、自己簡化（Chrome 給的 selector 通常太長）

簡化原則：
- 砍掉中間沒意義的層
- 保留 class / id / 屬性特徵
- 例：
  原始：body > main > section:nth-child(3) > div.list > div:nth-child(2) > a.product
  簡化：div.list a.product 或 .list .product
```

  
### 找不到 selector？試這些技巧

```text
1. F12 → Console → 跑 document.querySelectorAll('.your-selector')
   → 看回傳幾個元素、確認你的 selector 對不對

2. 找不到？元素可能是 JS 動態生成 → 用 Playwright（L3）

3. selector 寫了但回空？可能：
   - class 名有「動態 hash」（如 .price_a8b3c）→ 用屬性 selector
   - 元素在 iframe 內 → 要先切到 iframe
   - 用 BS4 但網頁有 broken HTML → 換 lxml parser
```

  
### 速度對比（解析 1MB HTML）

| 方法 | 時間 | 用法 |
|---|---|---|
| **BeautifulSoup + html.parser** | 慢（500ms） | 預設、新手用 |
| **BeautifulSoup + lxml** ⭐ | 中（100ms） | 業界推薦 |
| **lxml.html + XPath** | 快（50ms） | 大量資料用 |
| **lxml + SoupStrainer**（只解需要區塊）| 最快（30ms） | 超大頁面用 |
