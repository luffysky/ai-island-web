**儲存爬到的資料**——CSV / JSON / SQLite / PostgreSQL / MongoDB 5 種選擇、依**資料量 + 結構**選對的。

> 💡 **常見錯誤**：100 萬筆資料存 CSV、Excel 開不起來 / 查詢超慢。**對的格式選對 = 之後分析超順、選錯 = 重做**。

  
### CSV（最簡單、Excel 友善、< 1 萬筆用）

```text
import pandas as pd

df = pd.DataFrame(items)
df.to_csv(
    'data.csv',
    index=False,
    encoding='utf-8-sig'                      # ⚠️ Excel 開中文不亂碼必加
)
```

**讀回來：**

```text
df = pd.read_csv('data.csv', encoding='utf-8-sig')
```

> ⚠️ **utf-8** vs **utf-8-sig** 差別：sig 多了 BOM、Excel 才認得。**Mac 用 Numbers 也吃 utf-8-sig**。

  
### JSONL（JSON Lines、大檔友善）

**JSONL** = 「**一行一個 JSON 物件**」、不是傳統 JSON array。優勢：**可以一行一行讀寫**、不用整個檔載入記憶體、爬 100 萬筆超適合。

```text
import json

# === 寫 ===
with open('data.jsonl', 'w', encoding='utf-8') as f:
    for item in items:
        f.write(json.dumps(item, ensure_ascii=False) + '\n')
        # ensure_ascii=False = 中文不 escape（更小檔 + 可讀）

# === 讀（記憶體友善、一行處理一個）===
with open('data.jsonl', encoding='utf-8') as f:
    for line in f:
        item = json.loads(line)
        process(item)
```

> 💡 **JSONL 是業界大資料標配**——Hugging Face / OpenAI fine-tune 資料都用這格式。

  
### SQLite（< 100 萬筆、單檔 DB、零安裝）

```text
import sqlite3

conn = sqlite3.connect('data.db')             # 開啟（沒有自動建）
c = conn.cursor()

# === 建表（IF NOT EXISTS 重複跑不錯）===
c.execute("""
CREATE TABLE IF NOT EXISTS products (
    id INTEGER PRIMARY KEY,
    name TEXT,
    price REAL,
    link TEXT UNIQUE                          -- UNIQUE 防重複插入
)
""")

# === 批次插入（executemany 比 execute 快 10 倍）===
c.executemany(
    'INSERT OR IGNORE INTO products (name, price, link) VALUES (?, ?, ?)',
    [(p['name'], p['price'], p['link']) for p in products]
)
# INSERT OR IGNORE = link 重複的自動 skip、不 raise

conn.commit()
conn.close()
```

  
**讀回來（直接給 Pandas 分析）：**

```text
import pandas as pd
import sqlite3

conn = sqlite3.connect('data.db')
df = pd.read_sql("""
    SELECT * FROM products
    WHERE price < 1000
    ORDER BY price DESC
""", conn)
```

> 💡 **SQLite 優勢**：零安裝、單檔、直接給 Pandas 用、適合個人專案 + 接案小型 case。

  
### PostgreSQL（> 100 萬筆、多人共用、production）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**（裝 psycopg2 PostgreSQL 驅動）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
uv add psycopg2-binary
```

**這行做什麼？** 裝 `psycopg2-binary`（不用編譯版）、PostgreSQL 標準 Python 驅動。

  
```text
import psycopg2
from psycopg2.extras import execute_values

conn = psycopg2.connect(
    host='localhost',
    database='scraper',
    user='postgres',
    password='secret'
)
cur = conn.cursor()

# === 批次插入 + ON CONFLICT 防重複（PostgreSQL 強項）===
execute_values(
    cur,
    """
    INSERT INTO products (name, price, link) VALUES %s
    ON CONFLICT (link) DO NOTHING
    """,
    [(p['name'], p['price'], p['link']) for p in products]
)
# execute_values 比 executemany 快 50 倍（PG 專屬）

conn.commit()
cur.close()
conn.close()
```

> 💡 **PG vs SQLite**：SQLite 單檔、單機、單 writer；PG 多 user / 多 writer / 強 SQL。爬蟲跑在 server + 多人查、用 PG。

  
### MongoDB（不規則資料 / 巢狀 JSON、文件型 DB）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🖥️ **在終端機輸入**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
uv add pymongo
```

**這行做什麼？** 裝 `pymongo`、MongoDB Python 驅動。

  
```text
from pymongo import MongoClient

client = MongoClient('mongodb://localhost:27017')
db = client['scraper']
collection = db['products']

# === 直接塞 dict、不用先建 schema ===
collection.insert_many(items)

# === 防重複（用 upsert）===
for item in items:
    collection.update_one(
        {'link': item['link']},               # 找 link 相同的
        {'$set': item},                       # 更新
        upsert=True                           # 沒有就插入
    )
```

> 💡 **何時用 MongoDB？** 爬到的資料**結構不固定**（每個商品欄位不一樣）、或有**深層巢狀 JSON**（如評論 + 子評論 + 圖片）、MongoDB 比 SQL 友善。

  
### 5 種儲存對比表（依資料量選）

| 資料量 | 推薦 | 理由 |
|---|---|---|
| **< 1 萬** | CSV / JSON | 簡單、Excel 開 |
| **< 100 萬** | **SQLite** ⭐ | 零安裝、單檔、Pandas 友善 |
| **< 1 億** | **PostgreSQL** ⭐ | 強 SQL、多人共用、production |
| **> 1 億** | PostgreSQL + 分表 / BigQuery / Snowflake | 大資料倉儲 |
| **不規則 / 巢狀** | **MongoDB** | schema-less |
| **超大 + 分析** | Parquet + DuckDB（ch27 L19） | 欄式儲存、分析超快 |

  
### 寫入時的反重複策略

**所有 DB 都要處理「**爬到重複資料怎辦**」**——有 4 種策略：

```text
1. ❌ 直接插入
   → 重複資料堆積、之後分析爆

2. ✓ UNIQUE constraint + INSERT OR IGNORE（SQLite）
   c.execute('INSERT OR IGNORE INTO ...')

3. ✓ ON CONFLICT DO NOTHING（PostgreSQL）
   INSERT ... ON CONFLICT (link) DO NOTHING

4. ✓ Upsert（MongoDB）
   collection.update_one({...}, {'$set': ...}, upsert=True)

5. ✓ Pandas drop_duplicates（小資料）
   df.drop_duplicates(subset=['link'], keep='last')
```

  
### 加 timestamp 追歷史變化（價格監控用）

```text
from datetime import datetime

for item in items:
    item['scraped_at'] = datetime.now().isoformat()
    # 之後可以 SELECT * WHERE scraped_at > X 查歷史變化
```

  
### 自動備份（重要！爬資料萬一丟了哭都來不及）

```text
import shutil
from datetime import datetime

# === SQLite 直接複製檔案 ===
backup = f'data_backup_{datetime.now().strftime("%Y%m%d")}.db'
shutil.copy('data.db', backup)

# === 上 S3 / Google Drive（更安全）===
# 用 boto3（S3）或 pydrive（Google Drive）
```

儲存 5 大選擇：(1) **CSV** < 1 萬筆、Excel 開 (2) **JSONL** 大檔友善、ML 訓練必用 (3) **SQLite** ⭐ < 100 萬、單檔、Pandas 友善 (4) **PostgreSQL** ⭐ > 100 萬、多人共用、production (5) **MongoDB** 不規則 / 巢狀 JSON。**心法**：(1) 永遠加 UNIQUE 防重複 (2) 永遠加 timestamp 追歷史 (3) 永遠備份。**新手起手**：先 CSV、規模大轉 SQLite、production 上 PG。
