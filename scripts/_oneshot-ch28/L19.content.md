**Selenium = 「**爬蟲傳統王者**」**——2004 年就有、用了 20 年、現在被 Playwright 取代。**這課重點不是學 Selenium、是了解差別 + 為什麼新專案選 Playwright**。

> 💡 **何時用 Selenium？** 維護老 codebase（公司用 Selenium 寫一堆測試）、或團隊只會 Selenium。**新專案 100% 選 Playwright**（L3 / L10）。

  
### 安裝（vs Playwright 複雜）

🖥️ **在終端機輸入**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```text
uv add selenium webdriver-manager
```

**這行做什麼？**
- `selenium` —— 主套件
- `webdriver-manager` —— 自動下載 ChromeDriver、不用手動

> 💡 vs Playwright：Playwright 一個 `playwright install` 搞定瀏覽器、Selenium 還要 webdriver-manager 自動下載 driver、麻煩一點。

  
### 基本範例

```text
from selenium import webdriver
from selenium.webdriver.chrome.service import Service
from webdriver_manager.chrome import ChromeDriverManager
from selenium.webdriver.common.by import By
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC

# === 啟動瀏覽器（要寫一堆 import） ===
driver = webdriver.Chrome(
    service=Service(ChromeDriverManager().install())
)

# === 開頁 ===
driver.get('https://example.com')

# === 找元素（5 種 By） ===
title = driver.find_element(By.TAG_NAME, 'h1').text
btn = driver.find_element(By.CLASS_NAME, 'submit')
link = driver.find_element(By.LINK_TEXT, '點我')
input_el = driver.find_element(By.CSS_SELECTOR, 'input[name="email"]')

# === 等元素出現（要手動 wait、Playwright 自動）===
WebDriverWait(driver, 10).until(
    EC.presence_of_element_located((By.CLASS_NAME, 'product'))
)

# === 互動 ===
input_el.send_keys('user@example.com')
driver.find_element(By.CSS_SELECTOR, 'button[type="submit"]').click()

# === 結束（記得關、不然瀏覽器殘留）===
driver.quit()
```

> ⚠️ **vs Playwright 痛點**：Selenium 要 import 一堆東西 / 要手動 WebDriverWait / 沒自動 wait、新手寫法囉嗦。


  
### undetected-chromedriver（反 Selenium 偵測必用、L11 教過）

**Selenium 預設有 `navigator.webdriver = true` flag、網站一秒識破**。

```text
import undetected_chromedriver as uc

driver = uc.Chrome(
    headless=True,
    version_main=131                           # 指定 Chrome 版本
)
driver.get('https://cloudflare-protected.com')
print(driver.page_source)
driver.quit()
# 連 Cloudflare Bot Protection 都騙過
```

  
### Selenium vs Playwright 全方位對比

| 面向 | **Playwright** ⭐ | Selenium |
|---|---|---|
| 發布年 | 2020（Microsoft） | 2004 |
| **速度** | 快 5 倍 | 慢（每個 action 都有 RPC overhead） |
| **自動 wait** | ✓ 內建 | ❌ 要手動 WebDriverWait |
| **反偵測** | playwright-stealth | undetected-chromedriver |
| **API 設計** | modern、async-first | 老派、繁瑣 |
| **多瀏覽器** | Chromium / Firefox / WebKit | 全部（更多） |
| **截攔 API** | ✓ 原生 | 需 extension |
| **Codegen** | ✓ 內建錄製 | ❌ |
| **Trace Viewer** | ✓ 內建 | ❌ |
| **學習資源** | 中（較新） | 多（20 年累積） |
| **公司常用** | 新創 / modern | 大型企業 / 老 codebase |

  
### Selenium 何時還是會用

```text
1. 公司有大量 Selenium 寫的 E2E 測試（不可能全 rewrite）
2. 團隊只會 Selenium（Java / .NET / Ruby 生態）
3. 需要支援超舊瀏覽器（IE / 老 Safari、Playwright 不支援）
4. 用 Selenium Grid 跑分散式測試（成熟）
```

  
### Selenium → Playwright 遷移建議

**如果你正在用 Selenium、考慮換 Playwright：**

```text
1. 新功能用 Playwright 寫（不要再加 Selenium code）
2. 高頻維護的舊測試逐步換成 Playwright
3. 低頻 / 穩定的舊 Selenium code 保留不動
4. 完全遷移約 3-6 個月（看代碼量）

收益：
- 跑得快 5 倍（CI 時間縮短）
- 自動 wait 減少 50% 寫 code
- Trace Viewer 大幅減少 debug 時間
```

      🎯
      面試常考
    
    Selenium / Playwright 面試常問：(1) **為什麼 Playwright 比 Selenium 快** —— Playwright 用 WebSocket 跟瀏覽器溝通、Selenium 用 HTTP（多 RPC overhead） (2) **自動 wait 機制差別** —— Playwright 內建 actionability check（元素可見 + 可點才執行）、Selenium 要手動 WebDriverWait (3) **反偵測** —— Playwright 用 playwright-stealth、Selenium 用 undetected-chromedriver。**面試加分**：如果你用過 Selenium、但說「**新專案我會選 Playwright、因為...**」就大加分（顯示有 update 知識）。
