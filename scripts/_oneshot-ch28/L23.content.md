**爬蟲法律 + 道德**——做爬蟲之前一定要看完。**爬錯被告坐牢、爬對是技能變現**。

> ⚠️ **這課重點**：L1 / L5 / L11 都提過法律、這課完整版彙整。**不看完不要做爬蟲**。

  
### 台灣 4 大法律風險（依嚴重度排序）

**1. 妨害電腦使用罪（刑法 358-363 條、最重）**

```text
- 未經授權存取電腦 → 5 年以下有期徒刑
- 衝擊伺服器（DDoS 等級）→ 加重
- 商業利益 → 加重

何時觸發：
- 破解登入 / 帳號（即使你猜對密碼）
- 大量爬蟲打掛對方
- 繞過明顯的存取控制（如付費牆）
```

  
**2. 著作權法**

```text
- 抓的內容有版權（文章 / 圖片 / 影片）
- 商業利用 = 侵權
- 即使「公開」也算

例：爬 Medium 文章商用 = 侵權
例：爬蘋果日報新聞商用 = 侵權
```

  
**3. 個資法**

```text
- 抓「個資」就違法（即使公開）
- 個資 = 姓名 + 聯絡方式 + 任何能識別的資訊
- 從社群網站爬 user 資訊風險爆高

例：爬 LinkedIn 抓 email = 違法
例：爬 FB 個人頁 = 違法
```

  
**4. GDPR / CCPA（歐盟 / 美國法律）**

```text
- 罰金可達 4% 全球營收（GDPR）
- 即使你是台灣公司、爬歐盟用戶資料也適用
- Clearview AI 被罰超過 €20M

何時觸發：
- 爬歐盟 user 資料（社群 / 電商）
- 賣資料給第三方
```

  
### robots.txt（不是法律、是禮貌）

**位置**：`https://example.com/robots.txt`

**範例：**

```text
User-agent: *
Disallow: /private/                # 禁止爬 /private/
Disallow: /api/internal/
Allow: /public/
Crawl-delay: 10                    # 每次間隔 10 秒
```

**爬蟲心法**：
- robots.txt **不是法律**、技術上可以無視
- 但**業界共識**：遵守 = 你是好公民、不遵守 = 對方有理由告你
- 法庭上「**對方有 robots.txt 你不遵守**」會被當成「**故意違反**」、加重判決

  
### 倫理快速分類

**✅ 通常 OK（低風險）：**
```text
- 抓公開資料（沒個資 / 沒版權）
- 非商業（學術研究 / 個人專案）
- 比價（不商業利用對方資料）
- 政府開放資料
- 看 robots.txt + 加 delay
```

  
**⚠️ 灰色（要評估）：**
```text
- 商業使用公開資料（要看 ToS）
- 抓社群（FB / IG）— ToS 禁止但很多人在做
- 大量爬電商商品資料
- AI 訓練資料
```

  
**❌ 危險（不要做）：**
```text
- 抓個資、賣個資（個資法）
- 抓版權內容轉賣（著作權法）
- 衝擊網站（妨害電腦使用罪）
- 破解付費內容（破壞防護措施）
- 繞過 2FA / Captcha 強行爬登入後內容
```

  
### 真實案例（從中學）

**1. LinkedIn vs hiQ Labs（2017-2022）**

```text
背景：hiQ 爬 LinkedIn 公開個人頁、做 HR 分析賣給公司
結果：hiQ 勝訴（公開資料爬蟲合法）
代價：訴訟費用巨大、hiQ 後來破產
教訓：「合法」不代表沒事、會被巨頭告到破產
```

  
**2. Clearview AI（2020-）**

```text
背景：從社群（FB / IG / Twitter）爬 30 億張臉、賣給警方做臉部識別
結果：歐盟多國罰款（義大利 €20M、法國 €20M、希臘 €20M）
教訓：個資 + 大規模商用 = 全球被罰
```

  
**3. 台灣 2020 蝦皮商品爬蟲案**

```text
背景：個人爬蝦皮商品資料
判決：「爬公開資料無罪」
但：「大量爬導致對方損失」可能涉刑法 360
教訓：規模 + 商業目的會影響判決
```

  
### 自保 6 招

```text
1. ✅ 看 robots.txt、能不爬就不爬
2. ✅ 加 delay（1-3 秒/request、不要打爆對方）
3. ✅ 不抓個資（最大紅線）
4. ✅ 不商業使用版權內容（即使爬到）
5. ✅ 聯絡網站問是否有官方 API（最安全）
6. ✅ 商業專案前**諮詢法律顧問**（NT$ 5,000-20,000 諮詢費、防百萬罰金）
```

  
### Indie 開發者建議（依用途）

| 用途 | 風險 | 建議 |
|---|---|---|
| **個人 / 學習** | 低 | 隨便玩、注意 delay |
| **學術研究** | 低 | 引用來源、不商用 |
| **個人比價工具** | 低中 | 個人使用 OK、不要公開分享 |
| **接案做 SaaS** | 高 | **必須**先諮詢律師 + 簽合約 |
| **賣資料給第三方** | 極高 | **不要做**、99% 違法 |

  
### 合約條款（接案時保護自己）

```text
合約應該寫清楚：
1. 客戶承擔法律責任（爬資料的用途由客戶決定）
2. 你只負責「**技術實作**」、不負責「**用途合法性**」
3. 如果爬的網站告人、客戶賠
4. 不爬個資、不爬付費內容、不破解防護
5. 客戶提供目標網站 ToS、你確認沒明確禁止才開始
```

  
### 如果不確定怎辦？決策樹

```text
我可以爬這個網站嗎？

1. 是公開資料嗎？（不用登入看得到）
   ❌ → 不要爬（涉破解 / 個資）
   ✅ → 進 2

2. 含個資嗎？（姓名 / email / 電話）
   ✅ → 不要爬（個資法）
   ❌ → 進 3

3. 有官方 API 嗎？
   ✅ → 用官方 API（合法 + 穩定）
   ❌ → 進 4

4. robots.txt 禁止嗎？
   ✅ → 不要爬（業界共識）
   ❌ → 進 5

5. ToS 禁止商業爬蟲嗎？
   ✅ → 不要做商用（個人 OK）
   ❌ → 進 6

6. 你會打爆對方嗎？
   ✅ → 加 delay + 限制並發
   ❌ → OK、爬吧
```

爬蟲 4 大法律風險：(1) **妨害電腦使用罪**（衝擊伺服器 / 破解登入、5 年以下） (2) **著作權法**（商業利用版權內容） (3) **個資法**（即使公開的個資也違法） (4) **GDPR / CCPA**（4% 全球營收罰款）。**自保 6 招**：robots.txt / delay / 不抓個資 / 不商用版權 / 用官方 API / 商業專案諮詢律師。**indie 心法**：個人玩低風險、商業必須諮詢。

  

  
    
      ⚠️
      最後警告
    
    **看完還是想做不確定合法的爬蟲？** 3 個原則：(1) **規模小**（一天 < 1 萬 request、不會打爆對方） (2) **不商用**（個人 / 研究、不轉賣） (3) **不抓個資**（即使公開也別碰）。**遇到含糊的情況**：(1) 寄信問網站「**我要爬 X 資料、可以嗎**」 (2) 諮詢律師 NT$ 5,000-20,000、防百萬罰款 (3) **不確定就不做**——爬資料賺的不多、被告賠的很多、CP 值算不過來。
