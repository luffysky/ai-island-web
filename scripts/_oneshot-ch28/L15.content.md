**登入 + Session 爬蟲 = 「**爬「登入後才看得到」的內容**」**——個人化推薦、訂單記錄、私訊內容。**3 種登入方式 + cookie 重用技巧**。

> ⚠️ **法律警告**：爬「**登入後個人化內容**」涉及個資 + 帳號授權、商用前一定要先確認合法性（L1 / L5 / L11 法律段重溫）。

  
### 方式 1：requests + Session（最快、適合無 JS 登入）

```text
import requests
from bs4 import BeautifulSoup

session = requests.Session()

# === 1. 訪問登入頁、拿 CSRF token（很多網站防 CSRF 必有）===
login_page = session.get('https://example.com/login')
soup = BeautifulSoup(login_page.text, 'lxml')
csrf = soup.select_one('input[name="csrf_token"]')['value']

# === 2. POST 登入（同時帶 csrf）===
session.post('https://example.com/login', data={
    'email': 'user@example.com',
    'password': 'password',
    'csrf_token': csrf,
})

# === 3. 之後同 session 自動帶 cookie ===
res = session.get('https://example.com/dashboard')
```

> 💡 **CSRF token 是什麼？** Cross-Site Request Forgery（跨站請求偽造）防護 token——表單內藏的隱藏值、要把這值送回去 server 才認可。**沒帶 CSRF 直接 403**。

  
### 方式 2：Playwright 自動化登入（適合複雜 JS / OAuth）

```text
from playwright.sync_api import sync_playwright
import requests

with sync_playwright() as p:
    browser = p.chromium.launch(headless=False)   # 開發時看畫面、production 改 True
    page = browser.new_page()
    
    # === 自動操作登入 ===
    page.goto('https://example.com/login')
    page.fill('input[name="email"]', 'user@example.com')
    page.fill('input[name="password"]', 'password')
    page.click('button[type="submit"]')
    page.wait_for_url('**/dashboard')
    
    # === 拿到登入後的 cookie ===
    cookies = page.context.cookies()
    browser.close()

# === 把 cookie 倒給 requests 用（之後爬就不用每次都跑 Playwright、超快）===
session = requests.Session()
for cookie in cookies:
    session.cookies.set(cookie['name'], cookie['value'])
```

> 💡 **這招超實用**——Playwright 跑一次拿 cookie、之後 requests 用 cookie 爬。**Playwright 啟動慢、requests 快、混合用最強**。

  
### 方式 3：直接打 login API（最快、需要先逆向）

**很多網站登入是「**前端 POST 到 /api/login、回 JSON 含 token**」**——直接打這個 API、不用模擬整個瀏覽器。

```text
# === F12 → Network → 看登入 POST 到哪個 endpoint ===
res = requests.post(
    'https://example.com/api/login',
    json={'email': 'user@example.com', 'password': 'pw'}
)
token = res.json()['access_token']

# === 之後請求帶 token（看網站用 cookie 還是 header）===
headers = {'Authorization': f'Bearer {token}'}
data = requests.get(
    'https://example.com/api/profile',
    headers=headers
).json()
```

  
### Cookie 儲存 + 重用（重要、不要每次都重新登入）

```text
import pickle

# === 儲存 cookies ===
with open('cookies.pkl', 'wb') as f:
    pickle.dump(session.cookies, f)

# === 之後直接載入（不用重登入）===
session = requests.Session()
with open('cookies.pkl', 'rb') as f:
    session.cookies.update(pickle.load(f))

# === 直接用（如果 cookie 還沒過期就 work）===
res = session.get('https://example.com/dashboard')
```

> 💡 **cookie 通常 1 天 ~ 30 天有效**——存起來重用、不要每次都 brute force 登入（會觸發風控）。

  
### 處理 2FA（兩步驟驗證、很多網站有）

```text
# === 通常 2FA 流程 ===
# 1. 帳密登入 → 回「需要 2FA」
# 2. 系統寄 OTP 給你（email / SMS / Authenticator App）
# 3. 你填 OTP → 完成登入

# === 爬蟲處理 2FA 3 種方式 ===

# 方式 1：手動輸入（最簡單、適合一次性）
otp = input('輸入 OTP：')
session.post('/api/verify-2fa', data={'otp': otp})

# 方式 2：用 TOTP 套件（自動生成 OTP）
# uv add pyotp
import pyotp
totp = pyotp.TOTP('YOUR_SECRET')              # 從 QR code 拿 secret
otp = totp.now()                              # 自動生 6 位數
session.post('/api/verify-2fa', data={'otp': otp})

# 方式 3：cookie 重用避開 2FA
# 登入一次（含 2FA）→ 存 cookie → 之後用 cookie、不用再 2FA
```

  
### 處理「**Cloudflare Turnstile / hCaptcha 在登入頁**」

**最難爬的場景**——登入頁就有 captcha、必須先解 captcha 才能登入。

**解法（從便宜到貴）：**

```text
1. 手動登入一次 → 存 cookie → 用 cookie 爬
   ⭐ 最划算、適合長期專案

2. 用 Playwright + cookies session 持久化
   from playwright.sync_api import sync_playwright
   browser = p.chromium.launch_persistent_context('./user_data', headless=False)
   # 第一次手動登入、之後自動沿用

3. 上 2Captcha / Capsolver API 自動解（L11 教過）
   $2-3/1000 題、量大才划算
```

  
### 登入失敗常見原因 + 解法

```text
1. 沒帶 CSRF token → 加上
2. 沒帶 Referer header → 加 Referer: 'https://example.com/login'
3. 沒帶 User-Agent → 加完整 UA
4. IP 觸發風控（短時間多次嘗試）→ 換 IP + 慢慢試
5. 帳號被鎖（嘗試太多次）→ 等 30 分鐘 / 解鎖
6. 需要 device fingerprint → Playwright + stealth
```

  
    **☕ 用人話講**
    
登入爬蟲 3 種方式：(1) **requests + Session** 適合無 JS 登入（要拿 CSRF token） (2) **Playwright 自動化** 適合複雜 JS / OAuth、拿 cookie 後轉 requests 用 (3) **直接打 login API** 最快、要先 F12 逆向找 endpoint。**核心心法**：(1) **存 cookie 重用** 不要每次重登 (2) **2FA 用 pyotp 自動化** (3) **Captcha 用手動 + cookie 持久化** 最划算。**法律警告**：爬登入後個資商用前確認合法。

  

  
    
      ⚠️
      法律警告
    
    爬登入後內容法律風險高：(1) 涉「**個資法**」（爬到別 user 個資） (2) 涉「**著作權**」（爬付費內容） (3) 違反「**網站 ToS**」（封帳號 + 民事賠償）。**安全做法**：(1) **只爬自己帳號**的資料（如 export 自己 IG 貼文） (2) **明確使用者授權**（OAuth、跟 user 借 token） (3) **公開資料優先**（不登入也看得到的）。**接案時合約寫清楚客戶承擔法律責任**。
