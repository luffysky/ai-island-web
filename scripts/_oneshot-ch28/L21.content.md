### 完整 try / except 範本（業界做法）

```text
import requests
import time
from requests.exceptions import HTTPError, Timeout, ConnectionError

def fetch_with_retry(url, max_retries=3):
    """完整錯誤處理：依錯誤類型不同 retry 策略"""
    for attempt in range(max_retries):
        try:
            res = requests.get(url, timeout=10)
            res.raise_for_status()
            return res
        
        except HTTPError as e:
            # === 4xx / 5xx 錯誤 ===
            if res.status_code == 429:
                # Rate limit：等久一點（指數退避 * 60 秒）
                wait = 2 ** attempt * 60
                print(f'被限流、等 {wait} 秒')
                time.sleep(wait)
            elif res.status_code in (500, 502, 503, 504):
                # Server error：等短一點
                wait = 2 ** attempt
                time.sleep(wait)
            elif res.status_code == 404:
                # 404 = 永遠不存在、不要 retry
                return None
            else:
                raise                          # 其他 4xx 直接拋
        
        except (Timeout, ConnectionError) as e:
            # === 網路問題 → 等一下再試 ===
            wait = 2 ** attempt
            print(f'網路問題、等 {wait} 秒：{e}')
            time.sleep(wait)
    
    return None                                # 所有重試都失敗
```

  
### 用 tenacity（L14 教過、業界標準 retry decorator）

```text
from tenacity import retry, stop_after_attempt, wait_exponential, retry_if_exception_type
from requests.exceptions import RequestException

@retry(
    stop=stop_after_attempt(3),
    wait=wait_exponential(multiplier=1, min=2, max=60),
    retry=retry_if_exception_type(RequestException),   # 只 retry 特定錯誤
    reraise=True                                        # 最後失敗 raise 原錯誤
)
def fetch(url):
    res = requests.get(url, timeout=10)
    res.raise_for_status()
    return res
```

### Healthchecks.io（簡單監控「**爬蟲有沒有正常跑**」）

**核心概念**：爬蟲每次跑完 ping 一次 URL、超過排程時間沒 ping 就 alert（Email / SMS / Telegram）。

```text
import requests

CHECK_URL = 'https://hc-ping.com/YOUR-UUID'

def scheduled_scrape():
    requests.get(f'{CHECK_URL}/start')         # 開始
    
    try:
        scrape_all()
        requests.get(CHECK_URL)                # 成功
    except Exception as e:
        requests.get(f'{CHECK_URL}/fail')      # 失敗
        raise
```

**Healthchecks.io 自動：**
- 排程時間到沒收到 ping → 發 alert
- 收到 /fail → 發 alert（含失敗訊息）
- 連續 N 次失敗 → 升級 alert（pager duty）

**免費 tier**：20 個 check、無限通知。

from pydantic import BaseModel, ValidationError, HttpUrl, Field

class Product(BaseModel):
    name: str = Field(min_length=1, max_length=200)
    price: float = Field(ge=0)                  # >= 0
    link: HttpUrl                                # 必須是合法 URL

def parse_and_validate(raw_data):
    try:
        return Product(**raw_data).model_dump()

            return func(*args, **kwargs)

            raise
    return wrapper
