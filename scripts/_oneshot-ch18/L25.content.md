### 2. Year 1：基礎（必修）

```text
📚 Year 1 必學

   ⃣✓ SQL 基本語法
      → SELECT / WHERE / JOIN / GROUP BY / ORDER BY
      → UNION / subquery / CTE
      → 練習 100 道 LeetCode SQL

   ⃣✓ 1 個 ORM 熟練
      → Prisma（推薦新手）
      → 或 Drizzle（更貼 SQL）
      → CRUD 全部用過

   ⃣✓ Index 概念
      → 哪些欄位該加 index
      → 複合 index 順序
      → EXPLAIN 看執行計畫

   ⃣✓ 簡單 backup
      → pg_dump + cron
      → 上 S3 / GCS
      → 演練 restore 一次

   ⃣✓ Transaction 基本
      → ACID 是什麼
      → BEGIN / COMMIT / ROLLBACK
      → 何時要包 transaction

→ 完成 Year 1 = 能做小型專案
→ 月薪 NT$ 35-60k（接案 / 初階）
```

  
### 3. Year 2：效能 + Redis

```text
📚 Year 2 必學

   ⃣✓ EXPLAIN ANALYZE
      → 看 Seq Scan vs Index Scan
      → 找慢 query 的兇手
      → cost / actual time / rows

   ⃣✓ Connection pool
      → 為什麼要 pool
      → max 怎麼設
      → leak 怎麼 debug

   ⃣✓ Transaction 進階
      → Isolation level（read commited / repeatable read / serializable）
      → Deadlock 怎麼避免
      → SELECT FOR UPDATE

   ⃣✓ 慢查詢實戰
      → N+1 query
      → SELECT * 害處
      → OFFSET vs cursor pagination

   ⃣✓ Redis cache
      → cache-aside pattern
      → write-through / write-back
      → TTL 怎麼設

   ⃣✓ Replication 概念
      → primary / replica
      → 同步 / 非同步
      → read / write 分離

→ 完成 Year 2 = 能做中型 SaaS
→ 月薪 NT$ 60-100k（中級工程師）
```

  
### 4. Year 3：分散式

```text
📚 Year 3 必學

   ⃣✓ Replication 實戰
      → PG streaming replication
      → 設定 read replica
      → failover 怎麼做

   ⃣✓ Sharding
      → 水平切分（按 user_id / region）
      → 跨 shard query 痛點
      → consistent hashing

   ⃣✓ NoSQL 一個專精
      → MongoDB（彈性）/ DynamoDB（AWS）/ Cassandra（規模）
      → Redis 進階（cluster / Lua / streams）

   ⃣✓ 監控 + 告警
      → Prometheus + Grafana
      → DB metrics（QPS / 慢 query / 連線）
      → AlertManager

   ⃣✓ 多 DB 整合
      → CDC（Change Data Capture）
      → Debezium / Kafka Connect
      → 跨 DB transaction（saga pattern）

   ⃣✓ 大型 migration
      → zero-downtime schema change
      → backfill + double write
      → 工具：gh-ost / pt-online-schema-change

→ 完成 Year 3 = 能設計大型系統
→ 月薪 NT$ 100-180k（資深 / DBA）
```

  
### 5. Year 4+：DB 專家

```text
📚 Year 4+ 進階

```

### 9. 永遠不變的真理

```text
💎 業界 30 年不變的真理

   1. SQL 是命根
      ─────────
      → 學什麼新 DB 都可以
      → SQL 一定要會
      → 80% 工作場景用得到

   2. PostgreSQL 是 forever
      ─────────
      → 開源、免費、強大
      → 30 年了還在進化
      → 不會死、不會被取代

   3. 領域知識 > 技術深度
      ─────────
      → 懂業務的 backend > 純技術
      → 知道「電商怎麼運作」比「DB 內部」重要
      → 業務需求驅動技術選型

   4. 不要為新而新
      ─────────
      → 新 DB 天天出
      → 80% 兩年就死
      → PG + Redis 撐百萬 MAU 沒問題

   5. backup 是命根
      ─────────
      → 沒備份等於沒命
      → 沒測過的備份 ≈ 沒備份
      → 寫進工作流程

   6. 別怕慢、要懂為什麼慢
      ─────────
      → EXPLAIN ANALYZE 是好朋友
      → 找出兇手再優化
      → 別亂猜
```

  
### 10. 給未來的自己

```text
✉️ 從新手 → DB 工程師的 7 句話

   1. 每天寫一段 SQL、寫一年就熟了

   2. EXPLAIN ANALYZE 是你最好的朋友
      看不懂就 Google、學 1 個月就懂

   3. backup 設定完今天就演練 restore
      別等出事才發現備份壞了

   4. 不會的 DB 別硬上 production
      先在 dev / staging 玩熟

   5. ORM 是好東西、但 SQL 也要會
      ORM 出問題時你還能 debug

   6. 別追新 DB
      你的 PG 還沒榨乾、別搬 Mongo

   7. 寫 docs / 留紀錄
      schema 改了寫 ADR
      未來的自己會感謝你
```
