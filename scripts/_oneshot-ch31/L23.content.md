### 1. 內建 test runner（不用 Jest！）

**Node 18+ 內建 `node:test` + `node:assert`**——直接寫測試、不裝任何套件。

📄 **寫進程式檔**：`src/math.test.js`

```text
import { test, describe, before, after } from "node:test";
import assert from "node:assert/strict";
import { add } from "./math.js";

describe("add()", () => {
  before(() => {
    console.log("setup");
  });

  test("正常加法", () => {
    assert.equal(add(1, 2), 3);
  });

  test("負數加法", () => {
    assert.equal(add(-1, -2), -3);
  });

  test("async 測試", async () => {
    const result = await Promise.resolve(42);
    assert.equal(result, 42);
  });
});
```

**執行**：

🖥️ **在終端機輸入**：

```text
node --test
# 或跑特定檔案
node --test src/math.test.js

# Watch 模式
node --test --watch

# 平行跑
node --test --test-concurrency=4

# 出 TAP 格式（CI 友善）
node --test --test-reporter=tap

# 出 spec 格式（漂亮）
node --test --test-reporter=spec
```

**為什麼比 Jest 強？**

```text
✅ 0 依賴（不用裝 jest / vitest）
✅ 啟動快 10 倍（Jest 啟動 2 秒、node:test 0.2 秒）
✅ 原生 ESM 支援（Jest 設定 ESM 痛苦）
✅ 內建 mock / spy（node:test/mock）
✅ 內建 coverage（--experimental-test-coverage）
✅ 跟 TS 完美整合（tsx --test）
```

**內建 mock**：

```text
import { test, mock } from "node:test";

test("mock fn", () => {
  const fn = mock.fn();
  fn(1);
  fn(2);
  assert.equal(fn.mock.calls.length, 2);
  assert.equal(fn.mock.calls[0].arguments[0], 1);
});

test("mock module method", () => {
  mock.method(console, "log");
  console.log("hi");
  assert.equal(console.log.mock.calls.length, 1);
});
```

  
### 2. 內建 `fetch`（不用 axios / node-fetch）

**Node 18+ 內建 Web 標準 `fetch`**——跟瀏覽器同一個 API。

```text
// 不用 require / import
const res = await fetch("https://api.github.com/users/luffy");
const user = await res.json();
console.log(user);

// 完整選項
const res = await fetch("https://api.example.com/posts", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    "Authorization": "Bearer xxx"
  },
  body: JSON.stringify({ title: "Hi" }),
  signal: AbortSignal.timeout(5000)  // 5 秒 timeout
});
```

**Web 標準的 `Request` / `Response` / `Headers`**：

```text
const req = new Request("https://example.com");
const headers = new Headers({ "X-Custom": "val" });
const res = new Response(JSON.stringify({ ok: true }), {
  status: 200,
  headers: { "content-type": "application/json" }
});
```

**為什麼這超棒？**

```text
✅ 不用裝 axios / node-fetch
✅ 跟 Cloudflare Workers / Deno / Bun / 瀏覽器同一個 API
✅ AbortSignal 支援（timeout / cancel）
✅ Streaming 支援（res.body 是 ReadableStream）
```

### 4. `--watch` 模式（不用 nodemon）

```text
🖥️ 在終端機輸入：

# 改檔自動重啟
node --watch src/server.js

# Watch 多個檔
node --watch-path=./src --watch-path=./config src/server.js
```

**取代 nodemon**：

```text
之前：
   npm i -D nodemon
   nodemon src/server.js

現在：
   node --watch src/server.js
```

  
### 5. `--env-file`（不用 dotenv）

```text
🖥️ 在終端機輸入：

node --env-file=.env src/server.js
node --env-file=.env.local --env-file=.env src/server.js  # 多個
```

**`.env` 檔自動讀進 `process.env`**：

📄 **寫進 .env**：

```text
DATABASE_URL=postgresql://...
JWT_SECRET=supersecret
PORT=3000
```

📄 **寫進 src/server.js**：

```text
console.log(process.env.DATABASE_URL);  // 自動有了
console.log(process.env.JWT_SECRET);
```

**之前要 `dotenv` 套件 + `require("dotenv").config()`、現在內建**。

  
### 6. `require(ESM)`（Node 22+）

**長久以來的痛**：CJS 不能 `require` 純 ESM 套件。

**Node 22+ 開了實驗性 flag**：

```text
🖥️ 在終端機輸入：

node --experimental-require-module src/index.cjs
```

```text
// CJS 內可以 require ESM 套件
const chalk = require("chalk");  // chalk@5 純 ESM、現在也行
console.log(chalk.red("hi"));
```

**Node 23 預期穩定、預設啟用**。

  
### 7. 內建 `node:sqlite`（Node 22+）

**內建 SQLite client**——不用裝 better-sqlite3。

```text
import { DatabaseSync } from "node:sqlite";

const db = new DatabaseSync(":memory:");

db.exec(`CREATE TABLE users (id INTEGER PRIMARY KEY, name TEXT)`);

const insert = db.prepare(`INSERT INTO users (name) VALUES (?)`);
insert.run("Alice");
insert.run("Bob");

const rows = db.prepare(`SELECT * FROM users`).all();
console.log(rows);
// [{ id: 1, name: "Alice" }, { id: 2, name: "Bob" }]
```

### 9. `--inspect-wait`（debug 革命）

```text
🖥️ 在終端機輸入：

node --inspect-wait src/server.js
# server 不會啟動、等 Chrome DevTools 連上才跑
# 連上 chrome://inspect → 設斷點 → 第一行就停
```

**對比 `--inspect`**：

```text
--inspect       → 啟動後可以連 debugger（但 startup code 已跑過）
--inspect-brk   → 第一行斷點（要 debugger 連上才跑）
--inspect-wait  → 等 debugger 連上才開始
```

  
### 10. 完整 Node 22 開發 setup（不裝任何套件）

📄 **寫進 package.json**：

```text
{
  "name": "my-app",
  "type": "module",
  "scripts": {
    "dev": "node --watch --env-file=.env src/server.js",
    "test": "node --test --watch",
    "test:coverage": "node --test --experimental-test-coverage",
    "start": "node --env-file=.env src/server.js"
  }
  // 注意：dependencies 是空的！
}
```

**對比舊版本**：

```text
舊版 package.json：
```

### 11. 5 大常見坑

```text
❌ 1. 還在裝 nodemon
   → 浪費 install 時間 + 容易跟其他 watch 衝突
   → 修：node --watch

❌ 2. 還在用 axios
   → 體積大、不必要的抽象層
   → 修：原生 fetch

❌ 3. Jest 設 ESM 設定一團
   → next.config + babel + transform 一堆
   → 修：改用 node:test

❌ 4. dotenv 在 production 也用
   → 環境變數該由 platform 設、不該檔案
   → 修：dev 用 --env-file、prod 由 docker / vercel 設

❌ 5. 用 require(ESM) 沒加 flag
   → ERR_REQUIRE_ESM
   → 修：Node 22+ 加 --experimental-require-module
```
