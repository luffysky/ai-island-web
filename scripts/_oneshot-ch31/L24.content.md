**Hono（炎、日文「火焰」）= 取代 Express 的下一代 Node.js / Edge framework**——比 Express 快 5 倍、跑 Node / Bun / Deno / Cloudflare Workers / Vercel Edge / Lambda 全平台、API 跟瀏覽器 `fetch` 一模一樣。**2024 開始爆紅、2026 業界主流**。

> 💡 **這課的價值**：Express 是 2010 的東西、現在你用 Hono 寫 API、效能翻倍 + 跑 edge function + TS 完美支援 + bundle 30KB。會這課後、你能直接跳過 Express 學最現代的後端寫法。

  
### 1. 為什麼 Express 不夠用？

**Express 2010 年的設計**：

```text
❌ 不是 Web Standards（自己一套 req / res）
❌ TS 支援差（要裝 @types/express）
❌ 跑不了 Cloudflare Workers / Deno（不是 fetch API）
❌ middleware chain 效能差（Hono 快 5 倍）
❌ Async 支援差（要 next 串 callback）
❌ 沒內建 validation / OpenAPI
❌ Bundle 大（300KB+）
```

**Hono 的解法**：

```text
✅ 用 Web Standards Request / Response（fetch API）
✅ TS 第一公民（型別自動推、route param 自動推）
✅ 跑所有 runtime：Node / Bun / Deno / Workers / Vercel / Lambda
✅ Trie router 超快
✅ 完全 async/await
✅ 內建 validator（zod / valibot）
✅ Bundle 30KB（一個 worker 都裝得下）
```

### 3. 跟 Express 對比

**Express 寫法**：

```text
const express = require("express");
const app = express();
app.use(express.json());

app.get("/users/:id", (req, res) => {
  const id = req.params.id;
  res.json({ id, name: "Alice" });
});

app.post("/posts", (req, res) => {
  const body = req.body;
  res.status(201).json({ created: body });
});

app.listen(3000);
```

**Hono 寫法**：

```text
import { Hono } from "hono";
const app = new Hono();

app.get("/users/:id", (c) => {
  return c.json({ id: c.req.param("id"), name: "Alice" });
});

app.post("/posts", async (c) => {
  return c.json({ created: await c.req.json() }, 201);
});

export default app;  // 大多平台直接 export
```

**差別**：

```text
Express                Hono
────────────────────────────────
req / res 物件         c (Context) 一個
res.json(x)            return c.json(x)
res.status(201).json   return c.json(x, 201)
req.params             c.req.param
req.body（需 middleware）c.req.json() / form()
.listen()              export default
```

  
### 4. Context（`c`）—— Hono 的核心

`c` 是 Hono 的「上下文物件」、包所有 request / response 操作：

```text
// Request
c.req.param("id")        // path param
c.req.query("page")      // query string
c.req.header("X-API")    // header
await c.req.json()       // JSON body
await c.req.formData()   // form body
c.req.url                // 完整 URL

// Response（永遠 return）
c.text("hi")             // text/plain
c.html("<h1>hi</h1>")    // text/html
c.json({ a: 1 })         // application/json
c.body(stream)           // ReadableStream
c.redirect("/login")
c.notFound()
c.status(204)            // 只改 status

// State
c.set("user", user)      // 存
const u = c.get("user")  // 取（middleware 之間傳資料）
```

  
### 5. Middleware（中介軟體）

📄 **寫進程式檔**：`src/index.ts`

```text
import { Hono } from "hono";
import { logger } from "hono/logger";
import { cors } from "hono/cors";
import { etag } from "hono/etag";
import { compress } from "hono/compress";

const app = new Hono();

// 內建 middleware
app.use("*", logger());
app.use("*", cors({ origin: "https://app.example.com" }));
app.use("*", etag());
app.use("*", compress());

// 自訂 middleware
app.use("*", async (c, next) => {
  const start = Date.now();
  await next();
  console.log(`${c.req.method} ${c.req.path} - ${Date.now() - start}ms`);
});

// 認證 middleware
app.use("/api/*", async (c, next) => {
  const token = c.req.header("Authorization")?.replace("Bearer ", "");
  if (!token) return c.json({ error: "unauthorized" }, 401);

  const user = await verifyJWT(token);
  if (!user) return c.json({ error: "invalid token" }, 401);

  c.set("user", user);
  await next();
});
```

**內建 middleware 超齊全**：

```text
hono/logger       log
hono/cors         CORS
hono/etag         ETag
hono/compress     gzip / brotli
hono/jwt          JWT 認證
hono/bearer-auth  Bearer token
hono/basic-auth   Basic auth
hono/cache        Cache-Control
hono/csrf         CSRF 防護
hono/secure-headers  Security headers
hono/timing       Server-Timing
hono/timeout      Request timeout
```

  
### 6. Validation（zod 整合）

```text
import { Hono } from "hono";
import { zValidator } from "@hono/zod-validator";
import { z } from "zod";

const app = new Hono();

const createPostSchema = z.object({
  title: z.string().min(1).max(100),
  content: z.string().min(10),
  tags: z.array(z.string()).optional()
});

app.post(
  "/posts",
  zValidator("json", createPostSchema),
  (c) => {
    const data = c.req.valid("json");
    // data 完全型別安全：{ title, content, tags? }
    return c.json({ created: data }, 201);
  }
);
```

**驗證失敗自動回 400 + 錯誤詳情、不用自己寫**。

  
### 7. RPC mode（前後端型別共享）

**Hono 最神的功能**——前端直接呼叫後端、型別完全相通：

📄 **寫進後端**：`server.ts`

```text
import { Hono } from "hono";
import { zValidator } from "@hono/zod-validator";
import { z } from "zod";

const route = new Hono()
  .post("/posts",
    zValidator("json", z.object({ title: z.string() })),
    (c) => {
      const { title } = c.req.valid("json");
      return c.json({ id: 1, title });
    }
  );

export type AppType = typeof route;
export default route;
```

📄 **寫進前端**：`client.ts`

```text
import { hc } from "hono/client";
import type { AppType } from "../server/server";

const client = hc<AppType>("http://localhost:3000");

const res = await client.posts.$post({
  json: { title: "Hi" }   // ← 完全型別檢查
});

if (res.ok) {
  const data = await res.json();
  console.log(data.id, data.title);  // ← 型別也自動推
}
```

**像 tRPC 但跟 fetch 一模一樣、沒有額外抽象**。

### 10. 完整實戰：JWT + DB + validation

📄 **寫進程式檔**：`src/index.ts`

```text
import { Hono } from "hono";
import { jwt } from "hono/jwt";
import { logger } from "hono/logger";
import { zValidator } from "@hono/zod-validator";
import { z } from "zod";

type Env = {
  Variables: {
    user: { id: string; email: string };
  };
};

const app = new Hono<Env>();

app.use("*", logger());

app.post(
  "/login",
  zValidator("json", z.object({
    email: z.string().email(),
    password: z.string().min(8)
  })),
  async (c) => {
    const { email, password } = c.req.valid("json");
    const user = await authenticate(email, password);
    if (!user) return c.json({ error: "invalid" }, 401);

    const token = await signJWT({ sub: user.id, email });
    return c.json({ token });
  }
);

app.use("/api/*", jwt({ secret: process.env.JWT_SECRET! }));

app.get("/api/me", (c) => {
  const payload = c.get("jwtPayload");
  return c.json({ user: payload });
});

export default app;
```

  
### 11. 5 大常見坑

```text
❌ 1. 用 res.json() / req.body（Express 風格）
   → Hono 沒這 API
   → 修：return c.json(x) / await c.req.json()

❌ 2. 不 return response
   → Hono handler 必須 return
   → 修：永遠 return c.xxx()

❌ 3. middleware 沒 await next()
   → 後續路由不跑
   → 修：永遠 await next()

❌ 4. 部署 Workers 用 Node API（fs / path）
   → Workers 不支援 Node 內建
   → 修：用 Web Standards API（fetch / Request / Response）

❌ 5. RPC client URL 寫錯
   → Type 對但 runtime 噴 404
   → 修：base URL 環境變數注入、不要 hardcode
```
