你寫的 code 為什麼跑得快或慢？**演算法的選擇 = 速度差 1000 倍**。**Big O notation（大 O 標記）**—— 估算「資料量變大、程式時間怎麼變」的標準語言。

> 💡 **這課的價值**：面試一定問 Big O。實務上選對演算法、10 萬筆資料從 100 秒變 0.01 秒。學會這課、你的程式碼從業餘變專業。

  
### 1. 同個問題、不同寫法

```text
問題：找陣列裡有沒有某數字

📍 寫法 1：linear search（線性搜尋、一個一個找）

def find(arr, target):
    for x in arr:
        if x == target:
            return True
    return False

→ 1000 個元素：平均比 500 次
→ 100 萬個元素：平均比 50 萬次

📍 寫法 2：binary search（二元搜尋、先排序、對折找）

def find_sorted(arr, target):
    left, right = 0, len(arr) - 1
    while left <= right:
        mid = (left + right) // 2
        if arr[mid] == target: return True
        elif arr[mid] < target: left = mid + 1
        else: right = mid - 1
    return False

→ 1000 個元素：最多比 10 次
→ 100 萬個元素：最多比 20 次

→ 100 萬筆時、binary 快 25,000 倍
```

  
### 2. Big O 是什麼

```text
Big O = 「資料量 n 變很大時、所需操作數的成長趨勢」

只看成長趨勢、不看實際秒數：
- O(1)    — 不管多少筆都一樣快
- O(log n) — n 變 10 倍、時間只變 ~3 倍
- O(n)    — n 變 10 倍、時間變 10 倍
- O(n log n) — n 變 10 倍、時間變 ~33 倍
- O(n²)   — n 變 10 倍、時間變 100 倍
- O(2^n)  — n 變 10 倍、時間變 ∞（爆炸）

n = 資料量。
```

  
### 3. 7 大複雜度（從快到慢）

```text
⚡ O(1) — 常數時間
   不管 n 多大、時間固定
   範例：陣列存取 arr[5]、字典查找 dict[key]、hashmap put/get

📊 O(log n) — 對數時間
   每一步把問題縮一半
   範例：binary search、二元樹查找、二分法

✅ O(n) — 線性時間（最常見）
   每個元素都看一次
   範例：陣列遍歷、linear search、找最大值

🆗 O(n log n) — 線性對數時間
   分而治之的最佳排序
   範例：merge sort、quick sort（平均）、heap sort

⚠️ O(n²) — 平方時間
   雙層 nested loop、每個元素跟每個元素比一次
   範例：bubble sort、selection sort、找重複元素（暴力法）

⚠️ O(n³) — 立方時間
   三層 nested loop
   範例：矩陣相乘（暴力法）

🚨 O(2^n) — 指數時間（要避開）
   每多 1 個元素、時間 ×2
   範例：費氏數列遞迴、子集 / 排列暴力枚舉

🚨 O(n!) — 階乘時間（絕對避開）
   範例：旅行推銷員問題暴力解
```

**實際數字對照（n = 100）：**

| 複雜度 | 操作數 |
|---|---|
| O(1) | 1 |
| O(log n) | ~7 |
| O(n) | 100 |
| O(n log n) | ~700 |
| O(n²) | 10,000 |
| O(2^n) | 約 10³⁰（宇宙年齡內算不完）|
| O(n!) | 約 10¹⁵⁸ |

  
### 4. 怎麼算一段 code 的 Big O

**規則 1：常數忽略**

```text
def f(n):
    for i in range(n):    # n 次
        print(i)
    for j in range(n):    # 又 n 次
        print(j)
    
→ 2n 次、但 Big O = O(n)（常數丟掉）
```

**規則 2：取主導項**

```text
def f(n):
    for i in range(n):           # n
        for j in range(n):       # n²（主導）
            print(i, j)
    for k in range(n):           # n
        print(k)

→ n² + n、取主導 = O(n²)
```

**規則 3：nested loop 相乘**

```text
def f(n):
    for i in range(n):           # n
        for j in range(n):       # n
            print(i, j)

→ n × n = O(n²)
```

**規則 4：分而治之 = log n**

```text
def binary_search(arr, target):
    left, right = 0, len(arr) - 1
    while left <= right:
        mid = (left + right) // 2
        # 每次把範圍砍半
    
→ O(log n)（不管多少筆、log₂ n 步搞定）
```

  
### 5. 空間複雜度

```text
時間複雜度：執行用多少時間
空間複雜度：執行用多少記憶體

⚡ O(1) 空間：只用幾個變數
def sum(arr):
    s = 0
    for x in arr:
        s += x
    return s

✅ O(n) 空間：建一個跟輸入一樣大的東西
def double(arr):
    result = []
    for x in arr:
        result.append(x * 2)
    return result

⚠️ O(n²) 空間：建二維結構
def matrix(n):
    return [[0] * n for _ in range(n)]
```

  
### 6. 各語言常用操作的 Big O

```text
🐍 Python list:
   arr[i]        — O(1)   存取
   arr.append(x) — O(1) 平均
   arr.insert(0, x) — O(n)（要全部後移）
   x in arr      — O(n)
   arr.sort()    — O(n log n)

🐍 Python dict:
   d[key]        — O(1) 平均
   key in d      — O(1) 平均
   for k in d    — O(n)

🐍 Python set:
   s.add(x)      — O(1)
   x in s        — O(1)

📦 JS Array:
   類似 Python list

📦 JS Map / Set:
   類似 Python dict / set

📦 JS Object:
   property access — O(1) 平均
   Object.keys(obj) — O(n)
```

  
### 7. 實戰：選對結構救人一命

**情境**：寫一個 function、判斷陣列有沒有重複元素。

**暴力 O(n²) 解：**

```text
def has_duplicate_slow(arr):
    for i in range(len(arr)):
        for j in range(i + 1, len(arr)):
            if arr[i] == arr[j]:
                return True
    return False

→ 10 萬筆：100 億次比較、跑 10 分鐘
```

**用 set 的 O(n) 解：**

```text
def has_duplicate_fast(arr):
    seen = set()
    for x in arr:
        if x in seen:
            return True
        seen.add(x)
    return False

→ 10 萬筆：10 萬次、< 50ms
```

**Python 一行版：**

```text
def has_duplicate(arr):
    return len(arr) != len(set(arr))

→ 同樣 O(n)
```

> 💡 **新手寫 O(n²) 是常態、進階寫 O(n)**。差別在「用對資料結構」。Set / Dict / HashMap 是寫快程式碼最大的祕密武器。

  
### 8. 最壞 / 平均 / 最好

```text
Quick sort：
- 最好：O(n log n) — 每次切剛好對半
- 平均：O(n log n) — 大部分情況
- 最壞：O(n²) — 已排序的輸入

Hash table 查找：
- 平均：O(1)
- 最壞：O(n) — 所有元素都 hash 衝突（極罕見）

面試一般問「平均」、有時問「最壞」。
```

  
### 9. 5 大常見坑

```text
❌ 1. 把常數寫進 Big O
   → O(2n) ❌ 應該 O(n) ✅
   → 修：忽略常數係數

❌ 2. 沒看主導項
   → O(n² + n) ❌ 應該 O(n²) ✅
   → 修：n 很大時、低階項不重要

❌ 3. nested loop 沒乘
   → 兩層 for n、寫 O(2n) ❌ 應該 O(n²) ✅
   → 修：nested 是相乘不是相加

❌ 4. 把 String concat 當 O(1)
   → Python / JS string immutable、concat n 次 = O(n²)
   → 修：用 list.append + join，或 StringBuilder

❌ 5. 忽略空間複雜度
   → 寫了個 O(n) 時間但 O(n²) 空間的 function 也炸
   → 修：時間 + 空間都要考慮
```

  
### 🎯 重點回顧

- **Big O = n 變大時操作數的成長趨勢**
- **7 大複雜度從快到慢**：O(1) → log n → n → n log n → n² → 2^n → n!
- **3 大規則**：常數忽略、取主導項、nested 相乘
- **用對資料結構**（set / dict / hashmap）= O(n²) 變 O(n)
- **時間 + 空間都要算**、面試常問
