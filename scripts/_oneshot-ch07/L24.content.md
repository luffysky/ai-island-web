實戰用到的演算法不多——會這 6 大類就 **80% 場景搞定**：排序、搜尋、遞迴、動態規劃、貪心、圖（BFS/DFS）。

> 💡 **這課的價值**：面試 80% 題目就在這 6 類。實務上會這幾招、解決問題能力大幅提升。**不必背、理解 pattern 即可**。

  
### 1. 排序（Sort）

**所有語言都有內建排序——別自己寫**：

```text
🐍 Python
sorted_nums = sorted(nums)                          # 升冪
sorted_nums = sorted(nums, reverse=True)            # 降冪
sorted_users = sorted(users, key=lambda u: u.age)   # 依屬性
nums.sort()                                          # in-place

📦 JS
nums.sort((a, b) => a - b);                         # 升冪
nums.sort((a, b) => b - a);                         # 降冪
users.sort((a, b) => a.age - b.age);

☕ Java
Collections.sort(list);
list.sort(Comparator.comparingInt(u -> u.age));
```

**面試需要知道的 5 種排序算法（時間 / 空間）：**

```text
🐢 Bubble Sort（氣泡）    — O(n²) / O(1)
   兩兩比較換位、最慢但最簡單
   → 永遠不用、面試題

🐢 Selection Sort（選擇）  — O(n²) / O(1)
   每輪找最小放最前面
   → 同上

🐢 Insertion Sort（插入）  — O(n²) / O(1)
   像整理撲克牌、小資料快
   → 小陣列（< 10）快、JS sort 內部用

🚀 Merge Sort（合併）      — O(n log n) / O(n)
   分兩半 → 各自排好 → 合併
   → 穩定排序、Java sort 用

🚀 Quick Sort（快速）      — 平均 O(n log n) / O(log n)
   選 pivot → 比 pivot 小放左、大放右 → 遞迴
   → 平均最快、JS / C++ sort 用

🚀 Heap Sort（堆積）       — O(n log n) / O(1)
   建 heap → 一個個拿
   → 不穩定、實務少用
```

**Quick Sort 範例（手寫 + 看懂分而治之）：**

```text
def quick_sort(arr):
    if len(arr) <= 1:
        return arr
    pivot = arr[len(arr) // 2]
    left = [x for x in arr if x < pivot]
    mid = [x for x in arr if x == pivot]
    right = [x for x in arr if x > pivot]
    return quick_sort(left) + mid + quick_sort(right)
```

  
### 2. 搜尋（Search）

**Linear Search（線性搜尋）— O(n)**：

```text
def find(arr, target):
    for i, x in enumerate(arr):
        if x == target:
            return i
    return -1
```

**Binary Search（二元搜尋）— O(log n)、陣列必須先排序**：

```text
def binary_search(arr, target):
    left, right = 0, len(arr) - 1
    while left <= right:
        mid = (left + right) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    return -1
```

**Python / JS 內建：**

```text
# Python
from bisect import bisect_left
idx = bisect_left(sorted_arr, target)

# JS — 沒內建、自己寫或用 lodash sortedIndexOf
```

**何時用 binary**：
- 已排序的大陣列查找
- 找「第一個 >= X」「最後一個 <= X」這類邊界

  
### 3. 遞迴（Recursion）

**遞迴 = 函式呼叫自己、處理「自相似結構」**：

```text
經典：費氏數列（Fibonacci）

def fib(n):
    if n <= 1:
        return n
    return fib(n - 1) + fib(n - 2)

→ fib(40) 已經要算 30 秒
→ 因為 O(2^n) 爆炸（超多重複計算）
```

**遞迴三要素：**

```text
1. Base case（終止條件）— if n <= 1: return n
2. Recursive case（遞迴呼叫）— fib(n-1) + fib(n-2)
3. 朝 base case 收斂 — 每次 n 變小、最後到 0/1
```

**經典遞迴題：**

```text
🌲 樹的走訪
def traverse(node):
    if node is None: return
    print(node.value)
    traverse(node.left)
    traverse(node.right)

📁 資料夾遞迴
def list_files(folder):
    for item in os.listdir(folder):
        if os.path.isdir(item):
            list_files(item)  # 遞迴進子資料夾
        else:
            print(item)

🔢 階乘
def factorial(n):
    if n <= 1: return 1
    return n * factorial(n - 1)
```

> ⚠️ **遞迴陷阱**：(1) 沒 base case = 無窮遞迴 → stack overflow (2) 重複計算（fib）= 指數爆炸 → 改 DP

  
### 4. 動態規劃（Dynamic Programming、DP）

**核心**：「遞迴 + 記錄已算的答案、不重複算」。

**改良 fib：**

```text
# 方法 1：memoization（記憶化）
memo = {}
def fib(n):
    if n in memo: return memo[n]
    if n <= 1: return n
    memo[n] = fib(n - 1) + fib(n - 2)
    return memo[n]

→ O(n)、fib(40) < 1ms

# 方法 2：tabulation（表格法、從小算到大）
def fib_table(n):
    if n <= 1: return n
    dp = [0] * (n + 1)
    dp[1] = 1
    for i in range(2, n + 1):
        dp[i] = dp[i - 1] + dp[i - 2]
    return dp[n]
```

**經典 DP 題型：**

```text
🪜 climbing stairs（爬樓梯）
n 階樓梯、每次跨 1 或 2 階、有幾種走法？
→ dp[n] = dp[n-1] + dp[n-2]（fib 變種）

💰 coin change（找零錢）
有面額 [1, 5, 10, 25]、湊 X 元最少幾個硬幣？
→ dp[i] = min(dp[i - coin] + 1 for coin in coins)

📦 knapsack（背包問題）
n 個物品、各有重量 + 價值、背包容量 W、價值最大？
→ dp[i][w] = max(dp[i-1][w], dp[i-1][w-wt[i]] + val[i])

🔤 longest common subsequence（最長共同子序列）
diff / 拼字檢查 / DNA 比對
```

**DP 識別 signal：**

```text
✅ 「最大 / 最小」「幾種方法」「能不能達成」
✅ 子問題重複出現
✅ 暴力遞迴會 TLE
→ 考慮 DP
```

  
### 5. 貪心（Greedy）

**每一步都「當下看起來最好」、希望總和最佳**。

```text
經典：找零錢（單純面額）

面額 [25, 10, 5, 1]、湊 67 元最少幾個？

貪心：
- 67 / 25 = 2 個 25（剩 17）
- 17 / 10 = 1 個 10（剩 7）
- 7 / 5 = 1 個 5（剩 2）
- 2 / 1 = 2 個 1
- 總 6 個

→ 貪心對「美金面額」剛好對
→ 但 [1, 3, 4] 湊 6 用貪心拿 4+1+1=3 個、最佳是 3+3=2 個
→ 不是所有問題都能用貪心
```

**何時可用貪心**：
- 排程問題（活動選擇）
- 區間覆蓋
- Huffman 編碼
- 部分背包問題（可切塊）

  
### 6. 圖（Graph）BFS / DFS

**BFS（廣度優先搜尋、Breadth-First Search）— 用 queue**：

```text
from collections import deque

def bfs(graph, start):
    visited = set([start])
    queue = deque([start])
    while queue:
        node = queue.popleft()
        print(node)
        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)
```

**DFS（深度優先搜尋、Depth-First Search）— 用 stack / 遞迴**：

```text
def dfs(graph, start, visited=None):
    if visited is None: visited = set()
    visited.add(start)
    print(start)
    for neighbor in graph[start]:
        if neighbor not in visited:
            dfs(graph, neighbor, visited)
```

**何時用哪個：**

```text
🌊 BFS：
   - 找「最短路徑」（無權重圖）
   - 一層一層擴展
   - level order traversal

🤿 DFS：
   - 走遍所有節點
   - 找「有沒有路徑」
   - 拓樸排序
   - 偵測 cycle

社群網路六度分隔、迷宮最短路、檔案搜尋……全是這兩招。
```

```text

```

  
### 8. 5 大常見坑

```text
❌ 1. 為了學「演算法」自己手寫排序
   → 內建一定比你寫的快
   → 修：理解原理、實務用內建

❌ 2. 遞迴沒 base case
   → 無窮遞迴 → stack overflow
   → 修：永遠先寫 base case

❌ 3. 重複計算（純遞迴解 DP 題）
   → fib(40) 30 秒、不可接受
   → 修：加 memo / 改 tabulation

❌ 4. 把貪心當萬靈丹
   → 「找零錢 [1,3,4] 湊 6」貪心錯
   → 修：先確認「貪心性質」成立才用

❌ 5. BFS 用 list.pop(0)
   → Python list.pop(0) 是 O(n)、會慢
   → 修：用 collections.deque（O(1) popleft）
```
