## 0. 先講結論（誠實版）

> 這個平台的功能完整度已經很高——學習引擎、AI 導師、實作環境、社群、後台、變現、老師端、cron 自動化幾乎都有了。**現在最大的槓桿不是「再加功能」，而是「把已經做了一半的閉環收尾、把能收錢的那幾塊接通」。**
>
> 董，這份報告我刻意只挑「會推動營收或留存、且能複用你現有基建」的，不是把能想到的全倒給你。**如果只做三件，我會選 #B1（自動評測 + AI code review）、#B2（可驗證證書 + 課程市集收尾）、#C1（學習社群 / cohort）**——理由在第 5 段。其餘當作彈藥庫，做完這三個再回來挑。

## 2. B 類 — 深化學習閉環（最該先做、複用現有基建）

### ⭐ B1：自動評測 + 綠寶 AI Code Review（閉合實作環）
**現況缺口：** 你有 Playground + Pyodide + LeetCode，但程式**寫完之後沒有自動判對錯、也沒有 AI 點評**。學習閉環缺最後一哩。
**做什麼：**
- 每題附隱藏 **test cases**，提交後自動跑（Python 用既有 Pyodide、JS 用 sandbox）→ 顯示 pass/fail
- 通過 → 自動發 XP / Z-coin / 成就（接既有 gamification）
- 不論過不過，一鍵叫**綠寶 review 我的程式碼**（接既有 AI model pool）：給可讀性 / 效能 / 慣用法建議
**複用：** Pyodide、playgrounds、AI model pool、gamification、submissions 表
**價值：** 把「看 → 寫 → 對答案 → 被點評」閉環補滿，是學習型平台留存的核心
**工程量：** 中（test runner + UI + AI prompt）

### B2：間隔複習（SRS）排程
**現況：** 已有 `cron/student-daily-review` 的雛形 + LINE 每日推播——基建在了。
**做什麼：** 完成的 lesson / 答錯的 quiz 進入 SM-2 式複習排程，到期透過 LINE / 站內推「今天該複習這 3 題」。
**複用：** 既有 daily-review cron、LINE bot、quiz_attempts
**價值：** 直接拉高 DAU 與長期記憶留存（Duolingo 的核心引擎）
**工程量：** 小～中（加 `next_review` 欄位 + 排程邏輯）

### B3：實戰 Capstone / Build-in-Public 專案軌
**現況：** 你的 SnowRealm 生態（Insight / GLACÉRA / 毛行 / OrderEase）本身就是**活教材**——這是你記憶裡寫的「核心護城河」，但平台上還沒把它變成功能。
**做什麼：** 開「跟著做一個真產品」軌：學員照章節做出可部署的小產品，用既有 portfolio 展示、社群投票、Z-coin 打賞。優秀作品收進案例庫。
**複用：** portfolio、forum、Z-coin、courses
**價值：** 把「課程內容」升級成「作品成果」，是對外行銷與口碑的最強素材
**工程量：** 中（多為內容 + portfolio 串接）

## 3. B 類 — 變現缺口（這是課程平台，該優先接通）

### ⭐ B4：可驗證證書（LinkedIn 可分享）+ 課程市集收尾
**現況：** 有證書、有 courses/[slug]、有定價策略文件，但「可對外證明的憑證」與「完整市集體驗」還沒收尾。
**做什麼：**
- 證書給**唯一驗證網址 + QR**（`/verify/[certId]`），可一鍵分享到 LinkedIn / 設成 OG 卡
- 課程市集補：試看、購物車、bundle（你已規劃 Creator/Freelancer/Entrepreneur 三包）、優惠碼、購買後解鎖
**複用：** certificates、courses、orders、Newebpay/Stripe、OG 圖 route
**價值：** 證書可分享 = 免費招生漏斗；市集收尾 = 直接收錢
**工程量：** 中

### B5：接案 / 職缺媒合板（綁你的接案課）
**現況：** 有履歷 + 作品集 + 接案課規劃，但沒有「把學員推出去接案 / 找工作」的出口。
**做什麼：** 台灣 indie / 新創 friendly 的輕量職缺 / 接案需求板，學員用站內履歷一鍵應徵；雇主端可付費刊登或抽成。
**複用：** resume、portfolio、forum 基礎
**價值：** 形成「學 → 證 → 接案/就業」完整價值鏈，是高黏著、可抽成的差異化功能
**工程量：** 中～大（建議晚一點，先驗證需求）

### B6：1:1 導師預約市集
**現況：** 有 mentor 頁、Nami / 雪夫人是共同講師。
**做什麼：** 付費預約 1:1（程式 review / 職涯諮詢 / 作品集健檢），Z-coin 或金流付費、行事曆排程。
**複用：** mentor、Z-coin、orders、既有日曆需求
**價值：** 高客單價、把講師時間商品化
**工程量：** 中

## 4. C 類 — 社群 / 留存

### ⭐ C1：學習社群 / Cohort + 進度夥伴
**現況：** 有論壇、排行榜、Discord 整合，但缺「一起學」的結構。
**做什麼：** 期數制 cohort（同梯一起開課）、學習小組、進度夥伴配對、組隊 streak、組隊 Boss Battle。
**複用：** forum、leaderboard、gamification、Discord
**價值：** 同儕壓力是線上學習完課率的最強解；也餵養你規劃的 Discord 社群
**工程量：** 中

### C2：賽季制排行榜 + 衝刺活動
**做什麼：** 排行榜分賽季（月/季）、限時學習衝刺活動、賽季限定成就 / Z-coin 商城 DLC（你之前那套角色/服裝 DLC 思路可複用）。
**複用：** leaderboard、achievements、Z-coin
**工程量：** 小～中

### C3：綠寶語音導師 + 語音 mock interview
**現況：** mock-interview 目前純文字、有 LINE bot。
**做什麼：** Web Speech / TTS 讓綠寶能「講」、mock interview 可語音對答（更逼真）；LINE 語音訊息問答。
**複用：** AI model pool、mock-interview、LINE
**價值：** 體驗升級 + 無障礙，面試練習更真實
**工程量：** 中

## 5. D 類 — 生態系整合（你獨有的護城河）

### D1：Z-coin 跨產品錢包
**做什麼：** AI 島賺的 Z-coin 能在 Insight / 毛行天下 / GLACÉRA 通用消費（你記憶裡的「多宇宙經濟層」終於串起來）。
**複用：** 既有 Z-coin / coin_transactions、SnowRealm-Core 構想
**價值：** 把分散的產品綁成一個生態、提高整體黏著與切換成本
**工程量：** 大（跨產品、要先有中央錢包服務，建議列長線）

### D2：綠寶每週個人化複習報告
**做什麼：** 接 `summarize-memories` cron 思路，每週 AI 產一份「你這週學了什麼、卡在哪、下週建議」推到 LINE / email。
**複用：** AI、analytics、LINE、email
**工程量：** 小～中（你基建幾乎都有）

### D3：LINE 學習小程式深化
**現況：** LINE bot 已能對話。
**做什麼：** 把每日一題、簽到、複習、進度查詢做成 LINE Rich Menu 小程式級體驗（台灣市場的關鍵入口，你一直強調的差異化）。
**複用：** LINE bot、daily cron
**工程量：** 中

## 6. 我的誠實建議（排序 + 提醒）

**先做這三個（會直接推營收 / 留存，且複用現有基建最多）：**
1. **B1 自動評測 + AI code review** — 補學習閉環的最後一哩，留存核心
2. **B4 可驗證證書 + 課程市集收尾** — 直接收錢 + 免費招生漏斗
3. **C1 學習社群 / cohort** — 拉完課率、餵 Discord 社群

**做完上面再回來挑：** B2 複習排程、B3 capstone、D2 每週報告（這幾個工程量小、CP 值高，是好的第二批）。

**列為長線、先別動：** B5 職缺板、D1 跨產品錢包（大工程、需求要先驗證，現在做會分散火力）。

> 董，最後一句真心話：這平台的問題從來不是「功能不夠」，而是功能已經很多、戰線很長。**這份清單請當「收尾與變現指南」，不是「再開新坑指南」。** 每挑一個做之前，先問「這會幫我收到錢或留住人嗎？」——會，才動工。
