## 1. 它在做什麼（一段話）

這是一個給 owner / admin 用的**靈感重組引擎**。核心價值**不是「生點子」，而是「看見別人看不到的連結」**——
你平常把零碎的想法、回憶、看到的網頁、半成形的概念丟成一張張「碎片」，
按一下「✨ 給我一個點子」，AI 會在這些碎片之間找出**深層的、非顯而易見的關聯**，
重組成新的產品 / 故事 / 行銷 / 功能 / 品牌 / 歌曲 / 課程點子，並告訴你「**為什麼這些碎片值得被組合**」。

四層演進都已落地：
1. **基礎**：碎片 CRUD + 標籤 + AI 重組
2. **整理**：資料夾（拖放分類）+ 四種檢視（列表 / 時間軸 / 標籤雲 / 關聯圖）
3. **協作 + 自動**：多人碎片（誰丟的）、每日自動推薦點子
4. **語意引擎**：向量 embedding + 「意外配對」（表面遠、深層共振的碎片組合）+ 回饋學習

## 3. 左欄：碎片（素材）

### 新增碎片
- 欄位：**標題**（必填）、**內容**（可空、之後補）、**標籤**（逗號 / 、/ ，分隔）、**資料夾**。
- **貼網址自動抓**：標題框貼一個 URL，會冒出「從網址抓標題 / 摘要」按鈕 → 打 `fetch-url` 把網頁標題/描述塞進內容、自動加「網摘」標籤。
- 新碎片預設丟進「目前篩選中的資料夾」。

### 整理工具
- **🔍 搜尋**：標題 / 內容 / AI 摘要全文比對。
- **#標籤篩選**：點標籤過濾，再點取消。
- **📁 資料夾**：新增 / 改名（雙擊）/ 刪除（裡面碎片不刪、變未分類）。**把碎片卡拖到資料夾 chip 即可分類**（dnd-kit，樂觀更新）。
- **☑ 手選模式**：勾選 ≥2 個碎片 → 主按鈕變「組合這 N 個」，AI 只用這幾個。

### 四種檢視
| 檢視 | 看什麼 |
|---|---|
| 列表 | 預設，完整卡片 + 操作 |
| 時間軸 | 依時間排碎片，看靈感的時序脈絡 |
| 標籤雲 | 標籤大小=出現頻率，點擊回列表並篩選 |
| 關聯圖 | 碎片節點 + **語意意外配對連線**（吃 pairs） |

### 單張碎片卡
- 顯示：標題、內容、資料夾 / 分類 / 情緒 / 「🙋 誰丟的」徽章、#標籤。
- **AI 分析**：按「分析碎片」→ AI 產出 `ai_summary`（一句洞察）+ `potential_uses`（可能用途），顯示在卡內。
- 編輯（modal：標題/內容/分類/情緒/標籤/資料夾）、刪除。

## 4. 主行動：✨ 給我一個點子

按鈕的素材來源**優先序**：
1. 手選的碎片（≥2 個）→ 只組這幾個
2. 否則目前選中的資料夾 → 用該資料夾的碎片
3. 否則 → 最近全部碎片

AI（`generateIdeaRows`）會回傳數個點子，每個含：

- **title** — 有記憶點、不平庸的名稱
- **summary** — 2~3 句講清楚
- **ideaType** — 產品 / 故事 / 行銷 / 功能 / 品牌 / 歌曲 / 課程 擇一
- **connections** — 🔗 **這功能的靈魂**：具體指出「碎片 A 的某元素」和「碎片 B 的某元素」之間那條線（共享的情緒 / 主題 / 結構 / 對比 / 因果）。Prompt 明令禁止「都跟創作有關」這種空泛廢話。
- **whyItWorks** — 為什麼這個組合值得做、打中什麼
- **nextSteps** — 2~4 個具體下一步
- **sourceFragmentIds** — 用到的碎片（會驗證 id 真的存在）

> AI 用 `temperature 0.85`、`maxTokens 2500`，回傳純 JSON（`extractJson` 容忍 ```json fence / 前後雜訊）。

## 5. 語意引擎：🔗 意外配對

這是讓點子「驚喜」的關鍵，分兩步：

1. **建立語意向量**：`embed-backfill` 幫每個碎片算 OpenAI `text-embedding-3-small`（1536 維）存進 `idea_fragments.embedding`（pgvector）。
2. **找意外配對**：RPC `idea_surprising_pairs` 找**相似度落在「中間帶」（0.28~0.55）**的碎片對——
   - 太高（>0.55）= 根本在講同件事，無聊
   - 太低（<0.28）= 八竿子打不著，硬湊
   - **中間帶 = 表面遠、深層卻有張力 → 最容易擦出火花**

意外配對顯示成卡片，可「用這對」直接生點子，或「重新探索」（會先補齊缺向量再重算）。
生成時這些配對會以 `pairsHint` 餵進 prompt，要 AI **優先**嘗試把它們接起來。

> 沒有 OpenAI key 時，embedding 靜默跳過、意外配對區塊顯示「建立語意向量」引導，其餘功能不受影響。

## 6. 右欄：點子的下游（讓點子「動起來」）

每個生成的點子可以：

| 動作 | 結果 |
|---|---|
| 🔖 **儲存** | 標記 saved、卡片高亮、之後好找 |
| 👍 / 👎 **評分** | **回饋迴路**：AI 之後生成會參考你喜歡過的風格（`likedStyle`） |
| 📋 **轉成任務** | 寫進「待辦」 |
| 📄 **轉成文章草稿** | 建立部落格草稿 |
| 🚀 **轉成產品企劃** | AI 展開成完整企劃、存成部落格草稿（標題【產品企劃】…） |
| 💬 **深化 / 追問** | 開對話 modal，把這個點子當對象一直問（商業模式 / MVP / 風險 / 取名…），多輪對話 |
| 🗑️ **丟掉** | 刪除 |

## 7. 🌅 今日點子（每日自動推薦）

- 每天（台北時區換日）自動產生一個當日點子，顯示在最上方 `DailyIdeaCard`。
- 和手動「給我一個點子」**共用同一套** prompt / 解析（`generateIdeaRows`），只是觸發方式不同。
- 對應 `generated_ideas.daily_date` 欄位，一天一筆。

## 8. 後端 / 資料結構

### API 路由（全在 `/api/admin/` 下、admin/owner 守門）
| 路由 | 作用 |
|---|---|
| `idea-fragments` (GET/POST) | 列出 / 新增碎片 |
| `idea-fragments/[id]` (PATCH/DELETE) | 改 / 刪碎片（含搬資料夾） |
| `idea-fragments/[id]/analyze` (POST) | AI 分析單一碎片 |
| `idea-fragments/generate` (POST) | **重組生點子**（吃 fragmentIds / folderId / count） |
| `idea-fragments/daily` (POST) | 產生 / 取今日點子 |
| `idea-fragments/fetch-url` (POST) | 貼網址抓標題摘要 |
| `idea-fragments/embed-backfill` (POST) | 補算缺的語意向量 |
| `idea-fragments/pairs` (GET) | 取意外配對 |
| `idea-folders` (POST) + `[id]` (PATCH/DELETE) | 資料夾 CRUD |
| `generated-ideas/[id]` (PATCH/DELETE) | 儲存 / 評分 / 刪點子 |
| `generated-ideas/[id]/convert` (POST) | 轉任務 / 文章 / 產品企劃 |
| `generated-ideas/[id]/deepen` (POST) | 深化追問（多輪） |

### 資料表（migration 都在 `supabase/`）
- **`idea_fragments`** — 碎片：`title / content / tags[] / mood / category / ai_summary / potential_uses[] / folder_id / created_by / embedding(vector 1536)`。RLS 只放行 admin/owner、有 `updated_at` trigger、tags GIN index、embedding ivfflat index。
- **`generated_ideas`** — 點子：`title / summary / idea_type / source_fragment_ids[] / why_it_works / next_steps[] / connections[] / saved / feedback / daily_date / created_by`。
- **`idea_folders`** — 資料夾：`name / color / sort_order`。
- RPC **`idea_surprising_pairs(match_count, min_sim, max_sim, folder)`** — 中間相似帶配對。

### AI 模型解析（`resolveIdeaModel`）
從 `ai_models`（`is_active`）挑：**Anthropic 優先 → OpenAI → 任一**，再從 `ai_api_keys` 解密該 provider 的 key。沒啟用模型 / 沒 key → 回明確錯誤導去 `/admin/ai/models`。

## 9. 權限 / 安全
- 頁面：未登入 → `/login`；非 `admin` 且非 `is_owner` → 導回首頁。
- 所有表 RLS 開啟、policy 僅 admin/owner；後端走 service-role admin client。
- API key 一律加密存庫（`ai-crypto`），解密只在 server。

## 10. 設計亮點（為什麼這功能特別）
1. **賣點是「連結」不是「生成」** — connections 欄位 + prompt 鐵則，逼 AI 講出碎片「之間」的關係，而非各自摘要。
2. **意外配對用相似度中間帶** — 不是找最像的（無聊）也不是隨機（無關），專挑「驚喜帶」。
3. **完整下游** — 點子不會停在畫面上，能一鍵變任務 / 文章 / 企劃，真正接上生產流程。
4. **會學你的品味** — 👍👎 回饋餵回下次生成。
5. **協作 + 每日習慣** — 多人碎片署名、每日自動推一個點子，養成「持續收集 → 持續被啟發」的循環。
